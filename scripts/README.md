# Scripts Notes

Repo-local wrappers and diagnostics live here. Keep unrelated stack helpers out of this folder.

- `test-suite.sh`: standardized test runners for common agent loops.
  - `./scripts/test-suite.sh smoke`
  - `./scripts/test-suite.sh fast`
  - `./scripts/test-suite.sh domain <domain>`
  - `./scripts/test-suite.sh all-fast`
  - `./scripts/test-suite.sh full`
  - `make test-smoke`, `make test-fast`, `make test-domain DOMAIN=<domain>`, `make test-all-fast`, and `make test-full` forward to the same wrapper.
  - extra pytest args can be appended, for example `./scripts/test-suite.sh domain parsing --collect-only`
  - For agent workflows, prefer these `test-suite.sh` modes over raw `pytest` to avoid monolithic long-running runs.

- `interactive-with-codex-farm.sh`: launches interactive `cookimport` with Codex Farm env gate enabled for that process (`scripts/interactive-with-codex-farm.sh`).
- `with-codex-farm.sh`: one-shot opt-in wrapper that runs any command with `COOKIMPORT_ALLOW_CODEX_FARM=1` for that process only (`scripts/with-codex-farm.sh cookimport ...`).
- `markitdown_epub_smoke.py`: quick local smoke test for EPUB -> markdown conversion using MarkItDown (`python scripts/markitdown_epub_smoke.py <file.epub>`).
- `bench_sequence_matcher_impl.py`: quick local matcher timing/parity check for canonical eval across selectable modes (`python scripts/bench_sequence_matcher_impl.py --tokens 1400 --repeats 5 --modes stdlib,auto,dmp`).
- `benchmark_cutdown_for_external_ai.py`: builds a benchmark package for external AI review (per-run summaries + sampled diagnostics + codex-vs-baseline comparison), then optionally flattens it to markdown (`python scripts/benchmark_cutdown_for_external_ai.py <benchmark_folder> --output-dir <cutdown_folder> --overwrite`). Cutdown now prioritizes causality artifacts: root-level `changed_lines.codex_vs_vanilla.jsonl`, `per_recipe_or_per_span_breakdown.json`, `targeted_prompt_cases.md`, and `label_policy_adjudication_notes.md`; run-level codex outputs include `prompt_warning_aggregate.json` and `projection_trace.codex_to_benchmark.json` when full prompt logs are available. Root README now includes `## Project Context Digest` (benchmark contract + label ontology cheat-sheet + projection bridge + artifact legend), and root manifests include context pointer/hash metadata (`project_context_path`, `project_context_title`, `project_context_version_or_date`, `project_context_hash`; plus `project_context_digest_included` on `process_manifest.json`) with explicit `missing` fallback values if `docs/AI_Context.md` is unavailable. Line-level wrong/missed samples remain, while duplicate block-level error samples are removed, `unmatched_pred_blocks` is counts-only by default, and heavy alignment samples are emitted only for weak alignment quality. New per-run compressed diagnostics include `wrong_label_lines.with_context.full.jsonl.gz` (unsampled wrong-line context) and `preprocess_trace_failures.jsonl.gz` (failure traces joined to prediction-run/archive and prompt context when available; fallback reason is recorded in `need_to_know_summary.json` `sample_counts` when unavailable). `full_prompt_log.jsonl` is still preserved untruncated and `codexfarm_prompt_log.dedup.txt` remains a convenience sampled view. Starter-pack triage now also lifts per-recipe `llm_manifest.json` diagnostics (`pass1/2/3` statuses, clamped span-loss counters, pass2 degradation reasons, pass3 fallback reason, transport mismatch, evidence-normalization stats) into `01_recipe_triage.csv`, `04_warning_and_trace_summary.json`, and selected recipe packets/casebook. Without `--output-dir`, default naming is timestamp-driven from discovered run IDs (`<run_id>_cutdown` for one run, `<first>__to__<last>_cutdown` for multi-run). Set `--prompt-pairs-per-category 0` to keep all calls in the convenience file.
  - Reconstructed prompt logs from existing benchmark outputs now include pass4 knowledge rows when only raw `prediction-run/raw/llm/*/pass4_knowledge/{in,out}` artifacts are available.
  - Upload-consolidation mode: pass `--no-flatten --upload-3-files` to emit `upload_bundle_overview.md`, `upload_bundle_index.json`, and `upload_bundle_payload.jsonl` alongside the regular package; add `--upload-3-files-only` to prune output down to just those three non-CSV files while preserving full source artifact content inside the payload bundle.
  - `upload_bundle_index.json` now includes verified topline/self-check fields plus default triage views (per-label/per-recipe summaries, stage-separated comparison, failure ledger, stratified changed-line samples, compact regression casebook, call runtime/tokens/cost rollups, line-role confidence signals), with row locators into payload and alias metadata for duplicate-equivalent artifacts.
  - Upload-bundle index/overview now surfaces pass4 knowledge harvest explicitly via `analysis.pass4_knowledge` plus `navigation.row_locators.pass4_by_run`; high-level group bundles keep lightweight pass4 evidence (`prompt_type_samples_from_full_prompt_log.md`, `pass4_knowledge_manifest.json`) while leaving raw `prompt_task4_pass4_knowledge.txt` as follow-up-only heavy context.
  - Existing-output upload-bundle generation now derives codex diagnostic statuses directly from run artifacts when per-run `need_to_know_summary.json` is missing, and writes those derived artifacts into payload rows under `_upload_bundle_derived/runs/<run_id>/...`.
  - Row-locator resolution now has basename fallbacks (for example `codex_vs_vanilla_comparison.json` for comparison-summary lookups), and when root/starter files are absent it emits `_upload_bundle_derived/...` payload rows so locator coverage remains useful without changing the 3-file contract.
  - Call-runtime summaries include explicit observed-cost `cost_signal` plus token-based `estimated_cost_signal` fallback (using default pricing) so first-pass triage still gets a cost-ordering signal when telemetry omits billing cost.
  - Benchmark pair inventory includes `generalization_readiness` details (`minimum_pairs_for_generalization`, `additional_pairs_needed_for_generalization`) for quick triage of single-pair overfitting risk.
  - Line-role candidate-label reporting accepts multiple producer field shapes (`candidate_labels`, `label_candidates`, `candidates`, `label_scores`) for forward compatibility.
  - The script now also exposes `build_starter_pack_for_existing_runs(...)` so CLI single-offline paired runs can generate `starter_pack_v1/` directly in an existing session folder; pass `write_flattened_summary=True` to also write in-place `benchmark_summary.md`.
  - `build_upload_bundle_for_existing_output(...)` builds the same 3-file upload bundle from an existing artifact folder into a chosen output directory (used by CLI benchmark defaults). Pass `high_level_only=True` and `target_bundle_size_bytes=...` for a group-level bundle that keeps per-book highlights within a fixed size budget (default target: ~40MB).
- `replay_seaandsmoke_codex_transport.py`: fixture-backed replay for Pro3 transport and outside-span trace policies (`python scripts/replay_seaandsmoke_codex_transport.py --all`). It reads `tests/fixtures/codex_transport_cases.json` and `tests/fixtures/codex_outside_span_bridge.json`, prints per-case transport parity lines, and exits non-zero on policy regressions.
