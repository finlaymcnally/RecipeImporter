# Scripts Notes

Repo-local wrappers and diagnostics live here. Keep unrelated stack helpers out of this folder.

Change map:
- If you are changing `test-suite.sh`, read `docs/12-testing/12-testing_README.md` first and prove the change with the narrowest wrapper invocation that exercises the contract.
- If you are changing `benchmark_cutdown_for_external_ai.py`, start in the wrapper plus `cookimport/bench/external_ai_cutdown/`, read `cookimport/bench/README.md` and `docs/07-bench/07-bench_README.md`, and run the focused cutdown pytest slice before any broad bench run.

- `test-suite.sh`: standardized test runners for common agent loops.
  - `./scripts/test-suite.sh smoke`
  - `./scripts/test-suite.sh fast`
  - `./scripts/test-suite.sh domain <domain>`
  - `./scripts/test-suite.sh all-fast`
  - `./scripts/test-suite.sh full`
  - `make test-smoke`, `make test-fast`, `make test-domain DOMAIN=<domain>`, `make test-all-fast`, and `make test-full` forward to the same wrapper.
  - extra pytest args can be appended, for example `./scripts/test-suite.sh domain parsing --collect-only`
  - For agent workflows, prefer these `test-suite.sh` modes over raw `pytest` to avoid monolithic long-running runs.

- `fake-codex-farm.py`: zero-token stand-in for `codex-farm`; point `--codex-farm-cmd` at this script to exercise real RecipeImport shard/runtime folder handoffs without live model calls.
- `migrate_pulled_labelstudio_gold_to_source_rows.py`: batch-migrates old pulled Label Studio exports into row-authoritative artifacts in place (`python scripts/migrate_pulled_labelstudio_gold_to_source_rows.py`). It prefers the saved `sent-to-labelstudio/.../extracted_archive.json` for each gold set and falls back to deterministic source conversion only when no saved archive exists; it writes `source_rows.jsonl`, `row_gold_labels.jsonl`, `row_gold_ambiguous.jsonl`, `row_gold_conflicts.jsonl`, `row_seed_tasks.jsonl`, and a batch summary under `data/golden/pulled-from-labelstudio/`. The migrated `row_seed_tasks.jsonl` output now uses non-overlapping 120-row focus windows. Add `--prefer-live-row-gold` when the existing replacement project in Label Studio has newer annotation edits than the archived pulled export; the script will back up that live replacement export under `live_row_gold_backups/` and remigrate from those live spans instead of the older archive.
- `upload_row_gold_seed_projects_to_labelstudio.py`: creates fresh Label Studio replacement projects from migrated `row_seed_tasks.jsonl` (`python scripts/upload_row_gold_seed_projects_to_labelstudio.py`). The uploader converts the seeded row results into real Label Studio `annotations`, rewrites seeded result ids into Label Studio-safe CSS-friendly ids, and by default also mirrors those same results into `predictions` plus enables interactive preannotation reveal because some Label Studio builds surface imported overlays more reliably that way. After import it exports the new project back from Label Studio and verifies that per-task annotation label counts still match the seed package; if they do not, it retries once with a clean recreate instead of silently accepting a corrupted project. It writes per-book `exports/row_gold_labelstudio_project.json` plus a batch summary at `data/golden/pulled-from-labelstudio/source_rows_labelstudio_upload_summary.json`.
- `sync_cookbook_tag_catalog.py`: refreshes the checked-in `cookimport/llm/knowledge_tag_catalog.json` runtime copy from the Cookbook seed SQL (`python scripts/sync_cookbook_tag_catalog.py --catalog-sql /home/mcnal/projects/cookbook/supabase/seeds/catalog.sql`). The script is offline and only rewrites the local JSON snapshot.
- `markitdown_epub_smoke.py`: quick local smoke test for EPUB -> markdown conversion using MarkItDown (`python scripts/markitdown_epub_smoke.py <file.epub>`).
- `bench_sequence_matcher_impl.py`: quick local matcher timing/parity check for the benchmark matcher selector across selectable modes (`python scripts/bench_sequence_matcher_impl.py --tokens 1400 --repeats 5 --modes stdlib,auto,dmp`).
- `benchmark_cutdown_for_external_ai.py`: builds a benchmark package for external AI review (per-run summaries + sampled diagnostics + baseline/candidate comparison), then optionally flattens it to markdown (`python scripts/benchmark_cutdown_for_external_ai.py <benchmark_folder> --output-dir <cutdown_folder> --overwrite`). The script is still the public wrapper, but extracted owner code now starts in `cookimport/bench/external_ai_cutdown/` for run discovery/output-root selection, prompt/prediction/processed-output artifact-path resolution, per-run cutdown assembly and existing-run record hydration, root README/flattened-summary generation, prompt-row excerpt/scoring helpers used by pair diagnostics, pair-diagnostics/comparison-summary aggregation, starter-pack recipe selection/casebook/readme helpers plus the heavier starter-pack assembly root, deterministic `full_prompt_log.jsonl` reconstruction from staged `raw/llm` outputs, recipe-span projection and line-prediction views, prompt-response warning/correction interpretation, wrong-label/preprocess compressed packet generation, alignment-health checks plus projection-trace summaries and prompt-row helper views, deterministic JSON/JSONL I/O, clipping/sampling, prompt-log parsing/convenience rendering, project-context metadata/digest rendering, scored-row sampling, and shared row/line projection helpers. The script keeps thin wrappers for the patch-sensitive `PROJECT_CONTEXT_REL_PATH` seam and for shared internal helper names such as `_prompt_category_sort_key`, `_resolve_knowledge_prompt_path`, `_build_run_cutdown`, `_build_run_record_from_existing_run`, `_build_pair_diagnostics`, `_build_comparison_summary`, `_build_warning_and_trace_summary`, `_select_starter_pack_recipe_cases`, `_build_selected_recipe_packets`, `_prompt_case_score`, `_input_excerpt_for_prompt_row`, `_reconstruct_full_prompt_log`, `_build_recipe_spans_from_full_prompt_rows`, `_summarize_prompt_warning_aggregate`, `_build_preprocess_trace_failure_rows`, `_build_projection_trace`, `_write_readme`, `_flatten_output`, `write_flattened_summary_for_existing_runs`, and `_write_starter_pack_v1` while the deeper upload-bundle code is still in the wrapper. Cutdown now prioritizes causality artifacts: root-level `changed_lines.benchmark_comparison.jsonl` (legacy runs may still use `changed_lines.codex_vs_vanilla.jsonl`), `per_recipe_or_per_span_breakdown.json`, `targeted_prompt_cases.md`, and `label_policy_adjudication_notes.md`; run-level codex outputs include `prompt_warning_aggregate.json` and `projection_trace.codex_to_benchmark.json` when full prompt logs are available. Root README now includes `## Project Context Digest` (benchmark contract + label ontology cheat-sheet + projection bridge + artifact legend), and root manifests include context pointer/hash metadata (`project_context_path`, `project_context_title`, `project_context_version_or_date`, `project_context_hash`; plus `project_context_digest_included` on `process_manifest.json`) with explicit `missing` fallback values if `docs/AI_context.md` is unavailable. Line-level wrong/missed samples remain, while duplicate block-level error samples are removed, `unmatched_pred_blocks` is counts-only by default, and heavy alignment samples are emitted only for weak alignment quality. New per-run compressed diagnostics include `wrong_label_lines.with_context.full.jsonl.gz` (unsampled wrong-line context) and `preprocess_trace_failures.jsonl.gz` (failure traces joined to prediction-run/archive and prompt context when available; fallback reason is recorded in `need_to_know_summary.json` `sample_counts` when unavailable). `full_prompt_log.jsonl` is still preserved untruncated and `codex_exec_prompt_log.dedup.txt` remains a convenience sampled view. Starter-pack triage and stage scoring now read the live single-correction artifact set directly: `recipe_manifest.json` semantic statuses, `recipe_correction_audit/*.json`, `recipe_phase_runtime/inputs`, `recipe_phase_runtime/proposals`, and `recipe_build_final/out`, with selected packets and stage-separated comparison keyed by `recipe_build_intermediate`, `recipe_refine`, and `recipe_build_final`. Upload-bundle recipe topology is now reviewer-facing only through semantic `recipe_topology_key` plus ordered `recipe_stages`. Without `--output-dir`, default naming is timestamp-driven from discovered run IDs (`<run_id>_cutdown` for one run, `<first>__to__<last>_cutdown` for multi-run). Set `--prompt-pairs-per-category 0` to keep all calls in the convenience file.
  - Reconstructed prompt logs from existing benchmark outputs now include knowledge-stage rows when only raw `prediction-run/raw/llm/*/nonrecipe_finalize/{in,out}` artifacts are available.
  - Upload-consolidation mode: pass `--no-flatten --upload-3-files` to emit two lane folders, `upload_bundle_v1/quality/` and `upload_bundle_v1/token/`, each with `overview.md`, `index.json`, and `payload.json`; add `--upload-3-files-only` to prune output down to just those lane folders.
  - Lane `index.json` includes verified topline/self-check fields plus default triage views (per-label/per-recipe summaries, stage-separated comparison, failure ledger, stratified changed-line samples, compact regression casebook, call runtime/tokens/cost rollups, line-role escalation signals), with row locators into payload and alias metadata for duplicate-equivalent artifacts.
  - Upload-bundle index/overview now surfaces knowledge extraction explicitly via `analysis.knowledge` plus `navigation.row_locators.knowledge_by_run`; high-level group bundles keep lightweight knowledge evidence (`prompt_type_samples_from_full_prompt_log.md`, `knowledge_manifest.json`) while leaving raw `prompt_nonrecipe_finalize.txt` as follow-up-only heavy context.
  - Existing-output upload-bundle generation now derives codex diagnostic statuses directly from run artifacts when per-run `need_to_know_summary.json` is missing, and writes those derived artifacts into payload rows under `_upload_bundle_derived/runs/<run_id>/...`.
  - Row-locator resolution now has basename fallbacks (for example `benchmark_comparison.json`, with legacy `codex_vs_vanilla_comparison.json` still accepted for comparison-summary lookups), and when root/starter files are absent it emits `_upload_bundle_derived/...` payload rows so locator coverage remains useful without changing the lane-local 3-file contract.
  - Call-runtime summaries include explicit observed-cost `cost_signal` plus token-based `estimated_cost_signal` fallback (using default pricing) so first-pass triage still gets a cost-ordering signal when telemetry omits billing cost.
  - Benchmark pair inventory includes `generalization_readiness` details (`minimum_pairs_for_generalization`, `additional_pairs_needed_for_generalization`) for quick triage of single-pair overfitting risk.
  - Upload-bundle line-role summaries now report reason-based escalation signals under `analysis.line_role_escalation`, and the reviewer packet uses `analysis.explicit_escalation_changed_lines_packet`.
  - The script now also exposes `build_starter_pack_for_existing_runs(...)` so CLI single-book paired runs can generate `starter_pack_v1/` directly in an existing session folder; pass `write_flattened_summary=True` to also write in-place `benchmark_summary.md`.
  - `build_upload_bundle_for_existing_output(...)` builds the same lane-local upload bundle from an existing artifact folder into a chosen output directory (used by CLI benchmark defaults). Pass `high_level_only=True` and `target_bundle_size_bytes=...` for a group-level bundle that keeps per-book highlights within a fixed size budget.
