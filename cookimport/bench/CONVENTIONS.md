# Benchmark Conventions

Durable scoring and orchestration contracts for benchmark flows in `cookimport/bench/` and related CLI paths.

## Benchmark Contract Rule

- `labelstudio-benchmark` is row-native only. `source-rows` is the only supported evaluator mode.
- Interactive benchmark modes (`single_book`, `selected_matched_books`, and `all_matched_books`) run `labelstudio-benchmark` in `source-rows` mode for extractor-agnostic comparison against freeform gold.
- Stage evidence manifests are generated by staging writers at `.bench/<workbook_slug>/semantic_row_predictions.json` and copied into prediction-run roots as `semantic_row_predictions.json` for stable benchmark lookup.
- Pipelined benchmark eval must score the replayed semantic-row bundle built from streamed prediction records, while manifests and interruption recovery still point at the original prediction-run root.
- Benchmark eval outputs now use row/line mismatch artifacts as the only supported diagnostics contract; earlier alias files such as `missed_gold_spans.jsonl` and `false_positive_preds.jsonl` are retired and should not be reintroduced.
- Source-row eval outputs should include `aligned_prediction_blocks.jsonl` so joined-line diagnostics can be audited byte-for-byte across runs.
- Source-row metrics are the benchmark source of truth: `strict_accuracy`, `overall_line_accuracy`, `macro_f1_excluding_other`, and `worst_label_recall`. `eval_report.json` payloads should expose explicit benchmark metrics and not emit alias key duplicates.
- Segmentation boundary metrics use structural label projection `core_structural_v1` (`RECIPE_TITLE`, `INGREDIENT_LINE`, `INSTRUCTION_LINE`, all others mapped to `OTHER`) and report boundary PRF for ingredient/instruction start/end plus recipe-split boundaries; optional `segeval` metrics (`pk`, `windowdiff`, `boundary_similarity`) are computed only when explicitly requested.
- Interactive all-method benchmark must keep processed cookbook outputs under the interactive stage output root (`<settings.output_dir>/<benchmark_timestamp>/all-method-benchmark/<source_slug>/config_*/<prediction_timestamp>/...`, defaulting to `data/output/...`), not under the golden eval root.
- Interactive all-method EPUB sweeps default to extractor variants `unstructured` + `beautifulsoup`; `markdown`/`markitdown` variants are excluded unless explicitly enabled via `COOKIMPORT_ALL_METHOD_INCLUDE_MARKDOWN_EXTRACTORS=1`.
- Interactive all-method all-matched scope should continue after per-source failures, record failed sources in the combined report, and keep successful per-source reports intact.
- All-matched orchestration should use bounded source-level concurrency (`all_method_max_parallel_sources`) so multiple sources can run concurrently, while report ordering remains deterministic by discovery order.
- `labelstudio-eval` freeform span-overlap scoring dedupes overlapping gold spans using `(source_hash, source_file, start_block_index, end_block_index)` keys; benchmark stage-block scoring does not use span-IoU matching.
- Freeform golden recipe count source-of-truth is `exports/summary.json` -> `recipe_counts.recipe_headers`, derived from deduped `RECIPE_TITLE` spans by block range. Benchmark/eval reports should surface this against predicted recipe counts from prediction-run context.
- Non-interactive `labelstudio-benchmark` supports an explicit offline path via `--no-upload`; this mode must skip Label Studio credential resolution and never call upload APIs.
- `labelstudio-benchmark` currently mutates process-global `C3IMP_EPUB_*` runtime env vars and all-method per-source config runs append to a shared history CSV path; all-method source-level orchestration should use a bounded thread dispatcher around existing per-source runners, while per-source config execution stays process-based.
- Interactive/non-suppressed `labelstudio-benchmark` should keep a visible spinner during evaluation so long longer scoring phases do not appear frozen after prediction generation completes.
- All-method parallel benchmark execution should use bounded process concurrency: split-heavy conversion is gate-limited via shared split-slot locking (`all_method_max_split_phase_slots`), while smart mode admission keeps phase-aware prewarm behavior and applies eval-tail cap semantics `max_active_during_eval = configured_inflight + eval_tail_headroom_effective`.
- Smart all-method eval-tail headroom defaults should be CPU-aware per source (`cpu_budget_per_source - configured_inflight`, clamped at `>=0`) when `all_method_max_eval_tail_pipelines` is not explicitly set; explicit overrides are allowed but effective headroom is bounded by per-source CPU budget and remaining variants.
- All-method per-source parallel scheduling supports per-config timeout/retry controls (`all_method_config_timeout_seconds`, `all_method_retry_failed_configs`): timeouts should mark configs failed, restart worker pools safely, and optional retries should rerun only failed configs.
- All-method per-source reports should include a `scheduler` block with heavy-slot capacity/busy/utilization, wing backlog metrics, eval-tail activity rollups (`max_eval_active_observed`), idle-gap seconds, and resolved scheduler control values including `eval_tail_headroom_mode`, `eval_tail_headroom_configured`, `eval_tail_headroom_effective`, and `max_active_during_eval`; multi-source reports should include a matching `scheduler_summary` rollup.
- Scheduler heavy-slot utilization/idle-gap telemetry reflects split-convert slot activity only; evaluation time runs after `post_finished` and can dominate wall time without appearing as heavy-slot busy time.
- Benchmark timing telemetry must be persisted end-to-end: prediction `manifest.json` `timing`, benchmark `eval_report.json` `timing`, and benchmark `run_manifest.json` artifacts `timing`.
- Evaluators should emit `evaluation_telemetry` in `eval_report.json` with subphase timers (`load_gold_seconds`, `load_prediction_seconds`, etc.), resource deltas (CPU and peak RSS when available), and work-unit counts; benchmark timing checkpoints should mirror these as `evaluate_*`, `evaluate_resource_*`, and `evaluate_work_*`.
- `labelstudio-benchmark` supports opt-in slow-eval profiling via env vars:
  - `COOKIMPORT_BENCHMARK_EVAL_PROFILE_MIN_SECONDS` enables cProfile capture when eval time meets/exceeds the threshold.
  - `COOKIMPORT_BENCHMARK_EVAL_PROFILE_TOP_N` controls top-call count written to `eval_profile_top.txt` (default `60`).
  - when captured, artifacts `eval_profile.pstats` and `eval_profile_top.txt` are written into eval run roots and referenced in run manifests.
- Benchmark CSV timing precedence is: explicit benchmark timing payload first, then `processed_report_path` `timing`, then blanks; when timing exists, populate stage runtime columns (`total/parsing/writing/ocr`) and benchmark-specific runtime columns (`benchmark_*_seconds`).
- All-method benchmark reports must include timing rollups (`timing_summary`) at both per-source and multi-source levels, plus per-config `timing` for successful config rows.
- Run-producing flows must emit `run_manifest.json` so source identity (`path` + `source_hash`), effective config, and key artifacts are inspectable without reading code.
- When codex-farm is enabled for stage/pred-run, report + manifest payloads should expose `llmCodexFarm`/`llm_codex_farm` metadata, and deterministic fallback semantics must be explicit (`codex_farm_failure_mode=fail|fallback`).
