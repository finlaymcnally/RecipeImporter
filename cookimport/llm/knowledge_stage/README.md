Private knowledge-stage package. Start with `planning.py` for shard planning and progress accounting, `runtime.py` for stage orchestration and summary artifacts, `workspace_run.py` for the live worker path, `recovery.py` for retries/watchdogs and terminal classification, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is now one happy-path workspace session per worker assignment: the worker edits classification `task.json`, runs the repo-owned `cookimport.llm.knowledge_same_session_handoff` helper, and then either repairs in place, continues straight into grouping in the same session, or finishes final shard outputs directly when everything is `other`. Classification `task.json` now carries immutable top-level `ontology` plus per-unit `candidate_tag_keys`; a kept row must emit `category=knowledge`, `reviewer_category=knowledge`, a short `retrieval_concept`, and non-empty `grounding` to an existing tag or a proposed tag under an existing category. `runtime.py` writes the sidecar `knowledge_tag_proposals.jsonl` plus grounding counts that flow through `promotion_report.json`, `knowledge_manifest.json`, `09_nonrecipe_authority.json`, and `knowledge_stage_summary.json`. `reporting.py` still emits the packet-economics rollups into stage `telemetry.json` so later stage summaries and prompt-budget summaries reuse the same packet-count, same-session transition, repair-churn, and per-owned-row cost numbers. The final reviewer artifacts come from promoted shard outputs rather than worker-edited phase ledgers. `__init__.py` exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
