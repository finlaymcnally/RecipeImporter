Private knowledge-stage package. Start with `planning.py` for shard planning and progress accounting, `runtime.py` for stage orchestration and summary artifacts, `workspace_run.py` for the live leased-packet worker loop, `recovery.py` for retries/watchdogs and terminal classification, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is packet-lease-first: `knowledge/in/*.json` stays one shard per file, `task_manifest.jsonl` still mirrors shard ids for reporting, and worker roots expose `assigned_shards.json` only as context plus one repo-owned active lease through `current_packet.json`, `current_hint.md`, `current_result_path.txt`, and `packet_lease_status.json`. Pass 1 returns per-row `knowledge|other` decisions, Pass 2 returns per-row `group_key` plus `topic_label` only for kept rows, repo code validates structure and coverage, generates structural repair packets for unresolved subsets, and assembles the final shard output itself. `reporting.py` now emits packet-economics rollups into stage `telemetry.json` so later stage summaries and prompt-budget summaries can reuse the same packet-count / repair-churn / per-owned-row cost numbers. Knowledge hints in `hints/<shard_id>.md` stay intentionally compact worker-calibration sidecars rather than second prompts. The final reviewer artifacts come from promoted shard outputs rather than worker-edited phase ledgers. `__init__.py` exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
