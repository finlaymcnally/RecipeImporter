Private knowledge-stage package. Start with `planning.py` for shard planning and progress accounting, `runtime.py` for stage orchestration and summary artifacts, `workspace_run.py` for the live worker path, `structured_session_contract.py` for inline-JSON packet/prompt/answer helpers, `recovery.py` for retries/watchdogs and terminal classification, `recovery_status.py` for task-status/stale-followup/stage-status bookkeeping, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is one happy-path workspace session per worker assignment, and both classification and grouping now use the same direct-batch editing model: open `task.json`, read the full assignment, edit only `/units/*/answer`, run `task-handoff`, and continue in the same session if the helper rewrites the file into repair mode or grouping mode. For these direct-batch stages the worker-facing task file is deliberately slim: it keeps `answer_schema`, stage-specific metadata such as `ontology` when needed, and `units`, but omits worker-facing `helper_commands`, `workflow`, `next_action`, and `editable_json_pointers`. `task-status`, `task-doctor`, and `task-handoff` remain the repo-owned helper surface; ordinary local reads of `task.json` and `AGENTS.md` are allowed, and workflow drift is warning-first rather than kill-first. Knowledge runtime may still spend one bounded fresh-session recovery attempt when the first clean session preserved useful state without completing, and one bounded fresh-worker replacement attempt after a catastrophic first-session failure. Classification `task.json` still carries immutable top-level `ontology` plus per-unit `candidate_tag_keys`, but the answer surface is now just `category` plus `grounding`; the old reviewer-category and retrieval-concept metadata are gone. Grounding is now a keep gate rather than repair paperwork, so a candidate only stays `knowledge` if deterministic normalization leaves at least one real existing tag or one valid proposed tag. Under-grounded `knowledge` rows are demoted to canonical `other` instead of going through tag-invention repair. Same-session handoff validation still fails closed on immutable task-file tampering by rewriting the current file into repair mode instead of silently treating the run as unchanged or complete. Structured inline-JSON classification/grouping no longer keeps its packet/prompt/answer shapers in `workspace_run.py`; those helpers now live in `structured_session_contract.py`, while `workspace_run.py` keeps the runner loop and artifact orchestration. Recovery bookkeeping also no longer lives entirely in `recovery.py`: task-status tracking, stale follow-up finalization, and `stage_status.json` writing now live in `recovery_status.py`, while `recovery.py` keeps the watchdog/retry logic. `runtime.py` and `recovery.py` now import their `_shared`/owner dependencies explicitly instead of cloning `_shared` into module globals. `runtime.py` writes the sidecar `knowledge_tag_proposals.jsonl` plus grounding counts that flow through `promotion_report.json`, `knowledge_manifest.json`, `09_nonrecipe_authority.json`, and `knowledge_stage_summary.json`. `reporting.py` still emits the packet-economics rollups into stage `telemetry.json`; for the same-session path those rollups describe classification/grouping step counts rather than extra worker-session counts. The final reviewer artifacts come from promoted shard outputs rather than worker-edited phase ledgers. `__init__.py` exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
