Private knowledge-stage package. Start with `planning.py` for shard planning and progress accounting, `runtime.py` for stage orchestration and summary artifacts, `workspace_run.py` for the live phase-first per-worker session loop, `recovery.py` for retries/watchdogs and terminal classification, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is shard-ledger and phase-first: `knowledge/in/*.json` stays one shard per file, `task_manifest.jsonl` mirrors those shard ids for reporting only, worker roots expose `assigned_shards.json` plus `current_phase.json`, and the final reviewer artifacts come from promoted shard outputs rather than any hidden packet queue. `__init__.py` now exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
