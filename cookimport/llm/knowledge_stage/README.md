Private knowledge-stage package. Start with `planning.py` for shard planning and progress accounting, `runtime.py` for stage orchestration and summary artifacts, `workspace_run.py` for the live worker path, `recovery.py` for retries/watchdogs and terminal classification, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is now one happy-path workspace session per worker assignment: the worker edits classification `task.json`, runs the repo-owned `cookimport.llm.knowledge_same_session_handoff` helper, and then either repairs in place, continues straight into grouping in the same session, or finishes final shard outputs directly when everything is `other`. `task.json` now also carries repo-written `helper_commands`, `next_action`, and `answer_schema` metadata, and the same-session helper exposes read-only `--status` and `--doctor` entrypoints from the workspace root. Those helper modes auto-discover hidden state from `./_repo_control/`, and repair rewrites keep the same visible guidance plus immutable stage-specific metadata in the rewritten `task.json`. Knowledge runtime may spend one bounded fresh-session recovery attempt when the first clean session preserved useful state without completing; that restart is separate from watchdog policy. Classification `task.json` still carries immutable top-level `ontology` plus per-unit `candidate_tag_keys`; a kept row must emit `category=knowledge`, `reviewer_category=knowledge`, a short `retrieval_concept`, and non-empty `grounding` to an existing tag or a proposed tag under an existing category. Same-session handoff validation now fails closed on any task-file contract tampering, including immutable top-level edits, by rewriting the current file into repair mode instead of silently treating the run as unchanged or complete. `runtime.py` writes the sidecar `knowledge_tag_proposals.jsonl` plus grounding counts that flow through `promotion_report.json`, `knowledge_manifest.json`, `09_nonrecipe_authority.json`, and `knowledge_stage_summary.json`. `reporting.py` still emits the packet-economics rollups into stage `telemetry.json`; for the same-session path those rollups now describe classification/grouping step counts, not extra worker-session counts. The final reviewer artifacts come from promoted shard outputs rather than worker-edited phase ledgers. `__init__.py` exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
