Private knowledge-stage package. Start with `planning.py` for shard/task planning, `runtime.py` for stage-level orchestration, `workspace_run.py` for worker-session execution and post-session packet/shard finalization, `recovery.py` for retries/watchdogs, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is packet-native on disk and task-native at review time: `knowledge/in/*.json` stays one packet per file, shard manifests can bundle several packets, task manifests split those shard bundles back into one packet-owned task per result file, and the final reviewer artifacts are rendered from promoted packet groups rather than deterministic chunks. `__init__.py` now exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
