Private knowledge-stage package. Start with `planning.py` for shard planning and progress accounting, `runtime.py` for stage orchestration and summary artifacts, `workspace_run.py` for the live phase-first per-worker session loop, `recovery.py` for retries/watchdogs and terminal classification, `promotion.py` for no-op/final report helpers, and `reporting.py` for manifest/telemetry writers. The live contract is shard-ledger and phase-first: `knowledge/in/*.json` stays one shard per file, `task_manifest.jsonl` mirrors those shard ids for reporting only, worker roots expose `assigned_shards.json` plus `current_phase.json`, Pass 1 starts from raw owned rows instead of repo-authored semantic defaults, Pass 2 continues only over accepted Pass 1 knowledge rows, and the repo canonicalizes final `group_id` values after the worker supplies grouping semantics. Knowledge hints in `hints/<shard_id>.md` are intentionally compact worker-calibration sidecars, not second prompts: keep shard posture, example pointers, and attention rows, then fall back to the authoritative shard input. The final reviewer artifacts come from promoted shard outputs rather than any hidden packet queue. `__init__.py` now exposes only the thin public seam behind `codex_farm_knowledge_orchestrator.py`; shared internals live in `_shared.py`.
