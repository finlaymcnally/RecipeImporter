Private canonical line-role package. The live worker contract is now single-file and assignment-first: `planning.py` builds contiguous shard owners, `runtime.py` writes one editable `task.json` plus repo-owned debug artifacts, and `validation.py` expands validated answers back into shard `out/<shard>.json` outputs. Worker-visible line evidence is raw-text-first and stable-provenance-only; deterministic label hints and deterministic reason hints stay repo-side. `task.json` now also carries repo-authored `helper_commands`, `next_action`, and `answer_schema` metadata, workers should start with `python3 -m cookimport.llm.editable_task_file --summary` plus narrow `--show-unit` or `--show-unanswered` reads instead of dumping the full file, the same-session helper exposes read-only `--status` and `--doctor`, and the runtime may spend one bounded fresh-session recovery attempt when the first clean session preserved useful state without completing. Those helper modes auto-discover hidden state from `./_repo_control/`, and repair rewrites keep the same visible guidance metadata in the rewritten `task.json`. That restart is separate from watchdog policy. Workspace-worker supervision stays warning-first for ordinary drift, but line-role now has one explicit fail-fast seam after a final agent message: if the worker says it is done and `out/<shard>.json` is still missing after a 2-second grace window, runtime records a deterministic recovery assessment, may spend the one shared fresh-session retry budget on the preserved workspace, and otherwise fails closed with explicit missing-output metadata instead of grinding until context exhaustion. Same-session worker repair is still the primary recovery path, retryable watchdog kills may still get one bounded structured retry, and otherwise the shard fails closed. Deterministic fallback is not final authority on the live worker path. Start with `contracts.py` for the prediction model/normalization helpers, `prompt_inputs.py` for row serialization, `artifacts.py` for runtime/prompt artifact helpers, `planning.py`, `policy.py`, `validation.py`, and `runtime.py`. `__init__.py` is the small package seam.
