Private canonical line-role package. The live worker contract is single-file, assignment-first, and now direct-batch: `planning.py` builds contiguous shard owners, `runtime.py` writes one readable `task.json` plus repo-owned debug artifacts, and `validation.py` expands validated answers back into shard `out/<shard>.json` outputs. Workers should open `task.json` directly, read the full shard in order, fill every `answer.label` in place, run `task-handoff`, and reopen the rewritten file only if repair feedback appears. `task-status` and `task-doctor` remain optional troubleshooting helpers, while `task-show-unit` and `task-show-unanswered` are fallback-only helpers for local ambiguity, not the default path. The visible execution workspace is still just `task.json` plus `AGENTS.md`; repo-side `debug/`, `in/`, `out/`, and related worker-root artifacts remain observability surfaces, not the startup contract. The task file now advertises editability through `answer_schema.editable_pointer_pattern` instead of `editable_json_pointers`, and line-role no longer uses `answers.json`, `task-template`, or `task-apply` on the happy path. Ordinary local reads of `task.json` and `AGENTS.md` are allowed, while real boundary escapes still fail closed. One bounded fresh-session recovery attempt and one bounded fresh-worker replacement attempt remain available when repo-owned state shows recoverable progress. Start with `contracts.py` for the prediction model/normalization helpers, `prompt_inputs.py` for row serialization, `artifacts.py` for runtime/prompt artifact helpers, `planning.py`, `policy.py`, `validation.py`, and `runtime.py`. `__init__.py` is the small package seam.
