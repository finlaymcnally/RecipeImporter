# External Knowledge Manifest Derived Row

The normal bundle payload sweep only includes files physically under the benchmark session root. A processed-output knowledge manifest usually lives outside that tree, under `data/output/.../raw/llm/.../knowledge_manifest.json`.

That means resolver fixes alone are not enough. If the bundle wants `knowledge_manifest_in_bundle = true` and a working `knowledge_by_run` locator, it has to copy the external manifest into a derived payload row, currently `_upload_bundle_derived/runs/<run_id>/knowledge_manifest.json`.
