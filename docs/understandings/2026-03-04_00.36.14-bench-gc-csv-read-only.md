# Discovery

- `cookimport bench gc` previously rewrote `performance_history.csv` in apply mode by hydrating missing durable fields and pruning stale non-durable rows.
- The GC path also wrote timestamped `*.gc.bak.csv` backups whenever it mutated history.

# Resolution

- GC is now CSV read-only: no row hydration writes, no stale-row deletion, and no history backup writes.
- Run-root pruning now requires durable benchmark metrics already present in CSV rows.
