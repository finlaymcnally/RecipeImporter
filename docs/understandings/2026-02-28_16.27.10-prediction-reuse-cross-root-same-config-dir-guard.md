# Discovery

Prediction reuse cache entries already stored enough signal to reuse across roots (`prediction_reuse_key`), but materialization logic rejected entries when `config_dir` matched the current config dir name.

Because config dir names are deterministic (`config_{index}_{hash}_{slug}`), two different run roots can share the same name for equivalent configs. That guard blocked valid cross-root reuse and forced extra prediction work.

# Outcome

- Reuse now allows same `config_dir` when source artifact paths point to a different run root.
- Cache entries include absolute source artifact dirs and reuse can copy from those paths.
- Quality runner and tournament now resolve/report a shared prediction reuse cache root so reuse can span rounds, experiments, and folds by default.
