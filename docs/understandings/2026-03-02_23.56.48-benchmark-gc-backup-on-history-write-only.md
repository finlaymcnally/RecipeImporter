# Benchmark GC backup behavior

Current contract:
- `bench gc --apply` writes `performance_history.<YYYY-MM-DD_HH.MM.SS>.gc.bak.csv` before deleting confirmed run roots or rewriting benchmark history rows, when history CSV exists.
- If apply has no confirmed deletion and no history rewrite, no new backup is written.

This keeps backup guarantees aligned with apply-time mutations while preserving idempotent no-op runs.
