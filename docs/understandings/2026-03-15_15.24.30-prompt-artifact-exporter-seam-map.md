The old prompt exporter coupled three concerns in one CLI helper: raw-run discovery (`pass1/pass2/...` folders + manifest names), row normalization (`full_prompt_log.jsonl` payload construction), and artifact rendering (`prompt_request_response_log.txt`, category files, sample markdown).

The stable refactor seam is now:

- `discover_codexfarm_prompt_run_descriptors(...)` in `cookimport/llm/prompt_artifacts.py` for current-layout discovery.
- `render_prompt_artifacts_from_descriptors(...)` for topology-agnostic rendering from normalized descriptors.
- `build_codex_farm_prompt_response_log(...)` as the wrapper used by CLI call sites.

Downstream pass4 tooling was brittle because it assumed exactly `prompt_task4_pass4_knowledge.txt`. Both `scripts/benchmark_cutdown_for_external_ai.py` and `cookimport/bench/followup_bundle.py` now resolve pass4 prompt files through `prompt_category_logs_manifest.txt` plus `prompt_task4_*.txt` fallback discovery, so dynamic pass4 stage stems are included without adding new hardcoded filenames.
