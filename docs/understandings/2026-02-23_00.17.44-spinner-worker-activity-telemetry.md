# Discovery

- `_run_with_progress_status(...)` is the central spinner renderer for Label Studio import/benchmark paths, so worker summaries should be implemented there once instead of per command.
- Raw progress callbacks are string-only; adding per-worker UI needs a small serialized side-channel so worker updates do not overwrite the main phase/task status line.
- The stable contract is a hidden worker-activity payload (`format_worker_activity(...)` / `format_worker_activity_reset()`) that spinner code parses (`parse_worker_activity(...)`) and renders as multiline worker rows under the decorated main status message.
- Freeform prelabel worker threads can provide useful worker summaries by assigning stable thread->slot ids and emitting `task X/Y` + segment range activity updates, then idle/reset events.
- Split conversion loops can reuse the same telemetry contract by assigning virtual worker slots to in-flight jobs and emitting `job X/Y` activity updates as slots are reused.
