# Compare & Control live dashboard state

The visible Compare & Control panel is already browser-driven from `cookimport/analytics/dashboard_render.py`. When the dashboard is opened with `cookimport stats-dashboard --serve`, the page polls `assets/dashboard_ui_state.json` every 3 seconds and reapplies newer payloads when `saved_at` increases.

Two details matter for live control:

- Writing the file is enough; there is no separate compare/control HTTP API beyond the served `assets/dashboard_ui_state.json` route.
- The chart does not need a separate "activate" write. If the incoming state has a valid `compare_field` and `view_mode` is not `discover`, `shouldAutoActivateCompareControlChart(...)` turns the chart on during the next render pass.

That means an external CLI/agent only needs to update `previous_runs.compare_control` plus a fresh ISO `saved_at` timestamp for the visible panel and chart to move on their own.
