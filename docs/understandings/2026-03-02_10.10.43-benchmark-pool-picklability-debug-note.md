---
summary: "Global benchmark queue regression found: process pool submission fails when monkeypatched local callbacks are unpicklable."
read_when:
  - "When this test suite fails with 'Can't pickle local object' in benchmark worker execution."
---

- `cookimport/cli.py` uses `ProcessPoolExecutor` in `_run_all_method_benchmark_global_queue` for parallel config execution.
- The tests monkeypatch `_run_all_method_prediction_once` with local helper functions, which are not picklable under process pools.
- Worker submissions then fail with `Can't pickle local object ...`, producing zero successful runs and forced retries with empty assertions.
- The probe was added to validate picklability of `_run_all_method_prediction_once` before enabling process concurrency.
- If pickling is not supported, execution now falls back to `ThreadPoolExecutor` automatically.
