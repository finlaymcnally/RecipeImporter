The Oracle benchmark follow-up loop is strong on the packet-generation side but weak on the orchestration side. `cf-debug` already supports `request-template`, `select-cases`, `pack`, and `build-followup` for additive `followup_dataN/` packets, but there is no current bridge that converts Oracle's natural-language first-pass follow-up asks into a machine-readable `cf.followup_request.v1` manifest or automatically launches a second Oracle turn from that manifest.
