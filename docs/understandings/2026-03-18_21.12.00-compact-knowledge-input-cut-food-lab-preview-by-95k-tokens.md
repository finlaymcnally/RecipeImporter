## Compact knowledge input cut Food Lab preview cost

- The live knowledge-stage input contract is now compact and alias-based:
  - top level `v`, `bid`, `c`
  - per chunk `cid`, `b`, optional `h`
  - per block `i`, `t`, optional `hl`, `th`
  - optional `x` context and `g` guardrails only when non-empty
- We also stopped pretty-printing the inline knowledge JSON in `build_knowledge_direct_prompt(...)`.

Evidence from zero-token full preview:

- Old preview: `data/output/2026-03-18_17.15.00-foodlab-current-preview/prompt_preview_budget_summary.json`
  - `nonrecipe_knowledge_review`: `214,665` input / `47,029` output / `261,694` total
  - full preview: `475,494` total
- New preview: `data/output/2026-03-18_21.05.00-foodlab-compact-knowledge-preview/prompt_preview_budget_summary.json`
  - `nonrecipe_knowledge_review`: `120,077` input / `47,029` output / `167,106` total
  - full preview: `382,326` total

Net effect:

- knowledge total tokens dropped by `94,588` (`-36.1%`)
- full 3-stage preview dropped by `93,168` (`-19.6%`)
- recipe stayed flat
- line-role stayed effectively flat

Using the same `o200k_base` tokenization on the new Food Lab knowledge shard inputs:

- compact knowledge payload bodies: about `146,048` input tokens
- raw `blocks[*].text` only: about `86,599` tokens
- remaining overhead: about `59,449` tokens

That overhead was previously about `125,467`, so the compact contract removed roughly half of the non-text scaffolding without dropping the underlying chunk evidence.
