## Discovery Summary

- Interactive run-settings Codex model picker now sources discovered models from `codex-farm models list --json` (`cookimport/cli_ui/run_settings_flow.py` via `cookimport/llm/codex_farm_runner.py:list_codex_farm_models`).
- Recipe pass1/2/3, pass4 knowledge, and pass5 tags subprocess paths now validate configured pipeline ids before execution using `codex-farm pipelines list --root <pack> --json`.
- Subprocess runner still executes `codex-farm process --json`, but now parses process JSON and checks both subprocess return code and payload `exit_code`.
- On failure with `run_id`, runner issues `codex-farm run errors --run-id <run_id> --json` and includes first-error detail in `CodexFarmRunnerError`.
