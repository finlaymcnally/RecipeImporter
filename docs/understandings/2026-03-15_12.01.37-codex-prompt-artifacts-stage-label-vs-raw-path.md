---
summary: "Prompt artifact export can label stages dynamically even though raw Codex folders still use legacy pass-slot names."
read_when:
  - "When prompt artifact files mention the wrong pass/stage for merged-repair or other repurposed pipelines"
  - "When deciding whether to rename raw `raw/llm/.../pass2_schemaorg` storage folders"
---

`_build_codex_farm_prompt_response_log()` had its own hardcoded maps for prompt sample headings and `prompt_task*.txt` filenames, so reviewer-facing artifacts kept saying `pass2 schemaorg` even when the actual pass2 pipeline id was `recipe.merged-repair.compact.v1`.

The exporter already had enough runtime data to avoid that: the per-pass manifest payload exposes `pipeline_id`, and the full prompt log rows can carry derived stage metadata. Fixing the prompt collection surface only required deriving labels from observed stage metadata while keeping `row["pass"]` as a compatibility alias.

This does not rename the underlying raw LLM folders. `cookimport/llm/codex_farm_orchestrator.py` still writes merged-repair inputs/outputs under `raw/llm/<book>/pass2_schemaorg/{in,out}` because the recipe orchestrator itself is still organized around legacy pass slots. So there are two seams:

- reviewer-facing prompt artifacts can be stage-accurate now,
- raw storage layout is still slot-compatibility-first and would need a separate refactor if you want those folders renamed too.
