# Oracle Browser Upload Still Hits `readFiles(...)` Cap

- The installed Oracle CLI here is `0.8.6`.
- In that version, browser runs still call `readFiles(...)` before prompt assembly and browser upload handling.
- `readFiles(...)` enforces a hard per-input-file limit of `1 * 1024 * 1024` bytes and throws `The following files exceed the 1 MB limit` before the browser attachment path runs.
- The headless wrapper already passes `--browser-attachments always --browser-bundle-files`, but that does not help because the raw `--file` inputs are rejected before bundling/upload.
- Repo-side workaround: keep the on-disk `upload_bundle_v1` contract unchanged, but build a temporary Oracle-only attachment set for browser mode where oversized text files are split into ordered shard files under the cap.
