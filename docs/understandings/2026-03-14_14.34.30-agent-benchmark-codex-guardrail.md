# Agent Benchmark Codex Guardrail

The old `labelstudio-benchmark` command boundary only required `--allow-codex` for live Codex-backed runs. That stopped accidental Codex use, but it did not stop a coding agent from self-approving by adding the flag.

The fix is two-layered. First, live `labelstudio-benchmark` Codex runs now require both `--allow-codex` and `--benchmark-codex-confirmation I_HAVE_EXPLICIT_USER_CONFIRMATION`. Second, agent-run environments are hard-blocked from live Codex benchmark execution entirely; they must use `--codex-execution-policy plan` or defer to a human-run shell outside the agent environment.

The same agent-env hard block now also covers `bench speed-run --include-codex-farm`, which can indirectly launch Codex-backed benchmark work.
