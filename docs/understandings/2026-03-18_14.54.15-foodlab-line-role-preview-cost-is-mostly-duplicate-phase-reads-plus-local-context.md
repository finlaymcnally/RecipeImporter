# Food Lab Cost Shape After File-Backed Fix

Using the same `o200k_base` tokenization method as prompt preview, the Food Lab line-role preview spends only about `23.6k` input tokens on wrapper prompts and about `1.224M` input tokens on the deposited task-file bodies themselves.

Within those task files, the biggest drivers are:

- duplicate full-book reads across both phases: about `600.5k` task-body input tokens for `recipe_region_gate` and about `624.0k` for `recipe_structure_label`
- local context duplication: adding `prev_text` and `next_text` around every row costs about `527.3k` task-body input tokens total
- non-context metadata plus JSON framing beyond the minimal `atomic_index` + `current_line` seam costs about `370.0k` task-body input tokens total

The practical lesson is that the file-backed transport solved wrapper bloat and context-window failures caused by giant inline prompts, but the next spend cut has to come from narrowing which rows reach the model, shrinking repeated local context, or avoiding a full second pass over the same rows.
