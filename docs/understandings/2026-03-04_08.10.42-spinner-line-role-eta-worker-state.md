## Discovery

- Spinner worker overlays were leaking codex state between phases.
- `_run_with_progress_status` kept `latest_running_workers/latest_active_tasks` from codex-farm messages and did not clear them when the next non-codex phase message arrived.
- That made later messages like `Running canonical line-role pipeline...` render with stale `active workers: 0`.

## Implementation

- Clear codex worker/stage state on non-codex, non-worker callback messages in `_update_progress_common`.
- Add canonical line-role codex batch progress callbacks in `label_atomic_lines(...)` as:
  - `Running canonical line-role pipeline... task X/Y | running N`
- Wire ingest to pass `_notify` into `label_atomic_lines(...)` so spinner gets task counters.

## Outcome

- No stale `active workers: 0` line after codex->non-codex phase transitions.
- Existing spinner ETA logic now applies to canonical line-role stage automatically via `task X/Y` counters.
