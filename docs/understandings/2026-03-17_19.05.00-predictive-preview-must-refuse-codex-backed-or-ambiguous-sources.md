---
summary: "Predictive prompt preview must refuse Codex-backed or ambiguous processed runs; otherwise it leaks retrospective data and stops being a real prediction."
read_when:
  - When changing prompt-preview source resolution for predictive mode
  - When a benchmark wrapper points only at a Codex-backed processed run
  - When deciding whether predictive preview should fall back to a non-vanilla source
---

If predictive preview cannot positively identify a deterministic or `vanilla` workload source, it should fail. Falling back to a Codex-backed processed run is not a degraded prediction; it is retrospective leakage.

The safe rule is:

- predictive accepts only deterministic/`vanilla` artifacts
- observed is the only mode that may read Codex-backed processed runs and live telemetry
- ambiguous sources should be treated the same as Codex-backed ones and rejected in predictive mode
