# Dashboard filter clause edit flow (2026-03-04)

- Active clause summaries in `renderPreviousRunsTableColumns(...)` are rendered per clause; wiring edit on the summary text lets users jump directly into the existing popover flow.
- The filter editor state is `previousRunsOpenFilterDraft`; carrying an optional `edit_index` in that draft enables in-place updates while preserving the existing append-on-save behavior for new clauses.
- Replacing one clause without resetting the full stack is safest through a dedicated helper (`updatePreviousRunsColumnFilterAt`) that reuses existing normalization/dedupe rules.
