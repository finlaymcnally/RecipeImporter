# Discovery

`generate_pred_run_artifacts(...)` prelabeling runs one task per threadpool slot and collects per-task failures. Without a shared stop signal, a 429 can still allow queued tasks to keep calling the provider, which risks amplifying throttling.

# Contract

- Detect `HTTP 429`/rate-limit failures from provider error text.
- Set one shared stop event as soon as the first 429 is observed.
- Before each task provider call, check the stop event; if set, skip that queued task (recorded in `prelabel_errors.jsonl`) instead of calling the provider.
- Emit an explicit warning message containing `429` so CLI users can immediately see why task processing stopped early.
