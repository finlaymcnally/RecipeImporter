# Legacy codexfarm benchmark classification

- March 5 dashboard JS tightened `aiAssistanceProfileForRecord(...)` / `isOfficialGoldenBenchmarkRecord(...)` so official single-offline rows only count as `codexfarm` when they look like full-stack AI (`llm_recipe_pipeline` on and `line_role_pipeline` on).
- Older benchmark CSV rows often recorded the recipe pipeline but not `line_role_pipeline`.
- Older benchmark CSV rows also often omitted `artifact_dir`, leaving the variant-identifying path only in `run_dir` / `report_path`.
- Result: historical path-based `.../single-offline-benchmark/.../codexfarm` rows were reclassified as `recipe_only`, and because the trend chart only split benchmark variants into `vanilla`, `codexfarm`, and `other`, those rows vanished from the chart entirely.
- Backward-compatible fix: for official paired benchmark paths, fall back to the path variant when line-role metadata is missing, and read that path from `artifact_dir`, `run_dir`, or `report_path`. That keeps legacy `vanilla`/`codexfarm` trend history visible without changing newer rows that already carry explicit metadata.
