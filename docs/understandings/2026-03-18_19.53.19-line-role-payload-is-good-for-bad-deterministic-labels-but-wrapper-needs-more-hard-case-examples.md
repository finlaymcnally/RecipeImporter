# Line-Role Review Under Bad Deterministic Labels

With the assumption that deterministic labels are often wrong, the current line-role payload shape still looks sensible:

- the model sees one ordered contiguous shard, not isolated rows
- each row still carries the raw line text
- `label_code` is framed as a first-pass hint, not authority
- the prompt explicitly tells the model to use neighboring rows for context

That means the model can often recover from bad first-pass labels by reading the local run of lines instead of trusting the deterministic code.

The main remaining weakness is prompt teaching, not payload density.

The active file-backed wrapper now has better semantic rules, but it still does not give explicit examples for the hardest deterministic-only failure modes, especially:

- endorsements and dedications versus `KNOWLEDGE`
- table-of-contents menus versus `INGREDIENT_LINE`
- chapter / part headings versus recipe-side labels
- section menus and one-word headings such as `Light`

So the next line-role improvement should probably be a small set of targeted few-shot examples in the active file-backed wrapper, not re-expanding the billed task payload with more deterministic metadata.
