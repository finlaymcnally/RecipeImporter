summary: "Recipeimport prompt-sample thinking traces depend on CodexFarm classifying nested `item.completed` reasoning wrappers upstream."
read_when:
  - "When prompt_type_samples_from_full_prompt_log.md shows trace metadata but no reasoning excerpt"
  - "When deciding whether a missing thinking trace is a recipeimport export bug or an upstream CodexFarm capture bug"
---

`cookimport/cli.py` already extracts reasoning snippets from event keys including `text`, so the prompt-sample exporter does not need a special-case patch for nested Codex events. The real failure mode is upstream: if CodexFarm writes a `.trace.json` with `reasoning_events=[]` because it missed `{"type":"item.completed","item":{"type":"reasoning","text":...}}`, recipeimport can only render the fallback `_No thinking trace captured for this sample._`.

Practical debugging rule: inspect the linked trace file first. If `reasoning_event_count` is zero but raw Codex output included nested `item.type="reasoning"`, fix CodexFarm trace classification rather than recipeimport report rendering.
