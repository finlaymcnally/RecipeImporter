## Discovery

`docs/plans/OGplan/speed2-2.md` is partially useful but stale in key assumptions:

- Runtime no longer hardcodes stdlib `difflib`; canonical eval now selects matcher implementation via `COOKIMPORT_BENCHMARK_SEQUENCE_MATCHER` with `auto|stdlib|cydifflib|cdifflib` and prefers accelerated implementations when available.
- Canonical eval already has disk-backed alignment reuse keyed by canonical text + prediction text + block boundaries; all-method runs share this cache per source.
- Evaluator telemetry already records matcher implementation/mode and cache hit/miss fields.

Observed run evidence from `data/golden/benchmark-vs-golden/2026-02-27_17.54.41`:

- All scanned eval reports in that run recorded matcher impl `cydifflib` with mode `auto`.
- Cache hits were common and reduced alignment SequenceMatcher time to `0.0` on hit cases.
- Cache misses still showed alignment dominating eval wall time, so remaining work should target miss-path cost in current architecture (not pre-selector assumptions).

## Practical implication

Use `speed2-2.md` only as idea source (identity short-circuit + strict parity validation). Do not execute it verbatim; it should be rewritten around the current selector + disk-cache design and current telemetry contracts.
