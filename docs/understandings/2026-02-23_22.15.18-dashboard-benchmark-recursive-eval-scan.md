# Dashboard benchmark recursive eval scan

- A blank all-method page can occur when benchmark collection only scans shallow `eval_report.json` paths.
- All-method benchmark outputs store `eval_report.json` under nested config folders:
  - `.../eval-vs-pipeline/<run_ts>/all-method-benchmark/<source_slug>/config_*/eval_report.json`
- Dashboard collector now scans `eval_report.json` recursively under `golden_root`, still excluding `prediction-run` and pytest temp paths.
- Nested config eval rows should use the nearest timestamped parent folder (for example `2026-02-23_16.01.06`) as `run_timestamp` for stable sorting and run grouping.
