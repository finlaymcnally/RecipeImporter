# QualitySuite Race Finalists No-Prune Overhead

- Current all-method EPUB base grid is 13 variants per target (`unstructured` parser/preprocess/skip combinations plus `beautifulsoup`), before optional deterministic sweeps.
- Race keep logic preserves at least `race_finalists` configs each round (`_compute_keep_count(...)` in `cookimport/bench/quality_runner.py`), so if `race_finalists >= variant_count`, no hashes are pruned.
- With the fast no-sweeps thresholds (`race_probe_targets=1`, `race_mid_targets=2`, `race_finalists=24`, sweeps off), one 6-target experiment does approximate work:
  - race: `13 (probe) + 26 (mid) + 78 (final) = 117` config evaluations
  - exhaustive: `6 * 13 = 78` config evaluations
- With sweeps enabled (about 143 variants/target), the same race settings do prune materially (probe -> ~26 survivors, mid -> ~24 finalists), so race remains beneficial versus exhaustive.
- Practical tuning rule: if sweeps are off and per-target variants are low, either lower `race_finalists` below variant count or use `--search-strategy exhaustive`; keep race for high-cardinality sweeps-on runs.
