# Post-refactor onboarding note

The current repo should be explained as a label-first staging system, not as an importer-first writer pipeline with optional extras.

Key boundaries verified in code:

- `cookimport/staging/import_session.py` now builds authoritative labels first and writes `label_det`, optional `label_llm_correct`, and `group_recipe_spans` artifacts before recipe drafting.
- If importer candidates existed but authoritative regrouping finds zero recipes, the run keeps the label-first result and writes `group_recipe_spans/<workbook>/authority_mismatch.json`.
- `08_nonrecipe_spans.json` is the machine-readable authority for outside-recipe `knowledge` vs `other`.
- `stage_observability.json` is the run-level semantic stage index and should be the main naming backbone for onboarding docs.
- Inline recipe tags are part of recipe correction plus deterministic normalization; there is no standalone tagging runtime package anymore.
