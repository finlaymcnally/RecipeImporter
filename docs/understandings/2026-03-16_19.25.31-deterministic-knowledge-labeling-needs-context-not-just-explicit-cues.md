The benchmark path currently scores authoritative Stage 2 labels, so improving `cookimport/parsing/canonical_line_roles.py` matters more than tuning later knowledge extraction if the goal is benchmark recall.

Two heuristics were the main blockers in the missed `saltfatacidheatcutdown` cases:

- outside-recipe prose only promoted to `KNOWLEDGE` on a narrow explicit-cue regex (`conduct heat`, `in other words`, `temperature`, etc.), which missed explanatory paragraphs about salt, flavor, texture, chemistry, evaporation, and similar cooking-science prose;
- `_looks_recipe_note_prose(...)` treated any first-person cooking-context prose as `RECIPE_NOTES`, which pushed memoir-ish but still instructional explanation paragraphs out of `KNOWLEDGE`.

The low-risk deterministic improvement is:

- keep the hard denials for obvious narrative/front-matter cases;
- require advisory/editorial language before first-person prose becomes `RECIPE_NOTES`;
- add domain/explanation cues for outside-recipe science prose;
- let compact domain headings like `SALT AND FLAVOR` inherit `KNOWLEDGE` when neighboring prose is already knowledge-like.
