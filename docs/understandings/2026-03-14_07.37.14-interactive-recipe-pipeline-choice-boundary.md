# Interactive Recipe Pipeline Choice Boundary

The interactive run setup does not select `llm_recipe_pipeline` directly. It first resolves a top-tier profile in `cookimport/cli_ui/run_settings_flow.py`, then harmonizes that profile into concrete `RunSettings`. That means adding a new interactive recipe-pipeline choice requires overriding the profile-produced `llm_recipe_pipeline` after harmonization; otherwise the codex profile always snaps back to its contract default.

Benchmark setup has a second seam in `cookimport/cli.py:_interactive_single_offline_variants`. When the selected run settings have any non-`off` recipe pipeline, that helper rebuilds a vanilla baseline and a codex variant from the benchmark contract. If the helper hard-codes the codex recipe pipeline, an interactive choice like `codex-farm-2stage-repair-v1` is lost even though the initial chooser accepted it.
