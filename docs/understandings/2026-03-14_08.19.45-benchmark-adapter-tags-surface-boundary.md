# Benchmark Adapter Tags Surface Boundary

`build_benchmark_call_kwargs_from_run_settings(...)` is not a dump-all `RunSettings` serializer. It has to track the actual `labelstudio_benchmark(...)` call surface.

As of the March 14 benchmark cleanup, tag-pipeline knobs are stage-only:
- `llm_tags_pipeline`
- `tag_catalog_json`
- `codex_farm_pipeline_pass5_tags`

If the benchmark adapter forwards those fields, paired single-offline and other helper-backed benchmark runs fail before prediction generation starts because Python rejects the unexpected kwargs at the `labelstudio_benchmark(...)` call boundary.
