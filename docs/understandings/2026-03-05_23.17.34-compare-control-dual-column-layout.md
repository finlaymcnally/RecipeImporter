## Discovery

- The current dual-set DOM is already split into separate controls, results, and chart rows; the missing "whole way down" effect comes from the results row staying single-column and the chart row defaulting to `stacked`.
- A low-risk fix is to keep the existing IDs/state model, switch the results row to two columns in dual mode, and auto-promote the chart layout from `stacked` to `side_by_side` when Set 2 is opened.
- This preserves combined-chart behavior as an explicit opt-in override instead of forcing a larger markup refactor.
