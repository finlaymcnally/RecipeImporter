# Interactive Pipeline Warning Dump Source

The menu prompt in `cookimport/cli_ui/run_settings_flow.py` is not appending that long text itself. After the operator selects a recipe pipeline, `choose_run_settings(...)` loads saved QualitySuite winner settings when the profile is codex, then re-runs the top-tier harmonization through `RunSettings.from_dict(...)`.

That produces two warning classes:
- retired-key warnings from stale fields still present in `data/.history/qualitysuite_winner_run_settings.json`
- unknown-key warnings when `to_run_config_dict()` carries `bucket1_fixed_behavior_version` metadata back through `RunSettings.from_dict(...)`

The dump is therefore compatibility/logging noise in the interactive flow. It is not the selected pipeline value, and it does not mean the run setup failed.
