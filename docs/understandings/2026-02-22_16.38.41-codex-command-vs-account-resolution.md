# Codex command vs account resolution

- Interactive freeform prelabel previously always passed `codex_cmd=None`, so runtime used global default command/home and could hit the wrong login even when another command (for example `codex2`) was intended.
- Correct fix is command-aware resolution without extra prompts: resolve command (`COOKIMPORT_CODEX_CMD` or `codex exec -`), then resolve model defaults/cache/account identity from that command's Codex home roots (`CODEX_HOME` honored).
- `codex2` (and other `codexN`) should be treated as Codex executables for `--model` injection and plain-command `exec -` fallback behavior.
- Showing account email in the interactive flow prevents silent wrong-account runs before expensive prelabel loops start.
