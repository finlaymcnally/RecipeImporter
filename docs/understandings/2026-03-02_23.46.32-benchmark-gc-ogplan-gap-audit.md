# Benchmark GC OG Plan Gap Audit

- `bench gc` currently prunes selected run roots even when no benchmark history CSV is found, so deletion can proceed without confirming durable metric retention.
- The OG plan required proof that old benchmark rows stay visible after pruning; current tests hydrate CSV then prune, but do not assert dashboard visibility post-prune.
- The OG plan required idempotence and backup timestamp-format proofs; current tests do not explicitly assert second-run idempotence or `YYYY-MM-DD_HH.MM.SS` backup naming format.
