# Discovery

`codex_farm_pass1_pattern_hints_enabled` was added as a run setting and propagated through stage/benchmark surfaces, but automatic top-tier profile baseline patches were not pinning it explicitly.

# Resolution

- Built-in codex fallback baseline now pins `codex_farm_pass1_pattern_hints_enabled=false`.
- Vanilla automatic baseline also pins `codex_farm_pass1_pattern_hints_enabled=false` (inert while `llm_recipe_pipeline=off`).
- Codex winner harmonization still only enforces the codex pipeline trio and does not overwrite winner-provided pass1 pattern-hints tuning.
