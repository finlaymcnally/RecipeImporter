# Knowledge Token Cuts: Chunk Collapse, Context 1, Short Keys

After bundle planning was fixed, the next big prompt-cost cut did not come from changing bundle limits again. It came from three smaller seams working together:

- collapse standalone heading fragments and tiny bridge chunks before bundling
- lower default knowledge context from `2` blocks to `1`
- shorten the structured-output contract keys

Evidence on `data/golden/benchmark-vs-golden/2026-03-16_20.23.25/.../line_role_only`:

- after the first bundle pass: `91` knowledge prompts, `~703k` estimated total tokens
- after chunk collapse + context `1` + short output keys: `41` knowledge prompts, `~579k` estimated total tokens

The final knowledge preview packed `299` surviving chunks into `41` prompts. Distribution:

- `12` chunks: `19` prompts
- `10` chunks: `2` prompts
- `9` chunks: `1` prompt
- `6` chunks: `1` prompt
- `4` chunks: `4` prompts
- `3` chunks: `1` prompt
- `2` chunks: `4` prompts
- `1` chunk: `9` prompts

The durable lesson is that token reduction here is not only about bigger bundles. Chunk-count suppression before bundling is just as important.
