read_when:
  - When replacing or debugging the line-role live transport

  - When updating line-role prompt export or `telemetry_summary.json` readers
---

The repo already had the hard part of the transport swap: `cookimport/llm/codex_exec_runner.py` can run one structured `codex exec` call, capture JSON events, and normalize usage totals. That meant the live line-role cutover did not need another subprocess wrapper.

The part that still had to stay line-role-specific was telemetry shaping. `line-role-pipeline/telemetry_summary.json`, prompt exports, and prompt-budget readers do not only want raw usage totals; they also want batch-facing metadata such as `prompt_index`, `shard_id`, requested `atomic_index` ownership, and one attempt payload per shard. Reusing the shared runner was straightforward, but the direct runtime still needed to map each shard run back into that existing line-role summary shape.
