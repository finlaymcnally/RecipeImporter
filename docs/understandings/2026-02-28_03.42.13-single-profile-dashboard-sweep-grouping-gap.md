## Discovery

- Dashboard run-index grouping in `cookimport/analytics/dashboard_render.py` previously only accepted paths shaped like:
  - `.../all-method-benchmark/<source_slug>/config_*`
- Single-profile all-matched runs are written as:
  - `.../single-profile-benchmark/<source_slug>/eval_report.json`
- Because no `config_*` path segment exists in single-profile runs, those rows were excluded from `_collect_all_method_groups(...)` and never rendered in `all-method-benchmark/index.html`.

## Resolution

- Extend sweep grouping to accept both path families:
  - `all-method-benchmark/<source_slug>/config_*`
  - `single-profile-benchmark/<source_slug>`
- Treat single-profile rows as one run-profile (using `run_config_hash` when present) so run-level config aggregation stays stable across books.
