# Discovery: benchmark telemetry is run-local first

Observed behavior:

- `data/.history/performance_history.csv` only had 2 non-pytest Feb 25-26 production rows.
- Full Feb 25-26 benchmark data lived in:
  - `data/golden/benchmark-vs-golden/**/eval_report.json`
  - `data/golden/benchmark-vs-golden/**/all_method_benchmark_report.json`
  - `data/output/**/all-method-benchmark/**/.history/performance_history.csv`

Durable rule:

- For benchmark performance analysis, treat run-local eval/source reports and run-local history CSVs as primary truth.
- Use top-level history only as a convenience index, not as complete benchmark telemetry.
