## Discovery

- Canonical line-role codex fanout is controlled only by `COOKIMPORT_LINE_ROLE_CODEX_MAX_INFLIGHT` in `cookimport/parsing/canonical_line_roles.py`.
- Interactive benchmark paths (`single_offline` and `single_profile`) did not override that env var, so both single-book and multi-book behavior inherited the same default.

## Decision (Historical)

- Use a scoped CLI env override for interactive benchmark runs:
  - single-book runs set `COOKIMPORT_LINE_ROLE_CODEX_MAX_INFLIGHT=8`
  - multi-book single-profile runs set `COOKIMPORT_LINE_ROLE_CODEX_MAX_INFLIGHT=4`

## Superseded By

- `docs/understandings/2026-03-04_08.50.26-shared-line-role-inflight-default-propagation.md`
- Newer behavior moved defaults into shared ingest prediction-generation seam, so benchmark/import paths inherit policy without per-flow CLI wiring.
