## Discovery

The benchmark/import path now uses `ProgressCallbackAdapter` + `ProgressDashboardCore` for both dashboard text and worker summary lines inside `_run_with_progress_status`, while keeping existing ETA/elapsed and callback compatibility logic intact.

Stage `Live(...)` rendering now uses a shared-core-backed worker-line pipeline (`ProgressDashboardCore`) for cached, debounced worker lines, while preserving filename/status/age formatting and merge/result state updates from stage jobs.

No new flow-level contracts were added; the migration was done by replacing local worker-line formatting calls with core-backed updates and keeping legacy scheduling/timer bookkeeping untouched.

## Why this matters

This reduces duplicate dashboard rendering paths and makes stage and benchmark progress observability follow the same shared shape without regressing operator-visible callback timing behavior.
