# All-method run-level dashboard hierarchy

- Existing per-book grouping still comes from benchmark artifact paths matching `all-method-benchmark/<source_slug>/config_*`.
- Dashboard generation now builds a second aggregation layer by run folder timestamp:
  - root index: `all-method-benchmark.html` (one row per run folder)
  - run summary: `all-method-benchmark-run__<run_timestamp>.html` (config means/wins across all books in that run)
  - per-book detail: `all-method-benchmark__<run_timestamp>__<source_slug>.html` (existing ranked config table + summary bars)
- Config aggregation key is `run_config_hash` when present, with config-slug fallback for older rows.
- The contract remains CSV-first; no new dashboard-only storage layer is introduced.
