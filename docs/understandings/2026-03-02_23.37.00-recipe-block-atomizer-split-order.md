# Discovery

For merged recipe blocks, split order matters more than individual regex quality.
The most reliable sequence here is:

1. split before explicit boundary markers (`NOTE:`, yield prefixes, `TO MAKE`/`FOR THE` headings, inline numbered steps),
2. then split yield-leading segments on later quantity starts so ingredient tails become separate lines,
3. then split remaining quantity-run segments.

Doing quantity-run splitting first can over-fragment headings or leave yield+ingredient bundles in one segment.

# Applied contract

- `cookimport/parsing/recipe_block_atomizer.py` now applies marker boundaries first, then yield-tail splitting, then general quantity-run splitting.
- Fixture coverage in `tests/fixtures/canonical_labeling/*.json` pins this order across known failure shapes (hollandaise merged block, inline numbered tails, range ingredients vs yield).
