# Spinner task-counter wiring map

- Spinner text shown in both interactive and non-interactive Label Studio import is driven by `progress_callback` messages emitted from `cookimport/labelstudio/ingest.py`.
- To keep both entrypoints consistent, task counters (`task X/Y`) should be generated in ingest loops where totals are known (prelabel/decorate task iteration), not in CLI-only wrappers.
- CLI wrappers should remain thin status presenters that render whatever progress callback messages they receive.
