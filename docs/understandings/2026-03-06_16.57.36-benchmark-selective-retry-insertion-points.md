- Benchmark config enters at `cookimport/cli.py::labelstudio_benchmark(...)`, flows through `cookimport/labelstudio/ingest.py::generate_pred_run_artifacts(...)`, and becomes canonical in `cookimport/config/run_settings.py::build_run_settings(...)`.
- The actual retry hook belongs in `cookimport/llm/codex_farm_orchestrator.py::run_codex_farm_recipe_pipeline(...)`, immediately after pass2/pass3 `codex_runner.run_pipeline(...)` returns and before the per-recipe output-validation loops mark missing bundles as errors.
- The runner already decides whether a non-zero exit is recoverable partial-output mode in benchmark runs, so the retry helper should consume serialized runner metadata instead of re-parsing stderr in a second place.
- Detailed retry provenance belongs in `raw/llm/<workbook_slug>/llm_manifest.json`; benchmark root manifests should only carry concise counts and the existing artifact link back to that LLM manifest.
