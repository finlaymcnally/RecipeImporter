## Discovery Summary

- Recipe pass1/2/3, pass4 knowledge, and pass5 tags all converge on `SubprocessCodexFarmRunner.run_pipeline(...)` when no fake/custom runner is injected.
- Injecting `--output-schema` at this shared subprocess boundary enforces schema contracts across all codex-farm production flows without changing every orchestrator callsite.
- Resolving schema from pack pipeline JSON by `pipeline_id` avoids filename-coupling and keeps enforcement aligned with pack metadata.
- Contract check can live at the same boundary: when `process --json` returns payload JSON, require `output_schema_path` and compare against the expected override path.
