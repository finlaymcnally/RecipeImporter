Line-role token telemetry gap summary:

- `cookimport/llm/codex_exec.py` can already extract token usage when `track_usage=True`.
- `cookimport/parsing/canonical_line_roles.py` currently calls `run_codex_json_prompt(..., track_usage=False)`, so line-role Codex usage is dropped at the source.
- `cookimport/cli.py:_load_pred_run_recipe_context()` only reads CodexFarm token telemetry from `llm_codex_farm` in `prediction-run/manifest.json`.
- `cookimport/analytics/perf_report.py` and `cookimport/analytics/dashboard_collect.py` already know how to backfill token fields from manifests when telemetry exists.
- The smallest truthful fix is to add a durable `prediction-run/line-role-pipeline/telemetry_summary.json` artifact, thread its path into manifests, and merge it into benchmark/history/dashboard token totals.
