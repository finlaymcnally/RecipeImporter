# Report metadata flow

Discovery:
- Single-file runs write reports in `cli_worker.stage_one_file`.
- Split EPUB/PDF runs write a fresh merged report in `cli._merge_split_jobs`.

Implication:
- Fields set only inside importer-level reports can be lost in split runs unless also written in merge-path report construction.

Current rule:
- `importerName` and `runConfig` are now set in both single-file and split-merge report paths so report metadata is consistent.
