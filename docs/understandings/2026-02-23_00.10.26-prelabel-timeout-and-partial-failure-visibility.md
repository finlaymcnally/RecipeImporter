summary: "Why freeform prelabel runs can look successful with mostly failed tasks and the CLI contract that prevents silent partial completion."
read_when:
  - "When investigating Label Studio runs that uploaded tasks but completed few prelabels"
  - "When changing prelabel timeout defaults or end-of-run CLI summaries"
---

Freeform prelabel failures can be hidden in practice when `allow_partial` is enabled: import/upload still completes, but only successful task annotations are attached.

Runtime truth markers:
- `prelabel_report.json` carries `task_count`, `success_count`, and `failure_count`.
- `prelabel_errors.jsonl` carries per-segment failures (timeouts, provider errors, parse failures).

Contract update from this task:
- Default prelabel timeout is now 300s per provider call.
- `labelstudio-import` completion output must print an explicit red `PRELABEL ERRORS: X/Y ...` summary with `prelabel_errors.jsonl` path whenever `failure_count > 0`, including allow-partial runs.
