For this benchmark run, `wrong_label_lines.with_context.full` (derived codex rows) carries `recipe_id`, but vanilla `wrong_label_lines.jsonl` does not.

Reliable recipe-level slicing flow:
1. Use codex derived context rows (`_upload_bundle_derived/.../wrong_label_lines.with_context.full.jsonl`) to map `line_index -> urn recipe_id`.
2. Join with codex `line-role-pipeline/joined_line_table.jsonl` to recover the local recipe token (`recipe:N`).
3. Use that token to collect all line indices for the recipe from joined table.
4. Filter vanilla `wrong_label_lines.jsonl` by those indices to recover baseline-only errors for that recipe.
