## 2026-03-06_00.44.16 per-label single-profile variant fallback

- Observed failure mode: per-label table showed all `-` in baseline/delta/rolling when latest run groups came from
  `/single-profile-benchmark/...` with no `.../codexfarm` or `.../vanilla` path segment.
- Cause: rendering filtered codexfarm rows using `benchmarkVariantForRecord(record) === "codexfarm"`, while that function returns `full_stack` for this single-profile shape.
- Fix approach: per-label calculations now treat `full_stack` as `codexfarm` only when the selected run group has no explicit `codexfarm` or `vanilla` variants.
- Impact: the single-profile full-stack path now renders baseline/rolling codexfarm columns as normal; mixed groups with explicit variants keep current behavior unchanged.
