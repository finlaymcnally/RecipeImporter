  - "When split-cache reuse appears stale after changing PDF extraction behavior knobs."

- `labelstudio-benchmark` had option normalization for `pdf_ocr_policy` / `pdf_column_gap_ratio` but the values were not forwarded through both prediction-generation branches (`generate_pred_run_artifacts` and `run_labelstudio_import`), so the knobs had no effect in those runs.
- `generate_pred_run_artifacts(...)` and `run_labelstudio_import(...)` needed explicit parameters plus passthrough into `build_run_settings(...)`; otherwise importer runtime stayed on defaults.
- Split-cache key semantics depend on `ALL_METHOD_SPLIT_CONVERT_INPUT_FIELDS`; new PDF knobs must be included there, or cached conversions can be incorrectly reused across different PDF settings.
