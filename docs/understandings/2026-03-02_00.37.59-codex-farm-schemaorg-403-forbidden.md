## Discovery: codex-farm `recipe.schemaorg.v1` failing with 403 Forbidden

While running offline benchmark prediction generation with `llm_recipe_pipeline=codex-farm-3pass-v1`, pass1 (`recipe.chunking.v1`) completed, but pass2 (`recipe.schemaorg.v1`) failed with `codex exec` returning exit code `1` for every task.

### Root cause
- The underlying `codex` CLI failed to connect to the Codex websocket endpoint:
  - `wss://chatgpt.com/backend-api/codex/responses`
  - Error: `HTTP error: 403 Forbidden`
- `codex-farm run errors` only surfaced the trailing warning (`no last agent message`), but the full failure is captured in codex-farm forensics bundles.

### Where to look
- List forensics bundles:
  - `codex-farm run forensics --run-id <RUN_ID> --json`
- Open the bundle and read:
  - `var/forensics/runs/<RUN_ID>/<TASK_ID>/attempt-*/stderr_tail.txt`
  - `var/forensics/runs/<RUN_ID>/<TASK_ID>/attempt-*/metadata.json`
