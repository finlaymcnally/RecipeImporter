summary: "The last real compatibility seam after shard cutover was accepting legacy pipeline ids in RunSettings; burn-the-boats means those ids must fail, not normalize."
read_when:
  - "When deciding whether shard-v1 cutover is complete on active CLI and run-setting surfaces."
  - "When a test or helper still tries to feed `codex-farm-*` or `codex-line-role-v1` into current run settings."
---

- `labelstudio/ingest.py` was already strict for recipe ids. The remaining migration seam lived in `cookimport/config/run_settings.py`, where recipe, knowledge, and line-role normalization still accepted legacy ids and silently upgraded them.
- That silent upgrade kept the old runtime names alive as valid operator inputs even after defaults/help/docs moved to shard-v1 ids.
- Burn-the-boats cutover means active run-setting surfaces should reject removed ids immediately. Historical readers and archived docs can still mention old ids, but current commands/config should not parse them.
