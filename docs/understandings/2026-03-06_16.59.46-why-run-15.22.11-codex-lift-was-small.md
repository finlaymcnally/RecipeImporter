The weak Codex lift in `data/golden/benchmark-vs-golden/2026-03-06_15.22.11` is mostly not a "Codex got dramatically worse" story. It is a convergence story plus one failed Codex book:

- `01_dinnerfor2cutdown` historically delivered a very large Codex gain, but in this run the Codex variant failed in pass2 (`content_filter` / mixed `timeout`) and never produced an eval. The shared progress log ends with Dinner at `v2/2 ok1 f1`, so the multi-book average lost one of the strongest Codex wins.
- The deterministic baseline in this run is much stronger and much closer to the old Codex shape than older `vanilla` runs:
  - `atomic_block_splitter=atomic-v1` instead of `off`
  - `line_role_pipeline=deterministic-v1` instead of `off`
  - `section_detector_backend=shared_v1` instead of `legacy`
  - `multi_recipe_splitter=rules_v1` instead of `legacy`
  - `instruction_step_segmentation_policy=always` instead of `auto`
  - `epub_unstructured_html_parser_version=v1` with `epub_unstructured_skip_headers_footers=true`
- Evidence of convergence:
  - Dinner vanilla improved from strict `0.4545` / macro `0.3164` to `0.6909` / `0.4499`, and its per-label recalls now match the old Codex run on `RECIPE_NOTES`, `YIELD_LINE`, and `TIME_LINE`, while `INGREDIENT_LINE` recall is only slightly below old Codex (`0.9245` vs `0.9326`).
  - Salt current Codex vs vanilla is identical on `INGREDIENT_LINE`, `KNOWLEDGE`, `RECIPE_NOTES`, `RECIPE_VARIANT`, and `YIELD_LINE`; Codex only adds small gains on `INSTRUCTION_LINE` and `RECIPE_TITLE`.
  - FoodLab current Codex vs vanilla mainly improves `RECIPE_TITLE`; most other label deltas are near zero.
- The newly enabled benchmark knowledge surface added cost but almost no benchmark lift on these books:
  - Salt current `KNOWLEDGE` metrics are identical between Codex and vanilla.
  - Sea current `KNOWLEDGE` metrics are also identical.
  - FoodLab current `KNOWLEDGE` delta is negligible.
  This means pass4 knowledge likely contributed mostly tokens/runtime rather than score in this canonical-text benchmark slice.
- Codex itself did not collapse everywhere:
  - Salt Codex quality in this run is actually better than the older 2026-03-04 Codex Salt run (`strict 0.5259` vs `0.4645`, `macro 0.4516` vs `0.3980`).
  The "bad lift" came from vanilla catching up, not from a broad Codex quality crash.
