# Five-Prompt Default Needs Preview Fallback And Knowledge Char-Cap Bypass

- Older processed runs do not carry newly added `*_prompt_target_count` fields in `runConfig`, so prompt preview must default missing values to the current shard-v1 default (`5`) instead of falling back to the legacy shard-size constants.
- Recipe preview also has to count shard prompts, not per-recipe inputs. Otherwise prompt preview overstates recipe interaction count even when live runtime already groups several recipes into one shard.
- Knowledge bundling had one remaining split source after target-count wiring: the old per-bundle char cap. On `data/golden/benchmark-vs-golden/2026-03-17_14.22.55`, that cap kept knowledge at `6` prompts until `_bundle_prepared_chunks(...)` stopped enforcing the char cap when `target_prompt_count` is explicitly driving the shard plan.
- After those fixes, the benchmark preview lands at `5` recipe prompts, `5` line-role prompts, and `5` knowledge prompts, `15` total.
