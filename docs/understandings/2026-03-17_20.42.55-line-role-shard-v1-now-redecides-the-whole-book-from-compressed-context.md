On `data/golden/benchmark-vs-golden/2026-03-17_20.27.19/single-offline-benchmark/saltfatacidheatcutdown`:

- `vanilla` is the known deterministic regression: `strict_accuracy=0.489768` with effectively no surviving `KNOWLEDGE` recall.
- `codexfarm` recovers a lot (`strict_accuracy=0.592769`), but the remaining misses are still dominated by outside-recipe classification, not recipe extraction.

What the current shard-v1 code is actually doing:

- `_label_atomic_lines_internal(...)` sends `codex_targets = ordered` whenever `line_role_pipeline=codex-line-role-shard-v1`, so Codex reviews the full ordered book, not just unresolved rows.
- `build_canonical_line_role_prompt(...)` gives Codex only `atomic_index|label_code|current_line`, explicitly says there is no prior recipe-span authority, and does not serialize per-row escalation reasons or inline neighbors.
- The benchmark run showed `slice_metrics.json` with near-perfect recipe slices but weak outside-recipe performance:
  - `recipe_instructions` accuracy `1.0`
  - `recipe_ingredients` accuracy `0.9818`
  - `outside_recipe` accuracy `0.5830`, macro F1 excluding other `0.3973`

Why that matters:

- The pipeline sounds like "deterministic first, LLM corrects edge cases", but the active line-role path is closer to "LLM re-labels the whole book from compressed context".
- On this benchmark, the hard class is not recipe structure; it is the semantic boundary between editorial/storytelling/book-navigation `OTHER` and explanatory cooking prose `KNOWLEDGE`.
- The gold set itself is strict here: many lines that read like useful cooking advice still score as `OTHER`, so broad Codex promotion to `KNOWLEDGE` raises recall but creates many false positives.

Practical implication:

- If the goal is 90% overall on canonical-text, fixing recipe extraction alone will not get there.
- The biggest leverage is to stop letting the line-role LLM freely re-decide obvious outside-recipe rows from minimal context, or to narrow the scored product goal so `KNOWLEDGE` vs `OTHER` is not carrying the whole benchmark.
