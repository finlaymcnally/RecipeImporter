## Discovery

- `single_offline` already normalized codex-selected settings into paired variants (`vanilla` then `codexfarm`) via `_interactive_single_offline_variants`.
- `single_profile_all_matched` previously bypassed that helper and executed one benchmark run per selected book, so codex selection did not force a per-book baseline pair.
- Reusing `_interactive_single_offline_variants` in single-profile mode keeps normalization consistent across all three interactive benchmark modes and enables per-book `codex_vs_vanilla_comparison.json` generation when both variant evals succeed.
