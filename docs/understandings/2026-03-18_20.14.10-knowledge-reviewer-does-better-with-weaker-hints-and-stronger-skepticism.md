# Knowledge reviewer does better with weaker hints and stronger skepticism

The knowledge-review model does not need richer machine guesses. It does better when the prompt is explicit that raw block text is authoritative and the hint object is weak metadata.

What changed:

- the prompt now frames the reviewer as skeptical and names the repo's common false-positive families directly
- `semantic_hint` was removed from the model-facing payload
- `suggested_lane` now appears only for strong `knowledge` evidence; menu-like, marketing, and high-noise chunks usually get only `text_form`

Why this mattered in fresh previews:

- Salt Fat praise blurbs, publisher signup copy, and `Recipes and Recommendations` bundles no longer arrive pre-biased toward `knowledge`
- the Food Lab chapter-menu bundle starting with `ESSENTIAL KITCHEN GEAR` lost its misleading `l=knowledge` hint
- real reference tables and technique prose still keep grounded block text, and they can still retain `l=knowledge` when the evidence is strong

The practical rule for future work is: if a machine-generated field is not strongly reliable, teach the rule in prompt prose instead of billing the reviewer model for another authoritative-sounding hint.
