## Discovery

- `cookimport/bench/eval_canonical_text.py` no longer instantiates stdlib matcher directly; it imports `SelectedSequenceMatcher` from `cookimport/bench/sequence_matcher_select.py`.
- Active env contract is `COOKIMPORT_BENCHMARK_SEQUENCE_MATCHER=auto|stdlib|cydifflib|cdifflib` (not `COOKIMPORT_CANONICAL_SEQUENCE_MATCHER_IMPL` / verify envs proposed in `speed2-4.md`).
- The planned MultiLayer module (`cookimport/bench/seqmatch_multilayer.py`) is not present.
- Parity coverage exists for accelerated drop-ins in `tests/bench/test_sequence_matcher_dropin_parity.py` (opcodes, matching blocks, and end-to-end canonical eval artifact byte equality checks).
- Benchmark telemetry wiring already records matcher impl/mode/version and the alignment subphase timer.

- In benchmark run `data/golden/benchmark-vs-golden/2026-02-27_17.54.41`, all parsed `eval_report.json` files used `alignment_sequence_matcher_impl=cydifflib` and `alignment_sequence_matcher_mode=auto`.

## Implication

- `docs/plans/OGplan/speed2-4.md` is still useful as background on parity-first alignment acceleration goals.
- As an implementation plan, it is stale for this repo state and would need a rewrite to match the existing selector-based architecture if reused.
