Discovery:
- AI runtime labels (`AI Model`/`AI Effort`) are derived from `run_config` metadata (`codex_farm_model`, `codex_model`, `llm_recipe_pipeline`), not from token telemetry.
- Token usage is sourced from `tokens_*` CSV/manifest fields; some benchmark rows have these fields missing.
- In dashboard JS, `maybeNumber` currently converts `null`/empty values to `0` (`Number("") === 0`), so missing token fields can render as `0 total | 0 in | 0 out` instead of `-`.
- Result: a row can legitimately show codex model metadata while token display appears zero, even when no token telemetry was persisted.

Code refs:
- `cookimport/analytics/dashboard_render.py`: `rawAiModelForRecord`/`aiModelLabelForRecord` and `previousRunsAllTokenUseDisplay` path.
- `cookimport/analytics/dashboard_render.py`: `maybeNumber` null/empty coercion behavior.
- `cookimport/analytics/dashboard_collect.py`: benchmark token extraction/backfill from CSV/manifest telemetry.
