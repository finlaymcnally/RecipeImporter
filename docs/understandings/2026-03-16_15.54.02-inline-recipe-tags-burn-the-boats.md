read_when:

The current standalone tags pass re-reads final recipe drafts after the main recipe-correction work is already done. That duplicates recipe text across LLM calls, adds a separate settings/CLI/docs surface, and forces late projection back into outputs. The cleaner architecture is to have the recipe-correction contract emit tags while it already has the authoritative recipe text in context.

The right follow-up is a small deterministic normalization step between `run_codex_farm_recipe_pipeline(...)` and `write_intermediate_outputs(...)` / `write_draft_outputs(...)` in `cookimport/staging/import_session.py`. That step should normalize casing, whitespace, and obvious punctuation variants across the workbook, then write the final normalized tags into `recipe.tags` and JSON-LD `keywords`. The old standalone tags pipeline, catalog wiring, tags CLI, and `raw/llm/.../tags/` artifacts should be removed rather than kept in parallel.
