# Discovery

- The all-method progress queue (`ok/fail`) increments on each config completion attempt during the initial pass.
- Retry passes (`all_method_retry_failed_configs`, default `1`) run after initial pass completion for that source and can recover failed configs, but the live queue counters are not retroactively corrected.
- Per-config timeout (`all_method_config_timeout_seconds`, default `900`) marks active configs failed, restarts the process pool, and requeues unfinished work.
- Scheduler event files are per-config and are overwritten on each rerun attempt, so first-attempt failure details are not preserved there.

# Practical Implication

- Seeing non-zero `fail` while a source is still running is expected; final truth is in that source's `all_method_benchmark_report.json`:
  - `failed_variants`
  - `retry_failed_configs_requested`
  - `retry_passes_executed`
  - `retry_recovered_configs`
