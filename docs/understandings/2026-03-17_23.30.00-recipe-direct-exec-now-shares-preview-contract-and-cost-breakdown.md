# Recipe Direct Exec Shares Preview Contract

The recipe phase no longer depends on `classic_task_farm_v1` for live shard execution. It now renders the same inline shard JSON that preview tokenizes, sends that prompt once through the direct Codex runner, and writes per-shard `prompt.txt`, `events.jsonl`, `usage.json`, and `cost_breakdown.json` artifacts under `recipe_phase_runtime/workers/.../shards/...`.

The practical seam is: compact recipe shard/input/output aliases reduce billed prompt and response size, while `prompt_budget` can now report the same `visible_input_tokens`, `cached_input_tokens`, `visible_output_tokens`, and `wrapper_overhead_tokens` vocabulary for both preview and actual runs.
