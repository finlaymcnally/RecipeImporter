# Discovery

- Codexfarm token usage is persisted per pass in each `*.excel_import_report.json` under:
  - `llmCodexFarm.process_runs.pass1.telemetry.rows[*].tokens_total`
  - `llmCodexFarm.process_runs.pass2.telemetry.rows[*].tokens_total`
  - `llmCodexFarm.process_runs.pass3.telemetry.rows[*].tokens_total`
- Summing `tokens_total` over `pass3.telemetry.rows` gives the effective run3/pass3 token load for that run.

# Quick Query

Use this pattern from repo root to list recent pass totals:

```bash
find data/output -type f -name '*.excel_import_report.json' | while read -r f; do
  jq -r --arg f "$f" 'if (.llmCodexFarm.enabled // false) then
    ( .llmCodexFarm.process_runs.pass1.telemetry.rows // [] | map(.tokens_total // 0) | add // 0 ) as $p1 |
    ( .llmCodexFarm.process_runs.pass2.telemetry.rows // [] | map(.tokens_total // 0) | add // 0 ) as $p2 |
    ( .llmCodexFarm.process_runs.pass3.telemetry.rows // [] | map(.tokens_total // 0) | add // 0 ) as $p3 |
    [ .runTimestamp, $p1, $p2, $p3, ($p1 + $p2 + $p3), $f ] | @tsv
  else empty end' "$f"
done | sort -r
```

- Ignore rows where pass telemetry is empty (`0/0/0`) when codex execution fell back before telemetry rows were emitted.
