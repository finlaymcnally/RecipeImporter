# EPUB extractor auto mode removed

- Runtime extraction mode selection is now explicit-only: `unstructured`, `legacy`, `markdown`, `markitdown`.
- `epub_extractor=auto` is rejected by CLI/prediction validators.
- Legacy saved run settings with `epub_extractor=auto` are migrated to `unstructured` in `RunSettings.from_dict(...)` with a warning.
- Stage/pred-run no longer produce auto-selection artifacts; requested/effective extractor run-config fields remain and now match for new runs.
- Deterministic auto-selection code remains available for extractor race/debug workflows only.
