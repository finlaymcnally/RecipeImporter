Discovery:
- `AI Effort` label derivation is implemented independently in dashboard JS (`aiEffortLabelForRecord`) and backend compare-control engine (`ai_effort_label_for_record`), so parity requires touching both.
- The correct seam for separating categories is label formatting, not raw effort extraction (`aiEffortForRecord` / `_ai_effort_for_record`), which should remain `null` for vanilla rows.
- Pipeline-off detection via `llm_recipe_pipeline=off` cleanly supports `AI off` while preserving `-` for unknown/missing effort.

Code refs:
- `cookimport/analytics/dashboard_render.py`: `aiEffortLabelForRecord`.
- `cookimport/analytics/compare_control_engine.py`: `ai_effort_label_for_record`.
- `tests/analytics/test_stats_dashboard.py` and `tests/analytics/test_compare_control_engine.py` for behavior assertions.
