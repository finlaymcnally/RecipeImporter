# Profeedback implementation findings (line-role + pass1 gates)

## What changed

- `cookimport/parsing/canonical_line_roles.py` now applies a runtime do-no-harm arbitration step in codex mode after sanitization.
- Outside-span containment is stricter:
  - `HOWTO_SECTION` is hard-denied outside recipe spans.
  - Outside-span `RECIPE_TITLE`/`RECIPE_VARIANT` require compact-heading shape plus nearby (±2 lines) structural evidence.
  - Outside-span `INSTRUCTION_LINE`/`INGREDIENT_LINE` require local evidence; otherwise labels are downgraded.
- Outside-span low-confidence escalation is disabled by default (`COOKIMPORT_LINE_ROLE_OUTSIDE_SPAN_LOW_CONFIDENCE_ESCALATION` can opt in).
- Codex line-role runs now emit:
  - `line-role-pipeline/do_no_harm_diagnostics.json`
  - `line-role-pipeline/do_no_harm_changed_rows.jsonl`

## Pass1 eligibility gate

- `cookimport/llm/codex_farm_orchestrator.py` now scores pass1 spans before pass2:
  - `+2` ingredient evidence
  - `+2` instruction evidence
  - `+1` heading/yield context
  - `-2` high prose dominance
- Actions:
  - `score >= 3` -> `proceed`
  - `score 1-2` -> `clamp` to heuristic bounds
  - `score <= 0` -> `drop` before pass2
- Diagnostics file:
  - `raw/llm/<workbook_slug>/pass1_recipe_eligibility_diagnostics.json`
- Manifest rows now include:
  - `eligibility_status`, `eligibility_action`, `eligibility_score`, `eligibility_score_components`, `eligibility_reasons`

## Evidence

- Parsing tests: `tests/parsing/test_canonical_line_roles.py` (new do-no-harm arbitration tests + outside-span policy assertions).
- LLM tests: `tests/llm/test_codex_farm_orchestrator.py` (new pass1 eligibility proceed/clamp/drop coverage).
- Label Studio projection test: `tests/labelstudio/test_canonical_line_projection.py` confirms do-no-harm artifact pointers are propagated when present.
