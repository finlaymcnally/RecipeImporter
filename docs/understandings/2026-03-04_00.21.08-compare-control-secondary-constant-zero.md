summary: "Compare/control secondary side-metrics should drop constant fields so group summaries do not show misleading 0.000 values."
read_when:
  - "When Compare & Control shows benchmark timing side-metrics as 0.000 for every group."
  - "When tuning secondary metric selection for compare/control categorical summaries."
---

# Discovery

- Compare/control secondary metric selection accepted fields with at least two numeric values, even when the values were constant across all visible rows.
- For timing-heavy preferred fields (for example `benchmark_total_seconds`), this could surface side-metrics where every group printed `0.000`, adding noise and looking like a rendering bug.

# Resolution

- Secondary metric selection now requires numeric variation (`max - min > 1e-12`) in both dashboard JS and backend compare-control engine.
- Constant fields are excluded from `secondary_fields`; varying fields (for example token totals) still render in `Group outcome means` summaries.
