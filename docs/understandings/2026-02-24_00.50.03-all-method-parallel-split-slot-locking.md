# All-Method Parallel Split-Slot Locking

Interactive all-method benchmark now runs configs through a bounded outer queue instead of a strict serial loop. The queue keeps up to 4 config pipelines in flight and refills immediately on each completion.

The heavy split-conversion phase is independently gated to 2 concurrent configs via shared split-slot file locks (`.split_phase_slots` under each source all-method root). Configs can still overlap prep/post work while waiting for split slots.

Because parallel configs append benchmark rows to the same history CSV, `append_history_csv(...)` and `append_benchmark_csv(...)` now hold an inter-process file lock around schema checks, header decisions, and row appends to prevent header duplication and partial writes.
