# Knowledge Schema Nested Required Gap

The benchmark knowledge stage failed before any shard completed because `recipe.knowledge.v1.output.schema.json` added nested property `rc` under `r[*].d[*]` but did not add it to that nested object's `required` list.

The existing pack test only enforced the "all keys in `properties` must be in `required`" rule at the schema top level, so local tests stayed green while the live structured-output validator rejected the nested object shape.

The durable fix is:

- keep the nested schema itself strict (`required: ["i", "c", "rc"]` here)
- keep a recursive test over shipped schemas so future nested additions cannot bypass pack validation
