# Detached Oracle Response Lives In Log And Console Should Stay Short

- The detached Oracle launcher currently writes a short preamble plus the live Oracle stdout/stderr stream into `upload_bundle_v1/.oracle_upload_runs/<timestamp>/oracle_upload.log`.
- If Oracle succeeds, that same log contains the final model response; there is no separate response artifact today.
- If Oracle fails before the model replies, the log only contains the Oracle/browser error, so "where is the answer?" is really "there is no answer for this failed run."
- The terminal summary should therefore point directly at `oracle_upload.log` and keep the rest of the output short; the full command, shard details, and metadata already live in `oracle_upload.log` and `oracle_upload.json`.
