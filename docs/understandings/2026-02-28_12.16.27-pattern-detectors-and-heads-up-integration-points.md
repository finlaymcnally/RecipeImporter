# Pattern Detectors and Heads Up Integration Points

- EPUB already skips explicit nav/TOC documents and nav-tag content (`cookimport/plugins/epub.py`, `cookimport/parsing/epub_html_normalize.py`), but there is no broad TOC-pattern suppression for plain text list pages.
- EPUB extraction health already computes duplicate-line rates and warning keys (`cookimport/parsing/epub_health.py`) but does not perform dedupe or candidate-level suppression.
- Candidate gating exists in both EPUB/PDF importers via `score_recipe_likeness(...)` + `recipe_gate_action(...)`; rejected candidates are converted into `non_recipe_blocks`, so pattern penalties can be applied without data loss.
- Best insertion point for deterministic pattern handling is after block extraction and before `_detect_candidates(...)` in EPUB/PDF importers, plus optional score penalties/reasons in `cookimport/core/scoring.py`.
- CodexFarm runner already captures and persists `heads_up_*` telemetry (`cookimport/llm/codex_farm_runner.py` and manifest/report `process_runs`), so downstream advice loops are available once explicit input hint fields are added to pass contracts/prompts.
