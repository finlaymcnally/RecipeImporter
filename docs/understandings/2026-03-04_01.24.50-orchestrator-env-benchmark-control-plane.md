# Discovery

`orchestrator/env` in current docs is shorthand for environment-override logic inside `cookimport/llm/codex_farm_orchestrator.py`, not a separate module/file.

# Current control plane

- Codex pass3 pass2-ok skip policy is settings-first via `RunSettings.codex_farm_pass3_skip_pass2_ok`, with env override `COOKIMPORT_CODEX_FARM_PASS3_SKIP_PASS2_OK`.
- Other benchmark runtime controls (parallelism cap, executor mode, WSL safety guard, cache roots, ETA poll interval) live in `cookimport/bench/quality_runner.py` env vars, not in the LLM orchestrator.
- Bench docs explicitly call out this shift: QualitySuite tuning should prefer run-settings patches over orchestrator/env edits.
