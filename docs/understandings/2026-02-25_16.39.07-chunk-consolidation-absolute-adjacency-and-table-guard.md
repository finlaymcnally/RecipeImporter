# Chunk consolidation: absolute adjacency + table guard

- `KnowledgeChunk.block_ids` are relative sequence indexes (not guaranteed absolute source indexes). Pass4 job bundling depends on this relative contract (`tests/llm/test_knowledge_job_bundles.py`).
- To safely enforce book adjacency without breaking pass4, chunking now stores `provenance.absolute_block_range` derived from per-block `features.source_block_index` when available.
- Adjacent-chunk consolidation checks inclusive adjacency (`left_end + 1 == right_start`) using that absolute range.
- Table chunks (`provenance.table_ids`) must never merge with other chunks. Guard this in both merge phases:
  - `merge_small_chunks`
  - `consolidate_adjacent_knowledge_chunks`
