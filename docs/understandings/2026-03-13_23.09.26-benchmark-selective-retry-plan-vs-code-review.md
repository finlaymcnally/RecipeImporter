# Benchmark Selective Retry Plan-vs-Code Review

Current code matches the OG plan on the core behavior: benchmark-only pass2/pass3 retry, missing-bundle-only reruns, no overwrite of successful outputs, detailed `llm_manifest.json` retry payloads, concise benchmark summaries, and a direct benchmark-root `llm_manifest_json` link.

The review originally found two plan-vs-code mismatches, and both were fixed in the same follow-up:

- `cookimport/cli.py` now exposes `--codex-farm-benchmark-selective-retry` and `--codex-farm-benchmark-selective-retry-max-attempts` in `labelstudio-benchmark --help`, and `tests/cli/test_cli_llm_flags.py` now asserts that visible help surface.
- `cookimport/llm/codex_farm_orchestrator.py::_selective_retry_eligible_process_run(...)` no longer adds its own non-empty failure-category precondition before delegating to `cookimport/llm/codex_farm_runner.py::_is_recoverable_no_last_agent_message_failure(...)`, so the retry gate now follows the same helper semantics as the runner.
