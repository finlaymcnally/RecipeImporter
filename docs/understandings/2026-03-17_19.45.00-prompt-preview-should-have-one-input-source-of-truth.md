Predictive prompt preview should rebuild recipe and knowledge shard inputs from the selected deterministic or vanilla processed artifact, not opportunistically reuse any old `raw/llm/.../{recipe_correction,knowledge}/in/*.json` files that happen to be lying around.

- Reusing those staged input files gives prompt preview a second source of truth and makes the estimate depend on stale Codex-era leftovers.
- After the predictive-only cleanup, prompt preview now has one behavior: resolve a predictive-safe processed run, rebuild the shard payloads locally, and estimate from those reconstructed prompts.
