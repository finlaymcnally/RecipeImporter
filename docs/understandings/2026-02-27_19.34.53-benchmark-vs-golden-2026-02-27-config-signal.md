# Discovery

- Run analyzed: `data/golden/benchmark-vs-golden/2026-02-27_17.54.41/all-method-benchmark`.
- Global shape: `91` planned configs, `82` successful, wall `4977.99s`; `thefoodlabCUTDOWN.epub` consumed `4602.45s` (~92.5% of wall).
- Quality signal is material by extractor family:
  - `unstructured` variants are the only consistently competitive family.
  - `markdown` / `markitdown` are usually much worse (including near-zero F1 on `DinnerFor2CUTDOWN`).
  - `beautifulsoup` is usually below the best unstructured variants.
- Within `unstructured` v1 top tier, several configs tie on F1 for most sources; runtime still differs materially (often ~1.3x to ~1.7x faster for the best tied config).
- `unstructured` v2 variants win on `DinnerFor2CUTDOWN` but all v2 (plus non-unstructured extractors) failed on `thefoodlabCUTDOWN` in this run due process-pool worker termination errors.
- Practical takeaway from this run: pick from stable unstructured v1 configs (especially faster tied variants) unless the v2 crash on large books is resolved.
