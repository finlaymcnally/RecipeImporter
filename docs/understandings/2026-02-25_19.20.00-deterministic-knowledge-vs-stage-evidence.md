# Discovery: deterministic knowledge path and stage evidence gap

There is a deterministic pre-LLM knowledge pipeline:
- plugins emit `non_recipe_blocks`,
- chunking builds `KnowledgeChunk` objects with lane assignment and highlight extraction,
- pass4 LLM knowledge harvest consumes those deterministic chunks as job hints/context.

Gap found:
- Stage evidence builder (`stage_block_predictions.json`) previously added `KNOWLEDGE` only from pass4 snippets (`knowledge/.../snippets.jsonl`).
- With `llm_knowledge_pipeline=off`, stage evidence produced `KNOWLEDGE=0` even when deterministic chunk lanes identified knowledge content.

Rule now:
- Stage block predictions prefer pass4 snippet provenance when present, and fall back to deterministic `ChunkLane.KNOWLEDGE` chunk->block mapping when snippets are absent.
