## Discovery

- CLI flow-specific wrappers were applying line-role inflight defaults, which risks drift when new orchestration paths are added.
- `generate_pred_run_artifacts(...)` is the common prediction-generation seam used by benchmark and Label Studio import flows, and already receives split-phase gating context.

## Implementation choice

- Keep parser default/env behavior intact in `label_atomic_lines(...)`.
- Add optional explicit `codex_max_inflight` override parameter.
- Resolve shared default in ingest from split context:
  - non-split => `8`
  - split-gated => `4`
  - explicit `COOKIMPORT_LINE_ROLE_CODEX_MAX_INFLIGHT` env => no ingest override (env wins).

## Outcome

- Single-vs-multi default behavior now propagates automatically across benchmark/import prediction-generation paths without per-flow CLI wiring.
