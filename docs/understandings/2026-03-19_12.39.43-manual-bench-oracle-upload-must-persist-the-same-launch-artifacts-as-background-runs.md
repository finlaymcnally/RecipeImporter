Manual `cookimport bench oracle-upload ...` used a separate foreground code path from the benchmark auto-background uploader.

That foreground path could complete a real Oracle browser session successfully, but it did not write `upload_bundle_v1/.oracle_upload_runs/<timestamp>/` or persist `oracle_upload.log`, `oracle_upload.json`, and `oracle_upload_status.json`.

The fix is to make foreground browser uploads mirror the background artifact contract:
- create a launch dir under `.oracle_upload_runs/`
- keep any sharded staging files there
- write the local Oracle log mirror there
- persist recovered session id and conversation URL there
