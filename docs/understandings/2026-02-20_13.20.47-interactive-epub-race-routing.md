# Interactive EPUB Race Routing

- The main interactive loop (`_interactive_mode`) builds menu actions from top-level `data/input` discovery each refresh.
- `epub_race` belongs in that menu (not Settings) and should appear only when at least one top-level `.epub` file is present.
- To keep behavior parity, interactive race should call the existing `race_epub_extractors(...)` command function with explicit args instead of duplicating scoring/report logic.
- Interactive prompts should only collect path/candidate/output/overwrite choices, then return to the main menu regardless of command success.
