# Authoritative LLM Contract

The right architectural seam in this repo is not “let the model edit an output file in place.” The active Codex-backed surfaces work best when the program writes an immutable input payload, the model returns a structured proposal for only its owned ids, deterministic code validates exact ownership and coverage, and only then deterministic promotion writes the durable outputs.

Recipe and knowledge already mostly follow that contract. Line-role also follows it now at runtime, but some docs and artifact naming still make it look more compatibility-heavy than it really is. The next cleanup should remove those misleading compatibility surfaces rather than adding more of them.
