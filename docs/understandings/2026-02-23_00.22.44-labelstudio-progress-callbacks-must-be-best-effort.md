---
summary: "Label Studio ingest should treat progress callbacks as best-effort telemetry, not control flow."
read_when:
  - "When tracing Label Studio import failures that mention status/spinner callback errors"
  - "When wiring progress callbacks through ingestion/importer convert paths"
---

`generate_pred_run_artifacts(...)` historically forwarded the caller's `progress_callback` directly into importer `convert(...)`.
If that callback raised (for example from CLI spinner state bugs), EPUB conversion failed and surfaced as a conversion error even though parsing itself was fine.

Hidden rule: progress callbacks are telemetry only. They must be wrapped so callback exceptions are logged and ignored, and core extraction/task generation continues.
