# Strict RunSettings fixtures and split-merge status

## Discovery

- `RunSettings.from_dict(...)` now enforces live schema keys only. Test fixtures that still pass retired compatibility keys (`codex_execution_policy`, `instruction_step_segmentation_policy`, `section_detector_backend`, `codex_farm_pipeline_pass5_tags`, etc.) fail immediately.
- Split-merge progress output is hybrid:
  - top-level merge milestones are emitted as `merge phase X/Y: ...`,
  - session-level callbacks from `execute_stage_import_session_from_result(...)` are plain status lines (for example `Generating knowledge chunks...`, `Writing outputs...`).
- `_merge_split_jobs(...)` must not reinitialize `OutputStats` after writing merged `raw/.../full_text.json`; doing so undercounts raw artifacts in `report.outputStats` versus a fresh directory walk.

## Durable rule

- For tests and helper fixtures, build `RunSettings` payloads from live model fields only (or project mixed payloads before `from_dict`).
- For split-merge status tests, assert merge-phase rows separately from plain forwarded session messages.
- Keep one `OutputStats` accumulator through split merge so merged full-text and moved raw artifacts both land in `outputStats`.
