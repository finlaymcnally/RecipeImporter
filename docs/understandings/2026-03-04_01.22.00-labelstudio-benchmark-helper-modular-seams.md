# Discovery

`tests/labelstudio/test_labelstudio_benchmark_helpers.py` had become a mixed test surface (interactive/UI routing, benchmark payload contracts, scheduler algorithms, single-profile orchestration). Focused iteration was difficult because one file implied one execution profile.

# Resolution

- Split by concern into dedicated files:
  - `..._eval_payload.py`
  - `..._scheduler.py`
  - `..._single_profile.py`
- Keep shared helper/import state centralized in the base module and selectively inject non-test globals in split modules.
- Keep only eval/scheduler split modules in the `slow` slice; keep single-profile and general helper flows in fast labelstudio runs.
