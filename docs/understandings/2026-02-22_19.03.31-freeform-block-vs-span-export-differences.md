# Discovery

Comparing `data/golden/saltfatacidheatcutdown_aiv1_block_based` vs `data/golden/saltfatacidheatcutdown_aiv2_freeform_fr` confirms the divergence is expected runtime behavior from `prelabel_granularity`, not an export pipeline mismatch.

## Mechanic differences (code)

- Interactive style mapping:
  - `actual freeform` -> `prelabel_granularity=span`
  - `legacy, block based` -> `prelabel_granularity=block`
  - Source: `cookimport/cli.py` menu wiring.
- Prompt template switch:
  - `block` uses `llm_pipelines/prompts/freeform-prelabel-full.prompt.md` (asks for one label per block).
  - `span` uses `llm_pipelines/prompts/freeform-prelabel-span.prompt.md` (asks for selective quote/offset spans).
  - Source: `cookimport/labelstudio/prelabel.py::_build_prompt`.
- Parsing + span construction switch:
  - `block`: parse `{block_index,label}` and emit full block range `[segment_start,segment_end]`.
  - `span`: parse quote/offset selections, resolve quotes, allow sub-block ranges, drop ambiguous/unresolvable items.
  - Source: `cookimport/labelstudio/prelabel.py` parse/build helpers.

## Observed output differences in this pair

- Same segmentation inputs in both exports: 42 segments and 1471 unique source blocks in segment manifests.
- Span rows:
  - block run: 1635 rows
  - span run: 1355 rows
- Geometry:
  - block run: 100% full-block spans, 0% sub-block spans
  - span run: 92.2% full-block spans, 7.8% sub-block spans (106 rows)
- Coverage:
  - block run touches 1440 unique blocks
  - span run touches 1201 unique blocks
- Label mix shifted:
  - block run overuses `OTHER` and `KNOWLEDGE`
  - span run increases `RECIPE_TITLE`, `INSTRUCTION_LINE`, `INGREDIENT_LINE`, and introduces `TIME_LINE`

## Why this happens

- `block` mode is forced one-label-per-block, so nearly every block gets a label, and each label covers the full block.
- `span` mode is selective and confidence-driven; unresolved/ambiguous quotes are dropped, and unclear content can be left unlabeled.
- Both runs still use the same task segmentation (`segment_blocks`, `segment_overlap`), so differences come from prelabel interpretation, not segment construction.
