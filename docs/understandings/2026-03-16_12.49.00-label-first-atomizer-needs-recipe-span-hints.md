summary: "Label-first Stage 2 still needs recipe-span hints during atomic line labeling or outside-span safeguards will erase real recipe structure."
read_when:
  - "When authoritative Stage 2 labels suddenly collapse recipe titles/howto/instructions to `OTHER`"
  - "When editing `_atomize_archive_blocks(...)` in `label_source_of_truth.py`"
---

`build_label_first_compatibility_result(...)` currently runs canonical line-role labeling before recipe spans are regrouped from labels, so the atomizer still needs an initial `within_recipe_span` hint to avoid over-downgrading recipe-ish lines.

The regression version of `_atomize_archive_blocks(...)` set `within_recipe_span=False` for every archive block. That made the line-role layer treat the whole book as outside recipe spans, and the outside-span safety rules in `canonical_line_roles.py` downgraded many legitimate labels to `OTHER`, especially:
- `RECIPE_TITLE`
- `HOWTO_SECTION`
- `INSTRUCTION_LINE`
- `RECIPE_VARIANT`

On `saltfatacidheatcutdown`, that dropped the recovered line-role artifact counts from the historical shape:
- baseline good run: `RECIPE_TITLE=27`, `INSTRUCTION_LINE=64`, `HOWTO_SECTION=28`
- broken label-first reuse: `RECIPE_TITLE=4`, `INSTRUCTION_LINE=35`, `HOWTO_SECTION=0`

The minimal safe fix is to derive provisional recipe ranges from existing recipe provenance (`start_block` / `end_block`, with line-index fallback) and feed those ranges into `_atomize_archive_blocks(...)` so canonical labeling gets the same span hint the old benchmark candidate builder used.
