## Discovery

- `_run_with_progress_status` live ticker crashed because `_run_with_progress_status` used
  `len(worker_dashboard_adapter.snapshot_workers()[0])`, but `snapshot_workers()[0]` already returns
  an integer worker count. Replacing this with direct assignment restores live spinner ticking, ETA and
  elapsed updates.
- Stage tests in `tests/cli/test_stage_progress_dashboard.py` were patching `cli.stage_one_file`, but stage
  workers are imported inside `cli.stage(...)` from `cookimport.cli_worker`. Monkeypatching `cli_worker.stage_one_file`
  (and `stage_pdf_job`/`_merge_pdf_jobs` for split flows) is required for reliable CI isolation.
- Added merge-phase coverage for split-worker snapshots in `tests/cli/test_stage_progress_dashboard.py` so parity claims
  match runtime assertions around merge messaging and worker-line updates.
