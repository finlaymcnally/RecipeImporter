# Line-Role Cost Was Mostly Row Shape

- Once transport wrapping was fixed, line-role was still expensive because every atomic line repeated long keys like `deterministic_label`, `escalation_reasons`, `previous_line`, `current_line`, and `next_line`.
- `within_recipe_span` was also repeated per row even though recipe membership is batch-level range metadata.
- Neighbor text on every in-recipe row was especially wasteful; only escalated rows really need it.
- The cheap fix is to keep the strict output contract but compact the input rows:
  - move recipe membership into batch-level atomic ranges
  - use short label/reason code legends once per batch
  - serialize rows as compact arrays
  - include neighbor context only when escalation reasons are present

- On the `saltfatacidheatcutdown` codex-shaped preview, that compaction plus a larger shared batch size cut line-role from `15` prompts / `~290,891` estimated total tokens to `8` prompts / `~174,884`.
