# Salt Fat Acid Heat Regression Note

Run `data/golden/benchmark-vs-golden/2026-03-18_21.17.59` regressed versus vanilla because Codex classification got worse, not because the scorer changed.

Key evidence:

- `single_offline_summary.md` and `codex_vs_vanilla_comparison.json` show `strict_accuracy` fell from `0.704638` to `0.607094` and the comparison attributes the loss to `classification_signal_only`.
- `codexfarm/eval_report.json` shows a large new `HOWTO_SECTION` error pattern that vanilla did not have:
  - gold `KNOWLEDGE` -> predicted `HOWTO_SECTION`: `109`
  - gold `OTHER` -> predicted `HOWTO_SECTION`: `48`
- `line_role_flips_vs_baseline.jsonl` shows many harmful flips into `HOWTO_SECTION`, especially:
  - baseline `KNOWLEDGE` -> candidate `HOWTO_SECTION` while gold stayed `KNOWLEDGE`: `92`
  - baseline `OTHER` -> candidate `HOWTO_SECTION` while gold stayed `OTHER`: `26`
- Sample bad flips are book headings or prose like `Salt and Pepper`, `Breaking and Fixing Emulsions`, `Balancing Fat`, `Cooking Acids`, `Starches`, `Stewing and Braising`, and `Coddling and Poaching`.
- `wrong_label_lines.sample.jsonl` also shows some actual recipe material became `KNOWLEDGE` after dropping out of recipe spans (`line_role_match_kind=unmatched`, `within_recipe_span=null`), for example recipe variants and instruction lines around the slaw recipe section.
- `line-role-pipeline/telemetry_summary.json` says the scored artifact used `authority_mode=knowledge_refined_final` and changed `195` blocks, so the final knowledge-refined projection materially altered the line-role result.

Working conclusion:

- Main failure: the line-role / final-authority path treated many cookbook section headings as `HOWTO_SECTION` instead of `KNOWLEDGE` or `OTHER`.
- Secondary failure: some recipe-span misses caused real recipe lines to be evaluated as non-recipe `KNOWLEDGE`.
