# Codex model cache vs runtime access

- Local Codex model menus come from cached metadata (`models_cache.json`), which may list models that are not currently usable for the active login/account mode.
- Runtime provider access is only confirmed when a real `codex exec` call runs against the selected model.
- Freeform prelabel should run one preflight probe before task loops so invalid model/account combinations fail once, with explicit provider error text.
- Codex JSON event `turn.failed` carries actionable provider detail and should be surfaced directly instead of being reduced to a generic parse failure.
