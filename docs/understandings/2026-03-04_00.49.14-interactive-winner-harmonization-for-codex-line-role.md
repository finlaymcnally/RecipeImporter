summary: "Interactive run-settings resolution now harmonizes codex/line-role/atomic knobs even when loading a saved quality-suite winner profile."
read_when:
  - "When interactive benchmark/import unexpectedly runs with llm codex enabled but line-role/atomic disabled."
  - "When changing choose_run_settings top-tier default behavior."
---

# Discovery

- The interactive resolver preferred saved quality-suite winner settings first.
- Existing winner snapshots can be stale (`llm_recipe_pipeline=off`, `line_role_pipeline=off`, `atomic_block_splitter=off`).
- That stale winner payload bypassed the built-in top-tier fallback and could silently reintroduce off/off behavior.

# Resolution

- `choose_run_settings(...)` now applies a post-resolution harmonization pass to force:
  - `llm_recipe_pipeline=codex-farm-3pass-v1`
  - `line_role_pipeline=codex-line-role-v1`
  - `atomic_block_splitter=atomic-v1`
- This harmonization applies regardless of whether settings came from saved winner or built-in fallback.
