  - "When changing benchmark progress rendering for parallel selected/all matched book runs."

# Discovery

`Single config, selected matched sets` runs selected books in parallel (`ThreadPoolExecutor`), and each worker calls `labelstudio_benchmark(...)` directly.

`labelstudio_benchmark(...)` uses `_run_with_progress_status(...)` for prediction/evaluation phases, which starts `console.status(...)` live spinners unless spinner suppression is enabled.

Rich allows only one live display per console, so concurrent worker spinners raise `LiveError: Only one live display may be active at once`.

# Practical Impact

This is a renderer/concurrency conflict, not a parsing/data correctness error. It can still fail per-variant benchmark runs if the exception bubbles out of `labelstudio_benchmark(...)` inside a worker.

Current operator workaround: run with plain progress (`COOKIMPORT_PLAIN_PROGRESS=1`) or avoid parallel selected-book runs.
