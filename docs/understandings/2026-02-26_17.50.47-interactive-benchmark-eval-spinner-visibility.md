# Discovery

- In `labelstudio_benchmark`, prediction generation ran under `_run_with_progress_status(...)`, but evaluation ran directly afterward with no spinner/status wrapper.
- Canonical-text evaluation can run for minutes on large books (for example SeaAndSmoke), so the terminal appeared idle even while CPU remained high and work continued.

# Durable Rule

- Interactive (non-suppressed) benchmark runs should keep a visible status spinner during the evaluation phase, with a clear message that includes eval mode.
