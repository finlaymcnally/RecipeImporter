summary: Why `2026-03-15_15.37.33` appears in Per-Label Breakdown but can disappear from Previous Runs.
---

`data/golden/benchmark-vs-golden/2026-03-15_15.37.33/single-offline-benchmark/saltfatacidheatcutdown/line_role_only` is present in both `.history/performance_history.csv` and `.history/dashboard/assets/dashboard_data.json`, so collection is not the problem.

The mismatch comes from frontend scope:
- `Per-Label Breakdown` reads `filteredBenchmarks()` directly and then picks the latest run-group with per-label data.
- `Previous Runs` trend/table read `currentPreviousRunsFilterResult()`, which applies quick filters and browser/program UI state first.

This run is a hybrid single-offline variant rooted at `.../line_role_only`, not an official paired `vanilla` or `codexfarm` run. The `Official benchmarks only` quick filter keeps only:
- `vanilla` + deterministic profile
- `codexfarm` + full-stack profile

So if that quick filter is on anywhere in UI state, this run is hidden from Previous Runs while still showing up in Per-Label Breakdown.
