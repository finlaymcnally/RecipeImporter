# Background Oracle Upload Needs Persistent Staging

- Browser-mode Oracle uploads can require temporary shard files when `upload_bundle_payload.jsonl` exceeds Oracle's 1 MB per-file input cap.
- The synchronous upload path can use `TemporaryDirectory` because the Oracle subprocess starts and finishes before that directory is cleaned up.
- A detached/background upload cannot rely on `TemporaryDirectory`: the parent benchmark process would return, clean the directory, and leave Oracle holding paths that no longer exist.
- The durable fix is to create a per-launch directory under `upload_bundle_v1/.oracle_upload_runs/<timestamp>/`, stage shard files there when needed, and write `oracle_upload.log` plus `oracle_upload.json` so the detached process has stable inputs and the operator has somewhere to inspect progress.
