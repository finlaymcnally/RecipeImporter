# Final Legacy Purge Clusters

The remaining destructive cleanup after the first purge batch was concentrated in compatibility helpers around the live code, not in the live recipe-correction algorithm itself.

The runner still carried two historical allowances: parsing old stderr-only progress lines and accepting empty or non-JSON stdout even when `codex-farm process --json` was requested. Removing those branches makes the current subprocess contract explicit: structured progress events are optional-with-retry, but structured JSON output is mandatory.

The upload-bundle path had already been refactored to semantic reviewer-facing stage names, but its adapter internals were still counting `observed_pass2_call_count` and `observed_pass3_call_count` and exposing `UploadBundleSourceModel.as_context_dict()` only to keep an older script shape alive. Those were dead compatibility projections, so the clean cut is to rename the counters semantically and have callers use the typed model directly.

The old Pro3 transport replay tooling turned out to be fully detached from live code. Once `cookimport/llm/codex_farm_transport.py` and `cookimport/llm/evidence_normalizer.py` were deleted, the remaining script and fixture JSONs had no live consumer and could be removed instead of rebuilt.
