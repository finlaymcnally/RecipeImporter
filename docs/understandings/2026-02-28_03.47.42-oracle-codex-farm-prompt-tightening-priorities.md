## Discovery

- Oracle review reported the current pass prompts are valid but under-specified for deterministic behavior: tie-breaks, missing-field policy, evidence strictness, and extra-key/type constraints are not explicit enough.
- Highest-impact prompt tightening was identified in pass2 (`recipe.schemaorg.v1`) and pass3 (`recipe.final.v1`), then pass1 (`recipe.chunking.v1`), with pass4/pass5 as medium impact.
- Cross-pass recommendation: add explicit "omit rather than guess" language, stable ordering rules, exact evidence requirements, and "no extra properties" constraints while keeping existing schema contracts.

## Why this matters

- Pass2 and pass3 feed final draft payload quality; small wording drift there can propagate into benchmark score variance.
- Pass1 boundary ambiguity causes non-deterministic span width differences that affect downstream extraction and scoring.
- Pass4/pass5 improve consistency mostly via stricter evidence and confidence wording, but are lower leverage for core recipe extraction.
