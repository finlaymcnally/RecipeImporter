# Discovery

Runtime sampling showed many files in `_SLOW_FILES` were no longer slow (often <3s). The main dashboard suite had one expensive browser pixel-overflow harness mixed into otherwise fast renderer tests.

# Resolution

- Move the pixel-overflow rerender harness test to `tests/analytics/test_stats_dashboard_slow.py`.
- Keep fast dashboard coverage in `tests/analytics/test_stats_dashboard.py`.
- Recalibrate `_SLOW_FILES` to only genuinely expensive files so `-m slow` is focused and useful.
