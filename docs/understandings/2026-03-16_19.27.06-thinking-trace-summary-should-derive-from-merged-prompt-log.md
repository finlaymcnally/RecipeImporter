The prompt export already normalizes all CodexFarm interactions into `prompts/full_prompt_log.jsonl`.

That merged log is the right source for new trace-summary artifacts because it already contains:
- recipe and knowledge trace payloads resolved from `.codex-farm-traces`
- copied line-role rows, including the explicit case where no trace payload exists
- final benchmark-local stage naming and call ids

So the low-risk trace-summary fix is:
- build reviewer-facing trace summaries from `full_prompt_log.jsonl` after the line-role merge step
- write those summary files into the same `prompts/` folder
- expose the new files through run-manifest artifact keys when present

This avoids a second reconstruction path and keeps the trace summary consistent with the exact prompt bundle the reviewer sees.
