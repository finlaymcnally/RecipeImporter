# Oracle Browser Answer Capture Can Truncate Multi-Block ChatGPT Responses To The First Markdown Block

- The missing turn-2 handoff was not only a recipeimport orchestration issue. For session `you-are-reviewing-a-benchmark-290`, ChatGPT showed a full four-section answer in the browser conversation, but Oracle only persisted a short two-sentence summary into `output.log` and `oracle_upload.log`.
- The most plausible code-level cause was Oracle's assistant DOM extractor selecting `querySelector('.markdown')` under the assistant turn, which can capture only the first markdown/content block when ChatGPT renders a response as multiple blocks.
- The Oracle workspace fix is to aggregate all top-level content blocks inside the assistant turn before persisting `answerText` / `answerMarkdown`. Recipeimport can only auto-launch turn 2 from whatever Oracle actually saved.
