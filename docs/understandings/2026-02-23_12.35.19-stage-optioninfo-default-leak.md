# Discovery

`cookimport.cli.stage(...)` is invoked from both Typer CLI dispatch and direct Python call paths (`_interactive_mode`, `entrypoint.main`, tests).

When direct callers omit some kwargs, parameters declared as `param: T = typer.Option(...)` keep `OptionInfo` sentinel objects at runtime. Normalizers expecting strings/ints then fail (for example `.strip()` on `OptionInfo`).

# Contract

- `stage(...)` must unwrap `OptionInfo` inputs to plain defaults before normalization and run-settings assembly.
- Interactive import must forward the complete selected run-settings payload into `stage(...)`, including `llm_knowledge_pipeline`, pass4 pipeline id, and pass4 context blocks.
- `import`/`C3import` entrypoint shims must pass the expanded run-settings surface so saved settings can affect stage behavior.
