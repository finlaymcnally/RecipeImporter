# QualitySuite Seed Variation And Tournament Cache/Dedupe

- `bench quality-discover` previously recorded `seed` but selection did not use it, and with `--prefer-curated` it returned curated IDs only (no representative fill), which made multi-seed folds often duplicate the same selected set.
- Discovery now keeps curated IDs first and fills remaining `--max-targets` slots using seeded representative selection, so seeds can produce different fold suites.
- `quality_top_tier_tournament.py` now sets shared canonical/eval cache (`COOKIMPORT_ALL_METHOD_ALIGNMENT_CACHE_ROOT`) for fold `quality-run` calls, enabling cross-fold eval/cache reuse.
- Tournament also computes a suite signature per fold and skips duplicate-suite folds from execution and gate denominators to avoid paying for repeated evidence.
