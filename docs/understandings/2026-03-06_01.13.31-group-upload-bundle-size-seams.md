# Group upload bundle size seams

- The March 6 multi-book group bundle at `data/golden/benchmark-vs-golden/2026-03-06_00.44.16/single-profile-benchmark/upload_bundle_v1/upload_bundle_payload.jsonl` was about `77.8 MB`, even though group high-level mode is supposed to target about `40 MB`.

- The dominant bytes were raw prompt logs that high-level selection still treats as required:
  - `04_thefoodlabcutdown/prompts/full_prompt_log.jsonl`: `24.0 MB`
  - `01_amatteroftastecutdown/prompts/full_prompt_log.jsonl`: `23.9 MB`
  - `03_seaandsmokecutdown/prompts/full_prompt_log.jsonl`: `13.7 MB`
  - `02_saltfatacidheatcutdown/prompts/full_prompt_log.jsonl`: `8.2 MB`

- Those four files alone already exceed the intended budget. The code path is in `scripts/benchmark_cutdown_for_external_ai.py::_upload_bundle_select_high_level_artifact_paths(...)`, which appends each resolved full prompt log with `required=True`, so the artifact-budget check never blocks them.

- The next largest contributors were derived full-context trace artifacts, especially `preprocess_trace_failures.jsonl` and `wrong_label_lines.with_context.full.jsonl`. Those are smaller than the prompt logs but still add several more megabytes after the prompt logs have already broken the budget.

- The current group budget logic only budgets:
  - selected source artifacts by raw file size,
  - sampled wrong-line rows inside `group_high_level_packet`.

- The current logic does **not** enforce a final ceiling on the serialized output payload. After selected artifacts and derived virtual payload rows are written, there is no final pass that trims optional sections until the resulting `upload_bundle_payload.jsonl` is within target.

- Practical implication: reducing `BENCHMARK_GROUP_UPLOAD_BUNDLE_TARGET_BYTES` alone will not solve the issue. The builder needs:
  - different inclusion rules for high-level bundles,
  - a hard serialized-size enforcement pass,
  - and a deliberate policy for deep prompt/trace evidence, likely via summaries and follow-up packets instead of raw inclusion in the first-pass group bundle.
