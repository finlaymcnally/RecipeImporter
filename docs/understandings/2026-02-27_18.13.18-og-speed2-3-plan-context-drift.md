## Discovery

- `docs/plans/OGplan/speed2-3.md` proposes a new `COOKIMPORT_CANONICAL_ALIGNMENT_BACKEND` abstraction and `dmp`/`edlib` backends.
- Current runtime instead uses `COOKIMPORT_BENCHMARK_SEQUENCE_MATCHER=auto|stdlib|cydifflib|cdifflib` through `cookimport/bench/sequence_matcher_select.py`, with evaluator telemetry and tests built around that selector.
- Canonical-text scoring still enforces legacy global alignment semantics and deprecates fast strategy requests; the speedups come from matcher implementation choice plus the canonical alignment cache.
- Real run artifacts under `data/golden/benchmark-vs-golden/2026-02-27_17.54.41` show `alignment_sequence_matcher_impl: "cydifflib"` in canonical eval telemetry, with repeated cache hits across configs.

## Practical implication

Treat `speed2-3` as historical problem framing and acceptance intent, not as an execution-ready plan. New speed work should be expressed against the current selector/cache architecture unless there is an explicit decision to reopen backend-family experimentation.
