# Run Settings Propagation Map

- Canonical source of truth is `cookimport/config/run_settings.py` (`RunSettings`, `summary()`, `stable_hash()`, UI metadata).
- Interactive selection path is `cookimport/cli_ui/run_settings_flow.py` + `cookimport/cli_ui/toggle_editor.py` (global/last/edit).
- Last-run snapshots are persisted via `cookimport/config/last_run_store.py` under `<output_dir>/.history/last_run_settings_{import|benchmark}.json`.
- Stage/report write path uses canonical settings in `cookimport/cli.py` and `cookimport/cli_worker.py`, writing `runConfig`, `runConfigHash`, `runConfigSummary` into each `*.excel_import_report.json` (`ConversionReport`).
- Benchmark prediction manifests from `cookimport/labelstudio/ingest.py` include `run_config`, `run_config_hash`, `run_config_summary` plus processed report linkage.
- CSV history (`cookimport/analytics/perf_report.py`) persists `run_config_hash`, `run_config_summary`, and `run_config_json` for stage and benchmark rows.
- Dashboard collector/renderer (`cookimport/analytics/dashboard_collect.py`, `cookimport/analytics/dashboard_render.py`) prefer CSV hash/summary, fallback to JSON/report manifest data, and display a stale warning when legacy report references are missing.
