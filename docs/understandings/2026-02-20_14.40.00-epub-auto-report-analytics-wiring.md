# EPUB Auto Metadata Wiring Map

Key discovery: extractor auto metadata has to be propagated in four layers, not just written as raw artifacts.

1. Stage orchestration (`cookimport/cli.py`) resolves `auto`, stores per-file payload, and passes it into worker + split merge write paths.
2. Report writers (`cookimport/cli_worker.py`, `cookimport/cli.py:_merge_split_jobs`, `cookimport/labelstudio/ingest.py:_write_processed_outputs`) must set:
   - `epubAutoSelection`
   - `epubAutoSelectedScore`
3. History CSV (`cookimport/analytics/perf_report.py`) must persist explicit columns:
   - `epub_extractor_requested`
   - `epub_extractor_effective`
   - `epub_auto_selected_score`
4. Dashboard (`cookimport/analytics/dashboard_schema.py`, `dashboard_collect.py`, `dashboard_render.py`) must map those CSV/report fields into stage records and render/filter directly, not infer from nested config blobs.

If any one layer is skipped, users can see extractor details in one surface but not the others.
