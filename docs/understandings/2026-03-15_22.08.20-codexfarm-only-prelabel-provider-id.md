- `cookimport/config/codex_decision.py` still treated `prelabel_provider=codex-cli` as valid Codex provider identity, which could preserve legacy IDs in decision paths even though runtime execution is CodexFarm-only.
- `cookimport/labelstudio/ingest.py` also normalized `codex-cli` into `codex-farm`, silently accepting the retired provider alias.
- Tightening both layers to accept only CodexFarm IDs (`codex-farm`, plus underscore normalization) keeps policy, runtime validation, and emitted metadata on one backend model.
