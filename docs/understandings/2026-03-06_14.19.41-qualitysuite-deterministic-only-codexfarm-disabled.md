# QualitySuite Deterministic-Only CodexFarm Disabled

- `bench quality-run` is now deterministic-only by contract.
- The CLI rejects `--include-codex-farm`, `--qualitysuite-codex-farm-confirmation`, `--codex-farm-model`, and `--codex-farm-thinking-effort` / `--codex-farm-reasoning-effort`.
- The runner also rejects requested settings that enable Codex Farm recipe/knowledge/tags surfaces, so Codex Farm cannot sneak in through `cookimport.json`, `--base-run-settings-file`, or experiment patches.
- Normalized benchmark baseline metadata still records `run_settings`, while `requested_run_settings` preserves raw requested provenance when it is deterministic-only.

Read when:
- Changing `bench quality-run` Codex Farm behavior
- Auditing whether QualitySuite can still execute Codex Farm-backed experiments through config files or experiment patches
