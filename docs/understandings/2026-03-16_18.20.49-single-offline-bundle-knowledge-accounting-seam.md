# Bundle Knowledge Accounting Seam

In the current post-refactor single-offline benchmark output, the run-level `codexfarm/prompt_budget_summary.json` may exist and show real knowledge calls even when the bundle reports:

- `analysis.knowledge.total_knowledge_call_count = 0`
- `prompt_budget_summary_status = "missing"`
- `preprocess_trace_failures_status = "missing_prediction_run"`

Why: the bundle helper resolves knowledge and prompt-budget artifacts through `_resolve_prediction_run_dir(...)` plus `prediction-run/prompt_budget_summary.json`. This run shape instead keeps the prompt budget summary at the benchmark run root and leaves `run_manifest.artifacts.pred_run_dir` unset, so the bundle misses it for accounting even though basename fallback can still make a locator row look populated.
