# Label Studio prelabel/fallback wiring map

- Freeform prelabel generation now happens inside `cookimport/labelstudio/ingest.py:generate_pred_run_artifacts(...)` after freeform tasks are sampled and before `label_studio_tasks.jsonl` is written.
- Offset contract is deterministic: `prelabel.py` converts `{block_index, label}` suggestions into spans using `task.data.source_map.blocks[*].segment_start/end` and the exact `task.data.segment_text`.
- Upload behavior in `run_labelstudio_import(...)`:
  - default `--prelabel-upload-as annotations`: tries inline `annotations` in import payload,
  - if Label Studio rejects inline annotations, it auto-falls back to:
    - upload plain tasks (annotations stripped),
    - fetch project tasks via API,
    - map deterministic task IDs (`segment_id`) to Label Studio `task.id`,
    - create annotations per task.
- Decorate behavior in `run_labelstudio_decorate(...)`:
  - selects latest annotation as base,
  - requests additive labels only,
  - writes a new merged annotation (original preserved),
  - marks annotations with `meta.cookimport_prelabel=true`, `mode=augment`, and `added_labels` for rerun idempotence.
