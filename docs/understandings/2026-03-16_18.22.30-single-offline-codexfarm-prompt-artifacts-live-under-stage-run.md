For `labelstudio-benchmark single_offline`, the benchmark eval root under `data/golden/.../codexfarm/` still points back to the real processed stage run via `run_manifest.json` artifact fields like `stage_run_dir` and `processed_output_run_dir`.

On the 2026-03-16 post-refactor run for `saltfatacidheatcutdown`, CodexFarm clearly executed:
- `raw/llm/<book>/recipe_correction/in` and `out` each contained 175 JSON payloads
- `raw/llm/<book>/knowledge/in` and `out` each contained 56 JSON payloads
- both `recipe_correction/out/.codex-farm-traces/` and `knowledge/out/.codex-farm-traces/` contained matching trace files

The missing piece is the reviewer-facing prompt export bundle (`prompts/` with `prompt_request_response_log.txt`, `full_prompt_log.jsonl`, etc.). That bundle was not present in either the benchmark eval root or the processed stage run root for this run, even though lower-level Codex input/output artifacts were present.

So: missing benchmark-level prompt export currently indicates an export/render regression, not necessarily a failed Codex run.
