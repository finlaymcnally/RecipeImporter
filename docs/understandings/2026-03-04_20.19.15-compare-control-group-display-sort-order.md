Discovery:
- Categorical compare analysis payloads are count-sorted by default; the best seam for user-friendly ordering is the render layer (table/group-option display), not analysis math.
- A shared display sort helper can keep group checkbox list and group-summary table consistent.
- Placeholder labels (e.g., `-`) should be pushed to the bottom for scanability; `ai_effort` needs semantic ordering (`low`, `medium`, `high`, `xhigh`) instead of lexical/count order.
