# External-AI preprocess trace join contract

For cutdown failure traces, the reliable join anchor is `wrong_label_lines.jsonl` line indices. Those indices are enriched from three existing sources in this order:

1. Canonical line context via `_build_line_prediction_view(...)` (gold/pred labels, previous/current/next line text, recipe-span region).
2. Prediction-run archive context via `run_manifest.artifacts.pred_run_dir` (or `prediction-run` fallback) and `prediction-run/extracted_archive.json` (raw block text + preprocess features).
3. Prompt-call context via `full_prompt_log.jsonl` (prefer per-recipe pass3/pass2 rows, include prompt candidate excerpt and warning buckets).

When any upstream dependency is missing, cutdown packaging should continue and record explicit status in `need_to_know_summary.json` `sample_counts` rather than failing (`missing_prediction_run`, `missing_extracted_archive`, `missing_full_prompt_log`).
