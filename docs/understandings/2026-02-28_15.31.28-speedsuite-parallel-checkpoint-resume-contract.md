# SpeedSuite Parallel + Resume Contract

- `run_speed_suite(...)` now resolves a deterministic `_SpeedTask` list and executes tasks either serially or via bounded `ThreadPoolExecutor` fanout.
- CLI `bench speed-run` now exposes:
  - `--max-parallel-tasks` for fixed task concurrency (`>=1`)
  - `--resume-run-dir` for reusing completed task snapshots in an existing run folder
- Auto parallel mode is active when `--max-parallel-tasks` is omitted and resolves to `min(total_tasks, cpu_count, 4)`.
- Crash-safe incremental artifacts are written during execution:
  - `checkpoint.json`
  - `summary.partial.json`
  - `report.partial.md`
  - `samples.partial.jsonl`
- Each completed sample phase now writes `speed_sample_result.json` under its existing scenario folder (`scenario_runs/<target>/<scenario>/<phase>/...`); resume mode reads these snapshots and skips already-completed tasks.
- Resume compatibility is validated against normalized run config fields (suite/targets/scenarios/warmups/repeats/run-settings hash/Codex flags) before task reuse to avoid mixed-config runs.
