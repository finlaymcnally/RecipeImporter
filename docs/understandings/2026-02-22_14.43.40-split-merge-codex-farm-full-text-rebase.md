# Understanding: split merge + codex-farm pass1 context

Codex-farm pass1 needs absolute block indices and surrounding context windows (`blocks_before`, `blocks_candidate`, `blocks_after`).

In split runs, worker payloads already include per-job `full_text.json` raw artifacts and local block indices. Without merge-time reconstruction, pass1 cannot build a single coherent full-block stream for the workbook.

Implementation takeaway:

- `_merge_split_jobs(...)` now loads each job's `.job_parts/<workbook>/job_<n>/raw/**/full_text.json`.
- It rebases each block `index` by cumulative prior-job block counts.
- It writes merged `raw/<importer>/<source_hash>/full_text.json` before codex-farm orchestration.
- The same offsets are applied to recipe/tip/topic location fields so provenance and pass1 boundaries share one coordinate system.

If codex-farm boundaries look shifted in split runs, first verify this merged `full_text.json` and offset application path.
