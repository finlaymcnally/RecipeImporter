# Benchmark CSV Backfill Resolution

- Benchmark CSV repair should inspect benchmark run roots from `performance_history.csv` row `run_dir`.
- Single eval runs usually carry manifest data at `run_dir/prediction-run/manifest.json` (or sometimes `run_dir/manifest.json`).
- Bench-suite runs can only recover recipe totals by summing `run_dir/per_item/*/pred_run/manifest.json`.
- Recovery precedence:
  - `recipe_count` from manifest
  - fallback `processed_report_path` -> report `totalRecipes`
- Backfill should only fill missing CSV fields (`recipes`, `report_path`, `file_name`) and avoid overwriting existing values.
