## Findings

- Repo size before cleanup: about `175G`.
- Primary contributor: `data/golden/bench/quality/tournaments` at about `143G`.
- Largest single run: `data/golden/bench/quality/tournaments/2026-02-28_20.25.10` at about `89G`.
- That run contained about `1,853,806` files and repeated `processed_output` trees across many config/round directories.

## Why It Happened

- `quality_runner` creates a `processed_output` tree for each benchmark execution branch (`cookimport/bench/quality_runner.py`).
- Processed output writer persists many per-recipe artifacts (drafts/sections/chunks/raw/predictions/reports) (`cookimport/labelstudio/ingest.py`).
- Tournament/race flows multiply runs by folds, experiments, rounds, and config variants, so file counts and storage grow quickly.

## Cleanup Outcome

- Removed `data/golden/bench/quality/tournaments`.
- Repo size after cleanup: about `38G`.
- `data` after cleanup: about `28G`.
- Remaining large non-data item: `.venv` about `10G` (ML/CUDA dependencies).
