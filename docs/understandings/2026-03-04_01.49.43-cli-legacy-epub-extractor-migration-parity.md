# Discovery

Two different migration policies existed:
- `RunSettings.from_dict(...)` migrated `epub_extractor=legacy` to `beautifulsoup`.
- Interactive/default config loader (`_load_settings()` in `cookimport/cli.py`) treated `legacy` as generic unsupported and forced `unstructured`.

This mismatch produced misleading warnings and inconsistent behavior across entry paths.

# Resolution

`cookimport/cli.py` now treats `legacy` as an explicit migration in both coercion and warning text:
- coercion: `legacy -> beautifulsoup`
- warning: migration-specific message, not generic unsupported fallback

A CLI regression test now locks this behavior.
