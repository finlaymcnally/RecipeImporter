For live benchmark runs, canonical line-role Codex artifacts are not written under `raw/llm/<workbook>/...`.

They live under the processed run root:
- `line-role-pipeline/prompts/prompt_*.txt`
- `line-role-pipeline/prompts/response_*.txt`
- `line-role-pipeline/prompts/parsed_*.json`
- `line-role-pipeline/prompts/parse_errors.json`
- `line-role-pipeline/telemetry_summary.json`

So a prompt exporter that only discovers `raw/llm/...` recipe and knowledge manifests will silently miss line-role even when Codex line-role definitely ran.

The low-risk export fix is:
- keep recipe/knowledge export on the existing prompt-artifact descriptor path
- supplement it with copied live line-role prompt files from `line-role-pipeline/prompts`
- append semantic `line_role` rows into `prompts/full_prompt_log.jsonl` using the live prompt text, response text / parsed JSON, and per-batch telemetry from `telemetry_summary.json`

This keeps the benchmark `codexfarm/prompts/` folder as the single place containing all CodexFarm interactions for that run.
