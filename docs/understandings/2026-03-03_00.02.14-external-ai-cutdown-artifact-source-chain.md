# External-AI cutdown artifact source chain

`scripts/benchmark_cutdown_for_external_ai.py` already emits the core causality contract (`full_prompt_log.jsonl`, changed-line diffs, per-span breakdowns, targeted prompt cases, prompt warning aggregate, projection trace). The two missing diagnosis surfaces are unsampled full-failure exports and preprocess-aware traces for failing lines.

The reliable source chain for new trace data is:

1. Benchmark eval run (`run_manifest.json`) -> `artifacts.pred_run_dir` (or `prediction-run` fallback).
2. Prediction run manifest (`prediction-run/run_manifest.json`) -> extracted archive artifacts.
3. Existing cutdown line/prompt loaders (`wrong_label_lines.jsonl`, `full_prompt_log.jsonl`, recipe-span reconstruction) for per-line failure anchoring and pass context.

Key implementation constraint: if prediction-run artifacts are missing, cutdown generation must still succeed and report explicit status in per-run `sample_counts` instead of failing.
