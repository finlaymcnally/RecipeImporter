---
summary: "Line-role prompt rows should be one ordered contiguous slice of the book, without inline previous/next line duplication."
read_when:
  - When changing `cookimport/llm/canonical_line_role_prompt.py` or line-role prompt shape
  - When line-role token costs look inflated relative to the source-book size
  - When deciding whether escalated rows should carry duplicate neighbor text inside the prompt
---

The active line-role prompt contract now treats each shard as one ordered contiguous slice of the book. Rows carry only `[atomic_index, label_code, reason_codes, current_line]`; previous/next line text is not duplicated inline anymore.

That keeps line-role aligned with the operator mental model of “process this slice once” instead of repeating in-slice text on escalated rows. When the batch is already contiguous and ordered, neighbor lines are already present elsewhere in the same prompt, so duplicating them is mostly wasted prompt budget.
