# Canonical line-role ingredient miss chain

- The canonical line-role benchmark path does **not** call `cookimport/parsing/ingredients.py::parse_ingredient_line`.
- Instead, prediction generation builds `AtomicLineCandidate` rows from extracted archive blocks in `cookimport/labelstudio/ingest.py::_build_line_role_candidates_from_archive`, which calls `cookimport/parsing/recipe_block_atomizer.py::atomize_blocks`.
- `ingredient_like` is assigned only by `recipe_block_atomizer.py::_is_ingredient_line`.
- If `_is_ingredient_line` misses a line, `_infer_candidate_labels(..., within_recipe_span=True)` returns `["OTHER", "KNOWLEDGE"]` with `["recipe_span_fallback"]` (plus `explicit_prose` when applicable).
- Then `cookimport/parsing/canonical_line_roles.py::_candidate_allowlist` removes `KNOWLEDGE` inside recipe spans unless explicit prose is allowed, leaving `["OTHER"]`.
- The Codex line-role prompt is strict: prompt rows include per-line `candidate_labels`, and `_parse_codex_line_role_response` rejects any label outside that allowlist.
- Result: Codex can only emit `OTHER` for such rows. It is not choosing freely among `INGREDIENT_LINE` / `OTHER`; the deterministic allowlist already boxed it into `OTHER`.

Examples from `2026-03-14_17.21.15` Salt Fat Acid Heat run:
- `1 large jalapeĂ±o, seeds and veins removed if desired, thinly sliced`
  - `_QUANTITY_START_RE` matches, but `_is_ingredient_line` still returns false because there is no recognized unit, no noun hint from the current list, and `_is_short_quantity_led_ingredient_shape` fails on the Unicode `Ă±` tokenization path.
- `Optional: 1-ounce chunk of Parmesan`
  - `_QUANTITY_START_RE` fails because the string starts with `Optional:` instead of a quantity.
- `1-pound loaf day-old country or sourdough bread`
  - `_QUANTITY_START_RE` fails because the quantity is hyphenated (`1-pound`) rather than space-separated (`1 pound`).

Net effect:
- deterministic vanilla labeled these rows `OTHER` via fallback,
- codex line-role also labeled them `OTHER` because its allowlist for each row had already collapsed to `["OTHER"]`.
