- `RunSettings` defaults are still AI-on for recipe and line-role paths (`llm_recipe_pipeline=codex-farm-3pass-v1`, `line_role_pipeline=codex-line-role-v1`), and QualitySuite `baseline` experiments on 2026-03-04 inherited `llm_recipe_pipeline=off` plus `line_role_pipeline=codex-line-role-v1`, so those baselines were hybrid rather than fully deterministic.
- The all-method prediction reuse key hashes the full `run_config` payload except `benchmark_sequence_matcher`, so non-LLM knobs such as EPUB extractor/parser/preprocess settings produce different reuse keys even when they likely drive the same line-role workload.
- The 2026-03-04 exhaustive run was not a full cache miss across every variant in retained artifacts: available `all_method_benchmark_report.json` files sum to `377` variants, `137` executed predictions, and `240` cross-run reuses. Cache fragmentation is real, but "every variant forced a fresh call" overstates what the repo artifacts show.
- Dashboard default `Previous Runs` view is filtered to official single-offline `vanilla`/`codexfarm` rows. Those single-offline vanilla variants explicitly set `line_role_pipeline=off`, so the default dashboard vanilla rows are not the same contaminated QualitySuite baselines.
- Current code already has a low-level LLM kill switch in `cookimport/llm/codex_exec.py` via `COOKIMPORT_ALLOW_LLM`, so reports proposing that exact safeguard are stale relative to current code.
