# Knowledge Failure Classes

Two different failure classes appeared in adjacent benchmark runs:

- `2026-03-18_21.10.12`: knowledge never executed successfully because the shipped nested schema was invalid for live structured outputs (`rc` missing from nested `required`).
- `2026-03-18_21.17.59`: knowledge executed, but 4 of 5 shards failed validation because the model dropped owned chunk results, emitted an empty result set, or invented a `processing_error` chunk with out-of-surface evidence.

The first bug is a schema/test-gap problem. The second bug is a shard-size / prompt-contract / output-integrity problem.
