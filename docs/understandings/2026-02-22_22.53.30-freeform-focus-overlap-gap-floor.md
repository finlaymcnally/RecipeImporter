# Discovery

With freeform prelabeling, `segment_focus_blocks` currently labels a contiguous focus subset inside each segment task. If `segment_overlap` is too small relative to focus size, adjacent focus windows do not meet.

Example from production run:
- `segment_blocks=45`
- `segment_overlap=5` (step `40`)
- `segment_focus_blocks=30`

Each task focused blocks `[start, start+29]`, so focus windows advanced by `40` but only covered `30` blocks each. That created deterministic unlabeled gaps (`10` blocks per step), even before model-quality effects.

Safe rule:
- enforce `segment_overlap_effective >= segment_blocks - segment_focus_blocks`.

This guarantees focus windows can hand off coverage between adjacent tasks.
