read_when:
  - "When deciding whether interactive all-method benchmark can reuse the shared CodexFarm process submenu."
  - "When changing `_build_all_method_variants(...)` and you need to know whether all-method Codex variants include knowledge harvest as well as recipe correction and line-role."
---

All-method benchmark previously asked only `Include Codex Farm permutations?`, but the variant builder's codex path already applied the full benchmark Codex contract. In practice that means recipe correction, Codex line-role, and knowledge harvest were all enabled together even though the menu text did not say so.

The safer shared contract is:
- reuse the shared interactive CodexFarm process submenu for all-method benchmark callers,
- allow the all-method builder to honor the exact selected Codex surfaces,
- keep the legacy full-codex behavior only as the default fallback when no explicit surface selection object is supplied.
