# QualitySuite Live ETA Queue Awareness

- Previous live ETA used only currently active experiment telemetry and reported the slowest active experiment time.
- In parallel runs with many queued experiments, this systematically under-reported remaining wall time.
- Current ETA model keeps active telemetry but adds queue-aware waves (`remaining - active`) and estimates queued waves using median active ETA.
- When active ETA samples are unavailable, it now falls back to median duration from completed experiments.
- Live status output now includes `queued_experiments=` so the ETA context is explicit while the run is in flight.
