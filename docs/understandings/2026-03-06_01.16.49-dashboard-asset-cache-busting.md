# Dashboard asset cache busting

`cookimport stats-dashboard` writes fresh inline benchmark data into `.history/dashboard/index.html`, but the page still loads `assets/dashboard.js` and `assets/style.css` separately.

That means a browser can theoretically render fresh data with stale JS if it reuses cached subresources. The defensive fix is to version those asset URLs in generated HTML and keep the JSON fetch fallback `no-store`.
