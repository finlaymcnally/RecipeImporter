# Dashboard compare/control harness data dependency

- `applyCompareControlSelectionSubset()` now calls `compareControlSourceRecords()`, which reads `filteredBenchmarks()`.
- `filteredBenchmarks()` dereferences `DATA.benchmark_records`; Node harness tests that disable bootstrap must seed `DATA` explicitly.
- Test harness fix: add a helper to inject minimal benchmark rows (`run_category=benchmark_eval`, timestamp) before invoking subset behavior.
- Assertion update: trend-chart test should assert `renderCompareControlSection();` wiring instead of older direct `renderCompareControlDynamicChart(...)` call site.
