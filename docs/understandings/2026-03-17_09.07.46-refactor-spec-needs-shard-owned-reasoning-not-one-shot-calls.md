# Refactor Spec Needs Shard-Owned Reasoning, Not One-Shot Calls

- The durable translation from the old model is not "same stages, but larger prompts."
- Line-role should become bounded contiguous label shards with exact ownership over reviewed rows.
- Knowledge should become bounded outside-recipe shards that return one normalized result per owned `chunk_id` or span.
- Recipe correction should become bounded multi-recipe shards that share framing once and return one corrected result per owned `recipe_id`.
- The deterministic stages still own segmentation, grouping, intermediate building, validation, and final assembly.
- Preview and benchmark language should therefore shift from prompt counts to shard counts, fresh-agent counts, and live shard telemetry.
