---
summary: "Single-profile multi-book group upload bundle seam: high-level-only mode must avoid recursive artifact sweep and cap per-book sampled wrong-line detail by a fixed size budget."
read_when:
  - "When adding or debugging group-level upload bundles under single-profile-benchmark roots."
  - "When upload_bundle_v1 payload size grows too large in multi-book benchmark sessions."
---

- Existing behavior: `build_upload_bundle_for_existing_output(...)` uses `_write_upload_bundle_three_files(...)`, which recursively includes every file under the source root in payload rows.
- Impact for multi-book roots: `single-profile-benchmark/` roots can contain many run subfolders and heavy codex artifacts (for example full prompt logs), so a naive root bundle grows quickly.
- Practical seam: add a dedicated `high_level_only` mode in `_write_upload_bundle_three_files(...)` that uses curated artifact selection and a derived group packet for per-run highlights.
- Size-control seam: target bytes should be fixed (about 40MB), then split into an artifact-budget slice plus per-run sampled wrong-line budget so each run gets less sampled detail as run count increases.
