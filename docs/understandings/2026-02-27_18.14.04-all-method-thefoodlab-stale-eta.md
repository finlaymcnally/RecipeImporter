## Discovery

Run: `data/golden/benchmark-vs-golden/2026-02-27_17.54.41`

- At `18:12 EST`, run progress was `71/91` with no new output files after `18:10:17`, but `C3imp` worker processes remained CPU-active.
- `saltfatacidheatcutdown` had 5 configs in evaluate; `thefoodlabcutdown` had 8 configs already in evaluate with 0 finished yet.
- Recent finished salt configs (`011`, `012`) showed `evaluate_seconds ~= 447s`, proving evaluate can dominate wall time.
- Canonical text size is very large for these sets:
  - `7_thefoodlabcutdown/exports/canonical_text.txt` = `527,221` chars
  - `saltfatacidheatcutdown_2/exports/canonical_text.txt` = `398,300` chars
- Spinner ETA is completion-rate based and can stay stale when active work is mostly long-running canonical alignment with few `config_finished` events.

## Practical implication

For big canonical sets, flat ETA for 10+ minutes can still be normal. Treat as stalled only if CPU drops and scheduler/config event files stop changing for a prolonged period (for this run, >30 minutes is a better threshold than a few minutes).
