On the 2026-03-17 `fresh-summary-check` run, the line-role wrapper prompt itself was tiny (`~720` chars), but the saved trace for worker 1 showed Codex immediately shelling out to read the deposited shard file:

- first `cat` of the full shard file
- then additional `awk` / `sed` / `jq` slices over the same file
- then a final agent message

Those command outputs are part of the same Codex thread. The turn-level `usage.input_tokens` reported by Codex is therefore not “wrapper prompt only”; it is the sum of repeated model passes over an ever-growing thread that now includes the file contents and later file slices. Cached-input telemetry becomes huge because the same prefix is re-read over and over across those subturns.

In this run, the five line-role shards triggered `32` command-execution reads across the deposited shard files. The raw command outputs summed to about `1.13M` chars, while a rough cumulative “visible chars seen across model subturns” estimate was about `8.2M` chars, which lines up with the observed `1.82M` input-token charge much better than the single exported prompt text does.
