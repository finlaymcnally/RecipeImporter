# Stage-backed benchmark unification seam

The key seam is not just canonical line-role. `cookimport/labelstudio/ingest.py::generate_pred_run_artifacts()` currently duplicates most of `cookimport/cli_worker.py::stage_one_file()`: conversion, optional Codex recipe pass, optional knowledge harvest, chunk/table generation, processed-output writing, stage-block prediction writing, and manifest assembly.

That means the clean unification target is one shared stage/import session that returns both the in-memory `ConversionResult` and the written stage run artifacts. Benchmark-specific work should happen after that shared session and should be limited to eval, task generation, upload prep, and compatibility manifests.

The most important semantic rule is that the primary benchmark score must come from the stage run’s `.bench/<workbook_slug>/stage_block_predictions.json` plus the same extracted block text. Canonical line-role can only survive as secondary diagnostics or a separate experiment. It cannot remain a pre-score mutation step if benchmarking is supposed to mean “score the real import run.”
