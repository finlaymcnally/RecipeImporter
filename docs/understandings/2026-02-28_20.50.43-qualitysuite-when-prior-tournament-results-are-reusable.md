# QualitySuite Result Reuse Boundary

- Fold-level work reruns when a new tournament root is created (`resume_tournament_dir: null`), even if some per-config artifacts can be reused from shared caches.
- Prediction/alignment reuse reduces repeated low-level work, but it is not a full tournament-result memoization key.
- Final result reuse is safe only when the full input tuple matches a previously completed run:
  - experiments file contents
  - thresholds file contents
  - fold/seed plan and duplicate-fold behavior
  - code semantics that affect scoring/output
- Snapshot from current investigation:
  - `2026-02-28_16.18.25` is completed, but it used shortlist + `fast-nosweeps` (`include_deterministic_sweeps=false`).
  - `2026-02-28_20.25.10` uses full-candidates + sweeps-decision thresholds (`include_deterministic_sweeps=true`) and has no prior completed twin run.
  - Therefore this current run cannot be replaced by the prior completed result.
