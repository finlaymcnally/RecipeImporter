The March 16, 2026 regression run at `data/golden/benchmark-vs-golden/2026-03-16_12.14.35/` wrote healthy `line-role-pipeline/stage_block_predictions.json` and `line-role-pipeline/extracted_archive.json` files, but `manifest.json` still pointed `stage_block_predictions_path` at the processed stage `.bench/.../stage_block_predictions.json` and `extracted_archive_path` at the raw `full_text.json`.

That happened because `generate_pred_run_artifacts(...)` only rewired the canonical scorer pair to `line_role_artifacts` when `stage_block_predictions_source_path is None`. After the label-first refactor, stage-backed artifacts now exist, so the rewiring branch stopped firing even though the benchmark docs still expect canonical-text line-role runs to score the projection artifact pair.

Symptom in the bad run:
- `line-role-pipeline/extracted_archive.json`: 1471 nonempty rows, about 395k chars
- `eval_report.json`: only 85 nonempty prediction blocks, about 7k chars matched

The scorer was not reading the projection archive at all; it was reading the wrong canonical pair from manifest metadata.
