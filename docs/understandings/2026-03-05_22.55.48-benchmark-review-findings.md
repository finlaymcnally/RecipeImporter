# Benchmark Review Findings

- The line-role telemetry writers now publish both `prediction-run/manifest.json: line_role_pipeline_telemetry_path` and `run_manifest.json: artifacts.line_role_pipeline_telemetry_json`, but the analytics readers in `cookimport/analytics/perf_report.py` and `cookimport/analytics/dashboard_collect.py` only look for the first key. A manifest payload that only exposes the artifact key still backfills as all-`None`.

- The single-offline planner in `cookimport/cli.py::_interactive_single_offline_variants(...)` still returns `[("vanilla", selected_benchmark_settings)]` for any non-`codex-farm-3pass-v1` recipe pipeline. That includes hybrid settings like `llm_recipe_pipeline=off` with `line_role_pipeline=codex-line-role-v1`, so the CLI artifact naming layer can still stamp non-deterministic runs into `vanilla` directories even though analytics now relabel them later.
