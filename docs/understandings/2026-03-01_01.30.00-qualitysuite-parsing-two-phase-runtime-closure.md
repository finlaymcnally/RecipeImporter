# QualitySuite Parsing Two-Phase Runtime Closure

- `scripts/quality_top_tier_tournament.py` now has deterministic seed-plan resolution (`--seed`, `--seed-list`, thresholds fallback) and records seed source/resolved plan in `tournament_resolved.json`.
- Fold-level live progress can be read from each fold `quality-run` checkpoint (`checkpoint.json` fields `experiment_count_total`, `experiment_count_completed`, `pending_experiment_ids`) and promoted to top-level `tournament_checkpoint.json` during active folds.
- `cookimport/bench/quality_runner.py` can compute full variant cardinality before race rounds; when `variants_effective <= race_finalists`, race cannot prune and the run now executes one exhaustive pass with explicit metadata (`reason=race_no_prune_variant_count_le_finalists`).
- `cookimport/cli.py` prediction reuse now distinguishes in-run vs cross-run by checking whether cached source artifacts live under the current run root. Reuse artifact materialization attempts hardlinks first and falls back to copy safely when linking is unavailable.
