# Compare & Control per-recipe derived fields

The Compare & Control field catalog only exposes fields that `previous_runs_field_value(...)` knows how to resolve plus paths collected by `collect_benchmark_field_paths(...)`. That means a metric like "time per recipe" does not exist automatically just because both `conversion_seconds` and `recipes` exist.

To make a derived metric selectable in both CLI/backend analysis and the served dashboard UI, add it in both places:

- `cookimport/analytics/compare_control_engine.py`
- `cookimport/analytics/dashboard_render.py`

For this case the minimal derived fields are:

- `conversion_seconds_per_recipe`
- `all_token_use_per_recipe`

Both are `metric / recipes` and should return `None` when `recipes <= 0` or the source metric is missing.
