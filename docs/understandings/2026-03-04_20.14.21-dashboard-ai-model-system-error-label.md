---
summary: "AI Model now displays System error when manifest reports Codex runtime fatal failure."
read_when:
  - "When AI Model column should distinguish successful codex runs from fallback/system failures"
  - "When benchmark rows have missing tokens and possible codex runtime failures"
---

Discovery and implementation:
- Codex runtime failure metadata is available in benchmark manifests via `llm_codex_farm.fatalError` (or equivalent), even when `process_runs`/token telemetry is absent.
- Dashboard collector now enriches benchmark `run_config` with `codex_farm_runtime_error` from manifest fatal-error fields.
- Dashboard and compare-control AI model labeling now prioritizes this error marker and returns `System error`.
- This avoids showing a normal model label for rows where codex execution failed and fallback path was used.

Scope note:
- Rows without manifest fatal-error evidence still keep existing model/off/unknown labeling.
- Missing token telemetry alone does not force `System error`; explicit failure metadata is required.
