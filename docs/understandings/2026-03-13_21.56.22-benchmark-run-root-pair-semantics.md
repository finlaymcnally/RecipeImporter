---
summary: "Benchmark runs do not intrinsically create two sibling timestamp roots; paired roots seen on 2026-03-06 came from separate benchmark invocations, while `single-profile-benchmark` nests per-book/per-variant runs under one session root."
read_when:
  - When interpreting multiple nearby timestamps under `data/golden/benchmark-vs-golden/`
  - When debugging `single-profile-benchmark` versus direct `labelstudio-benchmark` output roots
---

- Direct `labelstudio-benchmark` calls create one timestamped eval root when `--eval-output-dir` is omitted.
- Interactive `single-profile-benchmark` first creates one session root like `data/golden/benchmark-vs-golden/<timestamp>/single-profile-benchmark`, then calls `labelstudio_benchmark(...)` with explicit nested `eval_output_dir` paths inside that session root for each book and variant.
- Because of that explicit override, a single single-profile session should not create a second sibling timestamp directory for its child runs.
- The observed sibling pairs on 2026-03-06 are separate invocations close together in time, not an automatic "two folders per run" behavior.
- Evidence:
  - `cookimport/cli.py:29694-29697` defaults a direct benchmark run to one timestamped root only when `eval_output_dir is None`.
  - `cookimport/cli.py:2611-2624` and `cookimport/cli.py:2701-2746` route single-profile runs into `.../<timestamp>/single-profile-benchmark/...` and pass nested `eval_output_dir` values to child benchmark calls.
