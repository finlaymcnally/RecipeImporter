## Discovery

Date: 2026-03-02

- Interactive per-run worker settings are now normalized through a run-settings gate (`choose_run_settings`) before launch, which is where a utilization cap can be applied once for both import and benchmark flows.
- `C3imp` entrypoint now sets default pacing envs for safe defaults so every interactive session starts with bounded runtime pressure without needing shell env exports:
  - `COOKIMPORT_WORKER_UTILIZATION=90`
  - `COOKIMPORT_IO_PACE_EVERY_WRITES=16`
  - `COOKIMPORT_IO_PACE_SLEEP_MS=8`
- The worker utilization still allows explicit override (e.g. `COOKIMPORT_WORKER_UTILIZATION=85`) before running.
