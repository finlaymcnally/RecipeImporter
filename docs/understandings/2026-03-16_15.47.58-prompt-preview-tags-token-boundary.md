`cookimport/llm/prompt_preview.py` and `cookimport/cf_debug_cli.py` still expose only recipe, knowledge, and line-role preview surfaces. Existing `prompt_preview` artifacts therefore cannot show tag prompts or tag token counts.

The recent tag integration work in `cookimport/tagging/orchestrator.py` only writes accepted tags back into `final drafts/.../r*.json` as `recipe.tags` and `intermediate drafts/.../r*.jsonld` as `keywords`. That projection happens after the tags pass and does not alter any recipe, knowledge, or line-role prompt builders, so the input-token delta from that change is zero.
