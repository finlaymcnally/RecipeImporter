# Bench + merge progress counter wiring

- `bench run` counters belong in `cookimport/bench/runner.py:run_suite(...)` because only that loop has the item total and per-item lifecycle events.
- `bench sweep` should own outer `config X/Y` counters in `cookimport/bench/sweep.py` and forward nested runner messages as `config X/Y | <runner message>`.
- Split-job merge progress shown in the worker dashboard is driven by `status_callback` in `cookimport/cli.py:_merge_split_jobs(...)`; phase counters must be added there, not in the Live dashboard renderer.
- A shared formatter (`cookimport/core/progress_messages.py`) keeps `X/Y` clamping and message shapes consistent across Label Studio, bench, and merge flows.
