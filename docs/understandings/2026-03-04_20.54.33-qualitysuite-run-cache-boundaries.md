# QualitySuite run cache boundaries

- Active flow is `bench quality-discover` + `bench quality-run` (+ `quality-leaderboard`/`quality-compare`). `quality-lightweight-series` and tournament script are retired/disabled.
- A quality run executes per-experiment all-method benchmarks over suite-selected targets, not every book in the library.
- Resume reuse is run-level only: `--resume-run-dir` reuses completed experiment snapshots (`experiments/<id>/quality_experiment_result.json`) and runs only pending experiments.
- Prediction generation reuse is key-based (`source_file` + run config; `benchmark_sequence_matcher` excluded as evaluate-only). Matching keys can reuse prediction artifacts in-run or cross-run from `data/golden/bench/quality/.cache/prediction_reuse` (default).
- Evaluation reuse is eval-signature-based (gold fingerprint + prediction rows + matcher), with in-run dedupe and cross-run cache reuse under canonical alignment cache roots.
- Practical implication:
  - Changing prediction-affecting run settings usually reruns prediction/import work for that variant.
  - Changing evaluate-only knobs (for example matcher) can reuse prediction artifacts and rerun only evaluation.
  - Identical config signatures can short-circuit both prediction and evaluation via cache.
