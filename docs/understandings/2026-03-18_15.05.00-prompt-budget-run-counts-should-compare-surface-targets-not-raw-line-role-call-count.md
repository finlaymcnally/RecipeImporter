# Surface Targets Versus Raw Calls

The user-facing `3/4/5`-style setting is a surface target, not a promise about total raw model calls across every internal subphase.

That matters most for line-role. The current runtime splits line-role into two internal phases:

- `recipe_region_gate`
- `recipe_structure_label`

If the requested line-role target is `5` and both internal phases plan `5` shards, the correct reporting is:

- requested surface target: `5`
- actual surface shard count: `5`
- total raw model calls: `10`

If reporting compares the requested `5` directly to raw call count `10`, it produces a false deviation. The right explanation is: the surface target matched, but line-role has two internal phases, so total model calls doubled.
