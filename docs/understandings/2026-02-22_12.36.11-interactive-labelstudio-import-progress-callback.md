# Interactive Label Studio import progress wiring

- `run_labelstudio_import(...)` already emits phase updates through `progress_callback` (for example `Running freeform prelabeling...`).
- Non-interactive `cookimport labelstudio-import` passed that callback into a Rich status spinner; interactive mode did not.
- Reusing one shared CLI helper for Label Studio import status keeps interactive/non-interactive behavior aligned and avoids duplicate spinner implementations.
