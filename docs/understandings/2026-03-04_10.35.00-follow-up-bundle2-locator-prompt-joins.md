For `single-profile-benchmark/follow_up_bundle_2`, locator joins require the line-role archive path:
- `prediction-run/line-role-pipeline/extracted_archive.json`

That atomic archive does not carry original EPUB spine/page features directly; it carries `source_block_index`. To recover spine metadata reliably:
1. Load base extracted locations from `prediction-run/extracted_archive.json` keyed by base index.
2. For each atomic line-role row, map `source_block_index` back to the base location.
3. Merge base features (including `spine_index`) with line-role features.

Without this merge, spine coverage drops to zero on atomic rows.

Prompt excerpts for reviewer portability are read from each codex run’s:
- `codexfarm/full_prompt_log.jsonl`

Top regression packet selection for this follow-up is by per-recipe negative `delta_codex_minus_baseline` from:
- `_upload_bundle_derived/root/01_recipe_triage.packet.jsonl`

Then inline excerpts are embedded per recipe (prefer `pass2`, then `pass3`, then `pass1`) so reviewers can inspect request/response content without navigating locators.
