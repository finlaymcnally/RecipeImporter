# Compare & Control chart routing

- `buildCompareControlChartDefinition(...)` is the routing seam for dynamic chart type choice.
- Previous state persisted `chart_type`, but runtime behavior was effectively scatter-only.
- The safe extension point is to infer chart type from field metadata already available in context (`compare_info`/catalog):
  - numeric compare field -> scatter builder,
  - categorical compare field -> bar builder.
- Keep `renderCompareControlChartHost(...)` chart config type aligned with resolved definition (`scatter` -> Highcharts scatter, `bar` -> Highcharts column) so tooltips/fallback lifecycle remain shared.
