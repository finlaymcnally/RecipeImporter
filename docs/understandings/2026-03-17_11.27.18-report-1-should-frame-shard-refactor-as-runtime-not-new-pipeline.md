# Report 1 Should Frame Shard Refactor As Runtime, Not New Pipeline

- The memo's key correction is that the desired change is mainly execution-model, not stage-model.
- Keep the label-first staged architecture and deterministic promotion boundaries.
- Replace many fresh Codex calls with bounded phase workers that process multiple explicit shards.
- Treat shards as ownership units and workers as execution contexts; they are related but not the same thing.
- Emphasize clean phase-local workspaces, task-relevant files only, and structured proposed outputs rather than in-place mutation.
