# Oracle Auto Follow-Up Can Only Trigger From What Oracle Actually Saved

- The new detached follow-up worker can only parse the answer text that Oracle persisted into `oracle_upload.log` and the Oracle session store. If turn 1 completed but Oracle only saved a short summary answer, there is nothing for recipeimport to parse into `Requested follow-up data`.
- That exact failure happened on the `2026-03-19_17.13.29` benchmark run for session `you-are-reviewing-a-benchmark-290`: the chat conversation URL existed and turn 1 was grounded, but the saved answer was only a two-sentence 55-token summary with no structured follow-up block.
- The worker should therefore fail explicitly and leave `oracle_auto_followup.json` in the source run directory, rather than silently doing nothing and leaving the operator to infer whether turn 2 was skipped, lost, or never attempted.
