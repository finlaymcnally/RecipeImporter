# Dry run showed current worktree regressed to old inline line-role path

On 2026-03-18 I ran two non-live checks:

- a direct `label_atomic_lines(...)` dry run with `codex_farm_cmd=scripts/fake-codex-farm.py`
- `cf-debug preview-prompts` on a deterministic fixture processed run

Both checks still wrote the old artifact shape:

- single-phase shard ids like `line-role-shard-0001-a000000-a000003`
- `line-role-pipeline/runtime/workers/.../in/line-role-shard-....json`
- prompt files like `prompts/prompt_0001.txt`
- preview input files like `line-role-pipeline/in/line_role_prompt_0001.json`

They did **not** write the intended file-backed/two-phase artifact families:

- `prompts/recipe_region_gate/...`
- `prompts/recipe_structure_label/...`
- `runtime/recipe_region_gate/...`
- `runtime/recipe_structure_label/...`

That means the current worktree runtime/preview path is still effectively wired to the older inline single-phase implementation, even though the intended file-backed design exists in the recent plan and test expectations. Do not do a live run until `canonical_line_roles.py` and prompt-preview output both show the per-phase file-backed layout again.
