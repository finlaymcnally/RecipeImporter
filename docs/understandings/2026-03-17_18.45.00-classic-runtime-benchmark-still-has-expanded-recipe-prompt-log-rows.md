title: Classic runtime benchmark still has expanded recipe prompt-log rows
summary: On classic shard runtime, prompt-log row count can exceed true shard-call count because recipe outputs are expanded into per-item rows after one shard call returns.
read_when:
  - When `full_prompt_log_rows` is much higher than planned shard-call count on `classic_task_farm_v1`
  - When deciding whether prompt preview should calibrate against `prompt_budget_summary.json` or `prompts/full_prompt_log.jsonl`
  - When debugging why recipe correction appears to exceed prompt-target count despite `max_turns_per_shard = 1`
---

On the 2026-03-17 `fresh-summary-check` CodexFarm benchmark, the real stage budget stayed at the planned `5 / 5 / 5` shard calls (`15` total) in `prompt_budget_summary.json`, but `prompts/full_prompt_log.jsonl` still had `39` rows because recipe correction emitted one reconstructed prompt row per owned recipe item (`29`) instead of one row per shard call (`5`).
