# Current-Contract Legacy Cut Batch

The first deletion batch was safest where runtime had already converged on one real contract and the extra branch was only a name or a dormant reader. That made it practical to cut without changing the live product shape.

What was removed in this batch:

- line-role prompt formatting is now compact-only; there is no remaining `legacy` prompt serializer path
- ingredient missing-unit policy now uses `medium` instead of `legacy_medium`
- canonical-text benchmark reports/cache keys now call the enforced scorer `global`, not `legacy`
- multi-recipe splitter backends no longer accept a `legacy` alias; active values are `off` and `rules_v1`
- analytics no longer probes the old CSV location or dash-timestamp run folders
- Label Studio export validates `freeform-spans` generically instead of carrying explicit old-scope rejection code
- knowledge job building no longer carries a dormant non-compact serializer branch; live harvest and prompt preview both rebuild the same compact inputs

What is still clearly pending after this batch:

- recipe-correction internals still expose `pass1` / `pass2` / `pass3` names
- staging still writes draft compatibility aliases
- benchmark follow-up and runner code still keep historical-reader compatibility branches
