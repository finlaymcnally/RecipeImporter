# Codex prompt cut ranking for previewed benchmark run

- Corrected live-like prompt total for the previewed run is about `663k` input tokens, not the larger preview-log wrapper number. That is about `7.7x` the `~86k` token book.
- Stage share after correcting line-role preview seams: recipe correction `~53%`, knowledge `~30%`, line-role `~17%`.
- Biggest low-risk lever is pass-4 context width. Reducing `codex_farm_knowledge_context_blocks` from `12` to `4` per side saves about `96k` tokens; reducing to `2` per side saves about `121k`.
- Cheapest recipe cut is dropping `draft_hint`, which is currently framed as a downstream preview only. That saves about `92k` tokens on this run.
- Another cheap recipe trim is removing `recipe_candidate_hint.provenance`, saving about `47k` tokens without changing the visible recipe fields.
- Noise-lane knowledge prompts are still worth direct gating: 7 previewed calls would save about `25k` tokens if skipped.
- Line-role is smaller than first preview math suggested because live prompts use `compact_v1`, but blanking neighbor context for all rows would still save about `56k`; blanking neighbors only outside recipe spans saves about `27k`.
- A pragmatic low-risk bundle for this run is: drop `draft_hint`, trim pass-4 context to `2` per side, and skip noise-lane knowledge calls. That would save about `238k` tokens and bring the run to about `4.9x` book size before completions.
