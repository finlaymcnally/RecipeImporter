# Knowledge Bundling Is Now Gap-Limited, Not Cap-Limited

- On `saltfatacidheatcutdown`, the post-bundle preview `prompt_preview_codex_hypothetical_after_knowledge_bundle_2026-03-16_20.56.20` writes `91` knowledge prompts for `320` surviving chunks.
- Those `320` chunks fall into `89` strictly contiguous runs by block index, so hard contiguity is now the main reason prompt count stays high.
- Current bundle caps are not the main bottleneck:
  - only `3` bundles hit the `max_chunks_per_bundle=12` cap
  - only `2` bundles were near the `max_bundle_chars=14000` cap
- Simulating the same chunk list with soft gap bridging shows the likely path to `~40` prompts:
  - allow gaps up to `5` blocks: about `54` bundles
  - allow gaps up to `8` blocks: about `46` bundles
  - allow gaps up to `10` blocks: about `41` bundles
- Narrowing candidate spans to only final `knowledge_spans` does not help on this run; it actually increased jobs in a local simulation because it breaks contiguous packing opportunities.
