read_when:
  - "When validating whether full_prompt_log.jsonl rows are exact payload captures or reconstructed prompts."
  - "When debugging benchmark cutdown packages that appear to only contain sampled prompt examples."
---

- `full_prompt_log.jsonl` is now the source-of-truth artifact: one row per pass call with no sampling/truncation.
- Row field `request_payload_source` indicates how prompt text was derived:
  - `telemetry_csv`: prompt/model/runtime fields came from `codex_exec_activity.csv` per-call telemetry.
  - `reconstructed_from_prompt_template`: fallback rebuilt from pass input payload + prompt template.
- `request_telemetry` carries per-call runtime metadata (task/worker ids, attempt indices, token usage fields, hashes, and transport paths) so a reviewer can audit what was actually executed.
