# Interactive Esc Back Contract

- `_menu_select(...)` is the select-menu control point and binds `Esc` to `BACK_ACTION`.
- Typed interactive prompts must go through `_prompt_text(...)`, `_prompt_confirm(...)`, or `_prompt_password(...)`.
- Those wrappers bind `Esc` to a `None` result so existing call sites treat it as cancel/back one level.
- Direct `questionary.text/confirm/password` usage in interactive flows will silently lose this behavior.
