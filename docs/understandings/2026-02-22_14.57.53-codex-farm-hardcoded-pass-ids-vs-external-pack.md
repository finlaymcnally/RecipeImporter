# Understanding: codex-farm hardcoded pass IDs vs external packs

## Observation

The orchestrator previously hardcoded pass IDs (`recipe.chunking.v1`, `recipe.schemaorg.v1`, `recipe.final.v1`) and only set `CODEX_FARM_ROOT` env, which is fragile when external packs use different IDs or require explicit workspace behavior.

## Why it matters

External CodexFarm integration contracts now support explicit `--root`, optional `--workspace-root`, and pipeline-defined `codex_cd_mode`. If recipeimport cannot configure pass IDs/workspace root, it can fail with "unknown pipeline" or run in an unintended working directory.

## Practical fix

- Expose these as run settings and CLI knobs:
  - `codex_farm_workspace_root`
  - `codex_farm_pipeline_pass1/2/3`
- Thread them through both stage and prediction-run generation.
- Pass explicit `root_dir` and optional `workspace_root` in runner calls.
- Record effective pipeline IDs in `llm_manifest.json` for auditability.
