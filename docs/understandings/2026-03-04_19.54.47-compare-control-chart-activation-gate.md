# Compare & Control chart activation gate

- Problem seam: compare/control settings are persisted and restored, so the dynamic chart can render immediately on page load even before the user intentionally uses the panel.
- Added an explicit runtime gate (`compareControlChartActivated`) that is **not persisted**.
- Gate defaults to false and is reset to false on UI-state payload apply/reset.
- Gate flips true only when compare/control controls are actively used (view/outcome/compare/split/hold/group interactions, discovery-card selection, subset action) and when a preset is explicitly loaded.
- Render behavior:
  - gate false -> chart host stays empty with a short instruction message,
  - gate true -> chart definition/render pipeline runs normally.
