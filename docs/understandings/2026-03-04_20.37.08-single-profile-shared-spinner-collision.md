## Findings

- `single-profile` multi-book mode uses an outer `_run_with_progress_status(...)` live spinner and inner `labelstudio_benchmark(...)` calls per book/variant.
- Inner benchmark calls were only using `suppress_spinner=True`, so they still printed per-run summary lines and triggered dashboard rebuild output (`Wrote dashboard ...`) while the outer spinner was repainting.
- Those concurrent writes cause the boxed spinner panel to jump/interleave in PTY output.

## Applied seam

- In shared-spinner mode (`single_profile_dashboard is not None`), nested benchmark calls should also set `suppress_summary=True` and `suppress_dashboard_refresh=True`.
- After the batch completes, run one deferred `_refresh_dashboard_after_history_write(...)` for the single-profile batch output root.
