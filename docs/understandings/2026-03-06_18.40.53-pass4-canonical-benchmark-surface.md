---
summary: "Canonical-text benchmark scoring only reflected pass4 after merging snippets into the preferred line-role projection artifact."
---

Before this change, pass4 `knowledge/<workbook_slug>/snippets.jsonl` could affect the legacy stage evidence path but not the preferred canonical-text benchmark surface.

Why:

- canonical-text with `--line-role-pipeline` prefers `prediction-run/line-role-pipeline/stage_block_predictions.json`
- that artifact was built entirely from canonical line-role spans
- pass4 snippets were therefore absent from the scored artifact even when pass4 ran successfully

The lowest-risk fix was to enrich the preferred line-role projection artifact instead of changing the evaluator API:

- `cookimport/labelstudio/ingest.py` now resolves pass4 `knowledge_snippets_path` before writing line-role artifacts
- `cookimport/labelstudio/canonical_line_projection.py` loads pass4 snippet evidence and upgrades matching projected `OTHER` spans to `KNOWLEDGE`
- matching uses source block index plus snippet evidence quotes when possible, with source-block fallback for non-recipe spans

This keeps one canonical-text benchmark surface:

- recipe structure still comes from line-role projection
- pass4 contributes `KNOWLEDGE` labels on that same surface
- recipe-local labels are preserved because only projected `OTHER` spans are upgraded

Regression proof added:

- `tests/labelstudio/test_canonical_line_projection.py::test_projection_artifacts_merge_pass4_knowledge_into_other_spans`
- `tests/bench/test_eval_stage_blocks.py::test_evaluate_canonical_text_scores_pass4_knowledge_in_line_role_projection`
