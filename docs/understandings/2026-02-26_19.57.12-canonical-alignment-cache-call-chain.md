# Discovery: canonical alignment cache call chain and safety surface

Observed flow after implementation:

- `_run_all_method_benchmark(...)` now defines per-source cache root: `<source_root>/.cache/canonical_alignment`.
- `_run_all_method_config_once(...)` forwards that path to `labelstudio_benchmark(...)` as `alignment_cache_dir`.
- `labelstudio_benchmark(...)` forwards `alignment_cache_dir` only when canonical mode calls `evaluate_canonical_text(...)`.
- `evaluate_canonical_text(...)` passes cache path into `_align_prediction_blocks_to_canonical(...)`.
- `_align_prediction_blocks_to_canonical(...)` computes normalized text + boundary hashes, attempts cache load, and falls back to legacy `SequenceMatcher` alignment on miss.

Durable safety behavior:

- Cache hit requires matching canonical hash, prediction hash, boundary hash, normalization version, and algorithm version.
- Payload shape is validated before reuse; validation failures become misses and are recorded in `evaluation_telemetry.alignment_cache_validation_error`.
- Disk writes are atomic and guarded by per-key lock files with stale-lock cleanup.

Evidence anchors:

- `cookimport/bench/canonical_alignment_cache.py`
- `cookimport/bench/eval_canonical_text.py`
- `cookimport/cli.py` (`_run_all_method_benchmark`, `_run_all_method_config_once`, `labelstudio_benchmark`)
- `tests/bench/test_eval_stage_blocks.py`
