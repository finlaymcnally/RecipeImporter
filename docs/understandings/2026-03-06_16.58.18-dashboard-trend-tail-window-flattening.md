# Dashboard trend tail window flattening

- Area: `cookimport/analytics/dashboard_render.py`
- Symptom: In `Previous Runs -> History Table & Trend`, dashed rolling trend overlays could go horizontal for the last few timestamps even while the underlying scatter points were still changing.
- Cause: `buildRollingTrend(...)` shifted edge windows backward to preserve a full fixed-size slice. Near the end of the series, multiple tail points therefore used the exact same last window and got the same averaged `y`.
- Fix: Keep the centered window where possible, but let it shrink at the series edges (`slice(start, end)` with clipped bounds). That preserves distinct tail averages and keeps the overlay responsive to the latest runs.
- Regression coverage: `tests/analytics/test_stats_dashboard.py::test_benchmark_trend_overlay_recomputes_tail_windows`
