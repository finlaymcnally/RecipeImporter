# Discovery: pytest.ini defaults were bypassed by explicit CLI overrides

- Prior low-noise work lived mostly in `pytest.ini` (`-q`, `--no-summary`, etc) plus glyph suppression hooks.
- Running with `-o addopts=''` removes `pytest.ini` defaults, and `-vv` re-enables per-test verbose rows and header/footer separators.
- This is why separator-heavy output returned despite earlier cleanup passes.

# Current guard

- `tests/conftest.py:pytest_configure(...)` now enforces compact mode even under `-o addopts=''`:
  - `no_header = True`
  - `no_summary = True`
  - `disable_warnings = True`
  - `verbose` clamped to compact mode
- Explicit opt-out for intentional deep debug output:
  - `COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1`
