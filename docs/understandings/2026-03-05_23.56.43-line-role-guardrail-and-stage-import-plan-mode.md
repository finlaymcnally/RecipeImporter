# Line-Role Guardrail And Stage/Import Plan Mode

- `cookimport stage`, `cookimport labelstudio-import`, and `import` can now stop at the command boundary with `--codex-execution-policy plan`.
  - They write `run_manifest.json` plus `codex_execution_plan.json`.
  - They intentionally return before stage processing, upload, prelabel, or other live Codex work.

- `labelstudio-import` plan mode must short-circuit before Label Studio credential resolution and before prelabel handling.
  - `labelstudio-import --prelabel` is still its own Codex-backed surface in execute mode.

- Line-role guardrails now use explicit mode vocabulary in `RunSettings`:
  - `off`: no do-no-harm mutation, write only disabled-scope guardrail report.
  - `preview`: compute would-change rows and write artifacts without mutating accepted predictions.
  - `enforce`: current mutating downgrade/fallback behavior.

- New preferred artifact names are:
  - `line-role-pipeline/guardrail_report.json`
  - `line-role-pipeline/guardrail_changed_rows.jsonl`
  - legacy `do_no_harm_*` files are still written when diagnostics exist so older readers do not break.

- Important implementation trap:
  - `RunSettings.line_role_guardrail_mode` is enum-backed, so helper code must normalize enum `.value` before comparing to `"off"|"preview"|"enforce"`.
  - Without that flattening, preview can silently behave like enforce.

- Deterministic regression fix that mattered in the current code:
  - normalize spaced fractions (`1 / 2 -> 1/2`) before atomization and pass2 evidence normalization,
  - do not split dual-unit quantity text when the second quantity follows `/` (`1 quart/1 L water`).
