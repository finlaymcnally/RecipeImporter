# Benchmark Processed Output Paths

- `labelstudio-benchmark` writes processed cookbook outputs to `data/output/<timestamp>` by default because `--processed-output-dir` defaults to `DEFAULT_OUTPUT`.
- The benchmark CLI passes that value to prediction generation as `processed_output_root`, and ingest writes to `<processed_output_root>/<timestamp>`.
- The command prints the resolved path as `Processed output: ...` at completion.
- Interactive `all_method` benchmark now keeps processed outputs under the interactive output root (`cookimport.json.output_dir`, default `data/output`), scoped by benchmark timestamp:
  - `<output_dir>/<benchmark_timestamp>/all-method-benchmark/<source_slug>/config_*/<prediction_timestamp>/...`
- Interactive all-method completion prints that root as `All method processed outputs: ...`.
- Offline suite `cookimport bench run` currently builds prediction/eval artifacts only and does not pass `processed_output_root`, so it does not write processed cookbook outputs by default.
