# Discovery

- `config 91/91` tracks planned first-pass configs only. It does not include retry-pass attempts.
- With `all_method_retry_failed_configs=1`, failed configs are rerun after the first pass.
- Retry runs use `dashboard_tracking=False`, so per-source `ok/fail` row counters can look frozen (for example `15/15, ok 6, fail 9`) while retries are still executing.
- `scheduler heavy 0/4 | eval 8 | active 8 | pending 1` means split phase is done and 8 evaluate workers are busy, with 1 retry config queued.
- On `thefoodlabCUTDOWN.epub`, canonical-text eval is the tail bottleneck. Config event files can sit at `evaluate_started` for long periods before timeout/finish.
