# Discovery

- In `data/golden/benchmark-vs-golden/2026-02-27_17.54.41/all-method-benchmark`, successful configs spent most time in canonical eval/alignment (`evaluation_seconds` dominated parsing/writing).
- Cross-source aggregate for successful configs: alignment ~17796s, sequence matcher ~9506s, while parsing was ~1315s and writing was ~202s.
- Cache telemetry showed many `alignment_cache_hit=true` configs still spending hundreds/thousands of seconds in `alignment_seconds` with near-zero evaluator CPU.
- Cause: duplicate keys running in parallel wait on the same lock owner; they report cache hit after load, but wall time still includes lock wait.
- Existing all-method cache path was anchored under each timestamped run root, so expensive alignment work was not reused across separate reruns.

# Practical implication

- Within-run cache prevents duplicate CPU-heavy recompute but can still burn wall time via lock waits.
- Cross-run persistence is high leverage: reruns can skip prior sequence-matcher misses entirely when canonical/prediction text + boundaries are unchanged.
