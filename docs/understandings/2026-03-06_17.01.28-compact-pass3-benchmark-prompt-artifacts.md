# Compact pass3 benchmark prompt artifacts

For `data/golden/benchmark-vs-golden/2026-03-06_15.22.11/single-profile-benchmark/02_saltfatacidheatcutdown/codexfarm/`:

- `run_manifest.json` and `prediction-run/manifest.json` both record:
  - `codex_farm_pipeline_pass2 = recipe.schemaorg.compact.v1`
  - `codex_farm_pipeline_pass3 = recipe.final.compact.v1`
- The pass3 process-run section inside `prediction-run/manifest.json` also records `pipeline_id = recipe.final.compact.v1`.
- `full_prompt_log.jsonl` contains exactly:
  - 23 calls for `recipe.chunking.v1`
  - 23 calls for `recipe.schemaorg.compact.v1`
  - 2 calls for `recipe.final.compact.v1`
  - 149 calls for `recipe.knowledge.v1`
- Pass3 only made 2 live LLM calls in this run because 19 pass2-ok recipes were deterministically skipped (`pass3_pass2_ok_deterministic_skips = 19`).

Why the prompt samples still look long:

- The top-level `codexfarm/prompts/` folder is a mixed prompt-artifact export for all CodexFarm stages, not only compact pass3.
- `prompt_task3_pass3_final.txt` is misleadingly named: it is a convenience dump of pass3 input/output JSON files, not the literal rendered prompt text.
- The literal prompt text is in `full_prompt_log.jsonl` and the sampled literal export `prompt_type_samples_from_full_prompt_log.md`.
- The compact pass3 template is shorter than legacy mainly because it uses `recipe_metadata` instead of the larger `schemaorg_recipe` shape, but the literal prompt still includes inline JSON payloads, so it will not look tiny.

Observed prompt sizes in this run (character counts from `request_messages[0].content`):

- `recipe.final.compact.v1`: 2831 to 3320 chars
- `recipe.schemaorg.compact.v1`: 2633 to 6696 chars
- `recipe.chunking.v1`: 11506 to 25659 chars
- `recipe.knowledge.v1`: 12718 to 35223 chars
