# Labelstudio Benchmark Compare: Mode and Debug Gates

`cookimport labelstudio-benchmark compare` builds gate context per source from the winner eval artifact set under each all-method source report.

Key discovery: mode/pipeline resolution comes from the eval run manifest (`<winner eval dir>/run_manifest.json` `run_config.prediction_run_config`) before fallback to summary text tokens. If that manifest payload is absent, compare falls back to `codex_farm_recipe_mode=extract`, which makes debug-artifact gates non-required.

Debug gates are required only when candidate context is:

- `codex_farm_recipe_mode=benchmark`
- `llm_recipe_pipeline=codex-farm-3pass-v1`

When required, compare expects:

- canonical alignment artifact (`aligned_prediction_blocks.jsonl`)
- codex-farm prediction-run manifest pointer (`llm_manifest_json`)
- pass-level JSON in/out directories for pass1/pass2/pass3

If required artifacts are missing, compare marks the corresponding `*_debug_artifacts_present` gate as FAIL and overall verdict becomes FAIL (optionally exit non-zero with `--fail-on-regression`).
