Interactive import and benchmark now share the same Codex submenu mechanics in `cookimport/cli_ui/run_settings_flow.py`, but they do not expose the exact same yes/no step prompts.

Benchmark keeps the full Codex surface set because benchmark run-settings are serialized through `build_benchmark_call_kwargs_from_run_settings(...)`, which carries `line_role_pipeline` and `atomic_block_splitter` into `labelstudio_benchmark(...)`.

Import reuses the same submenu helper with a narrower option set, recipe correction plus knowledge harvest only, because `build_stage_call_kwargs_from_run_settings(...)` does not pass `line_role_pipeline` or `atomic_block_splitter` into `stage(...)`. Showing block labelling in import would therefore create a fake toggle with no runtime effect.
