# Zero-Token CodexFarm Rehearsal Needs Fake Subprocess Not Plan Mode

- `--codex-execution-policy plan` is useful for prompt/work planning, but it exits before `phase_worker_runtime.py` creates worker sandboxes, writes shard inputs, calls the runner, and reads outputs back.
- The repo already has the right seam for a real rehearsal: live orchestrators use `SubprocessCodexFarmRunner`, and that runner can be redirected with `--codex-farm-cmd`.
- A fake `codex-farm` executable lets RecipeImport exercise the actual `process -> worker out dir -> proposal validation -> promotion` loop with zero token use, because RecipeImport still believes it is in execute mode and therefore runs the real folder choreography.
