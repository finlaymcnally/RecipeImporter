---
summary: "Actual costs are a finished-run artifact, not a preview mode. Prompt preview should stay predictive-only."
read_when:
  - When changing `cf-debug preview-prompts` or adding cost-reporting debug commands
  - When deciding where retrospective Codex cost reporting should live
  - When someone asks whether preview should reuse finished-run telemetry
---

Preview and postmortem are different jobs:

- `preview-prompts` exists to estimate likely cost before a live run
- actual costs come from finished-run telemetry and already belong in the post-run artifact set as `prompt_budget_summary.json`

The safe product split is:

- keep preview predictive-only
- surface retrospective cost reporting under a separate `actual-costs` name that points at the post-run artifact
