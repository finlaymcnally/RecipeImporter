---
summary: "The last line-role projection preference in interactive canonical benchmark runs was dead compatibility code and could be removed cleanly."
read_when:
  - "When canonical benchmark runs appear to score `prediction-run/line-role-pipeline/*` instead of the stage-backed run"
  - "When deciding whether `_build_prediction_bundle_from_import_result` should keep line-role projection selection logic"
---

- The stage-backed refactor removed all legitimate callers that wanted line-role projection as the primary score input, but `_build_prediction_bundle_from_import_result()` still carried a `prefer_line_role_projection` escape hatch.
- The interactive `labelstudio_benchmark()` pipelined canonical-text path still passed that flag explicitly, even though it was always `False`; that left misleading code suggesting projected artifacts were still a supported primary scoring lane.
- Removing the flag and the projection resolver is safe because line-role artifacts are still written for diagnostics under `prediction-run/line-role-pipeline/`, while authoritative benchmark bundle loading now always uses the primary stage-backed `stage_block_predictions.json` plus `extracted_archive.json`.
