summary: "Stats dashboard can render blank if dashboard.js includes a raw newline inside a quoted string from template escaping."
read_when:
  - "When debugging a generated dashboard that opens but shows no stats"
  - "When editing runConfigCell tooltip assembly or JS escapes in cookimport/analytics/dashboard_render.py"
---

# Discovery

`cookimport/analytics/dashboard_render.py` stores JS in a Python triple-quoted string.
Using `"\n"` inside that Python template inserts an actual newline character into `dashboard.js`, which breaks JavaScript parsing.

# Contract

Keep the generated JS string literal escaped as `"\\n"` for the run-config hash tooltip path in `runConfigCell`.
If this regresses, browsers stop executing `dashboard.js` and the dashboard appears empty.
