# Discovery

- Page-level checks (`document.scrollWidth - clientWidth`) can stay flat while trend hosts (`#benchmark-trend-chart`, `#compare-control-trend-chart`) still gain internal horizontal overflow over repeated rerenders.
- This happens because host content width can drift even when section/page containment prevents global overflow growth.

# Outcome

- Add explicit host-level drift checks (`host.scrollWidth - host.clientWidth`) sampled over time.
- Pin Highcharts trend chart width to measured host width on each render to prevent rerender-time width creep.
