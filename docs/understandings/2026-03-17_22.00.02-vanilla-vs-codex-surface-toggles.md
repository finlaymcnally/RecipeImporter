# Vanilla vs Codex surface toggles

After the grounded line-role and richer Stage 7 authority work landed, the remaining cleanup question was whether to force those seams on all runs. The desired product shape is narrower than that:

- a run must still support fully vanilla execution with every Codex surface off
- a run must still support fully Codex-backed execution
- recipe correction, line-role, and knowledge harvest must remain independently toggleable

That means the old extra middle state is what should disappear, not vanilla itself. The live runtime surface is now:

- `llm_recipe_pipeline`: `off|codex-recipe-shard-v1`
- `line_role_pipeline`: `off|codex-line-role-shard-v1`
- `llm_knowledge_pipeline`: `off|codex-knowledge-shard-v1`

`deterministic-v1` for line-role was redundant because `off` already preserves the deterministic label-first baseline without running the Codex correction pass. The cleanup therefore removes that live setting from run-settings/UI/CLI while keeping historical readers free to recognize old benchmark artifacts if needed.
