# Benchmark cutdown pass4 reconstruction gap

The prompt-artifact gap was in the existing-output reconstruction path, not in fresh benchmark runs.

- `cookimport/cli.py::_build_codex_farm_prompt_response_log(...)` already knows about pass4 and writes `prompt_task4_pass4_knowledge.txt` for fresh runs when pass4 artifacts exist.
- `scripts/benchmark_cutdown_for_external_ai.py::_reconstruct_full_prompt_log(...)` still used `PASS_DIR_MAP` / `PASS_PIPELINE_MAP` that stopped at pass3, so any processed-book workflow that had to rebuild prompt evidence from `prediction-run/raw/llm/...` silently lost pass4 rows.
- The fix is to include `pass4 -> pass4_knowledge` in the reconstruction maps and source the pass4 `process_run` metadata from `prediction-run/manifest.json -> llm_codex_farm.knowledge.process_run`.
- For pass4 rows, `chunk_id` is the stable fallback identifier when there is no recipe-level `recipe_id`.
