`cookimport/cli.py` currently has three distinct progress systems:

1. `_AllMethodProgressDashboard` for all-method benchmark queue snapshots (multi-line dashboard state object with source/config rows and a task line).
2. `_run_with_progress_status(...)` for generic callback-driven status rendering with live/plain modes, ETA/elapsed augmentation, and worker-activity control-message parsing.
3. Stage import `Live(...)` worker panel (`WorkerDashboard`) with a separate status cache and formatting path.

All three are conceptually similar but implemented separately. The all-method dashboard already matches desired UX shape (overall counters, current focus, queue rows, task line), making it the best extraction seed for a shared core.
