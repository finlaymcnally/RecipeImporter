# Benchmark vs stage knowledge pass wiring

`recipe.knowledge.v1` is the optional pass4 knowledge harvest pipeline, but it is only wired into the stage path.

- `cookimport stage` exposes `--llm-knowledge-pipeline` and runs pass4 when that setting is not `off`.
- `labelstudio-benchmark` prediction generation does not expose or forward `llm_knowledge_pipeline`, so benchmark runs do not execute pass4.
- Benchmark all-method contracts explicitly normalize `llm_knowledge_pipeline` and `llm_tags_pipeline` to `off`, even for CodexFarm benchmark variants.
- `bench quality-run` is deterministic-only and explicitly rejects Codex Farm recipe/knowledge/tags surfaces.
- `bench speed-run` can still hit pass4 only in the `stage_import` scenario, because that scenario calls the full `stage(...)` path using `RunSettings`.
