# Codex Farm precheck failure before pass1 output

- In the 2026-03-06 single-profile benchmark failure, recipeimport successfully wrote `pass1_chunking/in/*.json` files but `pass1_chunking/out/` stayed empty for every book.
- Replaying one prepared `codex-farm process` call against a single pass1 input showed the failure happened before `recipe.chunking.v1` execution:
  - `codex execution precheck failed before process`
  - Codex reported usage-limit failure for the default CLI model `gpt-5.3-codex-spark` with reasoning effort `high`.
- Important detail: this precheck happened even though the benchmark run selected `gpt-5.1-codex-mini` with `medium`. The precheck smoke call uses the current Codex CLI login/runtime state before codex-farm starts the actual pipeline work, so a bad/default Codex CLI model state can block the run before the requested per-run override is used.
- Triage shortcut:
  - if failure text is only `subprocess_exit=1` and omits both `run_id=` and `first_error=`, inspect codex-farm precheck/auth/quota first.
  - empty `pass1_chunking/out/` plus populated `pass1_chunking/in/` is the key artifact signature.
- `codex-farm` documents this behavior and its bypasses (`--no-login-precheck` / `CODEX_FARM_SKIP_LOGIN_PRECHECK=1`) in the external tool docs.
