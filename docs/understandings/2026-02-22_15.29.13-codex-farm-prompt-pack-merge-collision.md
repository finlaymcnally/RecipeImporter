# Understanding: codex-farm prompt-pack merge collision

- The authoritative prompt filename is whatever each pass pipeline JSON references via `prompt_template_path`.
- If both dotted and hyphen prompt filename families exist, codex-farm only uses the referenced path and the duplicate set becomes dead/confusing.
- Safe reconciliation path:
  1. Keep a single filename scheme referenced by `llm_pipelines/pipelines/recipe.*.v1.json`.
  2. Ensure each referenced prompt includes `{{INPUT_PATH}}`.
  3. Keep schema linkage (`output_schema_path`) in sync.
  4. Run both local-pack tests plus orchestrator tests.
