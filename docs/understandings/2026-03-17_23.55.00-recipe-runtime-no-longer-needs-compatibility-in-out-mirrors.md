# Recipe Runtime Can Drop In/Out Mirrors

Once recipe moved to one direct structured call per shard, the old per-recipe `recipe_correction/{in,out}` files stopped being runtime truth. They were only being written as compatibility copies for readers that had not been migrated yet.

The durable replacement surfaces are:
- `recipe_phase_runtime/inputs/*.json` for the compact shard request payloads that match live direct-exec input
- `recipe_phase_runtime/proposals/*.json` for validated shard outputs
- `recipe_correction_audit/*.json` for per-recipe human/debug summaries, now carrying the serialized input/output payloads that older follow-up tooling used to read from separate per-recipe files

That split lets recipe delete the last legacy transport adapter and the compatibility `in/out` mirrors without losing prompt reconstruction, benchmark follow-up context, or per-recipe debugging.
