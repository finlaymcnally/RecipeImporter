# Discovery

- Trend series generation was hardcoded to `strict_accuracy` and `macro_f1_excluding_other` in `dashboard_render.py`, so users could not add or remove plotted fields.
- The chart renderer is shared by both trend panels (`History Table & Trend` and `Compare & Control Analysis`), so field selection must feed one shared series builder.

# Resolution

- Added a `Trend fields` checklist (`Select all` / `Clear`) that accepts any number of numeric benchmark fields.
- Added `trend_fields` to persisted dashboard UI state so selected fields survive reloads and `--serve` state sync.
- Updated series generation to use selected numeric fields while preserving run-group x-axis alignment, paired variant splitting, and trendline/std-band overlays.
