# Spark Usage Attribution

The shared repo log at `~/projects/shared/CodexFarm/var/codex_exec_activity.csv` can show `model=gpt-5.3-codex-spark` for fake tests without proving real Spark billing. The March 14 rows around `02:21Z` to `02:24Z` come from `tests/test_fake_codex_pipeline_pack_demo.py`, which injects a fake `codex` binary on `PATH`; those rows also mark usage as estimated (`estimated=true`, `method=chars_div_4`).

The real March 13/14 Spark usage in this workspace is recorded in `var/codex_exec_activity.csv` and `var/codex_farm.sqlite3`. The relevant run window is `2026-03-14T02:55:17Z` through `2026-03-14T02:55:52Z` UTC, produced by a live `cookimport labelstudio-benchmark run ... --allow-codex --llm-recipe-pipeline codex-farm-3pass-v1 ...` against `saltfatacidheatcutdown`, with artifacts under `data/golden/benchmark-vs-golden/2026-03-13_22.54.57`.

Those two live runs executed `recipe.chunking.v1` and `recipe.schemaorg.compact.v1` with no per-run `codex_model` override, so both fell back to their pipeline defaults. The pipeline JSON files currently default both passes to `gpt-5.3-codex-spark`.
