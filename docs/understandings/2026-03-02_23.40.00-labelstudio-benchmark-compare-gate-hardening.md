# Labelstudio Benchmark Compare Gate Hardening

- `cookimport labelstudio-benchmark compare` now requires explicit benchmark intent metadata for strict prompt/LLM debug checks when candidate source appears to require them.
- `prompt_inputs_manifest_txt` and `prompt_outputs_manifest_txt` are now treated as hard requirements for benchmark-mode compare gates when debug checks are enabled.
- `debug_artifacts` now evaluates raw prompt payload lists from those manifests (`*_manifest_txt` contains file entries), so compare fails when manifest files exist but referenced prompt payloads are missing.
- Missing explicit `prediction_run_config`/`llm_recipe_pipeline` metadata in candidate manifests is surfaced and fails the corresponding `*_debug_artifacts_present` gate to avoid silently relaxing benchmark checks.
