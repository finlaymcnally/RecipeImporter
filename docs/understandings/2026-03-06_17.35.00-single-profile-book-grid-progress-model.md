# Single-profile book grid progress model

- Interactive single-profile multi-book runs call `labelstudio_benchmark(...)` under `_benchmark_progress_overrides(...)` with `suppress_spinner=True`, `suppress_summary=True`, and a shared outer `progress_callback`.
- That means the outer runner receives raw nested callback messages, not pre-rendered inner Rich panels.
- Those nested messages already contain the state needed for a per-book dashboard:
  - `task X/Y` counters for per-book ETA,
  - CodexFarm `running N | active [...]` payloads for worker rows,
  - serialized `__worker_activity__ ...` payloads for non-Codex worker updates.
- The clean seam is `_interactive_single_profile_all_matched_benchmark(...)`: swap the reused all-method queue dashboard for a single-profile-specific dashboard model there, and leave `_run_with_progress_status(...)` as the shared outer box/spinner renderer.
