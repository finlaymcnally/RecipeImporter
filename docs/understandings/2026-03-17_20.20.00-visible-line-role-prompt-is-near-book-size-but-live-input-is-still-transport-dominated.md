On the `2026-03-17_16.37.00` `saltfatacidheatcutdown` run, the canonical source text is about `88,172` `cl100k_base` tokens, the atomized line text is about `88,569`, and the current visible line-role prompt text built from the same rows is about `102,146` across the `5` line-role prompts.

That means the prompt shape itself is only about `~15.8%` larger than the source book now. The real explosion is in live Codex telemetry: line-role reported `1,818,218` input tokens, about `17.8x` the visible prompt text. The cached-input split (`1,501,952`) is the clue that most of that remaining volume is repeated transport/runtime scaffold, not unique cookbook content.

So the next major reduction seam is no longer prompt row metadata. It is the Codex runtime/transport layer that turns one visible prompt into much larger counted input.
