# Discovery

Freeform task sizing now has two layers:
- requested values (`segment_blocks`, `segment_overlap`, optional `target_task_count`), and
- effective runtime overlap (`segment_overlap_effective`) resolved per file by testing valid overlaps and choosing the closest task-count result.

Focus labeling is a separate layer from context: each task carries full context blocks plus `source_map.focus_block_indices`. Prompt text includes both full blocks and explicit focus blocks, but parser/runtime filtering is the real guardrail:
- block mode drops labels whose `block_index` is outside focus,
- span mode drops quote selections for non-focus blocks,
- span mode drops absolute spans if they touch any non-focus block.

This keeps old behavior when focus equals context and `target_task_count` is unset, while allowing higher task counts with smaller per-task labeling burden when new knobs are provided.
