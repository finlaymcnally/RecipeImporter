# Discovery: advanced properties != runtime offload state (2026-03-01)

## Evidence

- `Get-NetAdapterAdvancedProperty` for `vEthernet (FSE HostVnic)` shows:
  - LSO v2 IPv4/IPv6 = Disabled
  - RSC IPv4/IPv6 = Disabled
- But runtime status still reports enabled:
  - `Get-NetAdapterLso -Name "vEthernet (FSE HostVnic)"` => `IPv4Enabled=True`, `IPv6Enabled=True`
  - `Get-NetAdapterRsc -Name "vEthernet (FSE HostVnic)"` => `IPv4Enabled=True`, `IPv6Enabled=True`
- Direct runtime disable attempts without elevation fail with `Access is denied`.

## Practical interpretation

- Prior mitigation likely did not fully disable active offload behavior in the stack.
- Need elevated cmdlet-level disable (`Disable-NetAdapterLso` and `Disable-NetAdapterRsc`) plus explicit post-check using `Get-*` cmdlets.
- Avoid `-ErrorAction SilentlyContinue` during this step so failures are visible.
