## Discovery
- Manual `data-metric-tooltip-key` tagging was not enough to cover repeated metric names rendered from dynamic UI text.
- A stable coverage seam is: build a normalized alias catalog from `Metric help (table)` + `PREVIOUS_RUNS_COLUMN_META`, then auto-resolve keys from candidate text nodes at render refresh.
- Auto-targeting chart axis/legend text and inline metric `<code>` nodes closes the most visible gaps without adding visual affordances.

## Wiring points
- Alias catalog + key resolver: `cookimport/analytics/dashboard_render.py` JS template (`buildMetricTooltipAliasCatalog`, `metricTooltipResolveKeyFromText`).
- Auto-target refresh pass: `refreshMetricTooltipTargets` with `METRIC_TOOLTIP_AUTO_TARGET_SELECTOR`.
