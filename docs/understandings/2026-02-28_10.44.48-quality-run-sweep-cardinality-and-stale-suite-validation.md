# Quality-Run Sweep Cardinality And Stale Suite Validation

- Existing `data/golden/bench/quality/suites/pulled_representative.json` can become stale: `bench quality-run` validates all `targets[]`, not only `selected_target_ids`, so missing gold paths in non-selected rows still fail the run.
- Regenerating a fresh suite with `bench quality-discover --prefer-curated` produced a valid 3-target curated selection (`saltfatacidheatcutdown`, `thefoodlabcutdown`, `seaandsmokecutdown`).
- Running `quality-run` on that fresh suite with `--include-deterministic-sweeps --search-strategy race` started round 1 (probe) with `286` configs, confirming large cardinality growth from sweep expansion.
- Interrupting this large threaded run can emit `cannot schedule new futures after interpreter shutdown`; treat that as interruption fallout, not benchmark-quality signal.
