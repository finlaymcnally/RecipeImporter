# AI readiness seam note

This repo is better than average for AI work because it has a strong docs tree, package READMEs, a domain-sliced test suite, and a real smoke lane (`116 passed` in about `10s` on 2026-03-17).

The main structural limit is that important runtime knowledge still collapses into a few huge coordinator modules, especially `cookimport/cli.py` (~30.6k lines), plus several cross-package mutual dependencies (`labelstudio <-> parsing`, `labelstudio <-> llm`, `labelstudio <-> staging`, `llm <-> parsing`, `llm <-> staging`, `core <-> parsing`, `bench <-> config`).

Practical consequence: AI can often start from docs and tests, but meaningful changes still tend to require loading large implementation files and understanding cross-domain internals instead of staying at a small explicit interface.
