## Discovery

- Previous coupling existed in two explicit seams:
  - `computePreviousRunsFilterResult()` called `renderCompareControlPanel({ records: matchedRecords })`.
  - `renderBenchmarkTrendChart()` passed filtered/sorted table records into `renderCompareControlDynamicChart(...)`.
- Previous coupling also existed in actions:
  - `Filter to subset` called `syncCompareControlSelectionToTableFilters()` and wrote compare groups into table column filters.
  - Previous Runs view presets captured/applied `compare_control` state together with table filter state.

## Decoupled behavior

- Compare & Control now has its own render path (`renderCompareControlSection`) fed by `compareControlSourceRecords()`.
- Table filter computation and trend-chart render no longer push data into Compare & Control.
- Group selection is local to Compare & Control (`applyCompareControlSelectionSubset` + `compareControlRecordsForState`) and no longer mutates table filters.
- Previous Runs view presets now save/apply table state only; compare/control state remains persisted separately in dashboard UI state.
