# Prompt Preview Line-Role Must Follow Whole-Book Correction

The drift:

- live `codex-line-role-v1` now reviews the full ordered candidate set in bounded local batches
- `cookimport/llm/prompt_preview.py` had still been selecting only rows that looked unresolved/non-rule from saved labeled-line metadata

Why it matters:

- prompt preview understated line-role prompt volume
- rebuilt preview prompts no longer matched the actual line-role contract after the whole-book correction refactor

Correct seam:

- preview should build the same ordered candidate list it always uses
- batch all candidates
- pass `deterministic_label` and `escalation_reasons` into `build_canonical_line_role_prompt(...)`
