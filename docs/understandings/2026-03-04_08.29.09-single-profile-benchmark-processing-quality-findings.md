## Scope
- Run analyzed: `data/output/2026-03-04_06.58.16/single-profile-benchmark`
- Profiles: dinnerfor2, roastchickenandotherstories, saltfatacidheat, seaandsmoke, thefoodlab
- Modes: `vanilla` and `codexfarm`

## High-signal findings
- Report-counter drift exists in 4/5 books: `totalRecipes`/`totalTipCandidates`/`totalTopicCandidates` report as `0` while artifacts contain many records.
  - Example: `thefoodlab` reports `totalRecipes=0`, but `final drafts` contains `39` recipe JSON files and `topic_candidates.json` contains `1230` rows.
- Stage-block blockization mismatch vs gold blocks causes eval hard-fail on 3 books:
  - roastchicken: `extra_gold=37`
  - saltfatacidheat: `extra_gold=223`
  - thefoodlab: `extra_gold=617`
- Only dinnerfor2 and seaandsmoke produce valid stage eval reports against current gold.
  - dinnerfor2: codexfarm improves strongly vs vanilla (`strict_accuracy 0.741 vs 0.459`, `macro_f1_excluding_other 0.496 vs 0.354`).
  - seaandsmoke: both poor (`strict_accuracy 0.209 codexfarm`, `0.187 vanilla`; macro-F1 both ~`0.085-0.089`).
- Final draft schema normalization is inconsistent:
  - Across `174` vanilla recipes: `name=0`, `ingredients array=0`, `instructions array=0`, while `recipe.title=174` and `steps with instruction=174`.
  - Across `174` codex recipes: partial normalization (`name=48`, `ingredients array=64`, `instructions array=43`).
- Tip/topic extraction is over-broad on narrative-heavy books.
  - `saltfatacidheat` tips: `167`, avg tip length ~`208` chars.
  - `thefoodlab` tips: `136`, avg tip length ~`241` chars.
  - topic candidates are very large (`1048` and `1230`) and include short/non-actionable text.

## Interpretation
- There are at least two quality problems:
  - true extraction/segmentation quality issues (especially seaandsmoke + blockization mismatch books), and
  - reporting/instrumentation drift where report totals no longer reflect produced artifacts.
- Codexfarm currently changes labels materially (up to ~60% block-label flips in some books) and is not consistently better; improvements are book-dependent.

## Immediate fix order
1. Stabilize report truth: derive report totals from produced artifacts when available (or fail loud on mismatch).
2. Fix blockization compatibility with pulled gold before scoring (or add deterministic remap path).
3. Normalize final draft fields (`name`, `ingredients`, `instructions`) from `recipe.title`/`steps` at write time.
4. Add stricter tip/topic candidate filters for narrative intro material and long prose spans.
