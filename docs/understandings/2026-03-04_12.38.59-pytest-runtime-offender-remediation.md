# Runtime offender remediation findings (2026-03-04)

- The biggest wall-clock wins came from removing real stage/worker orchestration from tests that only assert report/output contract shape. A shared helper (`tests/fast_stage_pipeline.py`) now mocks `stage_one_file` and writes minimal deterministic artifacts.
- `tests/cli/test_cli_output_structure_text_fast.py` and hot-path `tests/ingestion/test_performance_features.py` now force serial executor resolution in tests, avoiding process/thread pool probe overhead that dominated wall time.
- Dashboard browser drift tests were bottlenecked by fixed waits. Replacing long static waits with timer clamping + render-count polling kept behavior checks intact while cutting runtime sharply.
- Some slow modules were better handled by splitting:
  - EPUB backend contract assertions moved to `tests/cli/test_cli_output_structure_epub_fast.py`; real EPUB integration checks remain in `tests/cli/test_cli_output_structure_slow.py`.
  - OCR-heavy scanned-PDF conversion moved to `tests/ingestion/test_pdf_importer_ocr_slow.py`; `tests/ingestion/test_pdf_importer.py` now stays in the fast path.
- Scheduler/progress timing tests can regress if synthetic sleeps are cut too aggressively; keep sleep reductions above timeout/heuristic thresholds so scheduler utilization assertions remain meaningful.
