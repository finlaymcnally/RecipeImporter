# Current ChatGPT Browser Picker Is Mode-Based, Not Model-Id-Based

- On 2026-03-17, the local Oracle 0.8.6 browser run reported picker options like `Instant`, `Thinking`, `ProResearch-grade intelligence`, and `Configure...` rather than stable labels such as `GPT-5.2`.
- That means both `gpt-5.2-pro` and `gpt-5.2` are brittle browser picker targets in this environment; the UI has shifted away from the labels Oracle expects.
- The practical repo-side fix is to keep the browser visible and pass `--browser-model-strategy ignore` so Oracle skips picker automation entirely.
- With picker-ignore mode, the active/manual model in the visible ChatGPT window is the real source of truth. The CLI `--model` value is still recorded in launch metadata, but it does not drive browser-side switching.
- The separate `No cookies were applied` failure remains a profile/login issue; using a stable Oracle browser profile directory lets a manual login persist across runs.
