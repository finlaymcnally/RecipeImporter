summary: "AI Effort should classify vanilla benchmark runs as `AI off` even when pipeline keys are missing."
---

- `AI Effort` is derived in two places that must stay in sync:
  - Backend compare/control engine (`ai_effort_label_for_record`).
  - Dashboard JS renderer (`aiEffortLabelForRecord`).
- Relying only on `llm_recipe_pipeline=off` can miss some vanilla cases when metadata is sparse/stale.
- Using benchmark-variant inference (`vanilla`) as fallback keeps grouping/dropdowns stable and separates unknown (`-`) from true AI-off runs.
