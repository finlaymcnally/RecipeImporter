# Staging Contract Edge Cases

Cookbook's `RecipeDraftStagingV1` validator accepts unresolved ingredient IDs as non-empty raw placeholders and unresolved units as `null`, but it still enforces:
- `exact` / `approximate` lines require `input_qty > 0`
- `unquantified` lines must omit/null qty and unit

In a scan of existing recipeimport outputs, most direct failures were blank IDs/units that resolver can normalize, but a smaller set still failed after resolver due to invalid quantity-kind + qty combinations (for example `approximate` with null qty, or `exact` with `0`).
