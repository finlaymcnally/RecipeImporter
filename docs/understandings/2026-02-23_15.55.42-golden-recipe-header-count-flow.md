# Discovery

`labelstudio-export --export-scope freeform-spans` already had all data needed to count recipes, but only span/task counts were persisted.

# Implemented flow

1. Freeform export now writes recipe-header counts into `exports/summary.json`:
- `counts.recipe_headers`
- `recipe_counts.recipe_headers` (deduped)
- `recipe_counts.recipe_headers_raw` (raw rows)

2. Recipe-header counting is based on `RECIPE_TITLE` spans, normalized through freeform label normalization, deduped by:
- `source_hash`
- `source_file`
- `start_block_index=min(touched_block_indices)`
- `end_block_index=max(touched_block_indices)`

3. Benchmark/eval (`labelstudio-benchmark`, `labelstudio-eval`) now attach `recipe_counts` into `eval_report`:
- golden recipe headers from export summary when available
- predicted recipes from prediction-run context (`recipe_count` / processed report fallback)
- predicted-vs-golden delta/ratio

4. Freeform `eval_report.md` now shows a `Recipe count diagnostics` section so run reviewers can directly see "import picked up vs golden recipes".
