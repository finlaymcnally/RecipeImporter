Oracle's browser session support is now good enough to recover and inspect an existing ChatGPT conversation, but not yet good enough to continue it with a new user turn. The saved Oracle session metadata already stores `conversationUrl` and `conversationId`, and `resumeBrowserSession(...)` can reopen that chat and capture the assistant output, but the current CLI only exposes inspection via `oracle session <id>`. There is no supported command that takes an existing session id plus a new prompt and attachments and submits a second message into the same finished conversation. That same-chat turn-2 send capability needs to be added in the editable Oracle workspace, not papered over in recipeimport.
