Key findings from the 2026-03-06_00.44.16 run audit:

- The first fix removed the old `line_index -> atomic_index` corruption, but naive exact-text occurrence matching still mispaired some duplicate short texts (for example repeated `Salt` / headings). A global exact-text sequence alignment is a safer matcher because it respects surrounding order and leaves ambiguous duplicates unmatched.
- Separate from exporter matching, some raw `line_role_predictions.jsonl` rows were internally invalid before export. The common pattern was `_sanitize_prediction(...)` changing the final label during fallback (`sanitized_yield_to_instruction`, `outside_span_howto_hard_deny`, neighbor-fragment rescue, etc.) without re-adding that final label to `candidate_labels`.
- Representative source rows from the audited run:
  - `label=INSTRUCTION_LINE` with `candidate_labels=[YIELD_LINE, INGREDIENT_LINE, OTHER]`
  - `label=KNOWLEDGE` with `candidate_labels=[HOWTO_SECTION, OTHER]`
  - `label=INGREDIENT_LINE` with `candidate_labels=[OTHER]`
- Consequence: even with a correct exporter join, historical run artifacts remain inconsistent until the line-role pipeline is rerun with the sanitizer fix in place.
