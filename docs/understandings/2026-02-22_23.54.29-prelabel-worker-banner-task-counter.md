# Discovery

- The parallel prelabel path emitted a phase-only message (`Running freeform prelabeling with up to N workers...`) after the initial `task 0/Y` update.
- That phase-only update became the latest spinner text while workers were active, so users temporarily lost visible `X/Y` progress and ETA context until first completion.
- The fix is to keep the worker kickoff message in `task X/Y` shape and append worker metadata (`(workers=N)`), preserving spinner counter parsing throughout the parallel phase.
