# Benchmark Selective Retry Contract Sync

The review gap on selective retry was real because the orchestrator had been deciding retry eligibility from failure categories alone, while the runner's accepted partial-output contract also required `no last agent message` evidence plus benchmark coverage thresholds.

Current fix shape:

- `cookimport/llm/codex_farm_runner.py` now serializes `error_summary` into the per-pass `process_run` payload.
- `cookimport/llm/codex_farm_orchestrator.py::_selective_retry_eligible_process_run(...)` now calls the same `_is_recoverable_no_last_agent_message_failure(...)` helper the subprocess runner uses, with the orchestrator supplying the relevant input/output bundle counts.
- Focused orchestrator tests were refreshed so positive retry cases use 4-of-5 partial-output coverage, while negative cases prove retry is blocked when summary evidence is missing or coverage is too low.
- `cookimport/cli.py::labelstudio_benchmark(...)` now mirrors a direct `llm_manifest_json` artifact link into the benchmark root `run_manifest.json`, restoring the reviewer breadcrumb the OG plan expected.
