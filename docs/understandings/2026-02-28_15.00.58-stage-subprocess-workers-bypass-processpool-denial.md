# Stage Subprocess Worker Fallback Discovery

- In this environment, process-pool startup fails due SemLock restrictions (`PermissionError: [Errno 13] Permission denied`), but plain subprocess spawning still works.
- Using thread executor slots to orchestrate subprocess worker invocations allows stage to retain concurrent job fanout without relying on blocked process-pool primitives.
- Telemetry and worker dashboard visibility are preserved by assigning explicit `SubprocessWorker-N` labels and recording statuses in `processing_timeseries.jsonl`.
- This is a runtime workaround for stage concurrency, not a restoration of host process-pool capability; `/dev/shm` restrictions still apply.
