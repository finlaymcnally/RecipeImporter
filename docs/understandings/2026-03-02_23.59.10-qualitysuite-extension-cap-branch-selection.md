# QualitySuite Extension-Cap Selection Branch

In `cookimport/bench/quality_suite.py`, `_select_representative_target_ids(...)` groups targets by normalized source extension, shuffles extension keys with the suite seed, and picks one target from each winning extension when `max_targets` is smaller than total extension count.

Because capacity is fully consumed by extension winners in this branch, no strata-fill pass runs. The test contract should assert deterministic same-seed output and extension-diversity invariants, not one fixed ID pair.
