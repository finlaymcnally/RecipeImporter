# Compare & Control dynamic chart seam

- Previous behavior: `renderBenchmarkTrendChart()` rendered the same benchmark trend series into both chart hosts, so compare/control chart did not reflect compare/control state.
- Shared-row seam: `currentPreviousRunsFilterResult()` computes filtered rows once; `renderCompareControlPanel(...)` and chart rendering both consume that same visible row set.
- New seam: compare/control now builds its own chart definition (`buildCompareControlChartDefinition`) from:
  - filtered rows,
  - normalized compare/control state,
  - field catalog metadata.
- Current chart type is `scatter` (builder-based registry), with:
  - numeric compare fields -> row-level scatter,
  - categorical compare fields -> group-mean scatter,
  - split-by -> one scatter series per split segment.
- Rendering seam remains host-based (`renderCompareControlChartHost`) so future chart types can reuse fallback/cleanup/width-pin lifecycle.
