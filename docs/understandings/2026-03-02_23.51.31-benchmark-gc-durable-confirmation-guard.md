# Benchmark GC Durable Confirmation Guard

- `bench gc` now prunes run roots only when benchmark CSV rows under that run root already contain durable metrics (`per_label_json` or strict/macro/boundary fields).
- If history CSV is missing, or rows for a run root are not durable, GC keeps that run root and reports an unconfirmed-history warning instead of deleting.
- When a run root is deleted, GC preserves durable benchmark rows and prunes only stale non-durable rows referencing the deleted root.
- Regression tests now cover:
  - dashboard row retention after prune,
  - apply idempotence,
  - backup filename timestamp format (`YYYY-MM-DD_HH.MM.SS`).
