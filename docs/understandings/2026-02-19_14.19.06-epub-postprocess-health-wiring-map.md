# EPUB postprocess + health wiring map

- `cookimport/plugins/epub.py:_extract_docpack(...)` is now the join point:
  - `legacy`/`unstructured` extraction happens first (`_extract_docpack_with_ebooklib` or `_extract_docpack_with_zip`).
  - shared cleanup runs next: `postprocess_epub_blocks(blocks)`.
  - signal enrichment runs after cleanup so split/normalized lines get correct features.
- `markitdown` skips this shared postprocess on purpose and only gets normal signal enrichment.
- Conversion-level health is attached in `EpubImporter.convert(...)`:
  - `compute_epub_extraction_health(blocks)` writes raw artifact `epub_extraction_health.json`.
  - warning keys from health are appended directly into `ConversionReport.warnings`.
- Spine metadata contract changed from tuple to typed `EpubSpineItem(path, media_type, item_id, properties)` so zip-fallback extraction can apply the same nav/TOC skipping logic as ebooklib mode.
