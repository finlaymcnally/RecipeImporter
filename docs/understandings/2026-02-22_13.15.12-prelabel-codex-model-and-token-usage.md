# Prelabel Codex model + token-usage wiring

- Interactive prompts alone are insufficient; model controls must propagate through `run_labelstudio_import(...)` and `run_labelstudio_decorate(...)` into `generate_pred_run_artifacts(...)` and `_build_prelabel_provider(...)`.
- The effective command should be resolved centrally (`codex exec -` + optional `--model`) so report fields always show the real command/model used.
- Token usage should be provider-level aggregation (`CodexCliProvider.usage_summary()`), always enabled for prelabel/decorate runs, then copied into `prelabel_report.json` / `decorate_report.json`; this keeps CLI wrappers thin and consistent with non-interactive runs.
