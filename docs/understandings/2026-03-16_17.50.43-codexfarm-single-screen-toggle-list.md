The shared CodexFarm submenu now uses a small stateful prompt instead of a checkbox prompt or a chain of separate yes/no prompts. Each row renders with fixed-width `Yes` and `No` columns so the boxes line up vertically. Up/down moves between rows, left/right changes the current row's active box in place, the active box is marked directly in the row instead of relying only on terminal color, Enter can still flip the current row, and `Continue` commits the whole list at once.

This is the best fit for the current interactive CLI because the standard `_menu_select(...)` helper cannot keep a row-local yes/no highlight in place; the submenu needs a tiny custom prompt-toolkit surface to support row movement plus left/right toggles without regenerating the menu.

The submenu is still flow-scoped. Import shows recipe correction plus knowledge harvest; benchmark shows recipe correction, block labelling, and knowledge harvest; `Continue` commits the current row states back into `RunSettings`.
