# Discovery

- Showing `(workers=N)` only on the kickoff line is easy to miss because later completion updates replace that line.
- Operators reading a long `task X/Y` run can mistake parallel work for serial execution when worker metadata disappears.
- Keep `(workers=N)` on each parallel completion update to make concurrency visible throughout the phase.
