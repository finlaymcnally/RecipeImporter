# Single-Pass Line-Role Still Duplicates Neighbor Context

The current live single-pass line-role payload builder in `cookimport/parsing/canonical_line_roles.py` still writes `prev_text`, `current_line`, and `next_text` into every row of the worker `in/*.json` file. `cookimport/llm/prompt_preview.py` mirrors the same schema for predictive previews.

That made sense when line-role prompts reviewed isolated shortlist rows, but it is redundant in the current file-backed design because each shard file already contains one ordered contiguous slice of the book. Once the model has the full ordered row list, most rows can recover their left/right neighbors from adjacent rows in the same file without paying to copy those neighbors into every row object.
