# Sandbox Process-Pool Fallback Surface Map

- In this runtime, both `/dev/shm` writes and `ProcessPoolExecutor` probing fail with `PermissionError: [Errno 13]`.
- `quality-run` now resolves experiment executor mode to `subprocess` in auto mode when process-pool probe fails (`cookimport/bench/quality_runner.py`).
- QualitySuite fallback is layer-specific: if effective experiment parallelism is `1`, execution remains single-worker.
- All-method config fanout in `cookimport/cli.py` falls back from process workers to thread workers (not immediate serial).
- `cookimport stage` still falls back to serial on multiprocessing `PermissionError`.
- `cookimport/labelstudio/ingest.py` split conversion still falls back to serial job execution on multiprocessing `PermissionError`.
- This explains why “parallelism fixed” can be true for one workflow layer and still feel serial end-to-end in sandbox runs.
