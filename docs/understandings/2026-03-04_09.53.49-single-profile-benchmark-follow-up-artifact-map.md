# Discovery

The group `upload_bundle_v1` for `single-profile-benchmark` contains enough derived rows to build follow-up packets without rerunning benchmarks.

Key locators:
- Changed lines: `_upload_bundle_derived/root/changed_lines.codex_vs_vanilla.jsonl` (4136 rows).
- Triage + stage reasoning: `_upload_bundle_derived/root/01_recipe_triage.packet.jsonl` plus `analysis.stage_separated_comparison` in `upload_bundle_index.json`.
- Call telemetry: `_upload_bundle_derived/starter_pack_v1/02_call_inventory.jsonl` (434 calls) and `analysis.call_inventory_runtime` summary.
- Existing low-confidence packet can be empty when sampled prediction files are incomplete; joining changed lines against per-run codex `line-role-pipeline/line_role_predictions.jsonl` across all books restores a usable packet.
- Prompt traces for recipe-level regressions are fully recoverable via per-book `codexfarm/codexfarm/full_prompt_log.jsonl` with recipe_id/call_id joins.

Practical caveat:
- Candidate label margins are not currently emitted (candidate sets + confidence only), so regression packet should explicitly mark margin unavailable.
