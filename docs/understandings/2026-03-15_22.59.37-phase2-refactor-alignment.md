`docs/plans/phase2-label-source-of-truth.md` is spiritually aligned with `docs/reports/Refactor.md`.

They agree on the core move: labels become authoritative, grouping becomes deterministic and label-driven, stage artifacts become first-class, and Label Studio / benchmark flows should consume the same authoritative Stage 2 output instead of rerunning diagnostics later.

The only small seam is fallback wording. `docs/reports/Refactor.md` explicitly says grouping failure should preserve candidate evidence. The Phase 2 ExecPlan implies that through compatibility and diagnostics, but does not state the failure-path rule as directly. Future edits should keep that fallback visible so label-first does not accidentally erase useful comparison evidence during migration.
