# Discovery: active WSL host vNIC renamed to FSE HostVnic (2026-03-01)

## Evidence

- Current up Hyper-V adapter:
  - `vEthernet (FSE HostVnic)` (`Hyper-V Virtual Ethernet Container Adapter`)
- Current offload settings on that adapter remain enabled:
  - `*LsoV2IPv4 = Enabled`
  - `*LsoV2IPv6 = Enabled`
  - `*RscIPv4 = Enabled`
  - `*RscIPv6 = Enabled`
- New warnings after reboot (`17:58:54` boot) target:
  - `NicFriendlyName: FSE HostVnic`
  - Event `285` (`OID_GEN_STATISTICS took too long`)
- Linux journal in same boot still logs repeated:
  - `WSL (...) ERROR: CheckConnection: getaddrinfo() failed: -5`

## Practical interpretation

- Prior admin commands targeted a stale adapter alias (`vEthernet (WSL (Hyper-V firewall))`) and may not have affected the active WSL networking path.
- Remediation must target whichever Hyper-V/vEthernet adapter is currently up, not a hard-coded legacy name.

## Correct approach

- Enumerate active Hyper-V/vEthernet adapters dynamically.
- Disable LSO/RSC on each target.
- Restart adapter(s), then `wsl --shutdown`.
