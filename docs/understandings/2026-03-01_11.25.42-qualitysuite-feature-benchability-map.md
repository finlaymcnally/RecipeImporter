# Discovery: QualitySuite Feature Benchability Map

Date: 2026-03-01

Key findings from code/docs verification:

- QualitySuite deterministic sweeps (`bench quality-run --include-deterministic-sweeps`) currently auto-expand Priority 2-6 knobs only: section detector, multi-recipe splitter, ingredient missing-unit policy, instruction step segmentation, and p6 time/temp/yield selectors.
- Priority 1 (recipe-likeness thresholds) is still benchmarkable in QualitySuite, but via explicit `experiments`/`levers` `run_settings_patch` entries (not the built-in deterministic sweep set).
- Priority 7 webschema behavior is benchmarkable in QualitySuite when sources are webschema-capable (`.html/.htm/.jsonld`, or schema-like `.json`); all-method variants auto-expand `web_schema_policy` for those sources.
- Priority 8 boundary segmentation metrics are not part of QualitySuite `quality-run`/tournament gating; they are exposed through `bench eval-stage` knobs/artifacts.
- Current curated QualitySuite run `data/golden/bench/quality/runs/2026-03-01_11.08.23/suite_resolved.json` contains EPUB-only targets, so webschema/structured-lane effects are not exercised in that suite.

Practical implication:

- For P1/P7 comparisons, use schema-v2 experiments/levers with explicit patches (or source selection that includes web inputs).
- For P8, run `bench eval-stage` separately (possibly on winning candidates) to compare boundary diagnostics.
