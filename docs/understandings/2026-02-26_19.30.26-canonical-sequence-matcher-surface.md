# Discovery: current canonical SequenceMatcher surface

Observed behavior:

- `cookimport/bench/eval_canonical_text.py` imports stdlib `SequenceMatcher` directly and uses it in `_align_prediction_blocks_legacy(...)` with `autojunk=False`.
- `_align_prediction_blocks_to_canonical(...)` currently always runs legacy alignment; fast path remains deprecated and reports deprecation telemetry.
- Alignment timing is already split into subphases and emitted under `report["evaluation_telemetry"]["subphases"]`, including `alignment_sequence_matcher_seconds`.

Durable rule:

- Speed work for canonical eval should preserve alignment/scoring behavior and only swap matcher implementation behind the same call surface.
