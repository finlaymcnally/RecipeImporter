# Discovery: offload disable applied but core failure signals persisted (2026-03-01)

## Confirmed applied

- Adapter: `vEthernet (FSE HostVnic)`.
- Offloads now disabled:
  - `Large Send Offload Version 2 (IPv4)`
  - `Large Send Offload Version 2 (IPv6)`
  - `Recv Segment Coalescing (IPv4)`
  - `Recv Segment Coalescing (IPv6)`
- `Wi-Fi` packet coalescing also disabled.

## Persisting signals in the next boot

- Linux journal (`boot 0`) still shows repeated:
  - `WSL (...) ERROR: CheckConnection: getaddrinfo() failed: -5`
- Windows `Microsoft-Windows-Hyper-V-VmSwitch-Operational` still shows warning `285`:
  - `NicFriendlyName: FSE HostVnic`
  - plus intermittent `IOCTL_NIC_SEND_STATUS_EX` delays.

## Practical interpretation

- Adapter offload mitigation was necessary to test, but not sufficient.
- Remaining likely path is WSL networking architecture / Hyper-V firewall integration (`FSE HostVnic`) rather than workload-level pressure.
- Next trial should target `.wslconfig` networking mode / firewall path (e.g., disable WSL Hyper-V firewall integration) and re-check event cadence.
