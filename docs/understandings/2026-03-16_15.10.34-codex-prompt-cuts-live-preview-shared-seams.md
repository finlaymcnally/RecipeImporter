  - "When deciding whether a token optimization belongs in prompt reconstruction, the live orchestrator, or a shared contract helper."

Recipe correction prompt bodies and preview reconstructions both serialize `MergedRecipeRepairInput`, so omitting empty fields belongs in a shared serializer rather than only in `prompt_preview.py`. Knowledge prompt counts are also shared: both live harvest and preview reconstruction call `build_knowledge_jobs(...)`, so lane-gating there is the single place that shrinks prompt volume for both flows.

The one extra runtime seam is the knowledge orchestrator: once `build_knowledge_jobs(...)` can skip every chunk, `run_codex_farm_knowledge_harvest(...)` must short-circuit before invoking Codex or writing misleading empty-output manifests. Without that guard, preview counts would drop but live runs would still pay orchestration overhead and potentially fail on an empty job directory.
