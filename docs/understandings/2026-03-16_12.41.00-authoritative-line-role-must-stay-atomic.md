The authoritative Stage 2 cutover created a subtle benchmark regression surface: the prediction run could reuse correct labels but still serialize them with the wrong coordinate system.

Good historical canonical-text runs wrote `line-role-pipeline/stage_block_predictions.json` and `line-role-pipeline/extracted_archive.json` in canonical atomic-span coordinates:
- `block_count` matched the number of projected atomic lines
- notes said `Projected from canonical line-role predictions.`
- the archive rows carried `line_role_projection` metadata and `line_index` coordinates

The first label-first reuse version instead wrote:
- stage predictions from `build_authoritative_stage_block_predictions(...)` over source blocks
- extracted archive rows copied directly from `label_first_result.archive_blocks`

That preserved text coverage but collapsed the scored surface from `1768` atomic rows to `1471` source-block rows on `saltfatacidheatcutdown`, which especially hurt `RECIPE_TITLE`, `RECIPE_VARIANT`, and `INSTRUCTION_LINE`.

The fix is to keep using `authoritative_lines_to_canonical_predictions(...)` as the source of truth, but serialize the scored artifact pair through the existing canonical projection builders:
- `project_line_roles_to_freeform_spans(...)`
- `build_line_role_stage_prediction_payload(...)`
- `build_line_role_extracted_archive_payload(...)`
