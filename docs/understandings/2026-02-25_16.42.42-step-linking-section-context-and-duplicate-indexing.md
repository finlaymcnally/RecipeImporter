# Step Linking: Section Context + Duplicate Safety

- Section context is optional in `assign_ingredient_lines_to_steps(...)`:
  - `ingredient_section_key_by_line`
  - `step_section_key_by_step`
- The linker still uses global matching, but same-section candidates are preferred for near ties.
- Two special passes now respect section scope when multiple sections exist:
  - `all ingredients` phrases
  - collective-term fallback (`spices` / `herbs` / `seasonings`)
- Duplicate ingredient text cannot be tracked safely by text equality alone; internal assignment now tracks ingredient identity by original index while computing passes/sorting, then removes that internal index before returning staged lines.
