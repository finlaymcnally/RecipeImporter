For `data/golden/benchmark-vs-golden/2026-03-06_15.22.11`, the collapse in visible Codex-vs-vanilla lift is mostly baseline catch-up, not a pure Codex quality crash.

Useful factor split from on-disk runs:

- Deterministic line-role is a major direct lift-collapser.
  - Sea legacy stack with `atomic_block_splitter=atomic-v1` and `line_role_pipeline=off` scored strict `0.3966` / macro `0.3405` (`2026-03-03_12.22.40_sea-baseline-off`).
  - Sea legacy stack with `atomic_block_splitter=atomic-v1` and `line_role_pipeline=deterministic-v1` scored strict `0.5261` / macro `0.3324` (`2026-03-03_00.58.55_sea-line-role-det-canonical`).
  - FoodLab legacy stack with `atomic_block_splitter=atomic-v1` and `line_role_pipeline=off` scored strict `0.3736` / macro `0.3321` (`2026-03-03_12.22.10_foodlab-baseline-off`).
  - FoodLab legacy stack with `atomic_block_splitter=atomic-v1` and `line_role_pipeline=deterministic-v1` scored strict `0.4683` / macro `0.3090` (`2026-03-03_00.34.16_line-role-gated-foodlab`).
  - The deterministic line-role diagnostics in the new vanilla run also show many baseline corrections that used to be Codex-style wins, especially outside-recipe cleanup and title/ingredient recovery.

- `atomic-v1` by itself is not the main reason Codex lost lift. In older legacy-stack ablations it was mixed and sometimes negative:
  - Sea legacy `off/off` at `atomic=off` scored strict `0.4303` / macro `0.4117`.
  - Sea legacy `off/off` at `atomic=atomic-v1` scored strict `0.3966` / macro `0.3405`.
  - Sea legacy `deterministic-v1` with `atomic=off` slightly beat `atomic=atomic-v1` (`0.5311/0.3567` vs `0.5261/0.3324`).
  This suggests `atomic-v1` alone was not the boost killer; the later shared parser/splitter stack made the big difference.

- The new benchmark knowledge pass appears to contribute little or nothing to measured quality lift here.
  - New Codex runs enable `llm_knowledge_pipeline=codex-farm-knowledge-v1`, but same-run Codex vs vanilla is identical on `KNOWLEDGE` for Salt and Sea and nearly identical on FoodLab.
  - Old Codex (`2026-03-06_00.44.16`) vs new Codex (`2026-03-06_15.22.11`) quality gains are not concentrated in `KNOWLEDGE`; they show up more in ingredients, instructions, titles, and yield.
  - Conclusion: pass4 knowledge mostly added runtime/token cost in this benchmark slice rather than visible score lift.

- The biggest unasked-but-real factor is the shared deterministic parser stack, not atomic alone:
  - `section_detector_backend=shared_v1`
  - `multi_recipe_splitter=rules_v1`
  - `instruction_step_segmentation_policy=always`
  - `epub_unstructured_html_parser_version=v1`
  - `epub_unstructured_skip_headers_footers=true`
  This stack, combined with deterministic line-role, is what pushed vanilla much closer to Codex.
