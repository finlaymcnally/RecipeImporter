# Compact pipeline defaults surface audit

The repo had already flipped compact prompt assets on in the main control surfaces:

- `RunSettings` defaults use `recipe.schemaorg.compact.v1` and `recipe.final.compact.v1`.
- CLI options for benchmark/import paths also default to those compact pipeline ids.
- Line-role prompt format resolves to `compact_v1` when `COOKIMPORT_LINE_ROLE_PROMPT_FORMAT` is unset.

The remaining inconsistency was two helper function signatures in `cookimport/labelstudio/ingest.py` that still defaulted pass2/pass3 to legacy ids. Updating those signatures makes compact the default across the same Label Studio ingestion path as well.
