# benchmark_cutdown project-context digest wiring

`_write_readme(...)` is the single source for root README content, and `_write_root_summary_markdown(...)` embeds README into flattened `benchmark_summary.md`.
So context text should be generated once and appended in `_write_readme(...)`.

`main()` is the correct wiring point for pointer/hash metadata because it assembles both root JSON outputs:
- add `project_context` object to `comparison_summary.json`
- add `project_context_*` and `project_context_digest_included` to `process_manifest.json`

Metadata extraction can stay deterministic by reading `docs/AI_Context.md` bytes for SHA-256 and parsing title/date markers from markdown content with fallback to file mtime date.
