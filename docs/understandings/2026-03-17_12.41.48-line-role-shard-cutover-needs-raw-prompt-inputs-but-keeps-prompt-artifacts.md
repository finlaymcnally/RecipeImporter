read_when:

  - "When a shard-worker phase needs JSON manifests for inspection but raw text inputs for the live CodexFarm pipeline."

The line-role cutover could not use the shared runtime unchanged because `line-role.canonical.v1` still consumes raw prompt text, not a JSON task object. The durable fix was to let `ShardManifestEntryV1` carry both inspectable JSON `input_payload` and optional raw `input_text`, then have the runtime write raw text only for the worker input file while leaving the manifest JSON-readable.

That was not enough by itself because benchmark prompt exports and reviewer tooling still read `line-role-pipeline/prompts/*`. The line-role phase therefore has to keep writing `prompt_*.txt`, `response_*.txt`, `parsed_*.json`, `codex_prompt_log.dedup.txt`, and `parse_errors.json` from the shard runtime results even though the old one-shot batch executor is gone.
