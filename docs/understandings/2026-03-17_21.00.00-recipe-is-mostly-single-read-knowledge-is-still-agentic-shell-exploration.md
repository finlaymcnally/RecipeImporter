On the 2026-03-17 Codex benchmark, recipe correction is relatively clean: each shard does one `cat` of the task file and returns JSON. Knowledge extraction is not: workers repeatedly `cat`/`jq`/`sed`/`python` the shard file and subsets of its chunks, so the same payload is replayed through shell output and cached context much like line-role, just less severely.

Read when: When comparing Codex stage costs and deciding which stage to simplify first after line-role; When evaluating whether recipe and knowledge are safe to keep path-based; When someone asks whether all Codex shard stages have the same shell-loop token blow-up.
