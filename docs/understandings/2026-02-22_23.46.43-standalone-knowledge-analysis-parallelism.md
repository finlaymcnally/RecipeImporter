# Discovery

- The long "Analyzing standalone knowledge blocks..." phase came from serial container processing inside `_extract_standalone_tips(...)` in both `cookimport/plugins/epub.py` and `cookimport/plugins/pdf.py`.
- Safe parallel boundary: standalone topic containers are independent units after `chunk_standalone_blocks(...)`, so container analysis can run concurrently.
- Order contract: even with concurrent completion, outputs are sorted back by original container index before merging, so topic/tip ordering remains deterministic.
- Progress contract: this phase now emits `Analyzing standalone knowledge blocks... task X/Y` updates (including `0/Y` start message) so spinner ETA can track the loop.
- Worker tuning: container concurrency is bounded by `C3IMP_STANDALONE_ANALYSIS_WORKERS` (default `4`, minimum `1`).
