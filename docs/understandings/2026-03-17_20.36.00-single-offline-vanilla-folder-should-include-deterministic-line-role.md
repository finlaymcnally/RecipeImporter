The interactive vanilla top-tier profile already enables `line_role_pipeline=deterministic-v1` plus `atomic_block_splitter=atomic-v1`. For single-offline benchmarks, that still needs to land under `.../vanilla/`, because no Codex-backed surface ran. The old `line_role_only` slug was too broad: it treated any non-`off` line-role pipeline as hybrid, which made deterministic baseline runs look non-vanilla even though paired `vanilla` benchmark runs already use deterministic line-role internally.
