# Line-role projection wiring (Milestone 4)

- `generate_pred_run_artifacts` now writes projection artifacts under `prediction-run/line-role-pipeline/`:
  - `line_role_predictions.jsonl`
  - `freeform_span_predictions.jsonl`
  - `stage_block_predictions.json`
  - `extracted_archive.json`
- Canonical benchmark bundle building in `cookimport/cli.py` now prefers those projected stage/archive files when `prefer_line_role_projection=True`; it falls back to legacy stage artifacts if projection files are missing.
- When processed output writing is enabled, projected spans are also applied back into `ConversionResult.recipes` before draft writing, so benchmark predictions and draft fields can come from the same line-role labels.
- Integration coverage now exists in:
  - `tests/labelstudio/test_labelstudio_ingest_parallel.py::test_generate_pred_run_artifacts_line_role_projection_updates_draft_fields`
  - `tests/labelstudio/test_labelstudio_benchmark_helpers.py::test_build_prediction_bundle_prefers_line_role_projection_for_canonical_mode`
