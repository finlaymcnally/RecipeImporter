# Docs task home mapping

Discovery:

- `docs/07-bench/` is the durable home for benchmark orchestration, session-root layout, upload/follow-up artifacts, single-profile dashboard/runtime behavior, and benchmark-facing pass4 scoring seams.
- `docs/10-llm/` is the durable home for Codex execution policy, compact pipeline defaults, pass4 pack/output contracts, prompt-budget artifacts, and cross-surface guardrail/policy decisions.
- `docs/04-parsing/` is the durable home for deterministic canonical line-role rules and parser-side guardrails when the change is mostly about label semantics rather than benchmark/session wiring.

Implication:

- A pass4 change that alters output schema or writer artifacts belongs primarily in `10-llm`, even if benchmark docs also need the scored-surface contract updated.
- A benchmark task that changes where artifacts live or how follow-up/upload/session roots behave belongs primarily in `07-bench`, even if LLM artifacts are involved.
