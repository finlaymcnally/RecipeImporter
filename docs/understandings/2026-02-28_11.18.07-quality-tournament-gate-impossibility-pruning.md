# Quality Tournament Gate-Impossibility Pruning

- The tournament now does discovery for all configured seeds first, dedupes identical selected-target suites, and only executes unique folds.
- After each evaluated fold, each active candidate is checked against an optimistic upper bound using remaining unique folds:
  - max possible completed folds,
  - max possible non-regression and uplift ratios,
  - max possible mean strict/practical/source deltas (future folds assumed perfect `+1.0` deltas).
- If a candidate cannot satisfy one or more gates even under that optimistic bound, it is pruned and excluded from subsequent fold `quality-run` commands.
- Fold outputs include `pruned_candidates_after_fold`; tournament summary/report include `pruned_candidates`, `quality_runs_executed`, and early-stop state.
