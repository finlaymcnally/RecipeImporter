# Quality-Run Subprocess Experiment Fallback

- Root cause: on this host, process-pool probing fails (`PermissionError: [Errno 13]`) because `/dev/shm` is not writable for the runtime user context.
- Existing fallback path kept all-method config workers on thread executors; this preserved correctness but left CPU underutilized from GIL contention when many experiments were active.
- New behavior in `run_quality_suite(...)`:
  - resolves experiment executor mode (`auto|thread|subprocess`) via `COOKIMPORT_QUALITY_EXPERIMENT_EXECUTOR_MODE`,
  - `auto` picks subprocess fanout when process-pool probe is unavailable,
  - writes executor metadata into `experiments_resolved.json` (`experiment_executor_mode`, `experiment_executor_reason`, env key).
- Subprocess fanout runs each experiment in an isolated Python process using an internal worker request/result JSON contract, then merges results back into the normal summary/report artifacts.
