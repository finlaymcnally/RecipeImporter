## Discovery

Interactive single-offline benchmark mode now has explicit paired-run orchestration:
- `vanilla` executes first (`llm_recipe_pipeline=off`),
- `codexfarm` executes second when selected settings enable codex,
- variant failures are handled independently so successful outputs are preserved.

Comparison artifacts (`codex_vs_vanilla_comparison.json/.md`) are written only when both variant runs complete successfully.

## Why this matters

This avoids losing a baseline when codex fails and prevents misleading one-sided comparison files.
