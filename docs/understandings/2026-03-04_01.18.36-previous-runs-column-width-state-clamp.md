# Discovery

- Previous Runs column widths are persisted in browser/program UI-state and reapplied on each render.
- If persisted width values become very large in local state, layout can appear to keep expanding even after content-level overflow fixes.

# Outcome

- Column-width normalization now clamps persisted values to `72..1200px` and applies that clamp consistently for sanitization, drag-resize updates, and payload/preset persistence.
