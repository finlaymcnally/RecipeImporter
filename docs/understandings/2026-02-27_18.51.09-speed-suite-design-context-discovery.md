## Discovery

- Existing `cookimport bench run` is quality-first and currently aggregates stage-block evaluation metrics. It does not provide baseline-vs-candidate speed gating for future optimization work.
- The codebase already has rich timing artifacts for speed analysis:
  - stage run reports (`*.excel_import_report.json`),
  - benchmark eval reports (`eval_report.json`),
  - run manifests (`run_manifest.json`),
  - processing/scheduler telemetry JSONL (`processing_timeseries*.jsonl`, `scheduler_timeseries.jsonl`).
- All-method source matching logic already exists in `cookimport/cli.py:_resolve_all_method_targets(...)` and can map freeform exports to `data/input` by source filename hint.
- Some pulled gold exports have empty `run_manifest.json` source sections, but `exports/freeform_span_labels.jsonl` still includes `source_file`, so robust discovery must use fallback hints, not manifest-only matching.
- Top-level `data/.history/performance_history.csv` can be incomplete for recent benchmark runs; run-local artifacts are the reliable source of truth for deterministic speed comparisons.
