# Trend Tooltip Point Metadata

- Discovery: trend point payloads only carried run-group keys/labels, and tooltip rendering focused on grouped series values. In runs with multiple books per run-group, hover cards looked aggregated instead of dot-specific.
- Fix: each trend point now includes `sourceLabel`, `sourceTitle`, `variant`, and `runTimestamp` in `custom`, and the tooltip renders hovered-point score/book/variant/timestamp first, then optional run-group values.
- Code seam: `cookimport/analytics/dashboard_render.py` (`benchmarkSeriesFromRecords`, trend `tooltip.formatter`).
