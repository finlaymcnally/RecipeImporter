---
summary: "Prompt preview now reuses exact saved CodexFarm recipe/knowledge input payloads when they exist, and only reconstructs locally as fallback."
read_when:
  - "When `cf-debug preview-prompts` seems out of sync with a live CodexFarm run."
  - "When changing prompt preview reconstruction for recipe or knowledge stages."
---

The safest prompt-preview seam is the saved live CodexFarm input payload itself. For processed runs that already contain `raw/llm/<workbook_slug>/recipe_correction/in/*.json` or `raw/llm/<workbook_slug>/knowledge/in/*.json`, `cookimport/llm/prompt_preview.py` now copies and renders those exact payloads instead of rebuilding approximations from drafts. That keeps preview output aligned with the prompt that actually ran, including field order and any new payload fields such as inline tagging guidance.

When those live input files are absent, recipe preview now falls back to the shared `_build_recipe_correction_input(...)` helper from `codex_farm_orchestrator` instead of hand-assembling a separate preview-only recipe payload. Knowledge preview still falls back through `build_knowledge_jobs(...)`, and line-role preview remains prompt reconstruction only.
