# Single-profile benchmark terminal noise sources

- Interactive single-profile matched-book runs use one outer shared dashboard/status surface and suppress nested benchmark summaries via `_benchmark_progress_overrides(...)`.
- `labelstudio_benchmark(...)` still called `_print_codex_decision(...)` before checking `suppress_summary`, so each nested vanilla/codex variant could print its own decision banner directly into the terminal.
- `SubprocessCodexFarmRunner.run_pipeline(...)` logged recoverable `no last agent message` partial-output failures with `logger.warning(...)`, which bypassed the shared progress callback and dumped multiline auth/retry details into the PTY.
- The clean fix is:
  - suppress codex decision banners when benchmark summary suppression is active,
  - when a recoverable codex-farm failure happens during callback-driven runs, emit one short progress message and keep the detailed diagnostics in artifacts/debug logs instead of terminal warnings.
