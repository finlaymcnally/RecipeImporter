# Discovery: all-method bulk matching gap

- `_interactive_all_method_benchmark(...)` currently resolves exactly one `(gold_spans, source_file)` pair through `_resolve_benchmark_gold_and_source(...)`, then runs `_run_all_method_benchmark(...)` once.
- `_infer_source_file_from_freeform_gold(...)` only reads source metadata from run `manifest.json` or the first labeled row in `freeform_span_labels.jsonl`.
- If `freeform_span_labels.jsonl` has no labeled rows, source inference can fail even when `exports/freeform_segment_manifest.jsonl` contains `source_file` metadata.
- This is the key gap for a future "select all golden sets" mode: we need a bulk resolver that discovers all freeform exports, matches each to `data/input` importable files, and explicitly reports unmatched exports.
