# SpeedSuite Codex Farm Confirmation Contract

## Discovery

`bench speed-run` already had `--include-codex-farm`, but no explicit user-confirmation gate. Any caller (including AI automation) could enable Codex Farm permutations directly.

## Contract decision

- Add a CLI confirmation token gate: `--speedsuite-codex-farm-confirmation I_HAVE_EXPLICIT_USER_CONFIRMATION` is required when `--include-codex-farm` is used.
- Add a runner-level guard in `run_speed_suite(...)`: direct callers must pass `codex_farm_confirmed=True` when requesting Codex Farm permutations.

## Why both layers

- CLI guard keeps command-line workflows explicit.
- Runner guard prevents bypass when `run_speed_suite(...)` is invoked programmatically.
