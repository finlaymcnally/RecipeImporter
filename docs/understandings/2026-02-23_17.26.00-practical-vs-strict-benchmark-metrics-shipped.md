# Practical vs Strict benchmark metrics (implemented state)

- Freeform eval now writes both strict localization metrics (`precision`, `recall`, `f1`) and practical content-overlap metrics (`practical_*`, supported practical variants).
- Eval reports now include span width stats plus `granularity_mismatch` detection for coarse prediction ranges (high practical / near-zero strict).
- Bench aggregate + iteration packet now carry both tracks and avoid ranking strict-only boundary misses as catastrophic when practical overlap is already satisfied.
- Benchmark CSV/dashboard now persist and render both `Practical F1` and `Strict F1`, with mismatch tagging in Recent Benchmarks.
- Interactive benchmark combo-only drift was removed; interactive benchmark now stays offline (`single_offline` + `all_method`).
