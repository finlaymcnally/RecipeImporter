# Knowledge Prompt Target Must Stay Soft

Current knowledge bundling should treat `knowledge_prompt_target_count` as a planning hint only.

The bug in the March 18 builder was that `_bundle_prepared_chunks(...)` raised `effective_max_bundle_chars` to `total_chars + 1` and `effective_max_gap_blocks` to a huge value whenever `target_bundle_count` was set. That meant the prompt target could silently disable the normal shard safety limits.

The durable rule is: prompt target may influence `max_chunks_per_bundle`, but the hard char cap and local-gap cap still apply. If the book needs more shards to stay within those limits, the bundle count must exceed the prompt target.
