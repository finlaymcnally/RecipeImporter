# Oracle file attachment contract

- For this repo and Oracle CLI `0.8.6`, passing `--file <upload_bundle_v1 dir>` did not attach any files, even though the path existed.

- Globs against the bundle directory also failed in the same environment:
  - `--file "upload_bundle_v1/**"`
  - `--file "upload_bundle_v1/*"`

- The working contract was explicit file attachment. Oracle dry-run accepted:
  - `--file <bundle>/upload_bundle_overview.md`
  - `--file <bundle>/upload_bundle_index.json`
  - `--file <bundle>/upload_bundle_payload.jsonl`

- One more local constraint matters for dry-run: Oracle inline dry-run rejected the real sample `upload_bundle_payload.jsonl` because it was 22.2 MB and Oracle's inline file limit is 1 MB.

- Practical implication: benchmark-to-Oracle upload should still resolve a bundle directory as the user-facing unit, but the actual Oracle command should pass the three bundle files individually. For oversized real bundles, `--mode dry-run` needs a repo-local preview fallback instead of a literal Oracle dry-run subprocess.
