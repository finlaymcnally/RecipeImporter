# Discovery

`cookimport bench quality-run` supports `--include-deterministic-sweeps` (default off). When enabled, it applies the same all-method deterministic sweep expansion used by interactive all-method runs to each quality experiment.

To identify which sweep is helpful:

1. Run `quality-run` with `--include-deterministic-sweeps`.
2. Run `quality-leaderboard` for the experiment (`--experiment-id baseline` if using only baseline experiment).
3. Inspect `leaderboard.json` row `dimensions.deterministic_sweep` (or `base` when absent) alongside `mean_practical_f1`, `mean_strict_f1`, and duration.

For cleaner attribution of single knob changes, use schema-v2 `levers` experiments (baseline + one lever per experiment) and keep `--no-include-deterministic-sweeps`.
