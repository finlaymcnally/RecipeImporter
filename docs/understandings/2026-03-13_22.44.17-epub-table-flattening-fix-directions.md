summary: "Best fix direction for missed EPUB knowledge tables is to preserve row/cell structure at extraction time; current Food Lab-style failures happen because rows are joined into plain text before `tables.py` sees them."
read_when:
  - "When planning fixes for missed EPUB table extraction"
  - "When deciding whether to patch the detector or the EPUB extractors first"
---

- Current EPUB row builders in `cookimport/parsing/epub_extractors.py` and `cookimport/plugins/epub.py` join table cell text with plain spaces, which erases reliable column separators before downstream detection.
- `cookimport/parsing/tables.py` currently expects row-like text cues (pipe, tab, repeated spaces) or consistent row runs; once a whole table is collapsed into one long line, the detector has very little to work with.
- Best-first fix:
  - preserve structured cell data or a deterministic delimiter in the extracted block,
  - then teach `tables.py` to trust that metadata for EPUB table rows instead of reparsing ambiguous free text.
- Fallback-only fix:
  - add special salvage heuristics for flattened conversion/reference sections, but treat that as lower-confidence and lower-priority than upstream preservation.
