## Discovery

`_run_with_progress_status(...)` is the shared spinner/status surface for Label Studio import, benchmark prediction/eval wrappers, bench run, and bench sweep.

`stage(...)` does not use `_run_with_progress_status`; it owns a separate Rich progress/live dashboard path with worker queue updates and direct job-loop rendering.

Practical implication:
- Extending telemetry to "all processing" requires both:
  - shared spinner instrumentation (`_run_with_progress_status`) and
  - stage-local instrumentation in the stage job loop.
