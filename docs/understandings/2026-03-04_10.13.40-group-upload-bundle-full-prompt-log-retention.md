# Group Upload Bundle Full Prompt Log Retention

In `high_level_only` mode, artifact selection is driven by `GROUP_UPLOAD_BUNDLE_RUN_PRIORITY_FILES`, which only listed `run_manifest.json`, `eval_report.json`, and `need_to_know_summary.json`.

Because of that filter, per-run full prompt logs were available for diagnostics during generation but not retained as payload artifacts.

Safe fix path:
- keep the same high-level selection flow
- resolve each run's full prompt log via `_resolve_full_prompt_log_path(...)`
- include it only if the file is inside the source root
- keep it marked as heavy/deprioritized in navigation
