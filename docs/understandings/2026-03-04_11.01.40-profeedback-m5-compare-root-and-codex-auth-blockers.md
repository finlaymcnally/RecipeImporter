# Profeedback Milestone 5 run blockers (compare roots + codex auth)

- `cookimport labelstudio-benchmark compare` does not accept per-run `eval_report.json` outputs from `labelstudio-benchmark run`.
- Compare resolver requires an all-method root containing `all_method_benchmark_multi_source_report.json` (or nested `all-method-benchmark/` with that file).
- In this environment, full `codex-farm` pass1 calls failed auth (`403 Forbidden` websocket) unless `--codex-farm-failure-mode fallback` was used.
- Milestone 5 ablation packet can still be produced with fallback-mode full-stack rows, but those rows are not true codex execution and should be rerun after codex auth is restored.
