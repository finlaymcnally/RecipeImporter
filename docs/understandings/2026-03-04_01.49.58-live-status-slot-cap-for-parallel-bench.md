# Discovery

Rich live status is scoped to a console instance, so parallel benchmark threads can collide when they all try to open live displays on the shared CLI console.

# Resolution

- Added live-status slot gating in `_run_with_progress_status`.
- Added per-run override plumbing (`_benchmark_progress_overrides(..., live_status_slots=...)`).
- Interactive single-profile parallel runs now request `2` live slots.
- When no slot is available, the run uses plain status updates instead of raising a live-display error.
