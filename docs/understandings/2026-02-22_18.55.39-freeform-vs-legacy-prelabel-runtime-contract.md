# Freeform vs legacy prelabel runtime contract

- Interactive style choice maps to `prelabel_granularity` (`actual freeform` -> `span`, `legacy, block based` -> `block`).
- `span` mode parses quote-anchored selections (plus optional absolute `start/end`), resolves quotes against exact block substrings, and drops ambiguous/unresolvable selections unless `occurrence` disambiguates repeats.
- `block` mode maps `{block_index, label}` directly to full-block spans using `segment_start`/`segment_end`; prompt requests one label per block, but runtime does not strictly enforce full coverage.
- Both modes recompute `value.text` from `segment_text[start:end]`, enforce allowed-label normalization, dedupe identical `(label,start,end)` results, and treat item-level parse failures as drops.
- Task-level failure happens only when no valid spans remain (or provider call errors), with strict-vs-partial continuation controlled by `--prelabel-allow-partial`.
