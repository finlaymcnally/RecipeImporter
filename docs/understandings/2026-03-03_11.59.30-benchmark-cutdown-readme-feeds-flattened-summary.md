# benchmark_cutdown readme to flattened summary path

`scripts/benchmark_cutdown_for_external_ai.py` writes root `README.md` in `_write_readme(...)`.
`_flatten_output(...)` then copies root metadata files and `_write_root_summary_markdown(...)` embeds README content into `benchmark_summary.md`.
So project/benchmark interpretation text should be added once in root README; flattened output picks it up automatically without duplicating logic.
