# Codex prelabel thinking effort injection

- Codex thinking effort is a config override (`-c model_reasoning_effort=...`), so prelabel wiring should inject that override into the resolved Codex command instead of inventing a separate provider option.
- Injection must be command-aware and non-destructive: resolve command first, then add effort only when the command does not already set `model_reasoning_effort`.
- Interactive prelabel prompt order should stay: style -> account display -> model -> thinking effort.
- Prelabel run reports should include resolved `codex_reasoning_effort` next to command/model/account for run auditability.
