# Multi-Book Run-Level Upload Bundle Discovery

The run-level group upload bundle path is `build_upload_bundle_for_existing_output(..., high_level_only=True)` from `cookimport/cli.py` when `single-profile` has more than one target book.

Key behavior gap found: discovered run records used `run_dir.name` as `output_subdir` and also keyed runtime/diagnostic lookups by `run_id` only. In multi-book runs, repeated variant IDs (`vanilla`, `codexfarm`) can collide across books, which can lower critical row-locator coverage and cross-wire runtime lookups.

Implementation direction used:
- normalize derived run rows to source-root-relative `output_subdir`
- prefer output-subdir run-dir lookup for diagnostics/locators
- keep runtime/low-confidence collectors source-aware and able to iterate all discovered run dirs
- only surface new scorecard/ablation/outside-span/chapter-page/runtime/top-regression-trace views for true multi-book high-level bundles
