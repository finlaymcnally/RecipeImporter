# Spinner stale-after-candidate wiring note

- The `Label Studio import (...)` spinner only renders what `progress_callback` emits; if the importer stops emitting messages, the spinner appears frozen on the last message.
- EPUB/PDF converters previously ended known-size loops at `Extracting candidate N/N...` and then did standalone-tip/topic/finalization work without new callback text.
- The safest fix is dual-path:
  - emit new phase callbacks inside importer runtime loops where the extra work happens,
  - keep CLI wrappers generic and add elapsed-time suffixes for unchanged messages so users can still see liveness.
