# Discovery

`evaluate_canonical_text` reused `load_stage_block_labels(...)`, and that loader remaps `HOWTO_SECTION` to ingredient/instruction using neighbor context by default.  
That remap is desirable for stage-block/freeform comparability, but it suppressed explicit HOWTO accounting in canonical-text metrics.

# Current contract

- `load_stage_block_labels(..., resolve_howto_sections=True)` stays the default for stage/freeform behavior.
- Canonical-text evaluation now calls `load_stage_block_labels(..., resolve_howto_sections=False)` and keeps explicit HOWTO labels through metrics.
- Canonical helper label builders in `cookimport/bench/eval_canonical_text.py` no longer remap HOWTO labels before `compute_block_metrics`.

# Validation evidence

- Regression test: `tests/bench/test_eval_stage_blocks.py::test_evaluate_canonical_text_includes_howto_section_totals`.
- The test verifies `per_label.HOWTO_SECTION.gold_total == 1`, `pred_total == 1`, `tp == 1`, and confusion contains `HOWTO_SECTION -> HOWTO_SECTION`.
