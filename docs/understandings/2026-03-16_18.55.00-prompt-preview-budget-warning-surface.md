summary: "Prompt preview now emits a heuristic budget summary and blunt danger warnings based on rendered prompt volume and call fan-out."
read_when:
  - "When deciding whether a prompt preview should block or warn before an expensive CodexFarm run."
  - "When changing prompt-preview budget estimation or warning thresholds."
---

`cf-debug preview-prompts` now writes `prompt_preview_budget_summary.json` and `.md` beside the preview manifest. The summary estimates prompt spend from rendered prompt text length because preview runs do not have live tokenizer telemetry, and it warns using both the token heuristic and raw prompt volume signals (call count plus rendered prompt chars).

The warning surface is intentionally blunt. If previewed work reaches roughly `100+` recipe-correction calls, `100+` total calls with large prompt volume, or an extreme rendered-char budget, the summary emits warnings such as `EXTREME prompt budget ... Treat this as a multi-million-token danger zone.` The CLI prints those warnings to stderr while still keeping the manifest path alone on stdout for scripting.
