The live stage runtime is already mostly on the new architecture. The remaining runtime debt is concentrated in `cookimport/parsing/label_source_of_truth.py`, `cookimport/staging/import_session.py`, `cookimport/config/run_settings.py`, `cookimport/cli.py`, and `cookimport/labelstudio/ingest.py`, where current behavior is still wrapped in bridge names or hidden defaults called `legacy`.

Tooling debt is broader but mechanical. `cookimport/llm/prompt_artifacts.py` still serializes prompt stages through `task1/task4/task5` compatibility metadata, benchmark evaluators still emit old mismatch alias files, and analytics collectors still synthesize current metrics from deprecated field names or deprecated CSV locations.

The safest execution order is: remove runtime bridge naming first, then collapse hidden defaults/scheduler knobs, then delete prompt/benchmark aliases, then cut analytics/history fallbacks and update docs/tests at the end.
