## Discovery

Run: `data/golden/benchmark-vs-golden/2026-02-27_17.54.41`

- Host CPU capacity: `16` logical CPUs (`8` cores, SMT2).
- Late tail state showed only one active source (`thefoodlabcutdown`) with `7` remaining configs (`9-15/15`) in `evaluate`.
- Live canonical cache locks for this source were `5` unique keys, all owned by live worker PIDs.

## Practical implication

Late in all-method runs, total CPU usage is bounded by:

- remaining active configs for the last source, and
- number of unique canonical alignment keys currently computing (configs sharing a key wait for cache reuse).

So 50-60% CPU in this phase can be normal even when no deadlock exists; saturation cannot exceed available independent eval work units.
