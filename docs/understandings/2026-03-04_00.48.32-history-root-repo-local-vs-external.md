# Discovery

- `data/.history` was unreachable for git tracking because top-level `data/` is gitignored.
- History-path consumers rely heavily on `history_root_for_output(...)`, so changing that helper gives a clean central migration point.

# Resolution

- Canonical history root for repo-local outputs moved to `<repo>/.history`.
- External output roots still resolve history to `<output_root parent>/.history`.
- Dashboard collector and run-settings stores keep read fallbacks for prior history locations to reduce migration breakage.
