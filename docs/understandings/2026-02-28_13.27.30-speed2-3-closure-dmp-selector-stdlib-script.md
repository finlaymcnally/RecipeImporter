# Speed2-3 Closure: DMP Selector Lock + Stdlib Benchmark Script Fix

- Current runtime contract is dmp-only for `COOKIMPORT_BENCHMARK_SEQUENCE_MATCHER`; non-`dmp` values are intentionally rejected in `cookimport/bench/sequence_matcher_select.py`.
- Discovery during plan closure: `scripts/bench_sequence_matcher_impl.py` still tried to run `stdlib` mode through the runtime selector, causing a crash (`ValueError` on selector mode).
- Fix: benchmark `stdlib` directly with `difflib.SequenceMatcher` in the script, and use selector resolution only for non-stdlib modes (currently `dmp`).
- Validation: new regression test `test_sequence_matcher_benchmark_script_supports_stdlib_and_dmp_modes` plus SpeedSuite `speed-discover`/`speed-run`/`speed-compare` run ending with PASS.
