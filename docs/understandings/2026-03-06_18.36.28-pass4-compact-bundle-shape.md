# Pass4 compact bundle shape

The pass4 prompt wrapper was never the main size problem. The large token cost came from the inline JSON bundle written by `cookimport/llm/codex_farm_knowledge_jobs.py`.

The compact pass4 format now reduces prompt input size by:

- keeping chunk blocks to `block_index`, `text`, optional `heading_level`, and a trimmed `table_hint`
- dropping `block_id`, `page`, `spine_index`, and `features_subset` from compact chunk blocks
- shrinking context blocks to only `block_index`, `text`, and optional `heading_level`
- removing full-book `guardrails.recipe_spans` from the compact bundle
- replacing that guardrail with `guardrails.context_recipe_block_indices`, which marks only the nearby context blocks that belong to recipe spans
- omitting empty/default-valued fields when writing compact JSON bundles

Compatibility boundary:

- The shipped pack now uses `recipe.knowledge.compact.v1`.
- The runtime still supports explicit non-compact/custom pass4 pipeline ids by writing the legacy pass4 job shape when the selected pass4 pipeline id is not `recipe.knowledge.compact.v1`.
