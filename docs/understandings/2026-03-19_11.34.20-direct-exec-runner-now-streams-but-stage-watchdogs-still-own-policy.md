
# Direct Exec Runner Streams And Strict Stages Own Policy

- `cookimport/llm/codex_exec_runner.py` now has the generic capability the repo was missing earlier in the day:
  - streamed JSON event observation while the subprocess is still running
  - callback-driven early termination from those live snapshots
  - shard-attempt `workspace_manifest` payloads that describe the sterile cwd and mirrored inputs
- Knowledge and canonical line-role now actually consume that seam in their stage orchestrators:
  - deterministic preflight rejection for malformed strict-JSON shard payloads
  - live watchdog termination for forbidden tool use, repeated reasoning-without-output patterns, and cohort-relative runtime outliers
  - one fresh retry packet for retryable watchdog kills, using repo-prepared context instead of mid-run coaching
  - shard-local `live_status.json` plus final `status.json` fields that surface `state`, `reason_code`, `reason_detail`, and `retryable`
- Recipe correction now uses the same repo-owned one-shot repair pattern as knowledge and line-role, and it also owns a narrower strict-JSON policy layer:
  - deterministic preflight rejection for malformed recipe shard payloads
  - live watchdog termination for forbidden tool use or repeated reasoning-without-output
  - shard-local `live_status.json` plus final `status.json` / `repair_status.json` fields that surface `state`, `reason_code`, `reason_detail`, and `retryable`
  - no cohort-relative outlier kill or watchdog retry packet yet; recipe still stays narrower than knowledge/line-role there by design
- All three direct stages now write shard-local workspace manifest files into the normal runtime artifact tree.
- The remaining hardening work is not “push more policy into the runner.” The remaining work is stage-owned follow-through:
  - final wording polish in higher-level summaries if new retry/recovery reason codes accumulate
  - any later decision to give recipe correction the same cohort-outlier / watchdog-retry policy rather than keeping recipe supervision narrower
- Practical boundary: keep subprocess transport, live snapshot plumbing, and workspace provenance in `codex_exec_runner.py`; keep kill/retry/repair policy in `codex_farm_knowledge_orchestrator.py`, `canonical_line_roles.py`, and `codex_farm_orchestrator.py`.
