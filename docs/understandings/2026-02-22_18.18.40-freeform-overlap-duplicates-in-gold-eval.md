# Freeform overlap duplicates in gold eval

What happens:
- Freeform tasks intentionally repeat boundary blocks across segments (`segment_overlap`) in `cookimport/labelstudio/freeform_tasks.py`.
- Freeform export writes one span row per segment annotation, and `span_id` includes `segment_id` context in the hash input (`cookimport/labelstudio/export.py`), so identical block-range labels from adjacent segments become distinct rows.
- Freeform evaluation now dedupes gold rows by `(source_hash, source_file, start_block_index, end_block_index)` before scoring.

Metric implication:
- Overlap duplicates no longer double-count in strict/app/classification gold denominators.
- If duplicated overlap rows disagree on label, evaluator resolves conflicts by majority vote.
- Exact label ties for the same dedupe key are dropped from scored gold to avoid arbitrary label picks; they are reported in `gold_dedupe.conflicts`.

Quick detection:
- Group `freeform_span_labels.jsonl` by `(source_hash, source_file, label, min(touched_block_indices), max(touched_block_indices))`.
- If duplicate keys map to different `segment_id` values, they are overlap-induced duplicates.

Current report hooks:
- `eval_report.json` now includes `gold_dedupe` with input/deduped totals and conflict breakdown.
- `eval_report.md` includes a short "Gold dedupe" summary section.
