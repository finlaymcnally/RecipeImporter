# Benchmark GC + CSV-first retention

Key implementation discovery: benchmark CSV already carried strict/macro/boundary/runtime fields, and the missing durable dashboard field was per-label breakdown.

Current contract:
- `append_benchmark_csv` now persists `per_label_json` (compact per-label list) for new benchmark rows.
- `collect_dashboard_data` is benchmark CSV-first; recursive benchmark eval scan is opt-in via `--scan-benchmark-reports` (with fallback scan only when no benchmark CSV rows exist).
- `bench gc` hydrates missing benchmark CSV fields (`per_label_json`, strict/macro, boundary) from `run_dir/eval_report.json` before pruning run roots, then deletes selected run directories by policy.
- On apply writes that modify history CSV, `bench gc` creates `performance_history.<YYYY-MM-DD_HH.MM.SS>.gc.bak.csv` first.
