# Discovery

`choose_run_settings(...)` exposed a separate two-item profile picker (`CodexFarm automatic top-tier` / `Vanilla automatic top-tier`) before runs, even though downstream single-profile benchmark logic already interprets codex-on as a two-variant plan (vanilla baseline first, then codexfarm).

# Resolution

Prompt intent is now `Use Codex Farm recipe pipeline for this run?` and maps to the same two deterministic profile families internally. `Yes` resolves codex top-tier (winner settings when available, then harmonized codex knobs). `No` resolves vanilla deterministic top-tier. `COOKIMPORT_TOP_TIER_PROFILE` still provides a forced override.
