# Discovery

After fixing `legacy -> beautifulsoup` parity, `epub_extractor=auto` still flowed through the generic unsupported-value warning in interactive config loading.

`RunSettings.from_dict(...)` already handles this value as a specific migration/removal case (`auto -> unstructured`) with targeted warning text.

# Resolution

Interactive `cookimport.json` loading now has explicit branches for both historical values:
- `legacy` => migrate to `beautifulsoup` (migration warning)
- `auto` => force `unstructured` (auto mode removed warning)

This keeps migration semantics and operator-facing warnings aligned across entry paths.
