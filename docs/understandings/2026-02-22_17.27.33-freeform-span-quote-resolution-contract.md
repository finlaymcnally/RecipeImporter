# Freeform span-mode quote resolution contract

Span mode (`prelabel_granularity=span`) does not trust model-provided absolute offsets as the primary path.

Primary contract:
- Model outputs quote-anchored selections: `{"block_index", "label", "quote", "occurrence"?}`.
- Runtime resolves quotes inside the exact block substring derived from `segment_text[segment_start:segment_end]`.
- Resolved start/end are converted back to segment-global offsets.

Important guardrails:
- Repeated quote text in the same block requires `occurrence` (1-based).
- If quote resolution is ambiguous without `occurrence`, that selection is dropped.
- `value.text` is always computed from `segment_text[start:end]` (never copied from model output).
- Optional absolute `start/end` items are accepted only with strict bounds validation.

Operational effect:
- Span mode can produce multiple partial highlights inside one extracted block.
- Legacy mode remains available as `prelabel_granularity=block` (`legacy, block based` in interactive CLI).
