# MarkItDown Extractor + Split Planning Notes

- `epub_extractor` remains the single run-setting knob, but `markitdown` is whole-book conversion (`EPUB -> markdown`) and cannot run with spine-range split jobs.
- To keep stage and benchmark parity, split-planning must be updated in both places together:
  - `cookimport/cli.py:_plan_jobs(...)`
  - `cookimport/labelstudio/ingest.py:_plan_parallel_convert_jobs(...)`
- Keep run metadata honest when split capability changes:
  - `cookimport/config/run_settings.py:compute_effective_workers(...)` must account for extractor value.
- `labelstudio_benchmark(...)` now forwards the selected extractor into prediction generation explicitly (not only through env), so run manifests/history rows capture the exact extractor used.
