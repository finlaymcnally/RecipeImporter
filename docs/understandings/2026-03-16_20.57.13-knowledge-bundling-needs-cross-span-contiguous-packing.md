# Knowledge Bundling Needs Cross-Span Contiguous Packing

The first bundled knowledge implementation only grouped chunks inside each individual Stage 7 seed non-recipe span. That was too conservative.

Evidence:

- On `data/golden/benchmark-vs-golden/2026-03-16_20.23.25/.../line_role_only`, the first bundled preview still wrote `313` knowledge prompts for `320` surviving chunks.
- Inspecting the generated knowledge inputs showed only `7` multi-chunk prompts; `306` prompts still carried exactly one chunk.

The reason is that Stage 7 seed spans are already fragmented. Many neighboring outside-recipe regions that should be reviewed together land in separate seed spans, so “bundle within a single seed span” barely changes fan-out.

The better seam is:

- keep chunk creation deterministic
- skip obvious `noise`
- then bundle surviving chunks globally in block order
- but only while the chunks stay contiguous, local, and not table-heavy

After switching the bundle planner to pack contiguous chunks across neighboring seed spans, the same preview dropped from `324` knowledge prompts to `91`, with `62` multi-chunk prompts. That is the durable contract to keep.
