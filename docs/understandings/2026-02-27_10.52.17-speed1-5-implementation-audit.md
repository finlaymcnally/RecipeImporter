## Detailed deficiencies

### D-01: Prepared archive abstraction not implemented (Milestone 4)

- Plan expectation:
  - explicit dataclass + constructor: `PreparedExtractedArchive` and `prepare_extracted_archive(...)` (`speed1-5.md` lines 283-289, 474-483).
- Current implementation:
  - archive generation still uses `build_extracted_archive(...) -> list[ArchiveBlock]` in `cookimport/labelstudio/archive.py:93`.
  - ingest builds that list once and passes raw list around: `cookimport/labelstudio/ingest.py:1625`.
  - repository-wide symbol search finds no implementation of planned interface.
- Impact:
  - milestone intent (explicit prepared object contract for reuse/serialization) is not satisfied.
  - follow-on work has weaker type/contract boundaries than originally planned.
- Closure criteria:
  - add `PreparedExtractedArchive` and `prepare_extracted_archive(...)`.
  - move archive/task serialization to thin helpers that accept the prepared object.
  - add guardrail test for parsed archive equality on fixture input.

### D-02: Split-merge `outputStats` likely undercounts moved raw artifacts (Milestone 2 risk)

- Plan expectation:
  - "same `outputStats` values ... without directory walk" and split-merge should use same mechanism (`speed1-5.md` lines 162-164, 187, 207).
- Current implementation:
  - split merge writes report before raw-artifact merge:
    - report write at `cookimport/cli.py:8150`
    - raw merge at `cookimport/cli.py:8153`
  - raw merge physically moves files in `_merge_raw_artifacts(...)` (`cookimport/cli.py:7588-7608`) after stats were already captured.
- Impact:
  - `outputStats` in split-merge reports can omit files moved during post-report raw merge.
  - this weakens the "same values as prior full scan" claim in merged job paths.
- Closure criteria:
  - either merge raw artifacts before report emission, or record moved files into `OutputStats` during merge.
  - add a split-merge test asserting `outputStats` includes raw moved artifacts.

### D-03: Missing explicit output-stats parity test from Milestone 2

- Plan expectation:
  - test compares `report.outputStats` against a fresh directory walk (`speed1-5.md` lines 197-204).
- Current implementation:
  - repository test search shows no `tests/staging` or CLI test asserting `outputStats` parity with a directory scan.
  - only `tests/analytics/test_stats_dashboard.py` references `outputStats`, and that is analytics ingestion, not staging parity verification.
- Impact:
  - regression risk: missed output accounting can ship unnoticed.
- Closure criteria:
  - add targeted staging/split test that computes ground-truth scan and asserts equality with report payload.

### D-04: `bench run` does not expose direct write-toggle CLI flags (partial interface gap)

- Plan expectation:
  - add write-toggle knob in both benchmark and bench run entrypoints (`speed1-5.md` lines 243-244, 329-330).
- Current implementation:
  - `labelstudio-benchmark` has both flags:
    - `--write-markdown/--no-write-markdown` at `cookimport/cli.py:10262`
    - `--write-labelstudio-tasks/--no-write-labelstudio-tasks` at `cookimport/cli.py:10269`
  - `bench run` command signature has only `suite/out_dir/baseline/config` (`cookimport/cli.py:11395-11411`).
  - workaround exists: bench config forwards keys to pred-run builder (`cookimport/bench/pred_run.py:42-43`).
- Impact:
  - functionality is available, but not through the direct CLI shape implied by the plan text.
  - usability/documentation mismatch for users expecting first-class bench flags.
- Closure criteria:
  - either add direct `bench run` flags, or formally revise the plan to state "config-only for bench run" and mark as intentional.

### D-06: No strong no-drift proof artifact for write-toggle modes
