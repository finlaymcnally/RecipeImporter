Once inline neighbor duplication is gone, line-role row metadata is only a small slice of the remaining prompt size. On the 2026-03-17 `saltfatacidheatcutdown` line-role payload, the old compact JSON row shape was about `433,774` chars, while a smaller pipe-delimited row shape without per-row escalation reasons was about `409,483` chars, a reduction of about `24,291` chars (`~5.6%`).

That means the safe trim is:

- keep one row per target line
- keep only `atomic_index`, deterministic `label_code`, and `current_line`
- drop per-row escalation reasons from the Codex prompt path
- use a cheaper row syntax than JSON arrays

Trying to move labels and reasons into separate sparse side sections looked worse on this book because the extra section overhead ate the savings. After the neighbor fix, the dominant cost is still the line text itself, not the row metadata wrapper.
