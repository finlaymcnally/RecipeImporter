# Discovery

Recipe codex-farm enablement entered through three user-facing paths (`stage`, `labelstudio-import`, `labelstudio-benchmark`) plus persisted/global run settings.

A single docs-only warning was not enough for automation safety because older saved settings or future CLI invocations could still request `codex-farm-3pass-v1`.

# Contract

- `llm_recipe_pipeline` is policy-locked to `off` in CLI/pred-run normalizers.
- Non-`off` values fail fast with an explicit policy message.
- Run-settings migration (`RunSettings.from_dict`) coerces legacy non-`off` values back to `off` with a warning.
- Run-settings UI exposes only `off` for `llm_recipe_pipeline` to reduce accidental toggles.
