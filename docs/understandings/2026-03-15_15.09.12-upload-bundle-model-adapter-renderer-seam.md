---

The least-risk seam for `upload_bundle_v1` was to keep `scripts/benchmark_cutdown_for_external_ai.py` as orchestration while moving two brittle responsibilities behind stable module boundaries:

- Existing-root discovery + fallback derivation now lives behind `build_upload_bundle_source_model_from_existing_root(...)`, which returns one `UploadBundleSourceModel` instead of an ad-hoc context dictionary.
- Stage-separated comparison rendering now lives behind `build_stage_separated_comparison(...)`, so fixed `pass2/pass3` compatibility fields remain output-compatible while the renderer can accept alternate topology labels.

This split preserves the three-file bundle contract and existing wrapper entrypoint but makes future staging/topology changes primarily an adapter problem rather than a renderer rewrite.
