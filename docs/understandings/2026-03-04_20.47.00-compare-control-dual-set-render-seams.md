## Discovery

- Compare/control was single-state + single-host (`compareControlState`, one panel/result pair, one chart host), so dual-set support needed explicit secondary state and host wiring rather than CSS-only cloning.
- Decoupling contract had to be preserved: compare/control uses its own benchmark scope and local group subsets; it must not write Previous Runs table filters.
- The safest compatibility path was additive UI-state fields inside existing `previous_runs.compare_control` payload, leaving Set 1 legacy keys unchanged.

## Implemented seam decisions

- Added dual-set state as additive fields: `second_set_enabled`, `second_set`, `chart_layout`, `combined_axis_mode`.
- Kept existing Set 1 IDs/state logic as the compatibility anchor; Set 2 uses suffixed IDs.
- Introduced per-set panel rendering from shared analysis helpers (no duplicated compare/control math).
- Added combined-chart builder with explicit incompatibility fallback (mixed categorical/numeric charts).
