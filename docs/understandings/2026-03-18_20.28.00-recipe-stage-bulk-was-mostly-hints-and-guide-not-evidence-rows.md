## Recipe-stage bulk was mostly hints and guide, not evidence rows

- The recipe prompt budget problem was not `ev` evidence rows.
- The main avoidable bulk was:
  - the large per-recipe deterministic hint object `h`
  - the repeated tagging guide `tg`
  - shard-level fields the model did not use (`wb`, `sh`, `an`)
  - pretty-printed inline JSON

What changed:

- recipe shard top level is now `v`, `sid`, `ids`, `r`, `tg`
- each recipe input keeps `rid`, `txt`, `ev`
- `h` is now a compact hint with only the useful correction/tagging fields:
  - `n` title hint
  - `i` ingredient hints
  - `s` step hints
  - optional `d`, `y`, `g`
- the tagging guide is now compact too:
  - `v` guide version
  - `r` short rule list
  - `c[*].k` category key
  - `c[*].x` example labels
- recipe inline JSON is now minified for the billed prompt body
