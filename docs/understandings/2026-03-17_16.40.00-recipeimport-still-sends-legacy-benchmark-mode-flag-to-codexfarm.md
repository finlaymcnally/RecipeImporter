# RecipeImport Benchmark Mode Must Use CodexFarm's Namespaced Flag

- RecipeImport's `codex_farm_recipe_mode` remains a local semantic setting with values `extract` and `benchmark`, but the subprocess transport must only translate `benchmark` into CodexFarm's current namespaced flag.
- The live local `codex-farm` editable install at `/home/mcnal/projects/shared/CodexFarm` exposes `--recipeimport-benchmark-mode` and `--recipeimport-benchmark-debug` on `process`; it does not expose the retired `--benchmark-mode` flag.
- Evidence:
  - `cookimport/llm/codex_farm_runner.py` now maps `benchmark` to `process ... --recipeimport-benchmark-mode line_label_v1`.
  - `cookimport/llm/codex_farm_runner.py` sends no benchmark flag at all for `extract`.
  - `codex-farm process --help` shows `--recipeimport-benchmark-mode`.
  - CodexFarm docs under `shared/CodexFarm/docs/08-external-program-reference/benchmark-runtime-contracts.md` document only the new flag names.
- Practical consequence: if RecipeImport sends the retired flag, interactive benchmark CodexFarm runs can fail even when CodexFarm itself is up to date.
