# Discovery: disable fast canonical alignment for scoring safety

Observed behavior:
- Fast bounded alignment can mis-map enough blocks on real source material to trigger large metric drift risk.
- Auto fallback preserved correctness but still introduced complexity and confusion about whether fast path was active.

Durable rule:
- Canonical scoring should enforce legacy global `SequenceMatcher` alignment.
- `COOKIMPORT_CANONICAL_ALIGNMENT_STRATEGY=auto|fast` should be treated as deprecated aliases and forced to legacy.
- Eval reports must explicitly carry deprecation telemetry so reviewers can confirm fast path stayed disabled.
