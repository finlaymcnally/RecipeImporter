# Prompt Preview Budget Is Now Real Work, Not Wrapper Waste

On `data/golden/benchmark-vs-golden/2026-03-16_20.23.25/.../line_role_only/prompt_preview_2026-03-16_21.14.22`, the current preview totals are:

- `92` calls
- `~578,563` estimated total tokens
- `41` knowledge prompts, `15` line-role prompts, `36` recipe prompts

What changed:

- line-role transport overhead is now tiny (`4,440` chars total), so the old double-wrapping bug is no longer the main problem
- knowledge fan-out is down from `324` prompts to `41`, so bundling/pruning already removed the biggest obvious waste there

What remains:

- line-role is still the single largest stage by estimated tokens (`~290,891`), but that is now mostly genuine task text for batched correction windows rather than wrapper duplication
- knowledge is the biggest count multiplier (`41` prompts, `~183,168` estimated tokens), so stricter deterministic suppression or larger safe bundles are the main lever for total-call reduction
- recipe correction repeats a large fixed payload on every call: the rendered prompt has a flat `2,648` chars of non-payload overhead, and the serialized `tagging_guide` alone is about `2,414` chars

Practical takeaway:

- do not chase line-role transport again; that seam is basically fixed
- if the goal is fewer prompts, attack knowledge admission/bundling
- if the goal is fewer tokens per recipe call, shrink or remove the inline recipe tagging guide
