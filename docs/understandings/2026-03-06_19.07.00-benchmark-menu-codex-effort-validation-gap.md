---
summary: "Interactive benchmark run-settings prompt does not filter Codex Farm reasoning efforts by selected model support."
read_when:
  - When changing interactive benchmark or top-tier run-settings prompts
  - When debugging Codex Farm model/effort mismatches from C3imp or cookimport interactive flows
---

- Interactive benchmark runs go through `cookimport/cli.py` -> `choose_run_settings(...)` -> `cookimport/cli_ui/run_settings_flow.py::_choose_codex_ai_settings(...)`.
- That helper builds the `Codex Farm reasoning effort override` menu from a fixed list: `none`, `minimal`, `low`, `medium`, `high`, `xhigh`.
- It does call `list_codex_farm_models(...)` for model discovery, but it ignores each model row's `supported_reasoning_efforts`.
- Result: the menu can offer an invalid model/effort pair such as `gpt-5.1-codex-mini` + `low`, even though `codex-farm models list --json` reports only `medium` and `high` for that model.
- Downstream, recipeimport passes the chosen override through unchanged to codex-farm, and codex-farm passes it through unchanged to `codex exec` as `model_reasoning_effort="<selected>"`; there is no compatibility check in recipeimport or codex-farm for model-specific support.
- Contrast: the Label Studio AI prelabel interactive flow in `cookimport/cli.py` already filters effort choices using discovered `supported_reasoning_efforts`, so this validation logic exists elsewhere in the repo.
