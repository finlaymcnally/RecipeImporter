# All-method dimension columns

- Slug-only config labels are hard to compare quickly when many all-method rows are present.
- Detail rows now include explicit columns: `Extractor`, `Parser`, `Skip HF`, `Preprocess`.
- Values come from benchmark `run_config` first, with fallback parsing from config directory slug tokens (e.g. `extractor_auto__parser_v2__skiphf_true__pre_br_split_v1`).
- For non-unstructured extractors (`legacy`, `markdown`, `markitdown`), parser/skip/preprocess columns are shown as `-` to avoid misleading default noise.
