## Discovery

For run `data/golden/benchmark-vs-golden/2026-02-26_22.30.40/all-method-benchmark/thefoodlabcutdown`:

- Scheduler events show prediction phases completing quickly (`prep` + `post` in ~20-45s), then all active configs transition to `evaluate_started`.
- No `evaluate_finished` events are emitted yet for the first six configs while the spinner shows `config 0/15`.
- Canonical alignment cache directory has only lock files (no cache payload yet):
  - `.cache/canonical_alignment/92/...lock`
  - `.cache/canonical_alignment/17/...lock`
- Process list shows two CPU-saturated `C3imp` workers and the rest mostly idle/waiting.

Interpretation:

- This is not split/prediction starvation; it is canonical-text evaluation bottlenecked on long-running `SequenceMatcher` alignment for the first unique cache keys.
- Other configs that share a key wait for that alignment to finish and cache to materialize.
- `scheduler heavy 0/N` during this period is expected because workers are in evaluate phase, not split-active phase.

## Practical implication

Scheduler/inflight/split-slot tuning helps throughput only before eval. When canonical eval dominates, wall time is controlled mostly by number of unique alignment keys and per-key `SequenceMatcher` cost.
