# External AI Cutdown Current Contract

The remaining cutdown cleanup is not a simple rename. `scripts/benchmark_cutdown_for_external_ai.py` still assumes the old benchmark recipe topology in two places at once.

- Prompt reconstruction still walks `chunking/schemaorg/final` directories and emits prompt rows keyed by `pass1/pass2/pass3`.
- Recipe triage still reads `recipe_manifest.json` as if each recipe row exposes `pass1/pass2/pass3`, degradation, fallback, and bridge counters.

The live single-correction runtime no longer publishes that shape. Current `recipe_manifest.json` rows expose semantic stage statuses (`build_intermediate_det`, `recipe_llm_correct_and_link`, `build_final_recipe`) plus structural and mapping fields, and richer per-recipe details live in `recipe_correction_audit/*.json`.

That means the correct cutdown migration is:

- rebuild prompt/call inventory around `recipe_correction` plus current stage metadata,
- derive deterministic final-stage diagnostics from manifest/audit files instead of synthetic pass3 prompt rows,
- update starter-pack triage/casebook fields so they describe the semantic stages rather than `pass1/pass2/pass3`.
