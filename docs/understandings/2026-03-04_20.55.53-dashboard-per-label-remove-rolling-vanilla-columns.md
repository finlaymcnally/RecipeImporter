# Discovery

- The rolling comparison columns are hard-coded in two seams inside `cookimport/analytics/dashboard_render.py`:
  - static table header HTML (`per-label-header-primary`/`per-label-header-rolling`), including the rolling group `colspan`,
  - `renderPerLabel()` row assembly (`perLabelComparisonCell(...)` chain).
- Column-count changes must also update the no-data row `colspan`; otherwise empty-state table rendering misaligns.
- Removing rolling-vanilla columns safely means dropping:
  - the two rolling vanilla `<th>` headers,
  - the two rolling vanilla comparison cells in row assembly,
  - the rolling-vanilla aggregation map/lookup now unused.
