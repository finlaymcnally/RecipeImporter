---
summary: "Benchmark `execution_mode` is an orchestration setting with exactly three stored values: `legacy`, `pipelined`, and `predict-only`."
read_when:
  - When interpreting `run_manifest.json` entries that show `execution_mode`
  - When comparing benchmark runs that used `llm_recipe_pipeline=codex-farm-3pass-v1`
---

- `execution_mode` is the `labelstudio-benchmark` runner mode, not the Codex Farm pipeline id and not pass3's internal `pass3_execution_mode`.
- `legacy` is the sequential/default path: run the full prediction bundle, then convert it to per-block prediction records with `predict_stage(...)`.
- `pipelined` keeps the same benchmark semantics but overlaps prediction-record production with evaluation prewarm/replay work using producer/consumer threads.
- Tests assert `legacy` and `pipelined` produce the same eval report payload apart from timing.
- `predict-only` still generates prediction artifacts/records but skips evaluation entirely.
