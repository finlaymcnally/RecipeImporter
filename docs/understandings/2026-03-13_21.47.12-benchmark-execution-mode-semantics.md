---
summary: "Benchmark `execution_mode` is an orchestration setting and the active runtime now stores only `pipelined`."
read_when:
  - When interpreting `run_manifest.json` entries that show `execution_mode`
  - When comparing benchmark runs that used `llm_recipe_pipeline=codex-farm-3pass-v1`
---

- `execution_mode` is the `labelstudio-benchmark` runner mode, not the Codex Farm pipeline id and not pass3's internal `pass3_execution_mode`.
- Public `labelstudio-benchmark` runs now always use `pipelined`.
- `pipelined` overlaps prediction-record production with evaluation prewarm/replay work using producer/consumer threads.
- The old public `legacy` and `predict-only` settings were removed; all-method keeps a private skip-evaluation prediction pass instead of depending on a user-visible `predict-only` mode.
