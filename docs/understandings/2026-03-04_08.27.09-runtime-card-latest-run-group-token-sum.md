---
summary: "Why runtime diagnostics token use was per-book and how latest run-group summation now works"
read_when:
  - "When Benchmark Runtime token use looks too small for multi-book runs"
  - "When changing latest diagnostics run-group selection logic"
---

`Benchmark Runtime` previously selected one latest benchmark row (`latestPreferredBenchmarkRecord`) and computed token use from that single row.

Per-label and boundary cards already aggregate by benchmark run-group key (`benchmarkRunGroupInfo` + `latestRunGroupRecords`). Runtime diagnostics now follows the same run-group path and sums discounted token totals across all rows in the latest preferred run-group.

AI model/effort/pipeline context is chosen from the best-metadata row within that run-group so `off` rows do not hide codex runtime context when mixed rows exist.
