# Discovery: Stage Docs IA Mapping

Key mapping used for docs reorganization:

- `cookimport/cli.py` and entrypoint wrappers map to `docs/02-cli/`.
- Importer registry + plugins + split-job orchestration map to `docs/03-ingestion/`.
- Ingredient/instruction/step-link/tip/chunk logic maps to `docs/04-parsing/`.
- Draft conversion + writers + staging contracts map to `docs/05-staging/`.
- Label Studio workflows map to `docs/06-label-studio/`.
- Offline suite benchmarking maps to `docs/07-bench/`.
- Perf report/dashboard/metrics artifacts map to `docs/08-analytics/`.
- Auto-tagging modules and commands map to `docs/09-tagging/`.
- Static schemas and inventories map to `docs/11-reference/`.

Outcome: a new AI coder can choose docs by requested stage rather than by historical note type.
