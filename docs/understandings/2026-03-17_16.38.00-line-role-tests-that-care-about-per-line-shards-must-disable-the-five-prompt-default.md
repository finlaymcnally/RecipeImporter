The current line-role runtime no longer treats `codex_batch_size=1` as enough to guarantee one atomic line per shard. `RunSettings.line_role_prompt_target_count` now defaults to `5`, so small batches are grouped into one shard unless a test explicitly sets `line_role_prompt_target_count=None` or a concrete `line_role_shard_target_lines`.

That mainly affects tests asserting exact shard ids, proposal filenames, or worker assignments. Those tests are checking shard-shape behavior, not the default prompt-target policy, so they should opt out explicitly instead of relying on the old implicit one-line-per-shard default.
