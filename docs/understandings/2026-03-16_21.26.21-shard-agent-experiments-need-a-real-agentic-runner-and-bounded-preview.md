# Shard-Agent Replacement Needs A Real Agentic Runner And Bounded Preview

- The current CodexFarm transport in `cookimport/llm/codex_farm_runner.py` audits itself as `structured_output_non_agentic`.
- That means shard-agent replacement cannot honestly be implemented as "same runner, just fewer bigger JSON tasks" if the goal is fewer fresh Codex workers with internal looping.
- Preview for shard agents also cannot reuse the old exact-looking prompt-char heuristic as if one rendered prompt equals one full cost. It can estimate shard boundaries, first-turn payload size, and fresh-agent count, but live telemetry must supply the real total tokens once compaction/looping happens.
- The safe experimental shape is:
  - deterministic prep
  - bounded local shard agent
  - strict normalized final output
  - deterministic merge
  - preview sweep for shard counts and boot payloads
  - live benchmark compare outside the agent shell for real quality/token frontier data
