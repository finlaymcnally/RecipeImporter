## Discovery

Stage execution now routes through `ProgressDashboardCore` snapshot rendering instead of a local dashboard type, and the `COOKIMPORT_PLAIN_PROGRESS` env override is respected before entering the stage job loop.

The `_run_jobs` helper needed closure-scoped assignment fixes (`nonlocal`) for backend metadata so `stage_worker_resolution.json` records the resolved backend and messages after a stage run.

Live-status mode now updates a shared status widget in `status.update(...)`; plain mode strips rich tags and prints plain messages while still using the same rendered snapshot string.
