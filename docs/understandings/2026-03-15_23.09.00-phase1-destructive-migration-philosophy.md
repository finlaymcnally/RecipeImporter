The important seam is philosophical, not just technical. Phase 1 is no longer an additive observability cleanup that coexists with the old pass-slot worldview. It is the cutover point where the repo should stop teaching `pass1` through `pass5`, stop emitting compatibility-only fields such as `pass2_stage`, and stop carrying historical fallback readers whose only job is to preserve the old naming model.

Short rule for future edits:

- New runs should expose semantic stage names only.
- Raw stage-owned directories should also use semantic names.
- Prompt artifacts, run summaries, manifests, and benchmark bundles should all read the same shared stage truth.
- Old fixtures or benchmark roots should be regenerated, not supported via compatibility code.
