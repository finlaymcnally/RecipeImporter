summary: "Benchmark status rendering only promoted `running N` into worker rows for `codex-farm ...` messages, while some codex-farm builds also echoed redundant plain stderr run snapshots."
read_when:
  - "When canonical line-role benchmark progress shows only one boxed status line despite `task X/Y | running N` updates."
  - "When codex-farm benchmark runs emit repeated `run=... queued=... running=...` stderr snapshots and you need to decide whether they are real errors."
---

- `_run_with_progress_status(...)` already knew how to render worker rows, but the generic benchmark path only populated that state for codex-farm-prefixed messages. Canonical line-role progress used the same `task X/Y | running N` shape without the codex-farm prefix, so the worker panel stayed empty.
- `SubprocessCodexFarmRunner` can receive both structured `__codex_farm_progress__ ...` events and plain legacy stderr snapshots like `run=<id> queued=... running=...`. Those plain snapshots are just progress noise when the structured channel is active and should be suppressed instead of surfaced as normal stderr.
