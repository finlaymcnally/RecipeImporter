# Dashboard responsive layout notes (2026-03-04)

- The main non-responsive behavior came from `#previous-runs-table` using a hard `min-width: 1600px`, which prevented natural shrink on narrower viewports.
- Trend charts were rendered with a measured pixel width at render time, but there was no window-resize trigger to rebuild charts with the new host width.
- Fix strategy used:
  - Replace hard 1600px floor with a clamped baseline width plus narrow-screen wrapping rules for table cells.
  - Add a debounced resize handler that re-renders trend charts on viewport width changes.
