# Repo Legacy Audit: Current State

Quick grep over `cookimport/`, `tests/`, and `docs/` shows the legacy burden has shifted.

- Active runtime/code hits are down to a small set: `cookimport` has 25 matching files / 45 matching lines for `legacy|compatibility|deprecated|obsolete|pass1|pass2|pass3|pass4|pass5`, and most of those are comments, compatibility notes, rejected old inputs, or historical-reader handling rather than core recipe execution.
- Tests have 12 matching files / 21 matching lines. Most are fixture names, assertions about removed compatibility aliases, or validation that removed settings still fail.
- Docs are now the largest remaining bucket at 47 matching files / 495 matching lines.

The remaining repo-wide clusters are:

- Live but non-core compatibility seams in current runtime code, especially older artifact readers/helpers in `cookimport/cli.py`, historical metric aliases in analytics/dashboard surfaces, and a few wrapper-style comments/docstrings in parsing/core helpers.
- Current docs that still intentionally describe compatibility or historical-reader behavior, especially under `docs/01-architecture`, `docs/05-staging`, `docs/06-label-studio`, `docs/08-analytics`, and `docs/10-llm`.
- Archival docs that still contain heavy old terminology by design: `docs/plans/old/`, `docs/plans/OGplan/`, `docs/reports/Refactor.md`, today’s purge task files, and related understandings.
- Stale test scaffolding around deleted tagging surfaces: `tests/tagging/` now only contains `__pycache__`, while `tests/tagging_gold/` and several test-doc metadata files still refer to the old tagging domain.
- Checked-in `__pycache__` trees under `tests/`, which amplify the “old stuff still exists” feeling even when they are not live source.

If the goal is maximum purge impact per unit effort, the next passes should be:

1. remove the remaining live compatibility helpers/alias reads in current Python code;
2. prune stale test scaffolding (`tests/tagging`, `tests/tagging_gold`, tagging mentions in test docs/config);
3. decide whether archival docs should be retained as history or aggressively deleted in a docs-only cleanup.
