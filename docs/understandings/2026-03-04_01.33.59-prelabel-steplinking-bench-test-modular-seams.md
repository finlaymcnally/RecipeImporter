# Discovery

Three mixed modules still combined unrelated seams, so routine edits often triggered broad test collection and less focused failures.

# Resolution

- Prelabel tests are split into prompt/span labeling contracts and codex CLI transport/config/account contracts.
- Step ingredient linking tests are split into core assignment/split heuristics and semantic/fuzzy/collective matching behavior.
- Bench tests are split into helper/noise/cost contracts, speed CLI wiring, and quality CLI wiring.
- New split files reuse shared fixtures/imports from their base module via non-test global injection.
