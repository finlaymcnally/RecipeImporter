# Quality-Run Auto Cap Ceiling and Ramp

- The active tournament run (`data/golden/bench/quality/tournaments/2026-02-28_11.23.00_top_tier_tournament.log`) showed auto mode pinned at `workers_cap=8` while host load-per-cpu stayed low (`~0.12-0.13`), which left CPU headroom unused.
- `run_quality_suite(...)` now uses auto effective cap:
  - `min(total_experiments, cpu_count, auto_ceiling)`
  - default `auto_ceiling=16`
  - env override: `COOKIMPORT_QUALITY_AUTO_MAX_PARALLEL_EXPERIMENTS`
- Auto admission now seeds the initial worker target from current load and ramps up in larger bounded increments (`ceil(cap * 0.25)`), so it reaches available capacity faster when the host is cool.
- `experiments_resolved.json` now includes:
  - `max_parallel_experiments_auto_ceiling`
  - `max_parallel_experiments_auto_ceiling_source`
  - `max_parallel_experiments_auto_ceiling_env`
