---
summary: "Phase 2 cutover must replace candidate-owned runtime decisions in staging and Label Studio with one shared authoritative label bundle."
---

- `cookimport/staging/import_session.py` is the highest-leverage runtime seam: it already sits between importer conversion and all persisted stage artifacts, so Phase 2 can replace candidate-owned `ConversionResult` contents there without rewriting every downstream writer at once.
- `cookimport/labelstudio/ingest.py` currently duplicates authority by running `label_atomic_lines(...)` after processed stage output generation and then prefers the projected line-role artifacts for scoring. Phase 2 should stop rerunning that pass and instead reuse the stage-produced authoritative labels.
- `cookimport/staging/stage_block_predictions.py` is still recipe-derived today. To make labels authoritative, benchmark evidence should be written directly from normalized block labels instead of being reconstructed from staged recipes plus knowledge hints.
- The existing canonical line-role code in `cookimport/parsing/canonical_line_roles.py` is reusable, but its `within_recipe_span` input can no longer come from importer candidate ranges. Any hint carried into Phase 2 must be derived from Phase 2 labels themselves, not from `_detect_candidates(...)` or `_split_recipes(...)`.
