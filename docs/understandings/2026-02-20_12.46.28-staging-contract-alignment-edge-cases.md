# Staging Contract Alignment Edge Cases

Cookbook's `RecipeDraftStagingV1` allows `source` to be null, but rejects empty strings. `recipe_candidate_to_draft_v1` therefore must emit `null` for blank source values.

For ingredient lines, linked-recipe rows need stricter cleanup than normal ingredient rows:

- blank `linked_recipe_id` values must be cleared (`null`) so the line does not trip one-of ID validation
- recipe-line `input_qty` must stay within `> 0` and `<= 100`
- recipe-line `input_unit_id` must always be `null`

Without these normalizations, outputs can look structurally fine in local staging tests but fail on Cookbook's runtime contract parser.
