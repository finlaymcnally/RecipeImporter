## Discovery

The nine task files under `docs/tasks` were all runtime-behavior specs from the same date window and mapped cleanly to three domain folders:

- `docs/02-cli`: startup SemLock warning guard contract (`2026-02-28_14.46.37`).
- `docs/05-staging`: stage worker fallback/telemetry contracts (`2026-02-28_14.42.42`, `2026-02-28_15.00.57`).
- `docs/07-bench`: SpeedSuite/QualitySuite orchestration, profiles, ETA, lightweight series, two-phase parser workflow, and Oracle gap closure (`2026-02-28_14.55.16`, `15.49.40`, `20.35.43`, `21.43.13`, `22.08.25`, `2026-03-01_09.48.35`).

## Why this mapping

- Stage worker label/fallback behavior is stage runtime behavior first, even when surfaced by SpeedSuite telemetry.
- Joblib SemLock guard is import/startup behavior first, so CLI domain is the durable home.
- The remaining six tasks define benchmark command contracts, run-shape policy, and tournament orchestration; they belong in bench docs.

## Merge rule used

- Keep chronology explicit by timestamp.
- Keep durable contracts and anti-loop notes.
- Keep test/evidence commands for future verification.
- Remove dependency on `docs/tasks` files after merge.
