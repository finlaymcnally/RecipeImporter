# Discovery

The runtime chooser was already simplified to two profile families (`codexfarm` / `vanilla`), but surrounding surfaces still carried legacy behavior:
- dead interactive benchmark `all_method` branch remained in `cookimport/cli.py` despite no menu path,
- last/preferred settings persistence APIs were no longer consumed by runtime,
- full-screen toggle editor code/tests/docs remained after chooser simplification,
- docs still instructed global/last/edit chooser behavior.

# Resolution

- Keep interactive run selection strictly on two top-tier profiles.
- Remove dead interactive all-method routing and obsolete last/preferred persistence APIs.
- Retire unused toggle-editor module/tests.
- Align docs/tests with current behavior and keep qualitysuite winner persistence only.
