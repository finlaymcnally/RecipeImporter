## Discovery

- Global `avg_seconds_per_task` underestimates all-method tail time because early fast configs dominate the full-run average.
- All-method snapshots already include `task: ... eval <n> | active <n> ...`, which can be used to detect long eval tails with no new completions.
- A practical ETA improvement is:
  - use recent completion-rate samples as the primary average,
  - if eval is active and completion counter has stalled, apply a lower-bound floor `stalled_seconds / active`.

## Practical implication

When the run is deep in canonical eval, ETA no longer stays pinned to unrealistically small values from early-run throughput.
