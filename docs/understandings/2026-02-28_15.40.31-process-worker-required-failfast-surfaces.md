summary: "Process-worker-required mode now fails fast across stage/all-method/quality/speed and emits executor-resolution telemetry."
read_when:
  - "When enabling --require-process-workers for stage, speed-run, or quality-run."
  - "When a benchmark/stage run exits early instead of using fallback worker backends."
---

`--require-process-workers` is now a strict contract, not just a preference.

Discovery summary:
- Stage: `cookimport stage --require-process-workers` now raises immediately if process workers cannot be established (before subprocess/thread/serial fallback), and records worker resolution in `stage_worker_resolution.json`.
- All-method: both global and legacy scheduler paths now enforce strict process-worker availability and startup/restart checks; strict mode raises instead of degrading to thread/serial.
- QualitySuite and SpeedSuite: bench wrappers forward strict mode into all-method/stage internals, so strict behavior is enforceable from `bench quality-run` and `bench speed-run`.
- Telemetry: all-method reports now include `executor_resolution` (`process_workers_required`, probe status/error, and backends seen), and suite summaries include strict/probe fields for auditability.
