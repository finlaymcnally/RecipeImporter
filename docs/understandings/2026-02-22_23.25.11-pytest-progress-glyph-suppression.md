# Pytest Progress Glyph Suppression

In this repo (pytest 9.x), `-q` still prints per-test progress rows by default.

To make successful runs compact:
- Set `console_output_style = classic` in `pytest.ini`.
- Keep `tests/conftest.py:pytest_report_teststatus(...)` returning empty shortletters for pass/skip call-phase reports.

Without `console_output_style = classic`, suppressed glyphs still leave a progress line ending in `[100%]`.
With both controls, successful runs collapse to a single summary line (plus failure hints on errors).
