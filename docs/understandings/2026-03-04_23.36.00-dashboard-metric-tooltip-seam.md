## Discovery
- Tooltip descriptions can be sourced directly from `#previous-runs-section .metric-help-list li` (`<code>metric_key</code>: description`) so the help list stays the canonical copy.
- `PREVIOUS_RUNS_COLUMN_META` is a good fallback for keys not present in the long help list.
- A delegated tooltip handler on `[data-metric-tooltip-key]` avoids per-node listener churn when Previous Runs / Compare & Control rerender.

## Wiring points
- Tooltip engine + description catalog: `cookimport/analytics/dashboard_render.py` JS template (`setupMetricHoverTooltips`, `buildMetricTooltipDescriptions`).
- Metric target tagging: Previous Runs header render, trend/checklist render, runtime table rows, per-label headers, compare/control grouped-metric headers.
