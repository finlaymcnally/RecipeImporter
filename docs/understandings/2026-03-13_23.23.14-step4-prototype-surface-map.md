# Step4 Prototype Surface Map

The durable seam for a new CodexFarm recipe pipeline is wider than `cookimport/llm/codex_farm_orchestrator.py`. The pipeline value has to survive four layers: `RunSettings` and CLI or Label Studio normalization, `cookimport/config/codex_decision.py` recipe-surface detection for `--allow-codex`, orchestrator pass planning and manifest writing, and benchmark/helper code that decides when LLM debug artifacts are required.

The current repo still hard-codes `codex-farm-3pass-v1` in several places where the real meaning is “any non-off recipe Codex pipeline is enabled.” Those spots include `RunSettings.from_dict(...)`, CLI and Label Studio `_normalize_llm_recipe_pipeline(...)`, `classify_codex_surfaces(...)`, the interactive benchmark paired-variant chooser, and benchmark source-debug checks.

That means step4 should be implemented as an additive alternate recipe pipeline, not as a one-off branch only inside the orchestrator. If only the orchestrator changes, the new pipeline will fail earlier at normalization or skip approval/debug behavior incorrectly.
