# Knowledge Cost Shape

For the current Food Lab full preview (`data/output/2026-03-18_17.15.00-foodlab-current-preview`), the knowledge stage carries about `212k` task-body input tokens against only about `86.6k` tokens of raw `blocks[*].text`.

The expensive part is not duplicate chunk reuse:

- repeated chunk ids across bundles: `0`
- repeated block indices across bundles: `0`
- context blocks populated: `0`

The extra cost is the review scaffold itself:

- `465` chunk objects
- `1,883` block objects
- per-chunk ids, span refs, heuristics, review hints, block ranges
- top-level source/span/guardrail metadata
- a richer output contract than line-role (`d`, `rc`, `s`, evidence quotes, one result per chunk)

So the current knowledge design appears directionally sensible in grounding terms, but it is still expensive because each surviving chunk is wrapped in a fairly heavy machine-readable review contract.
