# Pass1 Eligibility Chapter/Page Negative Evidence

`cookimport/llm/codex_farm_orchestrator.py` now treats chapter/page heuristic metadata as explicit negative evidence in pass1 eligibility scoring, rather than leaving it implied under prose dominance.

Implementation details:
- Added metadata hint detection across block-level and `features`/`metadata` dictionaries.
- Added score component fields: `chapter_page_negative_evidence_high`, `chapter_page_negative_hits`, and `chapter_page_negative_score` (`-2` when triggered).
- Included a new reason tag: `chapter_page_metadata_negative_evidence_high`.

Coverage:
- `tests/llm/test_codex_farm_orchestrator.py::test_orchestrator_pass1_eligibility_uses_chapter_page_negative_metadata` verifies this signal can force a `proceed` candidate down to `clamp`.
