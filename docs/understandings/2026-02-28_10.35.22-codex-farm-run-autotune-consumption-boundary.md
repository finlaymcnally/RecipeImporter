# CodexFarm Run Autotune Consumption Boundary

## Discovery

CodexFarm `run autotune --json` already translates telemetry playbook output into actionable caller artifacts (`flag_overrides`, `command_preview`, unified diffs). This is higher-level guidance than raw telemetry fields and can be persisted directly in recipeimport pass metadata.

## Applied boundary

- Runner performs a best-effort autotune follow-up after successful `process` calls (requires `run_id`).
- Payload is persisted as top-level `autotune_report` in each pass `process_run`/`process_runs` entry.
- Failures in autotune retrieval are non-fatal and leave `autotune_report` null.

## Why this shape

- Keeps recipeimport aligned with CodexFarm’s caller automation contract without enabling any automatic mutation.
- Preserves existing telemetry layers:
  - `telemetry_report` for schema-v2 insights/playbook,
  - `autotune_report` for concrete override/diff suggestions,
  - `telemetry` for compact row-level details.
