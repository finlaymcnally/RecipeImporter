For codex-farm vs non-codex quality comparison, the high-signal files are:

- `run_manifest.json` (or extracted subset): confirms `llm_recipe_pipeline` and key run settings.
- `eval_report.md` and core metrics from `eval_report.json`: overall accuracy, macro F1, practical metrics.
- Small error samples from `wrong_label_lines.jsonl`, `missed_gold_lines.jsonl`, and `unmatched_pred_blocks.jsonl` for qualitative inspection.

Low-signal/high-volume files for this use case are:

- `prediction-run/raw/**` prompt/response payloads
- `prediction-run/extracted_archive.json`
- `prediction-run/label_studio_tasks.jsonl`
- `prediction-run/extracted_text.txt`
- full diagnostic JSONL files and processing timeseries

These large files are useful for pipeline debugging, but not required to judge codex-farm quality deltas.
