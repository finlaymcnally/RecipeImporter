# Discovery

Three regressions explained the remaining OG-plan closure failures:

- `labelstudio-benchmark` transient prune hook reused dashboard exclusion logic that intentionally marks pytest temp eval paths as excluded, which caused benchmark helper tests to delete their own `eval/` artifacts.
- Stage-block benchmark execution was inheriting top-tier defaults (`line_role_pipeline=codex-line-role-v1`, `atomic_block_splitter=atomic-v1`) even though line-role/atomic are canonical-text features in this flow.
- `_merge_split_jobs(...)` treated missing `run_config` as implicit top-tier defaults, so direct test calls (with `run_config=None`) attempted codex-farm recipe passes and failed without full-text blocks.

# Implemented Fixes

- `cookimport/cli.py::_prune_benchmark_outputs(...)` now bypasses prune for pytest temp eval fixtures while keeping transient slop prune behavior for real run roots.
- `cookimport/cli.py::labelstudio_benchmark(...)` now forces `line_role_pipeline=off` and `atomic_block_splitter=off` for `eval_mode=stage-blocks`.
- `cookimport/cli.py::_merge_split_jobs(...)` now gates LLM recipe/knowledge execution behind explicit `run_config` presence.
