# Single-profile paired variants restored

`_interactive_single_offline_variants()` is shared by interactive single-offline and single-profile benchmark planning.

The bug was that the `codex-farm-3pass-v1` branch returned only `("codexfarm", selected_benchmark_settings)`, so single-profile Codex-selected runs produced one full-stack eval per book instead of the documented paired `vanilla` then `codexfarm` layout.

The fix is to build a benchmark baseline payload with `apply_benchmark_baseline_contract(...)`, then derive the Codex variant from that baseline with `apply_benchmark_codex_contract_from_baseline(...)`, returning both variants in order.
