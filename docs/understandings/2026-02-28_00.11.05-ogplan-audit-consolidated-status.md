## All-method forwarding findings (key merged outcome)

### Execution-lane split

- Single benchmark lane (adapter-driven) forwards the full adapter payload into `labelstudio_benchmark(...)`.
- All-method lane (`_run_all_method_prediction_once(...)`) forwards a subset.
- Speed/quality all-method wrappers inherit the same subset behavior from that all-method lane.

### Coverage counts

- Adapter setting keys: `58`
- All-method forwarded setting keys: `25`
- Missing in all-method forwarding: `33`

### Forwarded families in all-method

- EPUB, worker, OCR, and pipeline controls
- `section_detector_backend`
- instruction segmentation controls (`instruction_step_segmentation_policy`, `instruction_step_segmenter`)
- matcher selection (`sequence_matcher`)

### Missing families in all-method

- Priority 1 recipe-score controls (`recipe_scorer_backend`, `recipe_score_*`)
- Priority 3 multi-recipe controls (`multi_recipe_*`)
- Priority 4 ingredient parser controls (`ingredient_*`)
- Priority 6 time/temperature/yield controls (`p6_*`)
- Priority 7 webschema controls (`web_schema_*`, `web_html_text_extractor`)
- output toggles (`write_label_studio_tasks`, `write_markdown`)

### Experiment-interpretation severity map

| Family | Severity | Why |
| --- | --- | --- |
| Priority 6 (`p6_*`) | High | Reported cooking-time/temperature/yield behavior can diverge from intended config in all-method runs. |
| Priority 7 (`web_schema_*`) | High | Variants can advertise schema-policy dimensions while effective extraction behavior falls back to defaults. |
| Priority 3 (`multi_recipe_*`) | High | Split behavior dimensions may not reflect actual prediction-time splitter/threshold controls. |
| Priority 1 (`recipe_scorer_backend`, `recipe_score_*`) | Medium | Recipe-likeness gating sensitivity in all-method experiments can be mischaracterized. |
| Priority 4 (`ingredient_*`) | Medium | Ingredient parser backend/normalization/policy tuning may not be exercised in all-method comparisons. |
| Output toggles (`write_label_studio_tasks`, `write_markdown`) | Low | Mostly affects artifact surfaces, not prediction semantics. |

## Reconciled interpretation

These two statements are simultaneously true:

- Priority-family implementations (including Priority 6) exist in runtime, adapters, and non-all-method benchmark lanes.
- All-method prediction execution still omits many of those same run-setting keys.

So the core features are implemented, but all-method variant execution is not yet a full-fidelity carrier of the complete run-settings surface.

## Reporting nuance and risk

- All-method rows include variant dimensions from variant generation.
- Effective prediction config can differ where forwarding is missing.
- Result: dimension labels can imply non-default behavior while prediction-generation kwargs fall back to defaults.
