## Discovery

- Canonical line-role ETA depended on `task X/Y` callbacks.
- Before this change, only codex batch escalation emitted `task X/Y`; deterministic labeling emitted only phase text.
- So long deterministic-only stretches showed elapsed time but no ETA.

## Change

- `label_atomic_lines(...)` now emits deterministic-phase progress callbacks:
  - `Running canonical line-role pipeline... task X/Y`
- Codex batch callbacks remain and continue to emit:
  - `Running canonical line-role pipeline... task X/Y | running N`
- Spinner worker summary now suppresses zero-only rows (`active workers: 0`).

## Effect

- Canonical line-role stage ETA behavior is unified across single-book and multi-book runs because both use the same shared callback/status path.
- Operators see ETA sooner in canonical line-role, including pre-codex deterministic work.
