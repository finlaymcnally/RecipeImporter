## Discovery

- Interactive all-method prompt asks `Try deterministic option sweeps too? ...` and passes `include_deterministic_sweeps` into `_build_all_method_target_variants(...)`.
- Sweep expansion logic exists in `_build_all_method_sweep_payloads(...)` and includes section detector, multi-recipe splitter, ingredient missing-unit policy, instruction segmentation, and p6 time/temp/yield knobs (time/temp gated by optional deps).
- `bench quality-run` now exposes `--include-deterministic-sweeps` and forwards it into `_build_all_method_target_variants(...)`.
- Default behavior is still `False` (`--no-include-deterministic-sweeps`), so existing runs stay stable unless this flag is enabled.
- QualitySuite can test these knobs either by enabling this sweep flag, by explicit `experiments`/`levers` run-settings patches, or both.
- Existing checked-in quality run artifacts under `data/golden/bench/quality/runs/*/experiments_resolved.json` show empty `run_settings_patch` rows for recorded runs.
- Existing example lever file (`data/golden/bench/quality/experiments/2026-02-28_01.18.41_qualitysuite-levers.json`) includes these deterministic levers but all are disabled by default and includes yield-only for P6 (no time/temp levers in that sample file).
