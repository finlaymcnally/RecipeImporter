# Discovery: benchmark-vs-gold extractor parity

Investigated benchmark run `data/golden/benchmark-vs-golden/2026-02-25_19.07.15` against gold export `data/golden/pulled-from-labelstudio/saltfatacidheatcutdown_2`.

Findings:
- Gold came from a Label Studio import run with `epub_extractor=unstructured`.
- Benchmark prediction run used `epub_extractor=beautifulsoup`.
- Gold-labeled block coverage ended around the 1470 range, while benchmark stage predictions labeled through block 1993.
- Stage-block eval defaults missing gold block rows to `OTHER`, so predictions on drifted indices count as false positives.
- In this run, all 7 predicted `RECIPE_VARIANT` blocks landed in missing-gold indices, so `RECIPE_VARIANT` precision and recall were both `0`.

Operational rule:
- Keep extractor/blockization settings aligned between:
  1. the run used to generate Label Studio tasks/gold, and
  2. the benchmark prediction run being compared.

Related processing note:
- `KNOWLEDGE` in stage-block predictions is only injected from `knowledge_snippets_path` (LLM knowledge pipeline output). With `llm_knowledge_pipeline=off`, `KNOWLEDGE` prediction count is expected to remain `0`.
