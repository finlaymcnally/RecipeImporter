read_when:
  - "When codexfarm and vanilla benchmark trend points appear horizontally misaligned for what should be one paired run."
  - "When modifying benchmark run-group timestamp extraction in dashboard JS."
---

# Discovery

- `benchmarkRunGroupInfo` previously took the last timestamp token when `benchmark-vs-golden/<timestamp>` was not immediate in the path.
- With paths like `.../benchmark-vs-golden/<source>/<run_ts>/.../<variant_local_ts>/vanilla|codexfarm`, that fallback selected different `variant_local_ts` values and split paired points across the x-axis.

# Resolution

- Run-group extraction now scans `artifact_dir`, `run_dir`, then `report_path`.
- If `benchmark-vs-golden` exists, it takes the first timestamp token after that marker (shared run token).
- If no marker exists, it falls back to the first timestamp token in path order, then `run_timestamp`.
