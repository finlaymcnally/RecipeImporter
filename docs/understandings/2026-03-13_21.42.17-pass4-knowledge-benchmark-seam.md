# Pass4 knowledge benchmark seam

- In benchmark session `2026-03-06_15.22.11`, the codex runs for Salt Fat Acid Heat, Sea and Smoke, and The Food Lab all wrote `prediction-run/raw/llm/<book>/pass4_knowledge_manifest.json` plus `prediction-run/knowledge/<book>/snippets.jsonl`.
- Those manifests show real Codex Farm execution (`done=149/24/238`), even though `upload_bundle_index.json` reports `total_pass4_call_count=0`.
- The zero-call report comes from `cookimport/llm/prompt_budget.py`, which only summarizes `pass1`, `pass2`, `pass3`, and `line_role`; pass4 is excluded from `prompt_budget_summary.json`, and the upload bundle derives `pass4_call_count` from that summary.
- Current pass4 integration is not a full non-recipe `knowledge` vs `other` classifier. `cookimport/labelstudio/canonical_line_projection.py` only uses pass4 snippets to upgrade projected outside-span `OTHER` lines to `KNOWLEDGE`.
- In this session, scored eval artifacts show little or no benchmark gain from that seam:
  - Salt Fat Acid Heat: outside-span `KNOWLEDGE` precision/recall/F1 stayed `0.873/0.117/0.206` for both vanilla and codex.
  - Sea and Smoke: outside-span `KNOWLEDGE` stayed `0` true positives for both vanilla and codex.
  - The Food Lab: codex increased outside-span predicted `KNOWLEDGE` from `64` to `65` with the same `42` true positives, so recall stayed `0.0627` and precision got slightly worse.
- The codex `prediction-run/line-role-pipeline/stage_block_predictions.json` files also lack the expected pass4 merge note (`Pass4 knowledge evidence merged...`), which suggests the current benchmark-scored projection path is either not receiving snippet evidence or is receiving it but not producing measurable upgrades in these books.
