## Discovery

In `cookimport/cli.py:_merge_split_jobs(...)`, report writing previously happened before `_merge_raw_artifacts(...)`.
That ordering could omit moved raw files from `report.outputStats` even though those files exist in final output.

## Evidence and implication

- `_merge_raw_artifacts(...)` physically moves files from `.job_parts/<workbook>/job_*/raw/**` into `raw/**` after writer output phases.
- If report JSON is emitted first, post-merge files are missing from stats.
- Correctness fix is to:
  1. record merged `raw/.../full_text.json` when written,
  2. record each moved raw destination in `_merge_raw_artifacts(...)`,
  3. write report after raw merge.

## Guardrail

`tests/staging/test_split_merge_status.py::test_merge_split_jobs_output_stats_match_fresh_directory_walk`
now compares report `outputStats` against a fresh directory walk categorized by output artifact type.
