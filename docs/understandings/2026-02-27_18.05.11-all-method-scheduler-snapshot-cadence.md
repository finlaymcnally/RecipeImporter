## Discovery

`_run_all_method_benchmark(...)` drives scheduler updates from a poll loop with `ALL_METHOD_SCHEDULER_POLL_SECONDS = 0.15`.

- Scheduler counts are recomputed each loop tick.
- Spinner/dashboard `task: scheduler heavy ...` updates emit only when the snapshot string changes.
- During unchanged states (long evaluate/post phases), spinner text can look "stuck" even though the loop is still ticking.

This makes a persisted heartbeat time-series useful for tuning: it keeps regular datapoints even when visible snapshot text does not change.
