## Discovery

Interactive `single_offline` currently executes exactly one `labelstudio_benchmark(...)` call from `_interactive_mode()` and has no built-in paired baseline behavior.

For paired codex-enabled runs, the safe contract is:
- run `vanilla` first,
- run `codexfarm` second,
- keep successful `vanilla` artifacts even if codex fails,
- write comparison artifact only when both runs succeed.

## Why this matters

This guarantees at least one usable baseline artifact for every requested codex-enabled interactive run, and prevents misleading partial comparison files.
