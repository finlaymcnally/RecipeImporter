# Prompt Preview Quality Risk Pattern

Reviewing fresh zero-token previews for `thefoodlabcutdown` and `saltfatacidheatcutdown` showed that the biggest quality risk is not raw prompt size by itself.

The main issue is instruction and payload mismatch:

- knowledge currently appends inline JSON in `build_knowledge_direct_prompt(...)`, but the checked-in template still tells Codex to read `{{INPUT_PATH}}`, so the rendered prompt gives contradictory transport guidance
- live/preview line-role uses the short file-wrapper prompt, not the richer inline prompt with precedence, negative rules, and examples
- recipe correction is sometimes asked to "correct" spans that are obviously not recipes at all (for example conversion tables, section headings, knife recommendations) while the prompt still frames them as authoritative recipe spans and offers no reject/non-recipe escape hatch

That means agent failures here are likely to look like hallucination or poor judgment even when the deeper cause is that the contract is asking the model to reconcile conflicting or misleading instructions.
