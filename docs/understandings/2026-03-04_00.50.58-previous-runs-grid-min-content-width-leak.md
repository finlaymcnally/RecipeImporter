# Discovery

- `Previous Runs` uses wide children by design (`#previous-runs-table` has `min-width: 1600px` for readability).
- Without explicit grid track constraints, implicit grid sizing can honor child min-content width and push the whole section wider.
- Fix belongs at container level: force track shrinkability (`minmax(0, 1fr)`) and set `min-width: 0` on subsection/host wrappers so width pressure stays inside local scrollers.

# Outcome

- The section no longer expands rightward when wide content appears; local horizontal scroll behavior remains the mechanism for wide columns.
