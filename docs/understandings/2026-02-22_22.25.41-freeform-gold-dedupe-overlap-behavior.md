# Discovery

Freeform export rows are converted to absolute block ranges using `touched_block_indices` (or `touched_blocks[].block_index`) before evaluation.

`evaluate_predicted_vs_freeform(...)` dedupes gold spans before scoring, keyed by:
- `source_hash`
- `source_file`
- `start_block_index`
- `end_block_index`

Because dedupe is range-key based, it is overlap-count-agnostic: configured overlap (`segment_overlap`) and auto-tuned effective overlap can change how many duplicate rows are produced, but duplicates collapse as long as they resolve to the same absolute block range.

Caveat: near-duplicates with different block ranges are not merged. Only exact `(source_hash, source_file, start, end)` matches dedupe; label conflicts on those exact matches resolve by majority vote (ties are dropped).
