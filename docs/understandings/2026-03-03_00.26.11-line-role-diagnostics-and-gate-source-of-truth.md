# Line-role diagnostics + gates source-of-truth

- The most reliable contract for cross-artifact consistency is a single joined canonical line table keyed by `sample_id=line:<line_index>`.
- `wrong_label_lines.sample`, `correct_label_lines.sample`, `aligned_prediction_blocks.sample`, and `line_role_flips_vs_baseline.sample` should all be derived from that one joined table slice so IDs and text stay stable.
- Regression gates need three metric sources:
  - candidate run (`eval_report.json`) for current macro/accuracy/recalls/confusions,
  - latest vanilla canonical baseline from benchmark history (`llm_recipe_pipeline=off`, `line_role_pipeline=off`),
  - latest Codex canonical baseline (`llm_recipe_pipeline=codex-farm-3pass-v1`, `line_role_pipeline=off`) for confusion-drop checks.
- Optional SeaAndSmoke non-regression gates are practical to evaluate from benchmark history when local source exists, instead of requiring one command invocation to benchmark both books.
