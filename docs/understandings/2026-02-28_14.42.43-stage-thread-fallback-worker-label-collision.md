# Stage Thread-Fallback Worker Label Collision

- In stage thread-fallback mode, worker identity came from process label only (`MainProcess (pid)`).
- ThreadPool workers share process identity, so worker-status maps collapsed all concurrent thread workers to one key.
- This made `processing_timeseries.jsonl` snapshots under-report active workers and looked like forced serial execution.
- Fix: append thread name for non-main threads in worker labels (for example `MainProcess (1234) / ThreadPoolExecutor-0_3`), while keeping main-thread labels unchanged.
