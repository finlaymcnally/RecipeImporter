# Label Studio resume gate for new projects

- Root cause: `run_labelstudio_import(...)` always checked local manifest resume metadata whenever `resume=True`, even when this run had just created a brand-new Label Studio project.
- Symptom: stale manifests from older runs could raise `Existing project uses task_scope=...` or skip uploads into an otherwise empty new project.
- Durable rule: apply resume metadata only when the target Label Studio project already existed before this run.
- Additional edge case: benchmark upload can still collide with an existing project title that truly exists in Label Studio under a different scope (for example freeform vs pipeline), which is common when project names are auto-derived from source stem.
- Benchmark-side recovery rule: when scope mismatch is found during resume checks and benchmark used auto project naming, create a deduped project title (`-1`, `-2`, ...) and continue instead of failing interactive flow.
- Patch location: `cookimport/labelstudio/ingest.py` (`had_existing_project` gate and `auto_project_name_on_scope_mismatch` fallback), wired from `cookimport/cli.py:labelstudio_benchmark`.
