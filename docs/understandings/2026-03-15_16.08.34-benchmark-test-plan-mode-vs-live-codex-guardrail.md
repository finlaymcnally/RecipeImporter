---
summary: "Benchmark helper tests that only validate local artifact wiring should use plan mode once agent environments hard-block live Codex benchmark execution."
read_when:
  - "When a benchmark helper test fails because live Codex benchmark execution is blocked in agent environments"
  - "When deciding whether a Label Studio benchmark test should use `codex_execution_policy=plan`"
---

- Non-interactive `labelstudio-benchmark` runs with live Codex-backed surfaces are intentionally blocked in agent environments.
- Tests that only need to validate local benchmark scratch paths, run-manifest wiring, or compatibility artifacts should use `codex_execution_policy=plan` instead of trying to exercise live execution.
- Once a benchmark helper test moves to plan mode, its expectations must match the plan contract: no live `llm_manifest_json`, no upload bundle, plan artifact path present, and prediction metadata persisted under plan-oriented manifest fields rather than execute-only outputs.
