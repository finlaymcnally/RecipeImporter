# QualitySuite Checkpoint/Resume Surface Map

- `run_quality_suite(...)` already persisted rich per-experiment all-method artifacts, but run-level `summary.json` and `report.md` were emitted only after all experiments completed.
- Because of that, host crashes left usable experiment folders but no canonical run-level state for compare/tournament flows.
- The lowest-risk recovery seam is:
  - write one normalized experiment result snapshot per completed experiment,
  - regenerate run-level checkpoint + partial summary/report after each completion,
  - add explicit `--resume-run-dir` to reload completed snapshots and execute only pending experiments.
- Tournament fold recovery can stay simple by reusing the latest partial fold run dir and forwarding `--resume-run-dir` to `bench quality-run`.
