---
summary: "How to derive low-token 'correct' snippet samples for benchmark runs without shipping full prediction payloads."
read_when:
  - "When preparing benchmark artifacts for external AI review with strict token budgets"
  - "When you need positive examples in addition to wrong/missed snippet files"
---

For canonical-text benchmark runs, `wrong_label_lines.jsonl` already provides all mismatched line indices and snippets.

A compact positive-sample file can be derived by:

1. Reading canonical text and canonical span labels from `eval_report.json` (`canonical.canonical_text_path`, `canonical.canonical_span_labels_path`).
2. Building per-line gold labels from canonical span overlap.
3. Marking a line as inferred-correct when its `line_index` is absent from `wrong_label_lines.jsonl`.
4. Writing a capped sample JSONL (e.g., 80 rows) to avoid token blowup.

This produces specific "went right" snippets while staying tiny compared to raw prediction artifacts.
