# Unstructured V2 + EPUB Option Propagation

## Discovery

Unstructured `partition_html(..., html_parser_version="v2")` can fail on normal EPUB XHTML with:

`No <body class='Document'> or <div class='Page'> element found in the HTML.`

This is not an EPUB corruption issue; it is a parser-v2 input-shape expectation mismatch.

## What works

- Keep default parser as `v1` for broad compatibility.
- For parser `v2`, pre-wrap/annotate input so the partitioner sees `body.Document` when missing.
- Keep this shim in the adapter layer (`cookimport/parsing/unstructured_adapter.py`) so stage, benchmark prediction, and debug tooling all get the same behavior.

## Related contract discovered during implementation

EPUB unstructured options must be propagated in every run-producing path, not only stage:

- stage path (`cookimport stage ...`)
- benchmark prediction generation path (`cookimport labelstudio-benchmark ...` -> `generate_pred_run_artifacts(...)`)

The option triplet is:

- `epub_unstructured_html_parser_version`
- `epub_unstructured_skip_headers_footers`
- `epub_unstructured_preprocess_mode`

These values should appear in `runConfig`, influence runtime env vars (`C3IMP_EPUB_UNSTRUCTURED_*`), and be recorded in unstructured diagnostics metadata.

## Debug artifacts that made this obvious

Writing both raw and normalized spine XHTML per spine index (`raw_spine_xhtml_*.xhtml` and `norm_spine_xhtml_*.xhtml`) made parser failures and preprocessing effects immediately visible without re-opening container internals.
