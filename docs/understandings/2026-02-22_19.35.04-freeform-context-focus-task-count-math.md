# Discovery

Freeform task count in current code is controlled by `cookimport/labelstudio/freeform_tasks.py::_segment_ranges(total_blocks, segment_blocks, segment_overlap)`.

The critical invariant is `segment_overlap < segment_blocks`. Task step is `segment_blocks - segment_overlap`, so larger overlap means more tasks for the same source.

With defaults (`segment_blocks=40`, `segment_overlap=5`), step is `35`. To nudge from ~42 tasks to ~55 for the same book, overlap should increase so step lands near `26-27` (typically overlap `13-14` at `segment_blocks=40`).

Current prelabel prompt generation (`cookimport/labelstudio/prelabel.py::_build_prompt`) includes all segment blocks and block-mode prompt text asks for one label per block. There is no existing focus subset contract, so a robust context-vs-focus feature must add both prompt-level instruction and parser-side filtering.
