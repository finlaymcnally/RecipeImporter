# Discovery

- `scheduler heavy X/Y` in all-method status only counts configs in `split_active` phase.
- After prediction/post finishes, configs emit `evaluate_started` and can run for many minutes with no additional scheduler event until `evaluate_finished`.
- During this window, status can show `heavy 0/N | active >0 | pending 0`, which is expected and indicates eval-bound work, not a dead scheduler.

# Practical Check

- Confirm liveness by checking per-config scheduler event files under:
  - `.../all-method-benchmark/<source_slug>/.scheduler_events/config_*.jsonl`
- If the latest event is `evaluate_started` and worker CPU remains high, the run is still progressing.
