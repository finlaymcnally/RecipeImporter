# Label Studio Eval Artifact Root

`labelstudio_eval` run manifests should carry `artifacts.artifact_root_dir` for the prediction-run path.

Why:

- newer loaders already resolve the prediction run through `artifact_root_dir`
- leaving only `pred_run_dir` makes the eval manifest a one-off compatibility seam
- deleting `pred_run_dir` without adding `artifact_root_dir` would make downstream consumers fall back to the eval output dir, which is the wrong root when prediction artifacts live elsewhere
