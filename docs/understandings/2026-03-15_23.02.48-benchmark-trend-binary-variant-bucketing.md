The dashboard keeps rich benchmark semantics (`deterministic`, `line_role_only`, `recipe_only`, `full_stack`, `other`) for non-trend analysis, but the trend chart now uses a dedicated binary mapper. This avoids ad hoc historical rows creating extra trend lines when the user really wants a simple `LLM used` versus `LLM off` history view.
