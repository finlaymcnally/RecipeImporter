- `scripts/benchmark_cutdown_for_external_ai.py` previously built `process_manifest.included_files` from root-level files only.
- Because codex run outputs are nested (`codexfarm/full_prompt_log.jsonl`), strict validators could fail even when the full log file existed.
- The manifest assembly now adds each `RunRecord.full_prompt_log_path` entry into `included_files` so external reviewers can assert full prompt payload presence directly.
