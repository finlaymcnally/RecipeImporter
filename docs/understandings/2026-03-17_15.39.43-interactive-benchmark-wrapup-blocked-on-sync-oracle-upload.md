# Interactive Benchmark Wrap-Up Blocked On Synchronous Oracle Upload

- In both interactive benchmark flows, writing `upload_bundle_v1` was not the last step.
- After printing the bundle path, the code immediately called `_maybe_upload_benchmark_bundle_to_oracle(...)`.
- `_maybe_upload_benchmark_bundle_to_oracle(...)` waits on `run_oracle_benchmark_upload(...)`, so the interactive benchmark did not finish until Oracle finished, failed, or was interrupted.
- For single-offline this happened right after `External-AI upload bundle: ...`.
- For multi-book single-profile this happened right after `External-AI group upload bundle: ...`.
- The simplest safe fix is to stop at bundle generation and print the manual `cookimport bench oracle-upload <bundle>` command instead of trying to background Oracle from the interactive path.
