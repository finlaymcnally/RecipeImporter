summary: "Pytest is already compact by default; agents are mainly being nudged into verbose reruns by docs and failure hints."
---

- Quiet mode is enforced in `pytest.ini` and `tests/conftest.py`; even `-v/-vv` gets clamped back to compact unless `COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1` is set.
- Routine wrapper guidance points agents toward `./scripts/test-suite.sh` / `make test-fast`, so the normal path is already low-noise.
- Agents still see the verbose path because failure hints print a full verbose rerun command, and `tests/AGENTS.md` / `tests/README.md` document that escape hatch prominently.
- Practical conclusion: the compact-mode implementation did work, but the repo still teaches agents to escalate to verbose quickly once anything fails.
