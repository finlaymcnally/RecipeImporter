# Mixed line-role task helps recipe-local books but hurts prose-heavy books

The March 17 benchmark comparison showed that the active `codex-line-role-shard-v1` setup is not uniformly bad or uniformly good. It helps when the dominant errors are recipe-local structure recovery, and it hurts when the dominant errors are outside-recipe semantic judgments.

Observed shape:

- `dinnerfor2cutdown` improved because Codex mostly corrected recipe-local mistakes such as `OTHER -> INSTRUCTION_LINE`, `OTHER -> INGREDIENT_LINE`, and `KNOWLEDGE -> RECIPE_TITLE`.
- `saltfatacidheatcutdown` cratered because Codex over-promoted outside-recipe prose and headings into `RECIPE_TITLE`, `HOWTO_SECTION`, `INSTRUCTION_LINE`, and `KNOWLEDGE`.

Why this happens:

- the current line-role shard path still gives the model the full global label set on one ordered slice
- that means the model is simultaneously solving two different problems:
  - “is this part of an active recipe region?”
  - “if not, is this cookbook teaching prose, front matter, navigation, memoir, endorsement, or some other outside-recipe material?”
- the same model behavior that helps repair local recipe structure in recipe-dense books becomes a hallucination source in prose-heavy books because title-like headings and useful-sounding prose are allowed to compete directly with real recipe labels

Important product constraint from the user:

- deterministic guesses are not good enough to act as the judge over LLM work
- the fix is therefore not “LLM proposes, heuristics approve”
- the fix is to make the LLM do a narrower job:
  - first decide recipe-region membership or non-membership
  - only then allow recipe-structure labels for rows that belong to a recipe region
  - route outside-recipe rows into a separate semantic-authority pass with a smaller, more natural label space

Practical implication:

- keep Codex as the authority on hard cases
- stop asking one mixed label pass to invent recipe structure in outside-recipe territory
- preserve the current wins on `dinnerfor2cutdown` while removing the failure mode that dominates `saltfatacidheatcutdown`
