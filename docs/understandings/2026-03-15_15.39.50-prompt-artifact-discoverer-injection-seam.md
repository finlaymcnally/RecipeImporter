summary: "Prompt artifact rendering only needs normalized descriptors; discovery can be injected as a small adapter seam."
read_when:
  - "When wiring prompt artifacts to a new stage/cookbook topology"
  - "When deciding whether prompt exporter changes belong in discovery or rendering"
---

`render_prompt_artifacts_from_descriptors(...)` is already topology-neutral; the main remaining coupling was the builder defaulting directly to CodexFarm discovery.

A small seam is enough to unblock future topology tie-ins:

- add `discover_prompt_run_descriptors(...)` as a dispatcher over injected discoverers,
- add `build_prompt_response_log(...)` to accept either explicit descriptors or injected discoverers,
- keep `build_codex_farm_prompt_response_log(...)` as a compatibility wrapper for current CLI paths.

This keeps current behavior stable while letting a future stage-backed/cookbook-backed adapter plug in without editing renderer internals.
