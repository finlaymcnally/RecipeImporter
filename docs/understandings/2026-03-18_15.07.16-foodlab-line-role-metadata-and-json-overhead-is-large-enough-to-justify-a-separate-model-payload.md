# Metadata Overhead Is Big Enough To Matter

Using the same `o200k_base` tokenizer as prompt preview on `data/output/2026-03-18_15.15.00-foodlab-file-preview/line-role-pipeline/in/*.json`, the combined Food Lab line-role task bodies break down roughly as:

- about `233.5k` tokens of raw `current_line` text
- about `93.7k` additional tokens from the minimal JSON/index seam needed to carry that text as structured rows
- about `527.3k` additional tokens from `prev_text` and `next_text`
- about `370.0k` additional tokens from the rest of the row metadata and JSON framing

That last category includes fields such as `deterministic_label`, `rule_tags`, `escalation_reasons`, `region_status`, `block_id`, `recipe_id`, `within_recipe_span`, plus the repeated object keys and pretty-printed JSON structure around them.

The practical lesson is that line-role should stop using one rich row object as both the debug artifact and the model-facing transport. Keeping the rich local artifact is fine, but the Codex-facing payload should be a much smaller schema that carries only the fields the model truly needs to answer the task.
