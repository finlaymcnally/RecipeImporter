# Discovery

- The interactive run-settings chooser exposed multiple profile branches (`global`, `preferred`, `quality-first`, `quality-suite winner`, `last`, `edit`), so stale profile reuse could silently select weak settings.
- The old codex prompt only toggled `llm_recipe_pipeline`; it did not normalize `line_role_pipeline` or `atomic_block_splitter`, so codex runs could still execute with `off/off`.
- Single-profile quality regressions traced back to this selection chain, not to benchmark scoring logic.

# Current rule

- Interactive Import + Benchmark now resolve one automatic top-tier profile only:
  - use saved quality-suite winner settings when available,
  - otherwise use built-in top-tier baseline (`quality-first` EPUB stack + `llm_recipe_pipeline=codex-farm-3pass-v1` + `line_role_pipeline=codex-line-role-v1` + `atomic_block_splitter=atomic-v1`).
