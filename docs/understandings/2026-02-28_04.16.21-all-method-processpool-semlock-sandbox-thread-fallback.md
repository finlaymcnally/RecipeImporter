## Discovery

- `ProcessPoolExecutor` initialization failed at `_multiprocessing.SemLock` with `PermissionError: [Errno 13] Permission denied`.
- In this sandbox, `/dev/shm` is not writable, which blocks Python multiprocessing semaphore primitives.
- Workaround implemented in runtime:
  - keep all-method `global` scheduler scope,
  - use `ThreadPoolExecutor` for config workers when process workers are unavailable,
  - reserve serial single-config fallback only for thread executor setup failure.
- Quality runner no longer forces a global-to-legacy scheduler downgrade on process-worker probe failure.
