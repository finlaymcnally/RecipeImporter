# Joblib SemLock Warning: Startup Noise

- The repeated `joblib ... Permission denied ... will operate in serial mode` message is triggered during import-time SemLock probing in restricted `/dev/shm` environments.
- This warning did not indicate a newly introduced serial fallback path; it reflected host constraints already affecting multiprocessing.
- Fix strategy: run a lightweight SemLock probe early and, when restricted, set `JOBLIB_MULTIPROCESSING=0` before `joblib` import so warning spam stops.
- Explicit `JOBLIB_MULTIPROCESSING` env settings remain authoritative, and guard behavior can be disabled with `COOKIMPORT_DISABLE_JOBLIB_SEMLOCK_GUARD`.
