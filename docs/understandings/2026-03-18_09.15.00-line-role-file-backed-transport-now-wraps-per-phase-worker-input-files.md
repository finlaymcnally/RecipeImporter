# Line-role file-backed transport now wraps worker input files

`codex-line-role-shard-v1` now runs as one live file-backed `line_role` phase. Codex gets a short wrapper prompt that points at a worker-local JSON task file instead of a giant inline shard body.

The important artifact split is:

- `line-role-pipeline/runtime/line_role/workers/<worker>/in/<shard>.json` is the authoritative model input.
- `line-role-pipeline/prompts/line_role/*_prompt_*.txt` is the saved wrapper prompt for reviewer/export surfaces.
- `line-role-pipeline/runtime/line_role/workers/<worker>/shards/<shard>/prompt.txt` is the exact wrapper prompt used for that live shard call.

Preview and actual-cost reporting now need to describe both pieces honestly. The visible prompt is small, but the real task payload lives in the JSON file, so prompt artifacts and budget summaries should carry `prompt_input_mode=path` plus `request_input_file` metadata instead of pretending the full shard still lives inside `prompt.txt`.
