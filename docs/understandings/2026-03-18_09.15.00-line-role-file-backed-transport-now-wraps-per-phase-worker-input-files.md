# Line-role file-backed transport now wraps per-phase worker input files

`codex-line-role-shard-v1` now runs as two internal phases, `recipe_region_gate` and `recipe_structure_label`, and both phases hand Codex a short wrapper prompt that points at a worker-local JSON task file.

The important artifact split is:

- `line-role-pipeline/runtime/<phase>/workers/<worker>/in/<shard>.json` is the authoritative model input.
- `line-role-pipeline/prompts/<phase>/*_prompt_*.txt` is the saved wrapper prompt for reviewer/export surfaces.
- `line-role-pipeline/runtime/<phase>/workers/<worker>/shards/<shard>/prompt.txt` is the exact wrapper prompt used for that live shard call.

Preview and actual-cost reporting now need to describe both pieces honestly. The visible prompt is small, but the real task payload lives in the JSON file, so prompt artifacts and budget summaries should carry `prompt_input_mode=path` plus `request_input_file` metadata instead of pretending the full shard still lives inside `prompt.txt`.
