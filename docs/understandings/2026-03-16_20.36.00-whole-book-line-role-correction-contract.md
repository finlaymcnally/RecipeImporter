# Whole-Book Line-Role Correction Contract

What the March 16 seam showed:

- `docs/reports/refactorOG.md` describes Stage 2 as rule-first labeling plus Codex correction over the labeled book in local windows.
- runtime had drifted into unresolved-only Codex batching in `cookimport/parsing/canonical_line_roles.py`
- benchmark projection had a second drift: recipe-local labels were reused, while outside-recipe `KNOWLEDGE`/`OTHER` could collapse back to seed authority if the final non-recipe stage result was not carried through

Why token counts looked wrong:

- unresolved-only batching meant line-role Codex calls could be tiny even when the book was large
- that made prompt artifacts and token budgets look inconsistent with the intended product surface

Restored contract:

- `codex-line-role-v1` now reviews the full ordered atomic candidate set in bounded local batches
- each prompt row includes the deterministic first-pass label and any escalation reasons, so Codex acts as a correction pass rather than a blank-slate classifier
- benchmark canonical projection still reuses recipe-local Stage 2 labels, but outside-recipe `KNOWLEDGE` versus `OTHER` is overwritten from the final non-recipe authority before scoring
