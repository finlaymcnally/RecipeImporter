# Discovery

`labelstudio-benchmark` already has a hard runtime boundary between prediction generation and evaluation: evaluators consume artifact files (`stage_block_predictions.json` + `extracted_archive.json`) only after prediction stage finishes.

For speed-4, the practical decoupling unit is therefore one run-level record that stores those artifact paths and minimal run metadata. This supports evaluate-only replay immediately, while keeping scoring behavior unchanged.

# Evidence

- Prediction stage: `cookimport/labelstudio/ingest.py::generate_pred_run_artifacts`
- Eval orchestration: `cookimport/cli.py::labelstudio_benchmark`
- Evaluator inputs: `cookimport/bench/eval_stage_blocks.py`, `cookimport/bench/eval_canonical_text.py`
- New contract: `cookimport/bench/prediction_records.py`
