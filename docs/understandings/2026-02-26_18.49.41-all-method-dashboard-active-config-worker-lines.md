# Discovery

- The all-method dashboard already tracked active config indices/slugs per source, but for `>1` active configs it rendered only a compressed range (`current configs A-B/N`).
- Scheduler phase telemetry (`prep`, `split_wait`, `split_active`, `post`, `evaluate`) was available in `_run_all_method_benchmark(...)` via `.scheduler_events`, but that per-config phase state was not forwarded into dashboard rows.

# Practical Implication

- During parallel all-method runs, operators lost the per-slot “what is this worker doing right now?” view and only saw aggregate scheduler counters.
- The dashboard now keeps per-config phase state and renders an `active config workers:` section for multi-active states so each active config line shows `config NN: <phase> | <slug>`.
