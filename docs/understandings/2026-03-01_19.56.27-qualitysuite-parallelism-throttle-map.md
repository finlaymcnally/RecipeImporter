## Discovery

Three controls were limiting throughput:

- Auto experiment fanout used `min(total_experiments, cpu_count, auto_ceiling)` with fixed default `auto_ceiling=16`.
- WSL in auto mode forced subprocess experiment executor for parallel and single-slot runs (`wsl_parallel_experiment_isolation`, `wsl_single_experiment_isolation`).
- WSL safety guard rewrote per-experiment run settings/runtime knobs to conservative values (workers capped, `max_parallel_sources=1`, `max_inflight_pipelines=1`, `max_concurrent_split_phases=1`, `max_eval_tail_pipelines=0`, `smart_scheduler=false`).

## Implication

Even when users requested higher outer parallelism, inner all-method work was intentionally constrained on WSL, leading to noticeably slower QualitySuite runs.

## Resolution Direction

Restore CPU-scaled defaults and remove WSL-only throttling in runtime behavior while keeping explicit env overrides available for manual control.
