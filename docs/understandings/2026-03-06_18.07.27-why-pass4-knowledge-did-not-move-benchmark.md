---
summary: "Why pass4 knowledge harvest produced artifacts but almost no benchmark lift in run 2026-03-06_15.22.11."
---

Pass4 knowledge did run in `data/golden/benchmark-vs-golden/2026-03-06_15.22.11` and wrote large snippet sets:

- Salt: `437` snippet rows / `551` unique `provenance.block_indices`
- Sea: `49` / `54`
- FoodLab: `540` / `748`

But the canonical-text benchmark for this run preferred `prediction-run/line-role-pipeline/stage_block_predictions.json` because `--line-role-pipeline` was enabled (`docs/07-bench/07-bench_README.md`).

That projected artifact is built only from canonical line-role predictions in `cookimport/labelstudio/canonical_line_projection.py`:

- `build_line_role_stage_prediction_payload(...)` consumes `spans`
- it does not read pass4 `snippets.jsonl`
- notes in the artifact say `Projected from canonical line-role predictions.`

So pass4 snippet semantics are not on the scored path here. The scored `KNOWLEDGE` labels came mostly from canonical line-role logic / deterministic fallback (`cookimport/parsing/canonical_line_roles.py`), which already has its own `KNOWLEDGE` heuristics and sanitizers.

Evidence from this run:

- Salt line-role stage artifact:
  - codex `KNOWLEDGE` blocks: `82`
  - vanilla `KNOWLEDGE` blocks: `82`
- Sea:
  - codex `4`
  - vanilla `3`
- FoodLab:
  - codex `67`
  - vanilla `65`

And final eval deltas were correspondingly tiny:

- Salt `KNOWLEDGE`: identical (`pred_total=73`, `tp=63`, `f1=0.1886`) for codex and vanilla
- Sea: both `f1=0.0`
- FoodLab: codex `pred_total=64`, `tp=41`, `f1=0.0886`; vanilla `62`, `40`, `0.0867`

There is still a secondary narrowing even on the legacy stage path:

- `cookimport/staging/stage_block_predictions.py` loads only `provenance.block_indices` from snippets, not snippet `title/body/tags`
- `_resolve_block_label(...)` removes `KNOWLEDGE` when any recipe-local label is present

So pass4 is currently useful as an artifact surface, but in this benchmark configuration it is largely disconnected from the scored prediction surface.
