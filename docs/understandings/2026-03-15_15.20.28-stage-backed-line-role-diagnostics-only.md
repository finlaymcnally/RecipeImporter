# Stage-backed line-role diagnostics only

The clean seam was not "teach canonical line-role to become the new benchmark prediction source." The clean seam was "run the real stage/import session first, then let benchmark-only diagnostics hang off that session."

Why this mattered in code:

- `cookimport/labelstudio/ingest.py::generate_pred_run_artifacts()` previously built line-role candidates before the benchmark-only processed-output write and could project those labels back into `ConversionResult`.
- Once `cookimport/staging/import_session.py` owned the authoritative stage-backed write, letting line-role keep mutating recipes would have recreated the same semantic fork under a different helper.
- Rebuilding line-role candidates after the shared stage session also avoids the stale-pre-LLM recipe-span problem, because the candidate builder now sees the post-stage-session `ConversionResult`.

Current rule:

- authoritative benchmark score inputs come from the stage run,
- `prediction-run/line-role-pipeline/*` remains for diagnostics,
- `line_role_pipeline_recipe_projection` is now a diagnostic summary that explicitly records that authoritative stage outputs were not mutated.
