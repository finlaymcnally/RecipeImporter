summary: "Per-label point-value columns could show 0.0000 when the comparison variant is missing because empty values were coerced to numeric zero."
read_when:
  - "When Per-Label Breakdown shows unexpected 0.0000 values for a missing variant."
  - "When debugging codexfarm-vs-vanilla comparison rendering in dashboard JS."
---

# Discovery

- Run-group `2026-03-03_23.27.32` had `4` evals, all classified as `codexfarm` (`single-profile-benchmark/...`, `llm_recipe_pipeline=codex-farm-3pass-v1`).
- `vanilla` per-label rows were absent for that run-group.
- In `perLabelComparisonCell`, `maybeNumber` converted `null`/empty values to `0` (`Number("") === 0`), so missing comparison values rendered as `0.0000` in point-value mode.

# Resolution

- Per-label comparison rendering now treats `null`/empty-string candidates as missing before numeric parsing and shows `-` instead of `0.0000`.
