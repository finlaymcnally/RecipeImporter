# All-method dashboard progress hook

Interactive all-method runs currently call `labelstudio_benchmark(...)` for each config, but that command normally owns its own spinner and prints a full completion block (`Benchmark complete`, report paths, recipe deltas) every run.

To keep one persistent dashboard during all-method sweeps, `cookimport/cli.py` uses scoped context overrides (`_benchmark_progress_overrides`) while each config runs:

- `_BENCHMARK_PROGRESS_CALLBACK`: forwards benchmark import task updates into the outer all-method dashboard spinner.
- `_BENCHMARK_SUPPRESS_SPINNER`: skips nested benchmark spinner rendering.
- `_BENCHMARK_SUPPRESS_SUMMARY`: suppresses per-config completion dumps.

The all-method dashboard state lives in `_AllMethodProgressDashboard`, which tracks source queue state (`pending/running/done/failed`), config completion counts, and the current task line. Worker telemetry payloads (`__worker_activity__ ...`) are passed through unchanged so worker lines still render under the single spinner.
