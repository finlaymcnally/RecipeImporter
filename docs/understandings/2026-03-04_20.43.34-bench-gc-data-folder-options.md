# Bench GC data-folder options

- `cookimport bench gc` is the explicit artifact-retention command for benchmark data roots.
- Dry-run is default (`--dry-run`); deletion requires `--apply`.
- Keep policy knobs are `--keep-full-runs`, `--keep-full-days`, and optional speed override `--drop-speed-artifacts`.
- Label Studio benchmark roots are only included when `--include-labelstudio-benchmark` is set; matching `data/output/<run_id>/` pruning is opt-in via `--prune-benchmark-processed-outputs`.
- Prune candidates are deleted only if durable benchmark history is already confirmed in `performance_history.csv` (`run_category` benchmark row + durable metrics present).
- GC does not mutate benchmark history CSV rows (no rewrite/prune/backup path).
- Pin sentinels (`.gc_keep*`, `.keep`, `.pinned`) prevent prune even when policy says drop; `cookimport bench pin` and `cookimport bench unpin` manage `.gc_keep*`.
- Separate from `bench gc`, `labelstudio-benchmark` auto-prunes only transient excluded benchmark artifacts after CSV append; interactive sessions suppress this prune path.
