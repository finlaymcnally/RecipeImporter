summary: "Interactive benchmark variant planning now runs only the selected profile, so Codex-backed choices produce one codexfarm run instead of a vanilla+codex pair."
read_when:
  - "When changing interactive single-offline or single-profile benchmark variant selection."
  - "When investigating why interactive codex benchmarks also scheduled vanilla."
---

The shared interactive benchmark variant planner used to expand any `llm_recipe_pipeline=codex-farm-3pass-v1` selection into two runs: `vanilla` first, then `codexfarm`.

That was useful for automatic comparison artifacts, but it made the interactive CLI violate the simpler expectation that the selected profile is the one that runs. It also explained why the terminal could print `Codex decision: no Codex-backed surfaces enabled.` during an interactive codex benchmark: that line belonged to the injected vanilla baseline run.

The current planner keeps interactive behavior literal. If the selected settings are Codex-backed, the interactive runner schedules one `codexfarm` variant and passes `allow_codex=True` through to that run.
