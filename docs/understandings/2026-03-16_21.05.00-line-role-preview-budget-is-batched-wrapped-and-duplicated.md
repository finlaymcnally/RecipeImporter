# Line-Role Preview Budget Is Batched, Wrapped, And Duplicated

- `codex-line-role-v1` is whole-book in authority, but not one whole-book prompt. It batches ordered atomic lines in windows of 40 (`cookimport/parsing/canonical_line_roles.py`).
- For the `saltfatacidheatcutdown` hypothetical preview on 2026-03-16, that became 45 prompts covering 1,858 target rows.
- Each target row is serialized as JSON with `atomic_index`, `within_recipe_span`, `deterministic_label`, `escalation_reasons`, and line text context, so prompt text is much larger than the original book text.
- Preview then wraps each inner line-role prompt inside the Codex-farm transport prompt (`llm_pipelines/prompts/line-role.canonical.v1.prompt.md`) under `{{INPUT_TEXT}}`, which re-embeds the whole prompt as JSON and adds escaping overhead.
- In that preview:
  - inner line-role prompts totaled 852,478 chars
  - wrapped prompts totaled 1,822,601 chars
  - transport wrapping added 970,123 chars, about 2.14x the inner prompt size overall
- `prompt_preview_budget_summary` is heuristic only: it divides rendered chars by 3.2 chars/token and adds a stage output ratio (`line_role`: 20%), so the reported `~683k` is not live tokenizer telemetry.
- The specific processed run that preview was built from still used `line_role_pipeline=deterministic-v1`, so those preview tokens were not actually spent by that run.
