# Test Wrapper Enforcement And Raw Pytest Warning

- Routine agent test loops in this repo are expected to use `./scripts/test-suite.sh` or the equivalent `make test-*` targets.
- `scripts/test-suite.sh` now exports `COOKIMPORT_TEST_SUITE=1` so pytest can tell wrapper-driven runs from ad hoc raw invocations.
- `tests/conftest.py` emits a one-line note for broad raw pytest runs, specifically when the invocation looks like a directory run or a multi-file cross-domain batch without a marker filter.
- Narrow single-file raw pytest runs still stay quiet so diagnostic reruns remain practical.
