## Why these homes

- Compare & Control and dashboard display tasks are analytics-first because the behavior lives in dashboard artifacts and renderer contracts even when the code path is a big JS string template.
- QualitySuite integrity, upload-bundle, `cf-debug`, and interactive benchmark operator flow are bench-first because the user-facing contract is benchmark execution and evidence review, not generic CLI mechanics.
- Safe defaults, token telemetry for Codex-backed line-role, reasoning trace capture, and the explicit Codex decision boundary are LLM-first because they define when Codex is active, how its artifacts are recorded, and how it is audited.
- The spinner readability task touched dashboard-like terminal output, but the operator-facing contract is the interactive benchmark runner, so it lands in `docs/07-bench/`.

## Anti-loop note

Two-surface merge is mandatory for these files:

- README: keep the current contract and “how it works / where it is / known bad” content.
- `_log.md`: keep the implementation chronology, failed serious attempts, and reasons the current design exists.

Deleting the source doc before both surfaces are updated is the easiest way to recreate future loops.
