# ProFixes Local Context Seams

- The repo does not have one small orchestrator/config stack. The main runtime seams are:
  - `cookimport/config/run_settings.py` for the typed settings model and stable hash helpers,
  - `cookimport/config/codex_decision.py` for profile/variant facts and explicit-activation decisions,
  - `cookimport/config/run_settings_adapters.py` for mapping `RunSettings` into stage/benchmark kwargs,
  - `cookimport/labelstudio/ingest.py` for prediction-run construction and benchmark primitive wiring,
  - `cookimport/cli.py` for the public command surface plus benchmark/profile helpers.

- The original execplan’s “new resolver / new presets / one orchestrator” idea should be translated into the existing `RunSettings` world instead of creating a second config system. This repo already has stable-hash, manifest, and baseline-vs-codex contract helpers; the real problem is that they are split across multiple call sites.

- Codex execution already has partial safety rails:
  - low-level kill switch in `cookimport/llm/codex_exec.py` via `COOKIMPORT_ALLOW_LLM`,
  - explicit benchmark confirmation tokens for `bench speed-run` and `bench quality-run`,
  - `resolve_codex_command_decision(...)` in `cookimport/config/codex_decision.py`.
  The missing piece is a single repo-wide execution-policy layer shared by stage, Label Studio prediction runs, interactive profiles, and benchmark helpers.

- Line-role guardrails are not greenfield work. `cookimport/parsing/canonical_line_roles.py` already has:
  - deterministic-first labeling,
  - selective Codex escalation,
  - outside-span sanitization,
  - source-level `do_no_harm` arbitration with diagnostics artifacts.
  The plan should extend and parameterize this behavior instead of inventing a new unrelated guardrail subsystem first.

- The most direct deterministic-fix files for the plan’s quality goals are:
  - `cookimport/parsing/recipe_block_atomizer.py`
  - `cookimport/parsing/canonical_line_roles.py`
  - `cookimport/llm/evidence_normalizer.py`
  - `cookimport/llm/canonical_line_role_prompt.py`
  - `cookimport/llm/codex_farm_orchestrator.py`

- For user-visible proof, the real command surface is `cookimport stage`, `cookimport labelstudio-benchmark`, `cookimport bench speed-run`, and `cookimport bench quality-run`, not a generic `python -m <package>.cli run --preset ...` flow.
