# Auto Probe Importer Init Rule

`select_epub_extractor_auto(...)` intentionally probes extraction by calling `EpubImporter._extract_docpack(...)` directly on sampled spine ranges.

That means probe paths do not pass through `EpubImporter.convert(...)`, so any runtime fields that `_extract_docpack(...)` depends on must be initialized in `EpubImporter.__init__` (or guarded with safe defaults), not only inside `convert(...)`.

Concrete bug seen during Milestone 0 baseline capture on `saltfatacidheatCUTDOWN.epub`:

- `stage --epub-extractor auto` failed with:
  - `'EpubImporter' object has no attribute '_overrides'`
- Root cause:
  - `_extract_docpack(...)` called `signals.enrich_block(..., overrides=self._overrides)`,
  - but `_overrides` was only set in `convert(...)`.
- Fix:
  - initialize `self._overrides = None` in `EpubImporter.__init__`.
- Regression coverage:
  - `tests/test_epub_auto_select.py::test_select_epub_extractor_auto_real_importer_supports_direct_probe`
