- `_build_all_method_variants(...)` now has two distinct payload paths in `cookimport/cli.py`: baseline-style variants are passed through `_all_method_apply_baseline_contract(...)` before `RunSettings.from_dict(...)`, while Codex variants are created with `_all_method_apply_codex_contract_from_baseline(...)`.
- The baseline contract is `llm_recipe_pipeline=off`, `llm_knowledge_pipeline=off`, `llm_tags_pipeline=off`, `line_role_pipeline=off`, and `atomic_block_splitter=off`. This matches the existing single-offline `vanilla` benchmark semantics.
- The easy regression is over-normalizing Codex payloads. If a Codex payload is sent back through the baseline helper before hashing, it collapses to the same stable hash as the baseline and all `__llm_recipe_codex_farm_3pass_v1` variants disappear during dedupe.
- A retained proof run under `/tmp/qualitysuite-baseline-validation/2026-03-05_22.36.13/experiments_resolved.json` showed the fixed baseline experiment with `llm_recipe_pipeline=off`, `llm_knowledge_pipeline=off`, `llm_tags_pipeline=off`, `line_role_pipeline=off`, and `atomic_block_splitter=off`.
