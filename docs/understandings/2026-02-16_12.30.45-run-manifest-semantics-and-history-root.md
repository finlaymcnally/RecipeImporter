# Run Manifest + History Root Notes

- `run_manifest.json` is now the cross-flow traceability file for run roots (`stage`, `labelstudio_import/export/eval/benchmark`, `bench_pred_run`, `bench_eval`, `bench_suite`).
- The manifest captures source identity (`path`, `source_hash`), effective run config, and artifact pointers so stage vs benchmark relationships are explicit without reading module internals.
- `labelstudio-benchmark --no-upload` is the non-interactive offline mode: it builds prediction artifacts locally and evaluates without resolving Label Studio credentials or uploading tasks.
- Stage and benchmark CSV writes now follow the actual output root used by that run:
  - stage appends to `<stage --out>/.history/performance_history.csv`
  - benchmark appends to `<processed_output_dir>/.history/performance_history.csv`
- `perf_report.resolve_run_dir()` now resolves both timestamp folder formats (`YYYY-MM-DD_HH.MM.SS` and legacy `YYYY-MM-DD-HH-MM-SS`), so auto-latest works across old and new run folders.
