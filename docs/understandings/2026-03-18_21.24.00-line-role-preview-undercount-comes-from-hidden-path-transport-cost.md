# Line-Role Preview Versus Actual Cost

For the March 18 `dinnerfor2cutdown` benchmark, actual line-role cost was much larger than preview because the live path-backed Codex call billed far more than the visible wrapper prompt.

- preview code sets `wrapper_overhead_tokens = 0` for structural prompt estimates and treats estimated input as visible prompt text
- live telemetry counted only the short wrapper prompt as `visible_input_tokens`
- the task-file transport showed up mostly as `wrapper_overhead_tokens` and `tokens_cached_input`

Concrete signal from the benchmark:

- 5 line-role calls had only `14,060` visible wrapper chars total and `119,615` task-file bytes total
- actual line-role telemetry still reported `1,665,694` total tokens, including `937,887` wrapper-overhead tokens and `714,752` cached-input tokens

Inference: local prompt preview can tokenize the wrapper prompt and the compact task file, but it cannot currently reconstruct the hidden Codex path-input accounting surface that the real run bills.
