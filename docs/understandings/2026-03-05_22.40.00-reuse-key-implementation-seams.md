Reuse-key implementation seam summary:

- Prediction artifact reuse only needed a narrow projection of `RunSettings`, not the full `stable_hash()` or full `run_config`.
- The safest shared source of truth is a small `cookimport/config/prediction_identity.py` module that projects `settings.to_run_config_dict()`.
- All-method split-convert grouping was also over-broad because it still included worker and job-splitting knobs; those had to be removed so the summary matches real safe-reuse candidates.
- Line-role cache identity can be reduced to `line_role_pipeline` because candidate text/context already lives in `_canonical_candidate_fingerprint(...)`, while `codex_timeout_seconds` and `codex_batch_size` stay in the cache key separately.
