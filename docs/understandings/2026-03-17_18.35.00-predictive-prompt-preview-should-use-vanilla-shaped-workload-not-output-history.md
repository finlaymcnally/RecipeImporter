---
summary: "Predictive prompt preview should estimate from the selected book's deterministic or vanilla-shaped workload, not from transient local `data/output` history."
read_when:
  - When changing `cf-debug preview-prompts` predictive estimation behavior
  - When deciding which manifest a benchmark-root preview should resolve for predictive preview
  - When someone asks why prompt preview ignores old `data/output` runs for future token prediction
---

Predictive prompt preview is a pre-run planning tool, so it should not depend on transient local history under `data/output`; that folder is operational scratch space and may be cleared regularly. The stable predictive inputs are:

- the selected book's deterministic or `vanilla` processed artifact, which tells preview how many recipes, line-role rows, and knowledge spans the book would generate before any Codex calls happen
- repo-owned per-stage token models calibrated from a known vanilla-to-Codex benchmark

That leads to the selection rule:

- prompt preview prefers the paired `vanilla` benchmark artifact when a benchmark root contains both `vanilla` and `codexfarm`
- prompt preview must fail instead of silently falling back to a Codex-backed or ambiguous processed run
- retrospective “what actually happened” reporting belongs to the separate post-run `actual-costs` surface
