Discovery:
- `benchmark-csv-backfill` can only fill token columns when manifest telemetry exists (typically `llm_codex_farm.process_runs.*.process_payload.telemetry*`).
- For `profeedback_foodlab_full_codex` (run timestamp `2026-03-04T12:15:33`), `prediction-run/manifest.json` has:
  - `llm_codex_farm.enabled=true`
  - `fallbackApplied=true`
  - `fatalError=...subprocess_exit=124...`
  - no `process_runs` payload.
- The run's codex prompt log states: `SKIP: missing pass manifests ...`, confirming pass manifests/telemetry were not produced.
- Canonical rows at `2026-03-04T12:35:37` (`..._full_codex_canonical`, `..._deterministic_canonical`) also contain no token telemetry payloads to backfill.
- Result: these rows are correctly rendered as token-unknown (`-`) and cannot be retroactively reconstructed from available artifacts.

Operational implication:
- To populate token usage for such rows, rerun the prediction path with successful codex pass completion and telemetry capture (no fallback timeout), then re-append/backfill history.
