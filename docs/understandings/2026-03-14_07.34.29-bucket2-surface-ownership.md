# Bucket 2 Surface Ownership

- `cookimport/config/run_settings.py` decides the public versus internal classification that drives generated UI specs and public summary projection.
- `cookimport/cli.py` still has handwritten `stage` / `labelstudio-benchmark` Typer options plus the old interactive settings editor, so hiding a field in `RunSettings` does not remove it from help or menus by itself.
- Analytics and dashboard surfaces had separate fallback summary builders (`dashboard_collect.py`, `perf_report.py`, and `dashboard_render.py`), so Bucket 2 keys could keep showing up in summaries even after the metadata split unless those helpers were routed back to `summarize_run_config_payload(...)`.
