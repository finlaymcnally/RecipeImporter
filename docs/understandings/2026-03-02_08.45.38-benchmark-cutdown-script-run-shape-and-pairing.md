Key discovery used by the automation:

- Reliable benchmark run detection is "folder contains both `eval_report.json` and `run_manifest.json`" (ignore `prediction-run/run_manifest.json` and cache dirs).
- `run_manifest.json` stores the meaningful settings under `run_config` and source identity under `source.path` + `source.source_hash`.
- Codex-vs-baseline grouping should use `source_hash` first (fallback: source filename), and baseline status comes from `run_config.llm_recipe_pipeline` in `{off, none, ""}`.
- `eval_report.json` already contains compact "why" signals (`worst_label_recall`, `confusion`, scalar `per_label` stats) so we should avoid copying index-heavy arrays.
- `correct_label_lines.sample.jsonl` can be derived deterministically from canonical lines + canonical span labels by taking line indices absent from `wrong_label_lines.jsonl`.
