# CodexFarm Telemetry Report V2 Alignment

## Discovery

CodexFarm now publishes the caller automation contract directly in `process --json.telemetry_report` (`schema_version=2`, including `insights` and `tuning_playbook`). Recipeimport was already preserving raw `process_payload`, but did not expose this report as a first-class top-level `process_run` field.

## Alignment decision

- Add explicit `telemetry_report` on `CodexFarmPipelineRunResult` and persist it in all recipe/pass4/pass5 `process_run(s)` payloads.
- Keep existing `telemetry` CSV slices from `codex_exec_activity.csv` unchanged as compact per-row context and backward-compatible fallback when embedded reports are missing.

## Why both surfaces remain

- `telemetry_report` is the source-of-truth caller contract for automation loops.
- `telemetry` row slices remain useful for debugging exact row-level details and for compatibility with older/no-report CodexFarm invocations.
