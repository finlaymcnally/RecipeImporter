# Discovery

- Throughput bottleneck: freeform prelabeling used one strict serial loop in `generate_pred_run_artifacts(...)` (`for task in tasks`), so each Codex call blocked the next even when tasks were independent.
- Safe parallelization boundary: each task prompt is already self-contained (no cross-task memory), so task-level provider calls can be run concurrently without changing label semantics.
- Important runtime constraints when parallel:
  - Keep progress callback contract on completions (`Running freeform prelabeling... task X/Y`).
  - Keep prompt-log/report determinism by persisting task metadata with stable task indexes.
  - Keep provider usage totals thread-safe when one provider instance is shared by worker threads.
