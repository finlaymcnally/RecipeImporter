# Fake Exec Must Read Task Files

The restored two-phase line-role runtime was correct before the dry run passed. The real blocker was the zero-token shim: `scripts/fake-codex-farm.py exec` was extracting the first JSON object from the wrapper prompt text, which was the illustrative return-shape example containing `atomic_index:123`, instead of opening the deposited `in/*.json` task file named by `{{INPUT_PATH}}`.

For path-backed direct-exec phases, the fake shim has to resolve the referenced task-file path and load that JSON as the structural payload. Once that was fixed, the same dry run stopped failing proposal validation and produced the intended two-phase file-backed artifact layout under `line-role-pipeline/runtime/{recipe_region_gate,recipe_structure_label}/workers/*/in/*.json`.
