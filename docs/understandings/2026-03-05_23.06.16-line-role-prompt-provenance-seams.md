## Key Findings

- The recipe `prompts/full_prompt_log.jsonl` files are not the source of line-role prompt provenance. They only cover the recipe pass stack (`pass1`/`pass2`/`pass3`).

- Canonical line-role prompt provenance lives under `prediction-run/line-role-pipeline/prompts/` inside a codex-enabled benchmark run. Each batch is stored as:
  - `prompt_XXXX.txt`
  - `response_XXXX.txt`
  - `parsed_XXXX.json`

- `parsed_XXXX.json` is the cleanest join surface for audits. It is a JSON array of `{atomic_index, label}` rows, so a follow-up exporter can map one canonical line directly to the exact prompt batch files that produced its parsed label.

- `line-role-pipeline/line_role_predictions.jsonl` is the postprocess/final surface. It carries `atomic_index`, `label`, `candidate_labels`, `confidence`, and `decided_by`. Comparing that file to `parsed_XXXX.json` is enough to detect postprocess rewrites or missing prompt artifacts.

- `prediction-run/line-role-pipeline/extracted_archive.json` is the simplest context surface for page/span-style exports in current benchmark runs. It is a list of `{index, text, location}` rows keyed by canonical line index / atomic index, so a deterministic exporter can reconstruct nearby context windows without parsing markdown summaries.
