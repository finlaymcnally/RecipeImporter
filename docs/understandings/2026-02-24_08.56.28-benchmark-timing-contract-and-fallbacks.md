# Benchmark timing contract and fallback behavior

- `generate_pred_run_artifacts(...)` now emits `timing` in prediction `manifest.json`, prediction `run_manifest.json` artifacts, and the returned metadata payload.
- `labelstudio_benchmark(...)` now merges prediction timing with eval/load/write/CSV append timing and writes final `timing` to both `eval_report.json` and benchmark `run_manifest.json` artifacts.
- `append_benchmark_csv(...)` resolves timing in strict order: explicit `timing` argument, then `processed_report_path` JSON `timing`, then blank runtime fields.
- To make fallback reliable, ingest now does a best-effort write of timing data into the benchmark-generated processed `*.excel_import_report.json`.
- Benchmark `timing.total_seconds` is floored against known subphase totals so totals do not under-report in fast/mocked execution paths.
- All-method benchmark reports now include per-config `timing` plus per-source/run `timing_summary`, which is enough for coarse runtime analysis without introducing a new CLI command yet.
