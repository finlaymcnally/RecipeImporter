- Keeping only the newest timestamped run folder per benchmark surface (`benchmark-vs-golden`, `bench/quality/runs`, `bench/speed/runs`, `bench/speed/matcher_probe`) removes most stale benchmark history quickly.
- `stats-dashboard` still scans `eval_report.json` from disk, so deleting old artifact folders is required; CSV-only cleanup is not sufficient.
- After artifact deletion, re-prune `performance_history.csv` benchmark rows to existing `data/golden` paths so history rows do not reference deleted artifacts.
