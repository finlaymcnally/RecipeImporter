# QualitySuite fast-loop knobs

- `bench quality-run` already supports modular scaling:
  - run-level resume (`--resume-run-dir`),
  - race pruning (`--search-strategy race` + race knobs),
  - experiment-level concurrency (`--max-parallel-experiments` or auto mode),
  - shared cross-run caches (alignment + prediction reuse roots).
- Default race finalists (`--race-finalists 64`) can be too high for small variant sets; when `variants_effective <= race_finalists`, race falls back to exhaustive.
- Fast deterministic loop pattern:
  - smaller discovered suite (`quality-discover --max-targets ...`),
  - aggressive race pruning (`probe/mid/keep_ratio/finalists`),
  - disable extras (`--no-qualitysuite-agent-bridge`, no deterministic sweeps),
  - keep cache roots stable between runs.
- Throughput caution: each experiment has internal all-method parallelism. Increasing outer experiment parallelism too far can oversubscribe CPU/IO; tune outer and inner knobs together.
