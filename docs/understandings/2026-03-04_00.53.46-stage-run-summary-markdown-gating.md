# Discovery

`stage --no-write-markdown` is intended to suppress markdown sidecars. The run summary helper can still violate this if `run_summary.md` is emitted outside the markdown gating path.

# Resolution

- Keep `run_summary.json` as always-on for machine-readable run metadata.
- Gate `run_summary.md` on the stage `write_markdown` flag.
- Point terminal `Run summary file` output to `run_summary.json` when markdown writing is disabled.
