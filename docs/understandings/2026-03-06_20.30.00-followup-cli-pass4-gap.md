# Follow-up CLI pass4 gap

- `cf-debug` and `cookimport/bench/followup_bundle.py` do not currently read `analysis.pass4_knowledge` or `navigation.row_locators.pass4_by_run` from `upload_bundle_index.json`.
- Existing follow-up selectors are stage-filtered only for `line_role`, and the main audits (`line_role_audit`, `prompt_link_audit`, `page_context`, `uncertainty`) are all atomic-line based.
- `write_case_export(...)` can attach recipe raw-artifact files, but `RunContext.recipe_artifact_paths(...)` only resolves pass1/pass2/pass3 plus normalization/transport files; there is no pass4 artifact export path yet.
- Result: no urgent follow-up CLI update is required just because `upload_bundle_v1` now surfaces pass4 better, but a pass4-specific follow-up workflow would need new selector/output shapes rather than trying to force pass4 into the existing line-role prompt-link audit model.
