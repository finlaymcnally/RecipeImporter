# SpeedSuite Serial Loop + Resume Contract Snapshot

- `run_speed_suite(...)` in `cookimport/bench/speed_runner.py` iterates `scenario_work_items` and phases with nested `for` loops and runs each sample synchronously; no task-level executor is used.
- CLI `bench speed-run` has no `--resume-run-dir` option and no `--max-parallel-*` control for suite task fanout.
- Speed run artifacts (`samples.jsonl`, `summary.json`, `report.md`) are written after the full loop completes, so interruption can leave per-sample folders but no official resumable checkpoint state.
- In contrast, QualitySuite (`bench quality-run`) does expose `--resume-run-dir` and documented checkpoint artifacts.
