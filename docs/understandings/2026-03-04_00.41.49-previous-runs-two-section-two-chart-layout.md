# Discovery

- Previous Runs previously rendered as one top grid with Compare & Control beside a single `benchmark-trend-chart` host, then quick filters + table below.
- Chart rendering was hard-coded to one host ID (`benchmark-trend-chart`), so adding another chart required a host-aware render helper rather than duplicating whole chart logic.

# Resolution

- Layout now uses two subsection cards:
  - `previous-runs-history-panel` (primary trend + quick filters + table),
  - `compare-control-analysis-section` (Compare & Control panel + second trend chart host).
- Trend rendering now goes through `renderBenchmarkTrendChartHost(config)` and is invoked twice from `renderBenchmarkTrendChart()` for:
  - `benchmark-trend-chart`,
  - `compare-control-trend-chart`.
- Both charts share the same filtered benchmark row pool and trend-series construction contract, so they stay in sync with table filters/quick filters.
