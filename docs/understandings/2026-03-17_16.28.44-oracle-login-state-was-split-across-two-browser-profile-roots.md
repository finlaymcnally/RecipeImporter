# Oracle Login State Was Split Across Two Browser Profile Roots

- Historical Oracle wrapper/xvfb runs in this environment launched Chromium with `--user-data-dir=/home/mcnal/.oracle/browser-profile`.
- Newer direct benchmark upload launches used `/home/mcnal/.local/share/oracle/browser-profile`.
- Both trees currently exist and both contain Chromium profile artifacts, including `Default/Cookies`, so repeated manual sign-ins could land in one tree while a later benchmark run reused the other.
- The practical repo-side fix is to auto-select the most recently active populated profile between those two roots and print the chosen profile path in benchmark wrap-up.
- This explains the apparent cookie churn better than "login was never saved": the login state was being saved, but not always under the same profile root.
