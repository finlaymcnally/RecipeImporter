- `cookimport/parsing/canonical_line_roles.py` now routes `RECIPE_TITLE` through contextual validation (`_looks_recipe_title_with_context`) so title-like rows need supportive next-line structural evidence when available.
- `YIELD_LINE` now passes strict header validation (`_looks_strict_yield_header`) and is sanitized to structural fallback when prefix-only prose is not a true yield header.
- New yield fallback behavior can produce `INSTRUCTION_LINE`, `RECIPE_NOTES`, or `OTHER` depending on local evidence (`_yield_fallback_label`), while obvious ingredient lines are still rescued to `INGREDIENT_LINE`.
- Regression tests cover both new seams in `tests/parsing/test_canonical_line_roles.py`:
  - `test_label_atomic_lines_non_header_yield_phrase_demotes_to_instruction`
  - `test_label_atomic_lines_title_like_line_without_supportive_next_line_is_not_title`
