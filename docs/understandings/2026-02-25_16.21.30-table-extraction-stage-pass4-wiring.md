# Table extraction wiring map (stage + pass4)

- Non-split path: `cookimport/cli_worker.py:stage_one_file` is the only place where non-split chunking and staged file writes happen together, so table detection/annotation must run there before `chunks_from_non_recipe_blocks(...)` and before pass4 knowledge harvest.
- Split path: split workers only write temporary raw artifacts; merged chunking/writes happen in `cookimport/cli.py:_merge_split_jobs`, so table detection/annotation must run there on merged `non_recipe_blocks`.
- Pass4 input builder (`cookimport/llm/codex_farm_knowledge_jobs.py`) builds chunk block payloads from `full_blocks` but can attach table hints by absolute index from `non_recipe_blocks`; if hints are missing, verify the non-recipe block indices still match merged `full_text` indices.
- Processed-output snapshots used by benchmark/Label Studio (`cookimport/labelstudio/ingest.py:_write_processed_outputs`) also run chunking/writers; table-enabled runs must apply the same annotation + table writer steps there for parity.
