# EPUB Debug CLI Integration Contract

- `cookimport epub blocks` and `cookimport epub candidates` must reuse `EpubImporter` extraction/segmentation internals:
  - `cookimport/plugins/epub.py:_extract_docpack(...)`
  - `cookimport/plugins/epub.py:_detect_candidates(...)`
  - `cookimport/plugins/epub.py:_extract_title(...)` (for candidate title guesses)
- Calling `_extract_docpack(...)` directly requires initializing importer state used by signal enrichment (`importer._overrides = None`), because the normal `convert(...)` path sets it.
- Keep Unstructured option parity by setting the same env vars stage uses before extraction:
  - `C3IMP_EPUB_UNSTRUCTURED_HTML_PARSER_VERSION`
  - `C3IMP_EPUB_UNSTRUCTURED_SKIP_HEADERS_FOOTERS`
  - `C3IMP_EPUB_UNSTRUCTURED_PREPROCESS_MODE`
- Debug command output directories should be explicit and safe:
  - refuse non-empty `--out` unless `--force`
  - never modify source EPUB files
- EPUB structural inspection uses zip/OPF parsing as the reliable baseline; optional `epub_utils` import is best-effort only.
- `epub-utils` is currently consumed as a pre-release optional dependency (`epub-utils==0.1.0a1`, install via `.[epubdebug]` or `pip install --pre epub-utils`).
