After confirming both a headful failure and successful xvfb-backed Oracle smoke tests, the stable cross-machine rule here is not one fixed launcher path. It is one auto launcher.

Evidence:

- direct headful Oracle smoke test from the agent shell failed before ChatGPT interaction because Chromium had no X display
- direct xvfb-backed Oracle smoke tests completed successfully and returned real `gpt-5.2-instant` answers
- a fresh generic `oracle ...` smoke test succeeded after routing through `/home/mcnal/.local/bin/chromium-oracle-auto`, and the auto launcher log recorded:
  - `MODE=xvfb`
  - `LAUNCHER=/home/mcnal/.local/bin/chromium-nosandbox-xvfb`

Conclusion: the correct machine-wide default is:

- generic `oracle` browser runs go through `/home/mcnal/.local/bin/oracle-browser-auto`
- that wrapper sets `--browser-chrome-path /home/mcnal/.local/bin/chromium-oracle-auto`
- `chromium-oracle-auto` chooses visible Chromium only when a usable display exists, otherwise it falls back to xvfb

That keeps one canonical Oracle profile and one browser model strategy everywhere, without forcing every environment to have the same display capabilities.
