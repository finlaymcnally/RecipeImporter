read_when:
  - "When stage run progress unexpectedly drops from live panel to plain status lines."
  - "When changing stage live status rendering or shared live-slot controls."
---

# Discovery

Stage progress used a direct `console.status(...)` live panel path that did not participate in shared live-slot gating, so it could still hit Rich live-display conflicts while benchmark/import callback paths were already slot-capped.

# Resolution

- Stage live status now acquires a shared live slot before opening the panel.
- Stage uses the same live-slot-cap env (`COOKIMPORT_LIVE_STATUS_SLOTS`, default `1`).
- When a slot is unavailable, stage automatically runs with plain status output (no live-display exception).
