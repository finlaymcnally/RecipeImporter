## Purpose / Big Picture

QualitySuite is the repo’s “run a small golden set across many parser/settings variants” harness (`cookimport bench quality-discover`, `quality-run`, `quality-compare`, `quality-leaderboard`). Today it is easy for suites (and the resulting leaderboards) to be effectively EPUB-only, even when PDF gold exists, because suite discovery/selection is not explicitly format-aware.

The user-visible goal of this plan is to make it easy to do **small, agile mixed-format testing** (especially PDF) without reintroducing resource-heavy tournament workflows:

1. Suite discovery clearly shows what formats are present (and selected) at a glance.
2. Representative selection does not “accidentally starve” PDFs when `--max-targets` is small.
3. Quality run summaries/reports include a minimal per-format breakdown so regressions on PDF are visible even when global metrics look stable.
4. Operator workflow stays lightweight: small suites, small experiment packs, and `quality-run` + `quality-compare` as the main loop.

Non-goals:

- Reintroducing tournament / lightweight-series orchestration. The CLI currently disables `bench quality-lightweight-series` due to runtime/disk blowups; this plan assumes we keep that stance and focus on smaller-scale runs.
- Enabling CodexFarm or any new LLM/AI-based parsing/cleaning for import. This plan is about deterministic selection, reporting, and deterministic parsing knobs that already exist.
- A redesign of scoring (canonical-text F1 remains the primary QualitySuite surface). Structural debugging remains a targeted `bench eval-stage` activity.

## Surprises & Discoveries

1. `bench quality-lightweight-series` is intentionally disabled in `cookimport/cli.py` with an explicit message about tournament/series retirement. This plan should not depend on series/tournament flows.

2. All-method variants already carry a `dimensions["source_extension"]` value, and it uses the leading-dot suffix (for example, `.epub`, `.pdf`) as seen in `cookimport/cli.py` (variant dimension construction). This is the best “format token” to align suite metadata with.

3. Curated suite selection (`saltfatacidheatcutdown`, `thefoodlabcutdown`, `seaandsmokecutdown`) does not representative-fill unless `--max-targets` is set. This is fine for a fast EPUB sanity baseline, but it is a common footgun for mixed-format work.

4. `validate_quality_suite(...)` validates file existence for every row in `suite.targets[]`, not just `selected_target_ids[]`. This makes hand-editing manifests brittle; discovery should emit only correct paths and avoid generating stale rows.

5. Directly calling Typer command functions in tests can pass `OptionInfo` defaults; `bench_quality_discover` needed `_unwrap_typer_option_default(...)` for all options once `--formats` was added, otherwise default-filter parsing silently removed all matches.

## Decision Log

- Decision: Use `source_extension` (including the leading dot, e.g. `.pdf`) as the canonical “format” key.  
  Rationale: This aligns with how all-method variants already label format via `dimensions["source_extension"]`, and it matches how `Path(...).suffix` behaves in Python.  
  Date/Author: 2026-03-02, Assistant.

- Decision: Keep default curated behavior unchanged; make mixed-format friendliness opt-in via representative selection improvements and explicit `--no-prefer-curated` / `--max-targets` / `--formats` usage.  
  Rationale: Avoid regressing the current “3 curated EPUB cutdowns” fast path.  
  Date/Author: 2026-03-02, Assistant.

- Decision: Do not reintroduce tournament/series workflows as part of “PDF readiness.”  
  Rationale: The project has explicitly moved to smaller-scale testing due to resource costs.  
  Date/Author: 2026-03-02, Assistant.

- Decision: Use `__none__` as the serialized bucket key when a target has no suffix, while keeping `QualityTarget.source_extension=None` for the row-level field.  
  Rationale: JSON count maps require string keys; `None` remains the clean per-target representation while `__none__` keeps totals additive.  
  Date/Author: 2026-03-02, Assistant.

- Decision: Keep per-format leaderboard output opt-in (`--by-source-extension`) rather than always-on.  
  Rationale: Avoid extra artifact churn for existing workflows while preserving a zero-friction path when format-specific diagnosis is needed.  
  Date/Author: 2026-03-02, Assistant.

## Context and Orientation

Key terms:

- “Gold export” means a Label Studio freeform span export at `**/exports/freeform_span_labels.jsonl` under a “gold root” directory.
- “QualitySuite” means a JSON manifest (`cookimport/bench/quality_suite.py:QualitySuite`) containing:
  - `targets[]`: every matched gold export linked to a repo source file,
  - `selected_target_ids[]`: the chosen subset for the run,
  - `selection`: metadata describing how selection happened.
- “All-method benchmark” means the config grid evaluation performed by `cookimport/bench/quality_runner.py`, delegated into `cookimport/cli.py` helpers.

Current relevant code:

- `cookimport/bench/quality_suite.py`
  - `QualityTarget` is the per-target descriptor model.
  - `discover_quality_suite(...)` performs discovery and selection.
  - `_select_quality_target_ids(...)` implements curated-first behavior and representative fill.
  - `_select_representative_target_ids(...)` first does seeded extension-aware pre-selection under caps, then fills remaining capacity with seeded strata round-robin (`size_bucket` + `label_bucket`).

- `cookimport/cli.py`
  - `bench quality-discover` command wires `--prefer-curated/--no-prefer-curated`, `--max-targets`, `--seed`, and writes the suite JSON.
  - `bench quality-run` runs experiments and writes a `summary.json` + `report.md` under `data/golden/bench/quality/runs/...`.
  - `bench quality-compare` is the regression gate.
  - `bench quality-leaderboard` aggregates one experiment’s multi-source report into a global leaderboard.

- `cookimport/bench/quality_runner.py`
  - Writes `summary.json` and `report.md` for a quality run.
  - Uses file suffix awareness in race probe selection (`_target_source_extension(...)`) but only after the suite is selected.

- `cookimport/bench/quality_leaderboard.py`
  - Aggregates configs across sources and intentionally ignores `source_extension` by default (global winner focus).

PDF parsing context (why extension matters):

- PDF behavior is primarily driven by shared deterministic knobs already present in `RunSettings` (for example `section_detector_backend`, `multi_recipe_splitter`, OCR settings) plus PDF importer heuristics (`cookimport/plugins/pdf.py`).
- Many PDF failures are structural (ordering, candidate splitting) rather than purely “label classification,” so format-aware coverage + reporting is the first priority.

## Plan of Work

This plan is intentionally incremental. The early milestones focus on deterministic observability and selection coverage rather than changing parsing logic.

### Milestone 1: Make suite discovery format-aware (metadata + selection)

Update `cookimport/bench/quality_suite.py` to:

1. Add `source_extension: str | None = None` to `QualityTarget`.
2. Populate it during `_annotate_quality_targets(...)` using `Path(source_file).suffix.lower()` (keep the leading dot; use `None` or `""` when suffix is missing).
3. Add `format_counts` (all matched) and `selected_format_counts` (selected subset) to `suite.selection`.
4. Update representative selection so that when multiple extensions are present and `max_targets` is a cap, selection reduces accidental starvation:
   - If `max_targets >= number_of_extensions`, select at least one target per extension first (deterministically, seeded).
   - Then fill remaining capacity using the existing seeded strata round-robin (size_bucket + label_bucket).
   - If `max_targets < number_of_extensions`, select targets from as many distinct extensions as possible (seeded choice of which extensions win).
5. Bump `_QUALITY_SELECTION_ALGORITHM_VERSION` from `quality_representative_v1` to `quality_representative_v2` if selection semantics change.

Update tests in `tests/bench/test_quality_suite_discovery.py` to lock in:

- mixed-format representative selection includes PDF when possible,
- the new fields are emitted (`source_extension`, `format_counts`, `selected_format_counts`),
- backwards compatibility: old suite JSON still loads when `source_extension` is absent.

### Milestone 2: Add a `--formats` filter for PDF-first small runs

Add an optional `--formats` flag to `cookimport bench quality-discover` that restricts discovery to a set of suffixes (e.g. `.pdf`, `.epub`).

Implementation shape:

- In `cookimport/cli.py:bench_quality_discover`, parse `--formats` into a normalized set of lowercase suffixes with leading dots.
- Pass that into `discover_quality_suite(...)` (new kwarg), and filter matched targets before bucketing/selection.

This enables a lightweight PDF-only or mixed-format-by-choice suite without changing curated defaults.

### Milestone 3: Per-format visibility in quality-run outputs (counts first)

Update `cookimport/bench/quality_runner.py` to include `selected_format_counts` (and ideally `format_counts`) in:

- `summary.json` (top-level keys),
- `report.md` (a short section near the header).

This should be derived from the loaded suite manifest (no need to re-scan inputs).

Optional extension (only if it stays simple): add per-format aggregates for each experiment by reading the experiment’s multi-source report JSON and grouping sources by suite target extension.

### Milestone 4: Per-format leaderboard artifacts (keep global leaderboard unchanged)

Extend `cookimport/bench/quality_leaderboard.py` and `cookimport bench quality-leaderboard` so that it can also emit a per-format slice:

- Keep existing `leaderboard.json/.csv` as the global ranking (current behavior).
- Add optional additional outputs like:
  - `leaderboard_by_source_extension.json`
  - `leaderboard_by_source_extension.csv`

The by-format slice should preserve the same row shape but computed within each extension bucket, so “what wins on PDF?” is visible without changing the global winner semantics.

### Milestone 5: Add a “PDF-first” experiments preset (small, deterministic)

Add a new experiments JSON in `data/golden/bench/quality/experiments/` with a small schema-v2 lever pack that only uses already-wired deterministic knobs that affect PDF:

- `section_detector_backend` (e.g. `legacy` vs `shared_v1`)
- `multi_recipe_splitter` (e.g. `legacy` vs `rules_v1` vs `off`)
- optionally `instruction_step_segmentation_policy` if it impacts PDF results

Avoid using throughput knobs (e.g. `pdf_pages_per_job`, `pdf_split_workers`) as “quality levers”; those belong in speed/ops tuning, not quality winner selection.

## Concrete Steps

All commands are run from the repo root (`/home/mcnal/projects/recipeimport`).

1. Run focused tests while iterating:

    .venv/bin/python -m pytest -q tests/bench/test_quality_suite_discovery.py

1b. Run focused validation for all touched benchmark modules:

    .venv/bin/python -m pytest tests/bench/test_quality_suite_discovery.py tests/bench/test_quality_suite_runner.py tests/bench/test_quality_leaderboard.py tests/bench/test_bench.py

2. Create a small mixed-format suite locally (after Milestone 1/2):

    .venv/bin/cookimport bench quality-discover --no-prefer-curated --max-targets 6

3. Run a small quality-run using a small experiments file:

    .venv/bin/cookimport bench quality-run --suite <suite.json> --experiments-file <experiments.json> --search-strategy exhaustive --max-parallel-experiments 1

4. Compare baseline vs candidate:

    .venv/bin/cookimport bench quality-compare --baseline <baseline_run_dir> --candidate <candidate_run_dir> --fail-on-regression

5. Generate leaderboard artifacts:

    .venv/bin/cookimport bench quality-leaderboard --run-dir <run_dir> --experiment-id baseline

## Validation and Acceptance

Acceptance is phrased in user-observable behaviors.

Milestone 1 is accepted when:

- Running `bench quality-discover` produces a suite JSON where each `targets[]` row includes `source_extension` (or `null` when unknown).
- `suite.selection.format_counts` exists and matches the distribution of `targets[]`.
- `suite.selection.selected_format_counts` exists and matches the distribution of `selected_target_ids[]`.
- With a mixed-format gold set and a small `--max-targets`, representative selection includes at least one PDF when `max_targets` permits.

Milestone 2 is accepted when:

- `bench quality-discover --formats .pdf` only selects/outputs PDF targets (and reports 0 matched/selected if none exist).

Milestone 3 is accepted when:

- `bench quality-run` outputs include the selected format counts in both `summary.json` and `report.md` for quick sanity checks.

Milestone 4 is accepted when:

- `bench quality-leaderboard` emits the existing global leaderboard artifacts unchanged, plus optional by-format artifacts when enabled.

Milestone 5 is accepted when:

- A new experiments preset exists and can be used to run a small `quality-run` without errors, producing comparable artifacts.

## Idempotence and Recovery

- Discovery and leaderboard commands are safe to re-run; they write new files at explicit output paths (or timestamped folders).
- Quality runs are resume-able via `bench quality-run --resume-run-dir <dir>`; plan changes should preserve resume contracts.
- If mixed-format selection causes an unexpected behavior change in existing tests, keep the old semantics behind a version bump and update tests explicitly.
