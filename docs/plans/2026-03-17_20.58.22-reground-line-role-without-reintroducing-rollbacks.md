# Reground Line-Role Without Reintroducing Rollbacks

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

After this change, the canonical line-role pass should keep the recent win we wanted, namely that Codex can preserve obvious recipe structure instead of being overwritten by deterministic rollback layers, while also stopping the new flood of false recipe structure on chapter headings, section headings, and front-matter rows. The user-visible result is simple: rerunning the `saltfatacidheatcutdown` single-offline benchmark should no longer turn lines like `SALT`, `What is Salt?`, and `The Four Elements of Good Cooking` into recipe-local `HOWTO_SECTION` or `RECIPE_TITLE` labels just because the model saw a compressed shard of the book.

The intended operator behavior is: `codex-line-role-shard-v1` still reviews the whole book and may override the deterministic baseline, but it does so as a grounded correction pass rather than a free-form relabel of a compressed contiguous slice. The old post-Codex rollback layers remain deleted. The fix must come from better grounding and better deterministic seed quality, not from vetoing the model after the fact.

## Surprises & Discoveries

- Observation: removing rollback layers did rescue real structure.
  Evidence: comparing `2026-03-17_16.06.55` to `2026-03-17_20.27.19`, codex `INGREDIENT_LINE` correct rows improved from `121` to `150`, and `INSTRUCTION_LINE` correct rows improved from `16` to `52`.

- Observation: the new regression is not mainly “Codex got worse at reading recipes.” It is over-promotion of non-recipe headings.
  Evidence: `OTHER -> HOWTO_SECTION` rose from `0` to `62`, and `OTHER -> RECIPE_TITLE` rose from `7` to `22` in `wrong_label_lines.jsonl`.

- Observation: the bad heading promotions are actual Codex outputs inside recipe-span hints, not leftover stale artifacts.
  Evidence: in `line_role_predictions.jsonl`, rows like `The Four Elements of Good Cooking` and `SALT` are emitted as `RECIPE_TITLE` / `HOWTO_SECTION` with `decided_by="codex"` and `within_recipe_span=true`.

- Observation: the prompt changed from a correction-style contract to a compressed whole-book re-decision contract.
  Evidence: the old prompt included `Recipe atomic index ranges for this batch`, per-row label/reason codes, and optional local neighbor context. The new prompt says `No prior recipe-span authority is provided for this batch`, uses only pipe-delimited `atomic_index|label_code|current_line`, and drops the local-context row shape entirely.

- Observation: the prompt change likely amplifies a second problem: the deterministic seed itself is now too willing to mark outside-recipe headings as structure.
  Evidence: in the regressed prompt, front-matter and heading-like rows already appear with structured seed labels such as `L2`, and deterministic-only benchmark behavior cratered at the same time.

## Context and Orientation

The relevant code path starts in [cookimport/parsing/canonical_line_roles.py](/home/mcnal/projects/recipeimport/cookimport/parsing/canonical_line_roles.py). This module builds deterministic canonical line-role predictions for each `AtomicLineCandidate`, optionally sends full shards to Codex when `line_role_pipeline=codex-line-role-shard-v1`, sanitizes clearly invalid label shapes such as bad `TIME_LINE` and `YIELD_LINE` uses, and writes line-role artifacts under `line-role-pipeline/`.

An `AtomicLineCandidate` is one atomized line of cookbook text produced earlier in parsing. Each candidate carries the raw text, a stable `atomic_index`, heuristic `rule_tags`, and a provisional `within_recipe_span` boolean. That boolean is not gold truth. It is an upstream hint based on earlier recipe-span grouping. In the current regression, too many chapter and section headings are arriving with `within_recipe_span=true`, and the line-role LLM is taking that hint too seriously.

The line-role prompt builder lives in [cookimport/llm/canonical_line_role_prompt.py](/home/mcnal/projects/recipeimport/cookimport/llm/canonical_line_role_prompt.py). The current prompt sends the whole shard as a compressed ordered slice with only `atomic_index|label_code|current_line`. The older prompt contract, visible in older benchmark prompt artifacts, also supplied recipe-span authority, escalation reasons, and optional local neighbor context. This plan treats that older grounded contract as the reference direction, not as something to copy blindly line for line.

Benchmark evidence lives under `data/golden/benchmark-vs-golden/<timestamp>/single-offline-benchmark/saltfatacidheatcutdown/`. The two comparison anchors for this plan are:

- `2026-03-17_16.06.55`: older run with weaker structure recall but better overall strict accuracy.
- `2026-03-17_20.27.19`: current run after rollback removal, with stronger structure recall but major heading/title over-promotion.

The single most important artifact for concrete mistakes is `codexfarm/line-role-pipeline/joined_line_table.jsonl`, which merges gold labels, final predictions, and span metadata line by line. The prompt artifacts under `codexfarm/prompts/line-role-pipeline/` show the exact model input contract for each run.

## Plan of Work

The work is split into three connected edits.

First, rebuild the line-role prompt as a grounded correction prompt again. In [cookimport/llm/canonical_line_role_prompt.py](/home/mcnal/projects/recipeimport/cookimport/llm/canonical_line_role_prompt.py), restore explicit prior-knowledge anchors that help the model avoid inventing recipe structure from heading-like lines. The minimum contract at the end of this milestone is:

- the prompt tells the model whether any provisional recipe-span authority exists for the shard, instead of always saying no authority exists
- each row carries the deterministic seed label
- each row can carry the seed escalation reason codes again
- each row can carry short local neighbor context again for ambiguous heading and fragment cases

This is not a return to selective unresolved-row prompting. The whole shard still goes to Codex. The change is that Codex sees enough grounding to behave like a reviewer rather than a fresh parser of compressed book text.

Second, fix deterministic seed quality for obvious chapter and section headings. In [cookimport/parsing/canonical_line_roles.py](/home/mcnal/projects/recipeimport/cookimport/parsing/canonical_line_roles.py), tighten the rule-path heuristics for `HOWTO_SECTION`, `RECIPE_TITLE`, and `RECIPE_VARIANT` when the line is outside a strong recipe context. Lines such as `SALT`, `What is Salt?`, `How Salt Works`, and `The Four Elements of Good Cooking` should not be seeded as recipe-local structure simply because they look like headings. This must be solved in the baseline heuristics, not by adding a second veto pass after Codex.

Third, keep the rollback deletion intact while validating the new balance. The generic sanitizers for blatantly invalid label shapes should stay. The removed ownership veto, outside-span structured-label downgrade, and do-no-harm rollback must not come back in any form. Validation should focus on proving that real structure recovery remains much better than the old run while heading-like `OTHER` rows stop flooding into `HOWTO_SECTION` and `RECIPE_TITLE`.

## Milestones

### Milestone 1: Restore a grounded line-role prompt

At the end of this milestone, a line-role prompt shard is still a full ordered slice of the book, but it once again carries enough information for Codex to behave like a correction pass. The prompt must expose the deterministic seed label on every row, reintroduce compact escalation reason codes, and provide short local neighbor text for rows where ambiguity is common. The prompt header must explicitly explain whether provisional recipe-span authority exists for the shard instead of always forcing “no prior authority.”

The reader should be able to prove this by regenerating a prompt for the regressed run and seeing a richer row contract than the current `atomic_index|label_code|current_line` shape. A good acceptance spot check is the first line-role shard for `saltfatacidheatcutdown`: it should once again carry more than the raw line text for rows like `SALT`, `What is Salt?`, and front-matter praise blurbs.

### Milestone 2: Repair deterministic heading seeds

At the end of this milestone, deterministic line-role should stop labeling obvious chapter, part, and section headings as recipe-local structure unless nearby recipe evidence is truly strong. The important point is not to ban all heading-like structure outside recipes. The important point is to stop seeding the model with structure labels for book-taxonomy headings that are clearly not recipe components.

The reader should be able to prove this with targeted tests for lines such as `SALT`, `What is Salt?`, `How Salt Works`, `The Four Elements of Good Cooking`, and similar chapter-taxonomy headings. Those tests should show that the deterministic path produces non-recipe labels for those cases while still preserving real recipe headings and subheads in recipe contexts.

### Milestone 3: Benchmark and compare behavior

At the end of this milestone, the same `saltfatacidheatcutdown` benchmark should show that the fix did not reintroduce rollback behavior and did materially reduce the new bad heading promotions. The expected shape is not necessarily a new absolute best score on the first try. The expected shape is that the run keeps most of the ingredient and instruction recovery from `2026-03-17_20.27.19` while cutting `OTHER -> HOWTO_SECTION` and `OTHER -> RECIPE_TITLE` much closer to the older run.

The proof must come from the benchmark artifacts, especially `codex_vs_vanilla_comparison.json`, `joined_line_table.jsonl`, `wrong_label_lines.jsonl`, and the prompt files themselves.

## Idempotence and Recovery

The edits in this plan are safe to repeat. Prompt-file regeneration and benchmark reruns overwrite their own artifacts under new timestamped directories. The main recovery rule is: if a prompt change improves heading precision but collapses ingredient or instruction recall, keep the rollback deletion intact and revert only the grounding change, not the rollback removal.

When comparing runs, always use new timestamped benchmark directories rather than editing old ones. If a candidate fix makes the prompt larger, that is acceptable as long as the prompt remains a single direct shard call and does not reintroduce the old agentic file-reading behavior.

## Interfaces and Dependencies

This plan changes the live contract of the line-role prompt builder and the deterministic seed heuristics.

  This file must contain explicit regressions for chapter-taxonomy headings and for the prompt contract that grounds Codex.
