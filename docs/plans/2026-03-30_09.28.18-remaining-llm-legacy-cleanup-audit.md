## Context and Orientation

This note only covers the remaining cleanup after the recipe packet-first and knowledge terminal-reason resets. The key live code file is `cookimport/llm/prompt_budget.py`, which turns stage artifacts into benchmark-facing summary counts and still needs to normalize older knowledge artifact shapes into current naming. The relevant historical docs are `docs/reports/19.17.50-feedback-tokens.md` and `docs/plans/2026-03-29_22.23.14-burn-the-boats-knowledge-codex-exec-cost-reset.md`, which quote pre-cutover baseline evidence and need that context stated explicitly. The earlier runtime mismatch files, `cookimport/llm/knowledge_stage/workspace_run.py` and `cookimport/llm/knowledge_stage/recovery.py`, are still important reference points because they now show the aligned current contract.

In this repository, `no_final_output` means the worker produced no promotable final packet result, while `validation_errors` stores the low-level validator reason strings attached to a proposal wrapper. The current cleanup goal is narrower than the original audit phrasing: keep old knowledge artifacts loadable, but normalize their old `missing_output_*` shard-count names into the current `no_final_output_*` knowledge summary vocabulary and clearly label older quoted evidence as historical.
