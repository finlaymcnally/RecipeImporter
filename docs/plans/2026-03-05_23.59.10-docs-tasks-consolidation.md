## Plan of Work

First, write down the routing map and the few ambiguous callouts in one understanding note. That keeps the classification logic from being lost after the source task files are deleted.

Second, patch each destination README with current-state material extracted from the task docs. For analytics, this means the Compare & Control dashboard behavior and UI seams. For bench, this means benchmark integrity contracts, upload/follow-up tooling, and operator-facing caveats. For LLM, this means explicit Codex-boundary policy and safe-default posture. For testing, this means the runtime-offender remediation state. For CLI and Label Studio, this means only the pieces that are true current behavior and belong primarily to those sections.

Third, patch each destination log with a dated merged ledger. Those entries must preserve serious attempts, decisions, and anti-loop evidence, especially for the QualitySuite incident cluster and the Compare & Control iteration burst.

Finally, delete every task file and re-run the docs list command.
