# Upgrade upload_bundle_v1 starterpack surfaces for first-pass triage

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

Maintain this document in accordance with `docs/PLANS.md` from the repository root.

## Purpose / Big Picture

After this change, `upload_bundle_v1` for single-offline benchmark sessions will provide a cleaner first-pass triage surface that does not depend on CSV as the primary navigation artifact, includes an explicit blame-summary view, exposes run config/version metadata at top level, includes a dedicated low-confidence changed-lines packet, and reports baseline-vs-codex diagnostics with symmetric statuses.

A reviewer should be able to open one bundle at:
`data/golden/benchmark-vs-golden/<timestamp>/single-offline-benchmark/<slug>/upload_bundle_v1`
and get direct answers to:
1) where quality moved,
2) which stage seems responsible,
3) whether this run is configuration-comparable,
4) which low-confidence changed lines to inspect first,
without relying on `01_recipe_triage.csv`.

Scope guard: this plan is benchmark artifact/analytics work only. It must not enable any LLM parsing path for normal ingestion/data import defaults.

## Surprises & Discoveries

- Observation: upload-bundle navigation still exposes starter triage as CSV-first (`triage_csv`) even in the 3-file package flow.
  Evidence: `navigation.row_locators.starter_pack.triage_csv` points at `_upload_bundle_derived/starter_pack_v1/01_recipe_triage.csv` in `upload_bundle_index.json`.

- Observation: alias-equivalent artifact detection already exists but is not used to simplify default starter-layer navigation.
  Evidence: `scripts/benchmark_cutdown_for_external_ai.py::_upload_bundle_build_alias_metadata(...)` computes `content_equivalent_groups`, but row-locator generation still lists both canonical and alias-equivalent starter/root paths in normal flows.

- Observation: low-confidence line-role information exists, but it is not a changed-lines packet.
  Evidence: `_upload_bundle_build_line_role_confidence_summary(...)` emits `low_confidence_examples`, yet no joined packet exists that intersects low-confidence rows with codex-vs-baseline changed lines.

- Observation: baseline diagnostic parity is currently asymmetric in run diagnostics.
  Evidence: non-codex runs can show `prompt_warning_aggregate_status=missing` and `projection_trace_status=missing` rather than explicit `not_applicable`.

## Decision Log

- Decision: keep payload lossless and additive; dedupe only navigation/default surfaces.
  Rationale: upload bundle contract promises all source artifacts are preserved; dedupe should reduce triage noise, not drop data.
  Date/Author: 2026-03-03 / assistant

- Decision: replace CSV-first triage with JSONL-first triage while retaining legacy CSV read compatibility for older source trees.
  Rationale: user requirement is to stop relying on CSV uploads; historical bundle roots still include CSV artifacts and must remain readable.
  Date/Author: 2026-03-03 / assistant

- Decision: define blame summary from existing deterministic signals (line deltas + pass diagnostics), not model-internal heuristics.
  Rationale: keeps the summary reproducible from bundle-contained artifacts and stable across reruns.
  Date/Author: 2026-03-03 / assistant

## Context and Orientation

The code that creates this bundle lives in one script:

- `scripts/benchmark_cutdown_for_external_ai.py`

The core path for existing-output session roots is:

- `build_upload_bundle_for_existing_output(...)`
  - calls `_write_upload_bundle_three_files(...)`
  - which builds context via `_upload_bundle_build_context(...)`
  - computes analyses (`per_label_metrics`, `stage_separated_comparison`, `failure_ledger`, `line_role_confidence_or_candidates`, etc.)
  - writes:
    - `upload_bundle_overview.md`
    - `upload_bundle_index.json`
    - `upload_bundle_payload.jsonl`

Current behavior relevant to this plan:

1) Starter triage is CSV-first:
   - constants and write path still use `STARTER_PACK_TRIAGE_FILE_NAME = "01_recipe_triage.csv"`.
   - bundle row locator key is `triage_csv`.

2) Blame-style stage attribution is not a standalone artifact:
   - stage-separated rows exist, but no explicit net-error blame summary object exists.

3) Config/version metadata exists in fragments but not in one top-level triage block:
   - run config snapshots are computed earlier in the cutdown pipeline.
   - upload-bundle index does not expose a concise “settings/version parity” summary for rapid review.

4) Low-confidence signals exist but are not joined with changed lines:
   - `line_role_confidence_or_candidates.low_confidence_examples` does not map directly to changed-line review packet semantics.

5) Alias metadata is computed but not used to simplify starter/root navigation:
   - `alias_metadata.content_equivalent_groups` is available, but default row locators still point to alias-heavy paths.

6) Baseline parity gap:
   - non-codex diagnostics can appear as `missing` for codex-specific traces rather than explicit `not_applicable`.

## Milestones

### Milestone 1: No-CSV-first top-level triage packet

Add a top-level triage artifact in JSONL (or JSON object with row list) under `_upload_bundle_derived/root/` and expose it in `navigation.row_locators` and `analysis` so first-pass flow does not depend on CSV semantics.

Acceptance:
- `upload_bundle_index.json` contains a non-CSV triage locator (for example `triage_packet_jsonl`).
- `navigation.default_initial_views` includes this new triage view.
- Existing bundles without new artifact still load via fallback.

### Milestone 2: Net-error blame summary artifact

Create `analysis.net_error_blame_summary` (and optional derived payload artifact) summarizing net error contributions grouped into:
- `line_role`
- `pass2_extraction`
- `pass3_mapping`
- `routing_or_fallback`

Each group must include counts and share-of-net-error fields with transparent definitions.

Acceptance:
- Index contains `analysis.net_error_blame_summary` with deterministic totals and per-bucket shares.
- Overview includes a compact blame summary section.

### Milestone 3: Config/version metadata summary

Add a top-level comparability summary (for example `analysis.config_version_metadata`) including:
- run config keys of interest per run,
- codex/baseline pair setting deltas,
- generator schema/version stamps used for bundle generation.

Acceptance:
- Reviewer can tell from one section whether compared runs are config-compatible.
- Metadata includes concrete run IDs and key settings (`llm_recipe_pipeline`, `line_role_pipeline`, `atomic_block_splitter`, etc.).

### Milestone 4: Low-confidence changed-lines packet

Create a first-class packet by intersecting changed lines with low-confidence line-role predictions (threshold from `UPLOAD_BUNDLE_LOW_CONFIDENCE_THRESHOLD`), including recipe ID, line index, confidence, gold/baseline/codex labels, and context excerpt.

Acceptance:
- New artifact is indexed in row locators and appears in analysis section.
- Packet is empty-but-present with explicit note when no qualifying rows exist.

### Milestone 5: Baseline trace parity normalization

Normalize run diagnostics so codex-only traces are `not_applicable` for non-codex runs instead of `missing`.

Acceptance:
- In paired vanilla/codex bundle, vanilla diagnostic statuses for codex-only trace artifacts are explicit `not_applicable`.

### Milestone 6: Starter-layer alias dedupe in navigation

Use alias metadata to promote canonical locator paths for starter/root files and present alias map separately (without removing payload rows).

Acceptance:
- Row locators choose canonical paths only.
- Alias map remains available under `alias_metadata`.
- Artifact count/lossless payload guarantee remains unchanged.

## Plan of Work

Then update tests in `tests/bench/test_benchmark_cutdown_for_external_ai.py` to cover new index fields, new locator keys, parity statuses, and dedupe behavior.

## Validation and Acceptance

All of the following must be true:

1. No-CSV-first triage:
   - default navigation has a JSON/JSONL triage locator and uses it in initial views.

2. Blame summary:
   - bundle index includes explicit stage-bucket net error attribution with counts and shares.

3. Config/version metadata:
   - bundle index includes run settings/version summary sufficient to judge comparability.

4. Low-confidence changed-lines packet:
   - bundle index includes a first-class packet keyed to changed lines and confidence threshold.

5. Baseline parity:
   - baseline run diagnostics show `not_applicable` for codex-only trace artifacts.

6. Dedupe:
   - canonical locator paths are preferred while alias metadata remains available.
   - payload remains lossless (artifact count not artificially reduced by navigation dedupe).

7. Regression safety:
   - targeted bench cutdown tests pass.

## Idempotence and Recovery

Changes are additive to index/overview schema and can be rerun safely on the same source root. If a new helper fails for older runs with sparse artifacts, fall back to explicit `available=false`/`unavailable_reason` payloads instead of raising. Keep CSV compatibility reader paths until all existing historical roots are migrated.
