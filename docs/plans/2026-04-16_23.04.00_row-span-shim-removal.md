# Remove Stale Row-Span Block Shims

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

The active benchmark, stage, and prelabel surfaces are row-native now, but two stale block-era seams still remained: deterministic prep bundle reload still normalized old recipe span block keys, and the prelabel prompt/template layer still exposed block-named placeholder tokens even though it already carried rows. This pass removes those stale names without touching real provenance fields like `source_block_index` or table-layout block coordinates.

## Surprises & Discoveries

- Observation: the remaining row-span block names are mostly contract residue, not real block semantics.
  Evidence: `cookimport/staging/deterministic_prep.py` rewrote `start_block_index` / `end_block_index` / `block_indices` / `title_block_index` into row fields right before validating `RecipeSpan` and `RecipeSpanDecision`, even though the live writers already emit row-native payloads.

- Observation: the active freeform prompt text was already row-native; the stale part was the placeholder surface and helper naming.
  Evidence: `llm_pipelines/prompts/freeform-prelabel-span.prompt.md` and the active prompt copy in `prelabel_prompt.py` already talk about rows, while the replacement keys and docs still used names like `{{BLOCKS_JSON_LINES}}`, `{{FOCUS_BLOCK_JSON_LINES}}`, and `{{FOCUS_BLOCK_INDICES}}`.

## Decision Log

- Decision: remove the deterministic-prep span-key shim instead of preserving it for old cache bundles.
  Rationale: the user explicitly wants stale block-era compatibility removed, and the touched benchmark/stage cache contract is already row-native on the live writer side.
  Date/Author: 2026-04-16_23.04.00 / Codex

- Decision: rename the active prelabel placeholder surface to rows all the way through prompt docs/templates/helpers.
  Rationale: leaving block-named placeholders behind would keep teaching future agents the wrong contract even though the injected values are rows.
  Date/Author: 2026-04-16_23.04.00 / Codex
