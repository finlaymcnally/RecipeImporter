# Delete Dead Full-Block Prelabel Template Surface

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

Label Studio prelabel now supports only `span` mode, but the repo still carried a dead full-block prelabel template/file/fallback surface. That dead surface kept block-era prompt language and implied a second supported template contract even though runtime never used it. This pass deletes that dead full-template plumbing and keeps only the active span prompt contract.

## Surprises & Discoveries

- Observation: the dead full-template surface was spread across multiple helper modules as copied constants, not one single owner.
  Evidence: `_FULL_PROMPT_TEMPLATE_PATH` or `_FULL_PROMPT_TEMPLATE_FALLBACK` appeared in `prelabel.py`, `prelabel_prompt.py`, `prelabel_mapping.py`, `prelabel_parse.py`, and `prelabel_codex.py`, even though prompt construction always uses the span template.

- Observation: docs/tests were keeping the dead path alive more than runtime was.
  Evidence: the remaining references to `freeform-prelabel-full.prompt.md` were in agent docs and `test_prelabel_prompt_uses_file_templates`, not in any live prompt selection branch.

## Decision Log

- Decision: delete the full prompt template file and its dead plumbing instead of trying to modernize it.
  Rationale: runtime already rejects non-span prelabel granularity, so maintaining a second prompt template would just preserve dead block-era code.
  Date/Author: 2026-04-16_23.08.00 / Codex

## Outcomes & Retrospective

This slice is implemented, validated, and closed in Beads. The active prelabel prompt/template plumbing now points only at the span prompt surface, and the dead full-block prompt file/fallback/constants are gone.
