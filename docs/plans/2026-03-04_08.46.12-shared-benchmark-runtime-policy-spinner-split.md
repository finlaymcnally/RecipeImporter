# Centralize benchmark/processing runtime policy for spinner + split behavior

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

Maintain this document in accordance with `docs/PLANS.md` from the repository root.

## Purpose / Big Picture

After this change, line-role codex inflight defaults and split/spinner runtime behavior for prediction-generation jobs are resolved in one shared seam, instead of being re-implemented in each orchestration flow. Single-source jobs default to higher inflight, while multi-source split-gated jobs default lower, and all benchmark/import entry points that use shared ingest generation inherit the same policy automatically.

## Surprises & Discoveries

- Observation: line-role inflight policy currently depends on CLI-level env override wrappers, so monkeypatched/alternate entry paths can drift unless each flow remembers to add the wrapper.
  Evidence: direct wiring existed in `_interactive_single_offline_benchmark` and `_interactive_single_profile_all_matched_benchmark`, while core ingest/parsing path used parser default/env resolution.

- Observation: moving the policy into `generate_pred_run_artifacts(...)` also covers Label Studio import command flows automatically because they call that same shared generation path before upload.
  Evidence: `run_labelstudio_import(...)` delegates to `generate_pred_run_artifacts(...)` with split-phase settings forwarded.

## Decision Log

- Decision: centralize line-role inflight policy at prediction generation seam (`generate_pred_run_artifacts`) and pass explicit override into `label_atomic_lines`.
  Rationale: this seam is shared by benchmark and Label Studio import processing jobs, so behavior propagates automatically.
  Date/Author: 2026-03-04 / assistant

## Outcomes & Retrospective

Shared line-role codex inflight defaults now live at the ingest/parsing seam instead of interactive-flow wrappers. Prediction-generation paths now inherit `8` for non-split runs and `4` for split-gated runs automatically, while explicit env override remains authoritative. CLI benchmark flows no longer carry bespoke inflight wiring, reducing drift risk when adding new processing/benchmark paths.

## Context and Orientation

Key shared runtime seam:
- `cookimport/labelstudio/ingest.py:generate_pred_run_artifacts(...)` performs extraction/conversion/task generation for benchmark prediction runs and Label Studio import runs.
- `cookimport/parsing/canonical_line_roles.py:label_atomic_lines(...)` performs deterministic + codex line-role labeling and currently resolves codex inflight from env/default.

Current orchestration wrappers in CLI:
- `cookimport/cli.py:_interactive_single_offline_benchmark(...)`
- `cookimport/cli.py:_interactive_single_profile_all_matched_benchmark(...)`
- all-method prediction/eval wrappers in `cookimport/cli.py`

Documentation currently references per-flow wiring in:
- `docs/02-cli/02-cli_README.md`
- `docs/07-bench/07-bench_README.md`

## Plan of Work

Implement shared policy in ingest/parsing so all processing/benchmark generation paths inherit the same behavior:
1. Extend `label_atomic_lines(...)` to accept an optional explicit codex inflight override.
2. Add ingest-level resolver that maps run context to defaults:
   - split-gated (`split_phase_slots` set) => lower inflight
   - non-split single-source default => higher inflight
   - explicit `COOKIMPORT_LINE_ROLE_CODEX_MAX_INFLIGHT` env remains highest priority (do not override when explicitly set).
3. Remove CLI per-flow inflight wrappers added earlier so policy is no longer flow-specific.
4. Add tests around ingest/parsing seam to prove defaults are applied correctly.
5. Refresh docs and add one short understanding note.

## Validation and Acceptance

Acceptance criteria:
- Single-source prediction generation paths default to higher codex inflight without requiring per-flow CLI wrappers.
- Split-gated multi-source benchmark processing defaults to lower codex inflight.
- Existing explicit env override still takes precedence when set.
- Spinner/split scheduling plumbing remains unchanged and tests still pass.

## Idempotence and Recovery

Changes are code-only and safe to re-run. If behavior regresses, rollback by restoring prior `label_atomic_lines` signature and removing ingest policy resolver wiring.
