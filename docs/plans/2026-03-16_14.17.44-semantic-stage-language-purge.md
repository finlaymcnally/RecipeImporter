## Purpose / Big Picture

After this change, active runtime, benchmark, and follow-up surfaces should stop teaching pre-refactor stage-slot language for the knowledge stage. A person running `cf-debug`, reading benchmark docs, inspecting prompt-budget summaries, or wiring the optional knowledge pipeline should see semantic stage names such as `knowledge` and `recipe_correction`, not old slot labels or generic legacy grouping payloads.

The visible proof is straightforward: CLI help, follow-up packet fields, prompt-budget summaries, and knowledge-stage runtime helpers should all use current semantic naming, and repo docs for active flows should match that language.

## Outcomes & Retrospective

## Context and Orientation

The current refactor architecture already treats the optional knowledge extraction lane as a semantic stage driven from Stage 7 non-recipe knowledge spans. The active runtime files are `cookimport/llm/codex_farm_knowledge_orchestrator.py`, `cookimport/llm/codex_farm_knowledge_jobs.py`, `cookimport/llm/codex_farm_knowledge_ingest.py`, and `cookimport/llm/codex_farm_knowledge_writer.py`. Those files previously used old slot-based helper names even though the stage key is now `extract_knowledge_optional`.

The active follow-up/debug product surface is `cookimport/cf_debug_cli.py` plus `cookimport/bench/followup_bundle.py`. Those files had exposed old knowledge-stage selectors, case kinds, and audit filenames. The benchmark/docs surface had repeated that language in `docs/02-cli/02-cli_README.md`, `docs/07-bench/07-bench_README.md`, `docs/10-llm/10-llm_README.md`, and `docs/10-llm/knowledge_harvest.md`.

The prediction-run token summary surface is `cookimport/llm/prompt_budget.py`, written by `cookimport/labelstudio/ingest.py` and consumed by bundle/follow-up code plus tests. It previously published a legacy slot-group container even though current stage names were already semantic.

## Plan of Work

First rename the active knowledge-stage helper layer. In `cookimport/llm/codex_farm_knowledge_orchestrator.py` and the adjacent knowledge modules, rename old slot-based helper names, constants, variables, docstrings, and error text to `knowledge` or `knowledge_stage` forms. Keep behavior the same; only the stage-language contract changes.

Next rename the prompt-budget surface. In `cookimport/llm/prompt_budget.py`, publish `by_stage` instead of the legacy slot-group container and use stage keys `recipe_correction`, `knowledge`, and `line_role`. Then update `cookimport/labelstudio/ingest.py`, `scripts/benchmark_cutdown_for_external_ai.py`, and tests so the summary contract is semantic-stage only.

Then rename the follow-up/debug surface. In `cookimport/bench/followup_bundle.py` and `cookimport/cf_debug_cli.py`, replace old knowledge-stage selectors, audit names, row-locator keys, and case kinds with `knowledge` equivalents. Update active docs and tests to the new command names and payload field names.

Finally rerun the targeted test slices and grep for remaining active old-slot references. Any references left after this pass should either be historical reports/plans or explicitly documented as deferred.
