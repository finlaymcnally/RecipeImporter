## Purpose / Big Picture

After this change, the remaining active line-role and knowledge prompt surfaces will show model-visible evidence in the same compact provenance-first style instead of verbose row objects or fully expanded per-row JSON. The goal is lower prompt overhead and stronger row grounding without crossing the authority boundary into deterministic semantic hints.

## Context and Orientation

The active line-role prompt/rendering seams live under `cookimport/parsing/canonical_line_roles/` and `cookimport/llm/canonical_line_role_prompt.py`. The active knowledge seams live in `cookimport/llm/knowledge_prompt_builder.py`, `cookimport/llm/knowledge_stage/structured_session_contract.py`, and `cookimport/llm/knowledge_stage/task_file_contracts.py`. Focused tests already exist under `tests/parsing/` and `tests/llm/` for prompt text, structured packet shape, task files, and fake-runner behavior.

For this plan, “compact provenance” means model-visible owned rows render as short ordered strings that preserve identity and block provenance, for example `r01 | 210 | Bright Cabbage Slaw`. Nearby context rows may omit the local row id and stay as `210 | Variations`.

## Plan of Work

First, patch the remaining line-role retry prompt surfaces so they reuse the same compact row rendering as the main inline path. That includes both the authoritative-row rendering helper and the retry prompt instructions so the watchdog path no longer falls back to verbose row-object examples.

Next, patch knowledge prompt shaping. The direct prompt builder should transform owned blocks and neighboring context into compact ordered row strings before serializing the inline payload. The structured-session helper should do the same for both classification and grouping packets, keeping row ids available for response validation while collapsing row evidence into compact strings.

Then update focused tests and short owning docs so they describe the final compact row contract truthfully. Validation should stay narrow: only prompt-visible evidence formatting changes, not deterministic semantic hinting or repo-side answer rewriting.
