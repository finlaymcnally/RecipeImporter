## Context and Orientation

The active knowledge classification contract has two main surfaces. The inline-json benchmark/runtime path uses `cookimport/llm/knowledge_stage/structured_session_contract.py` to build the worker-visible packet and prompt. The taskfile fallback path uses `cookimport/llm/knowledge_stage/task_file_contracts.py` for the semantic review contract and `cookimport/llm/knowledge_stage/recovery.py` for the worker prompt text. The benchmark run that motivated this work is saved under `data/golden/benchmark-vs-golden/2026-04-14_17.18.19/single-book-benchmark/saltfatacidheatcutdown/`.

The failure is not missing structure. The saved bad rows are outside-recipe memoir and broad explanatory prose that the model treated as knowledge. Examples include:

    Salt, Fat, Acid, and Heat were the four elements that guided basic decision making in every single dish, no matter what.
    It was a revelation.
    My pursuit of flavor has continued to lead me around the world.

The intended boundary is already documented in prose: memoir, framing, and broad inspiration should stay `other`, while a later row can still be `keep_for_review` if it independently states reusable cooking guidance. The current change is to make that contrast more concrete in the active prompts and tests.
