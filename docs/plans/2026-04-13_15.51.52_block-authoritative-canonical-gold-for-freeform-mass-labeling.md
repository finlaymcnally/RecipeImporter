# Make Label Studio Benchmark Gold Block-Authoritative For Freeform Mass-Labeling

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This document must be maintained in accordance with `docs/PLANS.md`.

## Purpose / Big Picture

After this change, a human annotator can keep using broad freeform highlights in Label Studio to label many adjacent `KNOWLEDGE` or `OTHER` paragraphs at once, while the benchmark gold remains stable and block-authoritative. One long drag across five adjacent blocks must mean the same thing as five separate per-block highlights. The export should still preserve the raw freeform spans as an archive of what the annotator drew, but benchmark authority must come from derived per-row and per-block labels, not from the geometry of the freeform drag.

The shipped user-visible proof is simple. After exporting a Label Studio project and running `labelstudio-benchmark --eval-mode source-rows`, the scored gold no longer changes merely because one highlight crossed row or block boundaries differently. The benchmark output and docs now point operators at `row_gold_labels.jsonl` as the scored artifact and treat older canonical files as unsupported historical output rather than active runtime inputs.

## Surprises & Discoveries

- Observation: The current stage-block evaluator already does the desired semantic move: it reads `freeform_span_labels.jsonl`, extracts touched block indices, and treats labels as block assignments rather than preserving the exact span geometry.
  Evidence: `cookimport/bench/eval_stage_blocks.py:load_gold_block_labels(...)` builds `assignments[block_index] -> set[label]` from `touched_block_indices`.

- Observation: The intermediate canonical export path was only a migration bridge; once row-native scoring landed, keeping canonical readers alive added complexity without any active product value.
  Evidence: the final runtime uses `cookimport/bench/eval_source_rows.py`, Label Studio export writes `row_gold_labels.jsonl`, and the canonical reader modules were deleted in the cleanup pass.

- Observation: The recent benchmark confusion came from a real export rewrite, not a benchmark counter bug.
  Evidence: `data/golden/pulled-from-labelstudio/saltfatacidheatcutdown/exports/*` was rewritten at `2026-04-13 15:00:14 -0400`, and `summary.json` now reports `canonical_span_count: 1888` while the earlier benchmark artifact recorded `1881`.

- Observation: The user-visible confusion was broader than wording. As long as old canonical readers stayed in the runtime, docs and tests kept drifting back toward describing canonical files as supported inputs.
  Evidence: the cleanup removed `cookimport/bench/eval_canonical_text.py`, `cookimport/bench/canonical_alignment_cache.py`, `cookimport/labelstudio/canonical_gold.py`, and the `canonical-text` CLI alias to make the row-authoritative contract explicit in code rather than only in prose.

- Observation: Making the scored artifact truly stable across equivalent labeling styles required dropping raw span-shape provenance from the benchmark-authoritative exports. Keeping fields derived from one exact drag would still make two semantically identical exports differ on disk.
  Evidence: row-gold and block-gold outputs stay stable because they preserve row/block identity plus labels, while the raw freeform archive remains available separately for debugging.

- Observation: The existing tests were enough to protect the stage-block semantics once the loader was refactored, but the export path still needed a dedicated regression proving equivalence between one broad drag and several per-block clicks.
  Evidence: `tests/bench/test_eval_stage_blocks.py` passed unchanged after the refactor, while `tests/labelstudio/test_labelstudio_export.py::test_labelstudio_export_mass_labeling_is_block_authoritative` was added to cover the new contract directly.

## Decision Log

- Decision: Benchmark authority will become block-authoritative for all exported gold labels, not only for `KNOWLEDGE` and `OTHER`.
  Rationale: The user's stated intent is that Label Studio freeform highlighting is only an ergonomic input surface. A long drag across several blocks should mean "label these blocks", not "preserve this exact character geometry forever". A single rule for all labels is simpler, more stable, and easier to explain than special-casing only some labels.
  Date/Author: 2026-04-13 / Codex

- Decision: Raw freeform spans will remain exported and preserved as the archive of what the human drew, but they will stop being the runtime source of truth for scored benchmark inputs.
  Rationale: This keeps the raw annotation record available for debugging and future tooling without forcing benchmark scoring to inherit all of the geometric instability of freeform drag behavior.
  Date/Author: 2026-04-13 / Codex

- Decision: Introduce dedicated exported `row_gold_labels.jsonl` and `block_gold_labels.jsonl` artifacts instead of keeping the block-authoritative contract implicit inside evaluators.
  Rationale: The runtime needs one obvious, stable artifact for scored gold and one convenience artifact for block-oriented tools. Making both explicit reduces confusion and improves testability.
  Date/Author: 2026-04-13 / Codex

- Decision: The final runtime will remove canonical scoring/readers entirely instead of keeping them as compatibility seams.
  Rationale: Once row-native scoring exists end to end, canonical readers only preserve a second unsupported contract. Deleting them makes the active benchmark path obvious and prevents future drift back toward canonical artifacts.
  Date/Author: 2026-04-13 / Codex

- Decision: Operator-facing benchmark output will explicitly describe the scored row-gold files in active mode.
  Rationale: The UX should match the real scoring contract and stop implying that freeform or canonical compatibility artifacts are live scoring inputs.
  Date/Author: 2026-04-13 / Codex

- Decision: `row_gold_labels.jsonl` and `block_gold_labels.jsonl` will only carry benchmark-authoritative row/block identity plus stable source metadata, not raw freeform drag provenance such as original span ids or overlap snippets.
  Rationale: Those provenance fields made equivalent labeling styles produce different artifacts even after blockization. The raw freeform archive already preserves what the human drew; the benchmark-authoritative artifacts should stay stable and minimal.

## Outcomes & Retrospective

The original block-authoritative migration landed, and the destructive follow-up cleanup went further: active benchmark authority is now row-native end to end, Label Studio export no longer writes canonical gold files, cutdown/reviewer tooling reads `row_gold_labels.jsonl` directly, and the old canonical reader modules/CLI aliases are gone.

## Context and Orientation

This repository now has one active scored gold representation plus a few archival artifacts that may still exist inside older pulled exports.

`freeform_span_labels.jsonl` is the raw Label Studio export. Each row describes what the annotator highlighted inside one Label Studio task segment, including the exact local offsets and the list of touched source blocks. This is a good record of human input, but it is not stable when the human labels broad runs of adjacent blocks with one drag instead of several smaller drags.

`row_gold_labels.jsonl` is the active benchmark gold. Each row is keyed by stable `row_id` and carries the deduped label set assigned to that source row. This is what active benchmark scoring reads.

`block_gold_labels.jsonl` is a derived convenience artifact for block-oriented tooling and diagnostics. It uses the same block-authoritative semantics but is not the primary scored artifact.

Older pulled exports may still contain `canonical_text.txt` and `canonical_span_labels.jsonl`, but new exports do not write them and active runtime code does not read them.

The goal of this plan was to unify the interpretations around block-authoritative gold while preserving the raw freeform archive. The final implementation completed that unification by promoting row gold to the only active scored interface.

The key files and modules are:

- `cookimport/labelstudio/export.py`: owns Label Studio export artifact writing and `summary.json`.
- `cookimport/labelstudio/row_gold.py`: owns row-gold derivation and loading.
- `cookimport/bench/eval_stage_blocks.py`: provides the block-oriented evaluator surface used by legacy block comparisons.
- `cookimport/bench/eval_source_rows.py`: reads row gold during active benchmark scoring.
- `tests/labelstudio/test_labelstudio_export.py`: the main export artifact regression file.
- `tests/bench/test_eval_stage_blocks.py`: gold-loading and block-eval seam coverage.
- `docs/06-label-studio/06-label-studio_README.md` and `docs/07-bench/07-bench_README.md`: long-lived docs that define the export and scoring contracts.

In this document, "block-authoritative" means the following concrete rule: if a freeform annotation row touches block indices `101`, `102`, and `103`, then each of those source blocks receives that label for benchmark authority, regardless of whether the drag covered the whole block or only part of it.

## Final Design

The shipped design has three layers:

1. `freeform_span_labels.jsonl` remains the raw archive of what the annotator drew.
2. `row_gold_labels.jsonl` is the only active scored gold artifact.
3. `block_gold_labels.jsonl` is a deterministic convenience projection for block-oriented tooling.

New exports write row gold and block gold directly. Active benchmark runtime reads row gold through `cookimport/bench/eval_source_rows.py`. Older canonical export artifacts may still exist in previously pulled data, but they are not part of the supported runtime contract anymore.

## Validation and Acceptance

This work is considered complete because all of the following landed:

- Broad freeform sweeps and per-block clicks now converge to the same derived scored gold.
- New exports stop writing canonical gold files.
- Active benchmark and helper code read row gold rather than canonical files.
- Tests/docs/CLI wording were updated so the active contract is row-authoritative rather than canonical-authoritative.

## Idempotence and Recovery

`labelstudio-export` still overwrites the target export folder in place, so the derived `row_gold_labels.jsonl` and `block_gold_labels.jsonl` artifacts are regenerated deterministically on every export. Old pulled exports can still be migrated into row-native artifacts, but the runtime no longer promises direct canonical-file compatibility.

## Interfaces and Dependencies

The final runtime surface is:

- `cookimport/labelstudio/export.py`: writes freeform, row-gold, and block-gold artifacts.
- `cookimport/labelstudio/row_gold.py`: loads and derives row-authoritative gold.
- `cookimport/bench/eval_source_rows.py`: active benchmark scorer.
- `cookimport/bench/eval_stage_blocks.py`: block-oriented evaluator built on the same blockization semantics.

The exported row-gold row shape includes stable row identity plus labels and source metadata. The exported block-gold row shape includes stable block identity plus labels and source metadata. Neither scored artifact preserves drag-geometry provenance that would make semantically equivalent exports differ on disk.
