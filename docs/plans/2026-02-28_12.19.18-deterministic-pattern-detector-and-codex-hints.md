# Deterministic Pattern Detector And Optional Codex Hints

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This plan follows [docs/PLANS.md](/home/mcnal/projects/recipeimport/docs/PLANS.md) requirements.

## Purpose / Big Picture

After this change, EPUB and PDF ingestion will handle recurring cookbook noise patterns deterministically before candidate extraction. The importer will identify likely table-of-contents blocks, duplicated title+ramble sections, and overlapping duplicate candidates, then apply deterministic actions (`exclude_from_candidate_detection`, `trim_candidate_start`, and score penalties) while preserving all dropped text in `non_recipe_blocks`. The run will also emit explicit diagnostics (`pattern_diagnostics.json`, warnings, and scoring-debug reasons) so behavior is inspectable.

A separate, later milestone adds a strict `pattern_hints` contract field for CodexFarm pass1 input. That handoff is metadata-only and does not turn on AI parsing/cleanup in ingestion; it only allows downstream pass1 boundary refinement to see deterministic hints if and when we explicitly enable it.

## Surprises & Discoveries

- Observation: EPUB already skips explicit nav/toc documents and nav-tag content, but plain-text TOC-like runs are not currently suppressed.
  Evidence: `cookimport/plugins/epub.py` (`_should_skip_spine_document`, `_document_has_toc_nav`, `_parse_soup_to_blocks`) and `cookimport/parsing/epub_html_normalize.py`.

- Observation: Duplicate-line telemetry exists for EPUB extraction health, but it is warning-only and not connected to candidate suppression.
  Evidence: `cookimport/parsing/epub_health.py` (`duplicate_block_rate`, `epub_duplicate_block_rate_high`).

- Observation: Candidate gating already has a deterministic rejection path that preserves text in `non_recipe_blocks`.
  Evidence: `cookimport/plugins/epub.py` and `cookimport/plugins/pdf.py` loops that record rejected candidate block details and append them to `non_recipe_blocks`.

- Observation: CodexFarm process telemetry already persists `heads_up_*` details in manifests/reports, so later hint loops can attach to existing observability.
  Evidence: `cookimport/llm/codex_farm_runner.py` telemetry row normalization and `cookimport/llm/codex_farm_orchestrator.py` `process_runs` persistence.

## Context and Orientation

In this repository, a "candidate" is a block span that the importer thinks might be one recipe (`start_block`, `end_block`). Candidate detection for EPUB and PDF happens before recipe field extraction, then each candidate is scored by deterministic recipe-likeness logic in `cookimport/core/scoring.py`. Rejected candidates are converted into `non_recipe_blocks`.

Current flow locations:

- EPUB convert path: `/home/mcnal/projects/recipeimport/cookimport/plugins/epub.py` (`convert` method, `_detect_candidates`, candidate extraction loop, rejection/non-recipe collection).
- PDF convert path: `/home/mcnal/projects/recipeimport/cookimport/plugins/pdf.py` (same structural phases as EPUB).
- Recipe-likeness scoring: `/home/mcnal/projects/recipeimport/cookimport/core/scoring.py` (`score_recipe_likeness`, `build_recipe_scoring_debug_row`).
- Run report and artifact contracts: `/home/mcnal/projects/recipeimport/cookimport/core/models.py` (`ConversionReport`, `RawArtifact`).
- CodexFarm pass1 input contract: `/home/mcnal/projects/recipeimport/cookimport/llm/codex_farm_contracts.py` (`Pass1RecipeChunkingInput`).
- Pass1 prompt and pipeline metadata: `/home/mcnal/projects/recipeimport/llm_pipelines/prompts/recipe.chunking.v1.prompt.md`, `/home/mcnal/projects/recipeimport/llm_pipelines/pipelines/recipe.chunking.v1.json`.

Terms used in this plan:

- Pattern flag: a deterministic label on one block or a contiguous block cluster, such as "toc_like_cluster" or "duplicate_title_intro".
- Contiguous cluster: neighboring block indices with no gaps.
- Candidate overlap: two candidate spans whose index ranges intersect.
- Deterministic action: a non-LLM transform decided only by fixed code rules and thresholds.

## Plan of Work

### Milestone 1: Shared pre-candidate pattern detection with diagnostics only

Create a new parser utility module (for example `cookimport/parsing/pattern_flags.py`) that accepts a block stream and emits a typed diagnostics payload. In this milestone, detection runs before `_detect_candidates(...)` in EPUB/PDF, but no candidate suppression is applied yet. The output should include per-block flags, cluster summaries, and confidence-like numeric scores derived from deterministic rules only.

This milestone is complete when both importers write a `pattern_diagnostics.json` raw artifact and add a short warning summary when high-confidence clusters are detected.

### Milestone 2: Deterministic actions before and after candidate detection

Add an action phase that consumes pattern diagnostics:

- pre-candidate: provide an `excluded_indices` set so `_detect_candidates(...)` ignores flagged TOC-like clusters;
- post-candidate pre-extraction: trim candidate starts when leading narrative/title-duplication patterns are detected;
- post-scoring: resolve overlapping duplicate candidates with normalized same-title detection and keep-highest-likeness behavior.

All dropped text must remain represented in `non_recipe_blocks`, with action metadata attached in block features or location payloads.

### Milestone 3: Score penalties and scoring-debug reasons

Extend `score_recipe_likeness(...)` to apply deterministic penalties when candidate-level pattern metadata indicates TOC/duplicate-title risk. Add explicit reasons (for example `pattern_toc_like_penalty`, `pattern_duplicate_title_penalty`) and expose penalty magnitudes in `result.features`. Ensure `recipe_scoring_debug.jsonl` includes enough context to explain why a candidate was downgraded or rejected.

### Milestone 4: Regression tests and documentation updates

Add focused tests in existing ingestion/core suites:

- `tests/ingestion/test_epub_importer.py`
- `tests/ingestion/test_pdf_importer.py`
- `tests/ingestion/test_epub_extraction_quickwins.py`
- `tests/core/test_recipe_likeness_scoring.py`

The tests must prove:

- TOC-like clusters are detected and surfaced in diagnostics,
- deterministic actions change candidate behavior as intended,
- overlapping duplicates collapse deterministically,
- suppressed text still appears in `non_recipe_blocks`,
- scoring reasons include pattern penalties.

Update docs (`docs/03-ingestion/03-ingestion_readme.md`, `docs/04-parsing/04-parsing_readme.md`) to describe new behavior and artifact contracts.

### Milestone 5: Optional, default-off CodexFarm pass1 `pattern_hints` handoff

After deterministic ingestion behavior is stable, add `pattern_hints` to `Pass1RecipeChunkingInput` and update pass1 prompt instructions to treat hints as advisory context, not source truth replacement. Keep this handoff default-off through run settings (or equivalent strict gate) so ingestion remains non-LLM-cleaning by default. Update `tests/llm/test_codex_farm_contracts.py` and `tests/llm/test_codex_farm_orchestrator.py` accordingly.

## Validation and Acceptance

Acceptance criteria for deterministic ingestion work:

- On EPUB/PDF inputs containing TOC-like pages, importer emits `pattern_diagnostics.json` and warning summaries naming the detected pattern classes.
- Candidate detection ignores flagged TOC clusters and trims duplicate-title narrative prefixes when rules fire.
- Overlapping duplicate candidates with same normalized titles resolve to one kept candidate (highest recipe-likeness) and one downgraded/rejected candidate.
- Dropped/suppressed text remains present in `non_recipe_blocks` with traceable metadata.
- `recipe_scoring_debug.jsonl` includes pattern penalty reasons and penalty values.
- Existing ingestion fixture tests continue passing.

Acceptance criteria for optional future pass1 handoff:

- `Pass1RecipeChunkingInput` accepts `pattern_hints` with strict schema validation.
- Pass1 prompt allows using hints only as advisory boundary context and still requires grounding to input blocks.
- Gate remains default-off, so this does not enable AI cleanup of ingestion text.

## Idempotence and Recovery

All changes are additive and can be applied incrementally. Re-running tests and docs commands is safe. If a threshold change causes regressions, revert only threshold constants or action-application toggles and re-run targeted ingestion/core tests. If pass1 hint handoff introduces instability, disable the handoff gate and keep deterministic ingestion behavior active.
