# ExecPlan: Dashboard Previous Runs Column Controls

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This document follows [`docs/PLANS.md`](docs/PLANS.md) and is self-contained for this specific dashboard feature.

## Purpose / Big Picture

After this change, `Previous Runs` is no longer fixed to a hard-coded column set. Users can reorder columns, resize widths, and add/remove fields from discovered benchmark keys directly in the generated dashboard page.

This is visible in `data/.history/dashboard/index.html` under a new `Table columns` control panel, with dynamic headers and drag-resize handles.

## Surprises & Discoveries

- Observation: `Previous Runs` currently mixes single benchmark rows with grouped all-method summary rows, so dynamic field rendering must handle both row shapes.
  Evidence: `renderPreviousRuns()` creates `type: "single"` and `type: "all_method"` rows with different available keys.

- Observation: Header markup is generated in the HTML template; switching to dynamic columns required moving header construction into JS and keeping empty-state `colspan` dynamic.
  Evidence: old static `<th>` markup replaced by runtime-generated `renderPreviousRunsTableColumns(...)`.

## Decision Log

- Decision: Keep column customization session-local in dashboard JS state (no persisted storage).
  Rationale: User requested interactive behavior; persistence was not required and would increase scope.
  Date/Author: 2026-03-02 / Assistant.

- Decision: Reuse `collectBenchmarkFieldPaths()` as the source for addable fields.
  Rationale: Keeps add/remove options automatically aligned with collector/schema fields and existing filter-builder behavior.
  Date/Author: 2026-03-02 / Assistant.

- Decision: Implement reorder with explicit `Left`/`Right` controls and resize via header drag handles.
  Rationale: Fast to use, keyboard accessible, and minimal implementation risk inside a static dashboard.
  Date/Author: 2026-03-02 / Assistant.

## Outcomes & Retrospective

The feature shipped with dynamic column state, runtime header generation, resize handles, and test coverage. Existing filter flow and all-method grouping remained intact.

No major gaps were found during tests. Future enhancement (out of scope): persist column presets across page reloads.
