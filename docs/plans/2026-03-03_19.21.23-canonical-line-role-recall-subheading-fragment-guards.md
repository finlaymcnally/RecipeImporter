# Raise canonical line-role quality via ingredient recall, subheading disambiguation, and atomizer split guards

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

After this work, the canonical line-role pipeline should recover more true ingredient lines, stop overcalling `RECIPE_TITLE` for subsection headings, and reduce fake ingredient predictions created by quantity-fragment split artifacts. The goal is to improve both strict accuracy and macro-F1 on SeaAndSmoke single-offline codex runs without undoing the recent title/note recall gains.

A novice should be able to verify the improvement by running targeted parsing tests and then re-running the same benchmark source (`SeaAndSmokeCUTDOWN.epub`) against the current baseline run (`2026-03-03_18.31.00`) to confirm confusion-bucket reductions.

## Context and Orientation

The relevant pipeline has two deterministic stages:

1. `cookimport/parsing/recipe_block_atomizer.py` converts source blocks into `AtomicLineCandidate` rows with candidate labels and rule tags.
2. `cookimport/parsing/canonical_line_roles.py` assigns final labels with deterministic-first logic and optional codex fallback.

In this plan:

- “short quantity-led lines” means short ingredient candidates (often <=10 words) beginning with quantity patterns such as `1`, `1/2`, `8`, `2.5`.
- “subsection heading” means an internal component heading within a recipe flow that should be `HOWTO_SECTION` (for example, component titles like `SHIITAKE STOCK`) rather than top-level `RECIPE_TITLE`.
- “quantity-fragment artifact” means a split atom that begins with a quantity token but is actually the tail of instruction prose or a broken sentence fragment.

Primary files:

- `cookimport/parsing/recipe_block_atomizer.py`
- `cookimport/parsing/canonical_line_roles.py`
- `tests/parsing/test_recipe_block_atomizer.py`
- `tests/parsing/test_canonical_line_roles.py`
- optional fixture updates under `tests/fixtures/canonical_labeling/`

## Milestones

### Milestone 1: Raise short quantity-led ingredient recall without broadening false positives

Adjust atomizer/canonical ingredient heuristics so short quantity-led ingredient lines no longer fall through to `OTHER`, especially outside strict recipe spans and in component blocks. Preserve existing safeguards against instruction and yield misclassification.

### Milestone 2: Add subsection-heading guard to stop `HOWTO_SECTION -> RECIPE_TITLE` overcalls

Introduce deterministic disambiguation for heading-like all-caps rows in recipe flow. When heading context indicates a component/subsection (not a top-level recipe title), prefer `HOWTO_SECTION` ahead of `RECIPE_TITLE`.

### Milestone 3: Suppress quantity-fragment split artifacts

Harden split logic in atomizer so instruction prose is not split into quantity-start fragments that then trigger `ingredient_like`. This includes guards around split boundaries and sentence-fragment detection.

### Milestone 4: Regression proof and benchmark comparison

Add tests that reproduce the three bucket failures and prove the fixes. Re-run SeaAndSmoke single-offline benchmark and compare confusion buckets and topline metrics against `2026-03-03_18.31.00`.

## Plan of Work

First, update `recipe_block_atomizer.py` to tighten `_split_quantity_runs(...)` and add a “do-not-split” gate for instruction-like prose segments and short trailing fragments. In the same file, improve `_is_ingredient_line(...)` for short quantity-led rows by adding bounded positive signals and keeping negative checks for instruction/time/control lines.

Next, update `canonical_line_roles.py` to disambiguate title-like headings using context. Implement a helper that detects likely internal subsection headings and route these to `HOWTO_SECTION` rather than `RECIPE_TITLE` in deterministic labeling. Keep current title recall improvements by limiting this guard to component-heading contexts.

Then extend tests in parsing suites with explicit regression rows for:

- short quantity-led ingredient lines currently lost to `OTHER`,
- subsection headings currently overcalled as `RECIPE_TITLE`,
- instruction prose lines that currently split into quantity fragments and become ingredient labels.

Finally, run targeted pytest, execute the benchmark rerun on SeaAndSmoke, and compare confusion counts and topline metrics versus baseline artifacts.
