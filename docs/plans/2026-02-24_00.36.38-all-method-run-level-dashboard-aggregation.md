# Run-level all-method benchmark aggregation

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

PLANS.md is checked into the repo at `docs/PLANS.md`. This document must be maintained in accordance with that file.

## Purpose / Big Picture

The current all-method dashboard page lists one row per source/book sweep. During a mega run this makes it hard to answer the higher-level question: which configuration performs best across all books in a single run folder.

After this change, the all-method dashboard hierarchy becomes:

- run index page (one row per all-method run folder),
- run detail page (aggregated configuration leaderboard across all books + per-book rows),
- existing per-book detail page (full config stats for one source/book).

This keeps current drilldown while adding one higher aggregation level.

## Surprises & Discoveries

- Observation: all-method page generation is renderer-only and already CSV-backed; no schema/collector changes are required for hierarchy updates.
  Evidence: `cookimport/analytics/dashboard_render.py` computes groups from `DashboardData.benchmark_records` using artifact paths.

- Observation: using only run timestamp for grouping can collide across different roots, so run aggregation should key by the path prefix ending at `all-method-benchmark` and use timestamp only for display/sort.
  Evidence: all-method grouping already derives stable path keys from `artifact_dir` segments.

## Decision Log

- Decision: Keep existing per-book detail page content and filenames as-is, and add a new run-detail layer above it.
  Rationale: Matches user request for one-step-higher aggregation without removing current per-book workflow.
  Date/Author: 2026-02-24 / Codex

- Decision: Aggregate config performance by run-config hash when present, with config-name fallback.
  Rationale: Hash is more stable than slug text, but fallback keeps old rows usable.
  Date/Author: 2026-02-24 / Codex

- Decision: Rank run-level aggregated configs by breadth first (`books` covered), then practical mean, strict mean, wins, and strict precision/recall.
  Rationale: In multi-book sweeps this favors consistently present configs while still surfacing practical+strict quality.
  Date/Author: 2026-02-24 / Codex
