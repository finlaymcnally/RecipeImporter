# Shrink Main Worker Watchdog To Egregious-Only Kills

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This document must be maintained in accordance with `docs/PLANS.md`.

## Purpose / Big Picture

The previous rollback already removed path- and heredoc-based main-worker kills, but it still treated some ordinary orientation or read-only tool use as kill-worthy just because of the executable name. The user clarified the target more precisely: reasonable kills are fine, but they must be seriously egregious. After this change, the main `workspace_worker` path should still kill on clearly external-system or package-manager commands, plus clearly mutating `git` verbs, while read-only `git` inspection and other non-egregious local tooling stop being fatal.

## Decision Log

- Decision: keep kills for clearly external-system commands.
  Rationale: package managers, network fetchers, remote shells, and container/orchestration tools are still egregious enough to stop a main worker immediately.
  Date/Author: 2026-03-22 / Codex

- Decision: make `git` verb-sensitive instead of killing every `git` command.
  Rationale: `git status` / `git diff` / `git show` style inspection is not in the same risk class as `git pull`, `git reset`, or `git checkout`, and the watchdog should reflect that difference.
  Date/Author: 2026-03-22 / Codex

## Outcomes & Retrospective

Implemented. Main workspace-worker command kills are still present, but the remaining kill set is much narrower and more defensible. The shared detector now treats package-manager, network, remote-shell, container, orchestration, privilege-escalation, and mutating `git` commands as egregious enough to kill. Read-only `git` inspection no longer dies just because the executable name is `git`.
