# QualitySuite Fast Profile + Shared Prediction Reuse

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This plan follows [docs/PLANS.md](/home/mcnal/projects/recipeimport/docs/PLANS.md).

## Purpose / Big Picture

After this change, top-tier tournament runs can use a lower-workload profile by default (no deterministic sweeps, narrower race finalists, and shortlist-first experiment files), and prediction artifact reuse can persist outside one local all-method run root. Practically, this means repeated prediction work can be skipped across race rounds, experiments, and folds when run settings/source match.

## Context and Orientation

Prediction reuse is implemented in `cookimport/cli.py` in `_run_all_method_prediction_once(...)`, which computes a prediction reuse key and stores a cache entry in `.prediction_reuse_cache`. Reuse currently copies prediction artifacts by relative config directory under the same `root_output_dir`, which prevents reuse across different round/experiment/fold roots.

QualitySuite orchestration lives in `cookimport/bench/quality_runner.py`. Top-tier tournament orchestration lives in `scripts/quality_top_tier_tournament.py`, which already exports a shared canonical alignment cache env var into fold `bench quality-run` subprocesses.

Profile files for tournament configuration live under:

- `data/golden/bench/quality/thresholds/`
- `data/golden/bench/quality/experiments/`

## Plan of Work

First, add a shared prediction cache root resolver in `cookimport/cli.py` (env override plus default behavior). Extend prediction cache entry payloads so a cache hit can copy from stored absolute source artifact paths even when current run root differs. Keep legacy cache-entry compatibility by accepting old entries that only include relative config directory.

Second, make QualitySuite use a stable prediction cache root by default and expose it in resolved metadata. Then update tournament env wiring to provide a shared prediction cache root across folds.

Third, create new threshold and experiment profile JSON files for fast runs: non-sweeps, narrower race finalists/keep ratio, and shortlist-first/full-finalists file pair.

Finally, update docs and run targeted tests that cover prediction reuse behavior and quality runner regressions.

## Idempotence and Recovery

Changes are additive and safe to rerun. Cache payload changes remain backward-compatible by accepting legacy cache entries. If shared cache causes unexpected behavior, unsetting `COOKIMPORT_ALL_METHOD_PREDICTION_REUSE_CACHE_ROOT` restores prior per-run-root behavior.
