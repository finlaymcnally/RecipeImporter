# Offline Benchmark Suite for Freeform Golden Sets

Implemented 2026-02-12. See also: `docs/bench-runbook.md` for the quick-start guide.

## What This Is

The `cookimport bench` subsystem provides a fully offline benchmark loop for iterating on the EPUB/PDF parsing pipeline. A developer (or AI coding agent) can:

1. Run a benchmark suite against freeform gold exports with one command — no Label Studio required.
2. Get a ranked iteration packet explaining what the pipeline missed and why.
3. Make a code or config change, rerun with `--baseline`, and see exactly what improved or regressed.

Everything lives under `cookimport/bench/` (11 modules) with CLI commands under `cookimport bench`.

## Architecture

```
cookimport bench run --suite dev.json
         │
         ▼
    suite.py          Load + validate BenchSuite (Pydantic model)
         │
         ▼
    runner.py          For each item in suite:
         │
    ┌────┴────┐
    │         │
pred_run.py  │        Build offline predictions (calls ingest.py's
    │         │        generate_pred_run_artifacts — no LS dependency)
    │         │
    │    eval_freeform  Evaluate pred vs gold using existing freeform eval
    │    (existing)     (evaluate_predicted_vs_freeform from labelstudio/)
    │         │
    └────┬────┘
         │
    report.py          Aggregate metrics across items, write report.md
         │
    packet.py          Build iteration_packet/ with ranked failure cases
```

### Module Inventory

| Module | Purpose |
|--------|---------|
| `suite.py` | `BenchSuite` / `BenchItem` Pydantic models; `load_suite()`, `validate_suite()` |
| `pred_run.py` | `build_pred_run_for_source()` — thin offline wrapper around `generate_pred_run_artifacts()` |
| `runner.py` | `run_suite()` — orchestrates pred-run + eval for each item, writes aggregate results |
| `report.py` | `aggregate_metrics()`, `format_suite_report_md()` — per-label rollup, weighted recall/precision |
| `packet.py` | `build_iteration_packet()` — severity-ranked failure cases with block text context |
| `trace.py` | `TraceCollector` — JSONL event recording; writes `trace.jsonl` per bench run |
| `knobs.py` | `Tunable` model, `KNOB_REGISTRY`, `effective_knobs()`, `validate_knobs()` |
| `sweep.py` | `run_sweep()` — seeded random search over knob values with leaderboard output |
| `noise.py` | `dedupe_predictions()`, `consolidate_predictions()`, `gate_noise()` — applied per-item during bench runs |
| `cost.py` | `estimate_llm_costs()` (counting only, no LLM calls), `write_escalation_queue()` — emits per-item + aggregate artifacts |
| `__init__.py` | Empty package marker |

### Key External Dependency

The bench suite reuses the **existing freeform eval contracts** from `cookimport/labelstudio/eval_freeform.py`. No parallel scoring format was created. This means:

- Gold data: `exports/freeform_span_labels.jsonl` + `exports/freeform_segment_manifest.jsonl`
- Evaluation: `evaluate_predicted_vs_freeform()` with Jaccard IoU overlap on block-index sets
- Eval outputs: `eval_report.json`, `eval_report.md`, `missed_gold_spans.jsonl`, `false_positive_preds.jsonl`

## How the Offline Pred-Run Works

The core extraction logic in `cookimport/labelstudio/ingest.py` was refactored to separate artifact generation from Label Studio upload:

- **`generate_pred_run_artifacts()`** (new, extracted) — handles extraction, conversion, archive building, task generation, and artifact writing. Returns a dict with `run_root`, `tasks_total`, `manifest_path`, etc. Does not require LS env vars, does not upload.
- **`run_labelstudio_import()`** (existing, modified) — now calls `generate_pred_run_artifacts()` first, then layers on LS-specific upload logic (project creation, task upload, manifest update with LS fields).

`bench/pred_run.py` wraps `generate_pred_run_artifacts()` with bench-specific defaults: `task_scope="pipeline"`, `chunk_level="both"`, and knob values from the config dict.

## Run Directory Structure

Each `cookimport bench run` produces:

```
data/golden/bench/runs/<timestamp>/
  suite_used.json           # Exact suite definition (reproducibility)
  report.md                 # Aggregate recall/precision report
  metrics.json              # Machine-readable aggregate metrics
  knobs_effective.json      # Config values used
  cost_summary.json         # Aggregate LLM cost estimates across all items
  trace.jsonl               # Decision trace events for the run
  iteration_packet/
    README.md               # How to use the packet
    summary.md              # Metric deltas, per-item overview
    cases.jsonl             # Ranked failure cases (severity-sorted)
    top_failures.md         # Top N failures with block text context
  per_item/<item_id>/
    pred_run/               # Generated prediction artifacts
    noise_stats.json        # Prediction dedup/consolidation statistics
    eval_freeform/          # Evaluation results vs gold
      cost_estimate.json    # Per-item LLM cost estimate
      escalation_queue.jsonl # Predictions for LLM review
```

## Iteration Packet Design

The packet is designed to be directly consumable by an AI coding agent. Each case in `cases.jsonl` is self-contained:

```json
{
  "case_type": "missed_gold",
  "item_id": "saltfatacidheat-cutdown",
  "label": "INGREDIENT_LINE",
  "gold_range": [10, 15],
  "pred_range": null,
  "block_text_window": ["block 8 text", "block 9 text", "..."],
  "severity": 0.95
}
```

Severity is computed as `label_importance * case_type_weight`:
- Label importance: RECIPE_TITLE=1.0, INGREDIENT_LINE=0.95, INSTRUCTION_LINE=0.90, down to OTHER=0.15
- Case type weight: missed_gold=1.0, false_positive=0.7

When a `--baseline` run is provided, `summary.md` includes metric deltas (recall, precision, gold_matched).

## Knob System

Three tunable parameters are registered in `KNOB_REGISTRY`:

| Knob | Type | Default | Bounds | Description |
|------|------|---------|--------|-------------|
| `segment_blocks` | int | 40 | [10, 100] | Blocks per freeform segment task |
| `segment_overlap` | int | 5 | [0, 20] | Overlapping blocks between segments |
| `workers` | int | 1 | [1, 8] | Parallel conversion workers |

Config is loaded from a JSON file, merged over defaults. Bounds are validated; out-of-range values produce clear errors.

## CLI Commands

```
cookimport bench validate --suite <path>       # Check source + gold files exist
```


## Key Design Decisions

**Reuse existing eval contracts, don't invent a parallel format.**
The freeform eval in `eval_freeform.py` already produces exactly the artifacts the bench loop needs (eval_report.json, missed_gold_spans.jsonl, false_positive_preds.jsonl). Building on top of this avoids schema drift and means improvements to eval automatically flow into bench.

**Block indices as the stable coordinate system.**
Freeform gold spans intentionally don't mirror internal chunk boundaries. Scoring uses touched block-index sets (Jaccard IoU overlap) which rewards "picked it up in the right region" regardless of chunking strategy.

**Precision reported with caveats; prediction density as primary noise metric.**
Unlabeled regions in freeform gold are not guaranteed negatives. Treating all unlabeled text as negative makes precision misleading. The suite reports prediction density (preds per gold span) as a more actionable cost-control metric.

**Offline-first: no LS dependency in the bench path.**
The `generate_pred_run_artifacts()` extraction ensures the bench loop never initializes a Label Studio client. This was achieved by cleanly splitting artifact generation from upload in `ingest.py`. The refactor preserved all 27 existing labelstudio tests.

**Output under `data/golden/bench/`** (not a new top-level directory).
Golden/eval artifacts already live under `data/golden/`; keeping bench runs adjacent reduces path confusion.

**Severity-ranked iteration packet for agent consumption.**
The packet is designed so an AI agent can read `top_failures.md` and immediately identify the highest-impact fix. Each case includes the surrounding block text (±2 context window) so the agent has enough context without needing to load the full archive.

## Known Footguns (and How They're Handled)

1. **Source identity mismatch → near-zero eval overlap.** Suite items carry `force_source_match` (default false). When false, eval fails fast on mismatch. When true, it's recorded prominently.

2. **Split-job block-index rebasing regression.** Offline pred-run reuses the same merge/rebase logic as the stage/import path via `generate_pred_run_artifacts()`.

3. **Unlabeled ≠ negative in freeform gold.** Recall is the primary metric. Precision is reported but explicitly de-emphasized. Prediction density provides a more honest noise signal.

4. **ID prefix drift** (`urn:cookimport:*` vs `urn:recipeimport:*`). New bench code doesn't hardcode ID prefixes — it keys off stable fields (source hash, block index).

## Surprises During Implementation

- **`run_labelstudio_import()` was cleanly separable.** The monolith mixed offline artifact generation with LS upload, but the boundary was clean enough that extracting `generate_pred_run_artifacts()` required zero changes to existing callers or tests.

- **Existing eval produced exactly what the packet needed.** `evaluate_predicted_vs_freeform()` already returns `missed_gold` and `false_positive_preds` as dicts with block indices — no new eval code was needed.
