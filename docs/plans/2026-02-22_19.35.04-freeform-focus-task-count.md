# ExecPlan: Add freeform context/focus controls with target task count tuning

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This plan must be maintained in accordance with `docs/PLANS.md`.

## Purpose / Big Picture

After this change, an operator can keep roughly the same reading context per freeform task while asking the AI prelabeler to label a smaller subset of blocks per task, and can target an approximate number of tasks directly from the same CLI settings flow.

User-visible behavior: in Label Studio freeform import, the operator can set (a) context blocks shown per task, (b) focus blocks to label, and (c) optional target task count (for example `55`). The system auto-tunes effective overlap to land near the requested task count, records effective settings in manifests, and constrains AI prelabels to focus blocks while still seeing broader context.

## Surprises & Discoveries

- Observation: freeform task count is controlled entirely by `segment_blocks` and `segment_overlap` through `_segment_ranges(...)` in `cookimport/labelstudio/freeform_tasks.py`, with strict invariant `segment_overlap < segment_blocks`.
  Evidence: current `_segment_ranges` raises `ValueError` when overlap is `>= segment_blocks`; task count emerges from step size `segment_blocks - segment_overlap`.

- Observation: current prelabel prompts include every block in a segment and block-mode prompt explicitly asks for one label per block, so there is no built-in concept of “label only subset while reading all context.”
  Evidence: `_build_prompt` in `cookimport/labelstudio/prelabel.py` renders `{{BLOCKS_JSON_LINES}}` for all segment blocks and full prompt text asks for exhaustive block coverage.

- Observation: in this codebase, command functions called directly from tests/helpers must use plain Python defaults (typically `Annotated[..., typer.Option(...)] = ...`) or direct calls receive `OptionInfo` objects.
  Evidence: `tests/test_labelstudio_benchmark_helpers.py::test_labelstudio_import_prints_processing_time` failed with `TypeError: int() argument must be ... not 'OptionInfo'` until the new options were converted to `Annotated` form.

## Decision Log

- Decision: introduce two explicit knobs for freeform scope: `segment_focus_blocks` (how many blocks to label per task) and `target_task_count` (optional, auto-tunes overlap), while preserving existing `segment_blocks` as context size.
  Rationale: this directly maps to user intent: keep context broad, narrow labeling burden, and nudge task count upward without manual overlap guesswork.
  Date/Author: 2026-02-22 / Codex

- Decision: preserve backward compatibility by defaulting `segment_focus_blocks` to `segment_blocks` and `target_task_count` to unset.
  Rationale: existing workflows must remain unchanged unless new knobs are explicitly set.
  Date/Author: 2026-02-22 / Codex

- Decision: enforce focus constraints at runtime (prompt instruction plus parser-side filtering), not just by prompt wording.
  Rationale: model output can drift; parser-side filtering guarantees contract correctness.
  Date/Author: 2026-02-22 / Codex

- Decision: resolve task-count tuning ties by leaning in the requested direction (higher overlap when targeting more tasks, lower overlap when targeting fewer), then by overlap distance from the requested overlap.
  Rationale: this keeps tuning deterministic while preserving operator intent and minimizing unexpected jumps.
  Date/Author: 2026-02-22 / Codex

## Context and Orientation

The freeform Label Studio task path spans four modules:

`cookimport/cli.py` handles both interactive and non-interactive CLI parameters. Freeform flows now collect/route `segment_blocks`, `segment_overlap`, `segment_focus_blocks`, and optional `target_task_count`.

`cookimport/labelstudio/ingest.py` orchestrates conversion, freeform task generation, optional AI prelabeling, artifact writing, and upload. It now resolves effective overlap via `resolve_segment_overlap_for_target(...)`, passes focus settings into task generation, and persists requested/effective values in `manifest.json`.

`cookimport/labelstudio/freeform_tasks.py` generates freeform segments and offsets. `_segment_ranges` still defines task ranges, and new helper `resolve_segment_overlap_for_target(...)` selects deterministic effective overlap for a requested task target.

`cookimport/labelstudio/prelabel.py` renders freeform prompts and parses outputs. It now includes focus scope instructions in prompts and enforces focus bounds parser-side for both block and span granularity modes.

In this plan, “context blocks” means blocks visible to labeler/model in a task, and “focus blocks” means blocks that should receive labels in that task. Context can be larger than focus.

## Plan of Work

Milestone 1 introduces new freeform settings plumbing without changing defaults. Add non-interactive CLI options `--segment-focus-blocks` and `--target-task-count` in `cookimport/cli.py`, plus interactive prompts in the existing freeform settings sequence. The interactive prompts should remain in the same branch where `segment_blocks` and `segment_overlap` are currently collected, so users can tune all related knobs together. Blank target value means “disabled.” Validation at this stage proves arguments route into `run_labelstudio_import(...)` unchanged.

Milestone 2 adds deterministic overlap auto-tuning and focus metadata in task generation. Implement helper logic in `cookimport/labelstudio/freeform_tasks.py` to compute effective overlap when target task count is provided, based on total extracted blocks and segment size, choosing the overlap whose resulting task count is closest to target while respecting `0 <= overlap < segment_blocks`. In `cookimport/labelstudio/ingest.py`, compute and record both requested and effective values. Extend task `source_map` with focus range fields so downstream prelabeling can enforce focus-only labeling.

Milestone 3 constrains AI prelabeling to focus blocks while preserving full context. Update prompt rendering in `cookimport/labelstudio/prelabel.py` (and prompt template files in `llm_pipelines/prompts/`) to include focus instructions and focus block listing. Block mode should request exactly one label per focus block, not per context block. Span mode should request spans only from focus blocks. Add parser-side filtering so out-of-focus selections are dropped deterministically, including absolute span selections that cross outside focus blocks.

Milestone 4 completes tests and docs. Extend freeform task tests to validate effective overlap computation and focus metadata. Extend interactive tests to validate new prompt sequence and captured kwargs. Extend ingest/prelabel tests to validate focus enforcement and manifest fields. Update `docs/02-cli/02-cli_README.md`, `docs/06-label-studio/06-label-studio_README.md`, and `docs/IMPORTANT CONVENTIONS.md` so operator guidance matches implementation.

## Validation and Acceptance

Acceptance is behavior-based:

For a freeform run configured with context `40`, focus `< 40` (for example `28`), and target task count `55`, generated tasks must still include context windows near 40 blocks (except edge segments), while prelabeled outputs only include labels anchored to focus blocks.

When target task count is provided, manifest/report surfaces must show both requested target and effective overlap used for this file, and task count should be near requested target (closest valid overlap solution for the extracted block count).

When target is not provided and focus equals context, behavior must match today’s defaults, including task IDs and prompt semantics.

Interactive mode must expose the new knobs in the freeform section without removing existing controls.

## Idempotence and Recovery

All changes are additive and safe to rerun. If target tuning behaves unexpectedly, operators can disable it by leaving target blank (interactive) or omitting `--target-task-count` (non-interactive), returning to manual overlap.

If focus filtering is too strict for a run, operators can set focus equal to context to recover legacy semantics without code rollback.

If prompt changes produce regressions, the deterministic parser-side focus filter remains the safety boundary and can be verified via unit tests independently of model quality.
