# Fix pipelined source-rows benchmark artifact handoff

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

Interactive single-book benchmark runs should be able to finish a `source-rows` evaluation after prediction succeeds. Right now the benchmark can crash at evaluation time because the pipelined replay bundle only reconstructs block-level artifacts, while `source-rows` scoring still needs row-level line-role predictions. After this change, both vanilla and Codex-backed single-book runs should evaluate against the authoritative row-backed prediction artifacts instead of failing on the replay folder layout, and the test suite should prove that behavior directly.

## Decision Log

- Decision: Fix the runtime by keeping `source-rows` evaluation on the authoritative prediction bundle paths instead of trying to synthesize new row-level replay artifacts.
  Rationale: The authoritative bundle already has the correct row-backed files and matches the documented scoring contract. Restricting replay usage to compatible modes is smaller, clearer, and less fragile than inventing another replay artifact shape.

## Outcomes & Retrospective

The fix is small and targeted: pipelined replay artifacts still support `stage-blocks` evaluation, but `source-rows` now evaluates against the authoritative row-backed prediction pointers from the prediction bundle. That preserves the real row-level files for both vanilla and Codex-backed single-book runs, which addresses the exact failure observed in the interactive benchmark.

The main lesson is that the benchmark helper tests already touched this seam, but they were asserting the wrong contract. The durable protection is therefore a better expectation, not a larger synthetic end-to-end harness.

## Context and Orientation

The active benchmark entrypoint is `cookimport/cli_commands/labelstudio.py`. Its `labelstudio_benchmark(...)` command can run prediction generation and evaluation together. In pipelined mode, prediction records stream through `run_pipelined(...)`, which can build a temporary replay bundle under `.prediction-record-replay/` from streamed stage-block records. That replay bundle is safe for block-level evaluation because it reconstructs `stage_block_predictions` and `extracted_archive`, but it does not reconstruct row-level line-role prediction files.

The `source-rows` evaluator lives in `cookimport/bench/eval_source_rows.py`. It still resolves row predictions relative to the scored stage artifact path by looking for files such as `row_label_predictions.jsonl` and `semantic_line_role_predictions.jsonl`. That means a replay folder that only has block-level files is incomplete for `source-rows` evaluation.

The benchmark helper regression coverage lives in `tests/labelstudio/test_labelstudio_benchmark_helpers_eval_payload_pipelined.py`. That file already exercises the pipelined path, so it is the right home for the regression instead of adding a second disconnected smoke test.

## Plan of Work

Update `cookimport/cli_commands/labelstudio.py` in the evaluation setup block where `evaluation_stage_predictions_path` and `evaluation_extracted_archive_path` are chosen. Keep the replay override for block-oriented evaluation, but leave `source-rows` on the authoritative `prediction_bundle` pointers so the evaluator can find the row-level artifacts that already exist in the prediction run.

Then revise the pipelined benchmark tests in `tests/labelstudio/test_labelstudio_benchmark_helpers_eval_payload_pipelined.py`. Replace the stale expectation that `source-rows` uses the replay bundle with the correct expectation that it uses the line-role-backed stage pointer from the prediction bundle. Add one explicit regression that would fail if replay-only `.from_records` inputs are used for `source-rows` again.

Finally, update the benchmark docs in `docs/07-bench/07-bench_README.md` with a short note explaining that pipelined replay artifacts are block-level helpers and that `source-rows` keeps the authoritative row-backed stage pointer.
