# Data layout refactor: golden workflow buckets and shared history root

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

PLANS.md is checked into the repo at `docs/PLANS.md`. This document is maintained in accordance with that file.

## Purpose / Big Picture

After this change, `data/` is organized by workflow intent instead of mixed roots. Label Studio task-generation runs, label exports, and benchmark/eval runs have dedicated golden subfolders. Shared history and dashboard artifacts move from `data/output/.history` to `data/.history`. Users can verify this by running interactive Label Studio + dashboard actions and confirming the new folders are populated.

## Surprises & Discoveries

- Observation: Tests monkeypatch `cli.DEFAULT_GOLDEN` and expect interactive flows to follow that value.
  Evidence: interactive Label Studio/benchmark tests failed until runtime subpaths were derived from `DEFAULT_GOLDEN` dynamically (instead of using import-time frozen subfolder constants).

- Observation: Existing dashboard tests and some local environments still materialize legacy `output/.history`.
  Evidence: stage/dashboard collector tests initially returned zero stage rows until collector fallback logic accepted legacy CSV location.

## Decision Log

- Decision: Keep `DEFAULT_GOLDEN` as the root constant, but route command defaults to dedicated subfolders (`sent-to-labelstudio`, `pulled-from-labelstudio`, `benchmark-vs-golden`).
  Rationale: preserves one canonical golden root while cleaning runtime artifact organization.
  Date/Author: 2026-02-23 / Codex

- Decision: Make `data/.history` the canonical history root and keep legacy read fallback for collector/settings loader.
  Rationale: allows immediate folder cleanup without breaking older artifacts or tests that still seed legacy locations.
  Date/Author: 2026-02-23 / Codex

## Outcomes & Retrospective

The data tree now reflects workflow intent, and dashboard/history artifacts no longer live under `data/output`. The main risk area was compatibility for existing local runs/tests; this was mitigated with targeted fallback reads and test updates. Documentation now states the new root contracts explicitly.
