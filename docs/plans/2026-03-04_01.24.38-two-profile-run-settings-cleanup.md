# Remove legacy interactive run-settings surfaces after two-profile simplification

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

Maintain this document in accordance with `docs/PLANS.md` from the repository root.

## Purpose / Big Picture

After this change, interactive run setup is cleanly centered on two options only: `CodexFarm automatic top-tier` and `Vanilla automatic top-tier`. Legacy chooser/store/editor branches (`global defaults`, `last run`, `preferred profile`, full-screen toggle editor, and dead interactive all-method route) are removed so docs, tests, and runtime behavior match.

## Surprises & Discoveries

- Observation: `save_last_run_settings(...)` was still called by interactive import/benchmark even though no runtime path reads those snapshots anymore.
  Evidence: search before edit showed only write callsites in `cookimport/cli.py` and no non-test load callsites.

- Observation: interactive benchmark still had an `all_method` branch despite no menu entry exposing that mode.
  Evidence: mode chooser choices omitted `all_method` while a large `elif benchmark_mode == "all_method"` block remained.

- Observation: two single-profile scheduler tests assumed one run per book from default `RunSettings()`, but defaults are codex-enabled and now generate paired `vanilla` + `codexfarm` runs.
  Evidence: focused pytest run showed expected call counts `2/1` vs actual `4/2` until tests explicitly set `llm_recipe_pipeline=off`.

## Decision Log

- Decision: keep `qualitysuite_winner_run_settings` persistence but remove last/preferred snapshot persistence.
  Rationale: winner profile is still consumed by the codex top-tier branch; last/preferred snapshots were dead runtime complexity.
  Date/Author: 2026-03-04 / assistant

- Decision: remove dead interactive all-method routing only from interactive menu flow, not non-interactive all-method machinery.
  Rationale: preserves bench/all-method command surface while aligning interactive behavior to current product scope.
  Date/Author: 2026-03-04 / assistant

- Decision: retire full-screen toggle editor implementation and tests.
  Rationale: two-profile chooser no longer uses manual per-run editor, and keeping it created stale docs and dead API pressure.
  Date/Author: 2026-03-04 / assistant

## Outcomes & Retrospective

Completed in one pass with targeted regression fixes. Interactive run-profile selection is now consistently two-option, legacy persistence/editor paths are removed, and docs/tests reflect the current contract. Non-interactive all-method benchmark machinery was intentionally left intact.

## Context and Orientation

Interactive run profile selection is implemented in `cookimport/cli_ui/run_settings_flow.py` and called from import/benchmark menu handlers in `cookimport/cli.py`. Quality-suite winner profile persistence lives in `cookimport/config/last_run_store.py`. The old full-screen editor was in `cookimport/cli_ui/toggle_editor.py` with tests in `tests/cli/test_toggle_editor.py`. Docs currently spread behavior notes across `README.md`, `cookimport/CONVENTIONS.md`, `cookimport/config/*`, and `docs/02-cli/02-cli_README.md`.

## Plan of Work

Implement in four slices: (1) runtime chooser/callsite simplification and dead-branch removal, (2) persistence helper cleanup, (3) test alignment, and (4) docs updates for user-facing and convention-level behavior. Keep all-method non-interactive features untouched.

## Concrete Steps

1. Edit `cookimport/cli_ui/run_settings_flow.py` to drop unused chooser params and remove placeholder no-op assignments.
2. Edit `cookimport/cli.py` to stop passing removed chooser params, remove `save_last_run_settings(...)` writes, and delete unreachable interactive `all_method` branch.
3. Edit `cookimport/config/last_run_store.py` and `cookimport/config/__init__.py` to keep only qualitysuite winner load/save APIs.
4. Remove `cookimport/cli_ui/toggle_editor.py` and `tests/cli/test_toggle_editor.py`, and align tests that referenced removed APIs.
5. Update docs in `README.md`, `cookimport/CONVENTIONS.md`, `cookimport/config/README.md`, `cookimport/config/CONVENTIONS.md`, and selected docs under `docs/` to reflect current two-profile interactive behavior.
6. Run focused tests from `.venv`.
