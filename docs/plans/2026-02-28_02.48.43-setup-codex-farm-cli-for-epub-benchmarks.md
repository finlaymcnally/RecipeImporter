# Setup `codex-farm` CLI for EPUB Benchmark Runs

This plan is a living document. Keep it short and practical.

## Purpose / Big Picture

After this, `cookimport` benchmark runs can actually invoke Codex Farm (the external `codex-farm` CLI) to run the 3-pass recipe pipeline against EPUB-derived block streams. This is required because `cookimport` shells out to `codex-farm` when `llm_recipe_pipeline=codex-farm-3pass-v1` is enabled.

## Steps

1. Install `codex-farm` somewhere on this machine.

   If you keep the sibling repo `../shared/CodexFarm`, install it into its own venv:

       cd ../shared/CodexFarm
       python3 -m venv .venv
       source .venv/bin/activate
       pip install -e ".[dev]"
       codex-farm doctor

2. Point `cookimport` at that executable.

   For one-off runs, pass `--codex-farm-cmd` (use an absolute path):

       export COOKIMPORT_ALLOW_CODEX_FARM=1
       cookimport bench speed-run ... --include-codex-farm \
         --run-settings-file <path.json> \
         # in <path.json>, set:
         #   "llm_recipe_pipeline": "codex-farm-3pass-v1",
         #   "codex_farm_cmd": "/abs/path/to/codex-farm"

   Or set `codex_farm_cmd` in `cookimport.json` so it applies to future runs.

3. Verify pipeline-pack roots.

   If you want to use this repo’s pipelines, set:

   - `codex_farm_root` to `<recipeimport_repo>/llm_pipelines`
   - `codex_farm_pipeline_pass1/2/3` to `recipe.chunking.v1` / `recipe.schemaorg.v1` / `recipe.final.v1`

## Acceptance

- `cookimport stage <epub> --llm-recipe-pipeline codex-farm-3pass-v1` fails fast with a clear `codex-farm` invocation error if Codex Farm is missing, and proceeds past the normalization gate when `COOKIMPORT_ALLOW_CODEX_FARM=1` is set.
- `cookimport bench speed-run --scenarios benchmark_all_method_multi_source --include-codex-farm` records `include_codex_farm_requested=true` and `include_codex_farm_effective=true` in the all-method report when the env gate is set.
