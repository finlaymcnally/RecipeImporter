# Add "Select all golden sets" to interactive all-method benchmark

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

PLANS.md is checked into the repo at `docs/PLANS.md`. This document must be maintained in accordance with that file.

## Purpose / Big Picture

Today, interactive all-method benchmark runs permutations for one selected freeform gold export and one source file. You want a bulk mode: choose all golden sets, then run the same permutation sweep for every input file that has a matching gold set.

After this change, interactive all-method benchmark will support two scopes: single golden set (current behavior) and all matched golden sets. In all-matched mode, the CLI will discover freeform exports, match them to source files in `data/input`, show exactly how many source/gold pairs and total permutations will run, and then execute all runs with one combined summary report.

User-visible proof:

- Interactive benchmark shows an all-golden-sets scope option under all-method mode.
- Selecting that option prints matched and skipped counts before execution.
- The run writes per-source all-method reports plus a top-level multi-source summary report.

## Surprises & Discoveries

- Observation: The current all-method implementation is single-pair only and starts by calling `_resolve_benchmark_gold_and_source(...)` once.
  Evidence: `cookimport/cli.py:_interactive_all_method_benchmark(...)` resolves one `(gold_spans, source_file)` tuple, then calls `_run_all_method_benchmark(...)` once.

- Observation: `_infer_source_file_from_freeform_gold(...)` reads source from run manifest first, then from the first non-empty line of `freeform_span_labels.jsonl`.
  Evidence: `cookimport/cli.py` implementation at `_infer_source_file_from_freeform_gold`.

- Observation: If a freeform export has no labeled rows, first-line inference can fail even if `freeform_segment_manifest.jsonl` still contains source metadata.
  Evidence: Current inference helper does not read `freeform_segment_manifest.jsonl`.

## Decision Log

- Decision: Add a first-class all-method scope chooser with two values: `single` (current behavior) and `all_matched` (new behavior).
  Rationale: This keeps current UX intact while adding explicit bulk execution, so users can pick small/quick or full sweep intentionally.
  Date/Author: 2026-02-23 / Codex

- Decision: Match gold exports to input files by source filename from export metadata (`source_file`) against top-level importable files in `data/input`.
  Rationale: This aligns with existing interactive file discovery and avoids ambiguous stem-based guesses.
  Date/Author: 2026-02-23 / Codex

- Decision: Keep all-method per-source report format unchanged and add one new top-level multi-source summary report.
  Rationale: Existing dashboard/report consumers keep working, while reviewers get one artifact to inspect overall bulk-run outcomes.
  Date/Author: 2026-02-23 / Codex

- Decision: Keep recipe Codex Farm permutations disabled in this feature (no policy change).
  Rationale: Project policy explicitly keeps recipe LLM pipeline off; this feature is breadth (more books), not policy unlock.
  Date/Author: 2026-02-23 / Codex

## Plan of Work

Milestone 1 adds a bulk scope selector to the interactive all-method flow. Milestone 2 introduces a robust matcher that turns discovered freeform exports into matched source/gold pairs (plus skipped reasons). Milestone 3 adds a multi-source orchestrator that calls the existing single-source all-method runner per pair and writes a combined report. Milestone 4 adds tests for matching and routing regressions. Milestone 5 updates docs and understandings.

### Milestone 1: Add all-method scope selection

In `cookimport/cli.py`, update `_interactive_all_method_benchmark(...)` so it no longer immediately resolves one gold/source pair. Instead, prompt for scope:

- `Single golden set` (existing behavior)
- `All golden sets with matching input files`

For `single`, reuse existing resolution behavior.
For `all matched`, call new matching helper(s) from Milestone 2.

Acceptance for Milestone 1:

- Interactive all-method path shows scope prompt.
- Selecting `single` keeps previous behavior.

### Milestone 2: Build robust gold-to-source matcher for bulk mode

Add helper types/functions in `cookimport/cli.py` (or an adjacent helper module if preferred):

- `AllMethodTarget` dataclass (minimum fields):
  - `gold_spans_path: Path`
  - `source_file: Path`
  - `source_file_name: str`
  - `gold_display: str`

- `AllMethodUnmatchedGold` dataclass:
  - `gold_spans_path: Path`
  - `reason: str`
  - `source_hint: str | None`

- `_load_source_hint_from_gold_export(gold_spans_path: Path) -> str | None`
  - read source hint in this order:
    1. existing manifest path inference (current behavior)
    2. first non-empty row from `freeform_span_labels.jsonl` (`source_file`)
    3. first non-empty row from sibling `freeform_segment_manifest.jsonl` (`source_file`)

- `_resolve_all_method_targets(output_dir: Path) -> tuple[list[AllMethodTarget], list[AllMethodUnmatchedGold]]`
  - discover freeform exports via `_discover_freeform_gold_exports(...)`
  - discover importable files via `_list_importable_files(DEFAULT_INPUT)`
  - exact filename match (`Path.name`) against `source_hint`
  - if no match, emit unmatched entry with reason
  - deterministic ordering: newest gold first or sorted by display path; keep ordering stable and documented

Before run execution, print:

- matched pairs count
- skipped/unmatched count
- total variant count across all matched pairs
- short skipped summary (first few reasons)

Then prompt `Proceed with N benchmark runs across M matched golden sets?` default `No`.

Acceptance for Milestone 2:

- Bulk mode can discover multiple pairs without extra manual source prompts.
- Empty-labeled exports still resolve when segment manifest has source metadata.
- Unmatched gold exports are reported clearly and skipped safely.

### Milestone 3: Multi-source orchestration and aggregate report

Add multi-source orchestration that wraps existing per-source all-method runs:

- New helper: `_run_all_method_benchmark_multi_source(...)` (name can vary)
  - input: matched targets, selected run settings, eval root, processed root, overlap/source-match flags, codex settings
  - for each target:
    - build variants with `_build_all_method_variants(...)`
    - run `_run_all_method_benchmark(...)` once per target
    - collect per-source result paths + winner metrics
  - continue after per-source failures; record failures in summary

Output layout under one interactive benchmark timestamp root:

- existing per-source roots unchanged:
  - `<benchmark_eval_output>/all-method-benchmark/<source_slug>/...`
- new top-level combined report:
  - `<benchmark_eval_output>/all-method-benchmark/all_method_benchmark_multi_source_report.json`
  - `<benchmark_eval_output>/all-method-benchmark/all_method_benchmark_multi_source_report.md`

Combined report minimum fields:

- `created_at`
- `matched_target_count`
- `unmatched_target_count`
- `total_config_runs_planned`
- `total_config_runs_completed`
- per-source entries:
  - source file
  - gold path
  - per-source report path
  - winner metrics (precision/recall/f1)
  - status (`ok`/`failed`)
- unmatched entries with reasons

Console completion summary:

- successful sources / total matched sources
- successful config runs / total planned config runs
- path to combined report

Acceptance for Milestone 3:

- One command runs all permutations for all matched golden sets.
- Per-source reports still exist exactly where expected.
- Combined report exists and summarizes both successful and skipped/failed work.

## Validation and Acceptance

The feature is accepted when all conditions are true:

1. Interactive all-method offers both scopes (single and all matched).
2. All-matched scope runs every permutation for every matched gold/source pair without manual source selection per pair.
3. Gold exports with no labeled rows can still match via segment manifest source metadata.
4. Unmatched gold exports are skipped with explicit reason (not silent).
5. Per-source reports still exist (`all_method_benchmark_report.json/.md` under each source slug).
6. Combined report exists at all-method root with accurate matched/unmatched and success/failure counts.
7. Targeted labelstudio benchmark helper tests pass, and smoke pass remains green.

## Idempotence and Recovery

- Run roots remain timestamped (`YYYY-MM-DD_HH.MM.SS`) so repeated runs do not overwrite prior runs.
- Bulk mode should be safe to rerun; each run writes a fresh timestamped benchmark folder.
- Failures in one source should not abort remaining sources; summary must show partial completion clearly.
- No data import parsing policy changes: keep `llm_recipe_pipeline=off` and do not introduce any LLM-based parsing/cleaning behavior.
