# I3.1 — Unstructured-first EPUB Block Extraction

## What This Is

EPUB ingestion uses [Unstructured](https://github.com/Unstructured-IO/unstructured)'s HTML partitioner (v0.18.32) to convert each spine document into semantically typed Blocks. This replaces the legacy BeautifulSoup tag-walking approach with richer element classification, explicit traceability metadata, and a persisted diagnostics artifact.

The legacy extractor is still available as a fallback.

## Why

The legacy `_parse_soup_to_blocks` method walks HTML block tags (`p`, `div`, `h1`-`h6`, `li`, `td`, `th`, `blockquote`) and emits `BlockType.TEXT` for everything. This loses semantic information that downstream systems need:

- **Segmentation** — recipes get chopped at the wrong places because headings, list items, and narrative prose all look the same to the legacy extractor.
- **Knowledge chunking** — technique/intro chapters become undifferentiated prose; the chunker can't distinguish "knowledge" from "noise."
- **Debugging** — when boundaries shift, there's no element-level provenance to diff.

Unstructured's partitioner classifies elements as `Title`, `NarrativeText`, `ListItem`, `Table`, etc., with hierarchy metadata (`category_depth`, `parent_id`). This gives us better heading levels, list detection, and a richer signal set for downstream processing.

## How It Works

### Data Flow

```
EPUB file
  │
  ├─ ebooklib or ZIP extraction → spine HTML bytes
  │
  ▼
partition_html_to_blocks(html, spine_index, source_location_id)
  │  (cookimport/parsing/unstructured_adapter.py)
  │
  ├─► list[Block]               — with traceability features
  └─► list[dict]                — diagnostics rows for JSONL artifact
         │
         ▼
signals.enrich_block()          — ingredient/instruction/yield/time detection
         │
         ▼
assign_block_roles()            — deterministic role labeling
  (cookimport/parsing/block_roles.py)
         │
         ▼
Downstream: segmentation → field extraction → chunking → output
```

### Extractor Switch

Controlled by `C3IMP_EPUB_EXTRACTOR` env var, or via the interactive CLI Settings menu (`EPUB Extractor`).

| Value | Behavior |
|---|---|
| `unstructured` (default) | Unstructured HTML partitioner |
| `legacy` | Original BeautifulSoup tag-walker |

CLI usage:
```bash
cookimport stage data/input/book.epub --epub-extractor unstructured
cookimport stage data/input/book.epub --epub-extractor legacy
```

The switch is read at call time (not import time) so the interactive settings menu takes effect without restarting.

**Failure mode**: When set to `unstructured`, partitioning errors are raised loudly — no silent fallback to legacy. This prevents hidden regressions.

### Adapter: Element → Block Mapping

`cookimport/parsing/unstructured_adapter.py` — single entry point:

```python
partition_html_to_blocks(
    html: str,
    *,
    spine_index: int,
    source_location_id: str,
) -> tuple[list[Block], list[dict]]
```

**Category mapping**:
- `Title` → `BlockType.TEXT`, `font_weight="bold"`, `is_heading=True`, `heading_level=clamp(depth+1, 1, 6)`
- `NarrativeText` / `UncategorizedText` → `BlockType.TEXT`
- `ListItem` → `BlockType.TEXT`, `is_list_item=True`
- `Table` → `BlockType.TABLE`
- Everything else → `BlockType.TEXT`

**Traceability features** stored on every Block:
- `spine_index` — absolute position in EPUB spine (for split-job merge ordering)
- `unstructured_element_id` — Unstructured's hash-based ID (may vary across versions)
- `unstructured_element_index` — sequential index within the spine document (stable)
- `unstructured_stable_key` — `"{source_location_id}:spine{spine_index}:e{element_index}"` (diff-friendly)
- `unstructured_category` — raw category string from Unstructured
- `unstructured_category_depth` — hierarchy depth (drives heading level)
- `unstructured_parent_id` — Unstructured parent element linkage
- `source_location_id` — book identifier for provenance correlation

**Text normalization**: Uses the same `cleaning.normalize_text()` pipeline as legacy (NFKC + whitespace standardization + mojibake repair). Applied in the adapter to avoid double-normalization.

**Ordering invariants**:
- Element order within each spine doc matches Unstructured's output order
- Spine docs iterated in ascending `spine_index`
- No sorting by text or category — ordering is the coordinate system for segmentation

### Block Roles

`cookimport/parsing/block_roles.py` — called after signal enrichment, for both extractors:

```python
assign_block_roles(blocks: list[Block]) -> None  # in-place
```

Deterministic, cheap heuristics. Stored as `Block.features["block_role"]`:

| Role | When assigned |
|---|---|
| `recipe_title` | Heading (h3+) with short text (≤80 chars) |
| `section_heading` | Heading (h1/h2) with short text |
| `ingredient_line` | `is_ingredient_likely`, `starts_with_quantity`, or ingredient header |
| `instruction_line` | `is_instruction_likely` or instruction header |
| `metadata` | `is_yield` or `is_time` |
| `tip_like` | Starts with "Note:", "Tip:", "Chef's note:", etc. |
| `narrative` | Substantive prose (≥15 words) |
| `other` | Everything else (short fragments, noise) |

**Block role counts are aggregated per chunk** during chunking (`chunks.py`) and stored in `provenance["block_role_counts"]`. The chunk lane scorer (`_score_lane`) uses these counts to adjust knowledge/noise scoring: recipe-structural roles (ingredient_line, instruction_line, recipe_title, metadata) boost the knowledge score proportionally, tip_like provides a small knowledge boost, and `other` roles add a slight noise bias.

### Diagnostics Artifact

When using the `unstructured` extractor, a `unstructured_elements.jsonl` raw artifact is emitted per run. Each line is a JSON object with:

```json
{
  "source_location_id": "book-name",
  "spine_index": 3,
  "element_index": 7,
  "element_id": "abc123...",
  "stable_key": "book-name:spine3:e7",
  "category": "Title",
  "category_depth": 1,
  "parent_id": "def456...",
  "text": "Chapter 2: Soups",
  "html_tag": null
}
```

The artifact metadata includes `unstructured_version` and `element_count`.

Located at: `raw/epub/<source_hash>/unstructured_elements.jsonl`

For split jobs (`--workers > 1`), each worker writes diagnostics under `.job_parts/`, merged to `raw/` on success.

### Split-Job Behavior

Unchanged from legacy. Each worker processes `[start_spine, end_spine)`:
- Blocks carry absolute `spine_index` for deterministic merge
- Diagnostics accumulate per-worker and merge with the block artifacts
- Known limitation preserved: recipes spanning spine boundaries can still be split

## Key Decisions

| Decision | Rationale |
|---|---|
| Partition per spine doc (not `partition_epub`) | Preserves start_spine/end_spine splitting; avoids Pandoc/system deps |
| Store element_id + stable_key on every Block | Traceability for regression diffing |
| EPUB-only scope (no PDF) | PDF uses different upstream primitives (layout/OCR); separate adapter needed |
| Text normalization in adapter via `cleaning.normalize_text()` | One source of truth; avoids double-normalization |
| Hard-fail on Unstructured errors (no silent fallback) | Silent fallback hides regressions |
| block_roles run for both extractors | Cheap deterministic heuristics; benefits all paths |
| Pin `unstructured>=0.18.32,<0.19` | Avoid behavior drift; tested and stable |
| Default to `unstructured` extractor | Better semantic extraction; legacy available as fallback |

## Telemetry

## Quirks & Notes

- Unstructured 0.18.32 classifies short paragraphs as `UncategorizedText` and longer ones as `NarrativeText`. Both map to `BlockType.TEXT`; `block_role` handles the semantic distinction.
- `category_depth` on Title elements reflects HTML heading hierarchy: h1→0, h2→1, etc. This drives `heading_level = clamp(depth+1, 1, 6)`.
- The HTML partitioner doesn't require NLTK/punkt downloads (unlike PDF partitioning).
