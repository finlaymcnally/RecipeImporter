## Surprises & Discoveries

- Observation: The repo uses three nearby names for different things: `codex-exec` as a benchmark/profile slug, `codex-farm` as an older subprocess term, and `codex exec` as the live correction transport.
  Evidence: `cookimport/config/codex_decision.py`, `cookimport/cli_ui/run_settings_flow.py`, and `cookimport/llm/codex_exec_runner.py` each describe a different layer of that stack.

- Observation: Analytics and benchmark code currently treat the codex-side variant as both a logical variant id and a directory-name convention.
  Evidence: `cookimport/analytics/benchmark_semantics.py`, `cookimport/cli_support/bench_single_book.py`, and the associated tests look for paired `vanilla` / codex-side output roots and rank the two variants together.

- Observation: Hyphenating the new slug broke analytics in two separate ways: JavaScript object literals needed quoted `"codex-exec"` keys, and the Python/JS variant normalizers could no longer round-trip a hyphenated explicit value through underscore-based normalization.
  Evidence: the first analytics rerun failed broadly until `script_bootstrap.py`, `script_compare_control.py`, and `benchmark_semantics.py` were updated together.

## Decision Log

- Decision: This plan renames the active `codex-exec` slug/profile/backend surface first, not every remaining `codex-farm` transport-specific identifier.
  Rationale: The operator asked to replace the stale project term with the name that matches the live correction path. The slug/profile/backend layer is the high-signal mismatch. Some literal `codex-farm` seams still describe a separate prelabel or legacy subprocess contract and should not be conflated blindly during this change.
  Date/Author: 2026-04-01 / Codex

## Outcomes & Retrospective

This plan is implemented. The active benchmark/profile/backend slug is now `codex-exec`, and the high-signal UI/docs copy that referred to the active correction path now says `Codex Exec`. The targeted `codex-exec` search surface is clean.

The one real trap was analytics. Hyphenated slugs are better operator-facing names, but they are less forgiving inside JavaScript literals and normalization helpers than the old bare word. Fixing the rename cleanly required updating both the renderer literals and the Python/JS normalization logic at the same time.

## Context and Orientation

The active rename surface is spread across a few clear areas. `cookimport/config/codex_decision.py` defines top-tier profile ids, benchmark variant ids, and the emitted `codex_backend` value. `cookimport/cli_ui/run_settings_flow.py` uses that profile id for the interactive preset chooser. `cookimport/cli_support/bench_single_book.py`, `cookimport/cli_support/bench_single_profile.py`, and Label Studio benchmark helpers build paired benchmark output folders keyed by the variant slug. `cookimport/analytics/benchmark_semantics.py`, `cookimport/analytics/compare_control_engine.py`, and the dashboard renderer files interpret that slug when grouping runs and rendering the comparison UI. Tests across `tests/cli`, `tests/analytics`, `tests/labelstudio`, and `tests/bench` assert the same naming.

This plan intentionally distinguishes the active slug `codex-exec` from the older subprocess term `codex-farm`. The slug is a repo-owned naming choice. The subprocess term can still refer to a distinct command seam and is therefore not renamed indiscriminately here.

## Plan of Work

Update config and CLI code first so the new canonical slug is `codex-exec`. Keep the normalization path tolerant of the older input aliases if that can be done without leaving the stale term visible in the steady-state output. Then update benchmark helpers, analytics grouping/rendering code, Label Studio benchmark/report helpers, and any path-building logic so new paired outputs and summaries use `codex-exec`.

Once the runtime code is aligned, update the affected tests to assert `codex-exec`. Then update the short docs that describe the active paired benchmark/profile/backend naming so the docs match the live code. Keep edits localized. Do not touch unrelated behavior.
