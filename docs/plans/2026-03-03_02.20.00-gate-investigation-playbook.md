## Context and Orientation

A “gate” here means one regression criterion emitted by `_build_line_role_regression_gate_payload` in `cookimport/cli.py`. The run fails if any gate is false.

A “baseline” means a historical canonical benchmark row from `data/.history/performance_history.csv` where both `llm_recipe_pipeline` and `line_role_pipeline` are off for the same source.

A “candidate” means the current line-role run under test (deterministic pipeline for this plan).

“Macro F1 excluding OTHER” means the average F1 across scored labels excluding `OTHER`, read from `eval_report.json` field `macro_f1_excluding_other`.

Files used in this investigation:

- Gate logic: `cookimport/cli.py` (`_build_line_role_regression_gate_payload`).
- Deterministic label logic: `cookimport/parsing/canonical_line_roles.py`.
- Existing regression tests: `tests/bench/test_cutdown_export_consistency.py` and `tests/parsing/test_canonical_line_roles.py`.
- Frozen run artifacts for triage:
  - `data/golden/benchmark-vs-golden/2026-03-03_02.10.00_foodlab-line-role-gated-fix7/eval_report.json`
  - `data/golden/benchmark-vs-golden/2026-03-03_02.10.00_foodlab-line-role-gated-fix7/line-role-pipeline/joined_line_table.jsonl`

## Plan of Work

### Milestone 1: Artifact-first triage (no new replay)

Read the frozen `fix7` artifacts and produce a ranked contribution view of macro drag by label. The output of this milestone is not code; it is a short evidence summary written into this plan and a `docs/understandings` note.

Compute per-label F1 deltas against the FoodLab vanilla baseline (`2026-03-03_12.22.10_foodlab-baseline-off`) and identify the worst negative contributors. Then extract false-positive and false-negative examples for the top two labels, with counts and representative line texts from `joined_line_table.jsonl`.

Acceptance for Milestone 1: you can answer, with counts, “which labels are blocking macro uplift most right now?”

### Milestone 2: Offline hypothesis simulation

Before editing runtime code, run a lightweight simulation from `joined_line_table.jsonl` using explicit relabel rules in a temporary script (in `/tmp` or a scratch file) to estimate whether a proposed rule change could plausibly add at least `+0.02` to macro F1. This is directional, not canonical truth, but it prevents expensive blind replays.

Acceptance for Milestone 2: at least one simulated rule bundle predicts meaningful macro gain without collapsing `RECIPE_NOTES`/`RECIPE_VARIANT` recall below `0.40`.

### Milestone 3: Minimal code experiment loop

Apply one narrow code change in `cookimport/parsing/canonical_line_roles.py` per experiment, run targeted tests, then run exactly one gated replay with a fresh timestamped output directory.

After each replay, update this plan with:

- changed rule(s),
- gate table delta vs previous replay,
- keep/revert decision and rationale.

Stop early if `foodlab_macro_f1_delta_min` passes.

Acceptance for Milestone 3: either the macro gate passes, or three replays complete with a clear evidence-backed explanation for why it still fails.

### Milestone 4: Finalize investigation handoff

Write a concise investigation report in `docs/understandings` including:

- final gate snapshot,

- recommended next fix (single highest ROI step),
- replay budget consumed and run directories.

Then update this ExecPlan `Outcomes & Retrospective` to match.

Acceptance for Milestone 4: a next-day operator can continue from this file and the understanding note alone.

If a replay is interrupted, start a new run with a new timestamp directory and record the abandoned run in `Progress` with a short reason.

No destructive git or filesystem operations are required. Do not delete historical run directories during investigation.
