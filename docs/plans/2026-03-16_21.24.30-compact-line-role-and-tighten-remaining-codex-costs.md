# Compact Line-Role And Tighten Remaining Codex Costs

This ExecPlan is a living document. Keep `Progress`, `Decision Log`, and `Outcomes & Retrospective` current as the work lands.

## Purpose / Big Picture

All Codex-backed stages stay on, but the prompt payloads need to get materially smaller. The largest remaining cost center is whole-book line-role correction, so the main change is to compact line-role input rows without changing the response parser contract. After that, apply the remaining low-risk knowledge trims that reduce chars without disabling review.

## Decision Log

- Decision: keep the line-role response contract unchanged and compact only the input-side representation.
  Rationale: the current parser and tests are already stable; input compaction is where the repeated token waste lives.
  Date/Author: 2026-03-16 / Codex

- Decision: put recipe-span metadata in the batch header instead of repeating `within_recipe_span` on every row.
  Rationale: recipe membership is range-like batch metadata, not line-specific prose.
  Date/Author: 2026-03-16 / Codex

- Decision: include neighbor context only for rows with explicit escalation reasons.
  Rationale: most rows do not need local context, and unconditional neighbor text is one of the largest repeated payload costs.
  Date/Author: 2026-03-16 / Codex

- Decision: make the remaining knowledge trim a default-policy change, not a preview-only trick.
  Rationale: the user wants live savings, so the context reduction and stricter low-signal cutoff need to live in the shared stage/CLI/preview defaults.
  Date/Author: 2026-03-16 / Codex

## Outcomes & Retrospective

This landed cleanly. Line-role now keeps the strict JSON response shape but sends a much smaller request body: batch-level label/reason legends, batch-level recipe atomic ranges, compact array rows, and neighbor context only for escalated in-recipe rows. The shared line-role batch default is now `240`, which further cuts repeated instruction overhead.

The remaining low-risk knowledge trims also landed in the shared defaults. Knowledge context now defaults to `0` blocks, and tiny low-signal chunks can be pruned up to `240` chars when they still have no heading context, highlights, or table content.

Measured on `data/golden/benchmark-vs-golden/2026-03-16_20.23.25/.../line_role_only/`, the codex-shaped preview moved from:

- `92` total calls, `~578,563` estimated total tokens
- `41` knowledge prompts / `~183,168` estimated total tokens
- `15` line-role prompts / `~290,891` estimated total tokens

to:

- `84` total calls, `~449,484` estimated total tokens
- `40` knowledge prompts / `~170,096` estimated total tokens
- `8` line-role prompts / `~174,884` estimated total tokens

The new preview directory is `prompt_preview_codex_hypothetical_after_compact_line_role_2026-03-16_21.35.34`.
