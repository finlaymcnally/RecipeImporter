# Remove Prelabel `block_index` Task-Payload Compatibility

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

The active freeform prelabel contract is row-native, but the task payload and prelabel reader path still carry `block_index` compatibility. New tasks should stop writing that alias, and the active prelabel reader path should stop accepting it for row identity.

## Progress

- [x] (2026-04-16_23.20.00) Claimed `recipeimport-95fq` and re-read the owning Label Studio docs/contracts.
- [x] (2026-04-16_23.22.00) Identified the exact writer/reader seams: `freeform_tasks.py` still writes `block_index` into `source_map.rows` and `context_*_rows`, while `prelabel_prompt.py`, `prelabel_parse.py`, and `prelabel_mapping.py` still fall back from `row_index` to `block_index`.
- [x] (2026-04-16_23.24.00) Removed `block_index` from newly written prelabel task payload rows/context rows and from freeform-task coverage on that active contract.
- [x] (2026-04-16_23.24.00) Removed `block_index` fallback from the active prelabel reader path and updated the focused tests/docs.
- [x] (2026-04-16_23.25.00) Ran the focused validation slice and closed `recipeimport-95fq`.

## Surprises & Discoveries

- Observation: the remaining compatibility is split between the task writer and the prelabel reader path.
  Evidence: `freeform_tasks.py` wrote `block_index` into new task payload rows/context rows, while `prelabel_prompt.py`, `prelabel_parse.py`, and `prelabel_mapping.py` still accepted `block_index` in task rows or model quote selections.

- Observation: true provenance still lives elsewhere and should stay.
  Evidence: `source_block_index` is still used in task payload rows and supporting archive/context code, and that provenance is distinct from the stale `block_index` alias on row-native task rows.

## Decision Log

- Decision: remove `block_index` only from the active prelabel task payload and reader path in this pass.
  Rationale: that is the stale compatibility surface. Provenance-bearing `source_block_index` and unrelated archive block metadata remain legitimate and out of scope.
  Date/Author: 2026-04-16_23.20.00 / Codex

## Outcomes & Retrospective

This slice is implemented, validated, and closed in Beads. New freeform task payload rows/context rows no longer write `block_index`, and the active prelabel reader path now requires `row_index` for row identity.
