# Add size-budgeted group upload bundle for single-profile multi-book runs

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

Maintain this document in accordance with `docs/PLANS.md` from the repository root.

## Purpose / Big Picture

After this change, multi-book single-profile benchmark sessions produce one extra group upload bundle at `single-profile-benchmark/upload_bundle_v1` in addition to per-book bundles. The group bundle stays high-level and size-budgeted (target about 40MB) by dynamically reducing per-book sampled detail as the number of books increases.

## Surprises & Discoveries

- Observation: upload-bundle row-locator tests expect root locator values to resolve to dictionaries, so new root locator keys cannot be left null.
  Evidence: `tests/bench/test_benchmark_cutdown_for_external_ai.py` asserts all root locator values are dicts.

## Decision Log

- Decision: implement group behavior as an opt-in mode on existing upload-bundle writer instead of adding a separate script path.
  Rationale: keeps one contract (`upload_bundle_v1`) and avoids duplicate logic for index/payload generation.
  Date/Author: 2026-03-04 / assistant

- Decision: budget by bytes (target ~40MB) and sample wrong-line details per run with a shared per-run byte allocation.
  Rationale: directly maps to user requirement to keep group bundle size near a fixed envelope while scaling detail with run count.
  Date/Author: 2026-03-04 / assistant

## Outcomes & Retrospective

Single-profile multi-book runs now produce a group bundle in the root benchmark folder with high-level per-run data and dynamic per-run detail sampling. Existing per-book bundle behavior remains unchanged.

## Context and Orientation

`cookimport/cli.py` runs interactive single-profile matched-book batches and currently writes per-target upload bundles. `scripts/benchmark_cutdown_for_external_ai.py` provides `build_upload_bundle_for_existing_output(...)`, which creates the 3-file upload bundle by scanning source artifacts and writing index/payload rows.

## Plan of Work

Extend the upload-bundle helper with a high-level-only mode that selects a compact artifact subset and writes a group packet containing per-run summary plus size-budgeted sampled wrong-line rows. Then invoke that mode from the single-profile multi-book CLI path at the shared `single-profile-benchmark` root.

## Concrete Steps

1. Update `scripts/benchmark_cutdown_for_external_ai.py`:
   - add high-level artifact selection
   - add group packet generation with per-run sample budgeting
   - thread new args through existing-output helper
2. Update `cookimport/cli.py` to call the helper for group bundle after multi-book runs complete.
3. Add/adjust tests in `tests/bench/` and `tests/labelstudio/`.
4. Update docs in `docs/07-bench/07-bench_README.md` and `scripts/README.md`.

## Validation and Acceptance

Run focused tests:

- `source .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py -k high_level_only -q`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k group_upload_bundle -q`

Expected behavior:

- multi-book single-profile runs write `single-profile-benchmark/upload_bundle_v1/*`.
- group bundle index includes `analysis.group_high_level`.
- sampled rows per run decrease when run count increases under the same target bundle size.

## Idempotence and Recovery

Changes are additive and localized. If rollback is needed, remove the `high_level_only` helper path and the single-profile root bundle call; per-book bundle behavior remains as the fallback.
