# Finish Remaining Block-Era Semantic Cleanup

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

The active row-native migration is no longer blocked on the obvious public shims. The remaining cleanup is the last destructive 20-30 percent: row-gold benchmark artifacts still expose block-era provenance names, benchmark readers still consume those names, a few staging/report/helper surfaces still describe row-semantic ownership with block-era words, and the largest remaining old-name island is the knowledge worker transport/runtime contract.

This plan finishes that remaining cleanup in one coordinated program while keeping the block-oriented things that are still real and useful:

- `source_block_index`, `source_block_id`, and similar provenance/layout metadata
- archive block ids and split-job block rebasing metadata
- true source-layout spans such as table/source block ranges where the thing being described is actually source layout rather than semantic row authority

The goal is not “delete every string containing `block`.” The goal is: semantic row-owned contracts should stop pretending they are block contracts, and the final repo state should have a clear keep/delete rule instead of mixed semantics.

  Evidence: a raw grep finds thousands of `block_*` matches, but the meaningful remaining semantic debt clusters into four islands: row-gold artifact/benchmark readers, staging/report summary naming, helper/prompt-preview residue, and the knowledge worker packet/runtime contract.

- Observation: the active Label Studio/freeform path is already mostly clean.
  Evidence: recent cleanup waves removed the public task/prelabel/export compatibility shims from the active row-native path. What remains nearby is mostly row-gold artifact naming and benchmark-reader fallout, not another broad Label Studio rewrite.

- Observation: the knowledge lane is the hardest remaining piece.
  Evidence: `knowledge_stage/task_file_contracts.py`, `structured_session_contract.py`, `codex_exec_runner.py`, `codex_farm_knowledge_ingest.py`, and related tests/docs still describe semantic reviewed units with names like `block_decisions`, `block_index`, and `idea_groups[*].block_indices`.

  Evidence: while finishing the knowledge wave, a prompt-preview/line-role assertion initially pushed the line-role prompt wording from `block_index` to `row_index`. That was reverted once it was clear the line-role field there is source-block provenance for atomic rows, not semantic row identity. The keep/delete rule still applies: provenance can stay block-oriented, semantic owned-unit contracts cannot.

- Observation: not every block-named field is debt.
  Evidence: source-block provenance, archive block ids, split-job block rebasing, and true table/source layout spans still describe layout/provenance rather than semantic row authority.

- Observation: the docs index step is currently unreliable in this environment.
  Evidence: `npm run docs:list` still fails with `ENOTSUP` from the local `tsx` IPC socket path under `/mnt/c/.../tsx-1000/*.pipe`.

- Observation: the first downstream failures after removing row-level block aliases were outdated helper fixtures, not live runtime regressions.
  Evidence: the failing benchmark-helper artifact test and related scheduler/signature fixtures were still building semantic-row prediction records with `block_index` / `block_text` / `block_features`. Updating them to `row_index` / `row_text` / `row_features` restored coverage without reintroducing the deleted compatibility path.

- Observation: the next honest cleanup seam was staging/knowledge summary vocabulary, not boundary ownership dataclasses.
  Evidence: `RecipeBoundaryResult.recipe_owned_blocks` and related block-boundary lists still describe real source-block ownership, but the active nonrecipe finalize and knowledge rollups were already counting row indices while publishing names like `candidate_block_count`, `kept_for_review_block_count`, and `changed_blocks`.

- Observation: the prompt-preview/helper residue was mostly gone already; the live remaining shim was one row-native labeled-line fallback.
  Evidence: targeted grep across `prompt_preview.py`, `cli_support`, and prompt-preview tests found one active fallback from `source_block_index` to `block_index` when rebuilding labeled-line line-role candidates. The other nearby `block_index` terms on that path were either true line-role provenance in prompt payloads or source-layout helpers, not row-semantic aliases.

## Keep / Delete Rule

### Keep

Keep block-era fields when the thing being described is truly block/layout/provenance:

- `source_block_index`
- `source_block_id`
- archive block ids and merged split-job block offsets
- table or layout spans that are genuinely source block coordinates
- historical plan files and archived notes that are intentionally historical

### Delete or Rename

Delete or rename block-era fields when the thing being described is semantically row-owned:

- row-gold artifact provenance fields that still imply the row semantic unit is a block
- benchmark readers that still consume row-gold `block_index` on the active contract
- staging/runtime/public summaries like `recipe_owned_blocks` when they really mean row-semantic ownership
- helper/prompt-preview residue that still normalizes row-native payloads through semantic block aliases
- knowledge packet/runtime fields like `block_decisions`, `block_index`, and `idea_groups[*].block_indices` where the reviewed semantic unit is now a row

## Plan of Work

### Milestone 1: Row-gold artifact/public reader cleanup

Rename the active row-gold provenance field from `block_index` to `source_block_index`. Then move the benchmark readers, helper summaries, and CLI support surfaces to the new artifact contract. This should remove the most visible remaining old-name surface without touching true provenance semantics.

### Milestone 2: Staging and helper semantic honesty

Audit staging/runtime/report/public dataclasses and summaries that still use block-era words for row-semantic ownership. Separately, sweep prompt-preview and row-facing helper residue that still tolerates or emits semantic block aliases around row-native payloads.

### Milestone 3: Knowledge worker transport/runtime migration

This is the largest remaining old-name island. Migrate the typed knowledge schema first, then the worker-visible task-file/runtime path, then the real/fake runner/tooling layer, and finally ingest/reporting/promotion consumers. The end state should make it obvious that semantic owned units are rows, while any remaining `source_block_*` fields are provenance only.

### Milestone 4: Final keep/delete audit and docs/fixture sweep

Once the substantive code moves are done, sweep the durable docs and active fixtures so the repo stops advertising retired semantic block names. Then run a final audit that explicitly lists what remains intentionally and what was removed.

## Idempotence and Recovery

This program should proceed in small destructive waves, not one giant flag day commit.

If a migration reveals that an old archived payload still needs translation:

- put that translation only in an explicit migration or archival repair seam;
- do not restore old aliases on the live runtime surface;
- record the exception in the bead and update this plan if the keep/delete rule changes.

If the parsing/staging edge audit finds a surface that is ambiguous rather than obviously keep/delete, file a precise follow-on bead instead of guessing.
