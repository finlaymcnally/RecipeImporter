# Remove EPUB extractor auto mode from runtime paths

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

PLANS.md is checked into the repo at `docs/PLANS.md`. This document is maintained in accordance with that file.

## Purpose / Big Picture

After this change, users must explicitly choose EPUB extraction backend (`unstructured`, `legacy`, `markdown`, or `markitdown`). The old `epub_extractor=auto` runtime path is removed from stage and prediction generation so behavior is direct and intentional. Users can verify this by running `stage --epub-extractor auto` and seeing a validation failure, then running with explicit extractors and seeing normal output.

## Surprises & Discoveries

- Observation: Typer failure text for invalid extractor input was not in `result.stdout` in the CLI runner assertion path.
  Evidence: `tests/cli/test_cli_output_structure.py::test_stage_rejects_auto_epub_extractor` failed when asserting stdout text; assertion was changed to check exit behavior + no output run dir creation.

## Decision Log

- Decision: Keep `select_epub_extractor_auto(...)` module available for debug/race tools, but remove it from stage and prediction runtime wiring.
  Rationale: User requested removal of runtime auto mode, while debug diagnostics remain useful and isolated.
  Date/Author: 2026-02-23 / Codex

- Decision: Preserve `epub_extractor_requested` and `epub_extractor_effective` run-config fields, but make them equal for new runs.
  Rationale: Avoid schema churn in analytics/history while still removing auto-resolution behavior.
  Date/Author: 2026-02-23 / Codex

- Decision: Add migration in `RunSettings.from_dict(...)` that coerces legacy `epub_extractor=auto` to `unstructured` with warning.
  Rationale: Prevent old saved settings payloads from breaking interactive/default flows after enum tightening.
  Date/Author: 2026-02-23 / Codex

## Outcomes & Retrospective

Runtime extractor selection now requires explicit choices end-to-end in stage and prediction-generation flows. All-method benchmark permutations dropped auto-derived combinations, reducing method count and making comparisons clearer. Historical data contracts were preserved via requested/effective fields and legacy settings migration. Debug-only auto scoring remains available outside runtime extraction mode selection.
