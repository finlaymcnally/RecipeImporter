# Task: QualitySuite WSL single-slot safety guard

## Problem statement

WSL crashes were still reported when `bench quality-run` used a single experiment slot (`--max-parallel-experiments=1`), because the previous guard only activated when outer experiment fanout was greater than one.

## Acceptance criteria

- WSL single-slot quality-run defaults to subprocess experiment isolation in auto executor mode.
- WSL safety guard applies even when `max_parallel_experiments_effective == 1` (unless explicitly disabled by env).
- Guarded runs cap worker/runtime knobs to conservative values and emit telemetry in `experiments_resolved.json`.
- Non-WSL behavior remains unchanged.
- Regression tests cover single-slot WSL executor/guard behavior.

## Verification steps

Run from repo root with venv active:

```bash
. .venv/bin/activate
pytest tests/bench/test_quality_suite_runner.py -k "wsl" -q
pytest tests/bench/test_bench.py -k "quality_run" -q
```

## Evidence

- Added `test_run_quality_suite_applies_wsl_safety_guard_for_single_experiment_slot`.
- Updated runner behavior so WSL single-slot runs use subprocess executor reason `wsl_single_experiment_isolation`.

## Constraints / gotchas and decision

- Constraint: preserve Linux/macOS single-slot quality-run behavior (`thread` path) to avoid changing normal local throughput defaults outside WSL.
- Decision: WSL-only path now always applies guardrails by default, with opt-out via `COOKIMPORT_QUALITY_WSL_DISABLE_SAFETY_GUARD=1`.

## Rollback / compensation plan

- Temporary bypass: set `COOKIMPORT_QUALITY_WSL_DISABLE_SAFETY_GUARD=1`.
- Full rollback: revert this task’s code/doc diff.
