summary: "Add a zero-token helper that reconstructs Codex prompt previews from an existing processed benchmark/stage run."
---

## Problem

After the codexfarm refactor, there is no direct way to inspect the concrete prompts a vanilla-to-codex replay would send without paying token costs. The existing `--codex-execution-policy plan` path only writes a high-level task plan and does not materialize prompt bodies.

## Acceptance Criteria

- Given an existing processed run root, a helper can rebuild Codex prompt preview artifacts without executing Codex.
- The helper can preview recipe correction prompts from authoritative stage outputs.
- The helper can preview optional knowledge prompts and line-role prompts when those surfaces are requested.
- The helper writes reviewer-friendly prompt artifacts and a machine-readable manifest under the preview output root.

## Verification

- Run focused tests covering recipe/knowledge/line-role preview generation from fixture-like existing outputs.
- Run the helper against `data/golden/benchmark-vs-golden/2026-03-16_13.34.01/.../2026-03-16_13.34.06` and confirm prompt preview artifacts are written with no Codex execution.

## Constraints / Decisions

- Use existing deterministic stage artifacts as the source of truth instead of rerunning Codex.
- Prefer current pipeline prompt templates from `llm_pipelines/` when no Codex run-assets directory exists.
- Keep the feature scoped to preview generation; do not add a fake execution path into the live orchestrators.

## Rollback

- Revert the preview helper, tests, and small doc updates. No stored runtime data or schemas are mutated.
