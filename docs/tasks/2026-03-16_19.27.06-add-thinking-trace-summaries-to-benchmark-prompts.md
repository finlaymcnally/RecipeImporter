# Problem

The benchmark prompt bundle already contains prompt/response logs and per-row `thinking_trace`
payloads inside `full_prompt_log.jsonl`, but there is no dedicated trace-summary surface inside
`codexfarm/prompts/`. Reviewers have to inspect JSONL manually or chase raw `.trace.json` paths
under `data/output`.

# Acceptance Criteria

- Prompt export writes reviewer-facing trace summary artifacts into `prompts/`.
- The trace summary is built from the final merged `full_prompt_log.jsonl`, so it includes recipe,
  knowledge, and line-role rows consistently.
- Run-manifest artifact collection exposes the new summary files when they exist.

# Verification

- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py`
- Backfill the target benchmark run and verify `prompts/` now contains trace summary files with
  rows covering all `233` exported CodexFarm interactions.

# Evidence

- The target `saltfatacidheatcutdown` benchmark currently has `233` prompt-log rows:
  `175` recipe, `2` line-role, `56` knowledge.
- Recipe and knowledge rows have resolved trace paths; line-role rows explicitly record missing
  trace payloads, so a merged-log-derived summary can represent the whole run without ambiguity.
- Verified after backfill:
  - `prompts/thinking_trace_summary.jsonl` has `233` rows
  - `prompts/thinking_trace_summary.md` summarizes trace coverage for recipe, line-role, and knowledge
  - current run breakdown: `231` rows with existing trace files, `0` rows with reasoning events

# Constraints / Decision

Do not invent a second trace-discovery path. Build the summary from `prompts/full_prompt_log.jsonl`
after prompt export finishes so the new files always describe the exact exported prompt bundle.
