Problem statement

RecipeImport's shard-v1 Codex phases were planning multiple prompt jobs per book, but the live worker defaults did not follow that plan: recipe and knowledge defaulted to `1` worker, and canonical line-role defaulted to `4`. Interactive benchmark runs therefore showed `task 1/5 | running 1` or `running 4` even when the planner had already split the book into 5 shard jobs.

Acceptance criteria

- When `recipe_worker_count`, `knowledge_worker_count`, and `line_role_worker_count` are unset, the active shard-v1 runtime defaults to the planned shard/job count for that phase.
- The implicit default never exceeds `20` concurrent workers for one book/phase run.
- Explicit worker overrides still win, and live/prompt-preview worker planning stays aligned.

Verification steps

- `source .venv/bin/activate && pytest tests/llm/test_phase_worker_runtime.py tests/llm/test_recipe_phase_workers.py tests/llm/test_codex_farm_knowledge_orchestrator.py tests/parsing/test_canonical_line_roles.py -q`
- `source .venv/bin/activate && pytest tests/llm/test_prompt_preview.py tests/parsing/test_canonical_line_role_env.py -q`

Evidence

- Added shared worker-resolution regression coverage for default shard-count fan-out and the `20` worker cap.
- Added recipe, knowledge, and canonical line-role regressions proving that unspecified worker counts now follow shard count instead of serializing or capping at the old phase-local defaults.

Constraints / gotchas

- Keep explicit worker overrides and the line-role env override working.
- Do not let one RecipeImport worker fan back out into multiple CodexFarm process workers; `codex-farm process --workers 1` remains correct inside each assigned worker sandbox.

Decision

- Centralized the implicit default in `resolve_phase_worker_count(...)` and reused it from recipe, knowledge, line-role, and prompt-preview planning.
- Treat "planned shards/jobs per phase, capped at 20" as the default only when the operator did not explicitly set a worker count override.

Rollback / compensation plan

- Revert the shared helper and its call sites together if the higher default fan-out proves too aggressive for a given operator environment.
