## Problem

Interactive single-book benchmark runs can crash during artifact reconstruction with `NameError: name '_report_optional_metric' is not defined`.
`cookimport.cli_support.bench_artifacts` still depends on the benchmark facade import order to provide `_report_optional_metric`, but `bench.py` loads `bench_artifacts` before `bench_all_method`, so that helper is absent on the live path.

## Acceptance Criteria

- `cookimport.cli_support.bench_artifacts` binds `_report_optional_metric` directly from its owning benchmark module instead of relying on the facade snapshot.
- A focused fast regression test fails on the broken baseline and passes after the fix.
- The targeted artifact-helper test slice passes in the project `.venv`.

## Verification

- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py::test_bench_artifacts_rebuilds_prediction_bundle_with_string_timing_metrics -q`
- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -q`

## Evidence

- Before the fix, the focused regression test raises `NameError: name '_report_optional_metric' is not defined` from `cookimport/cli_support/bench_artifacts.py`.
- After the fix, the focused regression test and the artifact-helper file pass in `.venv`.

## Constraints / Decision

- Keep scope to the split benchmark helper binding bug and the fast test coverage gap that let it slip past routine runs.
- Reuse the existing fast artifact-helper test file so routine `labelstudio`/fast loops catch this class of issue without depending on the slow single-book helper suite.
