Problem statement

Interactive benchmark menus still render the same book identity differently across paths.
The single-offline gold picker now uses concise book labels, but the matched-books picker still shows source filename plus bracketed gold label noise.

Acceptance criteria

- Interactive benchmark pickers reuse one concise book identity wherever the operator is choosing benchmark books.
- The matched-books picker uses the same concise book label style as the single-offline gold picker.
- Targeted tests cover the matched-books picker label rendering.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py::test_interactive_single_profile_selected_matched_uses_concise_book_labels -q`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_import_eval.py::test_display_gold_export_path_relative_to_golden_root tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py::test_interactive_single_profile_selected_matched_uses_concise_book_labels -q`

Evidence

- Before the change, the matched-books picker test failed because its row title still combined `source_file_name` and bracketed `gold_display`.
- After the change, the picker row uses the same concise book identity as the single-offline gold export picker.

Constraints/gotchas + decision

- Keep this scoped to operator-facing benchmark picker labels, not progress logs or report payload fields.
- Reuse a shared helper instead of hardcoding another benchmark-only menu format.
