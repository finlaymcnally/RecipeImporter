# Problem statement

Interactive export project picker only showed project names. User requested that each project entry include its category/type.

# Acceptance criteria

- Interactive export picker displays project labels as `<project> [type: <scope>]`.
- Scope detection uses known local manifest `task_scope` when available.
- Existing routing still returns the selected project title as the value.

# Verification steps

- `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k 'select_export_project_name or interactive_labelstudio_export'`

# Evidence

- Targeted tests pass, including assertions on picker labels containing `[type: ...]`.

# Constraints/gotchas

- Some projects may not have enough metadata to infer scope; these show as `unknown`.

# Decision

Use local `manifest.json` history for most accurate type mapping and fallback to project payload `label_config` inference when available.

# Rollback / compensation plan

Remove scope labeling from picker entries and revert helper functions in `cookimport/cli.py`.
