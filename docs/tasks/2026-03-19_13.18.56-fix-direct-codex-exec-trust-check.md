# Fix direct Codex exec trust check

Problem statement:
Direct shard runs launch `codex exec` from an isolated workspace outside the repository. The current runner does not pass `--skip-git-repo-check`, so interactive single-offline Codex benchmark variants fail immediately with "Not inside a trusted directory".

Acceptance criteria:
- Direct `codex exec` command construction includes `--skip-git-repo-check`.
- Focused runner coverage fails before and passes after this change.
- Direct-exec docs note why the flag is required.

Verification steps:
- `source .venv/bin/activate && pytest tests/llm/test_codex_exec_runner.py -q`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_smoke.py -q`

Evidence:
- The failing interactive run wrote shard errors like `codex exec failed (exit=1): Not inside a trusted directory and --skip-git-repo-check was not specified.`
- `tests/llm/test_codex_exec_runner.py` now asserts the generated command includes the missing flag.

Constraints / gotchas + brief decision:
- Keep the fix minimal at the shared direct-exec command builder so recipe, knowledge, and line-role all benefit.
- Do not remove the isolated workspace; it is still the right instruction/file-visibility boundary.

Rollback / compensation plan:
- Revert this task's code/doc changes if the Codex CLI trust model changes and isolated workspaces no longer need the bypass flag.
