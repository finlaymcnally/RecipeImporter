## Problem statement

Interactive benchmark users could not choose recipe Codex Farm in `Change run settings...` even when `COOKIMPORT_ALLOW_CODEX_FARM=1` was enabled.

## Acceptance criteria

- `run_settings_ui_specs()` exposes `llm_recipe_pipeline=codex-farm-3pass-v1` when `COOKIMPORT_ALLOW_CODEX_FARM=1`.
- Without env unlock, UI choices remain `("off",)`.
- Tests cover both locked and unlocked choice lists.

## Verification steps

- Run: `. .venv/bin/activate && pytest tests/llm/test_run_settings.py -q -k "run_settings_ui_specs"`

## Evidence

- Updated `cookimport/config/run_settings.py`.
- Updated `tests/llm/test_run_settings.py`.
- Updated CLI docs notes.

## Constraints and decisions

- Kept env gate behavior strict: only UI exposure changes when unlocked; no default behavior changes.

## Rollback plan

- Revert UI choice logic to always `("off",)` for `llm_recipe_pipeline`.
