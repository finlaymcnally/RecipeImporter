## Problem

Recent interactive single-book benchmark crashes were caused by missing helper/name bindings inside the split benchmark stack. The current smoke test only proves menu routing because it stubs `labelstudio_benchmark(...)`, so routine runs miss deeper runtime `NameError` failures.

## Acceptance Criteria

- Add a static undefined-name guard for the benchmark CLI/helper stack.
- Add an offline vanilla single-book benchmark sim that executes the real runtime path without live LLM calls.
- Add an offline codex-shaped single-book benchmark sim that executes the same real runtime path without live LLM calls.
- Focused pytest verification passes in the project `.venv`.

## Verification

- `source .venv/bin/activate && pip install -e .[dev]`
- `source .venv/bin/activate && pytest tests/core/test_benchmark_undefined_names.py -q`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_smoke.py -q`

## Evidence

- Before this change, the benchmark smoke file did not execute the real `labelstudio_benchmark(...)` runtime and therefore could not catch helper-binding crashes below the menu layer.
- After this change, the new static guard and both offline whole-run sims pass in `.venv`.

## Constraints / Decision

- Keep all whole-run coverage offline by stubbing only leaf prediction-generation/evaluation seams.
- Reuse tiny `tmp_path` fixtures and existing benchmark helper support instead of adding a heavy realism suite.
