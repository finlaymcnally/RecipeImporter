# Problem

The `Previous Runs` dashboard area could expand to the right as content widened, instead of keeping width fixed and using local horizontal scroll regions.

# Acceptance Criteria

- `Previous Runs` container tracks stay width-constrained to available viewport width.
- Wide table/chart/control content remains accessible through existing local scroll containers.
- Renderer CSS tests assert the containment contract.

# Verification

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k previous_runs_table_has_horizontal_scroll_css`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`

# Evidence

- `2 passed, 67 deselected in 0.85s` (targeted regression check)
- `69 passed in 9.38s` (full dashboard renderer test file)

# Constraints / Gotchas

- `#previous-runs-table` intentionally keeps a large `min-width` for readability; containment must happen at section/grid level, not by shrinking table contract.

# Decision

Applied explicit grid track containment (`grid-template-columns: minmax(0, 1fr)`) and `min-width: 0` guards on Previous Runs subsection containers and related hosts, then added regression assertions in dashboard CSS tests.
