# Task: interactive prelabel account resolution

## Problem statement

Interactive freeform prelabel uploads used the default Codex command/home implicitly, so users with multiple Codex logins could select models from one account cache but execute with another account.

## Acceptance criteria

- Interactive freeform prelabel does not prompt for command selection.
- Interactive flow uses resolved command (`COOKIMPORT_CODEX_CMD` or `codex exec -`) and `CODEX_HOME` to resolve account/model metadata.
- Interactive flow shows resolved account email (when discoverable) before model selection.
- Resolved command is passed into `run_labelstudio_import(...)` and persisted in prelabel reporting.
- Prelabel command handling treats `codex2` like `codex` for model injection and plain-command fallback.
- Model/cache/config discovery is command-aware (`codex_cmd` roots), not global-only.

## Verification steps

## Constraints / gotchas and decision

- Constraint: this is a local multi-account workflow, and the desired account may be controlled by `CODEX_HOME` rather than command alias selection.
- Decision: keep CLI/env precedence and remove interactive command-choice UI; rely on `COOKIMPORT_CODEX_CMD` + `CODEX_HOME`.
- Decision: show account email via local auth token claims when available; degrade gracefully when not resolvable.

## Rollback / compensation plan

- Revert interactive default-command wiring in `cookimport/cli.py` and command-aware home/account helpers in `cookimport/labelstudio/prelabel.py`.
- Keep preflight checks enabled so unsupported model/account combinations still fail fast.
