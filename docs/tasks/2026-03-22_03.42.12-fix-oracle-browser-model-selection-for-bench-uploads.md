# Task

## Problem

Benchmark Oracle browser uploads are preserving whatever ChatGPT browser mode is already active because the wrapper passes `--browser-model-strategy ignore`.
With no Oracle env override set, the genuine-run fallback is also `gpt-5.4`, so uploads can land on a non-Pro / heavy-thinking lane instead of the intended Pro review lane.

## Acceptance criteria

- Benchmark browser uploads pass `--browser-model-strategy select`.
- Genuine benchmark uploads default to a Pro model when no Oracle env override is set.
- Bench docs describe the new behavior accurately.

## Verification

- `source .venv/bin/activate && pytest tests/bench/test_benchmark_oracle_upload.py -q`

## Evidence

- Pre-fix, benchmark docs and code both said browser uploads used `--browser-model-strategy ignore` and fell back to `gpt-5.4`.
- Post-fix, the benchmark upload command and tests prove explicit model selection plus Pro fallback.
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_oracle_upload.py -q` passed.

## Constraints / decision

- Keep the change narrow to benchmark Oracle upload only.
- Preserve `ORACLE_GENUINE_MODEL` / compatibility env overrides and the helper/test lane behavior.
