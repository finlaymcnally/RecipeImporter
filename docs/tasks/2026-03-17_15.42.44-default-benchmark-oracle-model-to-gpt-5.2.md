Problem statement

Manual `cookimport bench oracle-upload` runs are failing on the Oracle browser model picker because the current ChatGPT UI no longer exposes a direct `GPT-5.2 Pro` option in the way Oracle 0.8.6 expects.

Acceptance criteria

- Benchmark Oracle uploads default to a browser model target that matches the current picker more reliably.
- Manual `--model ...` override still works.

Verification steps

- Activate `.venv`.
- Run `pytest tests/bench/test_benchmark_oracle_upload.py`.

Evidence

- The benchmark Oracle default model constant now resolves to `gpt-5.2`, which maps to the current `GPT-5.2` picker target instead of `GPT-5.2 Pro`.
- Existing Oracle-upload tests still pass with the updated default.
- Verification result: `source .venv/bin/activate && pytest tests/bench/test_benchmark_oracle_upload.py` -> `8 passed, 1 warning in 2.06s`.

Constraints/gotchas + decision

- This only fixes the stale default picker target. A missing logged-in Oracle browser profile can still cause separate browser/login failures.
- Keep the change narrow: update the default benchmark Oracle model rather than trying to patch Oracle's upstream model-switcher logic here.

Rollback / compensation plan

- Revert the default-model constant if Oracle's browser picker becomes stable again for `gpt-5.2-pro`.
