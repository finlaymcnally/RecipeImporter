Problem statement

Food Lab-style EPUB reference tables were losing row/cell structure before `cookimport/parsing/tables.py` saw them, and some conversion sections were still being emitted as fake recipes instead of `nonRecipeBlocks`.

Acceptance criteria

- BeautifulSoup and unstructured EPUB extraction preserve table rows as `epub_table_row` blocks with `features.epub_table_cells` and visible `|` delimiters.
- `detect_tables_from_non_recipe_blocks(...)` reconstructs tables from structured EPUB rows and can salvage a simple flattened conversion block.
- `cookimport stage data/input/thefoodlabCUTDOWN.epub --table-extraction on --write-markdown` writes conversion/reference tables under `tables/thefoodlabcutdown/tables.md`.

Verification steps

- `source .venv/bin/activate && pytest tests/core/test_recipe_likeness_scoring.py tests/ingestion/test_epub_extraction_quickwins.py tests/parsing/test_tables.py tests/parsing/test_chunks.py tests/llm/test_knowledge_job_bundles.py -vv -rA`
- `source .venv/bin/activate && cookimport stage data/input/thefoodlabCUTDOWN.epub --table-extraction on --write-markdown`
- Inspect `data/output/2026-03-13_23.01.23/tables/thefoodlabcutdown/tables.md`.

Evidence

- Focused test slice passed: `57 passed, 7 warnings in 4.02s`.
- Food Lab stage run `data/output/2026-03-13_23.01.23` wrote 6 tables, including `COMMON INGREDIENTS BY VOLUME AND MASS*`, `COMMON WEIGHT CONVERSIONS`, and `COMMON TEMPERATURE CONVERSIONS`.

Constraints/gotchas + decision

- Unstructured already exposes table HTML via `metadata.text_as_html`; use that to emit per-row blocks instead of inventing regex-only recovery.
- Conversion chapters still looked recipe-like after structure preservation, so a narrow reference-title penalty was added in recipe scoring to keep those sections in `nonRecipeBlocks`.

Rollback / compensation plan

- If the reference-title penalty proves too broad, remove that penalty and keep the structured EPUB row preservation plus table-detector changes; those are independently useful and already test-covered.
