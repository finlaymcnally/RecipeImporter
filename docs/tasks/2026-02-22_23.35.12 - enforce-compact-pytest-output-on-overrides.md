# Problem Statement

Low-noise pytest behavior depended on `pytest.ini` defaults.
Running `pytest -o addopts='' -vv ...` bypassed those defaults and brought back separator-heavy output.

# Acceptance Criteria

- Compact output remains enabled when callers pass `-o addopts=''`.
- Failure hints still include a deterministic opt-in command for full verbose debugging.
- Tests docs mention the opt-out environment variable for verbose output.

# Verification Steps

```bash
. .venv/bin/activate
pytest -o addopts='' -vv --tb=short --show-capture=all tests/labelstudio/test_labelstudio_prelabel.py::test_prelabel_span_prompt_marks_focus_window_without_block_duplication
COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1 pytest -o addopts='' -vv --tb=short --show-capture=all tests/labelstudio/test_labelstudio_prelabel.py::test_prelabel_span_prompt_marks_focus_window_without_block_duplication
pytest tests/labelstudio
```

# Evidence

- Without env opt-out, the override command no longer prints pytest header/summary separator lines.
- With `COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1`, full verbose output is restored intentionally.
- `pytest tests/labelstudio` remains green after the guard change.

# Constraints / Gotchas

- This is intentionally opinionated for AI-token efficiency: manual `-v/-vv` is clamped unless opt-out env is set.
- Use opt-out only when deep traceback/capture detail is explicitly needed.

# Decision

Add a `pytest_configure(...)` guard in `tests/conftest.py` to enforce compact terminal settings independent of `addopts`, while preserving an explicit verbose escape hatch via `COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1`.

# Rollback / Compensation

If this guard is too strict, remove `pytest_configure(...)` compact enforcement and keep only `pytest.ini` defaults + glyph suppression hook.
