Problem statement

The live Codex benchmark guardrail blocks agent-run environments entirely, which also blocks the interactive CLI menu even when a human is driving that menu locally. Interactive benchmark runs need to be allowed without reopening the non-interactive agent path.

Acceptance criteria

- Interactive CLI benchmark runs may launch live Codex-backed `labelstudio-benchmark` variants directly.
- Non-interactive agent-run `labelstudio-benchmark` executions remain blocked.
- A focused test proves the interactive-mode carve-out.

Verification steps

- `source .venv/bin/activate && pytest tests/cli/test_codex_decision_boundary.py -k 'labelstudio_benchmark_requires_benchmark_codex_confirmation or labelstudio_benchmark_live_codex_blocked_in_agent_environment or labelstudio_benchmark_live_codex_interactive_mode_bypasses_agent_block'`

Evidence

- Before the change, interactive single-offline codex variants failed with the agent-environment block even after the human selected the codex pipeline in the menu.
- After the change, the guard function skips the non-interactive agent block when `_INTERACTIVE_CLI_ACTIVE` is set.

Constraints/gotchas + decision

- Keep the carve-out narrow: interactive menu only.
- Do not weaken the non-interactive CLI requirement for `--allow-codex` and `--benchmark-codex-confirmation`.

Rollback / compensation plan

- Revert the `_INTERACTIVE_CLI_ACTIVE` carve-out if interactive runs need a different explicit-confirmation mechanism later.
