# Problem statement

`labelstudio-benchmark` split EPUB jobs failed with `cannot pickle 'module' object` while regular imports worked.
This blocked prediction import for freeform benchmark flows.

# Acceptance criteria

- Split benchmark worker payloads remain pickle-safe.
- EPUB diagnostics metadata stores `unstructured_version` as a plain string.
- Regression test covers module-valued `unstructured.__version__`.

# Verification steps

- `. .venv/bin/activate && pytest -q tests/test_epub_importer.py::test_resolve_unstructured_version_handles_module_value tests/test_labelstudio_ingest_parallel.py`
- `. .venv/bin/activate && python - <<'PY' ... pickle.dumps(_parallel_convert_worker(...)[1]) ... PY` on `data/input/saltfatacidheatCUTDOWN.epub`

# Evidence

- Targeted pytest command passed (`4 passed`).
- Manual reproduction now reports `diag_version_type str` and `pickle_ok` for split worker conversion result.

# Constraints/gotchas

- Keep existing artifact schema unchanged except normalizing metadata value type.
- Do not reduce split benchmark coverage by forcing serial fallback.

# Decision

Normalize unstructured package version through a helper that always returns a string (including module-valued version edge cases).

# Rollback / compensation plan

- Revert the helper and metadata assignment in `cookimport/plugins/epub.py`.
- Re-run split benchmark and confirm the original pickling error reproduces.
