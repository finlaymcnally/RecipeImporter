Problem statement

The live knowledge worker path is phase-first, but the repo still carries a dead batch-era helper module and generic direct-exec workspace hints that advertise `CURRENT_BATCH*` / `*-batch` commands. That leaves a stale mental model in code, tests, and docs.

Acceptance criteria

- The dead legacy knowledge batch helper module is removed from the repo.
- Generic direct-exec workspace metadata no longer mirrors or advertises `CURRENT_BATCH*` knowledge batch files or `complete-batch` / `check-batch` / `install-batch`.
- Knowledge phase workspaces still advertise the current phase-first helper loop.
- Active docs and targeted tests reflect only the phase-first knowledge contract.

Verification steps

- Run `source .venv/bin/activate && pytest tests/llm/test_codex_exec_runner_workspace.py tests/llm/test_codex_exec_runner.py tests/llm/test_knowledge_workspace_tools_packets.py`

Evidence

- Targeted LLM workspace/helper tests pass after the cleanup.

Constraints/gotchas + decision

- This is a burn-the-boats cleanup. The old batch helper path is dead code, so the change should delete it instead of deprecating it.
- Keep historical logs intact; update only active docs that describe current behavior.

Rollback / compensation plan

- Revert this change set to restore the deleted legacy module and generic batch hints if a hidden runtime dependency appears.
