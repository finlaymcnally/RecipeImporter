Problem statement

The repo-facing Codex runner and stage contract surface still uses the old transport-centric names `workspace_worker` and `structured_json` in methods, mode values, and telemetry. This change renames the active contract surface to `taskfile` and `packet` so future work can build on the decided naming without a separate rename pass.

Acceptance criteria

- Repo-facing runner methods use `run_taskfile_worker(...)` and `run_packet_worker(...)`.
- Repo-facing worker contract values use `taskfile` and `packet` instead of `workspace_worker` and `structured_json`.
- Stage/runtime telemetry and prompt-input mode values exposed to callers/tests use the new names where they refer to the worker contract.
- Relevant docs and focused tests match the renamed contract surface.
- Low-level helper implementation names may stay literal where they are explicitly about the current fenced workspace machinery rather than the public contract.

Verification steps

- Run focused pytest coverage for the direct-exec runner, recipe/knowledge/line-role runtimes, and prompt-budget/reporting surfaces touched by the rename.
- Run a final targeted `rg` over the touched repo-facing surface to confirm the old names were removed there.

Evidence

- Targeted pytest after the rename:
  - `tests/llm/test_codex_exec_runner.py tests/llm/test_codex_exec_runner_workspace.py tests/llm/test_recipe_phase_workers.py tests/llm/test_knowledge_orchestrator_runtime_leasing.py tests/llm/test_knowledge_orchestrator_runtime_progress.py tests/parsing/test_canonical_line_roles_prompting.py tests/parsing/test_canonical_line_roles_runtime.py tests/parsing/test_canonical_line_roles_runtime_recovery.py tests/labelstudio/test_labelstudio_benchmark_helpers_prompt_budget.py`: `166 passed`
- Targeted search over the active contract surface returned no remaining matches for the retired runner/mode names:
  - `rg -n 'run_structured_prompt|run_workspace_worker|"structured_json"|"workspace_worker"|workspace_worker_row_count|workspace_worker_session_count|main_workspace_workers' cookimport tests docs/10-llm docs/12-testing docs/tasks/'2026-04-01_21.54.27 - taskfile-packet-rename.md'`

Constraints/gotchas + brief decision

- Do not overwrite unrelated in-flight edits already present in the worktree.
- Rename the public contract first; keep deep helper module/function names literal where they are implementation-specific and not part of the new worker-contract vocabulary.

Rollback / compensation plan

- Revert the rename change atomically if downstream code or tests show that an old contract string is still required for a live boundary we missed.
