# Task: Label Studio prelabel + decorate

## Problem statement

Freeform golden-set creation is slow because all spans are manual. We need first-pass AI labeling for import runs and an additive decorate path for existing projects.

## Acceptance criteria

- `labelstudio-import --task-scope freeform-spans --prelabel` can generate completed annotations and upload them.
- Import handles inline annotation rejection by falling back to post-import annotation creation.
- `labelstudio-decorate` can add requested label types additively without deleting prior annotations.
- Decorate supports `--no-write` dry run and writes report artifacts.

## Verification steps

- `source .venv/bin/activate`
- `pytest -q tests/test_labelstudio_prelabel.py tests/test_labelstudio_ingest_parallel.py tests/test_labelstudio_benchmark_helpers.py`
- `pytest -q tests/test_labelstudio_freeform.py`

## Evidence

- New tests passed locally:
  - `tests/test_labelstudio_prelabel.py`
  - `tests/test_labelstudio_ingest_parallel.py`
  - `tests/test_labelstudio_benchmark_helpers.py`
  - `tests/test_labelstudio_freeform.py`

## Constraints / gotchas

- Offset integrity depends on exact `segment_text` + `source_map.blocks` positions.
- Label Studio inline `annotations` upload support can vary by version/config.
- Codex CLI output may wrap JSON in prose, so parser must extract first JSON payload robustly.

## Decision

- Use block-index labeling from LLM, never raw offsets.
- Default upload mode is completed `annotations`, keep `predictions` as explicit fallback/debug mode.
- Decorate is additive: new merged annotation with metadata marker for idempotence.

## Rollback / compensation

- Disable `--prelabel` and use standard import if provider behavior is unstable.
- Use `labelstudio-decorate --no-write` to inspect projected changes before writing.
- If decorate output is bad, prior human annotations remain intact (new annotation can be ignored/deleted in LS UI).
