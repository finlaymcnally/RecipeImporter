# Reduce Line-Role Codex Prompt Overhead

## Purpose / Big Picture

After this change, prompt preview will stop hiding the difference between the actual line-role task prompt and the extra Codex-farm transport wrapper, and the default `codex-line-role-v1` execution path will use fewer, cheaper prompts. A human should be able to run prompt preview for an existing benchmark run and see separate task-versus-wrapper budget numbers, then compare the new line-role preview against the old one and observe lower prompt counts and lower estimated token totals without changing the labeling contract.

## Surprises & Discoveries

- Observation: the current processed run the user pointed at never spent the reported line-role tokens because it still used deterministic line-role.
  Evidence: `data/output/2026-03-16_20.23.25/single-offline-benchmark/saltfatacidheatcutdown/line_role_only/2026-03-16_20.23.34/saltfatacidheatcutdown.excel_import_report.json` shows `line_role_pipeline: deterministic-v1`.

- Observation: the prompt preview inflation is not just preview-only drift. The live line-role Codex path also writes JSON input files containing the full prompt and then re-embeds that JSON into the pipeline template.
  Evidence: `cookimport/parsing/canonical_line_roles.py::_run_line_role_prompt_via_codex_farm` writes `json.dumps({"prompt": prompt})`, and `llm_pipelines/prompts/line-role.canonical.v1.prompt.md` inserts `{{INPUT_TEXT}}`.

- Observation: for the user’s preview artifact, transport wrapping added more text than the inner line-role prompts themselves.
  Evidence: inner line-role prompts totaled 852,478 chars while wrapped prompts totaled 1,822,601 chars; wrapper overhead was 970,123 chars.

- Observation: once the JSON envelope was removed and the batch size was raised, line-role stopped dominating the preview budget; knowledge harvest became the obvious remaining cost center.
  Evidence: regenerated preview reduced line-role from 45 calls / ~683k estimated total tokens to 15 calls / ~291k, while knowledge harvest still accounted for 313 calls / ~683k estimated total tokens.

## Outcomes & Retrospective

The transport simplification and larger shared batch default both landed cleanly. Prompt preview now reports the distinction between task chars and wrapper overhead instead of hiding them inside one rendered-char number, and line-role no longer pays the huge double-embedding tax. The measured regeneration against the user’s benchmark root confirmed the user-visible goal: line-role prompt count dropped from 45 to 15 and its estimated total tokens dropped from ~683,476 to ~290,891.

The remaining overall-budget problem is no longer line-role. Knowledge harvest still produces 313 prompts on the same cutdown book, so any further “make this cheap” work should probably target `codex-farm-knowledge-v1`.

## Context and Orientation

Canonical line-role labeling lives in `cookimport/parsing/canonical_line_roles.py`. When `RunSettings.line_role_pipeline` is `codex-line-role-v1`, the runtime labels all atomic lines deterministically first, then sends bounded batches to Codex for correction. The prompt text for one batch is built in `cookimport/llm/canonical_line_role_prompt.py`.

The Codex transport for this stage currently goes through Codex Farm, a local runner that executes pipeline definitions from `llm_pipelines/`. The line-role pipeline definition is `llm_pipelines/pipelines/line-role.canonical.v1.json`, and its prompt template is `llm_pipelines/prompts/line-role.canonical.v1.prompt.md`. Today the runtime writes a JSON file containing the whole inner prompt under key `prompt`, then the transport template repeats that JSON file contents as the actual user prompt. That is the largest avoidable source of overhead.

Prompt preview for existing runs lives in `cookimport/llm/prompt_preview.py`. It rebuilds hypothetical prompt payloads and then calls `cookimport/llm/prompt_budget.py` to estimate cost from rendered prompt text length. Right now the summary only reports rendered prompt chars, which hides how much of the total comes from wrapper overhead.

The targeted regression tests live in `tests/llm/test_prompt_preview.py`, `tests/parsing/test_canonical_line_roles.py`, and `tests/llm/test_codex_farm_orchestrator_runner_transport.py`.

## Plan of Work

First, change the line-role transport to use raw prompt text files instead of JSON envelopes. In `cookimport/parsing/canonical_line_roles.py::_run_line_role_prompt_via_codex_farm`, write `prompt` directly to the temporary `.json` input file rather than wrapping it in `{"prompt": ...}`. In `llm_pipelines/prompts/line-role.canonical.v1.prompt.md`, collapse the transport template so it simply forwards `{{INPUT_TEXT}}` and relies on the output schema for the outer `rows` object. Mirror that same behavior in `cookimport/llm/prompt_preview.py` by passing the raw prompt text as the serialized input for line-role preview rows so the preview matches live execution.

Next, reduce prompt count by defining one shared default batch-size constant in `cookimport/parsing/canonical_line_roles.py` and using it everywhere the line-role Codex plan or preview currently hardcodes `40`. Pick a larger default that materially lowers prompt count while keeping per-prompt payloads bounded; the implementation will validate this with current tests and updated preview expectations.

Then, extend preview budget reporting in `cookimport/llm/prompt_budget.py` so each stage tracks both task prompt chars and wrapped prompt chars. Add explicit wrapper-overhead totals and render them in the Markdown summary. Preserve existing top-level shape where practical so downstream code keeps working, but make the new fields visible and test-covered.

Finally, update narrow docs in the parsing reference and add or refresh tests that prove: line-role preview uses the raw task prompt as its inner prompt measure, wrapper overhead is reported explicitly, and the default line-role prompt count drops when batching more lines per call.
