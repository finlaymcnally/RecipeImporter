# Task

## Problem

## Acceptance criteria

- `final_recipe_empty_mapping` reflects only actual final-recipe empty mapping output.
- `analysis.recipe_pipeline_context.observed_recipe_stage_call_counts.build_final_recipe` counts only observed final-recipe calls.
- `16_baseline_trace_parity.json` recognizes derived bundle-local trace artifacts reflected in run diagnostics.

## Verification

- `source .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py tests/bench/test_upload_bundle_v1_existing_output.py -q`

## Evidence

- New regression tests cover:
  - correction-empty rows without final-recipe output staying out of `final_recipe_empty_mapping`
  - topology call counts not inventing final-recipe calls from correction calls
  - bundle trace parity aligning with derived run diagnostics for virtual codex-only trace artifacts

## Constraints / decision

- Keep the change narrow to upload-bundle diagnostic generation and its tests.
- Do not alter benchmark scoring or Oracle upload behavior.
