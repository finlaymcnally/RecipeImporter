# Task: codex-farm progress active noise

## Problem statement

Codex-farm progress events included `active <filename>` in status text. In plain-progress terminals this changed almost every callback tick and produced noisy, low-signal line spam.

## Acceptance criteria

- Codex-farm progress callback messages still report `task X/Y`, running count, and error count.
- Volatile `active ...` suffix is not emitted in callback status messages.
- Duplicate messages with unchanged counters are deduped by existing callback logic.
- Regression coverage proves same-counter/different-active events emit one `task 0/N` line.

## Verification steps

Run from repo root with venv active:

```bash
. .venv/bin/activate
pytest tests/llm/test_codex_farm_orchestrator.py -q
pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py::test_run_with_progress_status_defaults_to_plain_for_agent_env -q
```

Fail-before signal used while adding the regression assertion:

```bash
. .venv/bin/activate
COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1 pytest tests/llm/test_codex_farm_orchestrator.py::test_subprocess_runner_emits_progress_callback_from_progress_events -o addopts='' -vv --tb=short --show-capture=all --assert=rewrite
```

## Evidence

- `tests/llm/test_codex_farm_orchestrator.py` now simulates two progress events with identical counters but different `running_tasks` inputs and asserts one `task 0/2` emission.
- `cookimport/llm/codex_farm_runner.py` no longer appends `active ...` to formatted progress event status.
- `cookimport/README.md` and `docs/07-bench/07-bench_README.md` document the noise-reduction behavior.

## Constraints / gotchas and decision

- Constraint: keep user-visible progress counters (`task X/Y`) unchanged for ETA and status tracking.
- Decision: remove only the volatile active-file suffix while preserving task/running/error fields.

## Rollback / compensation plan

Revert this task diff to restore active-file suffixes in codex-farm callback progress messages.
