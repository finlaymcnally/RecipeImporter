# Problem

Benchmark trend charts were hardcoded to two metrics (`strict_accuracy`, `macro_f1_excluding_other`), so users could not add/remove arbitrary fields.

# Acceptance Criteria

- Dashboard shows controls to add/remove any number of numeric benchmark fields from the trend chart.
- Trend charts keep existing behavior for paired variant splitting, run-group x-axis alignment, and trend overlays.
- Selected trend fields persist in dashboard UI state.
- Dashboard renderer tests cover the new controls/state/series behavior.

# Verification

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`
- Confirm new test `test_benchmark_trend_supports_arbitrary_selected_numeric_fields` passes.

# Evidence

- Test run result: `69 passed in 8.62s`.

# Constraints / Gotchas

- Trend rendering is shared by both chart hosts (`benchmark-trend-chart` and `compare-control-trend-chart`), so field selection must flow through one shared series builder.

# Decision

Added a state-backed `Trend fields` checklist with `Select all` and `Clear`, normalized selected fields against numeric field options, and fed that selection into the existing shared trend-series build path.
