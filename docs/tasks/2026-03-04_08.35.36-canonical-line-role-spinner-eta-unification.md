## Problem statement
Canonical line-role stage could run for a while with only elapsed time (no ETA) because deterministic work emitted no `task X/Y` callbacks. Spinner also surfaced noisy `active workers: 0` rows.

## Acceptance criteria
- Canonical line-role emits `task X/Y` progress during deterministic labeling.
- Codex batch progress still emits `task X/Y | running N`.
- Shared spinner suppresses standalone `active workers: 0` rows.
- Tests cover deterministic and codex callback progress plus spinner zero-worker behavior.

## Verification steps
- `source .venv/bin/activate && pytest -q tests/parsing/test_canonical_line_roles.py -k "codex_progress_callback_reports_batch_counts or deterministic_progress_callback_reports_task_counts"`
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers_progress.py -k "clears_codex_worker_state_for_new_phase or hides_zero_active_workers_row or shows_eta_for_canonical_line_role_progress"`
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers_progress.py tests/parsing/test_canonical_line_roles.py`

## Evidence
- Updated `label_atomic_lines(...)` to emit deterministic and codex task counters through shared callback plumbing.
- Updated spinner worker summary rendering to hide zero-only worker rows.
- Added regression tests for deterministic progress callbacks and zero-worker row suppression.

## Constraints / decision
- Keep progress callback volume bounded with interval-based deterministic updates (max ~100 updates per run) to preserve responsiveness without callback spam.

## Rollback / compensation
- Revert deterministic callback emission in canonical line-role helper and restore prior worker summary rendering behavior.
