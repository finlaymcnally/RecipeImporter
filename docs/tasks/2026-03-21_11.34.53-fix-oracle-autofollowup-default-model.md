Problem statement

Background Oracle benchmark upload could print a false "upload not started" warning when only the auto-followup subprocess launch failed.
The concrete failure case was a blank model override passing `None` into the follow-up worker command.

Acceptance criteria

- Interactive benchmark wrap-up still starts Oracle upload when no explicit Oracle model override is chosen.
- Auto-followup launch uses the resolved Oracle model instead of a `None` CLI argument.
- If Oracle upload starts but the follow-up worker launch fails, the terminal reports the follow-up failure without claiming the upload never started.

Verification steps

- Activate the project venv.
- Run `pytest tests/bench/test_benchmark_oracle_upload.py -q`.

Evidence

- Added regression coverage for the blank-model auto-followup launch path.
- Added regression coverage for accurate terminal messaging when only the follow-up worker launch fails.

Constraints / decision

- Kept scope to the background benchmark Oracle launch path only.
- Reused the resolved `launch.model` so turn 2 matches the actual turn-1 model selection.

Rollback / compensation plan

- Revert this change if background Oracle launch behavior regresses, then use `cookimport bench oracle-upload ...` and `cookimport bench oracle-followup ...` manually for affected runs.
