Problem statement

Purge the remaining pre-refactor runtime/tooling compatibility seams identified in `docs/plans/2026-03-16_13.47.23-burn-the-boats-removal-audit.md`.
This change is intentionally destructive: new runs should teach only the current label-first, semantic-stage contract.

Acceptance criteria

- Label-first runtime no longer exposes bridge types/fields framed as compatibility layers.
- Hidden run-setting and CLI defaults stop using public values named `legacy` where current alternatives already exist.
- Benchmark and prompt tooling stop writing legacy alias artifacts/fields for new runs.
- Analytics/history readers are either removed or explicitly retained as an intentional exception.

Constraints/gotchas + brief decision

- The workspace already has unrelated in-flight edits in several purge-adjacent files, so edits must preserve those changes while removing only the audited seams.
- This is a hard cut, so tests/fixtures should move to the new contract instead of keeping adapters alive.
- Analytics/history cleanup is intentionally out of scope for this purge. `.history/performance_history.csv` and old dashboard CSV support stay preserved by explicit user instruction.

Rollback / compensation plan

- Revert the purge as one atomic change if an archived reader or old bundle contract turns out to still be required.
