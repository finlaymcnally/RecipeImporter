# Problem

Interactive run settings had two separate problems:

- stage/benchmark plumbing could drop the selected Codex prompt-target counts before the live run config was built
- benchmark interactive prompts asked for counts in `recipe, line_role, knowledge` order while runtime stages execute `line_role, recipe, knowledge`, which made operator-entered triplets easy to misapply

Recipe also still treated `recipe_prompt_target_count` as a preferred worker-session count instead of the requested recipe shard count.

# Acceptance Criteria

- Interactive `RunSettings` prompt-target counts are forwarded through stage and benchmark adapters.
- `stage`, `labelstudio-benchmark`, `run_labelstudio_import`, and `generate_pred_run_artifacts` preserve those counts in the live run config.
- Benchmark interactive Codex prompts ask for counts in runtime stage order: line-role, recipe, knowledge.
- Recipe planning uses `recipe_prompt_target_count` as the requested recipe shard count.
- Focused regression tests prove the benchmark ordering and recipe shard-count behavior.

# Verification

- `source .venv/bin/activate && pytest tests/llm/test_recipe_phase_workers.py tests/llm/test_run_settings.py tests/cli/test_c3imp_interactive_menu.py tests/cli/test_run_settings_adapters.py tests/labelstudio/test_labelstudio_benchmark_helpers_single_offline_run.py -q`

# Evidence

- Before the fix, focused tests failed because recipe planning still emitted one shard per recipe and benchmark interactive prompt ordering encouraged operator-entered triplets to land on the wrong stage.
- After the fix, the verification command exits `0`.

# Constraints / Decision

- Keep the fix scoped to prompt-target-count plumbing plus the operator-facing benchmark prompt ordering and recipe shard-planning contract.
- Preserve current default behavior (`5`) when callers do not override the counts.
- Keep the new CLI seams hidden/internal; the user-facing interactive workflow remains the main product surface.

# Rollback

- Revert the stage/benchmark prompt-target plumbing changes and the focused regression tests.
