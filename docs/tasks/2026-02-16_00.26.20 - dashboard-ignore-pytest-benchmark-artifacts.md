# Problem statement

`Recent Benchmarks` included rows generated by local Python tests, where CSV benchmark artifact paths looked like pytest temp folders (`pytest-46/test_.../eval`).
This polluted the dashboard with non-user benchmark data.

# Acceptance criteria

- Dashboard collector excludes benchmark rows whose artifact paths match pytest temp eval/prediction layout.
- Normal benchmark rows remain visible.
- Stats dashboard tests cover this regression.

# Verification steps

- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py -k pytest_temp_eval_artifacts`
- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py`

# Evidence

- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py -k pytest_temp_eval_artifacts` -> `1 passed, 27 deselected`.
- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py` -> `28 passed in 0.17s`.

# Constraints/gotchas

- Keep filtering narrow to pytest temp eval/prediction path layout so normal benchmark runs are not hidden.
- Preserve JSON + CSV benchmark merge behavior by artifact directory.

# Decision

Add a pytest-path detector in `dashboard_collect.py` and skip matching benchmark records during CSV and JSON benchmark collection.
