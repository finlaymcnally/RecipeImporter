Problem statement

The benchmark suite lacked a cheap smoke layer that runs the real interactive single-offline benchmark wiring and catches runtime crashes before deep helper tests or manual CLI runs do. The coverage needs to stay fully offline and token-free.

Acceptance criteria

- A dedicated smoke test file runs the real interactive single-offline benchmark flow for both vanilla and codex-shaped settings.
- The smoke tests exercise actual menu routing and `_interactive_single_offline_benchmark` variant planning.
- The outer `labelstudio_benchmark(...)` boundary is stubbed so no real CodexFarm work or token spend can occur.
- `pytest -m smoke` includes the new benchmark smoke file.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_smoke.py`
- `source .venv/bin/activate && pytest -m smoke -k labelstudio_benchmark_smoke`

Evidence

- The vanilla smoke proves the real interactive benchmark flow reaches the offline single-offline runner and completes one variant.
- The codex-shaped smoke proves the same flow survives paired variant planning and completes both variants without invoking real CodexFarm execution.

Constraints/gotchas + decision

- Keep the smoke tests broad at the interactive/helper layer, but stub `labelstudio_benchmark(...)` itself.
- This intentionally catches setup/runtime crashes like variant-planning failures without depending on external services or spending tokens.

Rollback / compensation plan

- Revert the smoke file and smoke-marker registration if it becomes flaky, but keep narrower variant-planning regressions in place.
