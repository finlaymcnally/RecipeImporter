A reviewer should be able to run the existing benchmark command, inspect the `prediction-run/line-role-pipeline/stage_block_predictions.json` artifact, and see pass4-driven `KNOWLEDGE` labels in the same scored surface as recipe labels.

## Outcomes & Retrospective

Canonical-text benchmark runs with line-role enabled now score pass4 knowledge on the preferred projection surface instead of silently dropping it. The implementation stayed small because the evaluator already consumes a generic stage/evidence pair; the only required change was to enrich `line-role-pipeline/stage_block_predictions.json` and `freeform_span_predictions.jsonl` before they are scored. The result is covered by one projection test, one canonical-text scoring test, and one bundle-selection test.

## Context and Orientation

The current benchmark stack has two relevant pieces. First, stage-time deterministic evidence is written by `cookimport/staging/stage_block_predictions.py`. That path already knows how to read pass4 `snippets.jsonl`, but it only uses snippet provenance block indices and produces the legacy `stage_block_predictions.json` artifact. Second, canonical-text benchmark runs with line-role enabled prefer the line-role projection artifacts under `prediction-run/line-role-pipeline/`. Those files are built by `cookimport/labelstudio/canonical_line_projection.py`. Today that projection code writes a new `stage_block_predictions.json` containing only the canonical line-role labels, so pass4 knowledge disappears from the preferred scored artifact.

The evaluator itself, `cookimport/bench/eval_canonical_text.py`, only needs a standard `stage_block_predictions.json` and a matching `extracted_archive.json`. It does not know or care whether those labels came from deterministic recipe extraction, line-role projection, or pass4 enrichment. That means the safest fix is to change artifact generation, not the evaluator API.

Pass4 knowledge outputs live in `knowledge/<workbook_slug>/snippets.jsonl`. Each snippet row includes `provenance.block_indices` and `evidence[*].block_index` plus quoted evidence text. The line-role projection archive rows already preserve the original source block index, so we can use that bridge to decide which projected lines should carry `KNOWLEDGE`.

## Idempotence and Recovery

This change is additive and local to artifact generation. Re-running the same prediction run will rewrite the same projection artifacts deterministically from the same line-role predictions and pass4 snippets. If the snippets path is missing or unreadable, the projection writer should fall back to the current line-role-only behavior.
