## Outcomes & Retrospective

The Label Studio benchmark-helper area no longer has 3k-4k line monoliths. Scheduler and eval-payload are split by behavior, and the old giant support file is gone. The remaining larger Label Studio files are outside this specific helper cluster (`test_labelstudio_ingest_parallel.py`, `test_labelstudio_freeform.py`, and to a lesser extent `test_labelstudio_benchmark_helpers_progress.py`), so a future cleanup should target those separately rather than folding them back into this refactor.

## Context and Orientation

The current Label Studio benchmark-helper surface lives under `tests/labelstudio/`. These tests cover import/eval helpers, interactive benchmark flows, all-method scheduler logic, compare payload rules, and single-profile benchmark routing. The repo assigns markers by filename in `tests/conftest.py`, so every new `test_*.py` file must be added there or it will fall back to `core`. Live test-running guidance lives in `tests/README.md` and `docs/12-testing/12-testing_README.md`.

The problematic files are large for different reasons. `tests/labelstudio/test_labelstudio_benchmark_helpers_scheduler.py` mixes target resolution, variant generation, scheduler math, prediction reuse, execution, multi-source batching, and interactive all-method entrypoints. `tests/labelstudio/test_labelstudio_benchmark_helpers_eval_payload.py` mixes compare-mode acceptance logic, benchmark output-root behavior, pipelined execution behavior, timing/prune behavior, and extractor validation. `tests/labelstudio/benchmark_helper_cases.py` is a non-test support module that still contains dozens of real test case bodies plus shared helper functions.

## Plan of Work

First, split the scheduler file by behavior. Create separate modules for all-method target/variant construction, scheduler planning/runtime math, prediction reuse/execution, and multi-source or interactive all-method flows. Update `tests/conftest.py`, `tests/README.md`, and `docs/12-testing/12-testing_README.md` so the new filenames are first-class supported entrypoints.

Next, split eval-payload by behavior. Create separate modules for compare payload acceptance, benchmark prediction/eval artifact behavior, pipelined streaming behavior, and cleanup/extractor validation. Keep shared helper imports coming from a small non-test support module instead of another `test_*.py` file.

Finally, replace `tests/labelstudio/benchmark_helper_cases.py` with a small helper module that only stores shared imports and helper writers, plus several small case modules or direct pytest files containing the actual tests. The end state should not leave one huge support file as a substitute mega-file.

## Idempotence and Recovery

This refactor is safe to repeat because it only rearranges test code and documentation. If a generated or moved file lands in the wrong group, rerun collect-only to catch import or discovery errors, then move the test body without touching runtime code. Historical docs logs should not be rewritten; only current-source docs should be updated.
