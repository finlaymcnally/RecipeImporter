Problem statement

The interactive benchmark gold-export picker shows long relative paths like `pulled-from-labelstudio/<book>/exports/freeform_span_labels.jsonl`.
That makes the menu harder to scan than necessary when the operator mostly needs the book identity.

Acceptance criteria

- The interactive freeform gold export picker shows discovered exports as the book slug when the path follows the normal `.../<book>/exports/<file>` layout.
- Existing fallback behavior remains intact for paths outside that layout.
- Targeted CLI benchmark helper tests cover the shortened display.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_import_eval.py::test_display_gold_export_path_relative_to_golden_root -q`

Evidence

- Before the change, the targeted test failed after updating the expected label to `dinnerfor2cutdown` because the helper still returned the longer relative path.
- After the change, the same targeted test passes with the shortened label format.

Constraints/gotchas + decision

- Keep the change localized to the display helper used by the prompt instead of rewriting prompt construction or export discovery.
- Preserve the old relative-path fallback for any nonstandard export locations.
