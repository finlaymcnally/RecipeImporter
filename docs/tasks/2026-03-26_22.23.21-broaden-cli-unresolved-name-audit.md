## Problem

The repo has flushed out several `NameError` regressions one at a time because the unresolved-name audit only covered selected benchmark and interactive modules.
That means a newly split helper or command module can still ship a missing binding until a human happens to hit the live branch.

## Acceptance Criteria

- The undefined-name test discovers CLI support/command modules automatically instead of relying on a short hand-maintained subset.
- The audit still bootstraps the benchmark facade before checking split benchmark modules.
- Focused pytest verification passes in `.venv`.

## Verification

- `. .venv/bin/activate && pytest tests/core/test_benchmark_undefined_names.py -q`

## Evidence

- A broad manual sweep across `cookimport.cli_support.*` and `cookimport.cli_commands.*` is clean in the current tree.
- After this change, the same protection runs in pytest instead of depending on manual one-off scripts.

## Constraints / Decision

- Keep this static and cheap; do not replace runtime smoke tests with the audit.
- Prefer broad discovery plus a small explicit skip list only if false positives appear.
