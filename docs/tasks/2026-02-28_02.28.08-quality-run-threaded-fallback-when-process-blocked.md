## Problem statement

QualitySuite runs in restricted environments were entering effectively serial all-method execution because process-pool workers were unavailable.

## Acceptance criteria

- quality-run detects process-pool unavailability before all-method round execution.
- When unavailable and scheduler scope is `global`, quality-run switches to `legacy` scope automatically.
- For multi-source rounds, quality-run enforces `max_parallel_sources >= 2` (up to source count) when prior config would be serial.
- Behavior is covered by tests and docs mention the runtime adaptation.

## Verification steps

- Run: `. .venv/bin/activate && pytest tests/bench/test_quality_suite_runner.py -q -k process_workers_unavailable`
- Run: `. .venv/bin/activate && pytest tests/bench/test_quality_suite_runner.py -q -k schema_v2_levers_expand_and_pass_runtime_knobs`

## Evidence

- `cookimport/bench/quality_runner.py` adds runtime adaptation and passes effective runtime to all-method invocation.
- `tests/bench/test_quality_suite_runner.py` adds a coverage test for process-worker-unavailable fallback to legacy source-thread scheduling.
- Bench/CLI docs updated in `cookimport/bench/README.md`, `docs/02-cli/02-cli_README.md`, and `docs/07-bench/07-bench_README.md`.

## Constraints and decisions

- Keep experiment loop behavior unchanged (still sequential by experiment id).
- Apply adaptation only when requested scope is `global`; explicit `legacy` requests are untouched.
- Preserve existing all-method runtime knobs unless overriding is required to avoid serial source scheduling.

## Rollback plan

- Revert the quality-run runtime adaptation helper and restore direct all-method runtime pass-through.
