# Preserve Oracle Pro Follow-Up Model

## Problem

Benchmark Oracle chats could visibly start on the shared browser profile's current ChatGPT model before settling onto the intended review model.
Turn 2 could also continue a benchmark chat without pinning the concrete turn-1 browser model.

## Acceptance Criteria

- Automatic benchmark Oracle follow-up reuses the saved concrete turn-1 browser model when that metadata is available.
- Benchmark Oracle turn 1 uses a dedicated browser profile per concrete review model by default, so benchmark reviews do not inherit the shared automation profile's current picker state.
- Explicit `--model` overrides for follow-up still take precedence.
- Regression tests cover explicit follow-up overrides, saved-model reuse, and the dedicated benchmark profile path.

## Verification

- `source .venv/bin/activate && pytest tests/bench/test_oracle_followup.py tests/bench/test_benchmark_oracle_upload.py -q`

## Evidence

- Added a follow-up regression test that expects turn 2 to pass `--browser-model-strategy select --model gpt-5-pro` when turn 1 saved `launch_model: gpt-5-pro`.
- Existing explicit-override coverage still checks that `instant` maps to the concrete browser-visible model.
- Benchmark upload tests now verify the dedicated benchmark browser profile path derived from the concrete review model.

## Constraints / Decisions

- Keep the fix local to benchmark Oracle upload/follow-up; do not change generic Oracle wrapper behavior for unrelated uses.
- Prefer saved launch metadata over guessing from the browser's current picker state.
- Isolate benchmark review chats from the normal shared Oracle ChatGPT profile instead of trying to change Oracle's internal page-open timing from this repo.
