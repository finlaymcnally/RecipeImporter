## Problem statement

`docs/understandings/2026-02-27_10.52.18-speed1-2-spec-audit.md` identified remaining OG speed1-2 gaps: explicit override safety, configured/effective headroom visibility, smart-admission cap semantics, and missing test/doc coverage for those contracts.

## Acceptance criteria

- `_resolve_all_method_scheduler_runtime(...)` exposes configured/effective eval-tail headroom, headroom mode, CPU-budget context, and max-active-during-eval values.
- Explicit `all_method_max_eval_tail_pipelines` overrides are bounded by per-source CPU budget (and remaining variants), not only by total variants.
- Smart scheduler admission/pool sizing follows `max_active_during_eval = configured_inflight + eval_tail_headroom_effective`.
- Per-source and multi-source scheduler reports include configured/effective headroom + max-active fields (legacy aliases may remain).
- Focused scheduler tests cover CPU-bound explicit override behavior and updated runtime/report semantics.

## Verification steps

From repo root:

1. `source .venv/bin/activate`
2. `PIP_CACHE_DIR=/tmp/.pip-cache python -m pip install -e '.[dev]'`
3. `pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers.py`
4. `pytest -q tests/labelstudio/test_labelstudio_ingest_parallel.py`
5. `pytest -q tests/bench`

## Evidence

- Editable install completed successfully in `.venv`.
- `pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers.py` exited `0` after scheduler-threshold rebalance for new cap semantics.
- `pytest -q tests/labelstudio/test_labelstudio_ingest_parallel.py` exited `0`.
- `pytest -q tests/bench` exited `0`.

## Constraints and decision

- Constraint: no changes to canonical scoring algorithm or metric definitions.
- Decision: keep changes scheduler-only (`cookimport/cli.py`) with additive report fields and backward-compatible alias keys.
- Constraint: OG speed1-2 asked for configured vs effective headroom observability.
- Decision: add explicit scheduler payload fields (`eval_tail_headroom_mode`, `eval_tail_headroom_configured`, `eval_tail_headroom_effective`, `max_active_during_eval`) and retain `max_eval_tail_pipelines`/`smart_tail_buffer_slots` as compatibility aliases.

## Rollback / compensation plan

- Set `all_method_smart_scheduler=false` to return to fixed refill behavior.
- Set `all_method_max_eval_tail_pipelines=0` (or low value) to reduce eval-tail concurrency pressure.
- Revert scheduler contract changes in:
  - `cookimport/cli.py`
  - `tests/labelstudio/test_labelstudio_benchmark_helpers.py`
  - benchmark docs updated in this task
