## Problem
`Previous Runs` already has strong `Metric help`, but metric/data labels across the dashboard do not expose that context inline on hover.

## Acceptance Criteria
- Hovering metric/data labels for about 1 second shows a tooltip description.
- Tooltip descriptions are sourced from `Metric help (table)` first, with metadata fallback.
- Coverage includes Previous Runs headers/checklists plus runtime/per-label/compare-control metric-name surfaces.
- Coverage also auto-tags repeated metric text instances without explicit metric keys (for example inline metric code tokens and chart axis/legend metric labels).
- Boundary Classification category labels use the same dark delayed tooltip (not browser-native title tooltips).
- Tooltip-enabled labels get a subtle smooth hover accent with no underline.
- Dashboard analytics tests cover tooltip seams.
