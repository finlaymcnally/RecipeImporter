Problem statement

The single-offline `codexfarm` benchmark variant crashes before evaluation with `NameError: name 'os' is not defined`.
The failure happens only on the Codex line-role path, so vanilla still completes and the comparison artifact is skipped.

Acceptance criteria

- The Codex line-role helper no longer throws `NameError` when reading its env-driven max-inflight setting.
- A narrow non-slow regression test covers that helper so routine fast runs catch future import cleanup mistakes before a live Codex benchmark does.

Verification steps

- `source .venv/bin/activate && python - <<'PY'`
  `from cookimport.parsing import canonical_line_roles as m`
  `print(m._resolve_line_role_codex_max_inflight())`
  `PY`
- `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_role_env.py -q`

Evidence

- Before the fix, calling `_resolve_line_role_codex_max_inflight()` raised `NameError: name 'os' is not defined`.
- After the fix, the helper returns the configured integer and the fast regression test passes.

Constraints/gotchas + decision

- Keep the fix minimal and localized to the active Codex line-role module.
- Do not widen this into benchmark or CLI refactoring; the crash is a missing module import plus a routine-suite coverage gap.
