# Task: Per-Label point-value toggle

## Problem statement
Per-Label Breakdown comparison columns were always delta-vs-baseline.
Operator requested a quick checkbox beside `Rolling N` to switch those columns between `delta` and raw `point value`.

## Acceptance criteria
- Per-Label controls include a checkbox beside `Rolling N` for point-value mode.
- All six comparison columns after run codexfarm precision/recall switch display mode together.
- Delta direction is `codexfarm baseline - comparison` so positive means codexfarm is higher.
- Mode persists in dashboard UI state reload/sync path.
- Headers/tooltips reflect delta vs point-value mode.

## Evidence

## Constraints / gotchas
- Keep codexfarm baseline columns raw.
- Keep comparison color semantics (better/worse vs codexfarm baseline) in both modes.

## Decision
Implement mode switch purely in dashboard render JS with a persisted `per_label_comparison_mode` UI-state field, keeping collector/CSV contracts unchanged.
