# Single-Offline Dashboard Auto Refresh Target

## Problem statement

Interactive `single_offline` benchmark runs were printing a dashboard update, but the refresh wrote to a nested run-local `.history/dashboard` path instead of the lifetime dashboard path (`data/.history/dashboard`).

## Acceptance criteria

- Single-offline batch refresh still runs once after all variants complete.
- Auto-refresh targets the lifetime dashboard directory for the configured output root.
- Existing benchmark CSV trigger path remains unchanged.
- Regression tests cover the refresh call payload.

## Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "single_offline_codex_enabled_runs_vanilla_then_codex_and_writes_comparison or single_offline_codex_disabled_runs_only_vanilla_and_skips_comparison"`

## Evidence

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "single_offline_codex_enabled_runs_vanilla_then_codex_and_writes_comparison or single_offline_codex_disabled_runs_only_vanilla_and_skips_comparison"` -> `2 passed`.

## Constraints/gotchas + decision

- Keep CSV-first analytics behavior intact; no new scan-only metrics.
- Decision: extend `_refresh_dashboard_after_history_write` with optional `dashboard_out_dir` and pass the lifetime dashboard path from the interactive single-offline batch refresh call.

## Rollback / compensation

- Revert `cookimport/cli.py` refresh override arguments and restore previous refresh-call assertions in `tests/labelstudio/test_labelstudio_benchmark_helpers.py`.
