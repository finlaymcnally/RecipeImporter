Problem statement

Interactive benchmark source selection asks for confirmation even when a source file is already inferred from the chosen gold export.
That adds a pointless `y/N` step to the common path without changing the outcome.

Acceptance criteria

- When interactive benchmark input resolution infers a source file from the chosen gold export, it uses that source automatically.
- The source-selection menu still appears when inference fails and no explicit source file was supplied.
- Targeted helper tests cover both the auto-infer and fallback-selection paths.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_import_eval.py::test_resolve_benchmark_gold_and_source_auto_uses_inferred_source tests/labelstudio/test_labelstudio_benchmark_helpers_import_eval.py::test_resolve_benchmark_gold_and_source_prompts_when_inference_missing -q`

Evidence

- Before the change, the auto-infer test failed because `_resolve_benchmark_gold_and_source(...)` still called `_prompt_confirm(...)` for the inferred source.
- After the change, the inferred-source path resolves immediately while the no-inference fallback still prompts for source selection.

Constraints/gotchas + decision

- Keep the change localized to benchmark input resolution instead of reworking the higher-level interactive benchmark flow.
- Do not change the manual fallback path when inference is unavailable.
