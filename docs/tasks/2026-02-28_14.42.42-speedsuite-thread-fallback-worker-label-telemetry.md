## Problem statement

In sandboxed runs where stage falls back from process workers to thread workers, worker status labels used only process identity. All thread workers reported as `MainProcess (pid)`, so `processing_timeseries.jsonl` collapsed concurrent workers into one visible worker and made runs look serial.

## Acceptance criteria

- Stage worker labels include thread identity when work executes on non-main threads.
- Main-thread/process-worker labels keep existing format for compatibility.
- A regression test covers thread-label and main-thread-label behavior.
- CLI/ingestion docs mention the thread-label behavior in telemetry.

## Constraints/gotchas and decision

- Kept the main-thread label unchanged to avoid breaking existing process-worker label expectations.
- Added only thread-name suffix because it is stable, low-noise, and sufficient to disambiguate concurrent thread workers.
