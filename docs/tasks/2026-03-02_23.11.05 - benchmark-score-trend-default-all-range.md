# Task: Benchmark Score Trend default range

## Problem statement

`Previous Runs` can show long benchmark history, but `Benchmark Score Trend` opened on a short recent range by default (`rangeSelector.selected = 1`), making history look truncated unless manually switched.

## Acceptance criteria

- Trend chart loads with full available benchmark timeline on first render.
- Previous Runs filters still drive both table and chart datasets.
- Renderer tests cover the `All` default range selector contract.

## Verification steps

1. `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k benchmark_trend_chart_uses_fixed_height`
2. `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`

## Evidence

- Renderer now emits explicit range selector buttons including `{ type: "all", text: "All" }`.
- Renderer now emits `rangeSelector.selected = 5` (the `All` button index).
- Analytics dashboard tests pass after the change.

## Constraints and decision

- Constraint: keep Highcharts stock chart behavior and existing filters/table wiring unchanged.
- Decision: define explicit range buttons and default to `All`, instead of relying on Highcharts default button set/index.

## Rollback plan

- Revert `rangeSelector` config in `cookimport/analytics/dashboard_render.py` and remove matching test/doc assertions.
