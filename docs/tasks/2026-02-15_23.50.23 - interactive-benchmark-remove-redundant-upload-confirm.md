# Problem statement

Interactive benchmark asked `Upload benchmark prediction tasks to Label Studio now?` even after the user explicitly selected upload mode.
This added a redundant step and made the flow feel contradictory.

# Acceptance criteria

- Interactive benchmark upload branch no longer asks a second upload confirmation y/n prompt.
- `eval-only` branch behavior remains unchanged.
- Interactive upload still resolves Label Studio credentials via interactive settings before invoking `labelstudio_benchmark(...)`.

# Verification steps

- Fail-before (old behavior):
  - `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_uses_golden_output_roots`
- Pass-after (new behavior):
  - `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_uses_golden_output_roots tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_eval_only_uses_existing_prediction_run`

# Evidence

- The updated benchmark interactive test now fails if any `questionary.confirm(...)` is used in upload mode.
- Post-change targeted tests pass for both upload and eval-only interactive benchmark paths.

# Constraints/gotchas

- Keep non-interactive `cookimport labelstudio-benchmark` behavior unchanged.
- Keep credential resolution flow unchanged for interactive benchmark upload.

# Decision

Treat upload-mode selection as sufficient intent; remove only the redundant confirmation prompt while preserving the rest of the branch.

# Rollback / compensation plan

- Reintroduce the upload confirmation prompt in the interactive upload branch if explicit re-confirmation is desired again.
