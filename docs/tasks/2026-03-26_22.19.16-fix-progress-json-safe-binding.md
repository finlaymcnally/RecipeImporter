## Problem

Interactive single-book benchmark runs can hit `NameError: name '_json_safe' is not defined` while writing `processing_timeseries_prediction.jsonl`.
`cookimport.cli_support.progress` moved `_append_processing_timeseries_marker(...)` into a split module but did not bind its local `_json_safe` helper, so the benchmark keeps running but silently drops marker rows behind a warning.

## Acceptance Criteria

- `cookimport.cli_support.progress` owns the `_json_safe` helper it calls.
- A focused progress-helper regression test fails on the broken baseline and passes after the fix.
- Broader undefined-name coverage continues to include the split benchmark/progress modules.

## Verification

- `pytest tests/labelstudio/test_labelstudio_benchmark_helpers_progress.py::test_progress_split_module_appends_processing_timeseries_marker_with_json_safe -q`
- `pytest tests/core/test_benchmark_undefined_names.py -q`

## Evidence

- Broken `HEAD` baseline is missing `_json_safe` in `cookimport/cli_support/progress.py`, and the focused progress-helper test is red there.
- The current tree passes the focused progress-helper test and the broader unresolved-global audit in `.venv`.

## Constraints / Decision

- Keep the fix local to the progress split module instead of relying on `cli_support.__init__` import order.
- Keep one direct runtime regression test even though the static unresolved-global audit should also catch this class of issue.
