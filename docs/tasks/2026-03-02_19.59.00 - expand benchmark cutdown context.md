## Problem

`scripts/benchmark_cutdown_for_external_ai.py` trims benchmark artifacts aggressively, which can hide useful signal for external AI review.

## Acceptance Criteria

- Default output includes more diagnostic rows and longer text excerpts.
- Sampled JSONL artifacts cover additional benchmark diagnostics (not only three canonical files).
- Sampling remains deterministic and no longer only keeps the first N rows.
- README note in `scripts/` reflects the richer defaults and knobs.

## Verification

- `python scripts/benchmark_cutdown_for_external_ai.py --help`
- `python scripts/benchmark_cutdown_for_external_ai.py <benchmark_run_or_root> --output-dir /tmp/benchmark-cutdown-more-info --overwrite --no-flatten`
- Inspect one generated run folder and confirm new `*.sample.jsonl` files are present with capped text excerpts.

## Constraints / Notes

- Keep the script deterministic.
- Preserve compact output goals while increasing useful context for model analysis.
