## Problem

Two public CLI surfaces are useless and misleading:
- `cookimport bench quality-lightweight-series` is a registered command that only exits with a disabled message.
- `cookimport inspect --write-mapping` exists only to write inferred mapping scaffold files that are not wanted anymore.

## Acceptance Criteria

- `cookimport bench --help` no longer lists `quality-lightweight-series`.
- `cookimport inspect --help` no longer lists `--write-mapping` or `--out`.
- `cookimport inspect` still prints layout guesses for a workbook path.
- Active docs stop advertising the removed command and mapping-stub workflow.

## Verification Steps

- `./scripts/test-suite.sh domain cli tests/cli/test_cli_llm_flags.py`
- `./scripts/test-suite.sh domain bench tests/bench/test_bench_quality_cli.py`
- `./scripts/test-suite.sh domain ingestion tests/ingestion/test_excel_importer.py -k mapping`

## Evidence

- `./scripts/test-suite.sh domain cli tests/cli/test_cli_llm_flags.py` -> `86 passed, 10 deselected in 7.03s`
- `./scripts/test-suite.sh domain bench tests/bench/test_bench_quality_cli.py` -> `337 passed in 28.16s`
- `./scripts/test-suite.sh domain ingestion tests/ingestion/test_excel_importer.py -k mapping` -> `1 passed, 126 deselected in 3.15s`

## Constraints / Decision

- Remove only the public dead/scaffold surfaces.
- Keep internal inferred workbook mapping data because Excel conversion still uses it when no mapping is supplied.

## Rollback / Compensation

- Revert this task's patch if operators actually still need either public surface.
