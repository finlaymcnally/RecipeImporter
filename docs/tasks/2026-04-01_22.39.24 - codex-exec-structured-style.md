Problem statement

The repo has a detailed plan for the structured packet resume runtime, but the live codex-exec path only exposes the newer transport as a replacement. This change adds the structured resume transport as an additional codex-exec style so a run can choose between the current taskfile worker contract and the new structured resume contract.

Acceptance criteria

- Codex-enabled run settings persist one explicit codex-exec style choice, with the current taskfile transport remaining the default.
- Interactive codex-enabled flows let the operator choose the codex-exec style for the run without adding a third top-level workflow family.
- When the structured style is selected, line-role and non-recipe finalize run through the structured resumed-session transport while recipe correction stays on the existing taskfile path.
- Prompt preview, fake-runner coverage, and stage/benchmark manifests describe the selected style accurately.

Verification steps

- Run focused pytest for run-settings selection, direct-exec runner resume behavior, line-role runtime, knowledge runtime, and prompt preview.
- Run targeted domain suites for the touched LLM/parsing/staging seams after focused tests pass.

Evidence

- Pending implementation.

Constraints/gotchas + brief decision

- Keep `codex-exec` vs `vanilla` as the only top-level workflow families; expose the transport choice as a codex-only sub-choice.
- Do not replace or delete the current taskfile transport; the new structured runtime is additive and selectable.
- Recipe correction stays on the current taskfile helper surface in this change.

Rollback / compensation plan

- Revert the change atomically if the new style leaks into existing default runs, breaks prompt preview, or produces ambiguous run-config/history metadata.
