# Task: Align trend timeline with table dates

## Problem statement

`Previous Runs` listed benchmark rows from 2026-02-15 onward, but `Benchmark Score Trend` appeared to start later because legacy rows lacked explicit `strict_accuracy` / `macro_f1_excluding_other` points.

## Acceptance criteria

- Trend chart timeline starts/ends at the full filtered benchmark timestamp span.
- Chart still plots only explicit benchmark metrics (`strict_accuracy`, `macro_f1_excluding_other`).
- Renderer tests assert the x-axis min/max initialization logic.

## Verification steps

1. `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k benchmark_trend_chart_uses_fixed_height`
2. `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`
3. `source .venv/bin/activate && cookimport stats-dashboard`

## Evidence

- Renderer computes timeline min/max from all filtered record timestamps and applies them via `xAxisConfig.min/max`.
- Generated dashboard keeps early dates visible even when no score point exists at those timestamps.
- Analytics dashboard tests pass.

## Constraints and decision

- Keep explicit metric plotting contract intact.
- Do not synthesize legacy score points from non-equivalent metrics.
- Decision: align timeline via x-axis bounds instead of changing score semantics.
