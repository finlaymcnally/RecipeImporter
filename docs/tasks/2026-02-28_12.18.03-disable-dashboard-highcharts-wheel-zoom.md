# Problem statement

The dashboard trend chart currently allows mouse-wheel zoom. This causes accidental zoom while scrolling. We need wheel zoom off globally for dashboard Highcharts charts, with an easy way to turn it back on later.

# Acceptance criteria

- Dashboard JS sets Highcharts global mouse-wheel zoom to disabled.
- A single clear toggle point exists to re-enable wheel zoom later.
- Existing benchmark trend chart render behavior stays intact.

# Verification steps

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k benchmark_trend_chart_uses_fixed_height`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`

# Evidence

- Targeted test: `1 passed, 46 deselected in 1.81s`
- Full dashboard analytics test file: `47 passed in 23.35s`

# Constraints/gotchas + decision

- Keep this as a global Highcharts default so any additional Highcharts charts in dashboard pages inherit the same behavior.
- Implemented via `HIGHCHARTS_MOUSE_WHEEL_ZOOM_ENABLED` constant and `Highcharts.setOptions(...)` during dashboard init.

# Rollback / compensation plan

- Fast rollback: set `HIGHCHARTS_MOUSE_WHEEL_ZOOM_ENABLED = true`.
- Full rollback: remove `applyHighchartsGlobalDefaults()` call and function from dashboard JS template.
