## Problem statement

`Compare & Control` only supported one active comparison set at a time. The requested UX needs two concurrent sets with clearer table/chart layout options.

## Acceptance criteria

- Set 2 can be expanded from the right and controls split left/right at half width.
- Set 1 and Set 2 result tables render stacked full-width with per-set visual shading.
- Charts support stacked, side-by-side, and combined layouts.
- Combined chart supports shared Y-axis or dual Y-axis (left/right) when compatible.
- Existing single-set behavior still works when Set 2 is collapsed.

2. `source .venv/bin/activate && pytest tests/analytics/test_dashboard_state_server.py`

## Evidence

- `8 passed, 71 deselected` for compare/control + dashboard contract slice.
- `2 passed` for dashboard state server coverage.

## Constraints / gotchas

- `dashboard_render.py` emits JS/CSS as Python string templates; small string changes can break JS/CSS contract assertions.
- Compare/control state load must remain backward-compatible with existing `previous_runs.compare_control` payloads.
- Combined chart mode is best-effort; mixed categorical/numeric chart mixes should fail with explicit fallback text.

## Decision

Implemented additive dual-set state (`second_set_enabled`, `second_set`, `chart_layout`, `combined_axis_mode`) while keeping Set 1 legacy fields unchanged. Added per-set control rendering + chart definition plumbing instead of duplicating analysis math.

## Rollback / compensation plan

Disable Set 2 (`second_set_enabled=false`) and keep `chart_layout=stacked` to restore single-set behavior while preserving existing compare/control analysis logic.
