# Prompt Preview Codex Command Default Resolution

## Problem

`cf-debug preview-prompts` rebuilt fallback model labels with `default_codex_model()` but did not pass the effective Codex command. That could misreport the runtime default model, and the same command-sensitive gap existed for reasoning-effort labels.

## Acceptance Criteria

- Prompt preview resolves fallback model labels with the effective Codex command.
- Prompt preview resolves fallback reasoning-effort labels with the effective Codex command.
- Existing-run preview falls back to `runConfig.codex_farm_cmd` when no explicit preview command is provided.
- The preview manifest records the effective `codex_farm_cmd`.

## Verification

- `./.venv/bin/pytest tests/llm/test_prompt_preview.py -q`

## Evidence

- Added focused prompt-preview coverage for `runConfig.codex_farm_cmd` forwarding into runtime-default model and reasoning resolution.
- Focused prompt-preview test file passes after the change.

## Decision

Thread `codex_cmd` through the preview row builder and resolve both model and reasoning defaults from one command-aware seam instead of letting preview fall back to environment-only behavior.

## Rollback

Revert the prompt-preview API/threading change and the matching test/doc updates together.
