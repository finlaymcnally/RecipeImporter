# Fix Knowledge Prompt Index Jump Assumptions

## Problem

The shard planner is allowed to honor the requested prompt count even when that means one packet contains distant source regions.
That only works if the model is told clearly that packet adjacency is weak context and that large block-index jumps should weaken grouping assumptions.

## Acceptance Criteria

- The direct knowledge prompt explicitly says packet adjacency is not semantic proof.
- The prompt tells the model to classify blocks on their own merits and to be conservative about grouping across large index jumps or abrupt topic changes.
- Workspace-worker guidance mirrors the same rule.

## Verification

- `. .venv/bin/activate && pytest tests/llm/test_knowledge_prompt_builder.py tests/llm/test_knowledge_job_bundles.py -q`
- `python -m py_compile cookimport/llm/knowledge_workspace_tools.py`

## Constraints / Decision

Keep `knowledge_prompt_target_count` authoritative for shard count.
Fix the model contract first rather than overriding the operator's chosen shard budget in planner code.
