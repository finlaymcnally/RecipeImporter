# Task: codex-farm external pack configurability wiring

## Problem statement

The recipe-side codex-farm path existed, but pass pipeline IDs were hardcoded and workspace-root control was not exposed through run settings/CLI/pred-run parity. This blocked clean integration with external codex-farm pipeline packs.

## Acceptance criteria

- Run settings define and persist `codex_farm_workspace_root` and `codex_farm_pipeline_pass1/2/3`.
- Stage + benchmark prediction generation + Label Studio import paths accept/thread those settings.
- Orchestrator invokes configured pass pipeline IDs and forwards root/workspace to runner calls.
- `llm_manifest.json` records effective pass pipeline IDs.
- No live codex-farm/token-spending execution required for verification.

## Verification steps

Run from repository root:

    source .venv/bin/activate
    python -m py_compile \
      cookimport/config/run_settings.py \
      cookimport/llm/codex_farm_runner.py \
      cookimport/llm/fake_codex_farm_runner.py \
      cookimport/llm/codex_farm_orchestrator.py \
      cookimport/cli.py \
      cookimport/labelstudio/ingest.py

    source .venv/bin/activate
    pytest -q \
      tests/test_run_settings.py \
      tests/test_cli_llm_flags.py \
      tests/test_codex_farm_orchestrator.py \
      tests/test_run_manifest_parity.py \
      tests/test_labelstudio_benchmark_helpers.py

## Evidence

- Compile step passed for all modified Python modules.
- Targeted test suite result: `52 passed, 2 warnings in 3.51s`.
- No live codex-farm subprocess run against real models was executed.

## Constraints / gotchas and decision

- Constraint: explicit request to avoid token-spending codex-farm runs.
- Gotcha: benchmark helper tests call `labelstudio_benchmark(...)` directly and required in-function normalization variables for new pipeline-id options.
- Decision: add option normalization in CLI benchmark path and extend parity tests/docs accordingly.

## Rollback / compensation plan

- Runtime rollback: run with `--llm-recipe-pipeline off`.
- Config rollback: keep defaults (`recipe.chunking.v1`, `recipe.schemaorg.v1`, `recipe.final.v1`) and unset workspace root.
- Code rollback: revert run-settings/CLI/orchestrator/runner changes in this task.
