# Task: Use non-interactive Codex CLI for prelabel/decorate

## Problem statement

First-run freeform AI prelabeling produced zero labels because the provider launched plain `codex`, which is interactive and fails in subprocess mode with `stdin is not a terminal`.

## Acceptance criteria

- Default prelabel/decorate Codex command uses non-interactive CLI mode.
- If users still configured plain `codex`, provider retries automatically with `codex exec -` on TTY error.
- CLI/docs reflect the new default command value.
- Tests cover default command and fallback retry behavior.

## Verification steps

- `. .venv/bin/activate && pytest -q tests/test_labelstudio_prelabel.py -k codex`
- `. .venv/bin/activate && pytest -q tests/test_labelstudio_prelabel.py -k prelabel_freeform_task_uses_block_offsets_and_exact_text`

## Evidence

- New tests verify:
  - default command resolves to `codex exec -`,
  - plain `codex` retry path executes `codex exec -` after the TTY failure.
- Existing prelabel offset test still passes.

## Constraints and decisions

- Keep user override surface unchanged (`--codex-cmd`, `COOKIMPORT_CODEX_CMD`).
- Limit automatic fallback to plain `codex` commands to avoid rewriting arbitrary custom command lines.

## Rollback plan

- Revert `cookimport/labelstudio/prelabel.py` fallback/default command changes and matching test/docs updates.
