# Problem

`upload_bundle_v1` needed stronger first-pass triage data and a trustworthy topline. Existing bundles could report `run_count/pair_count/changed_lines_total` as zero when root summary files were missing or stale, and default navigation still emphasized raw/heavy artifacts.

# Acceptance Criteria

- `upload_bundle_v1` remains exactly 3 files.
- Index includes verified topline counts and self-check block:
  - `starter_pack_present`, `pair_count_verified`, `changed_lines_verified`, `topline_consistent`.
- Index adds first-pass triage views:
  - per-label metrics + confusion deltas
  - per-recipe breakdown
  - stage-separated comparison
  - failure ledger
  - compact regression casebook
  - stratified changed-line sample
  - call runtime inventory (latency/tokens/cost)
  - line-role confidence/candidate signal summary
- Index provides payload row locators and alias metadata for equivalent artifacts.
- Heavy/raw artifacts are retained in payload (lossless) but explicitly deprioritized in navigation.
