# Task: Pass 4 codex-farm knowledge harvesting (recipeimport side)

## Problem statement

Add an opt-in, codex-farm-driven “pass 4” that extracts reusable cooking knowledge from **non-recipe blocks** after recipes have been processed, and writes stable JSON + Markdown artifacts into the stage run directory.

## Acceptance criteria

- Stage supports enabling pass4 via run settings + CLI:
  - `--llm-knowledge-pipeline codex-farm-knowledge-v1`
  - `--codex-farm-pipeline-pass4-knowledge recipe.knowledge.v1`
  - `--codex-farm-knowledge-context-blocks 12`
- When enabled on a block-first source (has `full_text` blocks + `nonRecipeBlocks`):
  - Job bundles are written under `raw/llm/<workbook_slug>/pass4_knowledge/in/`.
  - codex-farm outputs are read from `raw/llm/<workbook_slug>/pass4_knowledge/out/` and validated strictly.
  - User-facing artifacts are written:
    - `knowledge/<workbook_slug>/snippets.jsonl`
    - `knowledge/<workbook_slug>/knowledge.md`
    - `knowledge/knowledge_index.json` (run-level summary)
- No live codex-farm/token-spending execution is required for automated verification.
- Local pipeline pack assets exist and are linked correctly:
  - `llm_pipelines/pipelines/recipe.knowledge.v1.json`
  - `llm_pipelines/prompts/recipe.knowledge.v1.prompt.md`
  - `llm_pipelines/schemas/recipe.knowledge.v1.output.schema.json`

## Evidence

- Targeted pytest suite:
  - `25 passed, 2 warnings in 2.26s`
- No live codex-farm/token-spending execution was performed in verification.

## Constraints / gotchas and decision

- Constraint: Do not run live codex-farm in verification (avoid token spend).
- Decision: Keep pass4 artifacts under `raw/llm/<workbook_slug>/pass4_knowledge/` and write user-facing outputs under `knowledge/<workbook_slug>/`.

## Rollback / compensation plan

- Runtime rollback: omit `--llm-knowledge-pipeline` (or set it to `off`).
- Code rollback: revert the pass4 modules under `cookimport/llm/` plus the added `llm_pipelines/recipe.knowledge.v1` assets.
