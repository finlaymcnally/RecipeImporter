# Task: EPUB Debug CLI

## Problem Statement

EPUB debugging required manual ZIP inspection and ad hoc scripts. There was no first-class CLI to inspect spine structure, dump source XHTML/text, compare extracted blocks, or inspect segmentation candidates.

## Acceptance Criteria

- Add `cookimport epub` subcommands: `inspect`, `dump`, `unpack`, `blocks`, `candidates`, `validate`.
- `blocks` and `candidates` must reuse production EPUB importer extraction/segmentation logic.
- Add deterministic artifact outputs for inspect/blocks/candidates/validate workflows.
- Add automated tests using a synthetic EPUB fixture.
- Update CLI docs and ExecPlan record.

## Verification Steps

Run from repo root:

    source .venv/bin/activate
    pytest -q tests/test_epub_debug_cli.py tests/test_epub_debug_extract_cli.py tests/test_epub_importer.py

Manual smoke checks:

    cookimport epub inspect tests/fixtures/sample.epub --out /tmp/epub-inspect
    cookimport epub blocks tests/fixtures/sample.epub --extractor legacy --out /tmp/epub-blocks
    cookimport epub candidates tests/fixtures/sample.epub --extractor legacy --out /tmp/epub-candidates

## Evidence

- Test suite result for affected areas: `19 passed`.
- New artifacts are generated by CLI tests:
  - `inspect_report.json`
  - `blocks.jsonl`, `blocks_preview.md`, `blocks_stats.json`
  - `candidates.json`, `candidates_preview.md`

## Constraints / Gotchas and Decision

- Direct use of `EpubImporter._extract_docpack(...)` needs `_overrides` initialized (`None`) before calling.
- `epub-utils` exists but is currently pre-release-only; use `python -m pip index versions --pre epub-utils` or install pinned `epub-utils==0.1.0a1`.
- Keep structure inspection fallback on ZIP/OPF parsing when `epub_utils` is not installed.
- EPUBCheck remains optional and strict failure is opt-in via `--strict`.

## Rollback / Compensation Plan

- Revert new module `cookimport/epubdebug/` and root CLI wiring in `cookimport/cli.py`.
- Remove added tests and tool/docs entries tied to `cookimport epub`.
- Existing root commands (`stage`, `inspect`, Label Studio flows) remain unaffected by this rollback.
