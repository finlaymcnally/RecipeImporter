## Problem

`Previous Runs` columns compressed to fit container width, making scanability poor and preventing practical horizontal scrolling.

## Acceptance Criteria

- `Previous Runs` table supports horizontal scrolling in the existing scroll container.
- Table keeps cells on one line so wide datasets stay readable.
- Renderer test coverage asserts the generated CSS contract.

## Verification

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`

## Evidence

- Expected pass-after result recorded from the verification command above.

## Constraints / Decision

- Keep change CSS-only in dashboard renderer output (`style.css` template) to avoid changing collector/render data logic.
- Use table-level `min-width` + no-wrap cells so horizontal overflow is deterministic.

## Rollback / Compensation

- Revert this task's renderer CSS and test assertions to restore current fixed-width behavior.
