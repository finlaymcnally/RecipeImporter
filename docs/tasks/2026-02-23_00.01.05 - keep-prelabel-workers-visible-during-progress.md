# Problem Statement

Freeform prelabeling could run in parallel but spinner completion updates dropped worker metadata after kickoff, making long runs look serial.

# Acceptance Criteria

- Parallel freeform prelabel status includes `task X/Y` and `(workers=N)` on kickoff and completion updates.
- Serial mode keeps existing `task X/Y` format without worker suffix.
- Regression test covers worker metadata on completion updates.

# Verification Steps

```bash
. .venv/bin/activate
COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1 pytest tests/labelstudio/test_labelstudio_ingest_parallel.py
```

# Evidence

- `cookimport/labelstudio/ingest.py` now formats prelabel progress through one helper that appends workers metadata only when `effective_prelabel_workers > 1`.
- `tests/labelstudio/test_labelstudio_ingest_parallel.py` now checks `(workers=2)` appears on `task 1/2` and `task 2/2` updates.

# Constraints / Gotchas

- Spinner ETA parsing still depends on preserving `task X/Y` structure.

# Decision

Keep worker visibility in the message suffix so progress/ETA parsing remains unchanged and operators can confirm concurrency at a glance.

# Rollback / Compensation

Revert the helper/message assertions if worker suffixes prove too noisy.
