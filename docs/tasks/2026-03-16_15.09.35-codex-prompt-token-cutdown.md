summary: "Trim the lowest-risk Codex prompt inputs shared by live runs and prompt previews."

## Problem

The new prompt-preview tooling made it easy to see that three low-risk seams were still wasting tokens in both live Codex runs and reconstructed previews: recipe correction always serialized an empty `draft_hint`, knowledge harvesting still defaulted to a wide pass-4 context window, and obvious `noise` knowledge chunks still became real Codex jobs.

## Acceptance Criteria

- Recipe correction input bundles omit empty `draft_hint` payloads without breaking the contract loader.
- Knowledge harvesting defaults to a smaller context window and skips suggested-lane `noise` chunks before Codex execution.
- The shared changes affect both live orchestrators and `cf-debug preview-prompts`.

## Verification

- Activate the project venv, then run:
  `. .venv/bin/activate && pytest tests/llm/test_codex_farm_contracts.py tests/llm/test_codex_farm_orchestrator.py tests/llm/test_prompt_preview.py tests/llm/test_knowledge_job_bundles.py tests/llm/test_codex_farm_knowledge_orchestrator.py tests/llm/test_run_settings.py`
- Expect: `56 passed`.

## Evidence

- Recipe correction live inputs and prompt-preview inputs now assert that `draft_hint` is absent when empty.
- Knowledge job tests now assert `noise` chunks produce `jobs_written == 0` and `skipped_lane_counts == {"noise": 1}`.
- Knowledge orchestrator tests now assert the all-skipped case short-circuits without calling the Codex runner.

## Constraints / Decisions

- Kept `draft_hint` in the Pydantic contract as an optional backwards-compatible field, but stopped emitting it when empty.
- Reduced the default pass-4 context width from `12` to `4` rather than `2` to keep this cut conservative.
- Scoped lane gating to `noise` only; `narrative` still stays as a hint, not an enforced skip.

## Rollback

- Revert the shared serializer/helper, the knowledge job/orchestrator changes, the smaller default, and the focused tests. No stored data migration is involved.
