summary: "Let long-lived workspace workers keep using narrow local shell commands during real task work while keeping structured follow-up calls strict."
---

# Problem

After the long-lived worker migration, main worker sessions were still being killed for normal workspace-local shell activity such as opening named shard files with `cat` or `sed`. The March 19 benchmark reruns showed the same problem at a larger scale: recipe and knowledge workers died on local `rg` / `ls` reflexes inside their own workspaces, and line-role still got killed for a harmless `out/` existence check. That zero-tolerance rule still makes sense for strict one-shot structured retry/repair calls, but it is too harsh for the workspace-worker model where Codex is explicitly acting like a local file worker processing many tasks over time.

# Acceptance Criteria

- Workspace-worker main attempts for recipe, knowledge, and line-role can keep using bounded workspace-local helper commands for the whole session, including local navigation/search/existence checks that stay inside the isolated worker workspace.
- Workspace-worker main attempts only terminate for command loops without outputs or for commands that leave the allowed workspace-local surface.
- Structured retry / repair follow-up calls still kill immediately on any shell command.
- Live status artifacts record when command execution was tolerated instead of killed.
- Focused tests cover the shared command classifier plus stage-level workspace tolerance.

# Evidence

- The focused watchdog/worker suites passed after the change.
- `./scripts/test-suite.sh domain llm` passed with `138 passed, 30 deselected, 1 warning`.
- `./scripts/test-suite.sh domain parsing` passed with `227 passed, 79 deselected, 1 warning`.

# Constraints / Decision

- Keep the tolerance bounded to the isolated worker workspace even though it is much looser than the original startup-only policy: allow common local helper commands and bounded existence checks, but do not allow parent traversal, repo escape, or non-local interpreters.
- Do not relax the structured retry/repair watchdog policy.
- Prolonged command loops, broader exploration commands, and non-local shell use still terminate the worker.
