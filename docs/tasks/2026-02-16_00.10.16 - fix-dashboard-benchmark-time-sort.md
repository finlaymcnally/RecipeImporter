# Problem statement

`data/output/.history/dashboard/index.html` showed "Recent Benchmarks" in the wrong order when records mixed `YYYY-MM-DDTHH:MM:SS` and `YYYY-MM-DD_HH.MM.SS` timestamps.

# Acceptance criteria

- Dashboard benchmark/run tables are sorted by parsed chronological time.
- "Latest benchmark" selection for per-label and boundary sections uses parsed time ordering.
- Collector summary `latest_*_timestamp` values are chosen by parsed timestamps.
- Regression coverage exists for mixed timestamp formats.

# Verification steps

- `. .venv/bin/activate && pip install -e .[dev]`
- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py`
- `. .venv/bin/activate && cookimport stats-dashboard`

# Evidence

- `pytest -q tests/test_stats_dashboard.py` -> `26 passed in 0.22s`.
- `cookimport stats-dashboard` rewrote `data/output/.history/dashboard/index.html` and assets successfully.

# Constraints/gotchas

- Keep compatibility with both timestamp encodings already present in history data.
- Keep fallback behavior stable for unparseable timestamps (include them; sort last where possible).

# Decision

Add timestamp-parse comparators in renderer JS and parsed-timestamp sort keys in collector Python, then add mixed-format regression tests.
