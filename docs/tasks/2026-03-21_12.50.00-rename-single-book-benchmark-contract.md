After this change, the repository should stop using the older `single_offline` / `single-book-benchmark` naming for the active single-book benchmark flow. Operators should see and produce `single_book` terminology consistently in interactive benchmark mode selection, run scopes, and benchmark artifact roots. ## Context and Orientation

The interactive benchmark menu lives in `cookimport/cli.py`. The single-book benchmark helper is `_interactive_single_offline_benchmark`, which currently creates a benchmark session root under `single-book-benchmark`. The same file also passes a scope string into the detached Oracle uploader. Oracle upload scope inference lives in `cookimport/bench/oracle_upload.py`; analytics path classification lives in `cookimport/analytics/benchmark_semantics.py` and `cookimport/analytics/dashboard_render.py`. The current tests for these contracts live mainly in `tests/labelstudio/`, `tests/bench/`, and `tests/analytics/`.

In this repository, “single-book benchmark” means the one-book offline benchmark flow that compares one selected source file against one freeform gold export. “Scope string” means the short identifier persisted beside upload metadata so Oracle upload and follow-up tooling know which benchmark family produced a bundle. “Artifact root” means the folder name directly under `data/golden/benchmark-vs-golden/<timestamp>/` that groups the files written by that flow.

The new contract should be:

- Interactive mode id: `single_book` (already done).
- Oracle/upload scope string: `single_book`.
- Artifact root: `single-book-benchmark`.

The matched-books flow under `single-profile-benchmark` is not being renamed here.

Finally, update active docs that describe the current contract, especially `docs/02-cli/02-cli_README.md`, `docs/07-bench/07-bench_README.md`, and the current analytics dashboard readme if it mentions the old root.
