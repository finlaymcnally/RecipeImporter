## Purpose / Big Picture

After this change, the repository should stop using the legacy pre-rename naming for the single-book benchmark flow, not just in user-facing artifact roots but also in helper names, split-cache keys, schema/version strings, analytics field paths, and test module names. Operators should see `single_book` on the CLI surface, and developers should see `single_book` in the internal API surface too. The easiest proof is to run the benchmark helper, analytics, and ingest-oriented tests after the rename and confirm both behavior and test file references now use `single_book`.

## Context and Orientation

The interactive benchmark menu lives in `cookimport/cli.py`. The single-book benchmark helper now uses `_interactive_single_book_benchmark` plus a family of `_single_book_*` helpers for metrics, split-cache handling, and markdown/comparison generation. The Label Studio ingest path in `cookimport/labelstudio/ingest.py` now uses `single_book_split_cache` identifiers in function names, manifest keys, and local variables. Analytics field names in `cookimport/analytics/compare_control_engine.py` and `cookimport/analytics/dashboard_render.py` now read `run_config.single_book_split_cache.conversion_seconds`. The benchmark-helper tests that exercise this seam live mainly in `tests/labelstudio/`, and the two focused helper modules now use `single_book` in their filenames.

In this repository, “single-book benchmark” means the one-book offline benchmark flow that compares one selected source file against one freeform gold export. “Scope string” means the short identifier persisted beside upload metadata so Oracle upload and follow-up tooling know which benchmark family produced a bundle. “Artifact root” means the folder name directly under `data/golden/benchmark-vs-golden/<timestamp>/` that groups the files written by that flow.

The final contract should be:

- Interactive mode id: `single_book` (already done).
- Oracle/upload scope string: `single_book`.
- Artifact root: `single-book-benchmark`.
- Internal helper and manifest naming: `single_book`.
- Public split-cache flags: `--single-book-split-cache-*`.

The matched-books flow under `single-profile-benchmark` is not being renamed here.
