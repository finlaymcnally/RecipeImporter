# Task: Prompt template `.md` extension migration

## Problem statement

Prompt templates under `llm_pipelines/prompts` were editable as `.txt` files, but we now want them to be markdown files. Runtime/template paths and tests must follow the new extension.

## Acceptance criteria

- Every prompt template in `llm_pipelines/prompts/` uses `.prompt.md`.
- Freeform prelabel/decorate runtime template paths in `cookimport/labelstudio/prelabel.py` point to `.prompt.md` files.
- Local codex-farm pipeline specs in `llm_pipelines/pipelines/*.json` reference `.prompt.md` template paths.
- Tests that assert prompt template paths/files are updated and pass.
- Current docs that describe active prompt locations reference `.prompt.md`.

## Verification steps

Run from repo root:

    source .venv/bin/activate
    pytest -q tests/test_labelstudio_prelabel.py tests/test_llm_pipeline_pack_assets.py tests/test_llm_pipeline_pack.py

## Evidence

- Renamed prompt files:
  - `llm_pipelines/prompts/freeform-prelabel-full.prompt.md`
  - `llm_pipelines/prompts/freeform-prelabel-augment.prompt.md`
  - `llm_pipelines/prompts/recipe.chunking.v1.prompt.md`
  - `llm_pipelines/prompts/recipe.schemaorg.v1.prompt.md`
  - `llm_pipelines/prompts/recipe.final.v1.prompt.md`
- Updated runtime/template path references:
  - `cookimport/labelstudio/prelabel.py`
  - `llm_pipelines/pipelines/recipe.chunking.v1.json`
  - `llm_pipelines/pipelines/recipe.schemaorg.v1.json`
  - `llm_pipelines/pipelines/recipe.final.v1.json`
  - `tests/test_labelstudio_prelabel.py`
  - `tests/test_llm_pipeline_pack_assets.py`
- Verification result:
  - `15 passed, 2 warnings in 1.43s`

## Constraints/gotchas and decision

- Decision: keep filename pattern as `*.prompt.md` instead of plain `.md` to preserve existing naming semantics while changing the extension to markdown.
- Template behavior is unchanged; runtime still does placeholder replacement and fallback to built-in strings when files are missing/empty.

## Rollback / compensation plan

- Rename `llm_pipelines/prompts/*.prompt.md` files back to `*.prompt.txt`.
- Revert `.prompt.md` path references in runtime code, pipeline specs, tests, and docs.
