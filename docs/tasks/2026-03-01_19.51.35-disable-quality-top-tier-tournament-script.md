# Task: Disable quality_top_tier_tournament script

## Problem statement

Even after disabling `bench quality-lightweight-series`, the standalone script `scripts/quality_top_tier_tournament.py` still allowed launching tournament runs that can consume massive runtime and disk space.

## Acceptance criteria

- Running `scripts/quality_top_tier_tournament.py` exits immediately with a clear disabled/retired message.
- Tournament script no longer creates output roots or executes quality-run subprocesses.
- Tests assert the disabled behavior.
- Bench docs reflect that the script is retired/disabled.

## Verification steps

Run from repo root with venv active:

```bash
. .venv/bin/activate
pytest tests/bench/test_quality_top_tier_tournament.py -k "main_is_disabled" -q
python scripts/quality_top_tier_tournament.py
```

Expected:
- pytest command passes.
- script prints disabled message and exits non-zero.

## Evidence

- `scripts/quality_top_tier_tournament.py` now returns early from `main()` with a disabled message.
- Added `test_main_is_disabled` in `tests/bench/test_quality_top_tier_tournament.py`.
- Updated `docs/07-bench/07-bench_README.md` to mark tournament script paths as historical/disabled.

## Constraints / gotchas and decision

- Constraint: helper functions in the script are still imported by tests.
- Decision: disable only runtime entry (`main`) while keeping helper functions intact for existing tests and historical analysis.

## Rollback / compensation plan

- Restore prior `main()` body in `scripts/quality_top_tier_tournament.py` and adjust tests/docs accordingly.
