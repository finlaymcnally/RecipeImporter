## Problem

After the first prompt-budget cut bundle, the same benchmark run was still spending about `461,865` live-like input tokens. The next cheapest remaining cuts were removing recipe provenance from correction hints, shrinking pass-4 knowledge context from `4` to `2`, and blanking line-role neighbor context for outside-recipe rows.

## Acceptance Criteria

- Recipe correction inputs omit `recipe_candidate_hint.provenance` in both live runs and prompt previews.
- Knowledge context defaults move from `4` to `2` across shared builders and exposed CLI/run-setting defaults.
- Line-role prompt serialization blanks `previous_line` and `next_line` for outside-recipe rows in both legacy and compact formats.

## Verification

- Activate the project venv, then run:
  `. .venv/bin/activate && pytest tests/llm/test_codex_farm_contracts.py tests/llm/test_codex_farm_orchestrator.py tests/llm/test_prompt_preview.py tests/llm/test_knowledge_job_bundles.py tests/llm/test_codex_farm_knowledge_orchestrator.py tests/llm/test_run_settings.py tests/parsing/test_canonical_line_roles.py`
- Expect: `106 passed`.
- Rebuild the prompt preview for `data/golden/benchmark-vs-golden/2026-03-16_13.34.01/single-offline-benchmark/saltfatacidheatcutdown/line_role_only` and compare live-like prompt totals.

## Evidence

- The rebuilt preview at `data/golden/benchmark-vs-golden/2026-03-16_13.34.01/prompt_preview_after_next_cuts/` measures about `365,215` live-like input tokens.
- The previous post-cut baseline was about `461,865`, so this round saved about `96,650` more tokens.
- Stage totals on the rebuilt preview are:
  - recipe correction: `214,070`
  - line role: `83,239`
  - knowledge: `67,906`

## Constraints / Decisions

- Kept the recipe correction contract otherwise intact: only provenance was removed from the hint object.
- Applied line-role neighbor blanking only outside recipe spans; inside-recipe neighbors still carry structure.
- Kept the knowledge change as a default-width trim, not a new behavior flag.

## Rollback

- Revert the recipe hint change, the line-role serializer change, the `4 -> 2` context-default updates, and the focused tests/docs. No migration or stored-data rewrite is required.
