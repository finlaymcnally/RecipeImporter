## Problem statement

The active line-role prompt teaches `HOWTO_SECTION` too loosely. On `saltfatacidheatcutdown`, Codex promoted many book/chapter/topic headings into `HOWTO_SECTION` even though this label is meant only for recipe-internal subsection headings like `FOR THE SAUCE`.

## Acceptance criteria

- The active checked-in line-role prompts define `HOWTO_SECTION` as recipe-internal subsection structure only.
- The prompts explicitly say chapter/topic/book headings and explanatory section headers are not `HOWTO_SECTION`.
- Prompt-builder fallback strings in code stay aligned with the checked-in prompt files.
- A targeted prompt test covers the new contract text.

## Verification steps

- Run `./.venv/bin/pytest tests/parsing/test_canonical_line_roles.py -k prompt`

## Evidence

- Before change, the active prompt only said `HOWTO_SECTION: method subheads such as FOR THE ... or TO MAKE ...`, which was not enough to stop `Salt and Pepper`, `Cooking Acids`, and similar headings from being promoted on the 2026-03-18 `saltfatacidheatcutdown` benchmark run.
- After change, the prompt text and test should explicitly teach recipe-internal-only semantics and book-heading negatives.

## Constraints / gotchas + brief decision

- Keep scope to prompt semantics and tests only; do not retune deterministic label rules or downstream howto resolution in this task.
- Update both checked-in prompt templates and Python fallback strings so preview/live/fallback contracts do not drift.

## Rollback / compensation plan

- Revert the prompt and test changes together if the tighter wording causes regressions on recipe-internal subsection headings.
