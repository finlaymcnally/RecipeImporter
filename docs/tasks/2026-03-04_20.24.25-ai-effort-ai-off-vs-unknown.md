## Problem
`AI Effort` currently collapses pipeline-off and unknown effort into the same `-` label, making filtering and comparison ambiguous.

## Acceptance Criteria
- `AI Effort` shows `AI off` when `llm_recipe_pipeline=off`.
- `AI Effort` still shows `-` for unknown/missing effort.
- Behavior is consistent between dashboard renderer and compare-control backend engine.
- Targeted tests cover both labels.

## Verification

- `. .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k previous_runs_quality_per_million_tokens_calculation`

## Constraints / Decision
- Keep change at label layer only; no token/quality metric or compare-control math changes.

## Rollback
