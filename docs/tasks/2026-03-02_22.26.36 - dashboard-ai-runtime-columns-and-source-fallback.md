## Problem

Main dashboard lacked explicit AI runtime context (model + thinking effort) and `Previous Runs` `Source` labels were often too generic or blank for quick scan.

## Acceptance Criteria

- `Diagnostics (Latest Benchmark)` includes AI runtime metadata for latest benchmark row.
- `Previous Runs` includes an `AI Model + Effort` column.
- `Source` labels in `Previous Runs` use better fallbacks when `source_file` is missing.
- All-method bundled rows also show a non-generic source summary and AI model/effort value.
- Dashboard renderer tests cover the new columns/card/helper hooks.

## Constraints / Decision

- Keep CSV-first dashboard contract: AI runtime values come from benchmark `run_config` / `run_config_summary` already persisted in history rows.
- When run-config values are unset (default codex runtime), collector backfills model/effort from prediction-run manifest `llm_codex_farm` telemetry.
- Use artifact-path-aware source slug fallback (all-method/single-profile/speed/eval patterns) only when `source_file` is missing.
- Keep UI changes minimal: add one diagnostics card + one history column without reworking layout structure.
