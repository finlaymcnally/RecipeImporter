# Problem Statement

Test runs were easy to over-scope and produce verbose output.
Need modular groups plus lower output volume so AI-driven test runs consume fewer tokens.

# Acceptance Criteria

- Pytest defaults are low-noise in repo config.
- Tests can be run by domain marker without editing each test file.
- A small `smoke` slice exists for quick sanity checks.
- Failure output includes short pointers to relevant `docs/*_log.md`.
- A short tests-folder note explains minimal run commands.

# Verification Steps

```bash
. .venv/bin/activate
pip install -e .[dev]
pytest -m smoke
pytest -m "ingestion and not slow" --collect-only
pytest -m "labelstudio and not slow" --collect-only
pytest tests/test_paprika_importer.py
```

# Evidence

- `. .venv/bin/activate && pip install -e .[dev]` completed successfully.
- `pytest -m smoke` -> `33 passed, 485 deselected, 2 warnings in 2.72s`.
- `pytest -m "ingestion and not slow" --collect-only` -> `46/517 tests collected`.
- `pytest -m "labelstudio and not slow" --collect-only` -> `11/517 tests collected`.
- `pytest tests/test_paprika_importer.py` (known pre-existing failing file) now prints compact failure output:
  - `..FF..`
  - `log: docs/03-ingestion/03-ingestion_log.md`
  - `2 failed, 4 passed in 0.21s`

# Constraints / Gotchas

- Keep marker mapping centralized in `tests/conftest.py` to avoid touching many files.
- Keep output concise by default; deep debugging remains opt-in with `-vv --tb=short --show-capture=all --assert=rewrite`.

# Decision

Use central marker auto-assignment plus `pytest.ini` defaults instead of physically moving test files.

# Rollback / Compensation

If grouping/noise changes are problematic, remove `pytest.ini` and `tests/conftest.py` and revert to explicit per-file test commands.
