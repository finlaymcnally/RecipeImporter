Problem statement

The label-first line-role pipeline still carries `confidence`, `trust_score`, and `escalation_score` through runtime models, staged artifacts, and reviewer exports. This task removes those scalar score fields entirely so the refactor uses labels plus explicit reasons only.

Acceptance criteria

- Canonical line-role predictions and authoritative label-first artifacts no longer expose `confidence`, `trust_score`, or `escalation_score`.
- Score-threshold-based line-role escalation is removed; Codex escalation decisions use explicit reasons only.
- Downstream exports/tests/docs for the label-first seam are updated to the score-free contract.
