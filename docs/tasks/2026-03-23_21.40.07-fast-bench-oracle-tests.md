Problem statement

`./scripts/test-suite.sh domain bench` is too slow for routine coding loops because a few Oracle-related bench tests copy or poll large realistic benchmark artifacts. The fast default bench slice needs smaller fixtures and clearer fast/slow boundaries.

Acceptance criteria

- The main Oracle bench tests use minimal synthetic fixtures for fast coverage.
- Any remaining realism-heavy Oracle bench coverage is isolated in explicit slow files.
- `./scripts/test-suite.sh domain bench` is materially faster than before.

Verification steps

- `source .venv/bin/activate && pytest tests/bench/test_oracle_followup*.py -m 'not slow'`
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_oracle_upload*.py -m 'not slow'`
- `source .venv/bin/activate && pytest tests/bench/test_cf_debug_cli*.py -m 'not slow'`
- `source .venv/bin/activate && ./scripts/test-suite.sh domain bench`

Evidence

- Before: `tests/bench/test_oracle_followup.py` about `52.5s`, `tests/bench/test_benchmark_oracle_upload.py` about `18.7s`, `tests/bench/test_cf_debug_cli.py` about `11.6s`.
- After: `tests/bench/test_oracle_followup.py` about `10.5s`, `tests/bench/test_benchmark_oracle_upload.py` about `9.4s`, `tests/bench/test_cf_debug_cli.py` about `2.0s`, and `./scripts/test-suite.sh domain bench` about `29.4s`.

Constraints / gotchas + decision

- Do not weaken live-token guardrail coverage.
- Keep docs short and update only the testing surface that changed.
- Prefer tiny synthetic `upload_bundle_v1` fixtures over copied checked-in benchmark roots.

Rollback / compensation plan

- If the split causes routing drift, restore marker mapping in `tests/conftest.py`.
- If a synthetic fixture misses a contract, keep one explicit slow realism test for that gap instead of reintroducing large copied fixtures to the fast slice.
