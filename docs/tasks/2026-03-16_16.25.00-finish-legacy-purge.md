Problem statement

The repo still has live compatibility code after the refactor, especially around recipe runner fallbacks, upload-bundle adapters, and old transport replay assets. This task removes those branches so active code and current docs only describe the new semantic recipe pipeline.

Acceptance criteria

- `cookimport/llm/codex_farm_runner.py` no longer parses legacy stderr progress lines or accepts empty/non-JSON stdout when `--json` is requested.
- Upload-bundle source/adaptation code no longer uses pass-slot variable names or `UploadBundleSourceModel.as_context_dict()`.
- Dead transport replay code and fixtures are deleted, and tests/docs no longer reference them.
- Current staging/LLM/reference docs match the post-purge contract.

Evidence

- Before the change, repo search still found `as_context_dict()`, `observed_pass2_call_count`, legacy runner progress parsing, and the deleted transport replay script.
- After the change, those active-code references are gone and the targeted validation commands pass.

Constraints/gotchas + brief decision

- Historical planning/report docs can keep historical language; only current runtime/docs/tests are in scope.

Rollback / compensation plan
