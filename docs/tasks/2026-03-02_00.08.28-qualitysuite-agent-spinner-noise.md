# Task: QualitySuite agent spinner noise

## Problem statement

Agent-driven PTY polling captures rich spinner frames as repeated output noise during long `bench quality-run` sessions.

## Acceptance criteria

- Agent envs (`CODEX_CI=1`, `CODEX_THREAD_ID`, `CLAUDE_CODE_SSE_PORT`) default to plain, change-only progress output.
- Explicit progress override still works via `COOKIMPORT_PLAIN_PROGRESS=1|0`.
- Existing live status behavior remains available for normal terminal runs.
- Tests cover agent default and explicit live override.

## Verification steps

Run from repo root with venv active:

```bash
. .venv/bin/activate
pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -q
```

## Evidence

- `cookimport/cli.py`: added agent-env detection and plain-progress default path.
- `tests/labelstudio/test_labelstudio_benchmark_helpers.py`: added regression tests for agent default and explicit override.
- `cookimport/README.md`, `docs/07-bench/07-bench_README.md`: documented the behavior.

## Constraints / gotchas and decision

- Constraint: keep human-facing spinner UX unchanged by default in normal terminals.
- Decision: reuse existing plain-progress path and only alter live-status eligibility for known agent env markers unless explicitly overridden.
