## Problem

The interactive dashboard workflow asks `Open dashboard in your browser after generation?`, but browser auto-open does not work reliably and the prompt adds unnecessary friction.

## Acceptance Criteria

- Dashboard action in interactive mode does not ask a browser-open confirmation question.
- Interactive dashboard generation still runs `stats_dashboard` with the configured output root/history path.
- Interactive dashboard generation always uses `open_browser=False`.
- CLI docs and README reflect the updated behavior.

## Verification

- Fail-before:
  - `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k interactive_generate_dashboard_runs_without_browser_prompt`
- Pass-after:
  - `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k interactive_generate_dashboard_runs_without_browser_prompt`

## Evidence

- Fail-before result: `1 failed, 156 deselected, 1 warning`
- Pass-after result: `1 passed, 156 deselected, 1 warning`

## Constraints / Decision

- Keep change scoped to interactive dashboard flow only.
- Non-interactive `stats-dashboard` command behavior is unchanged.

## Rollback / Compensation

- Restore the removed confirmation prompt and pass the selected value back into `open_browser` for interactive dashboard calls.
