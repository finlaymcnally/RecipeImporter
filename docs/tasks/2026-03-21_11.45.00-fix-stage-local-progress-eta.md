Problem statement

The shared spinner ETA could inherit avg-per-task timing from the previous structured stage when the next stage reused the same `task X/Y` shape. This made new stages report cartoonishly wrong ETAs and averages.

Acceptance criteria

- Structured progress ETA history resets when `stage_label` changes.
- A new stage does not show an inherited `avg .../task` before it has its own timing samples.
- Existing progress/ETA tests still pass.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_progress.py -q`

Evidence

- Added a regression test that moves from `stage one` to `stage two` at the same counter shape and asserts stage two does not render inherited `avg .../task`.
- Targeted pytest passed after the code change.

Constraints/gotchas + decision

- The fix stays in the shared CLI progress samplers so all structured stage-progress surfaces pick it up together.
- Both the live spinner path and the single-profile dashboard path now clear their rate samples on stage transition.

Rollback / compensation plan

- Revert the sampler reset change in `cookimport/cli.py` and remove the regression test if this causes ETA gaps that are worse than the inherited-history bug.
