## Problem

Dashboard `Diagnostics` and `Previous Runs` AI column could show `off`/unknown for codex benchmark rows because model/effort was missing from `run_config` and only present in manifest `llm_codex_farm` runtime payloads.

## Acceptance Criteria

- Benchmark collector backfills `codex_farm_model` and `codex_farm_reasoning_effort` into benchmark `run_config` from manifest runtime payloads when missing.
- `AI Model + Effort` can show real codex model for single-offline codex rows that previously rendered as `off`/unknown.
- Latest diagnostics runtime tie-break prefers richer AI metadata when multiple rows share a timestamp.
- Analytics dashboard tests pass.

## Verification

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`
- `source .venv/bin/activate && cookimport stats-dashboard`
- Check `data/.history/dashboard/assets/dashboard_data.json` for single-offline codex row:
  - `run_config.codex_farm_model`
  - `run_config.codex_farm_reasoning_effort`

## Evidence

- `pytest tests/analytics/test_stats_dashboard.py` -> `51 passed in 83.40s`
- Regenerated dashboard data includes:
  - codex row: `gpt-5.3-codex-spark`, `<default>`
  - vanilla row: `off`

## Constraints / Decision

- Keep CSV-first behavior for existing persisted run-config fields.
- Use manifest runtime enrichment only as fallback when codex runtime values were not persisted in run-config.

## Rollback / Compensation

- Revert collector runtime enrichment + renderer tie-break changes to restore previous AI-label behavior.
