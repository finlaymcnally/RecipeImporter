# Remove Isolate For X from stats-dashboard

## Problem statement

`Previous Runs` had both `Isolate For X` and `Compare & Control`. We now want isolate removed entirely so the dashboard keeps one slicing path (quick filters + table filters) and one analysis panel (`Compare & Control`).

## Acceptance criteria

- `Isolate For X` panel and controls do not render in dashboard HTML.
- Dashboard JS does not include isolate state/control-source plumbing.
- `Compare & Control` still renders and works, including `Filter to subset` writing table filters.
- Previous Runs status text reports active filters/global mode without isolate/control-source messaging.
- Analytics docs describe the new behavior and non-technical how-to only documents `Compare & Control`.

## Verification steps

- In repo root:
  - `. .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -q`
  - `. .venv/bin/activate && cookimport stats-dashboard`

## Evidence

- `pytest tests/analytics/test_stats_dashboard.py -q` passed.
- `cookimport stats-dashboard` completed and wrote dashboard output.

## Constraints / gotchas

- Preserve backward compatibility for older saved UI-state/preset payloads that may still contain isolate keys.
- Keep `Compare & Control` and table filter engine behavior unchanged.

## Decision

Remove isolate end-to-end (HTML/CSS/JS + tests/docs), rather than hiding UI only, to avoid stale status/state complexity and prevent future confusion.
