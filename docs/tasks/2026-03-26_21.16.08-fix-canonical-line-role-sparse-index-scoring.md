# Task: Fix canonical line-role sparse-index scoring

## Problem

Canonical line-role benchmark artifacts were being serialized by sparse source `block_index` instead of dense canonical `line_index`.
On sources with sparse source block ids, this inflated `stage_block_predictions.json` with thousands of synthetic `OTHER` rows and cratered vanilla benchmark scores.

## Acceptance Criteria

- `line-role-pipeline/stage_block_predictions.json` uses dense canonical line indices for scored labels.
- Sparse source block ids do not increase `block_count` or create filler `OTHER` labels.
- Existing projection artifacts still preserve source block provenance through `projected_spans.jsonl` / `extracted_archive.json`.

## Verification

1. Baseline repro before fix:
   - `./.venv/bin/python - <<'PY' ... build_line_role_stage_prediction_payload(...) ... PY`
   - Expect sparse input like source block ids `100` and `200` to report `block_count=201`.
2. Focused tests after fix:
   - `./.venv/bin/pytest tests/labelstudio/test_canonical_line_projection.py`
3. Vanilla benchmark confirmation after fix:
   - `./.venv/bin/cookimport labelstudio-benchmark data/input/saltfatacidheatCUTDOWN.epub --eval-mode canonical-text --no-upload --llm-recipe-pipeline off --llm-knowledge-pipeline off --line-role-pipeline off`

## Evidence

- Before fix, sparse repro produced `{'block_count': 201, 'keys_sample': ['0', '1', '2'], 'last_key': '200'}`.
- After fix, the same repro produced `{'block_count': 2, 'block_labels': {'0': 'RECIPE_TITLE', '1': 'OTHER'}}`.
- `./.venv/bin/pytest tests/labelstudio/test_canonical_line_projection.py`
  - `6 passed, 1 warning in 1.43s`
- `./.venv/bin/pytest tests/labelstudio/test_labelstudio_ingest_parallel.py -k fail_closes_unreviewed_nonrecipe_knowledge`
  - `1 passed, 39 deselected, 1 warning in 1.43s`
- Fresh vanilla benchmark:
  - root: `data/golden/benchmark-vs-golden/2026-03-26_21.19.54`
  - `strict_accuracy=0.5470668485675307`
  - `macro_f1_excluding_other=0.6731951693465547`
  - scored `line-role-pipeline/stage_block_predictions.json` now reports `block_count=1471` instead of the cratered `8017`-block surface.

## Constraints / Decision

- Keep the fix local to line-role projection serialization.
- Preserve source block provenance in archive artifacts, but keep the scored artifact in canonical line coordinates.

## Rollback

- Revert this patch if another consumer truly requires source-block aggregation, then add a separate non-scoring artifact for that view instead of changing the scored canonical pointer pair.
