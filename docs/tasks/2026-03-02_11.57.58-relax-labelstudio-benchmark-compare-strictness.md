# Task: Relax `labelstudio-benchmark compare` strictness (inference + warnings)

## Goal
Make `labelstudio-benchmark compare` "work if it can": when benchmark-mode metadata is missing, infer benchmark-ness from the artifacts on disk and proceed, while emitting clear warnings in console output and the comparison artifacts.

## Current behavior (problem)
Some compare gates (notably debug-artifact completeness) require explicit `codex_farm_recipe_mode=benchmark` metadata; legacy outputs without this metadata fail strict gates even if the expected artifacts are present.

## Desired behavior
- If metadata explicitly says benchmark: behave as today (no warnings).
- If metadata is missing but artifacts strongly indicate benchmark: run benchmark-only checks and allow PASS, but emit warnings that benchmark mode was inferred.
- If neither metadata nor artifacts indicate benchmark: skip benchmark-only checks (or mark them as N/A) and warn that benchmark mode could not be determined.

## Implementation plan (short)
1. Extend the compare "mode resolver" to return both `mode` and `mode_source` (`metadata`, `inferred`, `unknown`).
2. Update debug-artifact gate logic to treat `mode_source=inferred` as eligible for benchmark debug checks, but add a warning like "benchmark mode inferred; metadata missing".
3. Add a `warnings` list to the comparison output payload (JSON + MD) and print warnings prominently in CLI output.
4. Update tests to cover:
   - legacy outputs with no mode metadata + benchmark artifacts => PASS with warnings
   - unknown mode => benchmark-only checks are skipped/N/A with warnings
5. Update docs to describe inference behavior and what warnings mean (`docs/07-bench/07-bench_README.md`).
