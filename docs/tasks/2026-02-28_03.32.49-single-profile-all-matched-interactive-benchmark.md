## Problem statement

Interactive benchmark had a gap between one-off single runs and full all-method permutations. We need a mode that runs one selected profile against every matched gold/source pair.

## Acceptance criteria

- Interactive benchmark mode picker includes a single-profile all-matched option.
- Mode runs one `labelstudio-benchmark` execution per matched target (no all-method variant expansion).
- Mode stays offline/no-upload and canonical-text eval.
- Tests cover mode routing and per-target execution wiring.
- CLI docs reflect the new mode.

## Verification steps

- Run: `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -q`
- Confirm new single-profile-all-matched tests pass.

## Evidence

- Added `_interactive_single_profile_all_matched_benchmark(...)` and menu wiring in `cookimport/cli.py`.
- Added tests in `tests/labelstudio/test_labelstudio_benchmark_helpers.py`.
- Updated docs in `docs/02-cli/02-cli_README.md` and `docs/02-cli/02-cli_log.md`.

## Constraints and decisions

- Reused existing `_resolve_all_method_targets(...)` matching logic to avoid duplicating source-hint matching behavior.
- Implemented this mode as sequential per-target `labelstudio_benchmark(...)` calls so behavior remains aligned with single-offline semantics.

## Rollback plan

- Remove the `single_offline_all_matched` menu option and helper, restoring the previous two benchmark modes.
