# Task: Interactive EPUB Race Default Output Root

## Problem Statement

Interactive EPUB race defaults to `/tmp/epub-race/<book>`, which is easy to lose and hard to find. We want the default under the repo output tree.

## Acceptance Criteria

- Interactive EPUB race prompt defaults to `data/output/EPUBextractorRace/<book_stem>`.
- Existing interactive race behavior remains the same (still prompts for custom output/candidates and overwrite confirmation).
- User docs reflect the new default path.
- A test asserts the new default output root.

## Verification Steps

Run from repo root:

    source .venv/bin/activate
    pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_epub_race

## Evidence

- `1 passed, 33 deselected` on:
  - `pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_epub_race`

## Constraints / Gotchas and Decision

- Kept change scoped to interactive-mode default only; direct `cookimport epub race --out ...` behavior is unchanged.
- Added assertion to the existing interactive race routing test to avoid duplicate coverage.

## Rollback / Compensation Plan

- Revert `DEFAULT_EPUB_RACE_OUTPUT_ROOT` and restore `/tmp/epub-race/<book_stem>` in `_interactive_epub_race`.
- Revert docs/test updates tied to the default path.
