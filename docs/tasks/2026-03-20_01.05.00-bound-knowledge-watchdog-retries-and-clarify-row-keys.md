---
summary: "Clarify compact knowledge row keys in workspace prompts and bound silent strict-JSON knowledge retries."
---

# Problem

Knowledge workers were drifting into verbose row keys like `block_decisions` / `snippets` instead of compact `d` / `s`, then self-debugging their own output with shell commands. When the strict JSON watchdog retry launched, silent Codex retries could sit in `running` forever with no output and no timeout.

# Acceptance Criteria

- Knowledge workspace-worker prompts explicitly name compact row keys and forbid verbose key names.
- Knowledge strict JSON watchdog retries terminate after bounded silence instead of waiting forever.
- Knowledge watchdog retries also carry an explicit overall subprocess timeout.

# Verification

- `. .venv/bin/activate && pytest -q tests/llm/test_codex_farm_knowledge_orchestrator.py -k 'knowledge_workspace_watchdog_allows_shell_work_until_command_loop or knowledge_strict_json_watchdog_kills_silent_retry or knowledge_orchestrator_writes_manifest_and_artifacts or knowledge_watchdog_retry_uses_bounded_timeout'`

# Evidence

- New tests assert the workspace prompt now includes exact compact row keys and forbids `block_decisions` / `snippets`.
- New tests assert strict JSON retries write live status with `watchdog_no_activity_timeout` after bounded silence and are launched with a non-null retry timeout.

# Constraints / Decision

- Keep scope to the knowledge path only.
- Use a narrow silence watchdog plus a hard retry timeout instead of widening workspace command allowances further.

# Rollback

- Revert the knowledge orchestrator, knowledge tests, and short LLM runtime doc note together.
