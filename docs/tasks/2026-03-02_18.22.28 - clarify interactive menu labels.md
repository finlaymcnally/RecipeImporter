## Problem

Interactive CLI menus currently use long, inconsistent option labels (often `name - description`), which makes it hard to quickly understand what each choice does.

## Acceptance Criteria

- Interactive select menus use the convention: `NAME: simple description` (Questionary supplies the numeric `#)` prefix).
- The "Evaluate vs freeform gold" submenu options are clearly distinguishable in one scan.
- Docs that list these menu options match runtime strings.
- Tests that assert benchmark mode option text continue to pass.

## Verification

- `pytest -q`
- Manual:
  - Run `cookimport` interactive mode.
  - Confirm main menu and benchmark mode submenu show `NAME: description` options and remain selectable via numeric shortcuts.

## Constraints / Notes

- Keep the descriptions plain-English and short.
- Do not change behavior; only improve wording/labels.
