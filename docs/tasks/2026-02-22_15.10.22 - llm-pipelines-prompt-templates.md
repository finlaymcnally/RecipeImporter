# Task: llm_pipelines-backed Label Studio prompt templates

## Problem statement

Make freeform prelabel/decorate prompt wording easy to iterate by moving prompt text out of Python string literals and into editable files under `llm_pipelines/prompts`.

## Acceptance criteria

- Runtime prompt text is read from `llm_pipelines/prompts/freeform-prelabel-full.prompt.txt` and `llm_pipelines/prompts/freeform-prelabel-augment.prompt.txt`.
- `_build_prompt(...)` still injects per-task dynamic values (segment id, allowed labels, block JSON lines, existing labels).
- If template files are missing or empty, runtime uses built-in fallback templates.
- Tests verify template file usage and placeholder injection.
- Label Studio docs describe the new editing workflow.

## Verification steps

Run from repo root:

    source .venv/bin/activate
    pip install -e .[dev]
    pytest -q tests/test_labelstudio_prelabel.py

## Evidence

- `pytest -q tests/test_labelstudio_prelabel.py`: `13 passed, 2 warnings in 1.42s`
- Added test coverage confirms runtime uses prompt template files and injects placeholders:
  - `tests/test_labelstudio_prelabel.py::test_prelabel_prompt_uses_file_templates`

## Constraints / gotchas and decision

- Constraint: prompt wording should be freely editable without constantly touching Python/tests.
- Decision: move only text to `.prompt.txt` files and keep stable runtime substitution tokens in code.
- Decision: keep fallback strings in code so runtime still works when docs files are unavailable.

## Rollback / compensation plan

- Revert the template-loader wiring in `cookimport/labelstudio/prelabel.py` and restore hardcoded prompt strings.
