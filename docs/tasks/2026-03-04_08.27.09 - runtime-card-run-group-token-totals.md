## Problem statement
Latest `Benchmark Runtime` token use was sourced from one benchmark row, which under-reported multi-book runs.

## Acceptance criteria
- Runtime diagnostics token use is summed across the latest preferred benchmark run-group.
- Runtime diagnostics title indicates run-group context and eval count.
- Runtime model/effort/pipeline context remains available and prefers richer metadata rows in that run-group.

## Verification steps
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k "runtime_summary_aggregates_tokens_across_latest_run_group or test_js_renders_previous_runs_table_and_links_timestamp_to_artifact"`

## Evidence
- Added Node harness-backed renderer test validating latest run-group token summation and context selection.

## Constraints / decision
- Keep runtime card source/importer display tied to one representative context row for now; this change focuses on run-group token totals.

## Rollback / compensation
- Revert runtime helper (`latestRuntimeSummaryForRecords`) and restore row-level token calculation in `renderLatestRuntime`.
