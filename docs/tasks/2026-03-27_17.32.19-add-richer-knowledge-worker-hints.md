# Add Richer Knowledge Worker Hints

## Problem

Knowledge already had `hints/<shard_id>.md`, but the hint surface was much thinner than line-role.
Workers mostly got packet summary plus generic reminders, not a shard profile, decision posture, example pointers, or flagged attention rows.

## Acceptance Criteria

- Knowledge worker prompts tell the worker to open `hints/<shard_id>.md` before the authoritative input when the hint exists.
- Knowledge hints include shard profile, shard interpretation/default posture, decision policy, example-file pointers, and attention rows.
- Focused contract coverage proves the richer hint surface is present.

## Verification

- `. .venv/bin/activate && pytest tests/llm/test_knowledge_orchestrator_contracts.py -q`
- `python -m py_compile cookimport/llm/knowledge_stage/recovery.py`

## Constraints / Decision

Keep the task contract unchanged.
This pass improves worker calibration using repo-written hints and existing example files rather than changing the knowledge packet schema or shard planner.
