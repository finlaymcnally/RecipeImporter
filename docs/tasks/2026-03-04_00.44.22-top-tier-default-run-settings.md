## Problem statement

Interactive import/benchmark runs still asked users to pick from multiple settings profiles, causing stale low-quality combinations (for example codex on with line-role/atomic off).

## Acceptance criteria

- Interactive run-settings chooser menu is not shown for import or benchmark flows.
- `choose_run_settings(...)` resolves one deterministic top-tier profile.
- Resolution prefers saved quality-suite winner settings; otherwise applies built-in top-tier baseline.
- Built-in baseline keeps codex recipe + codex line-role + atomic splitter enabled.
- CLI docs and local helper docs reflect the new behavior.

## Verification steps

- `. .venv/bin/activate && pytest tests/cli/test_c3imp_interactive_menu.py -q`

## Evidence

- Updated tests in `tests/cli/test_c3imp_interactive_menu.py` assert:
  - quality-suite winner profile is used when present,
  - fallback baseline enables unstructured semantic EPUB + codex/line-role/atomic,
  - no chooser/codex prompt callbacks are invoked.

## Constraints/gotchas

- Keep quality-suite winner file compatibility intact (`data/.history/qualitysuite_winner_run_settings.json`).
- Preserve worker utilization downscaling behavior in `choose_run_settings(...)`.

## Decision

Use `choose_run_settings(...)` as a strict resolver instead of a menu workflow, making one profile the only interactive default and eliminating per-run knob drift.

## Rollback / compensation

Revert `cookimport/cli_ui/run_settings_flow.py` and restore chooser-related tests/docs if manual per-run profile selection is needed again.
