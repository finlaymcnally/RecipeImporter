## Problem statement

`bench quality-discover` currently selects a representative stratified subset across all matched gold exports. We need the default suite to prioritize the three high-effort CUTDOWN sets: `saltfatacidheatcutdown`, `thefoodlabcutdown`, and `seaandsmokecutdown`.

## Acceptance criteria

- Running discovery with default settings selects curated IDs first when those targets are present/matched.
- If curated IDs are absent, discovery falls back to existing representative stratified selection.
- The behavior is covered by tests and documented in bench/CLI docs.

## Verification steps

- Run: `. .venv/bin/activate && pytest tests/bench/test_quality_suite_discovery.py tests/bench/test_bench.py -q`
- Confirm curated-target test passes and existing discovery wiring tests stay green.

## Constraints and decisions

- Keep existing representative selection algorithm as fallback for non-repo fixtures and other environments.
- Keep changes minimal: quality suite selection logic + tests + short docs.

## Rollback plan

- Revert the quality selection helper changes and remove curated-selection doc/test additions.
