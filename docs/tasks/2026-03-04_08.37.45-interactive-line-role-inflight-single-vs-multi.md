## Problem statement
Interactive benchmark runs used the same canonical line-role codex inflight setting regardless of single-book or multi-book mode.

## Acceptance criteria
- Single-offline interactive benchmark runs with `COOKIMPORT_LINE_ROLE_CODEX_MAX_INFLIGHT=8`.
- Single-profile interactive benchmark runs with one selected book use `8`.
- Single-profile interactive benchmark runs with multiple selected books use `4`.
- Tests cover both single and multi-book interactive paths.

## Verification steps
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "interactive_single_offline_codex_enabled_runs_vanilla_then_codex_and_writes_comparison or interactive_single_offline_codex_disabled_runs_only_vanilla_and_skips_comparison"`
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py -k "interactive_single_profile_parallel_requests_two_live_spinner_slots or interactive_single_profile_all_matched_codex_runs_vanilla_then_codex_per_book"`

## Evidence
- Added a scoped CLI env override helper in `cookimport/cli.py` and applied it to interactive single-offline and single-profile benchmark run wrappers.
- Updated tests to assert observed env values (`8` for single-book, `4` for multi-book).
- Updated benchmark/CLI docs to reflect the runtime override behavior.

## Constraints / decision
- Kept parser-level default inflight behavior unchanged and applied overrides only in interactive benchmark orchestration to avoid unexpected behavior drift in non-interactive paths.

## Rollback / compensation
- Remove the interactive override context manager usage so benchmark runs fall back to parser/env defaults.
