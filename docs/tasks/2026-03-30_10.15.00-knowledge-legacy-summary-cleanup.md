Problem statement

Legacy knowledge benchmark artifacts still surface old `missing_output_*` shard-count names in prompt-budget compatibility reads, even though the active knowledge runtime and summary docs now use `no_final_output_*`.
That leaves one small drift seam where old artifacts can reintroduce retired wording into current summaries.

Acceptance criteria

- Legacy knowledge prompt-budget inputs that only contain `missing_output_shards` normalize into `no_final_output_shard_count` on the surfaced knowledge summary.
- The surfaced knowledge prompt-budget summary no longer reports `missing_output_shard_count` for that legacy knowledge-only path.
- Historical docs updated by this cleanup clearly label old `missing_output` wording as baseline evidence rather than current knowledge guidance.
- The living ExecPlan at `docs/plans/2026-03-30_09.28.18-remaining-llm-legacy-cleanup-audit.md` reflects the actual completed state of the repo after this cleanup.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -k 'legacy_knowledge_missing_output or knowledge_packet_and_followup_counts' -x`
- `source .venv/bin/activate && pytest tests/staging/test_stage_observability.py tests/llm/test_knowledge_output_ingest.py -x`
- `rg -n "missing_output|no_final_output|missing_output_shard_count" cookimport/llm docs tests`

Evidence

- The new prompt-budget regression fails before the normalization change because the knowledge stage only exposes `missing_output_shard_count`.
- After the change, the same test passes with `knowledge_stage["no_final_output_shard_count"]` populated and `knowledge_stage.get("missing_output_shard_count") is None`.
- Verified after the patch: `2 passed, 34 deselected` for the focused labelstudio slice, and `20 passed` for `tests/staging/test_stage_observability.py tests/llm/test_knowledge_output_ingest.py`.

Constraints / gotchas + decision

- Keep line-role and recipe `missing_output_*` references out of scope; those are still live taxonomies in this repo.
- Keep compatibility for older knowledge artifacts by reading the legacy key, but normalize it into the current surfaced knowledge vocabulary instead of preserving the retired field name.
- Do not run CodexFarm-enabled live processing or benchmark reruns for this cleanup.

Rollback / compensation plan

- Revert this task's diff if downstream consumers unexpectedly require the legacy knowledge field name in surfaced prompt-budget summaries.
