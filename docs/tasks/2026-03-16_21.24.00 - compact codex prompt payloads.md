## Problem

The current codex-shaped preview is still too expensive even after earlier bundle/transport fixes. The largest remaining cost is line-role prompt payload size, followed by knowledge payload size.

## Acceptance Criteria

- `codex-line-role-v1` keeps the same output contract but sends a materially smaller prompt payload per atomic line.
- Line-role prompt rows no longer repeat the current verbose JSON-object shape for every row.
- Line-role prompt construction uses batch-level recipe span metadata and only includes neighbor text when a row actually needs local context.
- Knowledge harvest keeps Codex enabled but uses stricter low-risk defaults to reduce prompt chars.
- A regenerated codex-shaped preview for `saltfatacidheatcutdown/line_role_only` shows a lower total estimated token count than `prompt_preview_codex_hypothetical_after_gap10_low_signal_2026-03-16_21.14.22`.

## Verification

- `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles.py tests/llm/test_prompt_preview.py tests/llm/test_knowledge_job_bundles.py -q`
- `source .venv/bin/activate && ./scripts/test-suite.sh domain llm`
- Regenerate the codex-shaped preview on the same benchmark root and compare the new `prompt_preview_budget_summary.md` against the previous artifact.

## Evidence

- Before: `prompt_preview_codex_hypothetical_after_gap10_low_signal_2026-03-16_21.14.22`
- After: `prompt_preview_codex_hypothetical_after_compact_line_role_2026-03-16_21.35.34`
- Measured preview delta:
  - total estimated tokens `~578,563 -> ~449,484`
  - line-role `15 -> 8` prompts and `~290,891 -> ~174,884` estimated total tokens
  - knowledge `41 -> 40` prompts and `~183,168 -> ~170,096` estimated total tokens

## Constraints / Decisions

- All Codex-backed stages remain enabled.
- Line-role is the highest-priority cost target.
- Keep the line-role output parser stable; reduce cost by compacting input, not by widening the response contract.

## Rollback

- Revert the compact line-role prompt serialization and batch-size constant if the new prompt format harms reliability.
- Revert the tighter knowledge defaults independently if they prove too aggressive.
