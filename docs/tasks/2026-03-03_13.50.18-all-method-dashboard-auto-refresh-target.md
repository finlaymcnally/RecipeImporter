# All-Method Dashboard Auto Refresh Target

## Problem statement

All-method deferred refresh paths were still using nested benchmark run CSV roots for dashboard output, which can regenerate run-local dashboard snapshots instead of the lifetime dashboard path users rely on.

## Acceptance criteria

- Deferred all-method refreshes target the configured output root lifetime dashboard directory.
- Existing CSV trigger paths and batch-refresh timing behavior are preserved.
- Single-source and all-matched interactive all-method entrypoints forward the same dashboard target root.
- Regression tests verify refresh kwargs and forwarding.

## Evidence

## Constraints/gotchas + decision

- Preserve existing non-interactive call behavior unless a dashboard output root is explicitly provided.
- Decision: add optional `dashboard_output_root` plumbing through all-method runners and translate it once into `output_root` + `dashboard_out_dir` at refresh call sites.

## Rollback / compensation

- Revert `dashboard_output_root` runner plumbing and restore previous refresh assertions in all-method benchmark helper tests.
