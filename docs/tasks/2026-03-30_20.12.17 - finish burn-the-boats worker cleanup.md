Problem statement

The live single-file worker runtime shipped, but two gaps remain. Recipe reporting still uses leased-packet vocabulary in user-visible summaries, and the repository still keeps legacy knowledge/line-role packet or phase-ledger worker-contract code and tests around active modules.

Acceptance criteria

- Recipe runtime, summaries, and docs describe worker-owned task-file work rather than leased packets.
- Live knowledge runtime no longer imports or ships dead leased-packet worker helpers, and the old knowledge packet helper module/tests are removed from active code.
- Live line-role runtime/docs no longer export or test the old phase-ledger worker contract helpers; only deterministic validation/scaffold helpers needed by the task-file runtime remain.
- Focused LLM/parsing/staging/CLI tests pass after the cleanup.

Constraints / gotchas + brief decision

- Keep the live task-file runtime behavior unchanged; this is vocabulary and dead-contract cleanup, not a semantic redesign.
- Remove only legacy worker-contract code that is no longer needed by the shipped path. Keep deterministic validation/scaffold helpers that the current runtime still calls.
- Update docs only where they would otherwise describe the wrong live contract.

Rollback / compensation plan

- Revert this task as one atomic change if any live task-file runtime coverage regresses.
