# Speed Up Labelstudio Domain Fast Slice

## Problem

`./scripts/test-suite.sh domain labelstudio` was taking minutes because the non-slow slice still executed heavy single-offline benchmark-helper coverage. Routine agent loops were paying for helper internals that already have dedicated tests elsewhere.

## Acceptance Criteria

- `tests/labelstudio/test_labelstudio_benchmark_helpers_single_offline_run.py` is excluded from non-slow domain runs.
- Interactive benchmark routing tests stop at `_interactive_single_offline_benchmark(...)` instead of exercising the full helper internals.
- The targeted labelstudio fast slice used for routine loops is materially faster while preserving helper-specific coverage in focused runs.

## Verification

- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_interactive.py -m 'not slow'`
- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_offline_run.py -m slow`
- `. .venv/bin/activate && pytest tests/labelstudio -m 'not slow' --maxfail=1`
- `. .venv/bin/activate && pytest tests/labelstudio -m 'not slow' --durations=15 --durations-min=0.5`

## Evidence

- Before change, profiling found `test_labelstudio_benchmark_helpers_interactive.py` at about `121s` and `test_labelstudio_benchmark_helpers_single_offline_run.py` at about `79s`.
- After change, the interactive file should only validate menu/routing arguments, while the dedicated single-offline helper file remains available in the slow slice.

## Constraints / Decisions

- Keep production code unchanged; this is a test-scope/runtime classification fix.
- Preserve coverage boundaries: interactive menu tests verify routing, helper tests verify single-offline comparison/bundle/dashboard behavior.
