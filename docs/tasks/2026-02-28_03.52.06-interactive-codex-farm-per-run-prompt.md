## Problem statement

Interactive users needed an explicit per-run Codex Farm choice while setting up import/benchmark runs.
Before this change, Codex Farm selection required manual editor changes or all-method-only prompts.

## Acceptance criteria

- Interactive `choose_run_settings(...)` asks `Use Codex Farm recipe pipeline for this run?` for import and benchmark flows.
- Prompt default follows the selected profile's current `llm_recipe_pipeline` value.
- `None`/cancel on the prompt cancels run setup cleanly.
- Existing policy lock remains: enabling still requires `COOKIMPORT_ALLOW_CODEX_FARM=1`.
- Tests cover enable, disable, env-locked fallback, and cancel behavior.

## Verification steps

- Run: `. .venv/bin/activate && pytest tests/cli/test_c3imp_interactive_menu.py -q -k "choose_run_settings"`
- Run: `. .venv/bin/activate && pytest tests/cli/test_c3imp_interactive_menu.py -q`
- Run: `. .venv/bin/activate && pytest tests/llm/test_run_settings.py -q -k "run_settings_ui_specs"`

## Evidence

- Updated `cookimport/cli_ui/run_settings_flow.py` to add per-run Codex Farm confirm prompt logic.
- Updated `cookimport/cli.py` call sites to pass interactive confirm prompt into `choose_run_settings(...)`.
- Added chooser tests in `tests/cli/test_c3imp_interactive_menu.py` for enable/disable/gated/cancel cases.
- Updated docs:
  - `docs/02-cli/02-cli_README.md`
  - `docs/02-cli/02-cli_log.md`

## Constraints and decisions

- Kept existing `COOKIMPORT_ALLOW_CODEX_FARM` policy gate unchanged.
- Implemented prompt in shared chooser so import/single benchmark/single-profile benchmark/all-method setup all get one consistent per-run question.

## Rollback plan

- Revert per-run prompt wiring in `cookimport/cli_ui/run_settings_flow.py` and remove the new `prompt_confirm=` call-site arguments in `cookimport/cli.py`.
