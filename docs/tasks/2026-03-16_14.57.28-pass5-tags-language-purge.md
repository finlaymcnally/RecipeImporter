# Pass5 Tags Language Purge

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

This document must be maintained in accordance with `docs/PLANS.md`.

## Purpose / Big Picture

After this change, the active tagging surface should stop teaching the old `pass5` name for the optional LLM tag-suggestion stage. A person reading CLI help, run settings, stage manifests, raw tagging artifacts, or current tagging docs should see semantic `tags` naming instead of `pass5`.

The visible proof is simple: active CLI and config names use `codex_farm_pipeline_tags`, raw stage artifacts live under `raw/llm/<workbook>/tags/`, and current tests/docs no longer refer to `pass5`.

## Surprises & Discoveries

- Observation: stage observability already uses the semantic artifact stem `tags`, while nearby tagging docs/tests still mention `pass5_tags`.
  Evidence: `cookimport/runs/stage_observability.py` maps stage key `tags` to artifact stem `tags`, but `tests/tagging/test_tagging.py` still expects `raw/llm/<workbook>/pass5_tags/`.

- Observation: `codex_farm_pipeline_pass5_tags` is not a persisted `RunSettings` field; it is a fixed-behavior property threaded through stage/tagging adapters and CLI wiring.
  Evidence: `cookimport/config/run_settings.py` defines `RunSettings.codex_farm_pipeline_pass5_tags` as a property returning `bucket1_fixed_behavior().codex_farm_pipeline_pass5_tags`.

- Observation: the checked-in tagging test was already out of sync with the current runtime artifact contract before this rename because runtime writes `tags_manifest.json`, not `pass5_tags_manifest.json`.
  Evidence: `cookimport/runs/stage_observability.py` exposes `TAGS_MANIFEST_FILE_NAME = "tags_manifest.json"` and `cookimport/tagging/orchestrator.py` writes that file directly.

## Context and Orientation

The tagging code lives under `cookimport/tagging/`. The public standalone tagging CLI is `cookimport/tagging/cli.py`, while the main stage command is in `cookimport/cli.py`. The fixed-behavior run-settings seam is split between `cookimport/config/codex_decision.py`, `cookimport/config/run_settings.py`, and `cookimport/config/run_settings_adapters.py`.

The current runtime already thinks of the optional tagging work as the semantic `tags` stage in stage observability. The leftover `pass5` language is mostly naming debt around the fixed pipeline-id property, the CLI option name, the raw tagging directory examples, and a few focused tests/docs.

## Plan of Work

First rename the fixed-behavior/config seam. Replace `PASS5_TAGS_PIPELINE` and `codex_farm_pipeline_pass5_tags` with `TAGS_PIPELINE` and `codex_farm_pipeline_tags` in the config modules and stage/benchmark adapters.

Then rename the CLI/runtime surfaces. In `cookimport/cli.py` and `cookimport/tagging/cli.py`, replace `--codex-farm-pipeline-pass5-tags` with `--codex-farm-pipeline-tags` and update help text to talk about tag suggestions, not pass numbers. In the tagging provider/orchestrator, rename model/helper text and make raw temp/stage directories use `tags` instead of `pass5_tags`.

Finally update focused tests and current docs so they assert the new names and paths, then rerun the targeted test slices and grep the active tree for remaining live `pass5` references.
