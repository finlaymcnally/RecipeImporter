Problem statement

Direct Codex line-role shards are still drifting into agent behavior: initial passes try `ls`/`pwd`, and retry/repair passes narrate plans instead of emitting JSON. The prompt/runtime contract needs to stop inviting local file inspection and planner-mode responses.

Acceptance criteria

- The first-pass line-role prompt no longer tells Codex to open a local task file to do the work.
- The isolated direct-exec `AGENTS.md` no longer nudges the model to inspect local files just to orient itself.
- Line-role retry/repair prompts explicitly require immediate final JSON with no planning narration.
- Prompt preview/runtime artifacts reflect the new line-role input mode accurately.

Verification steps

- `./.venv/bin/pytest -q tests/parsing/test_canonical_line_roles.py -k "file_prompt or repairs_near_miss or retries_cohort_outlier or watchdog_killed_shards"`
- `./.venv/bin/pytest -q tests/llm/test_codex_exec_runner.py -k "prepare_direct_exec_workspace"`
- `./.venv/bin/pytest -q tests/llm/test_prompt_preview.py -k "structural_prompt_tokenization"`

Evidence

- Before: the live first-pass prompt said `Read the worker-local task file at ...`, and the isolated worker `AGENTS.md` said `Prefer reading the local task file directly`.
- After: the prompt embeds authoritative shard rows inline, the worker AGENTS text forbids file/workspace inspection for orientation, and retry/repair wrappers require final JSON immediately.

Constraints / gotchas

- Keep the mirrored `in/*.json` files and related artifact paths for traceability/debugging.
- Do not break prompt-preview token accounting; line-role should now count as inline prompt input.

Decision

- Inline the authoritative shard rows into the first-pass wrapper prompt, keep the mirrored file only as a trace artifact, and tighten retry/repair instructions instead of loosening the watchdog.
