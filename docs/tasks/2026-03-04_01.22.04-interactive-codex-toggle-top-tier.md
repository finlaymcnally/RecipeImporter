# Problem

Interactive import/benchmark runs prompted a separate automatic top-tier profile picker (`CodexFarm` vs `Vanilla`). Operator intent should be expressed as codex on/off; codex-enabled benchmark runs must still preserve vanilla baseline comparison behavior.

# Acceptance Criteria

- Interactive run settings ask `Use Codex Farm recipe pipeline for this run?` instead of a required profile picker menu.
- Prompt default follows global `llm_recipe_pipeline`.
- `COOKIMPORT_TOP_TIER_PROFILE` continues to force codexfarm/vanilla profile resolution.
- Codex-enabled single-offline benchmark still runs vanilla first, then codexfarm.
- CLI/LLM docs are updated to describe the prompt and profile mapping.

# Verification Steps

1. `source .venv/bin/activate && pytest tests/cli/test_c3imp_interactive_menu.py -k choose_run_settings`
2. `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py::test_interactive_single_offline_codex_enabled_runs_vanilla_then_codex_and_writes_comparison tests/labelstudio/test_labelstudio_benchmark_helpers.py::test_interactive_single_offline_codex_disabled_runs_only_vanilla_and_skips_comparison`

# Evidence

- `5 passed, 13 deselected, 1 warning in 1.80s`
- `2 passed, 1 warning in 2.37s`

# Constraints / Decisions

- Kept legacy menu-based selector as a fallback only when `prompt_confirm` is unavailable.
- Preserved deterministic top-tier profile harmonization behavior for both codex and vanilla branches.

# Rollback

- Revert `choose_run_settings` flow in `cookimport/cli_ui/run_settings_flow.py` to the prior menu-first profile selection path.
