# Problem statement

Benchmark prediction generation used `C3IMP_EPUB_EXTRACTOR` internally, but benchmark command paths did not expose a direct extractor choice.
Users could not reliably choose `unstructured` vs `legacy` per benchmark run.

# Acceptance criteria

- `cookimport labelstudio-benchmark` exposes `--epub-extractor` with `unstructured|legacy` validation.
- Interactive benchmark upload prompts for extractor choice before running upload mode.
- Benchmark prediction import receives the selected extractor at runtime.
- CLI docs mention the new benchmark extractor choice.

# Verification steps

- Fail-before:
  - `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_uses_golden_output_roots`
- Pass-after:
  - `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_uses_golden_output_roots tests/test_labelstudio_benchmark_helpers.py::test_labelstudio_benchmark_applies_epub_extractor_for_prediction_import tests/test_labelstudio_benchmark_helpers.py::test_labelstudio_benchmark_rejects_invalid_epub_extractor`

# Evidence

- Interactive benchmark upload test now asserts extractor selection is passed to `labelstudio_benchmark`.
- Benchmark command test asserts prediction import sees selected `C3IMP_EPUB_EXTRACTOR` and prior env is restored.

# Constraints/gotchas

- Benchmark prediction generation reads extractor from environment (`C3IMP_EPUB_EXTRACTOR`), so command wiring must set env for the benchmark import scope.
- Keep existing benchmark upload consent and credential behavior unchanged.

# Decision

Add a benchmark CLI option (`--epub-extractor`) and interactive upload prompt, then apply choice via a scoped env override around prediction import.

# Rollback / compensation plan

- Remove `--epub-extractor` from `labelstudio-benchmark` and restore previous interactive benchmark upload flow if this option causes regressions.
