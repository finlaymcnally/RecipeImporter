Constraints/gotchas + brief decision:
- Keep the old broad filenames where useful for existing smoke/slow routing, but move substantial seams into new focused files.
- Use shared support modules for helper-heavy suites so the new files stay readable.
- Do not change product behavior; this is a test-organization refactor plus small doc updates.
- Mechanical range splits can drop decorators at file seams; validate with real pytest runs, not just `ast.parse(...)` or `--collect-only`.
- When a focused split run fails in a behavior-heavy suite, compare against the original unsplit file before widening scope into unrelated product fixes.
- For prompt/assertion-heavy seams, it is enough to prove the original unsplit file fails the same assertion in-repo; do not turn a test-split task into a prompt-contract rewrite unless explicitly asked.

Rollback / compensation plan:
- Revert the split files and `tests/conftest.py` marker updates together if collection or marker routing regresses.
