# Codex Spinner Stage Readability

## Problem statement

Live benchmark spinner lines surfaced raw codex pipeline IDs and long worker payloads, making it hard to tell which codex-farm stage/pass is currently running.

## Acceptance criteria

- Spinner status shows human-readable codex pass labels instead of only raw pipeline IDs.
- Panel includes a dedicated `stage:` line for codex progress updates.
- Existing ETA behavior (`task X/Y`, `running`) remains intact.
- Regression tests verify stage readability and spinner width/ETA behavior.

## Constraints/gotchas + decision

- Keep deterministic parsing only; no LLM or heuristic outside existing progress payload structure.
- Decision: rewrite display strings inside CLI spinner rendering path while preserving raw counters and active-task worker rows.

## Rollback / compensation

- Revert codex progress summarization helpers and remove the new readability test if raw pipeline IDs are preferred.
