Problem statement

The offline single-book benchmark can finish scoring successfully and then crash in the CLI summary path with `name '_coerce_int' is not defined`.
That leaves the run marked failed even though prediction and evaluation artifacts were written.

Acceptance criteria

- The benchmark summary path no longer crashes when `report.recipe_counts.predicted_recipe_count` is present.
- The CLI uses an explicitly imported/shared integer coercion helper instead of an unresolved global.
- A focused benchmark test fails before the fix and passes after it.

Verification steps

- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_eval_payload_execution.py::test_labelstudio_benchmark_prints_predicted_recipe_count_without_name_error tests/core/test_benchmark_undefined_names.py::test_all_cli_modules_have_no_unresolved_global_loads`

Evidence

- Before fix: the scoped benchmark test crashes in the summary path with `_coerce_int` unresolved.
- After fix: the scoped benchmark test and CLI unresolved-global audit both pass.

Constraints / gotchas

- Keep the fix local to the labelstudio benchmark summary path.
- Do not revert unrelated workspace edits.

Decision

- Import the shared `_coerce_int` helper from `cookimport.cli_support.bench_artifacts` under an explicit local alias and use it in the summary print path.
