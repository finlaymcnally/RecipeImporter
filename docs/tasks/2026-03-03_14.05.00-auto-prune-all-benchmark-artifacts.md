# Task: Restrict benchmark auto-prune scope

## Problem statement
- Aggressive benchmark pruning removed normal interactive single-offline outputs that should be retained.
- Dashboard/history artifacts under interactive output roots could disappear when those roots were pruned.

## Acceptance criteria
- `labelstudio-benchmark` prunes only excluded transient benchmark artifacts (gate/test/regression slop and `/bench/`-scoped benchmark artifacts).
- Line-role gated `FAIL` exits prune before `_fail(...)` raises.
- Normal interactive single-offline outputs are not auto-pruned.
- Interactive `C3imp` benchmark menu runs suppress prune entirely unless cleanup is requested separately.
- Benchmark GC tests covering prune helper behavior pass.

## Verification steps
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_gc.py -k "prune_benchmark_outputs"`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "single_offline"`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "prunes_transient_artifacts_only_after_csv_append"`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k "skip_gated or skip_pytest_temp or csv_collector_merges_nested_benchmark_history_csv_rows_and_skips_gated_runs"`

## Evidence
- Restored classifier-based prune guard in `_prune_benchmark_outputs(...)` so only excluded/transient artifacts are deleted.
- Removed interactive single-offline batch-end root prune.
- Updated tests back to retained single-offline artifacts and classifier-driven prune behavior.
- Added regression test proving transient prune runs after CSV append and leaves durable history rows.
- Added regression test proving interactive benchmark context sets prune suppression.
- Updated benchmark docs to match the corrected retention scope.

## Constraints/gotchas and decision
- All-method/speed/single-offline flows read run artifacts after each `labelstudio_benchmark(...)` call.
- Decision: prune only excluded/transient artifacts and keep interactive outputs by default.

## Rollback / compensation plan
- Revert this task diff to restore previous aggressive pruning behavior (not recommended).
