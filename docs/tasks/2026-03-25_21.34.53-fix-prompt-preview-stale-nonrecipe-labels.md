# Prompt Preview Stale Nonrecipe Labels

## Problem

`cf-debug preview-prompts` failed on the March 22 `saltfatacidheatcutdown` vanilla artifact because knowledge preview rebuilt Stage 7 routing from saved `authoritative_block_labels.json`, and that older processed run still had recipe-local `final_label` values on some non-recipe blocks.

## Acceptance Criteria

- Prompt preview no longer crashes when a saved deterministic artifact has recipe-local labels outside recipe spans.
- Preview-only knowledge planning excludes those stale rows instead of widening the knowledge review queue.
- Focused prompt preview tests cover the stale-label case.
- `cf-debug preview-prompts` succeeds on `data/golden/benchmark-vs-golden/2026-03-22_21.47.46/single-book-benchmark/saltfatacidheatcutdown/vanilla`.

## Verification

- `./.venv/bin/pytest tests/llm/test_prompt_preview.py -q`
- `./.venv/bin/cf-debug preview-prompts --run data/golden/benchmark-vs-golden/2026-03-22_21.47.46/single-book-benchmark/saltfatacidheatcutdown/vanilla --out /tmp/2026-03-25_21.30.15-saltfatacidheatcutdown-vanilla-preview-fixed`

## Evidence

- Added a preview-only sanitization step before knowledge preview rebuilds `build_nonrecipe_stage_result(...)`.
- Added a regression test that mutates a non-recipe block to `RECIPE_TITLE` and confirms knowledge preview still only plans the valid knowledge block.
- Re-ran the failing vanilla preview command after the patch.

## Constraints / Decision

Keep live Stage 7 validation strict. The compatibility behavior belongs in prompt preview only, because the bug is stale saved artifact shape rather than a valid live non-recipe contract.

## Rollback

Revert the preview-only sanitization helper, the regression test, and this task file together.
