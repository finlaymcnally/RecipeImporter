Problem statement

Replace the live recipe, knowledge, and line-role workspace happy path with one editable `task.json` file per worker session. The model should read that file, edit only answer fields in place, and stop; deterministic repo code should validate and expand those answers into the existing stage artifacts.

Acceptance criteria

- Recipe, knowledge, and line-role workers build and consume `task.json` instead of model-written `out/*.json` payloads on the happy path.
- Deterministic validation rejects edits outside `/units/*/answer` and can rewrite a repair-mode `task.json` for one bounded retry.
- Focused runtime tests prove the worker prompt and fake runner center on `task.json`, and stage runtimes still emit valid final artifacts after deterministic expansion.

Verification steps

- `source .venv/bin/activate && pytest tests/llm/test_codex_exec_runner_workspace.py`
- `source .venv/bin/activate && pytest tests/llm/test_recipe_phase_workers.py`
- `source .venv/bin/activate && pytest tests/llm/test_knowledge_orchestrator_runtime_leasing.py`
- `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles.py -k "workspace or compact_prompt"`

Evidence

- Before the change, worker prompts and runtime code taught assignment files plus model-written `out/*.json` ledgers.
- After the change, worker prompts point at `task.json`, repo code validates in-place answer edits, and final shard artifacts are rebuilt deterministically.

Constraints/gotchas + brief decision

- Keep deterministic code authoritative for validation and final artifact assembly; the model only owns compact semantic answers.
- Do not run live Codex-enabled book processing or benchmarks while implementing this reset.
