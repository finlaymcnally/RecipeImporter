# Task: EPUB Common Issues Quick Wins

## Problem Statement

EPUB imports were vulnerable to common text/structure failures (soft hyphens, BR-collapsed ingredient lines, bullet prefixes, nav/pagebreak noise), which degraded segmentation and parsing quality downstream.

## Acceptance Criteria

- Add shared EPUB text normalization for Unicode/whitespace/fraction cleanup.
- Add shared EPUB postprocess pass used by both `legacy` and `unstructured` extractor paths.
- Skip nav/TOC spine docs and suppress obvious pagebreak noise.
- Add EPUB extraction health metrics + warning keys and emit raw artifact.
- Add synthetic regression tests for quick-win bug classes.
- Update relevant docs, conventions, understandings, and ExecPlan record.

## Evidence

- Targeted EPUB suite: `33 passed`.
- Additional affected suites: `34 passed`.
- New regression tests cover:
  - soft hyphen/zero-width/fraction normalization
  - BR line splitting
  - bullet prefix stripping
  - nav/TOC suppression
  - pagebreak suppression
  - legacy table-row ingredient shaping
  - health warning/artifact emission

## Constraints / Gotchas and Decision

- Shared cleanup was deliberately limited to HTML-based extractors (`legacy`, `unstructured`) and not applied to `markitdown`.
- `_read_epub_spine` needed richer typed metadata (`item_id`, `properties`) so zip-fallback extraction can skip nav docs consistently.
- Debug extraction tooling needed the same postprocess call as importer flow to maintain parity.

## Rollback / Compensation Plan

- Revert `cookimport/parsing/epub_postprocess.py` and `cookimport/parsing/epub_health.py`.
- Remove EPUB postprocess/health wiring from `cookimport/plugins/epub.py` and debug parity hook in `cookimport/cli.py`.
- Remove new quick-win test modules and keep existing extractor behavior.
