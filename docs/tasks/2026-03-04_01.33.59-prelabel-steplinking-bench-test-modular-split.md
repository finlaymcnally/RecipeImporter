## Problem statement

Three larger test files still mixed multiple concerns in one module, making focused triage and collect-only loops slower than needed.

## Acceptance criteria

- `test_labelstudio_prelabel.py` is split by seam (prompt/span contracts vs codex CLI contracts).
- `test_step_ingredient_linking.py` is split by seam (core assignment/split logic vs semantic/fuzzy/collective matching).
- `test_bench.py` is split by seam (helper/noise/cost contracts vs speed CLI wiring vs quality CLI wiring).
- `tests/conftest.py` marker mapping includes all new files.
- Split files collect and representative tests pass.

## Evidence

- Split collection: `17 + 19 + 16 + 15 + 11 + 9 + 11`.
- Representative run: `7 passed, 1 warning in 2.17s`.
- Marker-filtered collection confirms new files are tagged under their expected domains.

## Constraints/gotchas + decision

- Split modules import non-test globals from their base module to avoid copying shared helper scaffolding.
- Excluding `__dunder__` globals prevents pytest import mismatch behavior.
- Keep split boundaries concern-oriented so future runs can target only needed seams.

## Rollback / compensation plan

Merge moved tests back into original files and revert marker-map updates in `tests/conftest.py`.
