# Pass2 Page/Layout Warning Bucket Rename

## Problem statement

Pass2 degradation reason `warning_bucket:ocr_or_page_artifact` reads as OCR-specific, but EPUB failures are often layout/page-marker residue.

## Acceptance criteria

- New codex manifests emit `warning_bucket:page_or_layout_artifact`.
- Existing old-name manifests still normalize to the new bucket in cutdown reports.
- Impacted tests pass.

## Verification steps

- `source .venv/bin/activate && pytest tests/llm/test_codex_farm_orchestrator.py`
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py -k "starter_pack_v1_contract_files or includes_project_context_digest_and_metadata"`

## Evidence

- `source .venv/bin/activate && pytest tests/llm/test_codex_farm_orchestrator.py` -> `29 passed, 1 warning`
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py -k "starter_pack_v1_contract_files or includes_project_context_digest_and_metadata"` -> `2 passed, 18 deselected`
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py` -> `20 passed`

## Constraints/gotchas + decision

- Keep degrade behavior unchanged; rename diagnostics only.
- Decision: rename in orchestrator and add canonicalization helper in cutdown script for backward compatibility.

## Rollback / compensation

- Revert bucket rename and compatibility helpers if downstream tools require legacy naming only.
