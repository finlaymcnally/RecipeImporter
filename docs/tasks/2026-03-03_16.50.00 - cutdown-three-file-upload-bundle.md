# Problem

`scripts/benchmark_cutdown_for_external_ai.py` emits many artifacts, including CSV, which is inconvenient for web-based AI uploads that work better with a small number of files.

# Acceptance Criteria

- Script supports an opt-in 3-file upload bundle mode.
- Bundle includes topline summary + diagnostics navigation + full artifact payload content.
- Bundle payload is lossless for source artifacts (structured/text or safe encoded fallback).
- Optional mode can prune output directory to exactly 3 non-CSV files.
- Existing default flow remains intact when new flags are not provided.

# Verification Steps

1. Run targeted tests:
   - `./scripts/test-suite.sh domain bench tests/bench/test_benchmark_cutdown_for_external_ai.py -k upload_3_files_only`
2. Confirm the new test asserts:
   - output folder has only the three upload files in `--upload-3-files-only` mode
   - no uploaded file has `.csv` extension
   - starter-pack CSV content is preserved inside payload JSONL
   - process manifest fields reflect upload mode settings

# Evidence

- Added a new bench test:
  - `test_main_upload_3_files_only_consolidates_and_preserves_artifacts`

# Constraints And Decision

- Keep the existing multi-file contract unchanged by default.
- Introduce explicit opt-in flags (`--upload-3-files`, `--upload-3-files-only`) to avoid breaking existing flows while enabling the 3-file upload use case.

# Rollback Plan

- Remove the two CLI flags and helper functions for upload bundling.
- Re-run the benchmark cutdown test file to confirm baseline behavior is restored.
