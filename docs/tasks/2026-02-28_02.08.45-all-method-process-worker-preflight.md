## Problem statement

All-method runs in restricted environments repeatedly hit `ProcessPoolExecutor` startup errors and log "falling back to serial mode," which sounds like a runtime failure even when execution is healthy.

## Acceptance criteria

- All-method config scheduling probes process-worker availability before parallel startup.
- When process workers are unavailable, all-method runs single-config execution directly without the old fallback message wording.
- Behavior is covered by tests and CLI docs are updated.

## Verification steps

- Run: `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -q`
- Confirm the executor-unavailable test passes with the updated status wording.

## Evidence

- Updated all-method scheduler paths in `cookimport/cli.py`.
- Updated test assertion in `tests/labelstudio/test_labelstudio_benchmark_helpers.py`.
- Updated CLI docs note in `docs/02-cli/02-cli_README.md`.

## Constraints and decisions

- Keep runtime behavior the same for results/correctness; only startup scheduling path and operator-facing messaging changed.
- Keep a defensive executor exception handler in place for rare mid-run failures.

## Rollback plan

- Revert the new process-worker preflight helper and restore prior executor-start fallback behavior/messages.
