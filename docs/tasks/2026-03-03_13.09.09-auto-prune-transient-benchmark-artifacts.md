# Auto-prune transient benchmark artifacts

## Problem statement

Transient benchmark runs (gate/test/smoke iterations) leave heavy artifact folders under `data/golden/benchmark-vs-golden` and `data/output` even when only metrics are needed.

## Acceptance criteria

- `labelstudio-benchmark` auto-prunes transient benchmark output directories after benchmark CSV history append succeeds.
- Matching processed-output run roots are pruned in the same step.
- Non-transient official runs are not auto-pruned.
- Bench docs mention the new retention behavior.
- Tests cover transient prune + non-prune cases.

## Verification steps

- `source .venv/bin/activate && pytest tests/bench/test_benchmark_gc.py -k "prune_transient_benchmark_outputs"`
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_gc.py`

## Evidence

- `source .venv/bin/activate && pytest tests/bench/test_benchmark_gc.py -k "prune_transient_benchmark_outputs"` -> `2 passed, 13 deselected, 1 warning in 1.97s`
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_gc.py` -> `15 passed, 1 warning in 1.84s`

## Constraints/gotchas + decision

- Reuse existing transient artifact classifier (`dashboard_collect` path policy) so cleanup rules match dashboard exclusion rules.
- Prune happens only after history append point to preserve durable CSV metrics first.

## Rollback / compensation

- Remove `_prune_transient_benchmark_outputs` calls from `labelstudio_benchmark` and delete helper/tests/docs updates.
