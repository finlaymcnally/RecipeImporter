# Task

## Problem

Benchmark-side Oracle upload only launched one generic turn-1 review, even though the benchmark follow-up work had two different goals: push quality toward `>95%` and reduce token spend.
That single chat mixed accuracy triage with cost triage and shipped one broad starter packet instead of purpose-built review packets.

## Acceptance criteria

- Benchmark Oracle upload supports explicit review profiles `quality` and `token`.
- Interactive benchmark wrap-up and `cookimport bench oracle-upload` launch both profiles by default, with `--profile quality|token|all` for replay control.
- Each profile gets its own prompt, its own staged lane brief, its own logical-path-selected `upload_bundle_payload.jsonl` subset, and its own launch directory/metadata under `.oracle_upload_runs/<timestamp>-<profile>/`.
- Unit coverage proves prompt rendering, profile packet staging, metadata persistence, and CLI default behavior for both lanes.
- No live Oracle call is required for acceptance.

## Verification

- `./.venv/bin/pytest tests/bench/test_benchmark_oracle_upload.py tests/bench/test_oracle_followup.py -q`

## Evidence

- The implementation added two prompt templates: `llm_pipelines/prompts/benchmark.oracle-upload.prompt.md` for `quality` and `llm_pipelines/prompts/benchmark.oracle-upload.token.prompt.md` for `token`.
- The uploader now persists `review_profile` and `review_profile_display_name` in `oracle_upload.json` and `oracle_upload_status.json`, and browser/background runs use launch dirs suffixed with the profile id.
- Targeted pytest coverage passed locally after the change; no live Oracle upload was run.

## Constraints / decision

- Keep `upload_bundle_v1/` immutable on disk and stage profile-specific packets only under `.oracle_upload_runs/.../staging/`.
- Select payload rows by logical artifact `path`, not bundle-local row number, because row ordering is not durable across bundles.
- Preserve the existing follow-up request schema so turn-2 automation keeps working without a second protocol change.
- Keep live Oracle/token-spending actions approval-gated.

## Rollback / compensation

- Revert the profile-aware upload changes in `cookimport/bench/oracle_upload.py`, `cookimport/cli.py`, the new token prompt template, and the related tests/docs to restore the prior single-lane upload behavior.
