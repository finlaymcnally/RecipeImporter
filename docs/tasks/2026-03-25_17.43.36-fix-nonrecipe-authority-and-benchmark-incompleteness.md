# Fix Nonrecipe Authority And Benchmark Incompleteness

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

## Purpose / Big Picture

After this change, the non-recipe path will behave honestly in three places that currently drift apart: stage outputs, benchmark scoring, and knowledge planning. A stage run with knowledge review off or partially incomplete should still keep the correct outside-recipe rows for late deterministic outputs, benchmark artifacts should show unresolved review-eligible blocks as unresolved instead of silently downgrading them to `OTHER`, and the knowledge shard-count control should affect shard planning the way the repo now teaches.

You can see the result by running the focused staging, bench, labelstudio, and knowledge-planning tests listed below. The important observable outcomes are: unresolved outside-recipe blocks are carried in explicit artifact metadata, scoring reports expose unresolved coverage instead of pretending those blocks were semantically decided, and knowledge packet planning groups packets into real shard bundles when the requested shard count is lower than the packet count.

## Surprises & Discoveries

- Observation: the current non-recipe seed stage intentionally records final authority only for immediately excluded obvious-junk rows, which is fine for the authority artifact itself, but the shared stage output flow later reuses `final_nonrecipe_blocks` for tables and deterministic chunks.
  Evidence: `cookimport/staging/nonrecipe_stage.py` seeds `authoritative_block_indices` from `review_excluded_block_indices`, while `cookimport/staging/import_session_flows/output_stage.py` currently feeds `knowledge_final_result.final_nonrecipe_blocks` into table extraction and `chunks_from_non_recipe_blocks(...)`.

- Observation: the live knowledge planner was already bundling packets into shard entries; the real drift was the dead function parameters and stale assumptions in review notes, not the bundling algorithm itself.
  Evidence: `tests/llm/test_knowledge_packet_planning.py` already passes against `cookimport/llm/codex_farm_knowledge_jobs.py` with multi-packet shard ownership when `prompt_target_count=1`.

- Observation: the prediction-run projection artifact now keeps provisional outside-recipe block labels and uses explicit unresolved metadata rather than rewriting those block labels to `OTHER`.
  Evidence: `cookimport/labelstudio/canonical_line_projection.py` and `cookimport/labelstudio/ingest_flows/artifacts.py` now emit `unresolved_review_eligible_*` fields while preserving projected block labels for reviewer/debug use.

## Decision Log

- Decision: keep `OTHER` and `KNOWLEDGE` as the only semantic outside-recipe labels, and represent incompleteness through explicit unresolved metadata rather than a new semantic label.
  Rationale: benchmark and Label Studio consumers already treat the freeform label set as a stable external contract. Adding a new semantic label would create wider churn than needed. The real bug is hiding incompleteness as semantics, not lacking one more meaning-bearing label.

  Rationale: outside-recipe deterministic outputs such as tables and knowledge-off chunks still need the surviving review-eligible rows, while benchmark `KNOWLEDGE` evidence must stay tied to final semantic authority only.

## Context and Orientation

The affected runtime starts after the shared five-stage stage session is already in progress. `cookimport/staging/nonrecipe_stage.py` owns the deterministic outside-recipe routing result. In that module, “routing” means deciding which outside-recipe blocks are obvious junk and which survive for later review. “authority” means the final semantic truth surface that is allowed to say `knowledge` or final `other`. `cookimport/staging/pipeline_runtime.py` and `cookimport/staging/import_session_flows/output_stage.py` turn those stage results into late outputs such as tables, chunks, and benchmark artifacts.

`cookimport/staging/stage_block_predictions.py` writes `.bench/<workbook>/stage_block_predictions.json`. That artifact is the benchmark-facing per-block label manifest. `cookimport/bench/eval_stage_blocks.py` and `cookimport/bench/eval_canonical_text.py` are the scoring readers. `cookimport/labelstudio/ingest_flows/artifacts.py` and `cookimport/labelstudio/canonical_line_projection.py` build the corresponding prediction-run projection artifacts used by Label Studio benchmark flows.

`cookimport/llm/codex_farm_knowledge_jobs.py` plans model-facing knowledge packets and the shard manifest entries that later worker runtime code consumes. In this repo, a “packet” is the content slice owned by one task, and a “shard” is the scheduling bundle that can own one or more packets. The current docs and some tests already expect one shard to be able to own several packets when the requested shard count is lower than the packet count.

The changes in this plan must keep the current external semantic labels stable while making incompleteness explicit and making shard planning honest.

## Plan of Work

First, change the non-recipe runtime contracts so the code can distinguish three different block sets cleanly: review-eligible outside-recipe rows, final-authority outside-recipe rows, and the late deterministic outside-recipe rows used when knowledge review is off or partially incomplete. This work belongs in `cookimport/staging/nonrecipe_stage.py`, `cookimport/staging/pipeline_runtime.py`, and `cookimport/staging/import_session_flows/output_stage.py`. The important behavior change is that tables and knowledge-off chunks should operate on the surviving outside-recipe rows, while benchmark `KNOWLEDGE` evidence still comes only from final authority.

Second, extend the stage-block prediction artifact so unresolved review-eligible blocks are first-class metadata. Edit `cookimport/staging/stage_block_predictions.py` to write unresolved block indices and counts. Then teach `cookimport/bench/eval_stage_blocks.py`, `cookimport/bench/eval_canonical_text.py`, `cookimport/labelstudio/canonical_line_projection.py`, and `cookimport/labelstudio/ingest_flows/artifacts.py` to preserve or respect that unresolved metadata. Scoring should use only authoritative predictions for accuracy/F1, while also reporting unresolved coverage counts. Projection code should stop rewriting unresolved `KNOWLEDGE` candidates into semantic `OTHER`.

Third, fix the knowledge planner contract in `cookimport/llm/codex_farm_knowledge_jobs.py`. Keep packet files one-per-packet, but build shard manifest entries that can own several packet ids when the requested shard count is lower than the packet count. Remove or stop advertising planner parameters that are currently discarded. Keep the worker runtime contract intact by ensuring multi-packet shard payloads still match what `cookimport/llm/knowledge_stage/planning.py` already knows how to split into per-packet tasks.

Finally, update the focused tests first where the current behavior is wrong, then update the docs that own these contracts: `docs/05-staging/05-staging_readme.md`, `docs/07-bench/07-bench_README.md`, `docs/10-llm/10-llm_README.md`, and the high-level walkthrough if needed.
