## Problem

Single-offline paired `vanilla`/`codexfarm` runs were silently forcing different `atomic_block_splitter` values, which made the comparison drift on representation instead of only the intended Codex surfaces.

## Acceptance Criteria

- Paired single-offline variants use the same `atomic_block_splitter` value.
- The shared value comes from the selected run settings.
- Default single-offline paired runs keep `atomic_block_splitter=off` unless the operator explicitly selects `atomic-v1`.

## Verification

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_offline_run.py -q`

## Evidence

- Before the fix, the updated single-offline helper test failed because `codexfarm` still forced `atomic-v1`.
- After the fix, the same test file passes and covers both the default shared-`off` path and the explicit shared-`atomic-v1` path.

## Constraints / Decision

- Scope stays limited to single-offline paired variant planning.
- Recipe-vs-Codex surface selection is unchanged; only `atomic_block_splitter` alignment changed.

## Rollback

- Revert the helper change in `cookimport/cli.py` and the paired-run contract returns to the old forced split behavior.
