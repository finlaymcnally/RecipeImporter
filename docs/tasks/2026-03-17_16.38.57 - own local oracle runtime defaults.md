# Own Local Oracle Runtime Defaults

- Consolidated local Oracle browser state onto one canonical profile: `~/.local/share/oracle/browser-profile`.
- Left `~/.oracle/browser-profile` in place only as a compatibility symlink to the canonical profile.
- Patched the installed Oracle 0.8.6 runtime under `~/.nvm/.../@steipete/oracle/dist/` so its own defaults now prefer the canonical Oracle home/profile and default browser model strategy `ignore` instead of brittle picker automation.
- Added one shared wrapper helper at `~/.local/bin/oracle-env-common` so `oracle-browser-safe`, `oracle-browser-headless`, and `chromium-chatgpt-login` all resolve the same Oracle home/profile.
- Stopped forcing Chrome DevTools port `9222` in the local wrappers unless `ORACLE_BROWSER_PORT` is explicitly set.
- Restored `oracle-browser-headless` to the xvfb-backed launcher path so Codex-run Oracle smoke tests still work when this shell has no usable X display for a headful browser.
- Added `/home/mcnal/.local/bin/chromium-oracle-auto` and `/home/mcnal/.local/bin/oracle-browser-auto`, then made generic `oracle` browser runs default to that auto path so machine-wide Oracle usage prefers visible Chromium when a real display exists and xvfb otherwise.
