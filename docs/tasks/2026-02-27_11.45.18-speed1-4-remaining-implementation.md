## Problem statement

`docs/understandings/2026-02-27_10.51.52-speed1-4-implementation-gap-audit.md` identified speed1-4 gaps: no explicit stage-runner API surface, run-level prediction-record contract instead of per-example records, evaluate-only replay tied to path pointers, and limited pipelined producer/consumer behavior.

## Acceptance criteria

- Explicit benchmark stage/runner functions exist and are used by `labelstudio_benchmark`:
  - `predict_stage(...)`
  - `evaluate_stage(...)`
  - `run_legacy(...)`
  - `run_pipelined(...)`
- `--predictions-out` writes per-block/per-example records (`schema_kind=stage-block.v1`) with deterministic `example_id` and `example_index`.
- `--predictions-in` evaluate-only supports:
  - per-block replay records (new contract),
  - legacy single-record run-pointer artifacts (backward compatibility).
- Pipelined mode uses bounded producer/consumer record flow with EOS/error propagation.
- Regression tests cover per-block output, replay behavior, legacy compatibility, and legacy-vs-pipelined parity.
- Bench docs/plan/understandings reflect the updated contract and remaining open timing-evidence item.

## Evidence

- Fail-before reference captured while shifting tests from run-level to per-block record expectations:
  - `pytest ... -k 'predictions_out or ... predict_only_mode_skips_evaluation ...'` failed with:
    - `assert len(records) == 1` where `records == []` in two tests still using zero-block fixtures.
- Pass-after captured:
  - `python -m py_compile cookimport/cli.py tests/labelstudio/test_labelstudio_benchmark_helpers.py` exited `0`.
  - `pytest -q tests/bench/test_prediction_records.py` exited `0`.
  - `pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers.py` exited `0`.

## Constraints and decision

- Constraint: preserve existing scoring semantics and evaluator internals.
- Decision: implement stage boundary through record-driven replay reconstruction:
  - prediction stage emits per-block records,
  - evaluate stage uses reconstructed stage/extracted artifacts from records,
  - legacy run-pointer record input remains supported.
