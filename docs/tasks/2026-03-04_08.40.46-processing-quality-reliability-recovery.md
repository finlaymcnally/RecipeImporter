# Recover processing quality with adaptive cross-version benchmarking

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

Maintain this document in accordance with `docs/PLANS.md` from the repository root.

## Purpose / Big Picture

After this work lands, benchmark and processing outputs remain trustworthy while deterministic extraction settings continue evolving. Existing gold spans stay unchanged, and benchmark scoring adapts deterministically to the current extraction method so historical gold remains usable.

The user-visible outcome is that modern runs (for example `v2 + semantic_v1`) can still be compared against gold created under older settings (for example `v1 + br_split_v1`) without manually editing gold files. In the same pass, report totals must match produced artifacts, vanilla and codex final draft payloads must share the same compatibility shape, and narrative-heavy segmentation/tip-topic noise must be reduced with measurable evidence.

## Scope Guardrails (Intent Lock)

This section exists to preserve the original intent while improving execution quality.

- Gold data in `data/golden/pulled-from-labelstudio/**` is immutable for this plan.
- Benchmarking adapts to extraction/preprocess drift; extraction is not rolled back to gold-era settings.
- CodexFarm remains in-scope as a first-class variant in all quality/benchmark validation.
- Deterministic parsing/segmentation/filtering only. No LLM-based parsing/cleaning is introduced.
- The goal is reliability and comparability recovery, not product-surface expansion.

## Surprises & Discoveries

- Observation: report totals can be semantically wrong even when report files look healthy.
  Evidence: `cookimport/core/reporting.py::enrich_report_with_stats` does not set count totals; it assumes producers already populated them.

- Observation: labelstudio ingest parallel merge drops totals by constructing a warning-only report.
  Evidence: `cookimport/labelstudio/ingest.py::_merge_parallel_results` uses `ConversionReport(warnings=warnings)` and does not set recipe/tip/topic totals.

- Observation: the hard `extra_gold` failures on roast/saltfat/foodlab are consistent with extractor/preprocess drift, not source-text drift.
  Evidence: run configs show `epub_unstructured_html_parser_version=v2` + `epub_unstructured_preprocess_mode=semantic_v1`, while gold touched-block metadata shows `v1` + `br_split_v1`.

- Observation: seaandsmoke behavior is unexpectedly poor in this run versus historical expectation, so this must be investigated as a dedicated track, not treated as ordinary variance.
  Evidence: user-reported historical performance expectation conflicts with current poor metrics and high segmentation errors.

- Observation: final draft contract shape is inconsistent by variant, creating downstream fragility.
  Evidence: vanilla `174/174` files missing top-level `name/ingredients/instructions`; codexfarm only partially populated.

- Observation: alias-normalized `instructions` must include deterministic prep-step injections from draft conversion, not only source instruction strings.
  Evidence: staging test initially expected only source instructions and failed because `recipe_candidate_to_draft_v1(...)` prepended `Gather and prepare ingredients.` when ingredients were otherwise unassigned.

- Observation: adaptive remap can be safely enabled by default only when bounded by explicit quality gates.
  Evidence: adding `min_coverage` and `max_ambiguous` thresholds produced deterministic failure on synthetic unresolved gold coverage and deterministic pass on stable-key-aligned fixtures.

- Observation: many seaandsmoke benchmark artifacts in active roots are canonical-text and do not carry stage-block segmentation payloads.
  Evidence: current seaandsmoke run manifests under `2026-03-04_06.58.16/.../04_seaandsmokecutdown/*/run_manifest.json` show `eval_mode=canonical-text`, and corresponding `eval_report.json` files have no populated `segmentation` block.

- Observation: stage-block validation-matrix artifacts provide explicit seaandsmoke segmentation comparison and show current-method degradation versus baseline.
  Evidence: `data/golden/benchmark-vs-golden/2026-03-04_10.40.04-validation-matrix/sea-stage-block-current-v2-semantic/eval_report.json` reports overall-micro segmentation `f1=0.034364`, while `.../sea-stage-block-baseline-v1-brsplit/eval_report.json` reports `f1=0.243750`.

- Observation: implicit default-zero report totals were being rewritten without diagnostics despite OG intent asking for explicit mismatch surfacing.
  Evidence: previous test `test_write_processed_outputs_skips_report_total_mismatch_diagnostics_for_implicit_defaults` asserted no diagnostics file; behavior was updated to emit diagnostics with `prepopulated=false`.

- Observation: benchmark helper eval roots under pytest temp folders were being deleted unexpectedly after successful runs.
  Evidence: `_prune_benchmark_outputs(...)` reused `_is_excluded_benchmark_artifact(...)`, and that helper intentionally marks `/tmp/pytest-*/test_*/eval` as excluded for dashboard scans.

- Observation: split-merge unit tests can execute codex-farm paths accidentally when `run_config` is omitted.
  Evidence: `_merge_split_jobs(...)` called `RunSettings.from_dict(None)` and enabled LLM recipe pipeline from defaults; direct test invocations with no full-text blocks raised `CodexFarmRunnerError`.

- Observation: the previous version of this plan had strong direction but insufficient execution-level detail for a stateless implementer.
  Evidence: phase goals existed, but milestone completion criteria, output artifact expectations, and retry behavior were not specific enough to guarantee consistent implementation.

Definitions used in this plan:

- Gold set: exported freeform labels under `data/golden/pulled-from-labelstudio/<book>/exports/freeform_span_labels.jsonl`.

## Plan of Work

### Milestone 1: Report truth hardening with explicit mismatch evidence

Implement a shared report finalization path that computes count totals from the in-memory `ConversionResult` before write, then reuses that path in all ingest merge code paths.

Files to touch:

- `cookimport/core/reporting.py`
- `cookimport/labelstudio/ingest.py`
- `tests/labelstudio/test_labelstudio_ingest_parallel.py`

Implementation details:

- Add one canonical helper in reporting that sets `totalRecipes`, `totalTips`, `totalTipCandidates`, `totalTopicCandidates`, `totalStandaloneBlocks`, and `totalStandaloneTopicBlocks` from result objects.
- Replace any direct `ConversionReport(warnings=...)` merge-only construction with the canonical finalization helper.
- Add mismatch detection that compares finalized totals to in-memory counts and emits:
  - warning entries for human visibility, and
  - a machine-readable diagnostics payload under the run output root.

Milestone exit criteria:

- A reproduced merged-job scenario no longer emits zero totals when artifacts exist.
- Diagnostics payload is present when mismatch is forced in tests.

### Milestone 2: Adaptive gold remap for stage-block eval when fingerprints differ

Implement deterministic remap from immutable gold spans to current-run block indices when blockization metadata indicates mismatch.

Files to touch:

- `cookimport/bench/eval_stage_blocks.py`
- `cookimport/cli.py` (`labelstudio-benchmark` command path)
- `tests/bench/**` relevant eval/remap coverage files

Implementation details:

- Add explicit fingerprint comparison gate:
  - matching fingerprints: existing strict path unchanged,
  - mismatching fingerprints: adaptive remap path executes.
- Define remap algorithm in deterministic steps:
  - normalize comparable text units from gold and current blocks,
  - build candidate matches using overlap and offset consistency,
  - resolve ambiguity deterministically with score ordering and tie rules,
  - mark unresolved spans explicitly.
- Emit per-book remap diagnostics artifact that includes:
  - coverage ratio,
  - ambiguous count,
  - unresolved count,
  - mapping confidence summary.
- Add CLI controls (default behavior preserves intent):
  - `--gold-adaptation-mode off|auto|force` (default `auto`),
  - `--gold-adaptation-min-coverage <float>`,
  - `--gold-adaptation-max-ambiguous <int>`.

Milestone exit criteria:

- `auto` only adapts when mismatch is detected.
- `off` preserves legacy strict behavior and legacy failure mode.
- `force` always adapts and still emits diagnostics.
- Previously failing `extra_gold` books produce eval artifacts in adaptive mode when coverage/ambiguity thresholds are satisfied.

## Idempotence and Recovery

This plan is designed to be rerun safely.

- Keep each benchmark validation run in its own timestamped root.
- If adaptive remap has low coverage, preserve artifacts and rerun after improving remap heuristics; do not edit gold.
- If adaptive remap crosses `--gold-adaptation-max-ambiguous`, preserve diagnostics, tighten matching policy, and rerun only affected books.
- If draft normalization breaks a consumer, preserve canonical `recipe` + `steps` and adjust only compatibility alias derivation.
- If a segmentation hardening tweak regresses another book, gate it with deterministic policy switches and record comparison evidence.
- If any milestone fails mid-way, commit and record current state in `Progress` with explicit completed/remaining split before proceeding.
