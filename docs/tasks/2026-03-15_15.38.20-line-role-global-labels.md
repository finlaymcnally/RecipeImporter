Problem statement

Canonical line-role Codex batches currently enforce per-row `candidate_labels`, which can block the model from choosing the right label when heuristic shortlist construction is wrong. The line-role Codex path should instead expose the full global label vocabulary for every row.

Acceptance criteria

- Canonical line-role Codex prompts no longer include per-row `candidate_labels`.
- Canonical line-role response parsing accepts any valid global label and does not reject rows for being outside a per-row allowlist.
- Codex-produced line-role prediction metadata and planning artifacts no longer present heuristic shortlists as LLM constraints.
- Tests cover the new prompt contract and confirm that a Codex response using a label outside the old shortlist is accepted.

Evidence

Constraints/gotchas + brief decision

- Keep the atomizer’s heuristic `candidate_labels` field as internal signal for deterministic logic where needed, but stop using it as an LLM constraint.
- Update docs because this is an externally observable prompt/benchmark behavior change.

Rollback / compensation plan

- Revert the task’s code and doc changes together if downstream benchmark tooling depends on per-row shortlist semantics.
