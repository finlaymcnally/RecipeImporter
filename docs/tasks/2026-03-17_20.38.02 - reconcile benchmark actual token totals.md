# Reconcile Benchmark Actual Token Totals

## Problem statement

Benchmark history/dashboard token fields are understating whole-run live token drain for some Codex runs. The current manifest readers sum `llm_codex_farm.process_runs` but can miss sibling `knowledge` telemetry, and line-role telemetry can be present only in nested batch summaries instead of a filled top-level token summary.

## Acceptance criteria

- Benchmark token extraction includes recipe plus knowledge telemetry from manifests.
- Line-role token extraction can recover totals from nested telemetry summaries when the top-level summary does not expose token fields.
- `prompt_budget_summary.json` includes the recovered line-role stage totals for these runs.
- Narrow tests cover CLI/import-eval token capture, analytics backfill/dashboard enrichment, and actual-cost summary generation.

## Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_import_eval.py -q`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -q`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -q`

## Evidence

- Before fix: `2026-03-16_18.10.25` dashboard row showed the recipe-only slice (`460,774` raw; `201k` discounted display) while the run artifact `prompt_budget_summary.json` already showed recipe + knowledge `841,214` raw and nested line-role telemetry showed an additional `25,500`.
- After fix: manifest readers and actual-cost summary should recover the whole-run totals from those same artifacts.

## Constraints / gotchas + decision

- Keep scope narrow: fix token extraction and actual-cost summarization only, not broader dashboard UI redesign.
- Prefer repo-owned actual-cost/telemetry artifacts over introducing new external dependencies or new benchmark outputs.

## Rollback / compensation plan

- Revert the helper changes and tests together if they destabilize benchmark history or prompt-budget generation.
