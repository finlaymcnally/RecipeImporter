# Task: Interactive prelabel mode selector

## Problem statement

Interactive Label Studio freeform import only asked a yes/no prelabel question. That hid `prelabel_allow_partial` and made partial-failure uploads inaccessible from interactive mode.

## Acceptance criteria

- Interactive freeform import shows a select menu for AI prelabel mode instead of yes/no confirm.
- The menu includes at least:
  - no prelabel,
  - strict prelabel annotations,
  - allow-partial prelabel annotations.
- Selected mode is passed into `run_labelstudio_import(...)` via `prelabel`, `prelabel_upload_as`, and `prelabel_allow_partial`.
- Docs reflect the new interactive prompt behavior.

## Verification steps

- Run:
  - `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_labelstudio_freeform_scope_routes_to_freeform_import`
  - `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_labelstudio_import_forces_overwrite_without_prompt`

## Evidence

- Updated test asserts capture `prelabel=True`, `prelabel_upload_as='annotations'`, and `prelabel_allow_partial=True` for interactive freeform flow.
- Interactive docs now describe the mode picker and partial option.

## Constraints and decisions

- Kept existing `_menu_select` so numeric shortcut auto-select behavior remains consistent with other interactive menus.
- Exposed advanced predictions modes in the same picker to mirror existing CLI upload modes.

## Rollback plan

- Revert `cookimport/cli.py` freeform interactive prompt block and related docs/tests if the menu introduces UX regressions.
