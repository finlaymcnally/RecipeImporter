# Problem statement

Interactive and CLI freeform prelabeling only supported legacy one-label-per-block annotations. We need true span-level prelabels while keeping the old behavior available.

# Acceptance criteria

- `labelstudio-import` supports `--prelabel-granularity block|span` (default `block`).
- Interactive freeform import asks for style after AI prelabel mode and includes `legacy, block based` wording for old behavior.
- Span mode can emit multiple sub-block spans from one block via quote-anchored selections.
- Existing block mode behavior remains unchanged.
- Docs are updated for operator behavior and conventions.

# Constraints and decisions

- Keep default as `block` for backward compatibility.
- Use quote-anchored matching for deterministic offsets; require `occurrence` for repeated quotes.
- Preserve strict offset/text integrity: derive `value.text` from `segment_text[start:end]`.

# Rollback plan

- Run with `--prelabel-granularity block` (or interactive `legacy, block based`) to disable span mode without reverting code.
