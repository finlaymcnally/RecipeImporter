# Problem
Interactive benchmark markdown behavior was confusing: default markdown-off plus scattered per-variant markdown outputs. The requested UX is markdown-on by default with one top-level markdown summary file per single-offline session.

# Acceptance Criteria
- Interactive benchmark defaults markdown to on.
- Interactive single-offline runs emit one markdown file at session root: `single_offline_summary.md`.
- Per-variant `eval_report.md` and comparison `.md` are not emitted in single-offline interactive flow.
- Comparison JSON remains available when both variants succeed.
- Docs and tests reflect the new contract.

# Verification Steps
- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -q`

# Evidence
- Updated benchmark-helper tests cover:
  - single top-level markdown summary behavior,
  - interactive default markdown-on forwarding,
  - no per-run markdown artifacts in single-offline interactive flow.

# Constraints / Gotchas
- Keep JSON artifacts as stable machine-readable truth.
- Keep non-interactive `labelstudio-benchmark` behavior unchanged.

# Decision
Single-offline interactive now runs per-variant benchmark calls with markdown disabled and writes one consolidated markdown summary from generated JSON artifacts.

# Rollback / Compensation
- Revert changes in `cookimport/cli.py`, `cookimport/c3imp_entrypoint.py`, and matching tests/docs to restore previous per-variant markdown behavior.
