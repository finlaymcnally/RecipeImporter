# Problem statement

Interactive Label Studio export asks for `export_scope` before project name selection. This is backwards for normal user intent and causes unnecessary cognitive load.

# Acceptance criteria

- Interactive `labelstudio_export` asks project selection before `Export scope`.
- Existing scope values (`pipeline`, `canonical-blocks`, `freeform-spans`) still route to `run_labelstudio_export(...)` unchanged.
- Docs reflect the new order.

# Verification steps

- Fail-before: `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_labelstudio_export_selects_project_before_scope`
- Pass-after: `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k "interactive_labelstudio_export_selects_project_before_scope or interactive_labelstudio_export_routes_to_export_command or select_export_project_name"`

# Evidence

- Fail-before: new order test fails because `Export scope` appeared before project selection.
- Pass-after: targeted export-order and export-routing tests pass.

# Constraints/gotchas

- Keep credential resolution order unchanged.
- Keep manual-entry fallback behavior for project selection unchanged.

# Decision

Reordered only the interactive export prompts in `cookimport/cli.py` and added a focused regression test for prompt order. This is the smallest behavioral change that matches expected UX.

# Rollback / compensation plan

Revert prompt order in `cookimport/cli.py` and remove the new order regression test.
