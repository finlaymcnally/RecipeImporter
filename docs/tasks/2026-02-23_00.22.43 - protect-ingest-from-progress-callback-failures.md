# Task: Protect Ingest From Progress Callback Failures

## Problem
Label Studio import can pass a UI progress callback into ingestion/conversion code. If that callback raises (for example from spinner rendering), conversion can fail even though extraction logic is otherwise valid.

## Acceptance Criteria
- Label Studio ingest progress callback calls are wrapped so callback exceptions are logged and ignored.
- Single-job conversion in `generate_pred_run_artifacts(...)` uses the safe notifier wrapper when calling importer `convert(...)`.
- `run_labelstudio_import(...)` passes the same safe notifier into artifact generation.
- Regression test proves artifact generation still succeeds when the provided progress callback always raises.

- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_ingest_parallel.py::test_generate_pred_run_artifacts_reports_prelabel_task_progress tests/labelstudio/test_labelstudio_ingest_parallel.py::test_generate_pred_run_artifacts_ignores_progress_callback_errors`
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_ingest_parallel.py`
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers.py::test_run_with_progress_status_renders_worker_activity_summary`

## Evidence
- Added `_notify_progress_callback(...)` helper in `cookimport/labelstudio/ingest.py` and routed notify paths through it.
- Added `test_generate_pred_run_artifacts_ignores_progress_callback_errors` in `tests/labelstudio/test_labelstudio_ingest_parallel.py`.

## Constraints / Decisions
- Kept callback failure handling non-fatal and low-noise (`logger.warning(...)`), preserving existing phase/status messages when callbacks are healthy.
- Avoided broader refactors; change is localized to ingest callback plumbing.

## Rollback
- Revert changes in `cookimport/labelstudio/ingest.py`, `tests/labelstudio/test_labelstudio_ingest_parallel.py`, and docs updates listed in this task.
