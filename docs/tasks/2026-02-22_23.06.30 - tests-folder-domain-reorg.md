# Problem Statement

`tests/` had too many files at one level, making focused runs and navigation noisy.
Need domain folders while preserving modular runs and low-token output behavior.

# Acceptance Criteria

- `tests/test_*.py` files are moved into category subfolders.
- Pytest discovery still works with existing marker strategy.
- Path-sensitive tests still resolve shared fixtures/docs paths correctly.
- Tests docs/instructions reflect the new folder structure.

# Verification Steps

```bash
. .venv/bin/activate
pytest -m smoke
pytest tests/parsing -m "parsing and not slow" --collect-only
pytest tests/ingestion -m "ingestion and not slow" --collect-only
pytest tests/ingestion/test_paprika_importer.py
```

# Constraints / Gotchas

- Keep fixture/doc path resolution independent of test file depth.
- Keep output minimal: no verbose/decorative test output.

# Decision

Use domain subfolders plus a shared `tests/paths.py` helper to stabilize file paths.

# Rollback / Compensation

Move test files back to `tests/` root and revert path-helper imports if folder layout proves disruptive.
