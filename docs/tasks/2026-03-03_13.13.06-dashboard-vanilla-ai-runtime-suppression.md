# Problem

Paired `single-offline-benchmark` rows can show codex model/effort on the `vanilla` row when run-config metadata is backfilled from manifests.

# Acceptance Criteria

- In dashboard JS, vanilla-variant rows do not expose AI model/effort values in `AI Model` / `AI Effort`.
- Variant detection still classifies rows as `vanilla`/`codexfarm` for trend/per-label flows.
- Existing stats dashboard renderer tests pass.

# Verification Steps

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k "renders_previous_runs_table_and_links_timestamp_to_artifact"`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`

# Evidence

- Added renderer assertions for vanilla suppression guard and raw-AI helper usage in `test_js_renders_previous_runs_table_and_links_timestamp_to_artifact`.
- Local pytest runs passed after patch.

# Constraints / Gotchas

- Keep variant detection path/pipeline-first so official quick-filter behavior stays stable.
- Avoid breaking runtime diagnostics selection that scores records using AI runtime presence.

# Decision

Implemented suppression in dashboard JS AI helper functions (variant-aware), instead of rewriting historical CSV/runtime metadata. This keeps historical artifacts untouched while fixing display semantics immediately.

# Rollback / Compensation

Revert JS helper changes in `cookimport/analytics/dashboard_render.py` and the paired test assertions in `tests/analytics/test_stats_dashboard.py`.
