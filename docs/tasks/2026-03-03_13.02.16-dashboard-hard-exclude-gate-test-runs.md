# Dashboard Hard Exclude Gate/Test Runs

## Problem statement

Dashboard diagnostics can still select test/gated benchmark runs as the latest record because test/smoke filtering currently lives in UI quick filters.

## Acceptance criteria

- Collector excludes benchmark rows whose artifact paths indicate test/gate runs (`/bench/`, pytest temp layouts, and timestamp-suffix token names such as `..._...-gated-...`).
- Excluded rows are removed before both diagnostics and Previous Runs data are built.
- Analytics docs reflect the collector-side exclusion contract.
- Analytics tests cover CSV and eval-report scan paths for gated-row exclusion.

## Verification steps

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k "gated or pytest_temp_eval_artifacts"`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py`
- `source .venv/bin/activate && cookimport stats-dashboard`

## Evidence

- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py -k "gated or pytest_temp_eval_artifacts"` -> `3 passed, 58 deselected in 0.20s`
- `source .venv/bin/activate && pytest tests/analytics/test_stats_dashboard.py` -> `61 passed in 3.65s`
- `source .venv/bin/activate && cookimport stats-dashboard` -> wrote dashboard to `data/.history/dashboard/index.html`

## Constraints/gotchas + decision

- Keep filtering deterministic/path-based only (no LLM parsing).
- Implemented in collector (not JS quick filters) so all dashboard surfaces share the same exclusion behavior.

## Rollback / compensation

- Revert collector helper/callsite changes in `cookimport/analytics/dashboard_collect.py` and restore previous tests/docs assertions.
