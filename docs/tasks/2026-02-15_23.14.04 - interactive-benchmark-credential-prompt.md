# Problem statement

Interactive benchmark upload called `labelstudio_benchmark(...)` without interactive credential resolution.  
When `LABEL_STUDIO_URL`/`LABEL_STUDIO_API_KEY` were unset, benchmark upload exited instead of prompting in interactive mode.

# Acceptance criteria

- Interactive benchmark upload resolves Label Studio creds through the interactive settings path (env -> saved config -> prompt).
- Resolved URL/API key are passed to `labelstudio_benchmark(...)`.
- Existing interactive benchmark/eval tests continue to pass.

# Verification steps

- `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_uses_golden_output_roots tests/test_labelstudio_benchmark_helpers.py::test_interactive_benchmark_eval_only_uses_existing_prediction_run`
- `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py`

# Evidence

- `26 passed` in `tests/test_labelstudio_benchmark_helpers.py`.
- Updated benchmark interactive test now asserts `label_studio_url` and `label_studio_api_key` are forwarded into `labelstudio_benchmark(...)` (this assertion would fail on the pre-change behavior).

# Constraints/gotchas

- Keep non-interactive `labelstudio-benchmark` behavior unchanged (still accepts CLI args/env and fails fast when missing).

# Decision

Reuse `_resolve_interactive_labelstudio_settings(settings)` inside interactive benchmark upload branch, matching interactive import/export behavior.

# Rollback / compensation plan

- Revert interactive benchmark upload to call `labelstudio_benchmark(...)` without resolved credentials.
