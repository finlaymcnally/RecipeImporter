# Task: Interactive Label Studio import spinner/progress callback wiring

## Problem statement

Interactive Label Studio import called `run_labelstudio_import(...)` without a progress callback, so long AI prelabeling runs had no visible working indicator.

## Acceptance criteria

- Interactive Label Studio import shows a live spinner with phase/status updates while the import/prelabel/upload flow runs.
- Spinner wiring reuses the same status/progress callback pattern used by non-interactive `labelstudio-import`.
- Tests confirm interactive import passes a callable `progress_callback` into `run_labelstudio_import(...)`.

## Verification steps

- `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_labelstudio_import`
- `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_labelstudio_freeform_scope_routes_to_freeform_import`

## Evidence

- Interactive Label Studio tests now assert `progress_callback` is callable.
- Shared helper wraps Label Studio import status text and callback for both interactive and non-interactive paths.

## Constraints and decisions

- Kept the existing status wording/style (`Label Studio import (...)`) so output remains consistent with other spinner flows.
- Avoided introducing a second indicator mechanism; reused existing Rich status + `progress_callback`.

## Rollback plan

- Revert the shared status helper usage in `cookimport/cli.py` and remove the new `progress_callback` test assertions.
