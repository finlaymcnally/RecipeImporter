summary: "Treat zombie Oracle controller PIDs as exited so background follow-up finalizes failed turn-1 uploads."
read_when:
  - "When Oracle upload lanes stay stuck on `running` or `waiting_for_turn_1` after browser automation already failed"
  - "When debugging bench Oracle follow-up status refresh against saved controller PIDs"
---

## Problem

The Oracle follow-up worker used `kill(pid, 0)` to decide whether a saved controller PID was still alive. On Linux that returns success for zombie processes, so failed Oracle browser runs could stay stuck as `running` and never finalize.

## Fix

- Treat `/proc/<pid>/status` state `Z` as not running before falling back to `kill(pid, 0)`.
- This lets the background refresh keep Oracle's logged/session-store failure instead of re-promoting the lane back to `running`.
