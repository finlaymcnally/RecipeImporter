## Problem statement

`tests/llm/test_codex_farm_orchestrator.py` bundled multiple concerns in one large file, making focused debugging and targeted runs slower than necessary.

## Acceptance criteria

- Codex orchestrator tests are split into policy, runner transport, and stage seam modules.
- Marker mapping includes all new filenames.
- Slow slice captures policy + runner transport modules; stage seam module stays non-slow.
- Collection works without missing/duplicate tests.

## Verification steps

- `. .venv/bin/activate && pytest --collect-only -q tests/llm/test_codex_farm_orchestrator.py tests/llm/test_codex_farm_orchestrator_runner_transport.py tests/llm/test_codex_farm_orchestrator_stage_integration.py`
- `. .venv/bin/activate && pytest tests/llm/test_codex_farm_orchestrator.py::test_orchestrator_gates_pass3_when_pass2_degraded_missing_instruction_evidence tests/llm/test_codex_farm_orchestrator_runner_transport.py::test_subprocess_runner_reports_missing_binary tests/llm/test_codex_farm_orchestrator_stage_integration.py::test_orchestrator_runs_three_passes_and_writes_manifest`
- `. .venv/bin/activate && pytest -m slow --collect-only -q tests/llm`

## Evidence

- Split collection counts: `16 + 15 + 3`.
- Representative targeted run: `3 passed, 1 warning in 2.15s`.
- LLM slow slice now includes policy + runner transport modules, excludes stage seam module.

## Constraints/gotchas

- Split modules import shared helpers/imports from the base module; avoid copying `__dunder__` globals to prevent pytest import-file mismatch errors.
- Keep all new `test_*.py` basenames mapped in `tests/conftest.py`.

## Decision

Modularize codex orchestrator tests by seam (policy/transport/stage) for faster targeted iteration while preserving existing behavior.

## Rollback / compensation

Merge split tests back into `test_codex_farm_orchestrator.py` and revert `tests/conftest.py` mapping/slow updates.
