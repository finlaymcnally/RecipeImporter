# Problem

Current upload bundles already include most requested review artifacts, but two practical gaps remain for first-pass triage: per-call cost is often absent in telemetry, and candidate-label fields vary by producer format. Pair-count readiness is present but not explicit enough for quick generalization triage.

# Acceptance Criteria

- Upload bundle call-inventory summary exposes additive estimated-cost fields when observed `cost_usd` is missing.
- Estimated-cost calculations are clearly marked as estimates (not observed billing values).
- Line-role candidate-label signal accepts additional candidate field shapes beyond `candidate_labels`.
- Index includes explicit generalization-readiness detail (`minimum_pairs_for_generalization`, `additional_pairs_needed_for_generalization`).
- Existing 3-file contract is unchanged.

# Constraints And Decision

- Keep the change confined to upload-bundle generation; do not alter ingestion/runtime LLM behavior.
- Keep default outputs deterministic and backward-compatible.
- Use additive fields so existing consumers remain valid.
