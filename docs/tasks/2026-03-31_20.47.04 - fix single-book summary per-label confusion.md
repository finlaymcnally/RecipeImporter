## Problem

`single_book_summary.md` shows a single per-label table aggregated across both `vanilla` and `codexfarm`.
For paired runs this hides variant-local failures, including cases where one variant has `0` recall for a label.

## Acceptance Criteria

- `single_book_summary.md` reports per-label metrics separately for each variant.
- The summary no longer implies that the merged table is variant-local.
- Existing comparison artifacts can still keep their aggregate paired per-label metadata if needed.

## Verification Steps

- Run `pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_book_artifacts.py -q`

## Evidence

- Before: paired summary rendered one merged per-label table with `eval_count=2`.
- After: paired summary renders one per-label section per variant, sourced from that variant's own `eval_report.json`.
- Verified with `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_book_artifacts.py -q`.

## Constraints / Gotchas

- Keep the fix narrow to `single_book_summary.md`; do not redesign comparison JSON semantics in the same change.
- Preserve existing comparison-artifact tests unless they specifically cover the summary markdown surface.

## Decision

- Add variant-local per-label extraction for the single-book summary writer and leave comparison artifact aggregation unchanged.
