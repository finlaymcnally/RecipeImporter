# Task: legacy EPUB extractor migration for interactive benchmark

## Problem statement
Interactive benchmark mode crashed while loading global run settings because persisted `epub_extractor=legacy` is no longer a valid enum value.

## Acceptance criteria
- `RunSettings.from_dict(...)` loads payloads containing `epub_extractor=legacy`.
- Legacy value is migrated to `beautifulsoup`.
- A regression test covers this migration.

## Verification steps
- `source .venv/bin/activate`
- `python - <<'PY' ... RunSettings.from_dict({'epub_extractor': 'legacy'}) ... PY`
- `pytest tests/llm/test_run_settings.py`

## Evidence
- Repro before fix: `ValidationError` on `epub_extractor='legacy'`.
- Repro after fix: resolves to `beautifulsoup` with migration warning.
- Test result: `9 passed` in `tests/llm/test_run_settings.py`.

## Constraints / decision
Minimal compatibility fix: only persisted settings migration was changed; accepted user-facing extractor choices remain canonical (`unstructured|beautifulsoup|markdown|markitdown`).
