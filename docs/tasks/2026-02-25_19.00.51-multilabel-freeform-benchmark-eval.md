# Task: allow multi-label freeform gold in stage-block eval

## Problem statement
Offline benchmark eval crashes when a freeform gold block has multiple labels, even though freeform workflows intentionally allow multiple spans per block.

## Acceptance criteria
- `load_gold_block_labels(...)` accepts multi-label gold blocks.
- Stage-block evaluation treats a prediction as correct when it matches any allowed gold label for that block.
- Multi-label gold blocks are written to `gold_conflicts.jsonl` as diagnostics.
- Missing gold rows for predicted blocks default to `OTHER` and are written to `gold_conflicts.jsonl`.
- Regression tests cover loader + metric + end-to-end eval behavior.

## Verification steps
- `source .venv/bin/activate`
- `pip install -e .[dev]`
- `pytest tests/bench/test_eval_stage_blocks.py`

## Evidence
- Before: `ValueError: Gold conflicts detected...` during `labelstudio-benchmark --no-upload` with multi-label gold blocks.
- After: evaluation runs; multi-label and missing-gold defaults are logged in `gold_conflicts.jsonl`; metrics treat any allowed gold label as a valid match.

## Constraints / decision
Kept existing report schema stable (`gold_label`, per-label metrics, legacy aliases) and added `gold_labels` in mismatch rows for disambiguation.
