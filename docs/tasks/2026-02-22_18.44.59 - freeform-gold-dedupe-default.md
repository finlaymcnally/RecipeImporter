# Task: freeform gold dedupe default

## Problem statement

Freeform gold exports from overlapped segments can contain duplicate spans for the same block range, which can bias benchmark/eval denominators. We need dedupe on by default and a deterministic policy for conflicting labels.

## Acceptance criteria

- Freeform eval dedupes gold spans by default before scoring.
- Dedupe key is range-based (`source_hash`, `source_file`, `start_block_index`, `end_block_index`) so overlap-generated duplicates collapse.
- Conflicting duplicate labels resolve by majority vote; exact ties are dropped from scored gold.
- Eval reports expose dedupe/conflict diagnostics so dropped or resolved conflicts are visible.
- Regression coverage exists for:
  - same-label duplicates,
  - conflicting duplicates with majority winner,
  - conflicting duplicates with tie/drop behavior.

## Constraints / gotchas and decision

- Constraint: overlapping freeform segments are intentional for annotation quality; eval should not require users to hand-clean exports.
- Constraint: picking an arbitrary label on tie would hide annotation ambiguity.
- Decision: dedupe by block range key, apply majority vote for conflicts, and drop exact ties while surfacing conflict details in report JSON/Markdown.

## Rollback / compensation plan

- Revert `cookimport/labelstudio/eval_freeform.py` dedupe/conflict logic and related tests/docs if this behavior needs to be replaced with a different conflict policy (for example strict-fail or manual-resolution gate).
