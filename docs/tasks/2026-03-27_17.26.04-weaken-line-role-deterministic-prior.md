# Weaken Line-Role Deterministic Prior

## Problem

The line-role prompts and hint policy were telling workers to treat deterministic labels as a strong prior and to leave seeded labels in place whenever they looked merely plausible.
Even after the first downgrade, the prompts still treated the deterministic seed as a fallback tie-breaker under ambiguity.
That biases the correction pass toward preserving deterministic mistakes instead of actually reviewing the text.

## Acceptance Criteria

- Live and legacy line-role prompts describe deterministic labels as weak hints, not starting truth or tie-breakers.
- Worker guidance and flip-policy text tell the model to recompute from shard-local evidence without falling back to the deterministic seed under ambiguity.
- Focused prompt tests reflect the weaker prior language.

## Verification

- `. .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles.py -q`

## Constraints / Decision

Keep deterministic labels in the payload for now, but only as lightweight hints.
If correction quality still looks capped after this change, the next step is to evaluate removing the seeded labels from the model-facing line-role input entirely.
