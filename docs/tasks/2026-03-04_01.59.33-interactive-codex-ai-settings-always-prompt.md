# Problem

Interactive import/benchmark could enable codex pipeline without asking for codex model and reasoning effort in single-profile flows. Operator requirement is codex-on and AI settings selection must stay coupled in the shared interactive chooser.

# Acceptance Criteria

- `choose_run_settings(...)` asks codex model + reasoning effort prompts whenever the resolved profile is codex and prompting is enabled.
- Interactive Import and interactive Benchmark both pass through this behavior.
- Cancel/back from codex AI settings prompts cancels run setup cleanly.
- Docs reflect the updated chooser contract.

# Verification Steps

1. `source .venv/bin/activate && pytest tests/cli/test_c3imp_interactive_menu.py -k "choose_run_settings" -q`
2. `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py -k "all_matched_mode_routes_to_runner or selected_matched_mode_routes_to_runner" -q`

# Evidence

- `7 passed, 14 deselected, 1 warning in 1.82s` (`tests/cli/test_c3imp_interactive_menu.py -k "choose_run_settings"`)
- `2 passed, 6 deselected, 1 warning in 2.05s` (`tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py -k "all_matched_mode_routes_to_runner or selected_matched_mode_routes_to_runner"`)

# Constraints / Decisions

- Keep codex AI settings prompting behind an explicit `prompt_codex_ai_settings` argument in `choose_run_settings(...)` so non-interactive or legacy callsites can remain unchanged until intentionally migrated.
- Reuse the existing reasoning-effort option set used in benchmark all-method prompts (`pipeline default`, `none|minimal|low|medium|high|xhigh`).

# Rollback

- Revert chooser additions in `cookimport/cli_ui/run_settings_flow.py` and remove new interactive callsite args in `cookimport/cli.py`.
