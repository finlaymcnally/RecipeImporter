## Problem statement

On this host, `joblib` emits `Permission denied ... will operate in serial mode` at startup because SemLock is unavailable. The warning is noisy and repeated across CLI workflows, even though runtime fallback behavior is already expected in this environment.

## Acceptance criteria

- On SemLock-restricted hosts, startup auto-sets `JOBLIB_MULTIPROCESSING=0` before downstream imports trigger `joblib` warning spam.
- Explicit `JOBLIB_MULTIPROCESSING` env values are respected (no override).
- Guard can be explicitly disabled via project env for debugging.
- Focused tests cover restricted-host, explicit-env, and guard-disabled paths.

## Constraints/gotchas and decision

- The guard only forces `JOBLIB_MULTIPROCESSING=0` when SemLock restriction is detected and env is otherwise unset.
- Decision: keep an explicit opt-out (`COOKIMPORT_DISABLE_JOBLIB_SEMLOCK_GUARD`) for debugging or host-specific experimentation.
