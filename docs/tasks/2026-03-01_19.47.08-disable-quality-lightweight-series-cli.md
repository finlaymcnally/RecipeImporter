# Task: Disable bench quality-lightweight-series in CLI

## Problem statement

`bench quality-lightweight-series` remains callable even though tournament-style fold workflows are now considered too slow and too storage-heavy for this project.

## Acceptance criteria

- `cookimport bench quality-lightweight-series` fails fast with a clear disabled/retired message.
- The command no longer starts lightweight-series orchestration or writes run artifacts.
- Bench docs indicate the command is disabled and treat existing lightweight-series artifacts as historical.
- CLI tests cover the disabled behavior.

## Verification steps

Run from repo root with venv active:

```bash
. .venv/bin/activate
pytest tests/bench/test_bench.py -k "quality_lightweight_series" -q
```

## Evidence

- Updated `cookimport/cli.py` to fail immediately for `bench_quality_lightweight_series`.
- Updated `tests/bench/test_bench.py` lightweight-series tests to assert disabled behavior.
- Updated `docs/07-bench/07-bench_README.md` command surface and artifact wording.

## Constraints / gotchas and decision

- Constraint: preserve command discoverability while preventing accidental expensive runs.
- Decision: keep the command registered but make it exit immediately via `_fail(...)` with explicit alternatives (`quality-run` + `quality-compare`).

## Rollback / compensation plan

- Re-enable by restoring previous `bench_quality_lightweight_series` implementation and tests.
