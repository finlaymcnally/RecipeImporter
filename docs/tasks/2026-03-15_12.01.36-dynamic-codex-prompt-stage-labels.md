---
summary: "Make Codex prompt artifact labels derive from observed stage metadata instead of fixed pass2/pass3 names."
read_when:
  - "When changing benchmark/stage prompt artifact naming or prompt sample headings"
  - "When a legacy pass slot is reused by a different recipe pipeline stage"
---

Problem statement

Prompt artifact export in `cookimport/cli.py` was hardcoding reviewer-facing names from legacy pass slots (`pass1`..`pass5`). When a run reused the pass2 slot for `recipe.merged-repair.compact.v1`, prompt sample markdown and category files still said `pass2 schemaorg`, which misrepresented the actual prompt stage.

Acceptance criteria

- `full_prompt_log.jsonl` keeps compatibility `pass` values but also records dynamic stage metadata (`legacy_pass`, `stage_key`, `stage_label`, `stage_artifact_stem`, `stage_matches_legacy`, `stage_slot_index`).
- Prompt category files derive their filename from the observed stage when the stage no longer matches the legacy slot.
- `prompt_type_samples_from_full_prompt_log.md` groups rows by the observed stage label when a legacy slot is repurposed, without regressing the normal pass1/pass2/pass3 naming path.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -k dynamic_stage_labels_for_merged_repair -q`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -q`

Evidence

- Before the change, the merged-repair regression test failed because `prompt_task2_merged_repair.txt` was not written.
- After the change, the new regression test and the full prompt-artifact test file both pass.

Constraints/gotchas + decision

- Keep `row["pass"]` for compatibility because benchmark and follow-up tooling still consume that field.
- Limit the change to reviewer-facing artifact naming; raw Codex storage folders remain legacy-slot names for now.

Rollback / compensation plan

- Revert the prompt-stage metadata helpers and fall back to legacy pass naming if downstream tooling unexpectedly requires the old filenames everywhere.
