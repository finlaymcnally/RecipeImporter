Problem statement

Replace the live leased-packet and phase-ledger workspace worker contract with one fixed-assignment contract for recipe, knowledge, and line-role. The happy path should stop depending on `current_packet.json`, `current_result_path.txt`, `packet_lease_status.json`, or `CURRENT_PHASE*`.

Acceptance criteria

- Live worker roots use immutable assignment files plus stable `in/*.json` and `out/*.json` paths on the happy path.
- Recipe, knowledge, and line-role workspace prompts/manifests teach only the fixed-assignment contract.
- Focused runtime tests fail if happy-path current-packet or current-phase control files are still required.

Verification steps

- `source .venv/bin/activate && pytest tests/llm/test_codex_exec_runner_workspace.py`
- `source .venv/bin/activate && pytest tests/llm/test_recipe_phase_workers.py`
- `source .venv/bin/activate && pytest tests/llm/test_knowledge_orchestrator_runtime_leasing.py`
- `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles.py -k "workspace or compact_prompt"`

Evidence

- Before the change, the workspace contract and stage runtimes still referenced lease/phase control files.
- After the change, worker manifests and prompts point at assignment files and stable output paths instead.

Constraints/gotchas + brief decision

- The repo already has unrelated in-flight changes, so keep edits localized to the live worker/runtime seams.
- Keep deterministic validation authoritative; let the model write outputs, then let repo code validate and optionally trigger bounded repair.
