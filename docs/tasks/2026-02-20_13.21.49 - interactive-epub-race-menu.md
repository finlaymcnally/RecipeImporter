# Task: Interactive EPUB Race Menu

## Problem Statement

`cookimport epub race ...` existed only as a direct command. Users in interactive mode had no in-menu path to run the one-file extractor race.

## Acceptance Criteria

- Interactive main menu exposes an EPUB race action when at least one top-level `.epub` is available in `data/input`.
- The interactive race flow prompts for EPUB file, output folder, and candidate extractors.
- The flow reuses existing `cookimport epub race` behavior and returns to the main menu.
- README and CLI docs describe the new interactive path.

## Verification Steps

Run from repo root:

    source .venv/bin/activate
    pytest -q tests/test_labelstudio_benchmark_helpers.py tests/test_c3imp_interactive_menu.py

## Evidence

- `37 passed` on:
  - `tests/test_labelstudio_benchmark_helpers.py`
  - `tests/test_c3imp_interactive_menu.py`

## Constraints / Gotchas and Decision

- Kept extractor race logic single-sourced by calling `race_epub_extractors(...)` from interactive mode.
- Kept menu visibility scoped to top-level EPUB availability to match existing interactive file discovery behavior.

## Rollback / Compensation Plan

- Remove `epub_race` branch from `_interactive_mode` and `_interactive_epub_race(...)` helper.
- Remove added interactive tests and docs entries for race menu flow.
