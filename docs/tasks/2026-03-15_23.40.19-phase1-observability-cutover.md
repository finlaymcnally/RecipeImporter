Problem statement

Phase 1 of the refactor still has multiple stage taxonomies in runtime artifacts. New runs can expose semantic labels, pass-slot directory names, compatibility fields, and fallback readers at the same time.

Acceptance criteria

- New stage and prediction runs write `stage_observability.json` as the canonical stage index.
- New outputs stop exposing legacy pass-slot naming such as `pass2_schemaorg`, `pass3_stage`, `pass4_knowledge`, and `pass5_tags` in run-level observability and raw LLM stage storage.
- Prompt artifact export and benchmark upload-bundle rendering read shared stage metadata instead of rebuilding pass-slot mappings locally.

Verification steps

- `source .venv/bin/activate && pytest tests/staging/test_stage_observability.py tests/staging/test_run_manifest_parity.py tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py tests/bench/test_upload_bundle_v1_existing_output.py tests/bench/test_benchmark_cutdown_for_external_ai.py -q`
- `source .venv/bin/activate && cookimport stage data/input/saltfatacidheatCUTDOWN.epub --out /tmp/phase1-observability-det`
- Inspect the newest run root and confirm `stage_observability.json` exists and no pass-slot stage directories were written.

Evidence

- Targeted tests should fail before the cutover because the new artifact does not exist and legacy names are still emitted.
- The same slices should pass after the cutover with semantic-only stage names.

Constraints and gotchas

- The workspace already has unrelated in-flight edits in benchmark and Label Studio files. Preserve them and patch around them.
- This is a destructive migration. Do not add compatibility readers or legacy alias fields for old roots.

Decision

- Centralize stage naming and stage-owned paths in `cookimport/runs/stage_observability.py`, then route manifest/summary/prompt/bundle code through that shared model.

Rollback / compensation plan

- If the cutover breaks a consumer, fix that consumer against the new semantic contract instead of restoring pass-slot compatibility code.
