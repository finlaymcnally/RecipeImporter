## Problem
Single-profile benchmark live status was hard to read in terminal mode: spinner was visually small, panel width was capped too tightly, and long task/detail rows were clipped.

## Acceptance Criteria
- Live status uses a larger spinner for benchmark panel mode.
- Boxed panel can use a wider terminal footprint before clamping.
- Long panel lines wrap across rows instead of being truncated to a single ellipsis line.
- Single-profile dashboard task rows are no longer hard-capped at 180 characters.
- Regression tests cover line wrapping/tail visibility and spinner selection.

## Verification

## Evidence
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_progress.py -k "clamps_live_box_width_to_terminal or wraps_long_lines_and_uses_larger_spinner or preserves_long_task_message or preserves_eta_when_live_line_is_truncated or humanizes_codex_stage_in_live_panel"` -> `5 passed, 22 deselected, 1 warning in 2.65s`

## Constraints / Decision
- Keep this strictly in CLI live-status formatting/rendering; do not alter benchmark execution or scheduler behavior.
- Preserve existing terminal-width clamp guarantees so narrow terminals still render safely.

## Rollback
- Revert live panel wrapping/spinner/task-clamp edits in `cookimport/cli.py` and remove the added progress-helper regression tests.
