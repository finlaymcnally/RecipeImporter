Problem statement

Interactive CodexFarm process selection is still split across different benchmark paths.
Import and the main freeform benchmark flow use the shared per-process CodexFarm submenu, but all-method benchmark still uses a separate yes/no Codex prompt plus separate override prompts.

Acceptance criteria

- Interactive CodexFarm process selection uses the shared per-process submenu everywhere the interactive CLI exposes benchmark Codex choices.
- Interactive all-method benchmark reuses the shared CodexFarm process submenu and shared Codex model/reasoning chooser.
- All-method variant planning respects the exact Codex surfaces selected from that shared submenu.
- Targeted tests cover both the interactive all-method menu path and the all-method variant builder surface selection contract.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_scheduler_targets.py::test_build_all_method_variants_respects_selected_codex_surfaces tests/labelstudio/test_labelstudio_benchmark_helpers_scheduler_targets.py::test_interactive_all_method_benchmark_uses_shared_codex_surface_menu -q`

Evidence

- Before the change, all-method interactive benchmark still asked `Include Codex Farm permutations?` and then used its own Codex override prompts.
- After the change, all-method interactive benchmark reuses the shared per-surface CodexFarm chooser and plans only the selected Codex surfaces.

Constraints/gotchas + decision

- Keep this scoped to interactive CodexFarm selection surfaces; do not rewrite non-interactive CLI flags.
- Preserve existing all-method behavior when no explicit per-surface selection is provided by keeping the legacy full-codex path as the default internal fallback.
