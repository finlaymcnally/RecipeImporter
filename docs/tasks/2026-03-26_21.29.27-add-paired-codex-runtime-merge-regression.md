## Problem

Existing upload-bundle tests already covered pair rendering and some prompt-budget fallback behavior, but they did not exercise the hybrid case that most resembles a real codex run:
paired benchmark output with partial per-call telemetry on prompt rows plus stage-level totals in `prediction-run/prompt_budget_summary.json`.

## Acceptance Criteria

- A paired benchmark fixture with codex prompt rows that carry realistic `request_telemetry` still merges correctly with `prediction-run/prompt_budget_summary.json`.
- The merged runtime summary keeps fallback stage totals/tokens while preserving row-level slowest/cost views.
- The focused upload-bundle runtime tests pass in `.venv`.

## Verification

- `. .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py -q -k "prediction_manifest or prompt_budget_summary or realistic_codex_call_telemetry"`
- `. .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py -q -k upload_bundle`

## Evidence

- Added a paired synthetic regression with two codex prompt rows carrying `request_telemetry`, plus `prediction-run/manifest.json` and `prediction-run/prompt_budget_summary.json`.
- The assertions check merged runtime source, stage token totals, and preserved row-level slowest/estimated-cost views.

## Constraints / Decision

- Keep scope to tests and task tracking only; do not change runtime behavior unless the new regression exposes a real bug.
- Prefer one high-signal integration-style fixture over many tiny unit tests so the post-refactor artifact shapes stay exercised together.
