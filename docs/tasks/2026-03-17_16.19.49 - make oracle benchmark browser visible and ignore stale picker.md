Problem statement

Benchmark Oracle browser uploads should open a visible Linux browser window so the operator can watch the run, and they should stop failing on Oracle's stale ChatGPT model-picker automation.

Acceptance criteria

- Browser-mode benchmark uploads launch visible Linux Chromium instead of the xvfb wrapper.
- Browser-mode benchmark uploads reuse a persistent Oracle browser profile directory.
- Browser-mode benchmark uploads pass `--browser-model-strategy ignore` so they do not fail on stale picker labels.
- Focused Oracle upload tests still pass.

Verification steps

- Activate `.venv`.
- Run `pytest tests/bench/test_benchmark_oracle_upload.py`.

Evidence

- Browser command assembly now proves the uploader calls Oracle with `--browser-chrome-path /home/mcnal/.local/bin/chromium-nosandbox` and `--browser-model-strategy ignore`.
- Browser env assembly now keeps `ORACLE_HOME_DIR`, `ORACLE_BROWSER_PROFILE_DIR`, and `ORACLE_BROWSER_REMOTE_DEBUG_HOST` stable across runs.
- Verification result: `source .venv/bin/activate && pytest tests/bench/test_benchmark_oracle_upload.py` -> pending in this task until rerun.

Constraints/gotchas + decision

- The old off-screen path solved focus stealing but made it impossible for the operator to watch login/model state; the user explicitly wants the visible browser back.
- Oracle's current browser picker is mode-based, not model-id-based, so trying to auto-select `gpt-5.2` or `gpt-5.2-pro` is brittle here.
- Keep the repo-side fix narrow: visible browser plus picker-ignore plus stable profile env, rather than trying to patch Oracle upstream.

Rollback / compensation plan

- Revert the browser path and picker-ignore flag in `cookimport/bench/oracle_upload.py` if visible-browser operation proves worse than expected.
