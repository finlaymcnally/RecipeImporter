# Problem

Users want benchmark outputs to be immediately uploadable to web-based AI with only three files, without losing any benchmark data and without requiring extra flags.

# Acceptance Criteria

- Interactive benchmark menu runs emit `upload_bundle_v1/` by default.
- Direct CLI `labelstudio-benchmark` runs emit `upload_bundle_v1/` by default when eval roots are retained.
- Bundle stays lossless and navigation-first (overview + index + payload).
- Existing benchmark artifacts remain intact (bundle is additive, not destructive by default).

# Verification Steps

1. Run focused tests:
   - `./scripts/test-suite.sh domain bench tests/bench/test_benchmark_cutdown_for_external_ai.py`
   - `./scripts/test-suite.sh domain labelstudio tests/labelstudio/test_labelstudio_benchmark_helpers.py`
2. Confirm interactive single-offline test checks `single_offline_summary.md` and `upload_bundle_v1/` file set.
3. Confirm non-interactive `labelstudio_benchmark` test checks `<eval_root>/upload_bundle_v1/` file set.

# Evidence

- Added/updated helper wiring in `cookimport/cli.py` to call `build_upload_bundle_for_existing_output` across interactive and non-interactive benchmark flows.
- Added/updated tests in:
  - `tests/bench/test_benchmark_cutdown_for_external_ai.py`
  - `tests/labelstudio/test_labelstudio_benchmark_helpers.py`

# Constraints And Decision

- Keep benchmark run artifacts and summaries as-is; add a dedicated `upload_bundle_v1/` folder for upload workflows.
- Preserve no-data-loss guarantee by serializing every source artifact into `upload_bundle_payload.jsonl` with checksums and byte counts.

# Rollback Plan

- Remove CLI post-run upload-bundle helper calls and constants.
- Keep script-level bundle helpers for manual/explicit usage.
