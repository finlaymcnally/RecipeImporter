# Task: Recipe codex-farm integration (recipeimport side)

## Problem statement

Implement `docs/plans/000-Recipe-Codex-Farm.md` on the recipeimport side so stage and benchmark prediction generation can optionally run the 3-pass codex-farm recipe correction pipeline while preserving deterministic defaults.

## Acceptance criteria

- `RunSettings` defines codex-farm knobs (`llm_recipe_pipeline`, `codex_farm_cmd`, `codex_farm_root`, `codex_farm_context_blocks`, `codex_farm_failure_mode`) with defaults that keep behavior deterministic (`off`).
- `cookimport stage` and `cookimport labelstudio-benchmark` expose and thread the new flags through run config paths.
- New LLM orchestration modules exist (`contracts`, `ids`, `runner`, `fake_runner`, `orchestrator`) and orchestrator emits `raw/llm/<workbook_slug>/llm_manifest.json`.
- Stage single-file and split-merge paths apply orchestrator output plus writer overrides for intermediate/final drafts.
- Pred-run generation supports the same setting path and surfaces `llm_codex_farm` metadata.
- Tests pass without invoking live codex-farm.

## Constraints / gotchas and decision

- Constraint: user explicitly requested no token-spend testing via live codex-farm calls.
- Gotcha: Typer rich help truncates long flag names in default-width test output; exact-string assertions require wider columns.
- Decision: keep CLI-help test deterministic by invoking with `env={"COLUMNS": "240"}`.

## Rollback / compensation plan

- Runtime rollback is immediate: run with `--llm-recipe-pipeline off` (or omit the flag) to restore deterministic behavior.
- Code rollback is isolated to codex-farm modules and run-setting plumbing; disable by reverting those files if needed.
