# Task: Spinner task X/Y counters for AI labeling

## Problem statement

AI prelabel spinner updates showed only phase text and did not expose per-task throughput, making it hard to tell if progress was real.

## Acceptance criteria

- Freeform prelabel progress messages include `task X/Y` while iterating segment tasks.
- Freeform decorate progress messages include `task X/Y` while iterating project tasks.
- Tests verify those counters are emitted.
- Spinner counter policy is documented in the closest AGENTS file and project conventions.

## Verification steps

- `. .venv/bin/activate && pytest -q tests/test_labelstudio_ingest_parallel.py -k prelabel_task_progress`
- `. .venv/bin/activate && pytest -q tests/test_labelstudio_prelabel.py -k decorate_dry_run`
- `. .venv/bin/activate && pytest -q tests/test_labelstudio_benchmark_helpers.py -k interactive_labelstudio_import`

## Evidence

- New helper-based progress messages emit `... task 0/N`, `... task 1/N`, ..., `... task N/N`.
- Decorate dry-run test now captures progress callback messages and asserts `task 1/2` and `task 2/2`.
- New generate-pred test validates prelabel callback counters.

## Constraints and decisions

- Reused existing callback-driven spinner flow rather than introducing another indicator system.
- Counter format is normalized as `task X/Y` for consistency across import/decorate.

## Rollback plan

- Revert `cookimport/labelstudio/ingest.py` task-counter progress message changes and related tests/docs/AGENTS updates.
