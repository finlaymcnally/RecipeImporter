# Single-Offline Starter-Pack Fallback Loader

## Problem statement

Interactive single-offline runs could skip starter-pack generation when `scripts.benchmark_cutdown_for_external_ai` was not importable as a package module. The fallback script loader then failed with `AttributeError: 'NoneType' object has no attribute '__dict__'`.

## Acceptance criteria

- Single-offline fallback helper loading succeeds when the package import path is unavailable.
- Starter-pack output generation still writes `starter_pack_v1/` in that fallback scenario.
- Regression test proves dataclass-bearing fallback modules load successfully.

## Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "single_offline_starter_pack_fallback_loader_registers_module"`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "single_offline_comparison_artifacts_trigger_starter_pack"`

## Evidence

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "single_offline_starter_pack_fallback_loader_registers_module or single_offline_comparison_artifacts_trigger_starter_pack"` -> `2 passed, 167 deselected`.
- `source .venv/bin/activate && pytest tests/bench/test_benchmark_cutdown_for_external_ai.py -k "build_starter_pack_for_existing_runs"` -> `2 passed, 18 deselected`.

## Constraints/gotchas + decision

- Keep primary import path unchanged when `scripts.benchmark_cutdown_for_external_ai` is importable.
- Decision: keep fallback script loading, but register the module in `sys.modules` before `exec_module()` so dataclass/type evaluation can resolve the module namespace.

## Rollback / compensation

- Revert fallback loader changes in `cookimport/cli.py` and remove the fallback loader regression test.
