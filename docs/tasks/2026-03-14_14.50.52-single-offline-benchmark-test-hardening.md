Problem statement

The single-offline metadata crash was only guarded indirectly by broader benchmark helper tests. There was no fast, offline regression at both the variant-planner seam and the interactive CLI handoff seam.

Acceptance criteria

- `_interactive_single_offline_variants()` has a focused regression test for persistence-only metadata leakage.
- Interactive `labelstudio_benchmark -> single_offline` has a fast test that proves codex-enabled settings are handed to the helper without prompting for Label Studio credentials.
- New tests stay fully offline and do not invoke real CodexFarm work.

Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_single_offline_run.py -k 'codex_enabled_runs_only_codexfarm or preserves_selected_codex_recipe_pipeline or variants_ignore_persistence_only_metadata'`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_interactive.py -k 'single_offline_codex_pipeline_passes_settings_to_helper_without_credentials'`

Evidence

- The helper-level metadata regression now fails directly at variant planning instead of relying on a broader benchmark wrapper to notice it.
- The interactive CLI test now stops at `_interactive_single_offline_benchmark`, so it proves the handoff path while remaining token-free and fast.

Constraints/gotchas + decision

- Keep benchmark tests offline by stubbing `labelstudio_benchmark` or `_interactive_single_offline_benchmark` rather than letting interactive tests launch the full benchmark flow.
- Prefer narrow handoff tests for CLI selection logic and separate direct helper tests for benchmark variant planning.

Rollback / compensation plan

- Revert the added tests if they become coupled to unrelated menu wording, but preserve at least one direct metadata-boundary regression.
