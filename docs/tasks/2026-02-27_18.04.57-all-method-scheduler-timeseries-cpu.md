## Problem

All-method runs exposed scheduler status only in live spinner output and rollups.
There was no persisted run-local time-series combining scheduler snapshots with CPU utilization.

## Acceptance Criteria

- All-method writes `<source_root>/scheduler_timeseries.jsonl`.
- Each row includes scheduler snapshot/counters (`heavy/wing/eval/active/pending`) and timestamped elapsed time.
- Rows include host CPU utilization samples when available.
- Per-source report `scheduler` payload includes discoverability fields for the time-series artifact.

## Verification

1. `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k 'smart_scheduler_improves_heavy_slot_utilization or writes_scheduler_timeseries'`

## Evidence

- Added `test_run_all_method_benchmark_writes_scheduler_timeseries`.
- Targeted pytest run passed (`2 passed`).

## Constraints / Gotchas

- CPU utilization sampling must not add new dependencies.
- Keep compatibility with existing scheduler snapshot text contract.

## Decision

Implement lightweight Linux host CPU sampling from `/proc/stat` in scheduler loop, and persist time-series rows on snapshot change plus a 1.0s heartbeat.
