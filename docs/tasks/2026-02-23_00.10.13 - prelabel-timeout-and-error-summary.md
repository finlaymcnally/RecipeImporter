# Task: Prelabel Timeout + Error Summary

## Problem
Freeform AI prelabel runs were timing out at the default per-task timeout and allow-partial runs could finish without a loud end-of-run failure summary.

## Acceptance Criteria
- Default prelabel timeout is increased from 120s to 300s.
- Both interactive and non-interactive `labelstudio-import` completion output prints an explicit red summary when prelabel failures exist.
- Completion output includes guidance/path to failure artifacts when failures occur.

## Verification
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_benchmark_helpers.py`
- `source .venv/bin/activate && pytest -q tests/labelstudio/test_labelstudio_ingest_parallel.py tests/labelstudio/test_labelstudio_prelabel.py`

## Evidence
- Added test coverage for prelabel failure summary printing.
- Added assertions for routed default timeout value (`300`).

## Constraints / Decisions
- Kept `--prelabel-allow-partial` behavior intact (operator-controlled), but made failure visibility explicit at completion.
- Updated docs/contracts for timeout default and error-summary expectations.

## Rollback
- Revert changes in `cookimport/cli.py`, `cookimport/labelstudio/ingest.py`, and related docs/tests to restore previous defaults and summary behavior.
