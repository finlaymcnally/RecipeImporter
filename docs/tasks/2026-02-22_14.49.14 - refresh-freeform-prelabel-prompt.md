# Task: Refresh freeform prelabel prompt template

## Problem statement

Replace the existing full-mode freeform prelabel prompt with the stronger instructions in `docs/06-label-studio/AI-labelling-instructions.md` so model labeling is more consistent at block level.

## Acceptance criteria

- `cookimport/labelstudio/prelabel.py:_build_prompt(...)` uses the new full-mode instruction template content.
- Augment/decorate prompt behavior remains additive and includes existing-label context.
- Tests cover key prompt contracts for both full and augment modes.
- Label Studio docs reflect the new prompt template behavior.

## Verification steps

Run from repo root:

    source .venv/bin/activate
    pip install -e .[dev]
    pytest -q tests/test_labelstudio_prelabel.py

## Evidence

- `pytest -q tests/test_labelstudio_prelabel.py` result: `12 passed, 2 warnings in 1.35s`
- New assertions verify full-mode prompt contains the new instruction sections and that augment mode still emits additive constraints.

## Constraints / gotchas and decision

- Constraint: full-mode instructions require one label per block, but decorate mode must remain additive.
- Decision: apply the new template to `mode="full"` and keep a separate augment prompt path with explicit additive rules (`Only return blocks that should receive a NEW additional label`).

## Rollback / compensation plan

- Revert the prompt branch changes in `cookimport/labelstudio/prelabel.py` and remove the added tests if model quality regresses.
