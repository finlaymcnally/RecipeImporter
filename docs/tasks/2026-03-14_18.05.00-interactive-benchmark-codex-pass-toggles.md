Problem statement

Interactive benchmark setup currently resolves one top-tier profile and only exposes the recipe pipeline plus model/effort overrides. That makes it hard to benchmark recipe Codex, block-label Codex, and pass4 knowledge harvest independently from the menu.

Acceptance criteria

- Interactive benchmark setup exposes separate per-run choices for:
  - recipe Codex pipeline (`off`, `codex-farm-3pass-v1`, `codex-farm-2stage-repair-v1`)
  - block labeling (`deterministic-v1` vs `codex-line-role-v1`)
  - knowledge harvest (`off` vs `codex-farm-knowledge-v1`)
- The chooser still starts from the existing codexfarm/vanilla top-tier contract and then applies only the requested per-surface overrides.
- Interactive benchmark tests prove the selected settings are forwarded into the benchmark helper.

Verification steps

- `source .venv/bin/activate && pytest tests/cli/test_c3imp_interactive_menu.py -k 'benchmark_surfaces or recipe_pipeline_menu_can_select_merged_prototype'`
- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_interactive.py -k benchmark_single_offline_codex_pipeline_passes_settings`

Evidence

- Before the change, interactive benchmark setup could only switch the recipe Codex pipeline and global Codex model/effort.
- After the change, the selected benchmark settings can disable block-label Codex or pass4 knowledge independently while leaving the recipe choice unchanged.

Constraints/gotchas + decision

- Keep the existing top-tier profile contract as the base so import and benchmark still share one chooser implementation.
- Map “block labeling off” to deterministic line-role rather than fully disabling projection so the benchmark can isolate Codex line-role against the deterministic path.

Rollback / compensation plan

- Revert the chooser override layer if the menu becomes too noisy or if benchmark semantics need a different deterministic-vs-off split for line-role.
