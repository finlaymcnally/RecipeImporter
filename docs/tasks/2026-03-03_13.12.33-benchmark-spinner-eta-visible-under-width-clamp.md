# Benchmark Spinner ETA Visible Under Width Clamp

## Problem statement

Long codex-farm status lines (`active [...]` with long file IDs) were right-truncated in the boxed live spinner panel, hiding the appended `(eta ..., avg .../task)` suffix.

## Acceptance criteria

- Live boxed spinner keeps ETA/avg visible when long lines are clamped to terminal width.
- Existing width clamp behavior still holds.
- Regression test covers a narrow-terminal codex-farm progress line with long active-task payload.

## Verification steps

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "truncated or clamps_live_box_width_to_terminal or shows_eta_for_xy_progress or bootstraps_eta_when_first_counter_starts_above_one"`

## Evidence

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k "truncated or clamps_live_box_width_to_terminal or shows_eta_for_xy_progress or bootstraps_eta_when_first_counter_starts_above_one"` -> `4 passed, 161 deselected, 1 warning in 3.89s`

## Constraints/gotchas + decision

- Keep panel compact and ASCII-only; do not widen panel just to fit long lines.
- Decision: preserve timing parentheticals by middle-ellipsis truncation only when needed, leaving existing right-truncate behavior as fallback.

## Rollback / compensation

- Revert truncation logic in `cookimport/cli.py` and remove the new regression test if this display policy is not desired.
