# Problem Statement

Parallel freeform prelabel startup emitted a worker-only banner without `task X/Y`, which could make the spinner look stalled until first completion.

# Acceptance Criteria

- Parallel prelabel kickoff status keeps `task 0/Y` in the message.
- Worker count is still surfaced in kickoff status.
- Regression test covers the `workers=N` kickoff message retaining `task 0/Y`.

# Verification Steps

```bash
. .venv/bin/activate
COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1 pytest tests/labelstudio/test_labelstudio_ingest_parallel.py tests/labelstudio/test_labelstudio_prelabel.py tests/labelstudio/test_labelstudio_benchmark_helpers.py
```

# Evidence

- `cookimport/labelstudio/ingest.py` now emits `Running freeform prelabeling... task 0/Y (workers=N)` for parallel kickoff.
- `tests/labelstudio/test_labelstudio_ingest_parallel.py` asserts the workers kickoff message still includes `task 0/2`.
- Verification command passed: `87 passed, 2 warnings in 3.28s`.

# Constraints / Gotchas

- Spinner ETA parsing depends on retaining an `X/Y` counter in status text.

# Decision

Keep worker metadata as a suffix on the existing `task X/Y` format instead of using a separate phase-only banner.

# Rollback / Compensation

Revert the kickoff message/status assertion changes if worker metadata should be hidden.
