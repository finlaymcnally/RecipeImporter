## Problem

Benchmark cutdown packages were shipping sampled prompt examples, not a complete machine-readable record for every CodexFarm pass call.

## Acceptance Criteria

- Benchmark CodexFarm runs emit `codexfarm/full_prompt_log.jsonl` with one row per pass1/pass2/pass3 call.
- Rows are not sampled, deduped, or truncated.
- Benchmark `run_manifest.json` artifacts include:
  - `full_prompt_log_status`
  - `full_prompt_log_rows`
  - `full_prompt_log_path`
- `scripts/benchmark_cutdown_for_external_ai.py` copies `full_prompt_log.jsonl` into each run package without truncation.
- Sampled text log (`codexfarm_prompt_log.dedup.txt`) remains convenience-only.

## Verification

- `.venv/bin/pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -k 'build_codex_farm_prompt_response_log'`
- `python scripts/benchmark_cutdown_for_external_ai.py --help`

## Evidence

- Targeted helper tests pass after implementation (see verification output in task execution notes).

## Constraints / Notes

- Keep backward compatibility with existing prompt text artifacts.
- Preserve deterministic output ordering.
- Do not require CodexFarm forensics files to exist for baseline row export.

## Decision

- Generate full JSONL rows directly from pass input/output artifacts and run assets at benchmark artifact-build time.
- Treat cutdown sampled prompt text as optional convenience and full JSONL as required source of truth.

## Rollback / Compensation

- Revert benchmark prompt artifact generation to pre-change behavior and remove full prompt manifest fields if storage/perf impact is unacceptable.
