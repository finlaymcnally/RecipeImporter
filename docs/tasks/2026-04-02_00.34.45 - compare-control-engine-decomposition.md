# Compare-Control Engine Decomposition

## Problem statement

`cookimport/analytics/compare_control_engine.py` had grown into a multi-thousand-line owner for record normalization, field-catalog building, filter normalization, and statistical analysis. That made compare/control changes harder to review and blocked the pending Milestone 2 analytics decomposition work in `docs/plans/2026-04-01_23.18.31_large-file-decomposition-roadmap.md`.

## Acceptance criteria

- `cookimport/analytics/compare_control_engine.py` remains the import-stable public facade.
- Field normalization/catalog, filter handling, and analysis logic move into explicit sibling owner modules under `cookimport/analytics/`.
- Existing compare/control CLI and test surfaces keep the same payload shapes and behavior.

## Constraints / gotchas + brief decision

- Keep the existing public engine import path stable because CLI support code and tests import from `cookimport.analytics.compare_control_engine`.
- Put `CompareControlError` in its own module so filter/analysis helpers can raise structured domain errors without circular imports.
- Preserve analytics payload semantics exactly; this is a structural refactor, not a behavior change.

## Rollback / compensation plan

If a regression appears, revert this slice as one atomic change: the new compare/control owner modules, the engine facade rewrite, and the related doc updates. No data migrations or irreversible artifact changes are involved.
