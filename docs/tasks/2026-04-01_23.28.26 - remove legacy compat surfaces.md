Problem statement

The repo still ships old CLI aliases, benchmark replay compatibility keys, legacy Oracle bundle readers, a transition-only `ConversionResult.non_recipe_blocks` seam, and direct-exec adapters for older compact payloads. The goal is to delete those surfaces and leave one current contract in each area while keeping `C3imp`.

Acceptance criteria

- `pyproject.toml` exposes only `cookimport`, `cf-debug`, and `C3imp`.
- Benchmark winner replay and Oracle follow-up trust only current run-config and lane-layout bundle contracts.
- Stage/LLM code no longer uses the targeted legacy seams (`ConversionResult.non_recipe_blocks`, legacy Oracle layout readers, direct-exec compact payload adapters).
- Relevant docs and the ExecPlan reflect the implemented state.

Verification steps

- Run `npm run docs:list`.
- Run focused pytest files for CLI, bench, staging, parsing, and LLM surfaces changed by this cleanup.
- Run `./scripts/test-suite.sh` for the affected domains once focused tests are green.
- Run grep checks for removed aliases/adapters and confirm only intentional historical docs still match.

Evidence

- Pending implementation.

Constraints/gotchas + brief decision

- Keep `C3imp`; delete `import` and `C3import`.
- Do not run CodexFarm-enabled book processing or benchmarks.
- Prefer deleting compatibility code and rewriting callers/tests to the live contract over preserving fallback readers.

Rollback / compensation plan

- Revert the cleanup atomically if removing a seam reveals a still-required live caller that cannot be migrated within this task.
