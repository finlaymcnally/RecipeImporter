## Problem

The interactive CLI can still ship `NameError` regressions outside the benchmark helper stack.
`cookimport.cli_commands.stage` references `DEFAULT_GOLDEN` during dashboard refresh, but that command module does not bind the name directly, and the current undefined-name audit only covers benchmark modules.

## Acceptance Criteria

- `cookimport.cli_commands.stage` binds `DEFAULT_GOLDEN` directly from its owning CLI support surface.
- A static unresolved-global audit covers the interactive CLI modules that participate in the no-subcommand flow.
- The focused undefined-name pytest file fails on the broken baseline and passes after the fix in `.venv`.

## Verification

- `. .venv/bin/activate && pytest tests/core/test_benchmark_undefined_names.py -q`

## Evidence

- Before the fix, the broader unresolved-global audit reports `cookimport.cli_commands.stage:register:stage:DEFAULT_GOLDEN`.
- After the fix, the same pytest file passes with no unresolved interactive CLI globals.

## Constraints / Decision

- Keep scope to the missing stage binding and the test gap that let it escape.
- Prefer a static guard over a heavy interactive end-to-end test because the failure mode is an import/binding regression.
