Problem statement

Line-role runtime telemetry was coercing missing worker usage to zero, and downstream analytics were treating those zeroes as literal spend.

Acceptance criteria

- New line-role `telemetry_summary.json` artifacts keep missing token usage unavailable instead of serializing fake zero totals.
- Prompt-budget summaries mark line-role token coverage as `complete`, `partial`, or `unavailable` and blank token totals when coverage is incomplete.
- Benchmark history/backfill leaves line-role token fields blank when telemetry indicates incomplete usage.

Verification steps

- `. .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles.py::test_label_atomic_lines_leave_missing_line_role_usage_unavailable tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py::test_prompt_budget_summary_marks_line_role_partial_token_usage_unavailable tests/analytics/test_stats_dashboard.py::TestBackfillBenchmarkHistoryCsv::test_backfill_benchmark_csv_leaves_line_role_tokens_blank_when_usage_incomplete`

Evidence

- New parsing regression proves line-role runtime summaries record `attempts_without_usage` and leave `tokens_*` unavailable when worker usage is missing.
- New prompt-budget regression proves incomplete line-role usage blanks `tokens_*` and marks `token_usage_status`.
- New analytics regression proves benchmark CSV backfill leaves line-role token columns empty instead of writing `0`.

Constraints / gotchas + decision

- Existing saved artifacts are not rewritten, so readers must fail closed on old bad summaries as well as newly generated good ones.
- The fix therefore patches both the line-role telemetry writer and the analytics readers.

Rollback / compensation plan

- Revert this change set to restore the prior zero-coercion behavior if a downstream consumer unexpectedly depends on it.
