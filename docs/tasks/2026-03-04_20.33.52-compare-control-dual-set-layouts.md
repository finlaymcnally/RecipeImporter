# Ship dual Compare & Control sets with split controls, stacked tables, and flexible charts

This ExecPlan is a living document. The sections `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` must be kept up to date as work proceeds.

Maintain this document in accordance with `docs/PLANS.md` from the repository root.

## Purpose / Big Picture

After this change, `Compare & Control` supports two independent comparison setups at the same time. A user can keep one comparison in Set 1, expand Set 2 from the right, and evaluate both together without constantly reconfiguring controls.

User-visible outcomes:
- Controls: Set 2 can be expanded from the right, and both sets render side-by-side in a taller split workspace.
- Tables: each set’s result table renders full-width in a stacked flow (not squished side-by-side), with visual shading to distinguish Set 1 vs Set 2.
- Charts: two chart instances can be shown stacked or side-by-side; when compatible, they can also be combined into one chart with single-Y or dual-Y axes.

## Surprises & Discoveries

- Observation: compare/control currently has one monolithic state object and one chart host, so dual-set support requires explicit secondary-state plumbing rather than simple markup cloning.
  Evidence: `compareControlState` single instance, fixed IDs (`compare-control-*`) and single chart host (`compare-control-trend-chart`) in `dashboard_render.py`.

- Observation: compare/control was intentionally decoupled from Previous Runs filters in recent work; this must be preserved while adding dual sets.
  Evidence: docs/tasks + docs/understandings note decoupled local subset semantics (`Apply local subset` does not write table filters).

- Observation: combined categorical charts need category remapping before merge, otherwise set-specific category arrays drift and series lengths mismatch.
  Evidence: first combined-chart harness pass produced inflated point counts from per-series category expansion, requiring explicit non-null point counting in assertions.

## Decision Log

- Decision: keep Set 1 state shape fully backward-compatible and add Set 2 as additive fields under persisted `compare_control` payload.
  Rationale: existing dashboard state files and tests expect current Set 1 contract.
  Date/Author: 2026-03-04 / assistant

- Decision: keep tables stacked vertically even when controls/charts are side-by-side.
  Rationale: user explicitly requested no squished dual-table layout.
  Date/Author: 2026-03-04 / assistant

- Decision: implement combined charts as best-effort with explicit fallback messaging when axis/category compatibility is not available.
  Rationale: user asked for combined mode “where possible”; impossible mixes should fail clearly rather than silently misplot.
  Date/Author: 2026-03-04 / assistant

- Decision: keep compare/control Set 1 IDs untouched and add Set 2 IDs via suffixes.
  Rationale: existing test contracts and persisted UI-state compatibility are anchored on Set 1 IDs/fields.
  Date/Author: 2026-03-04 / assistant

- Decision: enforce local-only subset semantics for both sets in dual mode.
  Rationale: prior decoupling intentionally removed table-filter mutation; dual-set feature should not reintroduce hidden coupling.
  Date/Author: 2026-03-04 / assistant

## Plan of Work

### Milestone 1: Add dual compare/control state and compatibility handling

Update dashboard JS state to track:
- `compareControlState` (Set 1, unchanged shape)
- `compareControlSecondaryState` (Set 2)
- `compareControlSecondSetEnabled` toggle
- chart layout mode (`stacked`, `side_by_side`, `combined`)
- combined axis mode (`single`, `dual`)

Persist these fields in `previous_runs.compare_control` while keeping Set 1 legacy fields intact for compatibility.

### Milestone 2: Add dual controls UI with right-side expansion

Extend compare/control HTML/CSS to include:
- Set 2 expand/collapse control
- secondary control panel anchored on the right
- dual-panel split layout with increased panel height when Set 2 is open

Keep existing Set 1 IDs intact and add explicit Set 2 IDs.

### Milestone 3: Render dual result tables in stacked flow

Render Set 1 and Set 2 analysis output in two full-width stacked result cards below controls.

Apply subtle per-set shading and headings to maintain visual scan flow.

### Milestone 4: Add chart layout modes and combined rendering

Add UI controls for chart layout/combination. Render behavior:
- `stacked`: Set 1 chart above Set 2 chart
- `side_by_side`: Set 1 and Set 2 charts in two columns
- `combined`: one combined chart when compatible, with axis mode options:
  - `single`: both series share one Y-axis
  - `dual`: Set 2 series use right-side Y-axis

If combined mode is incompatible, show clear fallback message.

### Milestone 5: Regression tests and docs

Update tests for:
- new HTML IDs/controls
- state persistence strings/functions
- combined chart behavior via Node harness

Update docs:
- dashboard readme bullets for dual-set controls/tables/charts
- short task note for this change
- understanding note summarizing seam decisions and constraints
