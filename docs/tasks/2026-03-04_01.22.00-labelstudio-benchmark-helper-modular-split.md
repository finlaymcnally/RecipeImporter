## Problem statement

`test_labelstudio_benchmark_helpers.py` bundled too many concerns, making targeted iteration and slow/fast slicing inefficient.

## Acceptance criteria

- Benchmark-helper tests are split into focused modules by concern.
- Central marker mapping covers each new test file.
- Slow slice includes heavy eval/scheduler modules, not all benchmark-helper tests.
- Collection works without duplicate tests.

## Verification steps

- `. .venv/bin/activate && pytest --collect-only -q tests/labelstudio/test_labelstudio_benchmark_helpers.py tests/labelstudio/test_labelstudio_benchmark_helpers_eval_payload.py tests/labelstudio/test_labelstudio_benchmark_helpers_scheduler.py tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py`
- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_eval_payload.py::test_labelstudio_benchmark_direct_call_uses_real_defaults tests/labelstudio/test_labelstudio_benchmark_helpers_scheduler.py::test_build_all_method_variants_non_epub_single_variant tests/labelstudio/test_labelstudio_benchmark_helpers_single_profile.py::test_interactive_benchmark_single_profile_all_matched_mode_routes_to_runner`
- `. .venv/bin/activate && pytest -m slow --collect-only -q tests/labelstudio`

## Evidence

- Split collection counts: `75 + 26 + 66 + 7`.
- Targeted representative split tests: `3 passed, 1 warning in 2.19s`.
- Slow labelstudio slice now collects only:
  - `test_labelstudio_benchmark_helpers_eval_payload.py`
  - `test_labelstudio_benchmark_helpers_scheduler.py`

## Constraints/gotchas

- Split modules reuse shared helpers/import state from the base helper module; avoid importing `__dunder__` globals to prevent pytest import-file mismatches.
- Keep new test filenames unique and mapped in `tests/conftest.py`.

## Decision

Use concern-based modular files to preserve behavior while making benchmark-helper iteration faster and safer.

## Rollback / compensation

Merge split tests back into `test_labelstudio_benchmark_helpers.py` and revert corresponding `tests/conftest.py` mapping/slow-file updates.
