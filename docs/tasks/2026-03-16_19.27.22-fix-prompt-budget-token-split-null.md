# Problem

Single-offline benchmark `prompt_budget_summary.json` can show `tokens_input`, `tokens_cached_input`, and `tokens_output` as null even though the underlying CodexFarm manifests recorded those per-call values.

# Acceptance Criteria

- `build_prediction_run_prompt_budget_summary(...)` reads CodexFarm split-token rows from the single-offline manifest shape used by benchmark runs.
- CodexFarm stage summaries preserve `tokens_input`, `tokens_cached_input`, and `tokens_output` when top-level telemetry rows are present.
- A regression test covers the broken manifest layout and fails before the fix.

# Verification

- `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -k prompt_budget_summary`

# Evidence

- Before fix: regression test for top-level CodexFarm telemetry rows fails because split-token fields are null.
- After fix: prompt-budget regression tests pass and the new case shows non-null split totals.

# Constraints / Decisions

- Keep scope limited to prompt-budget aggregation. Do not change benchmark dashboard rendering or unrelated telemetry producers.
- Prefer supporting both existing nested telemetry and the single-offline top-level telemetry layout.
