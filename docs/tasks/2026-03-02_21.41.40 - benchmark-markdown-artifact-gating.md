# Problem
Interactive benchmark runs default to markdown-off (`COOKIMPORT_BENCH_WRITE_MARKDOWN=0`), but benchmark summary markdown files were still emitted (`eval_report.md` and `codex_vs_vanilla_comparison.md`). This created extra high-level artifacts and drifted from operator expectations.

# Acceptance Criteria
- `labelstudio-benchmark` with `write_markdown=False` does not write `eval_report.md`.
- `run_manifest.json` omits `artifacts.eval_report_md` when markdown is disabled.
- Interactive single-offline paired comparison keeps `codex_vs_vanilla_comparison.json` and skips `.md` when markdown is disabled.
- Markdown artifacts are still written when markdown is enabled.
- Bench/CLI/Label Studio docs reflect the markdown-gated behavior.

# Verification Steps
- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py -q`

# Evidence
- Updated tests in `tests/labelstudio/test_labelstudio_benchmark_helpers.py` assert markdown-off behavior for benchmark eval and single-offline comparison outputs.

# Constraints / Gotchas
- Keep JSON artifacts (`eval_report.json`, comparison JSON, run manifest) as stable machine-readable truth.
- Do not change default CLI flag values.

# Decision
Use the existing `write_markdown` control (already threaded through interactive and benchmark call paths) to gate benchmark markdown outputs as well, instead of introducing a second benchmark-only markdown flag.

# Rollback / Compensation
- Revert changes in `cookimport/cli.py` and corresponding tests/docs if markdown output must be unconditional again.
