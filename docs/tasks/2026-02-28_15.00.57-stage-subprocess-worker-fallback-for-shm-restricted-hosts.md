## Problem statement

On this host, process pool startup fails with `PermissionError: [Errno 13] Permission denied`, so stage cannot use process workers directly. We need stage to keep parallel fanout without relying on process-pool SemLock behavior.

## Acceptance criteria

- Stage fallback order becomes `process -> subprocess-backed workers -> thread -> serial`.
- When process workers are denied, stage warns that it switched to subprocess-backed worker concurrency.
- Subprocess workers support `single`, `pdf_split`, and `epub_split` stage job kinds.
- If subprocess worker probing fails, stage continues with in-process thread fallback.
- Focused tests cover subprocess worker request dispatch and self-test probe behavior.

## Constraints/gotchas and decision

- This change does not alter host `/dev/shm`/SemLock policy and does not make process pools available.
- Decision: use subprocess-per-job worker execution so stage regains CPU-parallel work under restricted hosts while preserving existing thread/serial safety fallbacks.
