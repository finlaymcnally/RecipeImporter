# Problem

`labelstudio-benchmark` canonical-text runs regressed after the refactor because prediction-run metadata started keeping scorer pointers on stage-backed artifacts even when `line-role-pipeline/` artifacts were present and intended to be scored.

# Acceptance Criteria

- `generate_pred_run_artifacts(...)` returns `stage_block_predictions_path` and `extracted_archive_path` pointing at `line-role-pipeline/` artifacts when those artifacts exist.
- Processed stage outputs still get written, but benchmark scoring reads the projection pair.
- A targeted test proves the scorer-pointer preference when both stage-backed and line-role artifacts are available.

# Verification

- Activate `.venv`.
- Run `pytest tests/labelstudio/test_labelstudio_ingest_parallel.py -k scorer_pointer`.

- After fix: `source .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_ingest_parallel.py -k 'projection_prefers_projection_for_scoring or authoritative_line_role_artifacts_after_llm_recipe_update'` passes (`2 passed`).

# Constraints / Decision

- Keep the fix local to prediction-run artifact selection in `cookimport/labelstudio/ingest.py`.
- Do not change scoring code or artifact formats; only restore the intended canonical pointer wiring.

# Rollback

- Revert this change to return to stage-backed scorer pointers, understanding that canonical-text line-role benchmark runs will score the wrong artifact pair again.
