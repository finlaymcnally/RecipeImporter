## Problem statement

`docs/understandings/2026-02-27_10.52.17-speed1-5-implementation-audit.md` identified remaining speed1-5 gaps: missing prepared-archive abstraction, split-merge `outputStats` undercount risk, missing stats parity test, missing direct `bench run` write toggles, and shallow no-drift proof depth.

## Acceptance criteria

- `PreparedExtractedArchive` + `prepare_extracted_archive(...)` exist and are used by prediction-run generation.
- Split-merge writes report after raw-artifact merge and records moved raw files into `outputStats`.
- Test coverage includes fresh directory-walk parity for split-merge `report.outputStats`.
- `bench run` exposes direct `--write-markdown/--no-write-markdown` and `--write-labelstudio-tasks/--no-write-labelstudio-tasks` flags, overriding config/defaults only when passed.
- No-drift coverage includes deterministic stage-prediction equality across markdown-toggle modes.
- Relevant bench/staging/labelstudio docs and speed1-5 plan living sections are updated.

## Verification steps

From repo root:

1. `source .venv/bin/activate`
2. `PIP_CACHE_DIR=/tmp/.pip-cache python -m pip install -e '.[dev]'`
3. `pytest -q tests/staging/test_split_merge_status.py`
4. `pytest -q tests/labelstudio/test_labelstudio_ingest_parallel.py -k "prepare_extracted_archive_matches_legacy_payload or markdown_toggle_keeps_stage_predictions_identical or passes_write_markdown_to_processed_outputs or can_skip_tasks_jsonl"`
5. `pytest -q tests/bench/test_bench.py -k "bench_run_direct_write_flags_override_config or build_pred_run_for_source_passes_epub_extractor"`

7. `python -m cookimport.cli bench run --help > /tmp/bench_run_help.txt && rg -n "write-markdown|write-labelstudi" /tmp/bench_run_help.txt`
