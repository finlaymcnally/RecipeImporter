# Problem statement

Benchmark rows already show run configuration summary in the dashboard, but stage/import rows did not.
You requested the same run-config detail for imports (for example `epub_extractor=legacy | ocr_device=auto | ocr_batch_size=1 | effective_workers=10`).

# Acceptance criteria

- Stage/import records carry run config through analytics surfaces.
- Throughput tables show `Importer` and `Run Config` columns for stage/import rows.
- Run-config summary format matches benchmark summary key order.
- Stats dashboard tests cover stage run-config collection/rendering behavior.

# Verification steps

- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py`

# Evidence

- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py` -> `27 passed in 0.20s`.

# Constraints/gotchas

- Preserve CSV backward compatibility with schema migration for older history files.
- Keep benchmark behavior unchanged.

# Decision

Persist stage run config in history CSV via `run_config_json`, collect it into `StageRecord.run_config`, and render importer/run-config columns in throughput tables using the same summary formatter as benchmark rows.
