## Purpose / Big Picture

After this work, a reviewer should be able to run `scripts/benchmark_cutdown_for_external_ai.py` on benchmark outputs and always get one predictable, high-signal package that explains both quality deltas and likely causes. Primary docs to update:

Core behavior this script orchestrates today:

1. Discover benchmark runs by requiring both `eval_report.json` and `run_manifest.json`.
2. Build per-run cutdown summaries and sampled diagnostics.
3. For codex-enabled runs, preserve full prompt logs (`full_prompt_log.jsonl`) and derive prompt warning/projection diagnostics when possible.
4. Emit codex-vs-baseline root diagnostics (`changed_lines.codex_vs_vanilla.jsonl`, `per_recipe_or_per_span_breakdown.json`, `targeted_prompt_cases.md`).
5. Optionally flatten output into `<output_dir>_md`.

Important term definitions used in this plan:

- Full-failure context export: compressed JSONL containing each wrong-label line plus neighboring line text and provenance fields.
- Preprocess trace failure export: compressed JSONL attempting to connect wrong-label lines to prompt-call context and extracted archive block context.
- Status contract: explicit per-artifact availability state stored under `sample_counts` in each `need_to_know_summary.json`.

## Plan of Work

### Milestone 1: Lock Contract Coverage in Tests

Extend `tests/bench/test_benchmark_cutdown_for_external_ai.py` with fixture-style run directories that exercise `_build_run_cutdown(...)` end-to-end at function level. Assert that:

- `wrong_label_lines.with_context.full.jsonl.gz` is created when wrong-label rows exist.
- decoded rows include stable required keys (`run_id`, `line_index`, `gold_label`, `pred_label`, `current_line`, `source_key`).
- `preprocess_trace_failures.jsonl.gz` is created only when codex + wrong-label rows + prediction-run + extracted archive + full prompt log are all available.
- fallback statuses are preserved in `sample_counts` when upstream data is missing.

This milestone ends when tests fail on intentionally removed behavior and pass with current behavior.

### Milestone 2: Determinism Regression Guard

Add one focused test that writes the same ordered rows twice through `_write_jsonl_gzip_deterministic(...)` into two output files and compares byte content for equality. This verifies deterministic gzip headers and sorted/minified JSON row encoding are preserved.

This milestone ends when byte-for-byte equality is asserted.

## Idempotence and Recovery

The cutdown operation remains idempotent with `--overwrite`. If output is partial or stale, rerun with `--overwrite` to regenerate from source benchmark artifacts.

If flattening is enabled and intermediate output should be inspected, use `--keep-cutdown` so both raw and flattened outputs are preserved.

If a new test fails due to missing fixture context, recover by building minimal synthetic fixtures in `tmp_path` within the test file rather than depending on large checked-in benchmark directories.
