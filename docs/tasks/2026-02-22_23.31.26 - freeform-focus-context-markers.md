# Problem Statement

Freeform span prelabel prompts currently place focus blocks at the start of each segment, so context-only blocks show up mostly after the labelable window. The scope boundary is also not clear enough in Label Studio tasks.

# Acceptance Criteria

- Freeform task generation centers the focus subset within each segment when possible.
- Prompt block streams expose explicit context-before/context-after markers around focus START/STOP markers.
- Span prompt instructions clearly describe labelable vs context-only zones.
- Freeform Label Studio config surfaces per-task scope hints/ranges so annotators can see what to label.
- Existing labelstudio freeform/prelabel/ingest/benchmark helper tests pass.

# Constraints / Gotchas

- Overlap floor (`segment_overlap_effective >= segment_blocks - segment_focus_blocks`) is still required so focus windows hand off without internal gaps.
- Edge segments can still be one-sided (for example no context-after at file tail).

# Decision

Center focus windows inside each segment by default and add explicit context-before/context-after markers plus Label Studio scope headers (`focus_scope_hint`, focus/context ranges) so boundary intent is visible in both prompt logs and annotation UI.

# Rollback / Compensation

Set `segment_focus_blocks == segment_blocks` to disable context-vs-focus separation for a run (all blocks become labelable), or revert this task diff to restore leading-focus behavior.
