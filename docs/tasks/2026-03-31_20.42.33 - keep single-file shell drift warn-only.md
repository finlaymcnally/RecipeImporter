# Keep single-file shell drift warn-only

Problem statement:
Single-file `task.json` workers currently get killed for repeated shell drift or ad hoc `task.json` rewrites even when they stay inside the workspace boundary. That wastes tokens and turns a steerable prompt-shape problem into a run failure.

Acceptance criteria:
- Recipe, knowledge, and canonical line-role workspace watchdogs record `single_file_shell_drift` warnings but do not terminate the session for that warning alone.
- Hard boundary violations still terminate the session.
- Tests cover repeated drift and ad hoc transform cases as warn-only behavior.

Verification steps:
- From `/home/mcnal/projects/recipeimport`, activate `.venv`.
- Run:
  `pytest -q tests/llm/test_recipe_phase_workers.py::test_recipe_workspace_watchdog_keeps_running_after_repeated_single_file_shell_drift tests/llm/test_knowledge_orchestrator_contracts.py::test_knowledge_workspace_watchdog_warns_on_egregious_single_file_shell_transform tests/parsing/test_canonical_line_roles.py::test_line_role_workspace_watchdog_keeps_running_after_repeated_single_file_shell_drift tests/parsing/test_canonical_line_roles.py::test_label_atomic_lines_allows_line_role_jq_fallback_operator_output_command tests/parsing/test_canonical_line_roles.py::test_label_atomic_lines_allows_line_role_node_transform`
- Run:
  `pytest -q tests/parsing/test_canonical_line_roles.py::test_line_role_strict_watchdog_still_kills_benign_commands_in_structured_mode tests/llm/test_codex_exec_runner.py::test_codex_exec_runner_detects_boundary_violations_separately_from_telemetry`

Evidence:
- The five warn-only nodeids above exited with code 0 in `.venv`.
- The two boundary checks above exited with code 0 in `.venv`.
- A broader selector that included `line_role_workspace_watchdog` also hit three pre-existing/stale timing assertions that still expect a 2-second line-role grace window, while the current runtime reports the shared 15-second default. That mismatch is outside this warn-only change.

Constraints/gotchas + brief decision:
- Keep the existing command classification and warning metadata because that is the steering signal and operator visibility surface.
- Change only the escalation behavior: boundary violations remain kill-worthy, shell drift becomes warn-only.
- Do not “fix” the unrelated line-role grace-window assertions in this task; they are a separate timing-contract issue.

Rollback / compensation plan:
- Revert the watchdog callback changes in the three stage runtimes if warn-only behavior proves too permissive.
