# Problem Statement

Default test runs still emitted non-informational formatting output (pytest dot flood and success-path test prints).
Need compact output that preserves only actionable pass/fail context.
Later audit found a gap: `-o addopts='' -vv` bypassed the compact defaults and reintroduced separator-heavy output.

# Acceptance Criteria

- Default `pytest` runs do not emit per-test dot/glyph progress rows.
- Compact mode remains on even when callers pass `-o addopts=''`.
- Success-path `print(...)` noise is removed from test files.
- `tests/AGENTS.md` documents every noise example found during audit.
- `tests/README.md` reflects compact output expectations.

# Verification Steps

```bash
. .venv/bin/activate
pytest -m smoke
pytest tests/core/test_phase1_manual.py tests/tagging/test_tagging.py
pytest -o addopts='' -vv tests/labelstudio/test_labelstudio_prelabel.py::test_prelabel_span_prompt_marks_focus_window_without_block_duplication
rg -n "print\\(" tests
```

# Evidence

- `pytest -m smoke` -> `33 passed, 489 deselected, 2 warnings in 2.08s` (no dot flood).
- `pytest tests/core/test_phase1_manual.py tests/tagging/test_tagging.py` -> `23 passed, 2 warnings in 1.24s`.
- `rg -n "print\\(" tests` now reports only:
  - `tests/fixtures/generate_scanned_pdf.py` (manual helper script)
  - documentation mentions in `tests/AGENTS.md`

# Constraints / Gotchas

- Keep ellipsis-bearing message assertions that validate product status-message contracts; those are not test-runner decoration.
- Keep manual utility script output (`tests/fixtures/generate_scanned_pdf.py`) for direct script usage.

# Decision

Use a deterministic pytest hook and reporter style:
- `tests/conftest.py:pytest_report_teststatus(...)` suppresses pass/skip glyphs.
- `tests/conftest.py:pytest_configure(...)` enforces compact reporter behavior even with `-o addopts=''` (verbose opt-out: `COOKIMPORT_PYTEST_VERBOSE_OUTPUT=1`).
- `pytest.ini:console_output_style = classic` removes the progress bar row.

Remove success-path prints from test modules and keep report text only in assertion failures.

# Rollback / Compensation

If this compact mode is undesirable, remove:
- `console_output_style = classic` from `pytest.ini`
- `pytest_report_teststatus(...)` glyph suppression hook from `tests/conftest.py`
- restored prints can be reapplied in affected test files.
