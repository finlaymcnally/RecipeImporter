## Acceptance Criteria

- A regression test fails on the old schema and passes once fixed.
- The knowledge output schema satisfies the strict nested `required` contract expected by live structured outputs.
- The focused LLM schema-pack tests pass in the project venv.

## Constraints / Gotchas

- Keep the fix tight to the schema/test seam; do not widen into unrelated prompt or runtime refactors.
- The workspace is dirty in unrelated areas; avoid touching nearby user changes.

## Decision

- Add a recursive schema-contract regression test so nested object shapes are checked, then fix the missing nested `required` entry in the knowledge schema.
