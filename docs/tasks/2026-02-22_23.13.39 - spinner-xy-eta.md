# Problem Statement

Callback-driven CLI spinners showed phase text and stale-phase elapsed seconds, but did not estimate remaining time for known-size `X/Y` loops.
Need ETA in the spinner using observed seconds-per-unit throughput.

# Acceptance Criteria

- Shared callback spinner path computes ETA for messages containing `X/Y` counters.
- ETA uses average seconds per completed unit multiplied by remaining units.
- Existing stale-phase elapsed-seconds behavior remains intact.
- Tests cover ETA formatting and runtime spinner updates.
- CLI docs mention ETA behavior for `X/Y` progress.

# Verification Steps

```bash
. .venv/bin/activate
pip install -e .[dev]
pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py
```

# Evidence

- `cookimport/cli.py` tracks `X/Y` progress rate and appends ETA/avg throughput in spinner status output.
- `tests/labelstudio/test_labelstudio_benchmark_helpers.py` includes ETA formatting and `X/Y` runtime spinner assertions.
- Docs updated in `cookimport/README.md`, `docs/02-cli/02-cli_README.md`, and `docs/IMPORTANT CONVENTIONS.md`.
- `. .venv/bin/activate && pip install -e .[dev] && pytest tests/labelstudio/test_labelstudio_benchmark_helpers.py` -> `45 passed, 2 warnings in 3.34s`.
- `. .venv/bin/activate && pytest tests/bench/test_progress_messages.py` -> `2 passed in 0.01s`.

# Constraints / Gotchas

- Progress callbacks may include nested counters (`config`, `item`, `task`); use the right-most counter as the active unit.
- ETA is only meaningful after at least one observed `X` increment.

# Decision

Implement ETA centrally in `_run_with_progress_status(...)` so all callback-driven spinners inherit it automatically.

# Rollback / Compensation

Revert the spinner helper and tests/docs from this task file if ETA output is noisy or misleading.
