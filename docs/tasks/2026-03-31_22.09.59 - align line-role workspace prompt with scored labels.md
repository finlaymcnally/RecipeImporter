# Align line-role workspace prompt with scored labels

Problem statement:
The live line-role workspace-worker prompt used in benchmark runs omits `RECIPE_TITLE` and `YIELD_LINE` guidance even though canonical scoring expects both labels. In the March 31 benchmark artifact, that drift helped produce `0.0` recall for those labels in the Codex lane.

Acceptance criteria:
- The workspace-worker prompt in `cookimport/parsing/canonical_line_roles/planning.py` explicitly teaches `RECIPE_TITLE` and `YIELD_LINE`.
- The same prompt explicitly says nearby `Variations` context must reset when a fresh title is followed by yield or ingredient rows.
- A targeted regression test fails before the code change and passes after it.

Verification steps:
- `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles_prompting.py -q -k "workspace_worker_prompt"`

Evidence:
- Pre-change benchmark artifact `data/golden/benchmark-vs-golden/2026-03-31_21.49.39/.../codexfarm/prompts/prompt_line_role.txt` lists worker rules for `INGREDIENT_LINE`, `INSTRUCTION_LINE`, `TIME_LINE`, `HOWTO_SECTION`, `RECIPE_VARIANT`, `RECIPE_NOTES`, `NONRECIPE_CANDIDATE`, and `NONRECIPE_EXCLUDE`, but not `RECIPE_TITLE` or `YIELD_LINE`.
- The same benchmark scored `RECIPE_TITLE` recall `0.0` and `YIELD_LINE` recall `0.0` for `codexfarm` in `single_book_summary.md`.
- Fail-before proof: `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles_prompting.py -q -k "workspace_worker_prompt"` exited with code `1`.
- Pass-after proof:
  - `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles_prompting.py -q -k "workspace_worker_prompt"` exited with code `0`.
  - `source .venv/bin/activate && pytest tests/parsing/test_canonical_line_roles_prompting.py -q` exited with code `0`.

Constraints/gotchas + brief decision:
- Keep this change prompt-only. Do not add deterministic relabeling logic.
- Add coverage in a clean prompt-focused test file instead of touching the already-dirty runtime test file.

Rollback / compensation plan:
- Revert the prompt-text and test changes together if the new assertions prove too brittle or the worker prompt needs a different wording.
