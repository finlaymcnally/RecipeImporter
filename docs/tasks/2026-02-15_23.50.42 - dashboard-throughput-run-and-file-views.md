# Problem statement

Import throughput in the dashboard was mostly presented as one combined timeline and a slowest-runs table.
It was hard to answer both:
- how import speed is changing by run/date, and
- how processing speed for one specific file changes over time.

# Acceptance criteria

- Throughput UI keeps a clear run/date organization (timeline + recent run table).
- Throughput UI adds a file-focused organization (selected file trend over time).
- New UI is generated by `cookimport stats-dashboard` output (`index.html`) without requiring backend schema changes.
- Renderer tests cover the presence of the new run/date + file-trend elements.

# Verification steps

- `. .venv/bin/activate && pip install -e .[dev]`
- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py`

# Evidence

- `. .venv/bin/activate && pytest -q tests/test_stats_dashboard.py` -> `24 passed in 0.19s`.

# Constraints/gotchas

- Keep this change scoped to dashboard rendering/UI; avoid changing collector/schema contracts.
- Preserve existing benchmark sections and file-scheme inline JSON behavior.

# Decision

Implement run/date and file-trend organization fully in `dashboard_render.py`:
- run/date: rename and sort table as "Recent Runs"
- file trend: add file selector + per-file chart (`total_seconds`) + per-file history table
