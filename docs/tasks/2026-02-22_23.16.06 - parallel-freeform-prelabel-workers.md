# Problem Statement

Freeform AI prelabeling was strictly serialized per task, making long imports feel stalled even though task prompts are independent.

# Acceptance Criteria

- Freeform prelabel generation supports bounded parallel task execution.
- CLI exposes a worker knob for non-interactive imports.
- Progress remains `task X/Y` based and reports completion counts while workers run.
- Prompt logging, prelabel summaries, and strict/allow-partial failure behavior remain intact.
- Existing labelstudio prelabel tests pass.

# Constraints / Gotchas

- Provider usage counters are shared state and must stay thread-safe when one provider instance is reused.
- Prompt-log output must stay deterministic enough for audit/readability even when task completions are out of order.

# Decision

Use a bounded `ThreadPoolExecutor` for task-level prelabel calls (`--prelabel-workers`, default `4`) and keep deterministic post-processing by sorting completed task results by task index before writing logs/errors.

# Rollback / Compensation

Set `--prelabel-workers 1` to restore serial runtime behavior without code rollback.
