## Problem statement

CodexFarm now exposes non-mutating caller guidance via `run autotune --json` (flag overrides, command preview, prompt/pipeline diffs). Recipeimport did not consume this payload, so pass manifests lacked immediate follow-up tuning actions.

## Acceptance criteria

- Subprocess runner attempts `run autotune --run-id <id> --json` after successful `process` calls.
- Runner persists returned payload as `autotune_report` in `CodexFarmPipelineRunResult` and serialized `process_run(s)` metadata.
- Missing/unsupported autotune command remains non-fatal (`autotune_report` stays null).
- Targeted LLM/tagging tests pass with updated contract assertions.
