# Fix Upload Bundle Recipe Span Attribution

## Problem

The benchmark upload bundle currently rebuilds active recipe spans from prompt-log block indices, then compares them directly against canonical line indices.
When projected scoring uses a different coordinate system, recipe-local scored lines are misclassified as outside-span and recipe-stage blame falls back to `line_role`.

## Acceptance Criteria

- Active recipe span breakout uses canonical line coordinates when projected span artifacts are available.
- Recipe-local changed lines keep their stable recipe ids in pair diagnostics, per-recipe breakdown, and blame summaries even when block and line coordinates differ.
- Focused benchmark tests cover the mismatched-coordinate regression and the recipe-stage blame regression.

## Evidence

- Added a pair-diagnostics regression that uses prompt spans in shifted block coordinates plus `projected_spans.jsonl` canonical line mappings.
- Added an upload-bundle regression that proves recipe-local changed lines are blamed to recipe stages instead of defaulting to `line_role`.

## Constraints / Decisions

- Keep the fix local to benchmark upload-bundle derivation; do not change scoring semantics or recipe correction output semantics.
- Prefer canonical projected span artifacts as the normalization seam because they already bridge source block indices to scored canonical line indices.
