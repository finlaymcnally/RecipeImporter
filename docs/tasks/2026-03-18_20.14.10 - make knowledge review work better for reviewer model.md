## Problem statement

The knowledge-review prompt and compact payload were still nudging the reviewer model toward bad deterministic priors.
Obvious junk families like praise blurbs, signup copy, and chapter menus were arriving with hints that sounded more confident than the evidence deserved.

## Acceptance criteria

- The knowledge prompt explicitly says raw block text is authoritative and hints are weak.
- The model-facing payload no longer emits `semantic_hint`.
- `suggested_lane` is only exposed for strong `knowledge` evidence, not as a default guess on noisy or menu-like chunks.
- Fresh Food Lab and Salt Fat prompt previews look more reviewer-friendly while keeping the stage compact.
- Focused knowledge tests pass.

## Constraints / gotchas

- Keep the change inside the knowledge-review stage; do not turn this into an upstream parser or line-role rewrite.
- Preserve grounded chunk text and compact transport; the goal is better reviewer guidance, not more metadata.
- Archived benchmark roots did not resolve cleanly through `cf-debug preview-prompts` in this checkout, so fresh deterministic stage runs were used for preview validation.

## Decision

- Make the prompt more skeptical and repo-specific.
- Remove `semantic_hint` from the billed payload.
- Expose `suggested_lane` only when deterministic evidence is strong enough that the soft prior is more helpful than harmful.

## Rollback / compensation plan

If this makes reviewer prompts worse, first revert the stronger hint suppression in `cookimport/llm/codex_farm_knowledge_jobs.py` while keeping the prompt rewrite. If the prompt itself is the problem, restore the previous prompt and rerun the same focused tests and preview commands.
