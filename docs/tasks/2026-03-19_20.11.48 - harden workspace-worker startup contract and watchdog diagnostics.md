summary: "Make long-lived Codex workers start from named local files instead of orientation commands, and preserve offending command text in watchdog kills."
---

# Problem

The first benchmark after the long-lived worker cutover still showed main worker sessions wasting their first action on trivial `ls` / `pwd` orientation commands. The watchdog correctly killed those off-contract runs, but the worker contract was too implicit and the resulting artifacts did not preserve the exact offending command in a high-signal place.

# Acceptance Criteria

- Workspace-worker roots contain a small repo-written `worker_manifest.json`.
- Workspace-worker AGENTS text and stage prompts tell Codex to open named local files directly and explicitly forbid orientation commands such as `pwd`, `ls`, `find`, and `tree`.
- Recipe, knowledge, and line-role worker prompts all reflect that tighter startup contract.
- Strict-JSON watchdog kills preserve the offending command text in `reason_detail`.

# Verification

- `./.venv/bin/pytest -q tests/llm/test_codex_exec_runner.py tests/parsing/test_canonical_line_roles.py tests/llm/test_codex_farm_knowledge_orchestrator.py tests/llm/test_recipe_phase_workers.py`
- `./scripts/test-suite.sh domain llm`

# Evidence

- The targeted prompt/runtime suites passed after the change.
- `./scripts/test-suite.sh domain llm` passed with `127 passed, 30 deselected, 1 warning`.

# Constraints / Decision

- Do not loosen the watchdog rule for tool use; `ls` / `pwd` are still contract violations for strict-JSON worker stages.
- Fix the startup contract instead: make the workspace entry files explicit, keep outputs file-backed, and improve diagnostics so future runs show exactly what command triggered a kill.
