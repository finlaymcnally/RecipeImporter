## Problem

Interactive single-book benchmark runs can crash during artifact reconstruction with `NameError: name '_coerce_int' is not defined`.
`cookimport.cli_support.bench_artifacts` relied on the shared `bench` runtime snapshot to provide `_coerce_int`, but that helper is not available when `bench_artifacts` is imported during `cli_support` bootstrap.

## Acceptance Criteria

- `cookimport.cli_support.bench_artifacts` owns a local `_coerce_int` helper instead of depending on shared import timing.
- A focused regression test fails on the broken baseline and passes after the fix.
- The targeted benchmark-helper artifact test slice passes in the project `.venv`.

## Verification

- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py::test_bench_artifacts_loads_string_block_indices_via_shared_int_helper -q`
- `. .venv/bin/activate && pytest tests/labelstudio/test_labelstudio_benchmark_helpers_artifacts.py -q`

## Evidence

- Verified before the fix that `_load_extracted_archive_blocks(...)` raised `NameError: name '_coerce_int' is not defined` from `cookimport/cli_support/bench_artifacts.py:642`.
- The focused regression test above was red before this patch and passes after it.

## Constraints / Decision

- Keep scope to the missing helper binding that breaks single-book benchmark artifact reconstruction.
- Fix the dependency at the module boundary by owning the tiny helper locally in `bench_artifacts.py`; importing `bench_compare` here would create a circular import during `cli_support` bootstrap.
