# Fast Suite Runtime Cleanup

## Problem

Routine non-slow pytest runs are still much slower than needed for agent loops. Several domains appear to include files that are effectively integration-heavy even though they are part of the default fast slice.

## Acceptance Criteria

- Remaining non-slow suite hotspots are measured and documented.
- Expensive files that are not needed for routine loops move to `slow`.
- Routing-only tests stop at helper boundaries where dedicated helper coverage already exists.
- Targeted reruns confirm materially lower runtime for the affected non-slow domains and broad fast slices.

## Verification

- `. .venv/bin/activate && pytest tests/<domain> -m 'not slow'`
- `. .venv/bin/activate && pytest tests/<file>.py -m 'not slow'`
- `. .venv/bin/activate && pytest tests/<file>.py -m slow`
- `. .venv/bin/activate && pytest tests -m 'not slow' --maxfail=1`

## Constraints / Decisions

- Keep production/runtime code unchanged unless a test currently has no stable seam to stub.
- Prefer preserving coverage by relocating heavy files to `slow` instead of deleting assertions.
- Keep docs short and focused on the fast-vs-slow policy.
