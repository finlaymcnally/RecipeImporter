# QualitySuite Data Integrity Audit
**Date:** 2026-03-04
**Verification pass updated:** 2026-03-05
**Status:** Revised for correctness

## Executive Summary
The real QualitySuite integrity problem is clear and narrow:

- retained March 4 QualitySuite baseline-style runs were hybrid,
- not deterministic,
- because they used `llm_recipe_pipeline=off` together with `line_role_pipeline=codex-line-role-v1`.

That invalidates those baselines as true no-AI reference points.

The correct interpretation is:

- within-cohort comparisons may still be useful,
- but absolute claims about deterministic baseline quality from those runs are invalid.

## What Is Verified
### 1. Baseline-style QualitySuite runs were hybrid
Retained `experiments_resolved.json` files for March 4 QualitySuite cohorts show baseline experiments with:

- `llm_recipe_pipeline=off`
- `line_role_pipeline=codex-line-role-v1`

This is the central integrity defect.

### 2. The March 4 exhaustive run contains many retained hybrid rows
For the March 4 exhaustive QualitySuite run family:

- `49` retained history rows are hybrid (`recipe off`, `line-role codex on`)
- the associated benchmark reports show `377` variants represented
- only `137` prediction runs executed fresh
- `240` predictions were reused cross-run

So the cohort was contaminated, but not every row triggered a new live call.

### 3. Official single-offline vanilla is a different population
Official single-offline `/vanilla` benchmark construction explicitly forces:

- `llm_recipe_pipeline=off`
- `line_role_pipeline=off`
- `atomic_block_splitter=off`

That means the contaminated QualitySuite baseline population should not be casually conflated with official single-offline vanilla rows.

## What This Invalidates
For the contaminated March 4 QualitySuite cohorts, the following uses are invalid:

- treating baseline accuracy as true deterministic performance
- presenting Codex uplift as uplift from a rules-only baseline
- using those baseline numbers as the absolute floor for later comparison

The baseline there is not "no AI". It is "recipe Codex off, line-role Codex still on".

## What May Still Be Useful
Within one contaminated cohort, many relative comparisons remain useful because the same hidden line-role assistance often applies across the cohort.

Examples of still-usable directional comparisons:

- parser A vs parser B
- preprocess mode X vs Y
- section detector variant comparisons
- other within-cohort setting comparisons that share the same hidden baseline shape

That does not rescue the baseline label, but it does mean the runs are not worthless.

## What Was Too Broad In Earlier Wording
The following claims should be retired:

- "most vanilla benchmark runs were hybrid"
- "current dashboard vanilla metrics are contaminated"
- "all historical vanilla data is suspect"

Those statements go beyond what the retained repo evidence currently proves.

The verified statement is narrower and better:

- **retained March 4 QualitySuite baseline-style cohorts were hybrid and should not be treated as deterministic baselines**

## Practical Fix Reading
If the goal is to fix the underlying problem, this report points to three required outcomes:

1. QualitySuite baseline generation must explicitly disable line-role Codex when the baseline is meant to be deterministic.
2. Hybrid rows should never be labeled or mentally treated as vanilla/no-AI baselines.
3. Any benchmark conclusions already drawn from these contaminated baselines should be re-read as hybrid-baseline conclusions, not deterministic-baseline conclusions.

## Final Verdict
### Verified
- retained March 4 QualitySuite baselines were hybrid
- those baselines are invalid as true deterministic references
- official single-offline vanilla rows are a different benchmark population and should not be accused by association

### Best current wording
- **integrity defect:** real
- **scope:** retained March 4 QualitySuite baseline-style cohorts
- **primary consequence:** baseline interpretation was wrong
