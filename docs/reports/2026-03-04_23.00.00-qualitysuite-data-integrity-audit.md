# QualitySuite Data Integrity Audit: Impact of Invisible AI Defaults
**Date:** 2026-03-04
**Audit Scope:** All QualitySuite and SpeedSuite data prior to 2026-03-04

## 1. Executive Summary
The discovery that `line_role_pipeline` defaults to `codex-line-role-v1` (AI-based) rather than `deterministic-v1` (rules-based) confirms that historical "Vanilla" baselines were assisted by LLMs. This has created a "Data Integrity Gap" where the system's baseline performance was significantly higher than its actual rules-based capability.

## 2. Key Findings

### A. Contaminated Baselines ("Vanilla is not Vanilla")
Most "Vanilla" benchmark runs in the history logs were actually **Hybrid runs**.
*   **The Reality:** While the `llm_recipe_pipeline` was off, the `line_role_pipeline` was active.
*   **The Impact:** Absolute accuracy scores for "Vanilla" mode are artificially inflated. The system appeared to be "smarter" in its rules-based mode than it actually is.

### B. Underreported Uplift
Because the baseline was already using AI, the measured "uplift" of switching to full Codex extraction (e.g., `codex-farm-3pass-v1`) was compressed.
*   **The Math:** You weren't measuring `0% AI vs 100% AI`; you were measuring `~20% AI vs 100% AI`.
*   **Decision Impact:** The perceived ROI of full AI pipelines was likely higher than the data suggested.

### C. Optimization Mirage in Deterministic Sweeps
The massive 143-config tournament on Feb 28th aimed to find optimal "deterministic" parser settings.
*   **The Reality:** The "winning" settings were optimized to work in tandem with the Codex Line-Role model.
*   **The Risk:** If the system is ever run in a "True Vanilla" mode (e.g., local-only with no API access), these "optimal" settings may perform worse than expected because the "safety net" of the AI Line-Role model has been removed.

## 3. Legacy Data Calibration
For future generations reviewing data prior to March 4th, 2026, the following assumptions must be applied:

| Metric | Historical Interpretation | Audited Interpretation |
|--------|---------------------------|-------------------------|
| **Absolute Accuracy** | Raw rules-based performance | Rules-based performance + AI assistance |
| **Uplift (%)** | Full value of AI | Incremental value of AI over Hybrid mode |
| **Vanilla Cost** | $0.00 (Implicitly assumed) | Significant (due to hidden LineRole calls) |

## 4. Current Dashboard Status (March 4th Update)
An exhaustive inspection of `experiments_resolved.json` from the March 4th Exhaustive Batch confirms that **current data remains contaminated.**

*   **Evidence:** The "baseline" experiment in recent runs explicitly inherits `"line_role_pipeline": "codex-line-role-v1"` from the system defaults.
*   **Certainty:** 100%. The "Vanilla" metrics displayed on the dashboard as of today are not rules-based; they are Hybrid-AI metrics.

## 5. Conclusion on Past Decisions
*   **Relative Validity:** Decisions made to promote one setting over another (e.g., "Parser A beats Parser B") remain **mostly valid**, as both likely benefited from the same hidden AI assistance.
*   **Absolute Validity:** Any conclusion regarding "the system is 90% accurate without AI" is **invalid**. The system's true rules-based floor has not yet been accurately measured.
