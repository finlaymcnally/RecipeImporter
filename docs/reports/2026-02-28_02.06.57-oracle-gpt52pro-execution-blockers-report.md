# Oracle gpt-5.2-pro Execution Blockers Report

## Goal

Run Oracle on:

- `docs/reports/2026-02-28_01.51.16-thefoodlab-all-method-hotspot-report.md`

with all mapped code files referenced by that report:

- `cookimport/cli.py`
- `cookimport/bench/eval_canonical_text.py`
- `cookimport/bench/sequence_matcher_select.py`
- `cookimport/bench/dmp_sequence_matcher.py`
- `cookimport/config/run_settings.py`

Model required: `gpt-5.2-pro`.

## What Failed (timeline)

1. Browser mode with default Oracle paths:
   - Error: `EACCES: permission denied, mkdir '/home/mcnal/.oracle/sessions/foodlab-hotspot-review'`
2. Browser mode retried with writable `/tmp` Oracle home/profile:
   - Launch started, then failed with:
     - `ERROR: Chrome window closed before oracle finished. Please keep it open until completion.`
     - `Chrome disconnected before completion; keeping session running for reattach.`
3. API mode with `--engine api --model gpt-5.2-pro`:
   - Error: `Missing OPENAI_API_KEY. Set it via the environment or a .env file.`
4. Browser mode using default profile path with writable Oracle home:
   - Error: `EACCES: permission denied, open '/home/mcnal/.oracle/browser-profile/chrome-out.log'`
5. Manual login helper run did not produce a persistent logged-in session usable by this sandboxed run path (human confirmed: "not logged in").

## Root Causes

1. **Sandbox write boundary mismatch**
   - This Codex run can write in workspace and `/tmp`, not in `~/.oracle`.
   - Oracle defaults for sessions/profile/logs target `~/.oracle`, causing `EACCES`.

2. **No API auth configured**
   - `OPENAI_API_KEY` is not set, so `--engine api` cannot execute.

3. **No confirmed ChatGPT auth state in writable browser profile**
   - Browser engine requires a logged-in ChatGPT session in the same writable profile directory Oracle uses.
   - Current run path did not have that authenticated state.

## Why the Request Is Still Blocked

The run requires either:

- API auth (`OPENAI_API_KEY`) for `--engine api`, or
- browser auth in a writable profile plus writable Oracle session/log paths.

Neither was simultaneously available in this run.

## Recovery Paths

### Path A (recommended in this sandbox): browser with writable paths

1. Perform login into a writable profile:
   - `ORACLE_BROWSER_PROFILE_DIR=/tmp/oracle-browser-profile /home/mcnal/.local/bin/chromium-chatgpt-login`
2. Run Oracle with writable session + same profile:
   - `ORACLE_HOME_DIR=/tmp/oracle-home ORACLE_BROWSER_PROFILE_DIR=/tmp/oracle-browser-profile /home/mcnal/.local/bin/oracle-browser-headless --model gpt-5.2-pro --wait --slug "foodlab hotspot review" -p "<prompt>" --file docs/reports/2026-02-28_01.51.16-thefoodlab-all-method-hotspot-report.md --file cookimport/cli.py --file cookimport/bench/eval_canonical_text.py --file cookimport/bench/sequence_matcher_select.py --file cookimport/bench/dmp_sequence_matcher.py --file cookimport/config/run_settings.py`
