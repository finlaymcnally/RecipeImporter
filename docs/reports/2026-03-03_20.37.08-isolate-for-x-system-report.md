# Isolate For X Report

## 1. System Overview

`Isolate For X` is a dashboard filtering feature that lets you define a slice of benchmark runs using rule-based conditions, then analyze that slice against the baseline dataset.

It lives in the `Previous Runs` area and is integrated with table filtering logic in `cookimport/analytics/dashboard_render.py`.

It is intentionally not a separate filtering subsystem anymore. The isolate panel is a UI for building rule clauses that are converted into the same table filter clauses used by column filter editors.

## 2. Feature Purpose

The core purpose is fast hypothesis testing on benchmark history, for questions like:

- What happens when I isolate `ai_model = X`?
- How do results look when `all_token_use > 500000`?
- Show runs where any of these conditions are true.

This reduces manual scanning and makes configuration/performance tradeoffs easier to spot.

## 3. Design Goals

- Use one filtering engine so isolate and table filters always agree.
- Support boolean combine modes (`AND` and `OR`).
- Support native cross-column OR (not an approximation).
- Support numeric threshold operators on numeric fields.
- Keep rule editing fast and readable in the dashboard UI.
- Persist isolate/filter state across reloads.

## 4. Feature Surface

Each isolate rule has:

- field
- operator
- value

Supported operators:

- Categorical fields: `is`, `is not`
- Numeric fields: `=`, `!=`, `>`, `>=`, `<`, `<=`

Combine mode:

- `all rules (AND)`
- `any rule (OR)`

Value control behavior:

- For categorical fields, value is selected from known values in the current run set.
- For numeric fields, value is entered as a numeric threshold input.

## 5. Filter Semantics

Isolate rules are translated directly into table column filters.

Mapping behavior:

- Isolate operators map to table operators (`eq`, `neq`, `gt`, `gte`, `lt`, `lte`).
- Combine mode maps to global column combine mode:
  - `all` -> `AND`
  - `any` -> `OR`
- Numeric clauses are only considered active when their values parse as numbers.

Evaluation behavior:

- Isolate row matching uses the same operator evaluator as table filters (`evaluatePreviousRunsFilterOperator(...)`) so semantics remain aligned.

Boolean behavior details:

- Per-clause logic is defined by each rule’s operator.
- Cross-rule logic is controlled by isolate combine mode (`all`/`any`).
- Cross-column boolean behavior is native because table filters now support global `AND`/`OR` across column groups.

## 6. Analytical Use Model

`Isolate For X` does two things:

- Selects a filtered slice of rows.
- Shows isolate-vs-baseline comparisons in the isolate insights panel.

Baseline is the currently visible run set before isolate slice reduction.

This means quick filters and any already-applied table constraints define the candidate pool first, then isolate defines the slice within that pool.

## 7. State and Persistence

Isolate state is persisted in dashboard UI state and view presets along with table filters.

This keeps isolate behavior reproducible across page reloads and saved dashboard views.

Persisted isolate-related state includes:

- isolate clauses and combine mode,
- synced table column clauses generated from isolate,
- global column combine mode (`AND`/`OR`).

## 8. Practical Outcome

The current design supports meaningful diagnostic filtering:

- threshold numeric logic,
- true cross-column OR,
- one coherent filter contract across isolate panel, table, and trend visualizations.

## 9. Example Use Cases

- Quality gating slice: `strict_accuracy >= 0.92` AND `macro_f1_excluding_other >= 0.88`.
- Spend slice: `all_token_use > 750000` OR `benchmark_total_seconds > 1200`.
- Variant comparison slice: `ai_model is gpt-5-codex` AND `ai_effort is medium`.

These can be used to quickly answer tradeoff questions in one dashboard pass without exporting data.

## 10. Implementation Pointers

Primary implementation areas in `cookimport/analytics/dashboard_render.py`:

- isolate operator sets and normalization helpers,
- isolate field-type detection and control rendering,
- isolate-to-table sync (`applyIsolateRulesToTableFilters`),
- unified operator evaluation (`evaluatePreviousRunsFilterOperator`),
- previous-runs filter grouping and global mode evaluation.
