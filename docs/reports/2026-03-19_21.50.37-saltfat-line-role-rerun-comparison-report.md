# Salt Fat Acid Heat Line-Role Rerun Comparison Report

This report compares three single-offline Salt Fat Acid Heat benchmark runs:

- baseline comparison run: `data/golden/benchmark-vs-golden/2026-03-19_17.08.33/single-book-benchmark/saltfatacidheatcutdown`
- bad post-change rerun: `data/golden/benchmark-vs-golden/2026-03-19_21.09.56/single-book-benchmark/saltfatacidheatcutdown`
- restart rerun: `data/golden/benchmark-vs-golden/2026-03-19_21.24.41/single-book-benchmark/saltfatacidheatcutdown`

The goal here is narrow:
- determine whether the restart changed benchmark behavior in a meaningful way
- separate "line-role heading contamination improved" from "overall quality recovered"
- record the strongest remaining blockers before another round of line-role work

## High-Level Result

The restart rerun at `21.24.41` is a real improvement over `21.09.56`, but it is not a clean win.

What improved relative to `21.09.56`:
- CodexFarm `strict_accuracy` recovered from `0.594816` to `0.656207`
- Codex-vs-vanilla delta improved from `-0.110505` to `-0.049113`
- changed lines dropped from `328` to `180`
- `HOWTO_SECTION` false positives dropped materially

What did not recover:
- the restart rerun is still slightly worse than the earlier `17.08.33` run (`0.656207` vs `0.663029`)
- macro F1 is still worse than vanilla and much worse than the earlier `17.08.33` CodexFarm run
- recipe-local instruction handling is still weak

Practical interpretation:
- the severe `21.09.56` collapse was not permanent or deterministic; restart changed behavior
- the line-role stage remains unhealthy enough that benchmark quality is still subpar
- the exact heading/prose failure mode improved, but not enough to make the run good

## What Actually Improved

The most important recovery is in the original target failure mode: non-recipe headings getting promoted into `HOWTO_SECTION`.

### `HOWTO_SECTION` pressure

`17.08.33`:
- total predicted `HOWTO_SECTION`: `137`
- gold `KNOWLEDGE -> HOWTO_SECTION`: `61`
- gold `OTHER -> HOWTO_SECTION`: `45`

`21.09.56`:
- total predicted `HOWTO_SECTION`: `139`
- gold `KNOWLEDGE -> HOWTO_SECTION`: `58`
- gold `OTHER -> HOWTO_SECTION`: `41`

`21.24.41`:
- total predicted `HOWTO_SECTION`: `98`
- gold `KNOWLEDGE -> HOWTO_SECTION`: `23`
- gold `OTHER -> HOWTO_SECTION`: `31`

This is the clearest real gain in the restart rerun.

The changed-line sample also supports that story:
- the restart rerun no longer shows as many obvious lesson headings being flipped into `HOWTO_SECTION`
- some former heading errors now fall back to `OTHER` or `KNOWLEDGE` instead

The line-role outside-recipe slice improved versus the bad rerun:
- `21.09.56` outside-recipe accuracy: `0.5721`
- `21.24.41` outside-recipe accuracy: `0.6392`

That outside-recipe recovery is the main reason the restart rerun became usable again.

## What Is Still Bad

The restart rerun still has a broad instruction-line problem.

### Instruction-line collapse

`17.08.33`:
- `INSTRUCTION_LINE` true positives: `54`
- recipe-instructions slice accuracy: `0.8974`

`21.09.56`:
- `INSTRUCTION_LINE` true positives: `20`
- recipe-instructions slice accuracy: `0.3000`

`21.24.41`:
- `INSTRUCTION_LINE` true positives: `18`
- recipe-instructions slice accuracy: `0.3750`

So the restart rerun barely improved the instruction side. It is still far below the earlier `17.08.33` run.

The failure pattern is visible in changed-line samples:
- some actual instructions still fall back to `OTHER`
- some non-recipe prose still gets promoted to `INSTRUCTION_LINE`
- some variant/title-like rows are still being flattened to `OTHER`

Other notable remaining weakness:
- `RECIPE_VARIANT` recall remains poor
- `RECIPE_TITLE` is weaker than the earlier `17.08.33` run
- macro F1 remains clearly below both vanilla and the earlier CodexFarm run

## Span Story

The three runs are not fully equivalent in span projection behavior.

`17.08.33` upload bundle:
- derived `recipe_span_count=0`
- all scored lines were reported outside active recipe spans

`21.09.56` upload bundle:
- `recipe_span_count=33`
- inside-span delta: `-0.091603`
- outside-span delta: `-0.114618`

`21.24.41` upload bundle:
- `recipe_span_count=30`
- inside-span delta: `-0.038278`
- outside-span delta: `-0.050915`

Compared with `21.09.56`, the restart rerun improved both regions:
- inside-span damage got much smaller
- outside-span damage also got much smaller

Compared with `17.08.33`, the restart rerun is harder to compare literally because the earlier bundle had the known active-span accounting mismatch. Directionally, though, `21.24.41` is still not a strong result.

## Runtime / Stage Health

The restart rerun is still not a healthy full-stack run.

Recipe stage:
- `recipe_correction_error=30`
- `build_final_recipe_error=30`
- recipe correction still did not produce usable final-stage outputs

Knowledge stage:
- `changed_blocks=0`
- `decisions_applied=0`
- `validated_shards=0`
- knowledge remained effectively dead and the scored effect stayed `seed_only`

Line-role runtime:
- 5 shards
- all 5 still show watchdog kills
- 1 invalid-output shard remained
- 0 repaired shards in the summary

So the restart rerun should still be read mainly as a line-role outcome with dead downstream rescue. The benchmark is still mostly living or dying on line-role quality.

## Prompt Propagation Check

The restart rerun did not pick up the stronger outside-recipe wording from the latest checked-in prompt assets.

The live worker prompt used in `21.24.41` still contains the older short rules:
- it says not to use `HOWTO_SECTION` for headings like `Cooking Acids` or `Starches`
- but it does not include the stronger rule that outside-recipe rows should default to `KNOWLEDGE` or `OTHER`
- it also does not include the explicit "a single outside-recipe heading is not enough" wording

Why that matters:
- part of the intended line-role fix was prompt-side, not only sanitizer-side
- this rerun therefore does not fully validate the latest prompt changes
- the benchmark result should be interpreted as "restart improved behavior under the older live prompt," not "the full new line-role prompt solved the problem"

## Best Current Interpretation

The strongest coherent reading of the three-run sequence is:

1. `21.09.56` was a broader bad run, not just the original heading problem.
2. Restarting the program changed behavior enough to recover most of that regression.
3. The original heading contamination problem did improve materially in `21.24.41`.
4. The restart rerun is still held back by recipe-local instruction weakness and a still-unhealthy line-role runtime.
5. The checked-in prompt hardening was not fully propagated into the live benchmark prompt, so there is still an unresolved prompt/runtime mismatch.

## Current Priority Queue

If the next round stays strictly on line-role, the highest-value queue is:

1. Fix prompt propagation so the live line-role worker prompt actually includes the stronger outside-recipe defaulting rules.
2. Investigate why recipe-local `INSTRUCTION_LINE` quality is still far below the earlier `17.08.33` run.
3. Keep the new outside-span fail-closed behavior; the `HOWTO_SECTION` counts suggest that seam is helping.
4. Do not spend more effort on knowledge or recipe-stage quality until line-role is stable enough to justify downstream rescue.

## Short Conclusion

`2026-03-19_21.24.41` is a meaningful recovery from `2026-03-19_21.09.56`.

It is not yet a benchmark success:
- the run is still slightly worse than the earlier `17.08.33` accuracy result
- macro F1 remains substantially weaker
- recipe-instruction handling is still poor
- the runtime is still effectively line-role-only because knowledge and recipe rescue remain dead

The most important positive signal is that the non-recipe `HOWTO_SECTION` contamination problem is down sharply in the restart rerun.
The most important unresolved signal is that the intended prompt hardening still does not appear in the live worker prompt used by the benchmark.
