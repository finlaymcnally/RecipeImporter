# Deep Dive: Ingredient-to-Step Linking & Lemmatization Bug Audit

**Date:** 2026-04-10  
**Author:** Gemini CLI Agent  
**Status:** Research Complete / Verified Bugs  
**Domain:** `cookimport/parsing/step_ingredients.py`, `cookimport/staging/draft_v1.py`

## Executive Summary

This report documents a deep-dive investigation into the deterministic ingredient-to-step linking logic. The audit revealed seven significant bugs ranging from low-level lemmatization errors to high-level architectural omissions in the draft conversion pipeline. These bugs collectively cause incorrect ingredient assignments, quantity loss, and broken output mappings in the final recipe drafts.

## 1. Domain Overview: Step-Ingredient Linking

The `assign_ingredient_lines_to_steps` function is the central authority for connecting ingredients to the instructions where they are used. It operates on a two-phase resolution strategy:
1. **Candidate Collection:** Exact, Semantic, and Fuzzy matching across all step-ingredient pairs.
2. **Global Resolution:** A "best-step-wins" approach with special-case exceptions for split-language and section-header groups.

The system relies heavily on deterministic heuristics (verb lookbacks, lemmatization, and alias generation) to maintain high precision without LLM overhead.

---

## 2. Detailed Bug Analysis

### Bug 1: Section Group Over-assignment (Priority: High)
**Description:** The "Section header group" logic (intended to handle "For the Gravy" style clusters) adds ingredients to *every* step that mentions a section alias, bypassing the global resolution and "earliest use" principles.
**Impact:** Ingredients are duplicated across multiple steps. If Step 1 uses the ingredient and Step 2 merely mentions the section name (e.g., "Simmer the gravy"), the ingredients are incorrectly assigned to Step 2 as well.
**Reproduction Proof:**
```python
ingredients = [
    {"raw_ingredient_text": "For the Gravy", "quantity_kind": "section_header"},
    {"raw_ingredient_text": "flour", "quantity_kind": "exact"},
]
steps = ["Whisk flour.", "Simmer the gravy."]
# Result: flour assigned to BOTH steps.
```

### Bug 2: Verb Context 3-Token Lookback Shadowing (Priority: Medium)
**Description:** `_classify_verb_context` (used for Exact/Semantic matches) only scans 3 tokens *before* a match for "use" verbs. If a verb is 4+ tokens away (e.g., "Whisk the flour and the sugar"), the later ingredients (sugar) are marked as `neutral` (score 0.0) instead of `use` (score 10.0).
**Impact:** Inconsistent scoring across ingredients in the same step.
**Architectural Note:** `_collect_fuzzy_candidates` uses a whole-step verb check, but since it only runs for unmatched items, the "shadowing" effect means the more restrictive exact-match scoring always wins.

### Bug 3: Broken Lemmatization for Plurals (Priority: High)
**Description:** The `_lemmatize_token` logic for plurals is incomplete. It handles `-ies` and basic `-s`, but fails on `-oes`, `-ses`, and `-xes`.
**Impact:**
- `potatoes` -> `potatoe` (should be `potato`)
- `tomatoes` -> `tomatoe` (should be `tomato`)
- `glasses` -> `glasse` (should be `glass`)
- `boxes` -> `boxe` (should be `box`)
**Result:** Singular/Plural matching is broken for these common food items.

### Bug 4: Aggressive Consonant Undoubling (Priority: Medium)
**Description:** The `_undouble_consonant` helper, used during `-ed` and `-ing` lemmatization, blindly strips double consonants.
**Impact:**
- `added` -> `ad` (breaks matching with `add` in `_USE_VERBS`)
- `rolling` -> `rol` (breaks matching with `roll` in `_USE_VERBS`)
- `filling` -> `fil`
**Result:** Common "use" verbs are not recognized when they appear in past tense or continuous form.

### Bug 5: Single-Step Split Quantity Loss (Priority: High)
**Description:** If a step contains a split signal (e.g., "half") but is the only step assigned to that ingredient (common when the parser only finds the first mention in a complex line), the `input_qty` is halved, and the remainder is lost.
**Impact:** Direct data loss in the final recipe quantity.
**Reproduction Proof:**
```python
# Step: "Add half the onions now, then the rest later."
# Result: Onions (2.0) -> Assigned to Step 0 with qty 1.0. Total 1.0.
```

### Bug 6: Incomplete `_USE_VERBS` Vocabulary (Priority: Medium)
**Description:** `_USE_VERBS` is the whitelist for priority assignment. It currently misses common preparation verbs.
**Missing Verbs:** `sift`, `grate`, `peel`, `chop`, `dice`, `mince`, `slice`, `carve`, `shred`.
**Result:** These actions are treated as `neutral` (priority 0) rather than `use` (priority 10), leading to incorrect tie-breaks in multi-step recipes.

### Bug 7: `draft_v1.py` Mapping Omission (Priority: Critical)
**Description:** `cookimport/staging/draft_v1.py` derives its `ingredientStepMapping` only from `debug.assignments`. It completely ignores `debug.group_assignments` and `debug.all_ingredients_steps`.
**Impact:**
- "Mix all ingredients" results in **zero** mappings in the final draft.
- Section-header group assignments (Bug 1) are lost in the final output.
**Result:** The most robust "safety net" features of the parser are effectively disconnected from the output pipeline.

## 3. Upstream & Downstream Impact

### Upstream: Importers
Importers (EPUB, PDF) that rely on deterministic linking to validate recipe spans will receive "empty" or "weak" signals from complex steps containing "all ingredients" or "remaining" phrases.

### Downstream: Staging & Drafting
The `AuthoritativeRecipeSemantics` object emitted by `draft_v1.py` carries a broken `ingredientStepMapping`. This means:
1. UI/UX surfaces (Cookbook web app) cannot highlight ingredients correctly.
2. Analytics on ingredient usage/wastage will be skewed.
3. Automated nutrition/scaling that depends on "when" an ingredient is added will be incorrect.

---

## 4. Technical Debt & Recommendations

1. **Unify Verb Classification:** `_collect_all_candidates` should use the same whole-step verb check as fuzzy matching to avoid the 3-token shadowing bug.
2. **Standardize Lemmatization:** Replace the hand-rolled plural/consonant logic with a more robust library (like `spacy` or `nltk` if already present in environment) or at least fix the `-oes`/`-ses` edge cases and whitelist `add`/`roll` from undoubling.
3. **Draft-v1 Sync:** `_mapping_from_assignment_debug` must be updated to merge `assignments`, `group_assignments`, and `all_ingredients_steps`.
4. **Quantity Safety Rail:** Add a final pass to `_build_final_step_lines` that ensures 100% of an ingredient's quantity is accounted for across all assigned steps.

---

## 5. Conclusion

The ingredient-to-step linking system is architecturally sound but suffers from "death by a thousand cuts" in its deterministic implementation. The most critical failure is the disconnect between the parser's special-case logic and the `draft_v1` output mapping, which renders the most advanced assignment features useless in the final product.
