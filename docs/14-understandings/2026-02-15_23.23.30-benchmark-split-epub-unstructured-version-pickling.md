# Benchmark split EPUB pickling failure

Finding:
- `generate_pred_run_artifacts(...)` runs split EPUB/PDF conversion in a `ProcessPoolExecutor` and returns `ConversionResult` payloads back to the parent process.
- One unpickleable value in `ConversionResult` fails the whole job result transfer.

Root cause:
- EPUB unstructured diagnostics metadata used `getattr(unstructured, "__version__", "unknown")`.
- In this runtime, that value can become the module `unstructured.__version__` (not a string), which is not pickleable.

Rule:
- Keep split worker return payloads strictly pickle-safe (`str`, `int`, `float`, `bool`, `None`, `dict`/`list` of primitives).
- Normalize unstructured version metadata to a plain string before attaching it to raw artifact metadata.
