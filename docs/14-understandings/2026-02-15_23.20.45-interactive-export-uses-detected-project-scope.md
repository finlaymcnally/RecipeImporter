# Interactive export scope carry-forward

Discovery:
- The picker already computes per-project detected scope (`manifest task_scope` first, then payload `label_config` inference) for labels like `[type: freeform-spans]`.
- Interactive export previously discarded that detected scope and always prompted `Export scope` afterward.

Decision:
- `_select_export_project(...)` now returns `(project_name, detected_scope)`.
- Interactive `labelstudio_export` uses `detected_scope` directly when valid and only prompts for scope when detection is missing/unknown.
