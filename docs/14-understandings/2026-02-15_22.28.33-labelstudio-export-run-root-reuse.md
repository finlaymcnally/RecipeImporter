# Label Studio export run-root reuse (2026-02-15)

Superseded:
- Current behavior changed at `2026-02-15_22.33.22`; export now creates a fresh timestamped run root by default.
- See `docs/14-understandings/2026-02-15_22.33.22-labelstudio-export-timestamped-project-root.md`.

Discovery:
- `run_labelstudio_export(...)` checks for the latest matching manifest by `project_name` under `output_dir` and sets `run_root` to that manifest parent.
- If `run_root` is already resolved this way, export writes to `<run_root>/exports` and does not create a new timestamp folder.
- A new timestamped run root is only created when no run root can be resolved from `--run-dir` or matching manifest lookup.

Code anchors:
- `cookimport/labelstudio/export.py`: `_find_latest_manifest(...)`, manifest lookup branch, and `if run_root is None` timestamp creation branch.
- `cookimport/cli.py`: interactive export passes `run_dir=None`, so manifest lookup drives run-root selection.

Concrete example observed:
- Project: `SFAHpipe1`
- Manifest run timestamp: `2026-02-14T01:58:47`
- Export output path: `data/golden/2026-02-14_01.58.47/labelstudio/saltfatacidheatcutdown/exports/summary.json`
- Export files had newer write times than the manifest, confirming refresh-in-place behavior.
