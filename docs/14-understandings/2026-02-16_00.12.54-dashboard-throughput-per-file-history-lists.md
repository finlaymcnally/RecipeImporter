# Throughput file-history rendering rule

Discovery:
- Throughput already carries `stage_records[*].file_name` and `run_timestamp`, so renderer-side grouping can produce per-file history without collector/schema changes.

Current rule:
- Keep run/date trend + recent-runs table for overall throughput.
- Render `Per-File History Lists` under throughput, with one file-named table per group.
- Order rows newest-first per file via parsed timestamp comparison.
