# Benchmark dashboard metadata flow

Discovery:
- `labelstudio-benchmark` stores enrichment metadata (`manifest.json`, `coverage.json`) under `<eval_dir>/prediction-run/`, not only at eval-root.
- The dashboard collector previously read only eval-root enrichment files, so benchmark context fields were often missing.
- CSV and JSON benchmark records can represent the same eval run with different timestamps, so dedupe by `(run_timestamp, artifact_dir)` can create duplicates.

Current rule:
- Collector should probe both eval-root and `prediction-run/` for manifest/coverage enrichment.
- CSV + JSON benchmark records should be merged by eval artifact directory, filling missing fields rather than appending duplicate rows.
