# Interactive generate-dashboard feedback

Finding:
- In `_interactive_mode`, selecting `generate_dashboard` already called `stats_dashboard(...)` and wrote `index.html` to `<output_root>/.history/dashboard`.
- The flow immediately returned to the main menu, which made the action feel like it did nothing for some users.

Current rule:
- Interactive `generate_dashboard` now asks whether to open the dashboard in the browser after generation.
- The response is passed through `open_browser` when calling `stats_dashboard(...)`.
