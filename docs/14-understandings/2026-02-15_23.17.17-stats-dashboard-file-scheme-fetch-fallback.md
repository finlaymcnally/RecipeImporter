# Stats dashboard file:// fetch fallback

Finding:
- `dashboard.js` originally fetched `assets/dashboard_data.json` unconditionally.
- Some browsers block local `fetch()` for `file://` pages, causing `TypeError: NetworkError when attempting to fetch resource`.

Current rule:
- `render_dashboard` still writes `assets/dashboard_data.json`, but also embeds the same JSON inline in `index.html` via `<script type="application/json">`.
- `dashboard.js` now tries inline JSON first and only falls back to `fetch()` when inline data is missing or invalid.
