# Direct-call defaults in labelstudio-eval

Finding:
- `_interactive_mode` calls `labelstudio_eval(...)` directly.
- `labelstudio_eval` previously declared defaults as `typer.Option(...)` on `overlap_threshold` and `force_source_match`, so direct calls received `OptionInfo` objects instead of `0.5` and `False`.

Fix:
- Declare those parameters with `typing.Annotated[..., typer.Option(...)]` plus real Python defaults (`= 0.5`, `= False`).
- Add regression coverage in `tests/test_labelstudio_benchmark_helpers.py::test_labelstudio_eval_direct_call_uses_real_defaults`.

Implication:
- Interactive eval-only now passes primitive defaults into `evaluate_predicted_vs_freeform`, so threshold comparisons no longer crash.
