# Interactive benchmark upload credential resolution

Finding:
- `_interactive_mode` benchmark upload confirmation previously called `labelstudio_benchmark(...)` without passing `label_studio_url`/`label_studio_api_key`.
- `labelstudio_benchmark` then used `_resolve_labelstudio_settings(...)`, which only checks CLI args and env vars, so missing env caused an immediate exit.

Current rule:
- Interactive benchmark upload now resolves credentials through `_resolve_interactive_labelstudio_settings(settings)` first (env -> saved settings -> prompt).
- The resolved URL/API key are passed into `labelstudio_benchmark(...)`, matching interactive import/export behavior.
