# Stage run-config flow rule

Discovery:
- Stage commands already attach `runConfig` to `ConversionReport` JSON.
- Dashboard stage rows usually come from `performance_history.csv`, and that path previously dropped `runConfig`.

Current rule:
- `perf_report` writes stage `runConfig` to CSV as `run_config_json`.
- `dashboard_collect` parses `run_config_json` into `StageRecord.run_config` (and keeps report-scan fallback support).
- `dashboard_render` uses the same run-config summary formatter for stage/import and benchmark tables.
