# Benchmark scoring contract vs stage output contract

Finding:
- `cookimport stage` writes cookbook outputs but does not produce scoring-ready prediction range artifacts.
- Benchmark scoring paths (`labelstudio-benchmark`, `bench run`, interactive eval-only) load predictions from `label_studio_tasks.jsonl` and compare against `freeform_span_labels.jsonl`.

Why this exists:
- Freeform gold annotations are span/block-index based.
- Prediction tasks are generated in the same coordinate space, so matching can be deterministic.

Practical implication:
- Benchmark may also write processed stage-like outputs for review, but those are not the primary scoring input.
