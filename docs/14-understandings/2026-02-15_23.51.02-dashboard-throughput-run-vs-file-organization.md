# Throughput organization rule

Discovery:
- Stage throughput data already contains both `run_timestamp` and `file_name` on each `StageRecord`.
- This supports two orthogonal views without schema changes:
  - run/date timeline across all records
  - per-file trend grouped by `file_name`

Current rule:
- Keep both views in the throughput section.
- Use newest-first ordering for run/date tables, and chronological ordering for trend charts.
