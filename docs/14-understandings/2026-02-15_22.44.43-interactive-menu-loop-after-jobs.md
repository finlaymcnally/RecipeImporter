# Interactive menu loop after jobs (2026-02-15)

Finding:
- `_interactive_mode()` in `cookimport/cli.py` had `break` statements at the end of successful `import`, `labelstudio`, `labelstudio_export`, and `labelstudio_benchmark` paths.
- Those `break` statements exited the `while True` loop, so the CLI session ended immediately after one successful job.

Current rule:
- Those action branches now `continue` back to the main menu.
- Interactive mode exits only when the main menu action is `exit` (or `None`).
