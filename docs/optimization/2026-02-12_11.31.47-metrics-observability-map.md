# Metrics Observability Map

Code verification showed three separate metric surfaces:
- Per-file conversion metrics in `<run_root>/<workbook_slug>.excel_import_report.json` (`ConversionReport`), including counts, warnings/errors, confidence aggregates, timing, and `outputStats`.
- Cross-run performance metrics in `data/output/.history/performance_history.csv`, generated from per-file reports by `cookimport/analytics/perf_report.py`.
- Label Studio metrics in run artifacts (`coverage.json`, `manifest.json`, `eval_report.json`, `eval_report.md`) for task coverage and canonical/freeform benchmark quality.

Notable detail: split PDF/EPUB merge reports aggregate timing from child jobs and add `timing.checkpoints.merge_seconds` to capture merge overhead.
